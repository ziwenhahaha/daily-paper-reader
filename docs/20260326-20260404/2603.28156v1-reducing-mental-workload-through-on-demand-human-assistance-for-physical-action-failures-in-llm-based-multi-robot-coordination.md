## 摘要
基于大语言模型（LLM）的多机器人协作引起了越来越多的关注，因为 LLM 能够通过分解任务和生成高层规划，将自然语言指令直接转化为机器人动作方案。然而，从物理执行失败中恢复仍然具有挑战性，任务往往会因为重复相同的失败动作而陷入停滞。尽管已经提出了使用混合现实（Mixed Reality）进行机器人远程操作的框架，但针对多机器人环境中物理失败的远程错误解决尝试仍然较少。在本研究中，我们提出了 REPAIR（Robot Execution with Planned And Interactive Recovery），这是一种将远程错误解决集成到基于 LLM 的多机器人规划中的人机回环（human-in-the-loop）框架。在该方法中，机器人自主执行任务；然而，当发生不可恢复的失败时，LLM 会向操作员请求协助，通过远程干预实现任务的连续性。在真实环境中使用多机器人垃圾清理任务进行的评估证实，与全自主方法相比，REPAIR 显著提高了任务进度（在时限内清理的物品数量）。此外，对于易于收集的物品，它实现了与完全远程控制相当的任务进度。结果还表明，操作员的心理负荷在身体需求和努力程度方面可能存在差异。项目网站为 https://emergentsystemlabstudent.github.io/REPAIR/。

## Abstract
Multi-robot coordination based on large language models (LLMs) has attracted growing attention, since LLMs enable the direct translation of natural language instructions into robot action plans by decomposing tasks and generating high-level plans. However, recovering from physical execution failures remains difficult, and tasks often stagnate due to the repetition of the same unsuccessful actions. While frameworks for remote robot operation using Mixed Reality were proposed, there have been few attempts to implement remote error resolution specifically for physical failures in multi-robot environments. In this study, we propose REPAIR (Robot Execution with Planned And Interactive Recovery), a human-in-the-loop framework that integrates remote error resolution into LLM-based multi-robot planning. In this method, robots execute tasks autonomously; however, when an irrecoverable failure occurs, the LLM requests assistance from an operator, enabling task continuity through remote intervention. Evaluations using a multi-robot trash collection task in a real-world environment confirmed that REPAIR significantly improves task progress (the number of items cleared within a time limit) compared to fully autonomous methods. Furthermore, for easily collectable items, it achieved task progress equivalent to full remote control. The results also suggested that the mental workload on the operator may differ in terms of physical demand and effort. The project website is https://emergentsystemlabstudent.github.io/REPAIR/.

### 论文总结：通过基于 LLM 的多机器人协作中针对物理动作失败的按需人工协助来减轻心理负荷

#### 1. 核心问题与研究动机
*   **核心问题**：基于大语言模型（LLM）的多机器人系统虽然擅长高层任务规划，但在面对**物理执行失败**（如抓取落空、目标丢失）时，往往会陷入无效的重复动作循环，导致任务停滞。
*   **研究动机**：
    *   **全自主系统的局限**：目前的 LLM 规划器难以从复杂的物理失败中自我恢复。
    *   **全远程操作的负担**：随着机器人数量增加，全程人工干预会给操作员带来极大的心理和体力负担。
    *   **研究空白**：缺乏一种针对多机器人协作中物理失败的、高效且低负荷的“按需”人工干预机制。

#### 2. 方法论（REPAIR 框架）
*   **核心思想**：提出 **REPAIR**（Robot Execution with Planned And Interactive Recovery）框架，将远程错误修复作为一种特殊的“技能”集成到 LLM 的规划循环中。
*   **关键流程**：
    1.  **任务分解与分配**：LLM 接收自然语言指令（如“清理房间”），根据机器人能力（移动机器人 Kachaka 和移动协作机器人 Fetch）分解并分配子任务序列。
    2.  **动作规划与执行**：机器人执行 `navigation`（导航）、`detection`（检测）、`pick`（抓取）等技能。
    3.  **按需请求协助（Help Skill）**：当物理动作连续失败或无法继续时，LLM 调用 `help` 技能，生成描述当前困境的请求消息并发送给操作员。
    4.  **远程错误修复**：操作员通过 VR/MR 界面接管机器人完成关键动作，并向 LLM 提供反馈。
    5.  **恢复规划**：LLM 根据人工反馈更新环境状态认知，重新生成后续规划并恢复自主运行。

#### 3. 实验设计
*   **实验场景**：真实室内环境（包含餐厅、客厅、工作区等 4 个区域），执行多机器人垃圾清理任务。
*   **实验对象**：12 名受试者（20-30 岁）。
*   **对比方法（Benchmarks）**：
    *   **Teleop（全远程操作）**：操作员全程手动控制两台机器人。
    *   **Auto（全自主）**：LLM 尝试自主规划和错误恢复，不请求人工帮助。
    *   **REPAIR（本文方法）**：自主规划为主，必要时请求人工干预。
*   **评估指标**：
    *   **任务进度**：规定时间内（25 分钟）收集的垃圾数量。
    *   **心理负荷**：使用 NASA-TLX 量表（包含心理需求、体力需求、努力程度等 6 个维度）。

#### 4. 资源与算力
*   **硬件环境**：
    *   **计算节点**：Intel Core i9-12900H CPU, NVIDIA GeForce RTX 3080 Ti GPU (12 GB)。
    *   **机器人**：Fetch（移动机械臂）、Kachaka（移动底座）。
    *   **交互设备**：Meta Quest 3（用于 Fetch 远程操作）、智能手机（用于 Kachaka 远程操作）。
*   **软件与模型**：
    *   **LLM**：OpenAI **o3-2025-0416**（用于规划和状态识别）。
    *   **视觉模型**：Detic（目标检测）、Vision-Language Model（用于判断抓取成功）。
    *   **系统架构**：Ubuntu 22.04, ROS Noetic, Docker。

#### 5. 实验数量与充分性
*   **实验规模**：12 名受试者分别在三种条件下完成任务，共计 36 组完整实验流程。
*   **充分性评价**：
    *   **统计严谨性**：采用了 Friedman 检验和 Wilcoxon 符号秩检验（配合 Holm 校正），确保了结果的统计显著性。
    *   **难度分级**：将垃圾分为 Level 1（易收集）和 Level 2（难收集，如易倒的瓶子、形状不规则的纸屑），测试了系统在不同难度下的鲁棒性。
    *   **客观性**：实验涵盖了真实物理环境中的不确定性，而非仅在仿真中进行。

#### 6. 主要结论与发现
*   **任务进度显著提升**：REPAIR 的任务完成度显著优于 Auto 方法（Auto 在物理失败后基本陷入僵局，进度接近 0）。
*   **简单任务媲美全人工**：在处理 Level 1 物品时，REPAIR 的效率与全远程操作（Teleop）相当。
*   **心理负荷优化**：虽然总分差异未达显著统计学意义，但在**体力需求（Physical Demand）**和**努力程度（Effort）**维度上，REPAIR 表现出低于 Teleop 的趋势（中等效应量），说明按需干预减轻了操作员的负担。

#### 7. 优点与亮点
*   **人机回环的深度集成**：将“寻求帮助”视为一种高层逻辑技能，使 LLM 能够像人类一样在遇到困难时主动沟通。
*   **真实环境验证**：在复杂的真实室内场景中验证了多机器人协作，具有较强的实用参考价值。
*   **多模态反馈**：操作员修复错误后向 LLM 提供文本反馈，实现了知识的闭环更新。

#### 8. 不足与局限
*   **操作界面复杂性**：针对不同机器人使用不同设备（VR vs 手机），增加了操作员的学习成本和切换负担。
*   **MR 影响未分离**：实验未能单独区分“混合现实界面本身”与“远程操作模式”对心理负荷的具体贡献。
*   **判定误差风险**：未深入探讨机器人“误报失败”或“漏报失败”（即该求助时没求助）对系统性能的影响。
*   **样本量限制**：12 名受试者虽然符合此类研究标准，但更大规模的样本可能有助于进一步明确 NASA-TLX 各维度的显著性差异。
