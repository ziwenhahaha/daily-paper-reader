## 摘要
大型基础模型为自主系统提供了强大的推理能力，但将语义意图映射到可靠的实时控制仍然具有挑战性。现有方法要么 (i) 让大语言模型 (LLM) 直接生成轨迹——这种方式脆弱、难以验证且容易产生延迟；要么 (ii) 在线调整模型预测控制 (MPC) 的目标——这混合了慢速深思熟虑与快速控制，模糊了接口界限。我们提出了智能体快慢规划 (Agentic Fast-Slow Planning)，这是一个在自然时间尺度上解耦感知、推理、规划和控制的分层框架。该框架包含两个桥梁：Perception2Decision 使用车载视觉语言模型 (VLM) 检测器将场景压缩为以自我为中心的拓扑结构，然后利用云端 LLM 决策器将其映射为符号化驾驶指令，在保持可解释性的同时减少了带宽和延迟；Decision2Trajectory 将指令转换为可执行路径，其中语义引导的 A* (Semantic-Guided A*) 将源自语言的软成本嵌入到经典搜索中，使解偏向于可行轨迹，而智能体细化模块 (Agentic Refinement Module) 则利用反馈和记忆来调整规划器的超参数。最后，MPC 实时跟踪轨迹，并针对困难情况提供可选的云端引导参考。在 CARLA 中的实验表明，与纯 MPC 和 A* 引导的 MPC 基准相比，智能体快慢规划提高了扰动下的鲁棒性，横向偏差减少了高达 45%，完成时间缩短了 12% 以上。代码可在 https://github.com/cjychenjiayi/icra2026_AFSP 获取。

## Abstract
Large foundation models enable powerful reasoning for autonomous systems, but mapping semantic intent to reliable real-time control remains challenging. Existing approaches either (i) let Large Language Models (LLMs) generate trajectories directly - brittle, hard to verify, and latency-prone - or (ii) adjust Model Predictive Control (MPC) objectives online - mixing slow deliberation with fast control and blurring interfaces. We propose Agentic Fast-Slow Planning, a hierarchical framework that decouples perception, reasoning, planning, and control across natural timescales. The framework contains two bridges. Perception2Decision compresses scenes into ego-centric topologies using an on-vehicle Vision-Language Model (VLM) detector, then maps them to symbolic driving directives in the cloud with an LLM decision maker - reducing bandwidth and delay while preserving interpretability. Decision2Trajectory converts directives into executable paths: Semantic-Guided A* embeds language-derived soft costs into classical search to bias solutions toward feasible trajectories, while an Agentic Refinement Module adapts planner hyperparameters using feedback and memory. Finally, MPC tracks the trajectories in real time, with optional cloud-guided references for difficult cases. Experiments in CARLA show that Agentic Fast-Slow Planning improves robustness under perturbations, reducing lateral deviation by up to 45% and completion time by over 12% compared to pure MPC and an A*-guided MPC baseline. Code is available at https://github.com/cjychenjiayi/icra2026_AFSP.

### 论文总结：通过智能体快慢规划连接大模型推理与实时控制

#### 1. 核心问题与整体含义（研究动机和背景）
论文针对自动驾驶领域中**大模型推理与实时控制之间的断层**问题展开研究。虽然大语言模型（LLM）具备强大的语义推理和常识判断能力，但在实际应用中面临三大挑战：
*   **脆弱性与不可验证性**：直接由 LLM 生成的轨迹缺乏物理可行性保证。
*   **高延迟**：大模型的推理速度远慢于车辆实时控制（通常需 10Hz 以上）的要求。
*   **接口模糊**：现有方法往往将慢速决策与快速控制混合，导致系统架构混乱。
**研究动机**是借鉴人类认知的“双系统理论”（快慢决策），提出一种分层解耦框架，使 LLM 负责高层语义决策，而经典算法负责底层物理执行。

#### 2. 方法论：核心思想与关键技术
论文提出了 **Agentic Fast-Slow Planning (AFSP)** 框架，核心思想是**“尺度感知解耦”**，将系统分为感知、推理、规划和控制四个层级，并通过两个关键“桥梁”连接：

*   **Perception2Decision（感知到决策）**：
    *   **车载端（边缘）**：使用轻量化视觉语言模型（VLM）作为拓扑检测器，将原始图像压缩为以自我为中心的**拓扑图（Topology Graph）**，包含物体类别、距离、方位等语义信息。
    *   **云端**：LLM 接收压缩后的拓扑文本，通过思维链（CoT）输出符号化驾驶指令（如 `LEFT`, `RIGHT`, `KEEP`）和驾驶风格。
*   **Decision2Trajectory（决策到轨迹）**：
    *   **语义引导 A* (Semantic-Guided A*)**：在经典 A* 算法的代价函数 $f(s) = g(s) + h(s)$ 中引入**语义软成本（Soft Costs）**。根据指令执行状态（正确、延迟、错误、过度执行）分配不同的惩罚值，引导搜索算法生成符合 LLM 意图且物理可行的路径。
    *   **智能体细化模块 (Agentic Refinement Module)**：利用 LLM 作为智能体，根据规划反馈和云端记忆库自动调整 A* 算法的超参数（如各项成本权重），实现闭环自适应调优。
*   **实时控制**：采用 **Cloud-Guided MPC**，在常规情况下跟踪本地参考路径，在复杂场景下切换至云端引导的全局参考路径。

#### 3. 实验设计
*   **实验场景**：基于 **CARLA** 仿真器，涵盖 Town2-7 和 Town10HD 等多个城镇地图。

*   **Benchmark 与对比方法**：
    *   **Baseline 1**：纯 MPC（仅依赖局部避障）。
    *   **Baseline 2**：A*-MPC（经典 A* 引导的 MPC）。
    *   **评估指标**：任务完成时间（Finish Time）、轨迹长度、平均/最大横向偏差（Lateral Deviation）、速度波动。

*   **推理端**：云端 LLM 使用了 DeepSeek-V3 接口，车载端 VLM 采用轻量化部署。

#### 5. 实验数量与充分性
论文进行了多维度的实验验证，整体较为充分且客观：
*   **感知实验**：对比了三种不同的 VLM 微调策略，验证了“两阶段微调”在 IoU 和类别准确率上的优越性。
*   **决策一致性实验**：对比了 VLM 直接决策与“VLM检测+LLM决策”的得分与延迟，证明了后者在保持高准确度的同时显著降低了延迟（从 10.24s 降至 4.13s）。
*   **鲁棒性实验**：在地图偏移（Shift）和扰动下测试 A* 算法，验证了语义引导能有效纠正空间偏差。

#### 6. 主要结论与发现
*   **性能提升**：AFSP 框架在所有场景下均表现最优。相比纯 MPC，**最大横向偏差降低了约 45%**，任务完成时间缩短了 **12% 以上**。
*   **鲁棒性增强**：语义引导的 A* 能够处理地图偏移和感知不确定性，避免了经典 A* 在扰动下的失效。
*   **效率优化**：通过拓扑图压缩，大幅减少了边缘与云端之间的通信带宽需求，同时 LLM 的推理效率优于直接处理图像的 VLM。

#### 7. 优点（亮点）
*   **架构优雅**：成功实现了慢速语义推理与快速物理控制的解耦，解决了 LLM 难以直接用于实时控制的痛点。
*   **软约束设计**：将 LLM 指令作为 A* 的“软成本”而非“硬约束”，既保留了 LLM 的意图，又确保了在指令错误时系统仍能通过几何搜索找到安全路径。
*   **参数自进化**：引入 Agentic Refinement 模块，模仿人类专家调参过程，提升了系统在长尾场景下的自适应能力。

#### 8. 不足与局限
*   **云端依赖性**：虽然优化了带宽，但系统仍高度依赖云端连接。在极端通信中断（断连）情况下的降级处理机制讨论较少。
*   **场景复杂度限制**：实验主要在 CARLA 仿真环境中完成，尚未在真实世界的复杂交通流（如高密度人车混行）中进行验证。
*   **实时性瓶颈**：尽管 LLM 决策时间降至 4 秒左右，但对于高速行驶或瞬息万变的紧急避障场景，该频率可能仍显不足，需进一步研究更高效的异步更新策略。
