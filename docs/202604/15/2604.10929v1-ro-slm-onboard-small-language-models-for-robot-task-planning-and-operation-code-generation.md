## 摘要
大语言模型（LLMs）的最新进展为机器人提供了理解人类指令的上下文推理能力。然而，目前支持 LLM 的机器人通常依赖于云端模型或高性能计算基础设施，这限制了它们在网络环境不可靠或计算资源受限的机器人（如无人机和小型地面车辆）上的部署。因此，部署支持机载运行的微调小语言模型（SLMs）提供了一个极具前景的替代方案。本文介绍了 Ro-SLM，这是一个通过蒸馏 LLMs 的知识和推理能力来实现可靠的 SLM 驱动机器人操作的框架。Ro-SLM 从数据集合成开始，利用 LLMs 生成多样化的任务指令，在极少人工辅助的情况下生成相应的标准答案代码，并将指令增强到现实世界的应用场景中。随后，Ro-SLM 使用该数据集进行微调，其中 LLM 作为奖励函数来指导训练。在无人机操作任务上的广泛实验表明，Ro-SLM 将 SLM 的性能从无法支持机器人任务规划和代码生成提升到了接近 LLM 的水平。

## Abstract
Recent advances in large language models (LLMs) provide robots with contextual reasoning abilities to comprehend human instructions. Yet, current LLM-enabled robots typically depend on cloud-based models or high-performance computing infrastructure, which limit their deployment on robots under unreliable internet environments or with constrained computational resources, such as UAVs and small ground vehicles. Thus, deploying fine-tuned small language models (SLMs) that support onboard deployment offers a promising alternative. This paper introduces Ro-SLM, a framework that enables reliable SLM-driven robot operation by distilling LLMs' knowledge and reasoning. Ro-SLM starts from dataset synthesis by leveraging LLMs to generate diverse task instructions, produce corresponding ground truth code with minimal human assistance, and augment instructions into real-world application scenarios. Ro-SLM is then fine-tuned with the dataset, in which LLM serves as a reward function to guide the training. Extensive experiments on UAV operation tasks demonstrate that Ro-SLM improves the performance of SLM from being incapable of supporting robotic task planning and code generation to achieving performance that approaches LLM.

### 2. 方法论
Ro-SLM 框架分为两个主要阶段：

#### 第一阶段：数据集合成（Dataset Synthesis）
1.  **任务指令生成**：利用 LLM 代理（如 GPT-5.1）根据预设的机器人操作策略和演示，自动生成不同复杂度的任务指令（从单步动作到多步复杂推理）。
2.  **标准答案代码生成**：采用纠错式代码生成框架。对于简单任务由 LLM 自动生成；对于高复杂度任务（5步以上动作），引入少量人工干预以确保代码的绝对正确性。
3.  **场景增强（Augmentation）**：利用 LLM 的常识知识，将简洁的指令（如“向上飞5米”）改写为更符合人类习惯的现实应用场景描述（如“检查仓库通风口，向上爬升5米”），提高模型的鲁棒性。
4.  **数据预处理**：移除代码中的所有注释，防止 SLM 过度拟合语言模式而非逻辑结构。

#### 第二阶段：SLM 微调（Fine-tune）
1.  **监督微调（SFT）**：使用合成的数据集对 SLM（如 Llama-3.1-8B）进行初步训练，并采用 **LoRA** 技术提高训练效率。
2.  **基于 LLM 反馈的优化（GRPO）**：引入组相对策略优化（GRPO）算法。此时 LLM 充当**奖励函数**，它不比较文本相似度，而是解释 SLM 生成的代码逻辑，判断其执行效果是否与标准答案在行为上等价，从而指导模型进一步优化。

### 3. 实验设计
*   **实验场景**：主要在 AirSim 模拟器环境下的无人机（UAV）操作任务中进行。

    *   **Advanced 训练集**：20 个长程任务（包含低、中、高、极高四个难度等级）。
    *   **增强数据集**：将上述任务映射到现实场景后的扩展版本。
*   **Benchmark 与对比方法**：

    *   **原始 Llama-3.1-8B**：未经过 Ro-SLM 微调的原始小模型。
*   **评估指标**：成功率（Success Rate, SR）和完成度（Completeness）。

### 4. 资源与算力

### 5. 实验数量与充分性

### 6. 主要结论与发现
*   **性能飞跃**：原始 8B 模型几乎无法完成任务，而经过 Ro-SLM 微调后，在基础任务上达到 **97.7%** 的成功率，在复杂任务上达到 **70%** 的成功率。
*   **接近 LLM**：Ro-SLM 的表现非常接近基于 o4-mini 的云端方案，甚至在“极高难度”任务中，由于学习了高质量的人工校对数据，其成功率超过了纯 LLM 方案。
*   **鲁棒性**：通过场景增强，模型对人类自然语言指令的理解能力显著增强。
*   **跨平台潜力**：在地面车辆上的成功应用证明该框架具有良好的迁移能力。

### 7. 优点与亮点
*   **行为级奖励函数**：不以词汇重合度（如 BLEU）为准，而是利用 LLM 理解代码行为来给出奖励，解决了代码生成评价难的问题。
*   **数据质量控制**：结合了 LLM 自动生成与关键环节的人工辅助，确保了微调数据的“高质量”和“多样性”。
*   **机载友好**：针对资源受限场景设计，证明了 8B 规模的模型在特定领域可以替代昂贵的云端 LLM。

### 8. 不足与局限
*   **显存门槛**：虽然是 SLM，但 8B 模型仍需约 16GB 显存，对于更小型的嵌入式设备（如 1B/3B 需求场景）尚未验证。
*   **人工依赖**：构建复杂任务的 Ground Truth 仍需领域专家参与，限制了其向全新领域快速扩张的规模化能力。
*   **单次推理限制**：Ro-SLM 采用单次生成模式，缺乏像 Corrective GSCE 那样的多轮迭代纠错机制，在处理极复杂逻辑时容错率较低。
*   **实机验证不足**：实验主要在模拟器中完成，真实物理环境中的传感器噪声、动力学误差等因素尚未充分讨论。
