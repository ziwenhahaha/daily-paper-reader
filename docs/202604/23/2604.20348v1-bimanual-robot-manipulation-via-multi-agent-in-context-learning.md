## 摘要
语言模型（LLMs）已成为具身控制的强大推理引擎。特别是，上下文学习（ICL）使现成的纯文本 LLM 能够在不进行任何特定任务训练的情况下预测机器人动作，同时保留其泛化能力。将 ICL 应用于双臂操作仍然具有挑战性，因为高维联合动作空间和紧密的手臂间协调约束会迅速超出标准上下文窗口的负荷。为解决这一问题，我们引入了 BiCICLe（双臂协调上下文学习），这是第一个使标准 LLM 能够在无需微调的情况下执行少样本双臂操作的框架。BiCICLe 将双臂控制建模为一个多智能体主从（leader-follower）问题，将动作空间解耦为顺序的、有条件的单臂预测。这自然地扩展到了“手臂辩论”（Arms' Debate）——一个迭代细化过程，并引入了第三个“LLM 作为裁判”（LLM-as-Judge）来评估并选择最合理的协调轨迹。在 TWIN 基准测试的 13 个任务上进行的评估显示，BiCICLe 实现了高达 71.1% 的平均成功率，比表现最好的免训练基准高出 6.7 个百分点，并超过了大多数监督学习方法。我们进一步展示了在多项新任务上强大的少样本泛化能力。

## Abstract
Language Models (LLMs) have emerged as powerful reasoning engines for embodied control. In particular, In-Context Learning (ICL) enables off-the-shelf, text-only LLMs to predict robot actions without any task-specific training while preserving their generalization capabilities. Applying ICL to bimanual manipulation remains challenging, as the high-dimensional joint action space and tight inter-arm coordination constraints rapidly overwhelm standard context windows. To address this, we introduce BiCICLe (Bimanual Coordinated In-Context Learning), the first framework that enables standard LLMs to perform few-shot bimanual manipulation without fine-tuning. BiCICLe frames bimanual control as a multi-agent leader-follower problem, decoupling the action space into sequential, conditioned single-arm predictions. This naturally extends to Arms' Debate, an iterative refinement process, and to the introduction of a third LLM-as-Judge to evaluate and select the most plausible coordinated trajectories. Evaluated on 13 tasks from the TWIN benchmark, BiCICLe achieves up to 71.1% average success rate, outperforming the best training-free baseline by 6.7 percentage points and surpassing most supervised methods. We further demonstrate strong few-shot generalization on novel tasks.

#### 1. 核心问题与整体含义
*   **研究动机**：双臂机器人操作（如拧瓶盖、抬托盘）比单臂控制复杂得多，主要挑战在于**高维动作空间**（双臂联合空间）和**紧密的时间/空间协调约束**。
*   **核心问题**：现有的上下文学习（In-Context Learning, ICL）方法在处理双臂任务时，若将双臂动作简单拼接，会迅速耗尽 LLM 的上下文窗口并导致推理逻辑混乱；若让双臂独立预测，则缺乏必要的协调。
*   **整体含义**：本文提出了 **BiCICLe** 框架，旨在不进行任何神经网络微调（Training-free）的前提下，利用现成的 LLM 通过多智能体协作实现复杂的双臂协调操作。

#### 2. 方法论
BiCICLe 的核心思想是将双臂控制解耦为“主-从”多智能体问题，并通过迭代和评估机制优化轨迹。
*   **主从分解 (Leader-Follower)**：
    *   **阶段1（主）**：指定一只手臂为 Leader，LLM 根据环境观察预测其完整的单臂轨迹（$Z^7$ 空间）。
    *   **阶段2（从）**：Follower 手臂在预测时，其输入不仅包含环境观察，还包含 Leader 已规划好的轨迹。这种顺序依赖强制实现了手臂间的协调。
*   **手臂辩论 (Arms’ Debate)**：一种迭代优化策略。在主从预测后，进行第二轮反向调节：Leader 根据 Follower 的初版计划重新修正，随后 Follower 再根据修正后的 Leader 计划进行最终预测。
*   **LLM 裁判 (Best-of-N)**：利用 LLM 的随机性生成 $N$ 组候选轨迹，引入第三个 LLM 充当裁判，根据碰撞风险、目标匹配度、夹持逻辑和工作空间可达性对轨迹评分，选出最优解。
*   **表示方法**：将环境中的物体位置离散化为 3D 体素坐标（Text-based），动作也离散化为体素索引和欧拉角，通过纯文本形式输入 LLM。

#### 3. 实验设计
*   **数据集/场景**：主要在 **TWIN 基准测试**（包含 13 个双臂操作任务，如推箱子、移交物品、抬托盘、开抽屉等）中进行仿真实验。
*   **对比方法**：
    *   **免训练基准**：RoboPrompt (单智能体/双智能体版本)、KAT、VLM-LF。
    *   **监督学习方法**：ACT、PerAct 2、$\pi_0$-keypose、AnyBimanual、3DFA（当前 SOTA）。
*   **泛化性测试**：设计了两个 TWIN 之外的新任务（关罐子、从箱中取物）测试少样本泛化能力。
*   **真实世界实验**：在物理 Franka Panda 双臂机器人上验证了“抬箱子”和“开锅盖”任务。

#### 4. 资源与算力
*   **模型后端**：主要使用 **GPT-5-mini** API 进行推理。
*   **消融实验后端**：使用了开源模型 **Qwen 2.5 7B** 和 **Qwen 2.5-VL 7B**。
*   **算力说明**：由于该方法是**免训练**的（Training-free），因此不涉及 GPU 训练时长。
*   **推理延迟**：文中记录了推理延迟，基础 BiCICLe 每集（episode）中位耗时约 125.3s，而带有“手臂辩论”的版本因多次 API 调用耗时增至约 278.2s。

#### 5. 实验数量与充分性
*   **实验规模**：在 13 个仿真任务上进行了测试，每个任务使用 3 个随机种子，每个种子运行 100 次实验，数据量较为充分。
*   **消融实验**：非常详尽，涵盖了主手臂分配（左手领头 vs 右手领头）、辩论机制 vs 简单对话、观察表示（是否包含旋转）、点云提取方法等多个维度。
*   **客观性**：通过对比监督学习 SOTA 模型并展示在分布外（OOD）任务上的优势，客观地评价了 ICL 方法的优劣。

#### 6. 主要结论与发现
*   **性能卓越**：BiCICLe + Best-of-N 实现了 **71.1%** 的平均成功率，比最强的免训练基准高出 6.7%，甚至超过了 PerAct 2 和 $\pi_0$-keypose 等需要大量数据训练的监督模型。
*   **解耦的必要性**：实验证明，主从分解比单智能体（动作空间太大）或独立双智能体（缺乏协调）效果更好。
*   **泛化能力强**：在面对全新任务时，BiCICLe 仅需 10 个示例即可成功，而监督模型 3DFA 在同等少样本条件下几乎失效（成功率仅 10% vs BiCICLe 54.5%）。
*   **模型无关性**：在 7B 规模的开源模型上依然保持了架构优势，证明该方法不依赖于特定闭源模型。

#### 7. 优点
*   **无需训练**：极大地降低了双臂机器人任务的部署门槛，无需昂贵的 GPU 训练集群和海量标注数据。
*   **架构优雅**：通过“主从分解”巧妙解决了高维空间导致的上下文过载问题。
*   **鲁棒的协调性**：引入“辩论”和“裁判”机制，有效减少了 LLM 输出的随机性和不合理动作。
*   **Sim-to-Real 潜力**：通过离散化表示，成功将仿真中学习到的模式迁移至真实物理机器人。

#### 8. 不足与局限
*   **精度受限**：由于采用了体素离散化（100^3 分辨率），对于需要极高精度的微操任务（如抓取细绳）表现尚不如连续空间的监督模型。
*   **推理成本高**：Best-of-N 和手臂辩论需要多次调用 LLM API，导致推理延迟较高，难以实现实时闭环控制。
*   **感知依赖**：实验依赖于准确的物体分割和点云提取，在视觉噪声巨大的真实复杂场景中，感知误差可能会被 LLM 放大。
*   **长程任务挑战**：受限于 LLM 的上下文窗口，无法处理步骤极多的超长程操作任务。
