## 摘要
机器人系统缺乏一种以统一方式组织智能、能力和执行的有原则的抽象。现有方法要么将技能耦合在单体架构中，要么将功能分解为松散协调的模块或多个智能体，通常缺乏连贯的身份和控制权模型。我们认为，机器人应该被建模为一个单一的持久智能主体，其能力通过可安装的软件包进行扩展。我们将这一观点形式化为 AEROS（智能体执行运行时操作系统），其中每个机器人对应一个持久智能体，能力通过具身能力模块（ECM）提供。每个 ECM 封装了可执行技能、模型和工具，而执行约束和安全保证则由策略分离的运行时强制执行。这种分离实现了模块化可扩展性、可组合的能力执行以及一致的系统级安全性。我们在 PyBullet 模拟中使用 Franka Panda 7 自由度机械臂评估了一个参考实现，涵盖了重新规划、故障恢复、策略执行、基准比较、跨任务通用性、ECM 热插拔、消融实验和故障边界分析等八项实验。在每种条件下超过 100 次的随机试验中，AEROS 在三项任务中实现了 100% 的任务成功率，优于基准测试（BehaviorTree.CPP 风格和 ProgPrompt 风格为 92-93%，扁平流水线为 67-73%）；策略层以零误收率拦截了所有无效动作，运行时的优势在无需针对特定任务进行调整的情况下即可推广到不同任务，且 ECM 支持运行时加载，热插拔后的成功率为 100%。

## Abstract
Robotic systems lack a principled abstraction for organizing intelligence, capabilities, and execution in a unified manner. Existing approaches either couple skills within monolithic architectures or decompose functionality into loosely coordinated modules or multiple agents, often without a coherent model of identity and control authority. We argue that a robot should be modeled as a single persistent intelligent subject whose capabilities are extended through installable packages. We formalize this view as AEROS (Agent Execution Runtime Operating System), in which each robot corresponds to one persistent agent and capabilities are provided through Embodied Capability Modules (ECMs). Each ECM encapsulates executable skills, models, and tools, while execution constraints and safety guarantees are enforced by a policy-separated runtime. This separation enables modular extensibility, composable capability execution, and consistent system-level safety. We evaluate a reference implementation in PyBullet simulation with a Franka Panda 7-DOF manipulator across eight experiments covering re-planning, failure recovery, policy enforcement, baseline comparison, cross-task generality, ECM hot-swapping, ablation, and failure boundary analysis. Over 100 randomized trials per condition, AEROS achieves 100% task success across three tasks versus baselines (BehaviorTree.CPP-style and ProgPrompt-style at 92--93%, flat pipeline at 67--73%), the policy layer blocks all invalid actions with zero false acceptances, runtime benefits generalize across tasks without task-specific tuning, and ECMs load at runtime with 100% post-swap success.

### 1. 核心问题与研究动机
*   **核心问题**：现有的机器人系统在组织智能、能力和执行方面缺乏统一且有原则的抽象。
*   **研究背景**：
    *   **单体架构**：技能与控制逻辑紧耦合，难以扩展或重用。
    *   **多智能体/松散模块**：缺乏统一的身份、记忆和控制权模型，导致系统行为碎片化，且难以保证安全性。
*   **核心主张**：机器人不应被视为多个智能体的集合，而应是一个**单一的持久智能主体**，其能力通过可安装的“具身能力模块（ECM）”进行扩展。

### 2. 方法论：核心思想与技术细节
AEROS 架构基于三大原则：**单智能体原则**、**能力打包原则**和**策略-逻辑分离原则**。系统分为三层：

*   **持久智能体层 (Agent Layer)**：
    *   作为唯一的决策主体，维护身份、记忆（情节、语义、工作记忆）、世界模型和规划器。
    *   负责将高级目标分解为任务结构，并调用 ECM 提供的技能。
*   **具身能力模块层 (ECM Layer)**：
    *   **ECM 结构**：包含能力定义、技能（原子执行单元）、模型/工具以及清单（Manifest，定义依赖和资源需求）。
    *   **生命周期**：支持安装、配置、激活、停用和移除。ECM 仅扩展能力，不引入新的决策主体。
*   **运行时层 (Runtime Layer)**：
    *   **策略引擎**：独立于能力逻辑，强制执行权限、安全规则和资源约束。
    *   **执行逻辑**：通过公式 $R = (A, E, \Pi)$ 表达，其中 $A$ 是智能体，$E$ 是 ECM 集合，$\Pi$ 是运行时策略配置。
    *   **闭环模型**：智能体观察环境 -> 规划任务 -> 调度技能 -> 运行时根据策略 $\Pi$ 过滤动作 -> 执行并反馈。

### 6. 主要结论与发现
*   **卓越的成功率**：在三项任务中，AEROS 均实现了 **100% 的成功率**，显著优于 BehaviorTree.CPP（92.7%）和 ProgPrompt（92.7%）。
*   **安全性保证**：策略层成功拦截了 100% 的非法动作请求，误报率为 0%。
*   **鲁棒性**：在技能失败概率高达 80% 的情况下，AEROS 仍能保持 >97% 的成功率，表现出极强的故障恢复能力。
*   **灵活性**：验证了 ECM 的**运行时热插拔**，新加载的能力可立即被智能体识别并用于规划，且无系统重启开销。

### 7. 优点与亮点
*   **架构创新**：提出了“单智能体+可插拔能力模块”的清晰抽象，解决了机器人系统中身份模糊和能力耦合的问题。
*   **策略分离**：将“能做什么（技能）”与“允许做什么（策略）”解耦，使得同一模块在不同安全约束下无需修改即可复用。
*   **闭环鲁棒性**：通过动态重规划和显式恢复动作（Recovery Actions），极大提升了系统在不确定环境下的生存能力。
*   **开源贡献**：提供了 ECM 模式规范和参考实现。

### 8. 不足与局限
*   **仿真局限**：所有实验均在 PyBullet 仿真中完成，尚未在真实物理机器人上验证传感器噪声、接触动力学等现实因素的影响。
*   **规划器简化**：为了实验的确定性，评估中使用了基于规则的规划器，虽然架构支持 LLM，但未深入探讨 LLM 推理延迟对实时安全策略的影响。
*   **扩展性瓶颈**：当 ECM 数量极大（如数百个）时，单智能体决策路径可能成为性能瓶颈。
*   **容错性缺失**：架构假设持久智能体本身是不死进程，未讨论智能体层崩溃时的系统级恢复机制。
