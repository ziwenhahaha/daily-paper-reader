## Abstract
We present the first theoretical guarantees for differentially private online reinforcement learning (RL) with general function approximation, extending beyond prior work restricted to tabular and linear settings. Our approach combines a batched policy update scheme with the exponential mechanism, together with a novel regret analysis. We show that, even under general function approximation, the regret in the model-free setting under differential privacy matches the state of the art for the linear case, scaling as $\widetilde{O}(K^{3/5})$, where $K$ denotes the number of episodes. As an important by-product, we also establish the first regret bound for online RL with batch update that depends on the standard complexity measure of coverability, complementing existing results based on a newly introduced Eluder-Condition class. In addition, we uncover fundamental gaps in recent results for private RL with linear function approximation, thereby clarifying its landscape.

# 论文总结

## 1. 核心问题与整体含义

**研究动机**：强化学习（RL）已广泛应用于个性化系统（推荐系统、医疗干预、社交机器人、大语言模型等），但在此过程中收集的用户反馈往往包含敏感信息，存在隐私泄露风险。例如在医疗决策中，状态可能包含患者隐私信息；在LLM中，用户提示和偏好反馈也可能暴露个人隐私。

**核心问题**：如何在保证差分隐私的前提下，实现高效的在线强化学习？现有工作主要局限于表格环境和线性函数逼近，无法处理现代RL系统中常见的非线性函数表示。

**研究目标**：首次为通用函数逼近下的差分私有在线RL提供理论保证，证明即使在通用函数逼近下也能实现次线性遗憾。

## 2. 方法论

### 核心思想

1. **无约束优化框架**：采用直接优化目标函数的方式（而非传统的置信集约束优化），该目标函数显式平衡探索与利用，使得可以直接应用标准隐私机制（如指数机制）。

2. **固定大小批量更新策略**：限制策略更新频率，控制隐私损失组合。每个批次大小为B，只在批次开始时更新策略。

3. **指数机制（Exponential Mechanism）**：使用该机制从函数类F中采样策略，而非直接取最大值，从而实现隐私保护。

### 关键技术细节

- **覆盖度（Coverability）**：作为探索复杂度度量，定义为 $C_{\text{cov}} = \min_{\mu_1,\cdots,\mu_H} \max_{h,\pi} \|d_{\pi}^h / \mu_h\|_\infty$
- **隐私定义**：采用联合差分隐私（JDP），保护用户信息免受其他用户串通攻击
- **敏感度计算**：分数函数 $S(f) := \eta f_1(s_1) - L_{BE}^D(f)$ 的敏感度为 $\Delta_S \leq 8H$

### 算法流程

**算法1（一般MDP）**：
1. 初始化空数据集D
2. 对每集k=1到K：
   - 若k为批次起始点（k = i·B + 1）：计算所有$f \in F$的分数，采样$f^{(k)}$，更新策略
   - 否则保持上一集策略不变
3. 执行策略收集轨迹，更新数据集
4. 返回均匀分布的平均策略

**算法2（确定性MDP + 结果奖励）**：
- 类似的批量更新框架
- 使用结果Bellman残差损失替代Bellman误差损失
- 使用分布式乐观项 $\bar{f}_1 = \mathbb{E}_{s_1\sim\rho}\max_a f_1(s_1,a)$

## 3. 实验设计

### 实验环境

- **设定**：确定性结果奖励MDP，视野H=4，二元动作空间
- **状态结构**：每集开始时采样二元上下文 $x \in \{0,1\}^6$，状态包含上下文和历史动作
- **奖励**：仅在结束时提供二元奖励（0/1），取决于动作序列是否匹配隐藏目标假设

### 函数类

- 有限非线性假设类，$|F| = 243$
- 每个假设由二元门 $g(x)$ 和每层动作规则 $u(x, \text{prefix})$ 组成
- 定义了两种难度场景：
  - **Easy环境**：目标门 $g_0(x)=1$，正奖励密度较高
  - **Hard环境**：目标门 $g_2(x)=x_1 \oplus x_3 \oplus x_5$，仅部分上下文可获得奖励

### 对比方法

1. `non_private_no_batch`：非私有、每集更新
2. `non_private_batched`：非私有、批量更新
3. `private_eps_8`：差分私有、$\varepsilon=8$、批量更新
4. `private_eps_5`：差分私有、$\varepsilon=5$、批量更新

### 评估指标

- **累计遗憾曲线**：记录K集后的累积regret
- **Plateau Episode**：达到最终累积遗憾95%的首个集数，作为收敛速度的标量度量

## 6. 论文的主要结论与发现

1. **首次理论保证**：首次为通用函数逼近下的差分私有在线RL提供了次线性遗憾保证，答案确认为"是"。

2. **遗憾上界**：
   - 一般MDP：$\tilde{O}(K^{3/5})$
   - 确定性MDP（结果奖励）：同样为$\tilde{O}(K^{3/5})$
   - 与线性函数逼近下模型无关的最优结果匹配

3. **隐私成本**：
   - 第一项（非隐私）与非私有情况匹配
   - 第二项（隐私成本）为$K^{3/5}$阶，取决于$\varepsilon$

4. **方法论贡献**：
   - 批量更新策略不仅适用于隐私保护，在非隐私情况下也可实现$\sqrt{K}$的更新频率下仍保持$\sqrt{K}$遗憾
   - 覆盖度成为衡量批量更新RL复杂度的标准工具

5. **澄清现有研究**：发现并详细说明了近期声称改进遗憾界限的论文（[NVW22; Sah25]）中存在根本性缺陷，主要涉及私有化设计矩阵时单调性假设不成立。

## 7. 优点

1. **理论创新性强**：
   - 首次处理通用函数逼近下的差分私有RL
   - 提供了首个依赖于覆盖度的批量更新遗憾界
   - 澄清了线性MDP下差分私有RL的现状

2. **算法设计巧妙**：
   - 无约束优化使得可以直接应用指数机制
   - 固定批量策略有效控制了隐私组合

3. **分析技术贡献**：

   - 对于确定性MDP的引理E.4完全利用了确定性特性

4. **理论与实践结合**：
   - 即使在一般函数逼近下也能扩展到无限函数类
   - 提供了explicit covering方法处理无限函数类

5. **实验验证清晰**：
   - plateau episode指标直观展示收敛速度
   - 实验清晰展示了非私有、私有批量、私有不同参数之间的差异

## 8. 不足与局限

1. **实验覆盖不足**：
   - 仅在小型自定义环境中验证，未使用标准基准（如Atari等）
   - 函数类规模有限（243个假设）
   - 缺乏与大规模实际应用的联系

2. **理论假设较强**：
   - **Bellman完备性**假设：要求$Th f_{h+1} \in F_h$对所有$f_{h+1} \in F_{h+1}$成立，远强于实现性
   - **覆盖度依赖**：性能直接依赖于覆盖度常数，但该参数的计算或估计在实践中可能困难

3. **隐私机制局限**：
   - 当前使用指数机制，需要有限函数类或显式覆盖
   - 未探索其他隐私机制（如高斯机制）在此框架下的应用
   - 隐私参数选择（$\varepsilon, \delta$）的工程实践指导有限

4. **计算复杂度**：
   - 需要维护和更新所有函数类的分数，在函数类较大时计算成本高
   - 对于连续函数类，需要构建$\beta$-cover，计算成本可能指数增长

5. **性能限制**：
   - $K^{3/5}$的遗憾上界虽然匹配线性情形，但对于一般函数逼近可能不是最优
   - 批量策略引入了延迟更新开销，在需要快速适应的场景中可能不理想

6. **应用场景限制**：
   - 主要针对批处理的策略更新，不适用于需要实时策略调整的在线服务场景
   - 确定性MDP假设在许多实际应用（如随机环境）中可能不成立

7. **缺乏敏感性分析**：
   - 未系统研究关键超参数（如批量大小B、探索参数η）的影响
   - 覆盖度参数对算法性能的敏感性未充分探讨
