## Abstract
On-policy self-distillation (self-OPD) densifies reinforcement learning with verifiable rewards (RLVR) by letting a policy teach itself under privileged context. We find that when this guidance spans the full response, all-token KL spends gradients on mostly redundant positions and amplifies privileged-information leakage, causing entropy rise, shortened reasoning, and out-of-distribution degradation in long-horizon math training. We propose Token-Routed Alignment for Critical rEasoning (TRACE), which distills only on annotator-marked critical spans: forward KL on key spans of correct rollouts, optional reverse KL on localized error spans, and GRPO on all remaining tokens, with the KL channel annealed away after a short warm-up. Our analysis explains TRACE through two effects: forward KL provides non-vanishing lift to teacher-supported tokens that the student under-allocates, while span masking and decay keep cumulative privileged-gradient exposure finite. On four held-out math benchmarks plus GPQA-Diamond, TRACE improves over GRPO by 2.76 percentage points on average and preserves the Qwen3-8B base OOD score on GPQA-Diamond, where GRPO and all-token self-OPD baselines degrade. Gains persist under online self-annotation (+1.90 percentage points, about 69% of the strong-API gain), reducing the concern that TRACE merely imports external annotator capability. Across scales, the best routed action is base-dependent: on Qwen3-8B it is forward KL on key spans, while on Qwen3-1.7B it shifts to reverse KL on error spans.

## 1. 论文的核心问题与整体含义

### 研究动机

- **背景**：强化学习与可验证奖励（RLVR）如 GRPO 已成为训练大型推理模型的核心范式，但其仅提供标量奖励，导致训练信号稀疏
- **现有方案的局限**：on-policy 自蒸馏（self-OPD）通过让策略在特权上下文下自教来密集化信号，但发现在长horizon 数学训练场景中，全响应 KL 散度导致：
  - **梯度冗余**：约 97% 的概率质量集中在共享的 student-teacher token 上，KL 梯度浪费在学生已正确生成的位置
  - **熵上升**：SDPO/SRPO 等方法使 actor 熵飙升至 GRPO 基线的 4 倍以上
  - **响应缩短**：SDPO 响应从 2027 token 缩减至 1042 token
  - **OOD 退化**：GRPO 和所有-token self-OPD 基线在 GPQA-Diamond 上均出现性能下降
- **核心洞察**：问题粒度不匹配——蒸馏的正确单位不是整个响应，而是少数需要修正的关键推理 token

## 2. 论文提出的方法论

### 核心思想

**Token-Routed Alignment for Critical rEasoning (TRACE)**：一种 token 路由的自 OPD 方法，仅在标注者标记的关键跨度上应用蒸馏，并通过 KL 通道衰减控制特权梯度暴露。

### 关键技术细节

#### 角色分工
- **学生策略** π_S：基于问题 x 生成 rollout
- **特权教师** π_T：与学生共享参数 θ，但额外接收诊断类型标签作为特权上下文
- **跨度标注器** π_A：识别关键跨度（Ky）和错误跨度（Ey），不产生 logits 或梯度

#### 三类 Token 的差异化处理

| Token 类别 | 定义 | 默认处理 |
|-----------|------|---------|
| **Ky**（关键跨度） | 正确 rollout 中决定性推理步骤 | **Forward KL (FKL)**：提升学生分配不足的教师支持 token |
| **Ey**（错误跨度） | 错误 rollout 中的根因错误步骤 | **Reverse KL (RKL)**（可选）：抑制学生过度自信的错误 token |
| **Ny**（非跨度） | 其余 token | **GRPO**：无教师 KL，验证器驱动的 RL |

#### 公式说明
- **Eq. (1)**：per-step 损失 = GRPO(Ny) + ρ_k·GRPO(Sy) + λ_k·路由 KL
- **Eq. (2)**：KL 权重衰减：warm-up 阶段保持 w₀=0.5，step 10-40 线性衰减至 0，step 40 后为纯 GRPO

#### 设计原则

1. **教师不接收 span 内容**：仅接收粗粒度类型标签（如 `case_split`、`missed_case`），防止内容泄漏
2. **KL 覆盖率上限**：|Ky ∪ Ey| ≤ α|ŷ|，α=0.25
3. **短期激活**：KL 通道仅活跃 40 步，保证累积特权梯度暴露有限（O(α·Λ²)）

#### 理论分析（核心定理）

- **Lemma 1**：FKL/RKL 的逐 token 梯度不对称性
- **Corollary 2**：FKL 提供与学生质量无关的 logit 提升（适合欠分配 key span）；RKL 是学生质量加权的下压力（适合过度自信错误 span）
- **Prop. 3**：累积暴露 bound = O(α·Λ²)，有限且与训练 horizon 无关
- **Prop. 4**：关键跨度信号下界，要求标注精度 q_K > B_K/(γ_K + B_K)

## 6. 论文的主要结论与发现

### 核心结果

1. **性能提升**：TRACE (FKL on Ky) 在 5 基准平均提升 **+2.76 pp**（78.75 → 81.51）
2. **OOD 保持**：唯一保留 Qwen3-8B base OOD 分数的方法（GPQA-Diamond 58.33 vs 58.27）
3. **自标注有效性**：在线自标注仍获 **+1.90 pp**（强 API 增益的 69%）
4. **跨规模 Corner 倒置**：
   - Qwen3-8B：FKL on Ky 主导
   - Qwen3-1.7B：RKL on Ey 主导（符合 Corollary 2 预测）
5. **稳定性优势**：TRACE 熵曲线与 GRPO 保持一致，SDPO/SRPO 熵飙升 4 倍+

### 关键机制发现

- **Span 定位是活性成分**：全-token + 相同衰减仍退化，span 定位（而非仅衰减）至关重要
- **Credit 集中度**：TRACE 8.5× 信用集中（vs 1.0-1.1× 基线）
- **关键 token 提升**：TRACE-FKL 实现 +0.145 nats 提升（+168% vs GRPO）

## 7. 优点

### 方法论亮点

1. **理论驱动**：基于 Lemma 1/Corollary 2 的 FKL/RKL 不对称性推导路由决策
2. **有限暴露保证**：Prop. 3 形式化证明累积特权梯度有限
3. **不引入额外模型**：复用同一参数 θ 作为教师，避免额外参数开销
4. **诊断信息最小化**：教师仅接收粗粒度类型标签，防止内容泄漏
5. **灵活路由**：支持 (μ_E, μ_K) ∈ {(0,1), (1,0), (1,1)} 三种 corner

### 实验设计亮点

1. **跨规模 Corner 预测验证**：Cor. 2 预测 1.7B 上 corner 倒置，实验完全吻合
2. **多维诊断面板**：熵、长度、EMA 反馈回路、credit 集中度全面覆盖
3. **负向实验**：测试全-token FKL+衰减、rubric 条件、asymmetric thinking 等均报告
4. **在线自标注削弱外部能力假设**：证明增益来自方法而非 annotator 能力
