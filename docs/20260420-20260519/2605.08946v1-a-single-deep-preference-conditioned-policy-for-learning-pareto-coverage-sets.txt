Abstract 

Preference-conditioned multi-objective reinforcement learning aims to learn a single policy that captures trade-offs across preferences, but under nonlinear scalar-ization the uniqueness and continuity of the preference-to-solution correspondence remain unclear. We study this problem in tabular multi-objective Markov deci-sion processes (MDPs) using smooth Tchebycheff scalarization as a monotone utility. Under mild interior conditions on the preference set, we prove that each preference induces a unique Pareto-optimal return vector and that this vector de-pends Lipschitz-continuously on the preference, providing a principled foundation for preference sweeping toward dense Pareto-front coverage. To compute these targets, we formulate the problem over occupancy measures and derive Concave Mirror Descent Policy Iteration (CMDPI), which achieves an O(1 /k ) objective-suboptimality rate. We further show that each update is equivalent to solving a Kullback-Leibler-regularized MDP with the previous policy as reference, yielding a policy-iteration interpretation and finite-iterate policy continuity across preferences. We instantiate the update as a deep actor-critic algorithm preserving previous-policy regularization. On eight MO-Gymnasium tasks, it achieves the best average hypervolume rank among recent baselines and strong expected-utility performance. Continuous-control experiments indicate gains beyond the discrete-action setting. 

1 Introduction 

Multi-Objective Reinforcement Learning (MORL) optimizes multiple possibly competing reward signals, so the central object is not a single optimal policy but a set of Pareto-optimal trade-offs (Pareto set / Pareto front) [ 1 ‚Äď3 ]. With deep reinforcement learning, MORL has been extended to high-dimensional and continuous-control domains [ 4, 5]. A prominent direction is preference-conditioned 

learning, where a user-specified preference vector ŌČ is input to a single deep model that outputs diverse trade-off policies on demand [ 6‚Äď 8, 5, 9 , 10 ]. We focus on dense coverage of Pareto-optimal objective vectors: by scanning preferences, we aim to reach a neighborhood of any Pareto-optimal objective vector in the objective space [3, 11, 2, 12, 5]. A standard route is to scalarize the objective vector by a utility function and solve a single-objective problem. In MORL taxonomies, if user utility is restricted to positive weighted sums, one targets a 

convex coverage set (CCS) ; if one allows arbitrary monotonically increasing utilities, one targets a 

Pareto coverage set (PCS) [ 3, 13 ]. CCS-oriented methods admit clean formulations (often compatible with dynamic programming / linear programming), but even for convex attainable sets, the optimal objective vector for a fixed preference can be non-unique (e.g., a whole supporting face), which makes the target ambiguous and can induce preference-wise switching/discontinuity [ 14 , 3, 2, 15 ]. In practice, they also tend to concentrate on extreme (vertex) solutions, making dense coverage chal-lenging [ 3 , 2 ]. PCS-oriented approaches can reach non-vertex trade-offs, but the resulting problems

4 Numerical Experiments 

We empirically evaluate the proposed methods in three stages. First, in Section 4.1, we study a tabular MOMDP to examine whether the proposed CMDPI update exhibits the intended planning behavior in a controlled setting. Second, in Section 4.2, we evaluate discrete-action MO-Gymnasium benchmarks to test whether the proposed deep methods, PCSAC and CMDPI, scale beyond the tabular setting and improve Pareto-front approximation quality. Finally, we evaluate continuous-control MO-Gymnasium tasks to investigate whether partially incorporating the proposed STCH-based method (PCSAC) remains effective in continuous-action domains. 2

4.1 Tabular MOMDP (planning) 

We study a two-objective tabular MOMDP ( |S| = 4 , |A| = 2 ) to isolate a key limitation of existing PCS methods: linear scalarization recovers only vertex solutions, while regularized methods may produce biased Pareto coverage. In a fully known planning setting, we compare linear scalarization, CAPQL [ 17 ], and the proposed CMDPI. For 100 preference vectors ŌČ, we compute the converged objective vectors and visualize them in Figure 1. Linear scalarization recovers only vertex solutions, while CAPQL produces non-vertex solutions but exhibits a bias‚Äďcoverage trade-off. In contrast, CMDPI consistently recovers Pareto-optimal solutions with more uniform (denser) coverage of the front. Full details and sweeps are provided in Appendix G.1. 

4.2 MO-Gymnasium Discrete-Action Tasks We first evaluate eight representative discrete-action MO-Gymnasium benchmarks [ 34 ] to investigate whether the proposed deep methods, PCSAC and CMDPI, scale beyond the tabular setting. We compare against linear scalarization (LS), CAPQL [17], PreCo [35], and C-MORL [9]. PCSAC and CMDPI are our proposed STCH-based methods. Here, CMDPI denotes a deep actor‚Äďcritic implementation of the CMDPI update, where the critic target incorporates KL regularization toward the previous policy following the mirror-descent derivation.

5 Related Work 

Coverage sets vs. preference sweeping (what ‚Äúdense‚ÄĚ means here). Coverage sets‚ÄĒCCS and PCS‚ÄĒand representative computation methods are surveyed in MORL [ 3, 13 ]. CCS-oriented planning typically returns a set sufficient for any (unknown) linear weight, e.g., via linear-support style enumeration of supporting solutions [ 38 , 39 ]. This is different from our question: whether 

sweeping preferences yields stable objective-space coverage . Even with a convex attainable set, a fixed preference can select a whole supporting face, making the preference-to-solution map set-valued and potentially causing abrupt switching under infinitesimal preference changes [14, 15]. Our work isolates this ‚Äúselection/regularity‚ÄĚ issue and shows that STCH admits a unique, stable objective-point selection in the tabular polytope regime. 

Nonlinear scalarization and SER/ESR. Beyond linear scalarization, MORL distinguishes scalar-ized expected returns (SER: scalarize the expected return vector) and expected scalarized returns 

(ESR: take expectation of a nonlinear utility) [ 3 ]. ESR is closely tied to risk-sensitive/dynamic-programming viewpoints and faces time-consistency and optimization challenges [ 40 , 41 , 18 ], while recent work develops practical algorithms for concave/nonlinear objectives [ 25 , 26 ]. However, for general nonlinear scalarizations it is still nontrivial whether a preference induces a unique objective point and whether preference sweeping avoids unstable switching. In contrast, we prove that under STCH with a polyhedral attainable set, the optimal objective vector is unique and ŌČ 7 ‚Üí JŌČ is Lipschitz on ő©mőī (SER viewpoint), making dense objective-space coverage analyzable. 

Deep MORL and an optimization view (single models, stability, and mirror descent). Classical approaches grow an explicit set of policies to approximate coverage sets [ 2, 11 , 12 ], whereas preference-conditioned deep methods aim to output a trade-off directly from a single model [ 6‚Äď8], including hypernetwork-style ‚Äúmany policies in one‚ÄĚ [ 10 ] and meta-policy formulations [ 35 ]. Other recent work broadens solution notions beyond linear scalarization (e.g., Pareto stationarity) [ 17 ]. Yet, scalarization-grounded guarantees that preference sweeping stably fills the objective space (via uniqueness/continuity rather than implicit generalization) remain limited. On the optimization side, mirror descent and regularized (soft) policy-iteration schemes have been clarified for regularized MDPs [ 16 , 42 ] and concave-utility RL [ 26 , 24 ]. Our CMDPI derives from mirror descent over occupancy measures using relative smoothness [ 23 ], yielding an O(1 /k ) guarantee without biasing the STCH objective and providing an explicit route to ŌČ-wise regularity of finite-iteration policies. 

6 Conclusion and Future Work 

We established that, for discounted tabular MOMDPs under STCH, each preference ŌČ induces a 

unique optimal objective vector JŌČ , and the mapping ŌČ 7 ‚Üí JŌČ is Lipschitz on ő©mőī . This makes preference sweeping a principled route to dense objective-space coverage. We then derived CMDPI as mirror descent over discounted occupancy measures, yielding an O(1 /k )

convergence guarantee without biasing the target objective . Empirically, CMDPI recovers broad Pareto-front coverage in tabular MOMDPs with improved robustness, and CMDPI-style updates im-prove sample efficiency on continuous-control MORL benchmarks relative to preference-conditioned baselines. For future work, we highlight two directions. G.2 Experimental Protocol and Baselines on MO-Gymnasium G.2.1 Experimental Protocol for Discrete-Action MO-Gymnasium Tasks Environments. We evaluate on eight MO-Gymnasium [ 27 ] tasks with discrete action spaces: 

deep-sea-treasure-concave-v0 , deep-sea-treasure-v0 , fishwood-v0 , four-room-v0 ,

fruit-tree-v0 , minecart-v0 , mo-lunar-lander-v3 , and mo-reacher-v5 . The reward di-mension ranges from two to six. We use the default episode-length settings of MO-Gymnasium and train each method for 2M environment interaction steps. 

Baselines and codebases. We compare LS (linear scalarization), CAPQL [ 17 ], PreCo [ 35 ], C-MORL [ 9], PCSAC, and CMDPI. LS, CAPQL, PreCo, PCSAC, and CMDPI are run within the same PreCo‚Äôs official codebase to reduce implementation-level confounders. PCSAC and CMDPI are our STCH-based preference-conditioned actor‚Äďcritic variants. For the PreCo-based implementations, we apply a common per-objective running reward normalization in critic learning, using the empirical mean and standard deviation of collected reward vectors. This normalization is used only during training; all reported return vectors and metrics are computed from unnormalized episodic returns. C-MORL is included as a strong multi-policy Pareto-front discovery baseline, although it is not a single preference-conditioned policy. We use the official public implementation and modify the interaction accounting and logging needed to align the training budget and post-hoc metric computation with the other methods. 

Training seeds. We run eight random seeds for each method and environment. 

Evaluation and metric computation. At each evaluation checkpoint, we collect unnormalized episodic return vectors from each method and save them for post-hoc evaluation. For preference-conditioned methods, we perform preference-sweep evaluation by conditioning the policy on a fixed rollout preference set and recording the resulting return vectors. For C-MORL, which outputs a set of policies rather than a single preference-conditioned policy, we evaluate the policies in the discovered set to obtain return vectors. We do not use the metric values logged by the original C-MORL implementation. Instead, after all runs are completed, we compute HV, EUM, and SP for all methods from the saved unnormalized return vectors using the same post-processing script. HV is computed with a common environment-wise reference point. EUM is computed using a common EUM evaluation preference set shared by all methods. 

Evaluation preferences. We distinguish between two preference sets. The rollout preference set is used as input to preference-conditioned policies during evaluation rollouts and therefore applies only to LS, CAPQL, PreCo, PCSAC, and CMDPI. The EUM evaluation preference set is used only for metric aggregation and is shared by all methods, including C-MORL. For C-MORL, the EUM preferences are used only at the aggregation stage because C-MORL returns a policy set rather than a 26 preference-conditioned policy. All preference sets are generated by the deterministic simplex-grid routine used in PreCo-based codebase and are fixed across methods, checkpoints, and seeds. 

Reference points. We distinguish between reference points used for reported HV computation and reference values used internally by C-MORL. For reported HV, we use a common environment-wise reference point shared by all methods. Following the same protocol as in the continuous-action experiments, we determine this reference point a posteriori as the component-wise minimum over the unnormalized evaluation return vectors collected across all methods, checkpoints, and random seeds. C-MORL additionally uses internal reference values in its online multi-policy selection procedure. These internal values are used only during C-MORL training and are never used to compute the reported HV, EUM, or SP values. When the official C-MORL implementation provides environment-specific internal values, we keep them; otherwise, we use a conservative lower-bound vector with all entries set to ‚ąí10 4.

Counting environment interactions. We count only environment interactions used for training and exclude evaluation rollouts for all methods. For C-MORL, we count interactions across parallel sampling workers when necessary, so that the logged sampling steps reflect the actual number of environment transitions used for training. This adjustment affects only sample-efficiency accounting. 

Hyperparameters. PCSAC uses the off-policy replay-buffer optimization scheme with entropy coefficient őĪent = 0 .3 tuned out of {0.03 , 0.1, 0.3}. CMDPI uses the on-policy optimization scheme motivated by the KL-regularized formulation, with tuned smoothness/KL parameter 0.001 out of 

{0.1, 0.01 , 0.001 }. For numerical stability of the KL-regularized actor update, the CMDPI actor optimizer uses Adam with weight decay 10 ‚ąí4. For C-MORL, we tune the fraction of the total 2M-step budget allocated to the initialization stage 0.4 out of {0.2, 0.4, 0.5}. All other shared optimizer and network hyperparameters in the PreCo-based implementations follow the default settings of the codebase.
