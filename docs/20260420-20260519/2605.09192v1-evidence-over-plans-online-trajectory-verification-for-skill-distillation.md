## 摘要
智能体技能通过使用人类编写的过程性文档可以显著提高任务成功率，但其质量在没有环境基础验证的情况下难以评估。现有的技能生成方法严重依赖偏好日志而非直接环境交互，通常产生微小甚至下降的收益。我们发现这是一个根本性的时间瓶颈：稳健的技能应该是后验的，从经验环境交互中提炼而非先验计划。在本研究中，我们引入了后验蒸馏指数（PDI），这是一个轨迹级别的指标，用于量化蒸馏技能在任务-环境证据中的扎根程度。为了操作化PDI，我们提出了SPARK（用于自主可运行任务和技能生成的结构化流水线），用于保存任务执行证据以进行完整的轨迹级别分析。SPARK生成用于计算PDI的环境验证轨迹，并应用PDI作为在线诊断和干预信号以确保后验技能形成。在86个可运行任务中，SPARK生成的技能始终超越无技能基线，并在学生模型上优于人类编写的技能（推理成本比教师模型低达1000倍）。这些发现表明，PDI引导的蒸馏产生了基于任务-环境交互的高效且可迁移的技能。我们在https://github.com/EtaYang10th/spark-skills发布我们的代码。

## Abstract
Agent skills can remarkably improve task success rates by using human-written procedural documents, but their quality is difficult to assess without environment-grounded verification. Existing skill generation methods heavily rely on preference logs rather than direct environment interaction, often yielding negligible or even degraded gains. We identify that it is a fundamental timing bottleneck: robust skills should be posterior-based, distilled from empirical environment interaction rather than prior plans. In this study, we introduce the Posterior Distillation Index (PDI), a trajectory-level metric that quantifies how well a distilled skill is grounded in the task-environment evidence. To operationalize PDI, we present SPARK (Structured Pipelines for Autonomous Runnable tasKs and sKill generation) for preserving task execution evidence towards full trajectory-level analysis. SPARK generates environment-verified trajectories used to compute PDI, and it applies PDI as an online diagnostic and intervention signal to ensure posterior skill formation. Across 86 runnable tasks, SPARK-generated skills consistently surpass no-skill baselines and outperform human-written skills on student models (inference cost up to 1,000x cheaper than teacher models). These findings show that PDI-guided distillation produces efficient and transferable skills grounded in the task-environment interaction. We release our code at https://github.com/EtaYang10th/spark-skills .

## 1. 论文的核心问题与整体含义（研究动机和背景）

**核心问题**：什么使得技能蒸馏能够高效、可靠地跨任务和模型迁移？

**研究背景与动机**：

- **技能的价值**：大语言模型（LLM）智能体在使用人类编写的过程性文档（称为"技能"，Skill）时，能够显著提高交互式软件任务的成功率。这些技能编码了任务特定知识，如执行顺序、环境约束和验证器合约。
- **现有瓶颈**：现有技能生成方法严重依赖偏好日志（preference logs）而非直接环境交互，导致收益微小甚至性能下降。
- **根本原因**：作者认为关键问题在于**技能形成的时机**——现有方法让模型在与环境交互之前就"计划"如何解决问题，产生受通用先验和启发式主导的技能。
- **核心观点**：有效的过程性知识应当是**后验的（posterior-based）**，即从环境交互经验中提炼，因为只有通过环境交互才能发现环境特定约束、执行依赖和失败模式。

## 2. 论文提出的方法论

### 核心思想

技能应该是**后验的**——从经验环境交互中蒸馏，而非依赖先验计划。技能质量应由其在任务-环境证据中的扎根程度来衡量。

### 关键技术：后验蒸馏指数（PDI）

PDI 是一个轨迹级别的指标，通过 Token 分布的 Jensen-Shannon 散度来量化技能是否基于环境验证证据。

**三个组成部分**：
- **执行落地（Execution Grounding, φ_exec）**：技能与成功执行轨迹中命令分布的相似度。高值表示技能聚焦于环境验证过的操作。
- **计划复制（Plan Copying, φ_plan）**：技能与探索备忘录（Exploration Memo）中"下一步策略"分布的相似度。高值表示技能复制了先验计划而非独立验证的知识。
- **备忘录固化（Memo Ossification, φ_oss）**：跨尝试的验证事实和失败测试集分布稳定性。高值表示轨迹陷入重复，任务理解未实质更新。

**PDI 公式**：
```
PDI = z(φ_exec) - z(φ_plan) - z(φ_oss)
```
其中 z(·) 表示 z-score 归一化。

**直觉解释**：高 PDI 表示技能分布对齐于环境验证证据（高 φ_exec），偏离备忘录级计划（低 φ_plan），且源于任务理解真正演化的轨迹（低 φ_oss）。

### SPARK 框架

SPARK（Structured Pipelines for Autonomous Runnable tasKs and sKill generation）包含两大流水线：

**技能生成流水线**：
1. 教师智能体在 Docker 容器化环境中交互（最多 N_max=7 次尝试）
2. 每次尝试后更新**探索备忘录**（结构化摘要，包含：尝试日志、关键命令、验证事实、当前错误模式、下一步策略）
3. 成功时，将六类证据块（任务模式、执行链、验证记录、教训、环境信息、原始支持尾）蒸馏为 SKILL.md
4. PDI 作为在线诊断和干预信号，在轨迹质量下降时触发软/强干预

**任务构建流水线**：
将自然语言提示转换为结构化任务蓝图 → 迭代修复 → 批评检查 → 验证，最终生成自包含的基准任务。

## 6. 论文的主要结论与发现

### 核心发现

1. **SPARK 技能显著优于人类技能**：SPARK 生成技能在大多数学生模型上持续超越人类编写技能。例如，GPT-5.4-mini 使用 SPARK 技能达到平均奖励 0.52，而人类技能为 0.47。

2. **小模型可超越大模型**：多个配备 SPARK 技能的小模型甚至超越无技能的教师模型。例如，GPT-5.4-nano + SPARK（0.41）超过 Claude Opus 4.6 无技能（0.37）。

3. **PDI 有效预测技能质量**：
   - PDI 与技能收益呈显著正相关（ρ = +0.364, p < 10⁻⁶）
   - 高 PDI 轨迹的通过-收益率在所有学生模型上最高
   - 备忘录固化与人类技能差距呈负相关（ρ = -0.277, p < 10⁻³）

4. **执行落地 + 低计划复制 = 最佳组合**：
   - 低 φ_plan + 高 φ_exec：平均收益 +0.377，相对人类技能差距 +0.312
   - 高 φ_plan + 低 φ_exec：平均收益仅 +0.028，相对人类技能差距 -0.040

5. **探索模式关键**：
   - **过度压缩证据伤害技能质量**（压缩比与收益负相关）
   - **3 次尝试后收益变得不稳定**
   - **发散探索优于收敛探索**（跨尝试备忘录相似度下降）

6. **PDI 在线干预有效**：启用 PDI 引导的干预后，通过率显著提升（如 DeepSeek-Chat 从 9.1% 升至 42.4%）。

7. **跨领域泛化有效**：在 ALFWorld 上，PDI 优化技能将成功率从 16.7% 提升至 40.0%。

## 7. 优点

1. **创新性的指标设计**：PDI 首次提出轨迹级别的"后验性"量化标准，将技能质量从主观偏好提升到可测量的证据扎根程度。

2. **完整闭环系统**：SPARK 框架实现了"探索 → 证据保存 → 质量诊断 → 在线干预 → 技能蒸馏 → 评估"的完整闭环，而非仅关注单一环节。

3. **可操作的在线信号**：PDI 不仅用于事后分析，还通过代理-PDI（proxy-PDI）实时监控探索质量，触发分级干预（软/强），避免暴露指标给模型。

4. **深度消融验证**：论文对 PDI 的每个组成部分、权重、平滑参数进行了全面的敏感性分析，确认其鲁棒性。

5. **成本效益显著**：通过一次性高成本教师探索换取低成本学生推理（最高 1,124 倍差异），同时提升性能，具有实用价值。

6. **跨领域泛化验证**：在 ALFWorld 上的实验证明了方法和指标并非仅适用于特定基准。

7. **解释性强**：通过 JSD 分布相似度而非黑盒预测，PDI 提供可解释的质量诊断。

---

## 8. 不足与局限

1. **环境覆盖有限**：聚焦于文本/终端环境，未验证 GUI、多模态、机器人等更复杂交互场景的适用性。

2. **PDI 形式简单**：采用等权重线性组合，虽然被验证鲁棒，但可能在高度异质任务分布上不如自适应权重。

3. **跨模型协同未探索**：实验评估的是单模型使用技能的效果，未探索多模型协作场景。

4. **任务规模相对有限**：尽管扩展到 300 变体，原始基准仅 86 个任务，可能影响统计结论的稳健性。

5. **教师探索成本高**：教师探索仍需大量 API 调用（$9.90–$24.72/任务），对小团队可能不友好。

6. **泛化机制未完全解释**：虽然发现"发散探索"和"执行落地"重要，但这些洞察如何指导自动化探索策略仍需进一步研究。

7. **验证器依赖**：整个框架依赖确定性 pytest 验证器，对于没有明确正确性标准（如开放式创意任务）的任务可能不适用。

8. **附录资源分散**：部分关键分析（如案例研究、详细统计数据）位于附录，可能影响核心贡献的突出性。
