## 摘要
通信在大语言模型预训练中逐渐占据主导成本，特别是在数据并行和分片训练方案下，梯度同步和参数重建开销随模型规模和系统规模增长。现有的通信缩减方法要么对原始梯度进行稀疏化，在高度稀疏下对现代Adam风格优化器可能不稳定；要么对通信进行量化，其节省受限于位宽，且常常带来额外运行时开销。我们提出SCAPE，一种面向大语言模型训练的高效通信分布式优化器，它利用AdamS一阶矩的稳定性，在不损失大语言模型质量的前提下实现激进稀疏化。SCAPE并非从原始梯度构建掩码，而是从基于一阶矩的统计量中导出掩码，将掩码生成分布到各工作节点以对齐优化器分片，并将掩码使用延迟一步，使掩码同步可与计算重叠。SCAPE还从单个同步的稀疏缓冲区中重构二阶矩更新所需的数据，避免了额外的集合通信。我们在Megatron-LM中实现SCAPE，并在TACC Vista上使用32个NVIDIA GH200 GPU，通过在OpenWebText上预训练GPT-345M和在SlimPajama-6B上预训练Llama-500M来评估其收敛性。在两个模型中，SCAPE在90%和99%稀疏度下均保持训练稳定性、验证损失和下游任务准确率。对于Llama-500M，SCAPE将端到端预训练时钟时间减少了最多43.3%，同时保持与稠密AdamW和AdamS相当的模型质量。对于Llama-1.8B，与稠密AdamS相比，SCAPE每步加速最多达3.26倍。

## Abstract
Communication increasingly dominates the cost of Large Language Model (LLM) pre-training, especially under data-parallel and sharded training schemes, where gradient synchronization and parameter reconstruction overhead increase with model size and system scale. Existing communication-reduction methods either sparsify raw gradients, which can be unstable for modern Adam-style optimizers at high sparsity, or quantize communication, whose savings are fundamentally bounded by bit width and often incur additional runtime overhead. We present SCAPE, a communication-efficient distributed optimizer for LLM training that exploits the stability of AdamS's first-moment to enable aggressive sparsification without loss of LLM quality. Instead of constructing masks from raw gradients, SCAPE derives them from first-moment-based statistics, partitions mask generation across workers to align with optimizer sharding, and delays mask usage by one step so that mask synchronization can overlap with computation. SCAPE also reconstructs the quantities required for second-moment updates from a single synchronized sparse buffer, avoiding an additional collective. We implement SCAPE in Megatron-LM and evaluate its convergence by pre-training GPT-345M on OpenWebText and Llama-500M on SlimPajama-6B using 32 NVIDIA GH200 GPUs on TACC Vista. In both models, SCAPE preserves training stability, validation loss, and downstream task accuracy under 90\% and 99\% sparsity. For Llama-500M, SCAPE reduces end-to-end pre-training wall-clock time by up to 43.3\% while maintaining model quality comparable to dense AdamW and AdamS. For Llama-1.8B, SCAPE achieves up to 3.26$\times$ speedup per step compared to dense AdamS.

## 2. 论文提出的方法论

### 核心思想
- 不再对原始梯度进行 top-k，而是对 **AdamS 的一阶矩（momentum）** 进行稀疏化，因为一阶矩是梯度的指数滑动平均，更稳定、噪声更小。
- 利用 **时间稳定性**：相邻两步的一阶矩 top-k 掩码具有高重叠率（图 6），因此可以将掩码生成延迟一步，与计算重叠。
- **分区掩码生成**：在 Megatron-LM 的分片分布式优化器中，每个 worker 只负责自己分片层的 top-k 计算，避免“填满效应”（所有 worker 的 top-k 索引聚集后变为稠密）。

### 关键技术细节
| 组件 | 说明 |
|------|------|
| **掩码构造** | 基于 `|β₁ m_{t-1} + (1-β₁)g_t + e_{t-1}|` 进行 top-k，其中 e 是误差反馈缓冲区。 |
| **一步延迟** | 第 t 步使用的掩码由第 t-1 步的临时一阶矩计算，此时同步掩码（all-gather）可与当前步的计算（前向/反向）重叠。 |
| **单次 all-reduce** | 只传输稀疏化的临时一阶矩缓冲区 `b_t`，再通过公式 `¯g_t + ¯e_{t-1} = (¯b_t - β₁m_{t-1})/(1-β₁)` 恢复全局平均梯度，用于二阶矩更新，避免两次通信。 |
| **参数更新简化** | 对于未被掩码选中的位置，模型参数仅执行权重衰减 `θ ← (1-ηλ)θ`，无需 all-gather 通信。 |
| **CPU offload 与双缓冲** | 将误差反馈缓冲区（FP32）和完整模型参数 FP32 副本 offload 到 CPU，通过 NVLink-C2C 高速互联（900GB/s）进行异步传输，减少 GPU 内存占用。 |

### 算法流程（文字描述）
1. 初始化全局掩码 M₀ 和误差缓冲区 e₀。  
2. 每步 t：  
   - 计算局部梯度 g_tⁿ。  
   - 计算临时一阶矩 `˜m_tⁿ = β₁m_{t-1} + (1-β₁)g_tⁿ + e_{t-1}ⁿ`。  
   - 使用前一步掩码 M_{t-1} 提取稀疏缓冲区 `b_tⁿ = ˜m_tⁿ [M_{t-1}]`，并将未选中位置的残差存入 e_tⁿ。  
   - 发起 all-reduce 得到全局稀疏缓冲区 `¯b_t`。  
   - 同时刷新当前步掩码 M_t（基于 |˜m_tⁿ| 进行分区 top-k）。  
   - 从 `¯b_t` 恢复梯度用于二阶矩 v_t 更新。  
   - 更新模型参数。

- **依赖 AdamS 特性**：仅适用于 AdamS，未验证在 AdamW 或其他优化器（如 Muon、Sophia）上的扩展性。

- **稀疏度选择**：仅测试了 d=0.1 和 0.01，未探索中间稀疏度或每层自适应稀疏。  
- **未报告多次运行统计量**（如方差），单次运行可能受随机种子影响，尤其是下游任务分数有波动。  

### 应用限制
- 当前实现基于 Megatron-LM，迁移到其他框架（如 DeepSpeed、PyTorch FSDP）需要额外工程。  
- CPU offload 双缓冲依赖 GH200 的特别高速互联，在普通集群可能引入不可忽略的 PCIe 传输开销。
