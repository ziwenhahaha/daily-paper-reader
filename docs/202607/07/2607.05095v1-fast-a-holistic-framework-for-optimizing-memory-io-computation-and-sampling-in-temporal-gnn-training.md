## 摘要
时序图神经网络（TGNN）广泛应用于推荐、社交网络分析和交通预测等动态图学习任务。然而，由于内存I/O、不规则计算和时序邻居采样这三个相互交织的瓶颈，将TGNN训练扩展到大规模动态图仍然具有挑战性。现有系统通常孤立地优化这些阶段，从而留下了显著的性能提升空间。我们提出了FAST，一个通过联合优化采样、内存I/O和计算来加速端到端TGNN训练的整体框架。FAST引入了SlimCache，它利用批内压缩和跨批缓存来在有限的GPU内存预算下减少主机-设备数据传输。进一步地，FAST设计了针对稀疏时序子图定制的线程高效图算子，提高了GPU缓存局部性，并降低了聚合和边softmax的延迟。此外，FAST采用了一种拓扑感知的采样策略，提高了CPU缓存局部性并加速了时序邻居采样。在真实世界大规模动态图上的大量实验表明，FAST在不牺牲模型准确性的前提下，相比最先进的系统实现了平均2.1倍（最高4.7倍）的加速。

## Abstract
Temporal Graph Neural Networks (TGNNs) are widely used for learning from dynamic graphs in applications such as recommendation, social network analysis, and traffic forecasting. However, scaling TGNN training to large dynamic graphs remains challenging due to three intertwined bottlenecks: memory I/O, irregular computation, and temporal neighbor sampling. Existing systems often optimize these stages in isolation, leaving substantial performance headroom on the table. We present FAST, a holistic framework that accelerates end-to-end TGNN training by jointly optimizing sampling, memory I/O, and computation. FAST introduces SlimCache, which exploits within-batch compression and cross-batch caching to reduce host-device data movement under limited GPU memory budgets. It further designs thread-efficient graph operators tailored to sparse temporal subgraphs, improving GPU cache locality and reducing the latency of aggregation and edge softmax. In addition, FAST employs a topology-aware sampling strategy that improves CPU cache locality and accelerates temporal neighbor sampling. Extensive experiments on real-world large dynamic graphs show that FAST achieves an average of 2.1x (up to 4.7x) speedup over state-of-the-art systems without sacrificing model accuracy.

### 2. 方法论：核心思想、关键技术细节

#### 核心思想
FAST是一个整体框架，基于**TGNN训练中采样子图存在显著的批内和跨批冗余**这一观察，通过协同一体化设计来加速TGNN训练。

#### 关键技术细节

1.  **SlimCache：联合压缩与缓存优化内存I/O**
    *   **核心思想**：同时利用批内重复性（within-batch repetition）和跨批重叠性（cross-batch overlap），并区分节点和边的不同冗余模式。
    *   **批内压缩**：在运行时，只传输该batch中唯一的节点/边ID及其特征，并附一个逆索引（inverse index）用于在GPU上重建，减少传输量。
    *   **跨批缓存**：使用一个GPU常驻缓存来存储频繁访问的特征。采用**贪心选择策略**：通过预采样（pre-sampling）统计多个mini-batch中节点和边的访问频率，并在固定缓存预算下，根据边际命中增益交替分配缓存空间给节点或边（Algorithm 1）。这比均匀分配或仅优先节点更高效，因为节点和边的冗余模式显著不同（节点重复率高、边重叠率高）。
    *   **效果**：协同压缩与缓存，最大限度减少主机-设备数据传输。

2.  **线程高效图算子：优化不规则计算**
    *   **核心思想**：针对TGNN中稀疏、小度的子图结构，定制CUDA kernel以消除负载不均衡并提高线程利用率。
    *   **平衡聚合（Balanced Aggregate, AGG）**：
        *   **问题**：传统节点并行实现中，每个线程负责一个节点的邻居聚合，由于节点度分布偏斜，导致严重的负载不均衡。
        *   **方案**：采用**边中心（edge-centric）** 执行方案，每个线程独立处理一条边，使用`atomicAdd`将结果累加到目标节点。由于TGNN子图通常很小（最大度≤10），原子操作竞争开销可忽略不计。该方案彻底消除了度相关的不均衡。
    *   **线程高效边软最大（Edge Softmax, ESM）**：
        *   **问题**：传统的warp shuffle规约需要根据最大邻居数设定规约宽度，当节点度小时会有大量非活动线程浪费。
        *   **方案**：采用**CSR风格的线程循环策略**，每个线程迭代处理多个规约元素（对应同一目标节点的多条边），消除无效活动线程。同时，该设计将每block处理的节点数从约64（warp shuffle）提升到1024，增大了工作集，改善了L1/L2缓存命中率。

3.  **拓扑感知采样：加速CPU邻居采样**
    *   **核心思想**：利用CPU缓存层级中的资源共享（如共享L2/L3缓存），将具有高度邻居重叠的采样线程绑定到同一物理核上，从而提升缓存局部性。
    *   **实现方法**（Algorithm 2）：
        *   **预采样**：根据训练轮次中所有batch的根节点信息，构建**线程亲和性矩阵**，其中元素为两个线程所处理子图的Jaccard相似度（只使用0跳根节点，开销很小）。
        *   **线程绑定**：采用Blossom算法（最大权重匹配）找出最优的线程配对，生成线程-核心绑定列表，并在训练开始前一次性应用，避免运行时开销。
    *   **效果**：显著提高了L2和L3缓存命中率，从而加速采样。
