## Abstract
Activation steering controls LLM behaviour towards target behaviour by intervening in internal representations, yet it often degrades reasoning and retrieval performance. We argue that a primary cause of this trade-off is attention rerouting: steering vectors alter query-key matching, shifting attention away from contextually important tokens toward less informative ones. To address this, we propose Steering via Key-Orthogonal Projections (SKOP), a steering method that constrains harmful attention rerouting without eliminating steering efficacy. SKOP achieves this by preserving attention patterns on a small set of focus tokens the model relies on for reasoning and retrieval, while allowing redistribution among less critical tail tokens. Across multiple steering benchmarks, we show that SKOP achieves the best joint steering-utility trade-off, reducing utility degradation by 5-7x while retaining over 95% of vanilla steering efficacy. Our results further suggest that, in long-context retrieval settings where vanilla steering approaches are ineffective, SKOP can maintain robust performance by avoiding attention rerouting.

# 论文详细总结：Don't Lose Focus: Activation Steering via Key-Orthogonal Projections

## 1. 论文的核心问题与整体含义（研究动机和背景）

**核心问题**：激活控制（Activation Steering）通过干预LLM内部表征来引导模型行为，但在实际应用中普遍存在一个关键权衡——steering efficacy（引导效能）与 utility preservation（效用保持）之间的矛盾。具体表现为：随着steering强度的增加，虽然目标行为被更有效地引导，但模型在推理、检索等核心能力上出现显著性能下降。

**根本原因**：论文首次揭示了这一权衡的内在机制——**注意力重路由（Attention Rerouting）**。作者论证指出，query-space steering通过修改query-key匹配的相对分数来改变注意力分布，使得原本聚焦于关键上下文token（focus tokens）的注意力质量被重新分配到不太重要的token（tail tokens）上，从而损害了模型依赖的关键信息获取能力。

**研究动机**：现有方法试图通过条件化steering、语义门控或特征分解等技术缓解这一问题，但均未从注意力机制层面直接解决重路由问题。论文旨在提出一种既能保持强大引导能力，又能保护模型核心效用的新方法。

## 2. 论文提出的方法论：SKOP

### 核心思想

**Steering via Key-Orthogonal Projections (SKOP)** 的核心思想是：不追求完全消除注意力重路由（这会完全消除steering效能），而是选择性约束那些将注意力从focus tokens重路由到tail tokens的有害成分，同时保留其他对引导有效的steering分量。

### 关键技术细节

**（1）识别Focus Sets与Tail Sets**

- 在utility calibration set上，对每个attention head计算baseline attention distribution
- Focus set H(ℓ,h)：包含捕获τ_high（如80%）注意力质量的最少token集合
- Tail set L(ℓ,h)：剩余token的补集

**（2）构建Key-Difference Covariance Matrix**

- 定义focus-tail token之间的key差异向量：Δk = k_i - k_j
- 计算其二阶矩矩阵：Σ^(Δk) = E[Δk · Δk^T]
- 该矩阵编码了哪些key方向会驱动focus-to-tail重路由

**（3）SKOP Projector构建**

- 对Σ^(Δk)进行特征分解，保留累积能量占比γ_energy的特征向量
- 构建正交投影矩阵：P^(Δk) = I - U·U^T
- 将steering vector投影到与有害重路由方向正交的子空间

**（4）选择性应用于High-Risk Heads**

- 通过Rayleigh quotient量化每个head的风险分数R(ℓ,h)
- 仅对top-k风险heads应用投影，其他heads保持原样
- 实验发现k≈20%的heads即可覆盖主要风险

### 算法流程

1. **离线校准阶段**：
   - 在utility数据集上运行单次前向传播
   - 计算每个head的focus/tail sets和Σ^(Δk)
   - 确定投影维度和风险分数

2. **在线推理阶段**：
   - 对高风险heads使用投影后的steering vector
   - 对其他heads使用原始steering vector

## 8. 不足与局限

### 方法局限

1. **仅分析query-space steering**：论文专注于mean-difference vectors in query space，未扩展到残差空间steering（那里query/key/value/MLP同时扰动，无干净的重路由项可校正）

2. **需要校准集**：虽然仅需250+样本，但获取多样化的utility数据仍是工程负担

3. **Steering效能的少量损失**：SKOP保留>95%效能，但仍有~5%的绝对损失

### 实验局限

1. **模型规模有限**：仅测试8B和9B模型，未验证在更大模型（如70B+）上的效果

2. **任务类型有限**：主要关注"行为引导"类任务，未覆盖代码生成、多语言等其他场景
