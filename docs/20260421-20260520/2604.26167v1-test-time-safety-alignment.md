## Abstract
Recent work has shown that a model's input word embeddings can serve as effective control variables for steering its behavior toward outputs that satisfy desired properties. However, this has only been demonstrated for pretrained text-completion models on the relatively simple objective of reducing surface-level profanity in short continuations. A natural and practically important question is how well input embeddings can control aligned models, which produce an imbalanced bimodal refuse-or-comply output distribution rather than the smooth distribution characteristic of open-ended generation. We explore this in the context of safety, showing that input word embeddings can be optimized in a sub-lexical manner to minimize the semantic harmfulness of aligned model responses. Our approach uses zeroth-order gradient estimation of a black-box text-moderation API with respect to the input embeddings, and then applies gradient descent on these embeddings to minimize the harmfulness of the generated text. Experiments show that the proposed method can neutralize every safety-flagged response on standard safety benchmarks.

### 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：大语言模型经过 RLHF、DPO 等安全训练后能拒绝有害请求，但对抗性提示（jailbreak prompts）可通过角色扮演、假设性叙事、结构性混淆等手段绕过安全机制。这是因为对齐训练基于有限样本，而输入空间近乎无限。

- **现有方法的局限**：当前测试时防御（prompt 级策略、随机平滑、guard 模型、激活 steering）存在结构性限制——或施加无向扰动于离散 token，或需要离线训练的辅助模块，或依赖静态校准集的固定干预。它们均未直接在模型的输入层面优化连续安全目标。

- **核心问题**：输入词嵌入能否控制已对齐模型的行为？已对齐模型产生的是"拒绝-服从"的双峰分布，而非开放生成中的平滑分布；危害概念是语义层面的（暴力、自残、非法活动），而非词汇层面的（脏话）；攻击提示专门设计用于规避安全训练——此前仅在预训练模型上针对表面级毒性的简单任务验证过该思路。

- **研究贡献**：将词嵌入级控制扩展到对齐模型的安全防护，证明该方法可消除几乎所有安全标记响应。

### 2. 论文提出的方法论

#### 核心思想

- **核心思路**：将模型的输入词嵌入矩阵作为可优化变量，利用黑盒文本审核 API 对嵌入进行零阶梯度估计，沿估计梯度下降以最小化生成文本的危害性。全程无需模型再训练、无需辅助模型、将语言模型视为黑盒（仅需访问输入词嵌入）。

#### 关键技术细节

1. **目标函数**：
   - 设 $f$ 为已对齐的自回归语言模型，$X \in \mathbb{R}^{T \times d}$ 为提示的嵌入矩阵（$T$ 个 token，每个 $d$ 维）
   - 使用 OpenAI Moderation API 作为黑盒 oracle $h$，返回 13 个伤害类别的 [0, 1] 分数，取最大值 $h(y) = \max_c s_c(y)$ 作为危害度
   - 目标：最小化 $\Phi(X) := h(f(X))$

2. **零阶梯度估计**：
   - 对嵌入施加 i.i.d. 高斯噪声 $U \sim \mathcal{N}(0, I_d)$，定义高斯平滑目标 $\Phi_\mu(X) = \mathbb{E}_U[\Phi(X + \mu U)]$
   - 根据 Nesterov & Spokoiny (2017) 的关键结果，平滑目标的梯度可表为方向有限差分：
     $$\nabla \Phi_\mu(X) = \mathbb{E}_U \left[ \frac{\Phi(X + \mu U) - \Phi(X)}{\mu} U \right]$$
   - 用 Monte Carlo 平均逼近：$g_k = \frac{1}{N} \sum_{i=1}^{N} \frac{\Phi(X_k + \mu U_i) - \Phi(X_k)}{\mu} U_i \approx \nabla \Phi_\mu(X_k)$

3. **正则化机制**：
   - **梯度归一化**：将梯度估计除以 $\ell_2$ 范数，确保步长由单一学习率 $\eta$ 统一控制
   - **余弦相似度约束**：限制更新后嵌入与原始嵌入的方向保持一定相似度（余弦相似度 $\geq \kappa$），超出则投影回余弦球边界

4. **算法流程**（TSA）：
   - 初始化嵌入 $X_0$ → 生成响应并评分 → 若分数 < 阈值（0.1）则提前停止
   - 否则采样 $N$ 个高斯扰动，计算零阶梯度估计 → 归一化 → 步进更新 → 余弦约束 → 迭代

### 6. 论文的主要结论与发现

1. **消除安全标记响应**：TSA 在 5 个模型中的 4 个上，几乎消除了所有基准上的安全标记完成（flagged = 0）；Llama 3.1 在一次梯度步内即收敛。

2. **不引入拒绝偏置**：在对抗性良性 split 上，已安全响应保持完整（moderation 分数不增加），说明方法不会强制产生" blanket refusal"。

3. **子词级扰动有效**：优化后的嵌入始终解码回原始 token（子词级扰动），但能可靠地将模型输出引导至安全行为——说明嵌入层在离散 token 粒度之下仍存在可操控的行为自由度。

4. **Gemma 3 的局限**：因其安全训练较薄弱和嵌入维度较小（1152），约 59–278 条有害提示无法恢复。

5. **计算效率**：通常在 1–2 步梯度步内收敛；oracle 延迟而非模型推理是主要运行时约束（占约 20%）。

6. **不对称性**：将优化方向反转为最大化危害无法产生有意义的有害输出——这间接证明安全训练已在对齐行为的邻域内构建了"几何上不友好"的拒绝盆。

---

### 7. 优点

- **黑盒零阶优化**：无需反向传播、无需访问模型内部，仅需输入嵌入——可作为"即插即用"安全层部署于任何开源模型。
- **无辅助模块与再训练**：不依赖校准集、辅助模型或离线训练，方法简洁独立。
- **子词级控制有效**：揭示了嵌入空间在 token 粒度之下存在行为控制表面，挑战了"嵌入仅为离散 token 查表"的传统认知。
- **多模型验证**：跨越 1B–20B 多个规模、多个开发者，证明方法的通用性。
- **自适应性**：实例级优化——对每个提示独立寻找安全方向，而非统一固定的干预。

---

### 8. 不足与局限

- **推理延迟**：每提示需数秒（1–2 步优化），oracle 网络往返延迟是主要瓶颈；对于极高吞吐场景可能有延迟负担。
- **依赖嵌入访问**：不支持纯 API 封装的闭源服务（无法获取嵌入层）。
- **安全对齐质量依赖**：方法放大而非替代现有安全对齐；若模型本身对齐薄弱（如 Gemma 3），效果显著下降。
- **最大 harm 类别的稳定性**：理论上最大操作不可微文中也提及，实际中最高 harm 类沿优化路径保持稳定，但这依赖于特定模型的行为性质。
- **未探索生成完整序列**：实验固定生成 128 tokens，若更长序列可能引入新的安全隐患。
