## Abstract
As large language model (LLM)-based multi-agent systems scale to handle increasingly complex tasks, balancing structural stability and dynamic adaptability becomes increasingly challenging. Existing systems typically adopt either structure-centric methods, committing to structures determined upfront that limit fine-grained control, or orchestration-centric methods, adapting decisions dynamically while leaving coordination structure implicit and unstable. To address this challenge, we revisit multi-agent coordination from a probabilistic perspective, casting it as posterior inference over the joint distribution of structure and orchestration. We introduce MACA, an automated coordination framework that learns a task- and budget-conditioned structural prior over agent participation and interactions. This prior guides a policy-based orchestration as an approximation to posterior inference, enabling efficient solutions with fine-grained control. Across benchmarks, MACA outperforms adaptive multi-agent baselines by an average of 8.42% while using 43.19% fewer tokens. Further investigation reveals that joint adaptation of structure and orchestration suppresses redundant interactions, converging coordination toward task-effective execution.

# 论文详细总结：Multi-Agent Coordination Adaptation via Structure-Guided Orchestration

## 1. 论文的核心问题与整体含义（研究动机和背景）

### 研究背景
- 大型语言模型（LLM）驱动的多智能体系统正在成为处理复杂任务的重要范式
- 系统需要在"结构稳定性"与"动态适应性"之间取得平衡

### 核心问题
现有方法存在两种极端倾向：
- **结构中心方法（Structure-Centric）**：预先确定拓扑结构，限制细粒度控制，难以适应任务状态演化
- **编排中心方法（Orchestration-Centric）**：动态适应但协调结构隐式且不稳定，随系统规模增长导致高方差协调

### 关键洞察
- 协调常被当作单一、整体的过程处理，结构与编排的分离导致系统无法联合推理协调的组织方式和时序演化
- 资源预算（如token限制）不仅是外部约束，还从根本上塑造了可行协调空间及其内在性能权衡

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
将多智能体协调重新表述为**后验推理问题**，其中结构先验定义约束的交互空间，编排策略在其内优化。

### 关键技术：MACA框架

**A. 结构先验学习（Structural Prior Learning）**
构建GraphSpec，包含两个组件：

1. **代理相关性评分（Agent Relevance Scoring）**
   - 建模每个代理与任务-预算上下文的语义兼容性
   - 公式：`qi = σ(si / β(b))`，其中`si = cos(e(x,b), ei)`为余弦相似度
   - 通过阈值γ过滤低置信度代理，保留参与强度`zi`

2. **交互可能性估计（Interaction Plausibility Estimation）**
   - 建模边级别的有向代理转换先验`vi → vj`
   - 使用伯努利分布采样边：`eij ~ Bernoulli(σ(ℓij))`
   - 通过MLP网络学习交互概率：`P(vi→vj|x) = MLPψ([ei, ej, e(x)])`

3. **GraphSpec构建**
   - 综合代理相关性和交互可能性：`p̃ij = pij · qj`
   - 最终得到`GS(x,b) = (Zprior, Pprior)`

**B. Token感知编排（Token-Aware Orchestration）**
- 形式化为MDP问题，使用GRPO方法优化策略
- 动作空间：`A = O ∪ {STOP}`
- 奖励函数：`r't = rt - λ·DKL(πθ∥πmix)`，平衡任务效用与token成本
- 混合监督机制：硬约束（结构掩码）+ 软正则化（先验参考分布）

### 优化目标
最大化证据下界（ELBO）：
```
max_θ E_τ~πθ [ U(τ, y*) + α·Ω(πθ|G, x, b) ]
```

## 6. 论文的主要结论与发现

### 核心结论
1. **性能提升**：MACA在六个基准上实现**68.62%平均准确率**，比自适应多智能体基线提升**8.42%**
2. **成本节约**：同时减少**43.19%的token消耗**
3. **跨尺度有效性**：Llama-3.1-8B规模下的MACA可超越Vanilla Llama-3.1-70B（如GSM-Hard：50.30% vs 48.87%）

### 关键发现
- **协调本身可诱导推理能力**：通过多样化代理间的协调缓解单模型自我强化错误循环
- **结构先验重塑编排**：GraphSpec使转换概率分布从弥散（0.37）聚焦到主导路径（0.71-0.72）
- **层次协调自然涌现**：Router/Designer → Parser/Analyzer → Solver/Checker的分层结构自动形成
- **自纠正机制**：Solver→Checker和Router→Checker的高转换概率（0.30, 0.27）验证了迭代反馈模式

## 7. 优点

### 方法设计亮点
1. **首个将多智能体优化表述为后验推理的工作**，将自适应协调从架构选择提升到原则性概率公式
2. **GraphSpec创新**：联合建模代理相关性+交互可能性，产生任务-预算条件的结构先验
3. **混合监督机制**：结合硬约束（结构掩码）与软正则化（先验KL惩罚），平衡探索与利用
4. **Token感知设计**：奖励函数显式建模token成本，实现效率-性能权衡

### 实验设计亮点
1. **广泛覆盖**：6个基准×3个领域，多模型规模验证
2. **公平比较**：统一实验协议，三次独立运行取平均
3. **深度消融**：验证结构先验各组件的独立贡献
4. **机制可解释**：分析协调模式揭示自纠正和层次协调的涌现行为

## 8. 不足与局限

### 实验覆盖局限
- 评估主要基于**相对结构化的基准**，对开放域、交互式或领域特定设置的泛化性尚未完全验证

### 方法依赖性
- 依赖**预定义代理池**和手动指定的能力/角色描述，部分有效性可能来自仔细的代理初始化而非协调本身
- 代理池需要针对不同任务家族专门设计，通用性受限

### 训练数据偏差风险
- GraphSpec从**过滤高质量轨迹**训练，监督信号来自最终结果质量
- 可能偏向**易于验证的协调模式**，难以捕捉有用但非即时 rewarded的交互
- 在credit assignment有噪声、中间协作重要、或成功无法被最终答案清晰反映的场景中鲁棒性可能下降

### 其他限制
- 额外训练成本（15-30小时/数据集），需权衡一次性投入与后续收益
- 参数敏感性（如γ和λ）需要针对不同场景调优
