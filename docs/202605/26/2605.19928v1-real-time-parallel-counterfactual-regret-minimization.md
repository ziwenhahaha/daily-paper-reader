## Abstract
Counterfactual Regret Minimization (CFR) is the dominant algorithmic family for solving large imperfect-information games, underpinning breakthroughs such as Libratus and Pluribus in No-Limit Texas Hold'em poker. In real-time game-playing systems, the solver must compute a near-equilibrium strategy within a strict time budget of only a few seconds per decision, and the number of CFR iterations completed in this window directly determines play strength. We present \textbf{Parallel CFR}, the first parallelization framework for real-time depth-limited CFR solving that seamlessly integrates pruning, abstraction, and advanced CFR variants. We decompose each CFR iteration into a pipeline of seven stages and identify two orthogonal dimensions of parallelism: \emph{by information set} and \emph{by tree node}. Leaf node evaluation is offloaded to GPUs via batched neural network inference, creating a heterogeneous CPU--GPU pipeline. Experiments on Heads-Up No-Limit Texas Hold'em demonstrate that Parallel CFR achieves $3.3$--$3.4\times$ speedup over the single-threaded baseline on postflop streets, with per-iteration time of ${\sim}47$--$54$~ms on a depth-limited game tree with over $1$ billion histories. All experiments run on a single desktop-class device (NVIDIA DGX Spark), enabling hundreds of CFR iterations within a typical real-time decision budget without requiring datacenter-scale infrastructure.

## 2. 论文提出的方法论

### 核心思想
- 提出 **Parallel CFR**，首个实时深度限制CFR并行化框架
- 将每轮CFR迭代分解为**七阶段流水线**
- 识别**两个正交的并行维度**：按信息集（∥ information set）和按树节点（∥ node）
- 创建**异构CPU-GPU流水线**：叶子节点评估卸载到GPU进行批量神经网络推理

### 七阶段流水线设计

| 阶段 | 操作 | 并行维度 | 数据流向 |
|------|------|----------|----------|
| S1: ForwardProfile | 前向分布传播 | ∥ 信息集 | σ→πᵢ, π⁻ⁱ |
| S2: AggregateProbability | 概率聚合 | ∥ 节点×卡牌 | πᵢ→Psum, x_leaf |
| S3: OpponentReach | 对手触及概率 | ∥ 节点→∥ 信息集 | Psum → π⁻ⁱ(I) |
| S4: ShowdownEquity | showdown权益 | ∥ 节点 | π⁻ⁱ(I)→v_sd |
| S5: GPULeafEval | GPU叶子评估 | GPU批量 | x_leaf → v̂_leaf |
| S6: BackwardCFV | 反事实值反向传播 | ∥ 节点+信息集 | v̂_leaf, v_sd → vᵢ(I) |
| S7: UpdateRegret | 遗憾更新 | ∥ 信息集 | vᵢ(I)→Rᵢ(I,a) |

### 关键技术创新

1. **前向传递（S1-S2）**：在信息集级别而非历史级别操作，链内顺序传播、链间并行处理

2. **并行中段（S3-S5）**：S3/S4（CPU）和S5（GPU）无数据依赖，可并行执行；实践中GPU时间被更长的CPU分支完全掩盖

3. **对手触及概率计算（S3）**：使用包含-排斥公式通过三层聚合（总触及、单卡触及、单手触及）实现O(1)卡牌阻塞修正

4. **Showdown权益（S4）**：利用扑克手牌排名的全序性质，将每节点复杂度从O(n²)降至O(n)

5. **GPU批量叶子评估（S5）**：单次GPU前向传递评估所有非终止叶子节点，减少kernel启动开销

6. **与剪枝和抽象的集成**：EVPA剪枝作为预处理步骤，乘法加速所有流水线阶段；支持无损和有损信息抽象

### 正确性保证
- 流水线产生与串行CFR实现**数值相同**的结果
- 无近似、采样或舍入误差引入
- 仅有的精度损失来自可选的有损信息抽象本身

## 6. 论文的主要结论与发现

### 核心结论
1. **加速效果**：Parallel CFR在翻牌后街道实现3.3-3.4×加速（5线程+1GPU）
2. **绝对性能**：在超10亿历史节点的深度限制游戏树上，每迭代仅需47-54ms
3. **收敛优势**：同等墙上时间下，比PokerRL达到约7×更低可利用性
4. **硬件可行性**：可在单桌面级设备上完成数百次CFR迭代，实现实时求解

### 关键发现
- **流水线瓶颈**：S6（反向CFV传播）和S3（对手触及计算）占CPU时间超过50%
- **扩展性特点**：亚线性加速（5线程约3.3×）因GPU叶子评估未CPU并行化，内存带宽成为瓶颈
- **抽象化效果**：桶数从169降至50时，总迭代时间进一步降低约3×
- **规模效应**：游戏树规模增大时，并行效率提升（因更好的负载均衡）

## 7. 优点

### 方法亮点
1. **计算精确性**：流水线产生与串行实现完全相同的结果，无近似误差
2. **模块化设计**：七阶段流水线各阶段接口清晰，CFR变体仅影响S7
3. **异构协同**：CPU-GPU流水线无缝集成，各自执行最擅长的计算任务
4. **正交并行维度**：信息集级和节点级并行互为补充，最大化并行度
5. **乘法加速**：剪枝作为预处理，对所有后续阶段提供乘法加速
6. **理论与实践结合**：继承选定CFR变体的收敛保证

### 实验亮点
1. **实际硬件验证**：在真实桌面级硬件上验证，而非模拟环境
2. **全面性能分解**：分阶段耗时分析揭示真正的性能瓶颈
3. **公平对比**：与成熟开源求解器在相同条件下的对比
4. **多参数覆盖**：系统验证了SPR、抽象化粒度等关键参数的影响
