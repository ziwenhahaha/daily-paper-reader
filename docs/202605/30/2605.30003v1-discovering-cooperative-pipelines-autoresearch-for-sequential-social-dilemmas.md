## 摘要
我们研究两层次的自动研究以实现合作：外层AI代理自主重新设计内层LLM策略合成系统的流水线，用于多智能体的序列社会困境（SSD）。研究者智能体\mathcal{R}（以编码智能体方式运行）读取内层源代码，编辑系统提示、反馈函数、辅助库和迭代逻辑，运行评估，决定保留什么，遵循自动研究范式。在两款游戏（Cleanup和Gathering）、两种策略合成LLM、两种福利目标（功利效率和罗尔斯式最大最小）下，研究者可靠地超越手工设计的基线，大幅缩小运行间方差，并优于仅提示优化。发现的流水线依赖于目标：仅在最大最小目标下，研究者向合成流水线注入明确的公平机制，这一机制类别不在其自身的目标无关系统提示中，也不在每个效率优化流水线中。这支持一种信息设计视角，即研究者根据福利目标选择向有限理性的合成器透露什么。代码位于https://github.com/vicgalle/autoresearch-social-dilemmas。

## Abstract
We study two-level autoresearch for cooperation: an outer-loop AI agent autonomously redesigns the inner-loop pipeline of an LLM policy-synthesis system for multi-agent Sequential Social Dilemmas (SSDs). A researcher agent $\mathcal{R}$ (run as a coding agent) reads the inner-loop source code, edits system prompts, feedback functions, helper libraries, and iteration logic, runs evaluations, and decides what to keep, following the autoresearch paradigm. Across two games (Cleanup and Gathering), two policy-synthesizer LLMs, and two welfare objectives (utilitarian efficiency and Rawlsian maximin), the researcher reliably exceeds hand-designed baselines, sharply tightens run-to-run variance, and outperforms prompt-only optimization. The discovered pipelines are objective-dependent: only under maximin does the researcher inject an explicit fairness mechanism into synthesizer pipelines, a class of mechanism that is absent from its own objective-agnostic system prompt and from every efficiency-optimized pipeline. This supports an information-design reading in which the researcher chooses what to reveal to the boundedly rational synthesizer as a function of the welfare objective. Code at https://github.com/vicgalle/autoresearch-social-dilemmas.

**研究动机**：
- **序列社会困境（SSDs）** 是多人囚徒困境在时间丰富的马尔可夫博弈中的扩展，个体理性行为导致集体次优结果（如污染、过度捕捞、公开冲突）

- **核心问题**：能否让 AI 智能体自主设计该合成流水线？

**核心观点**：
- 研究者智能体作为"信息设计师"，通过控制向有限理性的策略合成器透露哪些信息来实现机制设计目标
- 发现的流水线是目标依赖的——仅在最大化最小福利目标下，研究者才会注入明确的公平机制

## 2. 方法论

**核心思想**：
- **两层自动研究框架**：外层研究者智能体自主设计内层 LLM 策略合成系统的配置
- 遵循 Karpathy 的自动研究范式：编码智能体通过 git 操作修改代码库、观察结果、迭代优化

**关键技术细节**：

**配置空间**：
配置 `c = (p, φ, H, ι)` 包含：
- `p`: 系统提示（策略合成器的指令）
- `φ`: 反馈函数（如何构造反馈、诊断阈值）
- `H`: 辅助函数库（路径规划、状态查询等工具）
- `ι`: 迭代逻辑（内循环迭代次数 K、评估种子数 |S|、思考预算）

**外层循环（研究者智能体 R）**：
- 接收当前最佳配置代码、实验历史（含代码差异、分数、指标）、环境源码
- 生成代码修改提案，提交到专用 git 分支
- 评估新配置在 held-out 种子上的福利分数 J(c)
- 遵循保留规则（`Jj > J* + τ`）决定是否接受修改

**内层循环（策略合成）**：
- 冻结的 LLM M 根据系统提示 p 和反馈 Fk 生成 Python 策略函数
- 所有 N 个智能体执行同一程序（通过 agent_id 参数实现条件行为）
- 多种子评估后计算指标向量 m_k = (U, E, S, P)
- 反馈函数 φ 构造下一轮提示

**福利目标**：
- **功利效率** Φ_U = (1/HN) Σ_i R_i
- **罗尔斯式最大最小** Φ_min = min_i R_i（优化最弱势智能体的回报）

## 6. 主要结论与发现

**核心发现**：

1. **研究者可靠超越手工设计基线**
   - Cleanup 效率从 1.93/0.86 提升至 3.1-3.2
   - Gathering 效率从 0.03-2.42 提升至 2.47-2.52
   - 运行间方差大幅缩小

2. **目标依赖性：公平机制仅在最大最小目标下出现**
   - 4/4 最大最小运行中发现了显式公平机制（时间轮换）
   - 0/4 效率运行中发现该机制
   - 这验证了信息设计视角

3. **游戏结构决定策略类别**
   - Cleanup（不对称成本）：需要显式公平机制（轮换清扫职责）
   - Gathering（对称成本）：效率优化自然产生高公平性，无需单独最大最小优化

4. **效率-公平权衡取决于模型能力**
   - Gemini：最大最小仅牺牲 1% 效率（3.16 vs 3.20）
   - Sonnet：中等权衡（3.12 → 2.57）
   - Gemma：意外反转——最大最小目标反而产生更高效率（0.87 → 1.71）

5. **GEPA 对比结果**
   - 自动研究在 Cleanup 上优于 GEPA 2-3 倍
   - 在 Gathering 上优于 GEPA 约 20%
   - 差距在较弱模型上更明显

## 8. 不足与局限

**主要局限**：

1. **单一研究者 LLM**：
   - 所有实验使用同一研究者（Claude Opus 4.6）
   - 最重要的是进行研究者智能体的消融实验

2. **内层基础设施本身是设计产物**：
   - 验证管道、辅助库骨架、编排器 API 都是工程化产物
   - 对无等价脚手架环境的泛化未评估

3. **网格世界范围**：
   - 2D 网格世界（完全观测、离散动作、短episode）
   - 发现的特定机制（Voronoi 划分、时间轮换）可能无法迁移到高维、连续或部分可观测设置

4. **实验规模限制**：
   - Gemma 实验仅单次运行
   - 未探索对抗性目标设置
   - 对异构程序（不同智能体运行不同代码）未涉及

5. **Spec-gaming 风险**：
   - 配置空间无约束，理论上可暴露模拟器内部或硬编码动作
   - 检查未发现此类行为，但需持续警惕
