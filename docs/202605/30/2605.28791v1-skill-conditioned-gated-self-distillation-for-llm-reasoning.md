## Abstract
On-policy self-distillation (SD) improves LLM reasoning by using teacher-side privileged information (PI) to turn sparse verifier outcomes into dense token-level supervision. Existing methods usually assume trusted PI, such as reference answers or successful traces. We ask whether PI can instead come from an experience-derived skill bank, where retrieved skills are compact and reusable but may also be irrelevant or misleading. We propose Skill-Conditioned Gated Self-Distillation (SGSD), which formulates skill-based SD as teacher hypothesis validation rather than unconditional imitation. SGSD retrieves skill-mistake pairs, constructs a multi-teacher pool, and lets all skill-conditioned teachers score the same plain-prompt student rollout. The verifier validates each teacher's polarity: supporting a success or suppressing a failure gives positive supervision, while the opposite stance is reversed. A robust gated objective then distills informative teacher-student disagreements while suppressing uncertain or extreme signals. Experiments on multiple mathematical reasoning benchmarks show that SGSD consistently improves over GRPO and remains competitive with answer-conditioned OPSD under a weaker PI assumption. For example, on Qwen3-1.7B, SGSD outperforms GRPO by 6.2% and OPSD by 1.7% on average on AIME24, AIME25, and HMMT25. Our code is available at https://github.com/walawalagoose/SGSD.

# 论文总结：Skill-Conditioned Gated Self-Distillation for LLM Reasoning

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：强化学习与可验证奖励（RLVR）已成为提升大语言模型（LLM）推理能力的主流后训练范式（如 GRPO）。其核心弱点是奖励信号稀疏、延迟且在整个轨迹上几乎均匀分布。
- **已有方法局限**：on-policy 自蒸馏（SD）通过教师端的特权信息（PI）将稀疏的验证器结果转化为密集的 token 级监督。但现有方法假设 PI 是可信的（如参考答案、成功轨迹），需要外部提供的 ground truth。
- **核心问题**：能否用**经验衍生的技能库**替代人工标注的特权信息？技能检索虽然紧凑、可复用，但可能不相关、过时甚至误导。简单的无条件模仿会导致错误知识传播。
- **研究目标**：在**弱 PI 假设**下（即仅依赖验证器反馈和可能不可信的技能），实现可靠的在线自蒸馏训练。

## 2. 方法论：核心思想与技术细节

### 核心思想
将技能自蒸馏建模为**教师假设验证**，而非无条件模仿。核心假设：技能教师的价值是**结果依赖**的——支持成功 rollout 或抑制失败 rollout 的教师是有益的，反之是有害的。

### 关键技术组件

#### （1）结构化技能库构建
- 技能库 $B = \{G, E\}$，包含：
  - **G**（通用技能）：编码可复用的推理原则，带有标题、使用条件。
  - **E**（常见错误模式）：描述错误类型、成因、避免方法。
- **冷启动**：从训练集采样问题，用初始策略生成轨迹，成功轨迹提取通用技能，失败轨迹提取错误模式，再通过分层合并去除冗余。
- **动态更新**：训练过程中定期从最近的成功/失败 rollout 中提取新技能候选，合并入技能库，并维护容量上限。

#### （2）技能条件多教师池
- 给定问题 $x$，从技能库检索 top-K 个技能-错误对 $\{(g_k, e_k)\}$。
- 每个 pair 拼接问题构成教师上下文：$c_k(x) = g_k \oplus e_k \oplus x$。
- 教师分布：$p_T^{(k)}(y_t | x, y_{<t}) = \bar{\pi}_\theta(y_t | c_k(x), y_{<t})$（stop-gradient）。
- 学生分布：$p_S(y_t | x, y_{<t}) = \pi_\theta(y_t | x, y_{<t})$（无技能注入）。
- 教师权重 $\alpha_k(x)$ 由检索相似度归一化得到。

#### （3）结果验证的教师极性
- 计算 token 级 log-probability 差距：$\Delta_t^{(k)} = \log p_T^{(k)}(y_t | x, y_{<t}) - \log p_S(y_t | x, y_{<t})$。
- 聚合为教师级支持分数：$a_k = \frac{1}{T} \sum_t \Delta_t^{(k)}$（正=支持，负=抑制）。
- 极性判定规则（由验证器 outcome $r$ 验证）：

| 情况 | 条件 | 教师类型 |
|------|------|----------|
| 蒸馏 | $r>0, a_k>0$ 或 $r<0, a_k<0$ | 有帮助的 / 警示型教师 |
| 反转 | $r>0, a_k<0$ 或 $r<0, a_k>0$ | 过度抑制 / 误导型教师 |

#### （4）鲁棒支持估计
- **Token 掩码**：过滤特殊 token、模板 token、格式 token，保留有效推理位置。
- **差距裁剪**：$\tilde{\Delta}_t^{(k)} = \text{clip}(\Delta_t^{(k)}, -c_\Delta, c_\Delta)$，防止极端值扭曲立场。
- **置信阈值**：当 $|\tilde{a}_k| \leq \epsilon_a$ 时判定为不确定，忽略该教师。

#### （5）门控蒸馏目标
- 门控损失函数：$\ell_{\text{gate}}(\Delta) = \log 2 - \log(1 + e^{-\Delta^2 / (2\tau_g)})$。
- **特性**：
  - near-zero gap：损失接近 0，几乎不更新。
  - medium gap：损失增长最快，是**信息丰富区域**。
  - extreme gap：指数衰减饱和，防止极端差异主导优化。
- 整体目标：$L_{\text{SGSD}}(x) = \sum_{k=1}^{K_x} \alpha_k(x) \rho_k \bar{\ell}^{(k)}$，其中 $\rho_k$ 控制更新方向（+1=蒸馏，-1=反转，0=忽略）。

#### （6）理论性质
- 梯度形式 $g_\tau(\Delta) = \frac{\Delta}{\tau_g(1 + e^{\Delta^2/(2\tau_g)})}$ 具有：
  - **符号一致性**：$\text{sign}(g_\tau(\Delta)) = \text{sign}(\Delta)$，与极性结合后确保正确方向更新。
  - **有界性**：$|g_\tau(\Delta)| \leq \frac{1}{\sqrt{e}\tau_g}$，防止极端 gap 主导。
  - **局部 reverse-KL 近似**：在 gap 小的区域，行为类似 reverse-KL 的二阶展开。

## 6. 主要结论与发现

### 核心发现
1. **技能注入本身不充分**：Base+Skill、GRPO+Skill、OPSD+Skill 等直接暴露技能的方法表现不佳，说明简单地将技能放入学生上下文或教师上下文不能带来稳定提升，反而可能让模型依赖技能而非内化推理能力。

2. **SGSD 在弱 PI 假设下表现优异**：
   - Qwen3-1.7B：平均分从 37.4% → 43.7%，领先 GRPO 6.2%，领先 OPSD 1.7%。
   - Qwen3-4B：平均分 64.3%，领先 GRPO 1.6%，领先 OPSD 0.8%。
   - Qwen3-8B：平均分 65.5%，略低于 OPSD（66.8%），但大幅领先 GRPO（65.0%）。

3. **训练稳定性**：GRPO 提升有限且不稳定；OPSD 在中后期出现性能退化；SGSD 在整个训练过程中保持稳定提升，尤其在后期阶段优势明显。

4. **关键组件验证**：
   - 极性机制有效，但移除后会在后期崩溃。
   - 多教师池优于单教师。
   - 全词汇蒸馏优于 Top-K 截断。
   - 门控目标在各项消融中均表现最佳。
   - live（同步）教师策略表现最优，说明极性和鲁棒估计能稳定自蒸馏过程。

5. **答案 PI 对 SGSD 有害**：将参考答案同时加入技能条件教师上下文会导致训练崩溃，说明 SGSD 的优势来自**可复用的技能 PI**，而非简单地增加教师端信息量。

## 7. 优点

1. **问题定义精准**：将技能蒸馏重新建模为"教师假设验证"，而非传统的"条件模仿"，抓住了技能检索噪声的核心挑战。
2. **设计精巧且有理论支撑**：极性机制（基于 verifier outcome）、鲁棒支持估计（掩码+裁剪+阈值）、门控损失形成了一个完整的去噪蒸馏框架，且有梯度形式的理论分析。
3. **弱 PI 假设下的竞争力**：在不使用参考答案的情况下，SGSD 达到了与 OPSD 相当的性能，证明了经验衍生技能在数学推理训练中的可行性。
4. **工程实现完整**：提供了详细的技能库构造流程、冷启动提取、在线更新、合并算法，以及 prompt 模板，便于复现。
5. **多维度消融验证**：实验覆盖了方法的核心假设和组件，论证较为系统。

## 8. 不足与局限

1. **任务范围有限**：主要在**数学推理**任务上验证，扩展到开放式生成、多步规划等需要更复杂的 outcome 信号（如人工评判或任务完成检测）。
2. **技能库构建依赖初始模型**：冷启动阶段需要用初始策略生成轨迹，如果初始策略能力太弱，提取的技能质量受限。
3. **容量与维护开销**：技能库需要定期合并、更新、裁剪，动态管理增加了工程复杂度。
4. **模型规模覆盖不均**：消融实验集中于 1.7B 模型，大模型（4B、8B）的组件级消融不够充分，可能存在规模相关的特性未被揭示。
5. **模型选择策略**：采用"记录所有 checkpoint，取最佳"的策略，可能导致对训练动态的乐观估计。更保守的做法是使用 held-out 验证集做早停。
6. **评估指标单一**：仅使用 pass@12 准确率，未报告其他指标（如多样性、推理步骤长度、错误类型分布等），难以全面评估推理质量。
7. **极性判定依赖 verifier 准确性**：如果 verifier 本身有误（如某些边界情况的符号判断），极性机制可能引入错误监督。
8. **技能-错误配对方式简单**：采用 rank-wise 配对（top-k skills 配 top-k mistakes），可能不是最优匹配策略。
