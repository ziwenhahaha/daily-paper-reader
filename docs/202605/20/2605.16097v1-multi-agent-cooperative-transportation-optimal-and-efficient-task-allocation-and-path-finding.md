## 摘要
多机器人系统是现代物流的核心，但它们的能力通常局限于可由单个智能体执行的任务。本文解决了现有框架（如多智能体路径规划MAPF和任务分配及路径规划TAPF）中的一个关键空白——缺乏对需要多个智能体协作运输大型物品的真实协作的支持。为此，我们形式化了协作运输任务分配及路径规划（CT-TAPF）问题，整合了团队组建、任务分配和无碰撞路径规划。我们提出了一种最优求解器——协作运输任务冲突搜索（CT-TCBS），它采用新颖的增量扩展策略来解决团队组建中固有的组合爆炸问题。认识到最优性的计算成本，我们也开发了一族次优求解器，采用全局的、以任务为中心的视角，根据全局难度指标（最佳任务或最差任务）选择下一个要分配的任务。我们的综合实证评估展示了三个关键发现：（1）增量扩展策略通过成功剪枝主要的任务分配搜索空间显著优于朴素的组合方法；（2）我们识别了一个任务-冲突扩展困境，其中对大型智能体路径规划子问题有效的复杂冲突解决器可能在集成的CT-TAPF设置中是有害的；（3）我们提出的次优求解器在解质量-运行时间谱上相对于"nn-"智能体中心基线建立了新的、更有效的前沿。这项工作为协作多智能体问题的这一新实用类别提供了一个基础框架和一套有效算法。

## Abstract
Multi-robot systems are integral to modern logistics, but their capabilities are often limited to tasks executable by individual agents. This paper addresses a critical gap in existing frameworks like Multi-Agent Path Finding (MAPF) and Task Allocation and Path Finding (TAPF), which lack true cooperation for transporting large items that require multiple agents. To this end, we formalise the Cooperative Transportation Task Allocation and Path Finding (CT-TAPF) problem, which integrates team formation, task assignment, and collision-free pathfinding. We present an optimal solver, Cooperative Transportation Task Conflict-Based Search (CT-TCBS), which features a novel Incremental Expansion strategy to tackle the combinatorial explosion inherent in team formation. Recognising the computational cost of optimality, we also develop a family of sub-optimal solvers that employ a global, task-centric perspective, selecting the next task to assign based on a global difficulty metric (Best Task or Worst Task). Our comprehensive empirical evaluation demonstrates three key findings: (1) the incremental expansion strategy significantly outperforms the naive combinatorial approach by successfully pruning the dominant task-allocation search space; (2) we identify a task-conflict expansion dilemma, where sophisticated conflict resolvers effective for large-agent pathfinding subproblems can be detrimental in the integrated CT-TAPF setting; and (3) our proposed sub-optimal solvers establish a new, more efficient frontier on the solution quality-runtime spectrum compared to "nn-" agent-centric baselines. This work provides a foundational framework and a set of effective algorithms for a new, practical class of cooperative multi-agent problems.

- **核心问题**：如何将团队组建（team formation）、任务分配（task assignment）和碰撞无关路径规划（collision-free pathfinding）三者统一到一个框架中，并在满足几何约束的前提下找到最优解。
- **研究目标**：提出一种新的形式化问题——**Cooperative Transportation Task Allocation and Path Finding（CT-TAPF）**，并设计高效的最优和次优求解器。

## 2. 论文提出的方法论

### 2.1 问题形式化

- **环境**：无向二维图 $G = (V, E)$，$n$ 个智能体各有初始位置 $v_0^i \in V$，$m$ 个协作任务集合 $T = \{\tau_1, ..., \tau_m\}$。
- **任务定义**：每个任务 $\tau_i$ 需要 $k_i$ 个智能体，具有起始配置 $V_S^i$（$k_i$ 个不同的任务槽位）和目标配置 $V_G^i$，两者均为连通子图。
- **任务执行两阶段**：
  1. **Assembly Phase**：各智能体独立从当前位置移动到各自的任务槽位 $V_S^i$（此时槽位未被预留，其他智能体可通行）。
  2. **Convoy Phase**：最后一个智能体到达后同步，团队转化为一个"Convoy"（概念上等价于 LA-MAPF 中的 Large Agent），以统一形状 $S_i$ 和参考点 $r$ 联合移动至目标配置 $V_G^i$；任务完成后 Convoy 立即解散。
- **冲突定义**：采用**广义顶点冲突**（generalized vertex conflict）。对于两个实体 $E_i$ 和 $E_j$ 在时刻 $t$ 的位置 $u$ 和 $v$，若它们的占位足迹 $F(E_i, u, t)$ 与 $F(E_j, v, t)$ 交集非空，则发生冲突。冲突可发生在两个单体智能体、单体与 Convoy、以及两个 Convoy 之间。
- **优化目标**：最小化 Sum of Costs（SoC），即所有智能体的总操作时间。

### 2.2 核心算法——CT-TCBS（Cooperative Transportation Task Conflict-Based Search）

CT-TCBS 是一个**双层搜索算法**，上层在约束树上执行 A* 搜索，下层计算最优路径。

- **上层搜索（A* on Constraint Tree）**：
  - 每个节点 $N$ 由当前任务分配状态 TA 和约束集 CS 定义。
  - 代价函数 $f(N) = g(N) + h(N)$，其中 $g(N)$ 为已完成任务的真实最优成本（由下层计算），$h(N)$ 为未完成任务的 admissible 估计（仅使用 $H_1 + H_2$，不包含 $H_3$）。
  - 搜索顺序遵循严格优先级：**冲突扩展优先于任务扩展**。

- **增量扩展（Incremental Expansion）——核心贡献**：
  - **Naive 方法（Combinatorial Expansion）**：一次性为多智能体任务枚举所有 $k!$ 种团队排列，产生组合爆炸。
  - **增量策略**：仅在已有部分分配的任务上进行 Partial Task Completion Expansion（为下一个开放槽位分配一个可用智能体），直到该任务满员后才进行 Assign New Task Expansion（选择新任务并仅分配其第一个槽位）。
  - 大幅降低每步的分支因子，将搜索空间聚焦于任务分配这一主导瓶颈。

- **冲突扩展（Conflict Expansion）**：
  - 采用 MC-CBS 框架处理 Convoy 碰撞，提供三种策略：ASYM（非对称）、SYM（对称）和 MAX-d（最大权重）。
  - ASYM 和 SYM 在两个子节点间分别施加约束集；MAX-d 通过 MDD lookahead 选择能最大化子节点代价增加的约束对。
  - 发现：在 CT-TAPF 整体问题中，MAX-d 的"故意抬高 g 值"策略会迫使上层 A* 搜索过早探索其他任务分配，反而导致更多扩展节点。

- **下层路径规划**：
  - 统一使用 A* 计算最优路径，对已完全分配的任务执行三步同步：①计算各智能体到槽位的独立路径与到达时间；②确定同步时刻 $t_{sync} = \max(t_j^{arr})$；③从 $t_{sync}$ 开始规划 Convoy 的联合路径。

### 2.3 次优求解器（Task Selector Layer）

为平衡求解质量与运行时间，提出**全局任务视角**的次优方法：

- **Best Task（BT）**：贪心地选择全局难度最低（最小匈牙利算法分配成本）的任务优先处理，追求快速找到高质量解。
- **Worst Task（WT）**：选择全局难度最高的任务优先处理（fail-fast 策略），尽早剪枝不可行分支。
- 难度定义为：将当前可用智能体分配给任务槽位的最小估计成本，通过匈牙利算法求解分配问题得到。

## 6. 论文的主要结论与发现

1. **增量扩展策略显著优于组合扩展**：增量策略在任务分配的搜索空间中实现了更有效的剪枝，统计显著性确认（$p_{Holm} ≈ 0.001$）。任务扩展节点与冲突扩展节点的中位数比例 $\tilde{r} = 9.54$（IQR [5.64, 20.0]），说明任务分配的计算开销是主要瓶颈，而非路径规划冲突。

2. **任务-冲突扩展困境（Task-Conflict Expansion Dilemma）**：复杂的冲突解决器（如 MAX-d）在 LA-MAPF 子问题上表现最优，但在集成 CT-TAPF 场景下反而有害。MAX-d 通过有意增大 g 值来剪枝冲突搜索树，会导致上层 A* 搜索过早放弃当前任务分配、探索更多任务空间节点，最终整体扩展节点数更多。

3. **次优求解器的性能层级**：WT 在解质量（最优性差距）上显著优于 BT（$r = 0.294$, $p ≈ 10^{-11}$），因为优先处理大型多智能体任务能更早组建团队。BT 在空间聚集场景下成功率更高，WT 在随机分布场景下更优——体现了场景敏感性。

4. **新的效率前沿**：在运行时-解质量权衡谱上，本文提出的 WT 方法相比 1-nn/2-nn 等智能体中心基线，在保持可比较成功率和显著更低解质量差距的同时，大幅缩短运行时间，为次优求解器设立了新的更高效的前沿。

## 7. 优点

- **问题原创性**：首次形式化 CT-TAPF 问题，整合了团队组建、任务分配和碰撞无关路径规划三个关键环节，填补了多智能体协作运输领域的框架空白。
- **增量扩展策略设计精准**：通过"逐步填充槽位"而非"一次性枚举团队排列"，直接命中了问题的主要计算瓶颈——任务分配搜索空间的组合爆炸。
- **理论严谨**：完整的 optimality 证明（admissible 启发式、非负步长代价、CBS 约束框架的正确性），以及通过统计检验对实验结论的严格支撑。
- **冲突解决器的深入分析**：不仅比较了求解器的性能，还揭示了 MAX-d 在集成问题中表现反常的根本原因——任务-冲突扩展困境，为后续研究提供了重要警示。
- **次优求解器的设计思路清晰**：从智能体中心（最近邻）到任务中心（BT/WT）的视角转变，解决了"单一智能体局部最优导致全局团队低效甚至死锁"的根本问题。

## 8. 不足与局限

- **仅考虑顶点冲突**：论文明确指出未建模广义的边冲突（generalized edge conflicts），理由是引入显著的计算开销。这意味着在需要严格避免边碰撞的场景中，当前框架可能不完整。
- **启发式的简化**：排除了 $H_3$（团队同步等待时间）这一组件，理由是计算 admissible 下界需要求解 NP-hard 的广义分配问题（GAP）。这导致启发式可能不够紧，搜索效率存在提升空间。
- **规模限制**：最优求解器在 8×8 网格上仅测试到 6 智能体和 8 任务；16×16 的综合测试也仅涉及 5 智能体和 15 任务。对于更大规模（如数十个智能体、数百个任务）的实际仓储场景，算法的可扩展性仍有待验证。
- **Convoy 的刚体假设**：Convoy 的形状 $S_i$ 是固定的，论文未涉及动态变形或柔性团队——这在某些实际协作搬运场景中可能是限制。
- **竞争性协作未涉及**：当前模型假设所有任务均为协作搬运，但真实场景中可能存在竞争关系（如多个团队争抢同一任务），论文未涉及此类设置。
- **对比方法覆盖有限**：次优基线中未包含其他主流方法（如 LNS2、LACAM 等），且缺乏与最新学习型/强化学习方法的可比性分析。
