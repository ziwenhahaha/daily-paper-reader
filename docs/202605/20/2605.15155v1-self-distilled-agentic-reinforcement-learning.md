## 摘要
强化学习（RL）已成为后训练LLM智能体的核心范式，但其轨迹级奖励信号对长时域交互仅提供粗粒度监督。同策略自蒸馏（OPSD）通过引入来自增强教师分支的密集token级指导来补充RL。然而，将OPSD迁移到多轮智能体被证明是有问题的：累积的多轮不稳定性破坏了监督，而技能条件的特权指导需要对负教师拒绝进行不对称处理——这可能源于不完美的技能检索或利用。我们引入了SDAR（自蒸馏智能体强化学习），它将OPSD作为门控辅助目标，同时保持RL作为主要优化主干。SDAR通过sigmoid门将解耦的token级信号映射，对教师认可的的正gap令牌强化蒸馏，对负教师拒绝进行软衰减。在ALFWorld、WebShop和Search-QA上跨Qwen2.5和Qwen3家族，SDAR在ALFWorld上比GRPO提升9.4%，在Search-QA上提升7.0%，在WebShop-Acc上提升10.2%，避免了朴素GRPO+OPSD的不稳定性，并在模型规模上始终优于混合RL-OPSD基线。

## Abstract
Reinforcement learning (RL) has emerged as a central paradigm for post-training LLM agents, yet its trajectory-level reward signal provides only coarse supervision for long-horizon interaction. On-Policy Self-Distillation (OPSD) complements RL by introducing dense token-level guidance from a teacher branch augmented with privileged context. However, transferring OPSD to multi-turn agents proves problematic: compounding multi-turn instability destabilizes supervision, while skill-conditioned privileged guidance requires asymmetric treatment for negative teacher rejections may arise from imperfect skills retrieval or utilization. We introduce SDAR (Self-Distilled Agentic Reinforcement Learning), which treats OPSD as a gated auxiliary objective while keeping RL as the primary optimization backbone. SDAR maps detached token-level signals into a sigmoid gate, strengthening distillation on teacher-endorsed positive-gap tokens and softly attenuating negative teacher rejections. Across the Qwen2.5 and Qwen3 families on ALFWorld, WebShop, and Search-QA, SDAR substantially improves over GRPO (+9.4% on ALFWorld, +7.0% on Search-QA, +10.2% on WebShop-Acc), avoids the instability of naive GRPO+OPSD, and consistently outperforms hybrid RL--OPSD baselines across model scales.

# 论文详细总结：Self-Distilled Agentic Reinforcement Learning (SDAR)

## 1. 核心问题与整体含义

### 研究动机与背景

- **大语言模型智能体的后训练挑战**：随着 LLM 智能体在动态、开放环境中的广泛应用（如 GUI 自动化、游戏、具身控制），如何在长时域交互中有效训练决策能力成为核心问题。
- **RL 的局限性**：强化学习（以 GRPO 为代表）依赖轨迹级（trajectory-level）的奖励信号，仅提供粗粒度监督，对长-horizon 交互任务而言监督信号稀疏，难以支撑稳定训练。
- **OPSD 的潜力与困境**：On-Policy Self-Distillation（OPSD）通过同一策略的增强版本（配备特权上下文）提供密集的 token 级指导。然而，直接迁移至多轮智能体训练时面临两大问题：
  1. **多轮 OPSD 不稳定性**：一旦学生智能体偏离教师支持的轨迹，累积误差导致每轮 KL 散度急剧上升，性能灾难性下降。
  2. **特权引导的不对称信任**：负 gap（教师对采样 token 赋值更低概率）可能源于技能检索质量差、技能利用不当或多轮漂移，而非 token 本身应被抑制。统计显示负 gap token 占比超过 50%，处理需谨慎。

## 2. 论文提出的方法论

### 核心思想

- **主-辅分离架构**：保持 RL 作为主要的、无偏的优化主干，将 OPSD 降格为受控的辅助目标。
- **Token 级自适应门控**：通过 sigmoid 门让每个 token 自主决定其蒸馏强度，动态实现细粒度的自步进课程（self-paced curriculum）。
- **不对称蒸馏**：对正 gap token（教师认可）强化蒸馏，对负 gap token（教师拒绝）软衰减，避免盲目跟随不可靠的负向信号。

### 关键技术细节

#### 优化目标

整体训练目标为 GRPO 损失与门控自蒸馏损失的加权组合：

$$L(\theta) = L_{GRPO}(\theta) + \lambda_{SDAR} \cdot L_{SDAR}(\theta)$$

- **GRPO 损失**：标准策略优化，基于组内相对优势计算裁剪的替代目标。
- **LSDAR 损失**：Token 级加权似然损失。

#### Token 级门控机制

对每个采样的 student token $y_t$，计算脱离的教师-学生 log 概率 gap：

$$\Delta_t = sg\left(\log \pi_\theta^+(y_t | s_t^+) - \log \pi_\theta(y_t | s_t)\right)$$

其中 $s_t^+$ 为配备特权上下文（如检索到的技能）的教师上下文，$sg(\cdot)$ 为 stop-gradient 算子。

门控函数使用 logistic sigmoid：

$$g_t = \sigma(\beta \Delta_t)$$

- $\beta > 0$ 控制锐度（sharpness），决定从保守衰减到强激活的过渡速度。
- 门控在 $(0, 1)$ 区间内光滑、可微、有界。

#### 三种门控策略

1. **熵门控**：$g_t = \sigma(\beta h_t)$，基于学生熵 $h_t = -\sum_v \pi_\theta(v|s_t)\log\pi_\theta(v|s_t)$，针对高不确定性位置。
2. **Gap 门控**（默认采用）：$g_t = \sigma(\beta \Delta_t)$，直接基于教师-学生 gap 调节。
3. **Soft-OR 门控**：$g_t = \sigma(\beta[1 - (1-h_t)(1-\Delta_t)])$，融合不确定性与 gap。

#### Token 级蒸馏损失

$$\ell_t^{SDAR} = g_t \cdot \left(\log \pi_\theta^+(y_t | s_t^+) - \log \pi_\theta(y_t | s_t)\right)$$

$$L_{SDAR} = \text{Agg}(\ell_t^{SDAR})$$

#### 技能检索策略

论文实现了四种检索质量层级：

1. **UCB 检索**：基于上置信界解决多臂老虎机问题，平衡探索与利用。
2. **关键词匹配（KM）**：直接匹配任务描述中的关键词与预定义类别标签。
3. **全检索**：使用完整检索系统。
4. **随机检索**：作为噪声基准。

## 3. 实验设计

### 数据集与 Benchmark

| Benchmark | 描述 | 评估指标 |
|-----------|------|----------|
| **ALFWorld** | 基于 ALFRED 的文本游戏，包含 6 类家庭活动（Pick, Look, Clean, Heat, Cool, Pick2），共 3,827 个任务实例 | 成功率（%） |
| **WebShop** | 真实网购场景的 Web 交互环境 | Score / Acc（128 固定验证任务） |
| **Search-QA** | 搜索增强 QA，包括单跳 QA（NQ, TriviaQA, PopQA）和多跳 QA（HotpotQA, 2Wiki, MuSiQue, Bamboogle） | 准确率（%） |

### 对比方法（三大类别）

#### 无训练方法
- **Skill-Prompt**：推理时不训练，仅在输入前拼接检索到的技能。

#### 后训练方法
- **GRPO**：纯 RL 基线。
- **OPSD**：纯自蒸馏方法。
- **Skill-GRPO / Skill-GRPO***：GRPO + 技能注入，区分推理时是否使用技能。

#### 混合方法
- **GRPO+OPSD**：简单叠加 OPSD 辅助损失。
- **Skill-SD**：GRPO + 技能条件 K3 散度蒸馏。
- **RLSD**：利用 teacher-student gap 重加权 token 级 RL 优势。

### 实验覆盖范围

- **模型规模**：Qwen2.5-3B-Instruct、Qwen2.5-7B-Instruct、Qwen3-1.7B-Instruct。
- **训练数据**：ALFWorld 使用 GiGPO 训练集划分；Search-QA 使用 NQ 和 HotpotQA 作为域内训练数据，其余作为域外评估；WebShop 使用 1,000 个任务训练。
- **消融实验**：门控策略（熵、gap、soft-OR）、$\beta$ 锐度、$\lambda$ 蒸馏系数、蒸馏目标类型（reverse KL、forward KL、JSD）。

## 6. 论文的主要结论与发现

### 核心结论

1. **显著性能提升**：SDAR 在三个 benchmark 上全面超越 GRPO：
   - ALFWorld：+9.4%（Qwen2.5-3B: 84.4% vs 75.0%）
   - Search-QA：+7.0%
   - WebShop-Acc：+10.2%（Qwen2.5-7B: 82.8% vs 72.6%）

2. **稳定训练**：避免了 naive GRPO+OPSD 的灾难性不稳定性（在 Qwen3-1.7B 上 GRPO+OPSD 仅达 32.0%，而 SDAR 达 53.9%）。

3. **技能内化**：SDAR 将特权知识真正内化到策略参数中，推理时无需外部技能即可超越 Skill-GRPO*（如 ALFWorld-1.7B: 53.9% vs 28.1%）。

4. **更强泛化**：相比 Skill-SD 和 RLSD，SDAR 在跨模型规模上表现更稳健，尤其在小模型（1.7B）上优势明显。

5. **鲁棒性**：即使随机检索也能超越纯 GRPO 基线，说明门控机制有效过滤了低质量技能的噪声。

### 关键发现

- **Gap 门控最优**：直接基于教师-学生 gap 的门控显著优于间接的熵门控和混合的 soft-OR 门控。
- **$\beta=5$ 为最优锐度**：过于保守（$\beta$ 过小）或过于激进（$\beta$ 过大）均导致性能下降。
- **$\lambda=0.01$ 为最优权重**：过度加权会使蒸馏主导策略更新，因教师平均不如学生自信（负 gap）而强制学生走向劣势行为。
- **Reverse KL 优于 Forward KL 和 JSD**：mode-seeking 特性与门控机制形成天然互补，自动降权低教师概率 token。

## 7. 优点

1. **理论支撑扎实**：附录提供严格的理论分析（命题 1-5），证明门控的梯度有界性、单调性及停止梯度操作的必要性。
2. **设计简洁优雅**：核心仅引入一个 sigmoid 门，参数少（$\lambda, \beta$）、调参友好、易于实现。
3. **即插即用**：作为 GRPO 的辅助目标，可与其他 RL 方法无缝结合。
4. **自适应课程**：无需人工设计时间表或轨迹深度阈值，token 级自步进机制自动在训练过程中调整监督强度。
5. **对检索质量鲁棒**：门控机制过滤噪声，使方法对检索系统的依赖降低，提高了实用性。
6. **实验全面系统**：涵盖多模型规模、多 benchmark、多基线、多维度消融，结论可信度高。

---

## 8. 不足与局限

1. **Benchmark 覆盖有限**：仅在 ALFWorld、WebShop、Search-QA 三个环境中验证，未在更广泛的 agent 场景（如真实 GUI 自动化、机器人具身控制、代码生成智能体）中测试。
2. **技能库依赖**：虽然对检索质量鲁棒，但仍依赖预定义的 SkillBank，未探索完全无检索的设定（如完全由模型自我生成技能）。
3. **门控函数选择**：仅验证了 sigmoid 形式，其他门控形式（如 Gumbel-softmax、线性衰减）未探索。
4. **小模型性能受限**：在 Qwen3-1.7B 上虽显著优于其他混合方法，但绝对性能仍较低，说明小模型可能难以有效利用 token 级门控信号。
5. **计算开销**：每次训练迭代需两次前向传播（student 和 teacher），相比纯 GRPO 增加约一倍计算成本。
6. **缺乏真实环境验证**：所有 benchmark 均为模拟或文本环境，未涉及真实的网页交互或具身机器人任务。
