## 摘要

开放世界游戏中的长时域多模态智能体必须在紧张的 token 和延迟预算下在许多低级交互中保持目标导向。现有方法经常在每步昂贵推理和可能导致漂移、重复失败和恢复不良的反应式执行之间权衡。我们的关键思想是在局部稳定段上复用战略推理，并在事件边界处重新调用它。我们提出 SPIKE，这是一种用于成本高效长时域游戏控制的自适应双控制器框架。其战略控制器执行低频全局规划、失败分析和恢复，而其反应控制器在严格的 token 预算下处理快速局部执行。事件触发器监控视觉变化、任务进度、重复动作和失败信号，以决定控制应保持反应式还是升级到战略推理。层次记忆将状态-动作记忆库（SA-MB）中的短期经验复用与状态动作知识图（SA-KG）中的结构化证据分离，允许每个控制器检索其需要的上下文。这种设计在多个反应步骤上复用战略提议，在计划变陈旧时支持本地覆盖，并将昂贵的推理保留在额外推理有用的时刻。在 StarDojo 的 Lite-100 分割上，SPIKE 将 Lite-100 成功率（SR）相比最强的 Lite-100 基线提高了 5.0 个百分点（相对 38.5%），将预算 SR 相比最强的预算基线提高了 9.3 个百分点（相对 75.6%）。它还分别降低了 54.9% 的 token 消耗和 40.8% 的延迟。消融显示事件触发、反应覆盖和异构记忆各自对成功和恢复有贡献，支持选择性推理而非每步推理。

## Abstract
Long-horizon multimodal agents in open-world games must stay goal-directed across many low-level interactions under tight token and latency budgets. Existing approaches often trade off costly per-step reasoning against reactive execution that can drift, repeat failures, and recover poorly. Our key idea is to reuse strategic reasoning across locally stable segments and reinvoke it at event boundaries. We present SPIKE, an adaptive dual controller framework for cost-efficient long-horizon game control. Its Strategic Controller performs low-frequency global planning, failure analysis, and recovery, while its Reactive Controller handles fast local execution under a strict token budget. An Event Trigger monitors visual change, task progress, repeated actions, and failure signals to decide when control should stay reactive or escalate to strategic reasoning. Hierarchical Memory separates short-term experience reuse in the State-Action Memory Bank (SA-MB) from structured evidence in the State Action Knowledge Graph (SA-KG), allowing each controller to retrieve the context it needs. This design reuses strategic proposals over multiple reactive steps, supports local override when plans become stale, and reserves expensive reasoning for moments where extra deliberation is useful. On the Lite-100 split of StarDojo, SPIKE improves Lite-100 success rate (SR) by 5.0 percentage points (38.5% relative) over the strongest Lite-100 baseline and Budgeted SR by 9.3 points (75.6% relative) over the strongest budgeted baseline. It also reduces token consumption by 54.9% and latency by 40.8%. Ablations show that event triggering, reactive override, and heterogeneous memory each contribute to success and recovery, supporting selective reasoning rather than reasoning at every step.

## 1. 核心问题与整体含义（研究动机和背景）

**问题背景：**
- 大型语言模型（LLM）、视觉-语言模型（VLM）和视觉-动作基础模型的发展，使得开放世界游戏（如 StarDojo）、网页导航和通用计算机控制领域的智能体能力不断提升
- 与单步问答不同，这些环境要求智能体在长时间、变化的交互历史和延迟反馈下执行动作

**核心挑战：**
- **规划-延迟-记忆三难困境**（Planning-Latency-Memory Trilemma）：
  - 更深的推理提升决策质量，但增加推理成本
  - 轻量级反应式执行速度快，但视野短浅
  - 朴素记忆复用引入检索噪声
- 当前方法存在两个极端：
  - **纯战略控制器**（如 CRADLE）：每步都调用大模型，token 和延迟成本极高
  - **纯反应控制器**（如 ReAct）：执行廉价但易出现漂移、执行失败、重复动作和恢复困难

**研究动机：**
- 开放世界控制存在**成本-价值不匹配**：常规步骤需要快速执行，而罕见的场景变化、卡顿和失败才需要昂贵推理
- 关键洞察：**战略推理应在局部稳定段上摊销（amortized），仅在事件边界处重新调用**

## 2. 论文提出的方法论

### 核心思想

SPIKE 将长期多模态控制建模为**预算分配的策略推理**，核心包括：

1. **事件触发的摊销 deliberation**：战略推理应跨局部稳定段复用，仅在事件边界重新调用
2. **自适应双控制器执行**：战略规划用于提升点，有界反应覆盖用于稳定局部执行
3. **控制器专用层次记忆**：SA-MB 用于局部状态-动作复用，SA-KG 用于结构化战略证据

### 关键技术细节

#### 2.2.1 双控制器框架

**观察表示：**
- $o_t = (v_t, u_t, f_t)$：视觉截图 $v_t$、文本/UI 上下文 $u_t$、运行器反馈 $f_t$（进度、位置、物品栏、对话变化）
- 决策：$g_t \in \{0, 1\}$，1=战略控制器，0=反应控制器

**优化目标（公式 1）：**
$$\max_{\pi_R, \pi_S, g} E[S(\tau)] \quad \text{s.t.} \quad E\left[\sum_{t=1}^{H} C_t(g_t, h_t)\right] \leq B$$

**战略控制器职责：**
- 信息收集 → 可选自我反思 → 任务推理 → 动作提议
- 输出：接下来 T 步的短期规划

**反应控制器职责：**
- 使用最新战略提议 + 当前观察 + SA-MB 记忆提示
- 严格 token 预算下执行局部动作
- 允许有界局部修正（避免、重新定位、工具重选）
- 不能改变当前子目标，失败修正后返回事件触发器

#### 2.2.2 事件触发器（Event Trigger）

**决策逻辑（公式 3）：**
$$g_t = I[(c_t \geq T) \vee (d_t > \tau_v) \vee (z_t \geq \tau_z) \vee (r_t \geq \tau_r \wedge z_t \geq \tau_{rz}) \vee (\ell_t \geq \tau_\ell)]$$

**触发信号：**
- $c_t$：上次战略调用后的反应步数计数
- $d_t$：视觉不连续性（余弦距离 $d_t = 1 - \cos(\phi(v_t), \phi(v_{t-1}))$）
- $z_t$：零进步卡顿计数
- $r_t$：最近 W 步行为循环检测
- $\ell_t$：执行失败等级（F1-F3）

**失败等级定义：**
- $\ell=0$：正常执行
- $\ell=1$：孤立局部问题，有界本地重试
- $\ell=2$：硬执行错误或低进度循环，带失败痕迹升级
- $\ell=3$：重复硬失败，无效当前计划，刷新上下文并强制顶层重规划

#### 2.2.3 层次记忆

**全局记忆（Global Memory）：**
- 固定大小窗口保存最近观察和动作
- 活跃子目标、进度指示器、失败摘要

**状态-动作记忆库（SA-MB）：**
- 低延迟局部动作先验，用于反应执行
- **检索评分（公式 4）：**
$$S_{MB}(i|x_t) = \rho_i(\alpha_C C_i + \alpha_L L_i + \alpha_R R_i + \alpha_P P_i)$$
  - $L_i$：Jaccard 词汇重叠
  - $C_i$：token 频率余弦相似度
  - $R_i$：奖励 EMA 的 clip 后归一化
  - $P_i$：成功/尝试比
  - $\rho_i$：24 小时指数衰减

**状态-动作知识图（SA-KG）：**
- 战略证据图，用于计划检查和重规划
- 存储：状态节点、动作边、相似边
- **检索评分（公式 5）：**
$$S_{KG}(j|x_t) = \beta_C C_j + \beta_P P_j$$
  - $C_j$：向量相似度
  - $P_j$：最佳出边动作成功率

**融合检索：**
$$S_{fuse} = \lambda_{MB} \tilde{S}_{MB} + \lambda_{KG} \tilde{S}_{KG}$$

**写入策略：** 仅成功执行、可解释恢复或规划器验证的进展才写入，防止瞬时错误成为长期证据

### 算法流程

```
初始化: proposal p ← ∅, history h_0 ← ∅
for t = 1 to H:
    观察 o_t = (v_t, u_t, f_t) 并更新历史上下文
    计算视觉变化 d_t, 卡顿计数 z_t, 重复尾 r_t, 失败等级 ℓ_t
    g_t ← EventTrigger(d_t, z_t, r_t, ℓ_t, c_t; Θ)
    if g_t == 1:
        检索 SA-KG/全局证据
        用战略控制器更新 proposal p
    else:
        从 SA-MB 检索局部提示
    用反应控制器基于 o_t, p 和提示发出 a_t
    执行 a_t，接收反馈
    仅在成功/恢复/验证后写入记忆
```

## 6. 论文的主要结论与发现

### 6.1 核心性能提升

| 指标 | 提升幅度 |
|------|----------|
| **Lite-100 SR** | +5.0 pp（18.0% vs 13.0% CRADLE），相对 38.5% |

| **恢复/卡顿比** | +0.26（2.75 vs 2.49 CRADLE） |

### 6.2 RDR2 泛化结果

- SPIKE 达到 **56.4% SR**（22/39），领先 CRADLE 12.8 pp
- Token/步减少 48.5%，延迟降低 47.1%

### 6.3 消融关键发现

1. **事件触发必要性**：
   - 移除后 SR 下降 2.3 pp
   - 更多战略调用（2.355/步）反而降低成功率
   - 移除周期刷新导致最大下降（11.6% SR）

2. **反应覆盖必要性**：
   - 移除后 SR 下降 2.3 pp
   - 局部修正防止计划失效

3. **层次记忆贡献**：
   - 移除 SA-MB：-3.0 pp
   - 移除 SA-KG：-3.7 pp
   - 全局记忆替换后仅 12.3% SR

4. **控制器组合效果**：
   - 纯战略+层次记忆：16.3% SR，305.4k tokens
   - 纯反应+层次记忆：10.3% SR，135.4k tokens
   - SPIKE 双控制：18.0% SR，168.1k tokens

### 6.4 任务家族差异

- **农业最高**（34.9%）：结构化环境，工具使用明确
- **战斗最低**（8.3%）：移动敌人定位、导航、攻击时序耦合困难
- **社交次低**（10.7%）：长路线导航需求

|------|------|
| **事件触发决策** | 将战略推理建模为预算分配问题，而非固定调度 |
| **有界反应覆盖** | 反应控制器非被动执行器，可做局部修正但不能改变子目标 |
| **异构层次记忆** | SA-MB 和 SA-KG 分离角色，防止检索噪声 |
| **选择性写入策略** | 仅成功/验证后写入，防止瞬时错误污染记忆 |
| **失败等级设计** | F1-F3 分级恢复，平衡局部重试和全局重规划 |

### 7.2 实验设计亮点

- **双指标评估**：Lite-100 SR（步数限制）+ Budgeted SR（LLM 调用限制），揭示不同偏差
- **Pareto 前沿可视化**：明确展示效率-效果权衡
- **控制变量严谨**：所有方法使用相同基础设施，仅控制器分配不同
- **消融层级清晰**：框架级 → 组件级 → 灵敏度级，因果归因明确
- **跨基准泛化验证**：RDR2 压力测试证明非数据集特定优化

### 7.3 工程实现亮点

- **可复现性细节**：固定随机种子、阈值列表、超参数表完整
- **事件触发阈值固定化**：非任务级调参，提高泛化性
- **记忆融合机制**：SA-MB 主列表 + SA-KG 注入，避免单一检索池噪声

## 8. 不足与局限

### 8.1 实验覆盖局限

| 局限 | 说明 |
|------|------|
| **数据集范围有限** | 主要在 StarDojo（RDR2 仅 13 任务）验证 |
| **战斗/社交任务瓶颈** | Combat SR 仅 8.3%，Social SR 仅 10.7% |
| **代表性失败案例** | 岩石蟹战斗任务显示定位-导航-攻击耦合问题未解决 |

### 8.2 偏差风险

| 风险类型 | 描述 |
|----------|------|
| **触发阈值敏感性** | 默认设置是相对最优，但需针对新游戏类型重新校准 |
| **记忆过期风险** | 视觉subtle变化、失败信号噪声、屏幕差异小的情况可能误路由 |
| **归因模糊** | 多组件耦合设计，精细因果分析仍是未来工作 |
| **后端 API 干扰** | GPT-5.4/Gemini-3.1-pro 延迟数据受服务商优化影响 |

### 8.3 应用限制

| 限制 | 说明 |
|------|------|
| **无特权游戏 API** | 依赖屏幕观察，视觉噪声影响失败检测 |
| **多模态信号不完整** | 当前仅视觉+动作历史，音频等丰富反馈未建模 |
| **推理成本仍存在** | 恢复和重规划仍依赖显式模型调用，限制部署场景 |
| **动作空间依赖** | 需适配不同游戏类型的动作接口抽象 |
| **长期部署未测试** | 更长 horizon、更广游戏类型的适应性未知 |

### 8.4 框架复杂性

- 相比单控制器基线，SPIKE 引入更多移动部件
- 新增故障模式：过期记忆检索、局部控制持续过久、升级过晚/过频
- 调试和调参成本增加

---

## 总结评价

SPIKE 论文提出了一个系统性的双控制器框架，通过事件触发、反应覆盖和层次记忆的协同设计，有效解决了长时 horizon 开放世界游戏中的成本-效果权衡问题。实验设计严谨充分，消融层级清晰，跨基准泛化验证有力。局限性主要在于任务覆盖不够均衡、战斗类任务仍是瓶颈、以及对多模态信号的利用不完整。
