## Abstract
Counterfactual Regret Minimization (CFR) is the dominant algorithmic family for solving large imperfect-information games, underpinning breakthroughs such as Libratus and Pluribus in No-Limit Texas Hold'em poker. In real-time game-playing systems, the solver must compute a near-equilibrium strategy within a strict time budget of only a few seconds per decision, and the number of CFR iterations completed in this window directly determines play strength. We present \textbf{Parallel CFR}, the first parallelization framework for real-time depth-limited CFR solving that seamlessly integrates pruning, abstraction, and advanced CFR variants. We decompose each CFR iteration into a pipeline of seven stages and identify two orthogonal dimensions of parallelism: \emph{by information set} and \emph{by tree node}. Leaf node evaluation is offloaded to GPUs via batched neural network inference, creating a heterogeneous CPU--GPU pipeline. Experiments on Heads-Up No-Limit Texas Hold'em demonstrate that Parallel CFR achieves $3.3$--$3.4\times$ speedup over the single-threaded baseline on postflop streets, with per-iteration time of ${\sim}47$--$54$~ms on a depth-limited game tree with over $1$ billion histories. All experiments run on a single desktop-class device (NVIDIA DGX Spark), enabling hundreds of CFR iterations within a typical real-time decision budget without requiring datacenter-scale infrastructure.

## 论文总结：Real-Time Parallel Counterfactual Regret Minimization

### 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在实时游戏-playing系统中（如 Libratus、Pluribus），求解器必须在严格的时间预算（仅几秒）内计算近均衡策略，而 CFR（反事实遗憾最小化）迭代次数直接决定策略质量。现有并行化方法无法无缝整合深度限制求解、剪枝、抽象等现代组件，导致实时性能受限于单线程执行。

- **背景**：
  - CFR 是不完全信息博弈（如德州扑克）计算近似纳什均衡的主导算法家族。
  - 在实时决策场景中，每秒能完成的 CFR 迭代次数直接决定 AI 的博弈强度。
  - 此前超人类扑克 AI（如 Libratus）依赖超级计算机进行实时求解，部署门槛过高。
  - 现有并行化工作局限于表格 CFR，无法应用于现代实时游戏求解栈（深度限制 + 剪枝 + 抽象 + 高级 CFR 变体的异构流水线）。

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

#### 核心思想
提出 **Parallel CFR**，首个针对实时深度限制 CFR 求解的并行化框架。将每个 CFR 迭代分解为 7 个阶段，识别两个正交并行维度：**按信息集并行**（Information Set Parallelism）和**按树节点并行**（Tree Node Parallelism），并构建异构 CPU–GPU 流水线。

#### 七阶段流水线

| 阶段 | 操作 | 并行维度 | 数据流 |
|------|------|----------|--------|
| S1: ForwardProfile | 传播玩家达到概率 | ∥ 信息集 | σ→πi, π−i |
| S2: AggregateProbability | 聚合到多层级（总/牌/手）| ∥ 节点 × 卡 | πi→Psum, xleaf |
| S3: Compute π−i(I) | 计算对手反事实达到 | ∥ 节点→∥ 信息集 | Psum →π−i(I) |
| S4: ShowdownEquity | 计算摊牌权益 | ∥ 节点 | π−i(I)→vsd |
| S5: BatchLeafEval | GPU 批量叶子评估 | GPU batch | xleaf → ˆvleaf |
| S6: BackwardCFV | 反向传播反事实值 | ∥ 节点+信息集 | ˆvleaf, vsd→vi(I) |
| S7: UpdateRegret | 更新累积遗憾 | ∥ 信息集 | vi(I)→Ri(I,a) |

#### 关键设计

1. **流水线分解**：前向阶段（S1-S2）→ 并行分支（S3/S4 CPU + S5 GPU）→ 反向阶段（S6-S7），各阶段可独立优化。
2. **GPU 叶子评估**：所有非终结叶子节点批量打包为单个 GPU 前向传播，最小化 kernel 启动开销。
3. **剪枝集成**：使用 EVPA 框架，在迭代前预处理去除被支配动作，作为所有后续阶段的乘法加速器（ρ ∈ [0,1]）。
4. **抽象支持**：支持无损抽象（无精度损失）和有损抽象，通过稀疏投影矩阵桥接不同维度空间。
5. **正确性保证**：并行化不引入任何近似、采样或舍入误差，结果与串行实现数值相同。

### 6. 论文的主要结论与发现

1. **加速效果**：Parallel CFR 在 Flop/Turn/River 上实现 **3.3–3.4×** 加速（5 线程，1 GPU），Preflop 因本身极快（~3.5ms）加速仅 2.3×。
2. **绝对性能**：在超 10 亿历史节点的深度限制树上，单次迭代仅需 **~47–54 ms**，可在 5 秒预算内完成 **~100 次迭代**。
3. **收敛优势**：在相同墙钟时间下，Parallel CFR 的 exploitability 比 PokerRL 低约 **7 倍**。
4. **扩展性**：并行效率随树规模增大而提升（SPR 4→64，Flop 时间仅增 8.0×，节点增 15.0×）。
5. **CPU/GPU 瓶颈**：Postflop 以 CPU 阶段（S3/S6）为主（>50% 时间），GPU 评估完全被掩盖；Preflop 和 River 分别由 GPU 和 Showdown 主导。

### 7. 优点

1. **首个完整框架**：首次系统性地并行化实时深度限制 CFR 求解，整合了剪枝、抽象、深度限制、GPU 推理等全部现代组件。
2. **异构流水线设计**：CPU–GPU 协同，Stage 3/4 与 Stage 5 并行执行，最大化硬件利用率。
3. **模块化架构**：CFR 变体仅影响 Stage 7，其他阶段完全通用，便于扩展新算法。
4. **实际可行**：在单个消费级设备上达到实时性能，降低部署门槛，推动扑克 AI 走向边缘设备。
5. **正确性严格保证**：流水线数值等价于串行实现，无近似误差。
6. **实验扎实**：多维度消融、对比基线、收敛曲线，全面验证框架效果。

---

### 8. 不足与局限

1. **并行规模有限**：聚焦单实例 CFR，未涉及多机器分布式扩展。
2. **剪枝依赖性**：依赖 EVPA 剪枝方法，若无剪枝则树规模仍受限于内存。
3. **Preflop 收益有限**：因手牌同构性，预 flop 本身极快，多线程开销相对收益较大（2.3× vs 3.4×）。
4. **GPU 利用率不均**：Postflop 以 CPU 为瓶颈，GPU 仅在 Preflop 主导；在更大规模问题上可能需要更均衡的设计。
5. **未测试更大规模**：仅在 HUNL 上验证，未涉及更高复杂度游戏（如 6-max、Pot-Limit Omaha）。
6. **Future Work 提及**：GPU 原生实现全流水线、CPU–GPU 异步重叠可能在未来进一步降低延迟。
