## 摘要

本文提出了一个安全内建的面向多智能体系统的规划与控制框架，该框架受线性时序逻辑（LTL）规范约束。该框架保护敏感信息免受具有智能体运动部分观测能力的被动入侵者的攻击。多智能体协调中的安全性通过两个概念来体现，防止入侵者推断秘密任务是否已执行，以及识别执行任务的智能体。所提出的框架通过构建一个移除所有违反这些约束路径的安全有限转移系统，直接将安全约束纳入 LTL 合成过程。然后将标准 LTL 合成应用于这个安全抽象以生成离散计划，再将其细化为动态可行的连续轨迹。该合成过程形式化保证多智能体系统的最终行为满足全局 LTL 规范和安全约束。通过两无人机案例研究验证了所提框架的有效性。

## Abstract
This paper presents a secure-by-construction planning and control framework for multi-agent systems subject to linear temporal logic (LTL) specifications. The framework protects sensitive information from a passive intruder with partial observations of the agents' motion. Security in multi-agent coordination is captured by two notions that prevent the intruder from inferring whether a secret task has been executed and from identifying the agent responsible for its execution. The proposed framework incorporates the security constraints directly into the LTL synthesis procedure by constructing a secure finite transition system that removes all paths violating these constraints. Standard LTL synthesis is then applied to this secure abstraction to generate discrete plans, which are then refined into dynamically feasible continuous trajectories. This synthesis procedure provides formal guarantees that the resulting behavior of the multi-agent system satisfies both the global LTL specification and the security constraints. The effectiveness of the proposed framework is demonstrated through a two-drone case study.

**研究动机：**
- 多智能体系统（如无人机群、智能交通系统）在执行复杂任务时，往往需要指定复杂的时序行为，线性时序逻辑（LTL）能够很好地表达这类任务。
- 然而，现有研究在将LTL应用于多智能体规划时，**忽略了信息安全**。实际应用中，敏感任务（如访问秘密区域）可能被外部入侵者通过观察智能体运动推断出来。
- 被动入侵者具有**部分观测能力**，能够通过输出函数观察智能体状态，但无法直接获知其是否访问了秘密区域。

**核心问题（Problem 1）：**
设计一个安全感知的规划与控制框架，使得多智能体系统生成的轨迹同时满足：
1. **LTL任务规范**：全局轨迹的迹（trace）满足给定的LTL公式。
2. **安全约束**：防止入侵者推断（i）是否有智能体访问了秘密区域（Type-A安全），以及（ii）是哪个智能体执行了秘密任务（Type-B安全）。

**整体含义：**
该研究为多智能体系统提供了**安全内嵌（secure-by-construction）**的设计方法，即在规划阶段就将安全约束融入系统合成，而非事后补救。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想

将安全约束直接嵌入LTL合成过程：
1. 构建**安全有限转移系统**（Secure gWTS），移除所有违反安全约束的路径。
2. 在该安全抽象上应用标准LTL合成，生成离散计划。
3. 将离散计划细化（refine）为动态可行的连续轨迹。

### 2.2 系统模型

- **工作空间**：全维多面体 $P \subset \mathbb{R}^n$，被划分为 $p$ 个非重叠子多面体 $q_1, \dots, q_p$。
- **智能体动态**：每个智能体 $i \in A = \{1, \dots, M\}$ 满足线性仿射动力学：
  $$\dot{x}_i = A_i x_i + B_i u_i + b_i$$
  其中 $x_i \in P$ 为状态，$u_i \in U_i$ 为有界输入（$U_i$ 为凸多面体）。
- **抽象**：智能体的运动被抽象为加权转移系统（WTS），其中每个子多面体对应一个离散状态。

### 2.3 安全约束定义

#### Type-A Security（防推断秘密访问）：
对于全局路径 $\tau \in Path(T_g)$，如果存在另一路径 $\tau'$ 使得：
- 观察序列相同：$H_g(\tau) = H_g(\tau')$。
- 且当真实路径中某智能体访问秘密状态时，复制路径中该智能体**不在秘密状态**。
则 $\tau$ 是 Type-A 安全的。形式化定义（Def. 5）确保入侵者无法确定某智能体是否访问了秘密区域。

#### Type-B Security（防识别执行者）：
对于任意智能体 $i$ 和时间步 $j$，如果 $i$ 在秘密状态，则存在另一智能体 $k \neq i$ 产生相同观察：
$$\tau_i(j) \in Q_i^S \Rightarrow \exists k \neq i: H_k(\tau_k(j)) = H_i(\tau_i(j))$$
该约束防止入侵者将某智能体锁定为秘密任务的唯一执行者。

### 2.4 安全抽象的构建流程

**Type-A 安全抽象（Def. 8-9）：**
1. 构建**Twin-gWTS** $V$：状态为不可区分的全局状态对 $(q_1, q_2)$，其中 $H_g(q_1) = H_g(q_2)$。
2. 构建**Secure Twin-gWTS** $V_s$：从 $V$ 中移除所有存在智能体在真实状态和复制状态均为秘密的配对。

**Type-B 安全抽象（Def. 10）：**
- 限制全局状态集合为 $Q_B^g = \{(q_1, \dots, q_M) \in Q_g \mid q_i \in Q_i^S \Rightarrow \exists k \neq i: H_i(q_i) = H_k(q_k)\}$。

**Type-A/B 同时满足**：
先构建 Type-B gWTS $T_B^g$，再在其上构建 Twin-gWTS 和 Secure Twin-gWTS。

### 2.5 转移动态可行性验证

使用**控制屏障函数（CBF）**（Def. 3）验证离散转移的动态可行性（Sec. 4.2）：
- 目标：确保智能体从当前子多面体 $q_i$ 运动到下一子多面体 $q_i'$ 时，始终保持在各自子多面体内（除交叉时刻外）。
- 方法：通过求解**中心化二次规划（QP）**（Problem 2），生成满足 CBF 约束的连续控制器。

QP 约束包括：
1. 动力学约束 $\dot{x}_i = A_i x_i + B_i u_i + b_i$。
2. 输入约束 $u_i(t) \in U_i$。
3. 起点终点约束 $x_i(0) = \xi_i, x_i(t_f) = \xi_i'$。
4. 边界交叉约束 $h_{c,i}(x_i(t_c)) = 0$。
5. CBF 前进不变性约束（Lemma 3）。

### 2.6 LTL 合成与轨迹生成

**产品 Büchi 自动机（PBA）**（Def. 11）：
- 将 Secure Twin-gWTS $V_s$ 与 LTL 公式的 NBA $B_\phi$ 同步。
- 接受路径即满足 LTL 规范。

**算法流程（Alg. 1）**：
1. 将 LTL 公式转换为 NBA $B_\phi$。
2. 构建 PBA $\mathcal{P} = V_s \otimes B_\phi$。
3. 对每个接受状态 $q_{acc}$：
   - 使用 Dijkstra 算法计算到 $q_{acc}$ 的最短前缀路径。
   - 计算从 $q_{acc}$ 返回自身的最短后缀环。
   - 计算总代价 $J = \beta J_{prefix} + (1-\beta) J_{suffix}$。
4. 选择总代价最小的前缀-后缀路径 $\tau^*$。
5. 通过拼接每个离散转移对应的连续轨迹段，生成最终轨迹 $x^*(t)$。
