## 摘要
多任务强化学习（MTRL）旨在训练单一智能体同时高效优化多个任务性能。然而，联合优化所有任务往往导致学习不均衡：智能体快速解决简单任务，但在困难任务上学得很慢。虽然先前的工作主要将这种不均衡归因于冲突的任务梯度，并提出梯度操纵或专门架构来解决，我们则关注一个独特且探索不足的挑战：数据分配不均衡。标准的MTRL为每个任务分配相同数量的环境交互，导致对需要相对较少交互即可解决的简单任务过度分配数据，而对需要更多经验才能解决的困难任务分配不足。为应对这一挑战，我们提出了分布鲁棒自适应任务采样（DRATS），该算法自适应地优先采样最远离已解决状态的任务。我们通过将MTRL形式化为一个可行性问题来推导DRATS，并从中得到一个最小化最坏情况返回差距（即期望目标返回与智能体在任务上的返回之间的差距）的极小化极大目标。在MetaWorld-MT10和MT50等基准上，DRATS提升了数据效率并提高了最差任务的性能，相较于现有任务采样算法表现更佳。

## Abstract
Multi-task reinforcement learning (MTRL) aims to train a single agent to efficiently optimize performance across multiple tasks simultaneously. However, jointly optimizing all tasks often yields imbalanced learning: agents quickly solve easy tasks but learn slowly on harder ones. While prior work primarily attributes this imbalance to conflicting task gradients and proposes gradient manipulation or specialized architectures to address it, we instead focus on a distinct and under-explored challenge: imbalanced data allocation. Standard MTRL allocates an equal number of environment interactions to each task, which over-allocates data to easy tasks that require relatively few interactions to solve and under-allocates data to hard tasks that require substantially more experience to solve. To address this challenge, we introduce Distributionally Robust Adaptive Task Sampling (DRATS), an algorithm that adaptively prioritizes sampling tasks furthest from being solved. We derive DRATS by formalizing MTRL as a feasibility problem from which we derive a minimax objective for minimizing the worst-case return gap, the difference between a desired target return and the agent's return on a task. In benchmarks like MetaWorld-MT10 and MT50, DRATS improves data efficiency and increases worst-task performance compared to existing task sampling algorithms.

## 2. 论文提出的方法论

### 核心思想
将 MTRL 建模为**可行性问题**（feasibility problem），要求智能体在每个任务上达到目标收益，然后推导出一个**极小极大目标**，以最小化最坏情况下的收益差距（return gap）。

### 关键技术细节

- **收益差距定义**：$g_i(θ) := \max\{J_{ref,i} - J_i(θ), 0\}$，即目标收益与当前收益之间的差距
- **极小极大目标**：$\min_θ \max_{q∈Δ_k} \mathbb{E}_{i∼q}[g_i(θ)]$，其中 $q$ 定义任务采样分布
- **KL 正则化**：为避免采样分布过度集中，添加 KL 约束 $KL(q || p_0) ≤ ε$，其中 $p_0$ 为均匀分布
- **闭式解推导**：通过对偶分析和拉格朗日松弛，得到最优采样分布为 **softmax 形式**：$q^*_i = \exp(ηg_i(θ)) / \sum_j \exp(ηg_j(θ))$，其中 $η$ 为逆温度参数，控制分布锐度
- **Mirror Ascent 更新**：使用负熵作为镜射映射，迭代更新采样分布：$q_{t+1,i} \propto q_{t,i} \exp(αh_{t,i})$，其中 $h_{t,i} = g_i(θ_t) - \frac{1}{η}(\log(kq_{t,i}) + 1)$

### 算法流程
1. 从当前任务分布 $q_t$ 采样任务，收集轨迹数据填充缓冲区
2. 对每个任务计算蒙特卡洛收益估计 $\hat{J}_i$
3. 更新参考收益 $J_{ref}$（若适用）
4. 计算归一化收益差距 $\hat{g}_i$
5. 通过 Mirror Ascent 更新任务分布 $q_{t+1}$
6. 强制最小采样概率以避免任务被完全忽视
7. 使用 on-policy 算法（如 PPO）更新策略参数 $θ$

## 6. 论文的主要结论与发现

### 主要结论

1. **数据效率提升**：DRATS 在所有 benchmark 上均达到最高聚合收益，或在相似最终收益下减少所需交互次数
2. **最差任务性能改善**：DRATS 的提升主要来自最差任务性能的提高，而非简单任务表现的上限提升
3. **架构兼容性**：DRATS 可与 MOORE、Soft Modularization 等多任务网络架构结合，进一步提升性能
4. **普适性**：在 Gridworld 上，即使消除任务间转移（使用独立网络），DRATS 仍优于基线方法

### 关键发现

- **Hard First 的缺陷**：当任务被标记为"已解决"后突然转移采样概率，导致性能崩溃和重复学习
- **Learning Progress/Potential 的局限**：优先快速学习任务（斜率大的学习曲线），反而可能忽视距离解决很远但进展缓慢的困难任务
- **自适应采样的优势**：相比目标重权重，自适应采样在梯度估计准确性和数据分配效率上均更优

## 7. 优点

### 方法论亮点
- **问题建模创新**：将 MTRL 形式化为可行性问题，为任务优先级提供了理论支撑
- **收敛性保证**：提供了 $O(1/√T)$ 的收敛速率证明，这是多数任务采样方法所缺乏的
- **KL 正则化的巧妙应用**：避免采样分布过度集中，保证所有任务持续学习

### 实验设计亮点
- **全面基准测试**：覆盖从玩具问题到真实机器人操作的多种场景
- **公平对比**：所有基线使用相同的更新规则（Mirror Ascent），仅优先级指标不同
- **消融彻底**：系统验证了自适应采样相比目标重权重的优势
- **兼容性验证**：展示了 DRATS 与现有架构的正交性（可叠加）

### 实用性亮点
- **实现简洁**：仅修改数据收集方式，不改变策略优化本身，易于集成
- **参考收益在线估计**：无需用户手动指定阈值，降低使用门槛
- **超参数鲁棒**：$η ∈ [2, 16]$ 范围内性能稳定

## 8. 不足与局限

### 理论层面
- **参考收益估计依赖**：在线估计可能在训练早期低估真实最大值，导致困难任务初期分配不足
- **离散任务假设**：假设任务集合是离散的，连续任务空间需要额外建模（如用神经网络输出分布参数）

### 实验层面
- **MT50 样本量有限**：10 seeds 可能不足以可靠区分方法性能
- **Hard First 基线的适配问题**：原 SMT 针对 off-policy 设计，作者做了若干修改，可能未能完全复现原方法效果
- **任务间转移假设**：实验假设任务间存在正向转移（共享网络表现更好），但在完全冲突场景下 DRATS 的表现未被充分验证

### 实用性局限
- **超参数敏感**：$α$ 对性能影响较大，过大会导致采样分布振荡
- **需要频繁更新采样分布**：在 on-policy 设置下可能引入额外开销
- **仅针对任务采样**：未直接处理梯度冲突问题，在极端冲突场景下可能仍受限
