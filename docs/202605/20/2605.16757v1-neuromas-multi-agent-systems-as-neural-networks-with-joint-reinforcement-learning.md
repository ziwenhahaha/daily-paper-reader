## 摘要
NeuroMAS将多智能体语言系统抽象为可训练的神经网络式架构，以大语言模型为节点、文本信号为边，通过联合强化学习实现自主通信、专业化与协调，显著提升性能并揭示组织规模化的路径依赖性。

## Abstract
Multi-agent language systems are often built as hand-designed workflows, where agents are assigned semantic roles and communication protocols are specified in advance. We propose NeuroMAS, a method that first treats a multi-agent language system as a trainable and scalable neural-network-like architecture with LLM agents as nodes and intermediate textual signals as edges. In NeuroMAS, agent nodes are role-free but structure-aware: the topology only determines how information can flow in general, while reinforcement learning training determines how nodes communicate, specialize, and coordinate. This formulation shifts multi-agent design from workflow engineering toward architecture design, where depth, width, connectivity, and growth protocol become scalable sources of capability. Further, we provide a theoretical perspective showing why such modular textual computation is more parameter-efficient when tasks admit hierarchical decompositions. Experiments show that NeuroMAS improves significantly over both inference-time and trained multi-agent baselines. We further find that organizational scaling is path-dependent: larger systems can be challenging to train from scratch, but become feasible when grown progressively from smaller trained systems. These results suggest that learned neural multi-agent systems are a promising scaling axis for LLMs.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想

NeuroMAS（Neural Multi-Agent System）将多智能体语言系统抽象为**文本值（text-valued）的神经网络式架构**：

- **节点（Nodes）**：LLM 智能体，作为随机语言策略（stochastic text policy）。
- **边（Edges）**：节点之间传递的**中间文本消息**（intermediate textual signals），而非梯度或连续向量。
- **关键设计——Role-Free but Structure-Aware**：节点无手工语义角色，仅知道自己在拓扑中的位置和输出格式；特殊化行为通过训练从终端奖励中涌现。

### 2.2 架构形式化

- 拓扑 $\tau = (n_1, n_2, \ldots, n_L)$ 表示 $L$ 个隐藏层，每层 $n_\ell$ 个节点，外加一个输出聚合节点。
- 前向传播（分层前馈）：
  - 隐藏节点 $v_{\ell,j}$：输入上下文 $C_{\ell,j}$（来自原始查询 + 前一层所有消息），生成文本 $Z_{\ell,j} \sim \pi_{\theta_{\ell,j}}(\cdot|C_{\ell,j})$。
  - 消息解析器 $\sigma_{\ell,j}$：从 $Z_{\ell,j}$ 中提取发往下一层各节点的消息。
  - 输出节点：从最终隐藏层所有消息中聚合生成最终答案 $\hat{Y} \sim \pi_{\theta_{\text{out}}}(\cdot|C_{\text{out}})$。
- 当 $\tau = \emptyset$ 时，退化为单策略（Single-LLM RL 基线）。

### 2.3 训练方法：终端奖励的联合策略梯度

- 由于所有中间文本是离散 token，无法通过其内容反向传播。
- 采用 **REINFORCE / score-function 策略梯度**：所有节点共享同一个终端奖励 $r(Y, \hat{Y})$，但每个节点仅根据自身采样输出的 log-probability 更新参数。
- 节点级采样损失：
  $$L_{\ell,j} = -\left[r(Y, \hat{Y}) - b_{\ell,j}\right] \cdot \bar{\ell}_{\ell,j}$$
  其中 $\bar{\ell}_{\ell,j}$ 为该节点生成文本的平均 token log-probability，$b_{\ell,j}$ 为指数移动平均基线（降低方差）。
- 总损失为所有节点损失之和（公式 3）。

### 2.4 参数高效节点实例化

- 所有节点共享同一个**冻结的 LLM 主干**（backbone）。
- 每个节点附加**节点专属的 LoRA adapter**（低秩适配器），仅训练 adapter 参数，主干参数固定。
- 在 LoRA 实现中，新增节点的 adapter 初始化为 $\Delta W = 0$（零初始化），使其初始行为等价于冻结主干。

### 2.5 渐进增长（Progressive Growth）

- **Scaling 协议**：不是从头训练大拓扑，而是从已训练的小拓扑扩展。
- 扩展时：保留位置兼容的节点（直接复制 LoRA 参数），新增节点初始化为惰性（零 adapter）。
- 扩展链：$\tau_3 = [1,1]$ → $\tau_5 = [2,2]$ → $\tau_7 = [2,2,2]$，其中数字为每层节点数，1 表示输出节点始终存在。
- NeuroMAS-$c$：总共 $c$ 个 LLM 调用（即 $c-1$ 个隐藏节点 + 1 个输出节点）。

### 2.6 理论分析

- **定理 5.1（组合参数效率界）**：在任务具有层级可分解性（假设 1-3）的条件下，多智能体模块化生成器的误差为 $\varepsilon_{\text{multi},q} = O\left(K^{1+1/a} q^{-1/a}\right)$，其中 $K$ 为组件数，$a$ 为局部学习复杂度指数。
- **推论 1**：当单策略的有效近似指数 $b > a$ 时，对于足够大的参数预算 $q$，模块化系统优于单策略系统。
- **核心含义**：当任务有可利用的组合结构时，分布式计算比强迫单一模型一次性解决更参数高效。

## 6. 论文的主要结论与发现

1. **学习型组织有效**：NeuroMAS 在所有 6 个 benchmark 上同时优于最强固定主干基线和单模型 RL 控制，证明多智能体组织本身是可训练的能力来源，而非仅仅是参数增加的结果。
2. **角色自由设计的可行性**：不赋予节点手工语义角色，仅通过结构位置和终端奖励训练，也能诱导出有用的专业化行为。
3. **组织 scaling 是路径依赖的**：从零训练大拓扑反而导致性能下降（NeuroMAS-7 从零训练仅 40.5%，低于 NeuroMAS-3 的 45.5%）；但通过渐进增长策略，性能可从 45.5% 提升至 51.0%。
4. **参数效率不是唯一驱动**：参数匹配的 Single-LLM RL 始终弱于 NeuroMAS-3，说明分布式交互计算不能简单等价于在单一模型中投入等量参数。
5. **跨 backbone 鲁棒性**：在 Gemma-3-1B-IT 上的结果与 Qwen3-0.6B 趋势一致，表明优势具有跨主干泛化性。
6. **理论支持**：当任务具有层级可分解结构时，模块化架构在参数效率上优于单策略系统。

## 7. 优点

- **统一的框架视角**：首次将多智能体组织与智能体训练统一为同一优化目标，而非分别处理 workflow 工程和 agent 训练。
- **严格的实验控制**：特别设计了参数匹配的单模型 RL 基线，能够将"多智能体组织"带来的增益与"更多可训练参数"带来的增益解耦。
- **渐进增长协议的创新**：提出并验证了组织 scaling 的路径依赖性，提供了一种可操作的增长策略，而非盲目增大拓扑。
- **理论与实践结合**：提供了基于组合效率的理论分析（定理 5.1 + 推论 1），为实验观察提供了合理的解释框架。
- **Role-free 设计的简洁性**：不需要人工设计角色名称和职责描述，降低了设计成本且更通用。
- **模块化可扩展性**：拓扑家族（深度 × 宽度）提供了清晰的能力扩展维度，与神经网络架构设计思路一致。

## 8. 不足与局限

- **任务结构依赖性**：理论优势和实验收益依赖于任务具有可分解的层级结构，对于需要整体全局判断或不可分解的任务，优势可能消失。
- **消融不够充分**：渐进增长的消融仅在 Navigate 一个任务上验证，其他任务上是否成立尚未确认。
- **推理成本增加**：每次前向传播需要多个 LLM 调用（NeuroMAS-3 为 3 次），在延迟敏感场景中不如单模型经济。
- **固定 prompt 模板**：输入上下文构造依赖手工的 prompt 模板和解析规则，虽然模板本身非学习，但仍是设计选择。
- **小规模 backbone 限制**：实验刻意选择弱 backbone（Qwen3-0.6B）以保证任务不饱和，未在强 backbone（如 GPT-4 级别）上验证效果。
- **评估规模有限**：使用了相对较小的评估子集，论文明确指出"results should be interpreted as evidence for learned organization as a promising scaling axis, not as a universal replacement for single-model scaling"。
- **拓扑设计空间未充分探索**：仅测试了分层全连接前馈拓扑，其他连通模式（skip connection、循环、动态拓扑）的效果未知。
- **通信协议固定**：消息解析和路由规则基于确定性格式解析，解析失败时使用简单回退策略，未探索更鲁棒的通信机制。
