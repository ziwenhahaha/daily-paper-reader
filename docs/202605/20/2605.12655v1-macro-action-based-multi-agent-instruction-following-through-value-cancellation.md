## 摘要
在实际应用场景中，多智能体强化学习（MARL）可能需要适应外部自然语言指令，这些指令会中断正在进行的行为并与长期目标产生冲突。然而，将奖励条件设定在指令上会引入一种根本性失败模式，即Bellman更新在不同的指令上下文之间耦合价值估计，导致当指令中断宏动作时出现价值不一致。我们提出了宏动作价值修正以实现指令遵守（MAVIC），它通过校正进入的指令目标并恢复当前目标下的延续价值来修正指令边界的Bellman更新。与奖励塑造不同，MAVIC直接修改自助法目标本身，使得在统一策略内随机切换指令的情况下仍能实现一致的价值估计。我们提供了理论分析以及一种演员-评论家实现，并展示了MAVIC在越来越复杂的合作多智能体环境中实现了高指令遵守率同时保持基础任务性能。

## Abstract
Multi-agent reinforcement learning (MARL) in real-world use cases may need to adapt to external natural language instructions that interrupt ongoing behavior and conflict with long-horizon objectives. However, conditioning rewards on instructions introduces a fundamental failure mode as Bellman updates couple value estimates across instruction contexts, leading to inconsistent values when instructions interrupt macro-actions. We propose Macro-Action Value Correction for Instruction Compliance (MAVIC), which corrects Bellman backups at instruction boundaries by correcting the incoming instruction objective and restoring the continuation value under the current objective. Unlike reward shaping, MAVIC modifies the bootstrapping target itself, enabling consistent value estimation under stochastic instruction switching within a unified policy. We provide theoretical analysis and an actor-critic implementation, and show that MAVIC achieves high instruction compliance while preserving base task performance in increasingly complex cooperative multi-agent environments.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：在现实世界的多智能体系统中，机器人需要在**部分可观测环境**中动态响应人类自然语言指令。这些指令具有以下特征：
  - 不可预测地到达（stochastic arrival）
  - 会中断正在执行的宏动作（macro-action）
  - 可能与长期基础任务目标冲突（例如指令禁止使用某个资源，但该资源对完成基础任务必不可少）

- **关键挑战**：多智能体强化学习（MARL）通常使用宏动作（temporally extended actions）来处理长时域协作任务。现有方法（如符号规划、VLM/LLM方法）在灵活性、计算成本或扩展性上存在局限。**一种朴素做法是将奖励条件设定为当前指令——但这会引入根本性失败模式**。

- **核心问题（Cross-Contamination）**：当指令中断宏动作时，标准 Bellman 更新会从**不同指令上下文**引导价值估计，导致价值函数在指令类之间产生耦合。具体来说，在状态 $(s, c_\emptyset)$ 下的价值函数递归依赖于指令状态 $(s', c)$ 的价值（见公式3），这使得：
  - 智能体可能学会**忽略**某些指令（因为它们与基础任务冲突）
  - 智能体可能**提前调整**基础行为以应对预期指令
  - 优化某一指令类会**损害**其他指令类的性能

- **研究目标**：设计一种方法，使智能体能够在统一策略下实现：
  1. 高指令遵守率（instruction compliance）
  2. 保持基础任务性能（base task performance）

## 2. 论文提出的方法论

### 2.1 核心思想

论文提出 **MAVIC（Macro-Action Value Correction for Instruction Compliance）**，核心思路是：**在指令边界处修正 Bellman 更新**，通过去除进入指令的贡献并替换为当前指令下的延续价值，从而实现每个指令类内价值估计的一致性。

### 2.2 形式化建模

- **基础模型**：使用 **Macro-Dec-POMDP**（宏动作去中心化部分可观测马尔可夫决策过程），定义为 $M = \langle I, S, A, M, \Omega, \zeta, T, R, O, Z, \gamma \rangle$，其中宏动作 $m_{i,j} = \langle I_{i,j}, \pi_{i,j}, \beta_{i,j} \rangle$ 包含启动集、低层原始动作策略和终止函数，执行时持续 $\tau$ 步。

- **指令增强过程**：将指令建模为离散指令类集合 $C$，每个类 $c \in C$ 对应一类语义等价的自然语言命令（如“给我番茄”和“请拿番茄”属同一类），包含空指令 $c_\emptyset$。通过笛卡尔积将状态和观察空间扩展为 $(s, c)$ 和 $(\omega, c)$，奖励函数修改为 $R'((s,c),\vec{a},(s',c')) = R_c(s,\vec{a},s')$。**关键设计**：当新指令到达或指令终止时，强制中断当前执行的宏动作。

- **价值函数的递归耦合**：以简化的两类指令（$c_\emptyset$ 和 $c$）为例，概率为 $\beta$ 收到指令时：
  $$V_{\vec\Psi}((s, c_\emptyset)) = E[R_{c_\emptyset} + \gamma(\beta V_{\vec\Psi}((s',c)) + (1-\beta) V_{\vec\Psi}((s',c_\emptyset))]$$
  价值在非指令状态和指令状态之间递归耦合。

### 2.3 价值修正机制

MAVIC 的核心是**动态修正奖励函数**（表2）：

| 转换类型 | 奖励 |
|---|---|
| $c = c'$（指令不变） | $R_c(s, \vec{a}, s')$ |
| $c \neq c'$（指令切换） | $R_c(s, \vec{a}, s') - \gamma V_{\vec\Psi}((s', c')) + \gamma V_{\vec\Psi}((s', c))$ |

在指令切换时，该修正**减去**来自进入指令的引导值，**替换为**当前指令下的延续价值，确保未来指令转换不影响当前指令的价值估计，从而消除跨指令的价值污染。

### 2.4 关键理论结果

- **引理1**：策略 $\Psi$ 在 $M_C$ 中关于指令类 $c$ 的价值，等于其在特定指令问题 $M_c$ 中的价值（对所有 $s \in S$）。
- **定理1**：若策略 $\Psi^*$ 在 $M_C$ 中是最优的，则它也在每个特定指令问题 $M_c$ 中是最优的（对所有 $c \in C, s \in S$）。该解耦仅源于修正的 Bellman 更新，未对转换或策略施加额外限制。

### 2.5 与 reward shaping 的区别

MAVIC 引入的修正是**策略依赖**的，直接修改引导目标，无法表达为固定势函数。这与 Ng 等人(1999) 的势函数塑形方法本质上不同：势函数方法保留原始 Bellman 算子，而 MAVIC 在随机指令切换场景下在 Bellman 更新层面强制一致性。

### 2.6 算法实现（Algorithm 1）

MAVIC 基于 **macro-action actor-critic** 框架（如 Mac-IAC），具体流程：

1. **数据收集**：智能体执行宏动作直到宏动作终止或收到新指令；指令切换时终止所有宏动作并基于更新指令采样新宏动作；转换元组 $(h_t, c_t, h_{t+1}, c_{t+1}, \bar{r}_t, \vec{m}_t)$ 存入重放缓冲区。
2. **训练时价值修正**：对每个采样转换，若指令改变（$c_k \neq c_{k+1}$），应用修正：
   $$\bar{r}_k \leftarrow \bar{r}_k + \gamma^{\tau_m}[V_\phi((h_{k+1}, c_k)) - V_\phi((h_{k+1}, c_{k+1}))]$$
   使用修正后的回报 $G$（式5）计算 TD 误差并更新评论家，使用修正后的策略梯度（式6）更新演员。
3. **指令编码**：使用**冻结的 BERT 模型**将自然语言指令编码为嵌入，与智能体的观察历史拼接后输入网络。

### 3.1 环境与任务

| **Overcooked (OC)** | 3~4个智能体在7×7网格中协作完成切、搅拌、烘焙、上菜任务，包含资源约束和次优指令 | 3~4 | 约束型（"Don't use the left cutting board"）、次优请求（"Get me the lettuce"）、直接命令（"Move left"）|

### 3.2 对比方法

- **Naive**：朴素地直接使用指令条件奖励，无价值修正（对应图2顶部）

- **Vanilla Mac-IAC**：无指令条件训练的基准，作为基础任务性能上界

### 3.3 评估指标

- **Compliance（指令遵守率）**：在有指令的在线环境中评估，定义为遵循的指令数 / 给出的总指令数

## 6. 主要结论与发现

1. **MAVIC 是唯一同时实现高指令遵守率和高基础任务性能的方法**。在 OC 中达到 122.28 的基础性能（vs. Naive 的 3.62 和 Switch 的 104.00），同时保持 99% 的指令遵守率。

2. **Cross-Contamination 是朴素方法失败的根因**：
   - Naive 在 OC 和 WTD 中基础性能大幅下降（分别为 3.62 vs. 152.31 和 359.12 vs. 473.65），因为约束指令导致智能体避免使用必要资源
   - 在 BP 中 Naive 完全无法遵守指令（仅 5%），因为遵守指令需要放弃 +300 的大箱子奖励

3. **独立训练（Switch）不足以解决问题**：尽管避免了 cross-contamination，但 Switch 在 OC 中基础性能仍有下降（104 vs 152），且方差极高（σ=49.46），原因在于每回合单一任务上下文限制了探索多样性，且训练不稳定。

4. **价值修正的必要性取决于任务结构**：在 WTD 中，当指令仅重新排序现有奖励而不引入额外塑造项时，MAVIC 与基线性能相当；但当指令改变工具递送顺序时，MAVIC 以更低的方差实现了更高的正确递送率。

5. **分布外指令泛化有效**：MAVIC 对语义相似的新指令（如"the delivery is mine"）保持了 99.9% 的遵守率，说明冻结 BERT 编码器能够泛化到未见过的指令表述。

6. **不可行指令触发统一的保守行为**：当指令完全不可行时（如"fly"），智能体选择单一宏动作，反映了语言覆盖有限导致的泛化不足。

## 7. 优点

1. **问题定位精准**：首次系统性地识别并形式化了 MARL 中指令中断宏动作场景下的 **Bellman cross-contamination** 问题，揭示了价值函数耦合的根本原因。

2. **理论严谨**：提供了引理1和定理1的严格理论分析，证明 MAVIC 能够解耦指令条件价值函数，并保持统一策略的最优性。

3. **方案简洁优雅**：修正机制仅涉及修改引导目标，不改变 Bellman 算子结构，不限制转换或策略空间，可与现有 actor-critic 方法无缝集成。

4. **与 reward shaping 的本质区别**：明确澄清了 MAVIC 不同于传统势函数塑形，其修正具有策略依赖性，直接修改引导目标，在异步指令切换场景下具有独特优势。

5. **实验设计合理**：通过 Naive 和 Switch 两个基线分别暴露了价值污染和独立训练的局限性，清晰展示了 MAVIC 的优越性。

6. **指令编码轻量**：使用冻结 BERT 而非端到端训练，降低了计算成本，同时通过指令嵌入共享参数实现跨表述泛化。

## 8. 不足与局限

1. **算力信息缺失**：未报告训练时长、GPU 型号等硬件资源，实验可重复性和成本评估受限。

2. **部分可观测性挑战**：MAVIC 依赖准确估计指令条件价值函数 $V_\phi((h,c))$，在高度部分可观测环境下价值估计本身就很困难，修正项可能引入额外方差。

3. **语言泛化的局限性**：依赖 BERT 的冻结编码，假设语义等价的指令映射到相同嵌入，但对于多样或模糊的自然语言，编码质量可能下降。

4. **探索不平衡**：MAVIC 未显式解决稀有或不平衡指令的探索问题，可能导致某些指令类训练不足。

5. **指令集规模有限**：实验中的指令种类相对有限，未在极端规模（如数百种不同指令）的场景下验证。

6. **指令切换概率的外部依赖**：训练中指令切换概率 $\beta_c$ 需要预先设定，可能影响在不同实际指令频率下的表现。

7. **仅关注控制层面一致性**：MAVIC 聚焦于价值层面的修正，未涉及语言 grounding 问题，即智能体是否真正理解指令语义。
