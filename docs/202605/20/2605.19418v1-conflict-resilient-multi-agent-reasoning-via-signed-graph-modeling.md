## 摘要

基于大语言模型的多智能体系统（MAS）已展现出优于单 LLM 智能体的强推理和决策能力。然而，它们的性能经常因假设统一合作交互的朴素聚合机制而受到影响。通过仔细检查，我们观察到现有基于图的 MAS 框架：（1）当冲突信号出现而没有控制时会传播错误；且（2）缺乏对冲突智能体间关系的显式建模以及结构感知，无法识别可靠的交互模式。为弥合这一差距，我们引入 SIGMA，这是一种新颖的符号图感知多智能体推理框架，通过符号关系图显式捕获智能体之间的信任、冲突和中立关系。具体而言，给定一个查询，SIGMA 首先选择一组相关且多样的智能体，然后构建带有置信加权边的结构化符号交互图。推理通过冲突感知的符号消息传递进行，该传递强化来自可信智能体的信息同时抑制冲突信号，并以结构和冲突感知的加权聚合终止，产生根据符号图中净支持强度的全局一致且抗冲突的预测。在六个基准数据集上进行了广泛实验，跨越多个 LLM 主干和多样的多智能体配置，结果表明 SIGMA 一致地优于最先进的基线，在准确性和抗冲突性能方面都取得了显著提升。

## Abstract
LLM-based multi-agent systems (MAS) have demonstrated strong reasoning and decision-making capabilities that consistently surpass those of single LLM agents. However, their performance often suffers from naive aggregation mechanisms that assume uniformly cooperative interactions. Upon close inspection, we observe that existing graph-based MAS frameworks (1) propagate errors when conflicting signals arise without control, and (2) lack explicit modeling of conflicting inter-agent relations as well as structural awareness, failing to identify reliable interaction patterns. To bridge this gap, we introduce SIGMA, a novel SIgned Graph-informed Multi-Agent reasoning framework that explicitly captures trust, conflict, and neutral relations among agents via a signed relational graph. Specifically, given a query, SIGMA first selects a set of relevant and diverse agents, then constructs a structured signed interaction graph with confidence-weighted edges. Reasoning proceeds through conflict-aware signed message passing, which reinforces information from trustworthy agents while suppressing conflicting signals, and terminates with a structure- and conflict-aware weighted aggregation to yield globally consistent and conflict-resilient predictions. Extensive experiments on six benchmark datasets, across multiple LLM backbones and diverse multi-agent configurations, demonstrate that SIGMA consistently outperforms state-of-the-art baselines, achieving notable gains in both accuracy and conflict-resilient performance.

## 1. 核心问题与整体含义（研究动机和背景）

**问题背景：**
- 大语言模型（LLM）驱动的多智能体系统（MAS）在推理、规划和决策任务中展现出优于单智能体的能力
- 现有MAS方法可分为三类：链式（Chain）、树式（Tree）和图式（Graph）结构，其中图式MAS因能捕获复杂交互而备受关注

**核心问题：**
- **错误传播问题**：现有图式MAS假设所有交互都是合作的（交互矩阵非负），当智能体输出冲突时，错误信号会在多轮推理中不受控制地传播，导致误差累积
- **缺乏显式建模**：现有方法仅用相似度或通信链接构建图，无法显式建模智能体间的冲突关系，缺乏结构感知能力
- **鲁棒性不足**：在存在噪声或对抗性智能体的场景下，现有方法容易受到影响，无法识别可靠的交互模式

**研究动机：**
- 引入符号图（Signed Graph）建模，通过显式编码信任（Trust）、冲突（Conflict）和中立（Neutral）三种关系，使系统能够识别哪些智能体值得信任，哪些需要质疑

## 2. 论文提出的方法论

**框架名称：** SIGMA（SIgned Graph-informed Multi-Agent reasoning）

**核心思想：**
将多智能体交互从"均质合作"转变为"结构化推理动态"，通过符号图显式捕获交互的极性（正/负），使系统能够强化可信信号、抑制冲突信号。

**四阶段流程：**

### 阶段I：查询引导的智能体选择（Query-Guided Agent Selection）
- **目标**：从候选池中选择与查询相关且多样的智能体
- **评分函数**（公式5）：
  ```
  s(vi) = λ·rel(vi,Q) + (1-λ)/2 · [div(vi) + conf(vi)]
  ```
  - `rel`：语义相关性（通过嵌入相似度或LLM评判）
  - `div`：结构多样性（1 - 与其他智能体的相似度）
  - `conf`：置信度（自一致性或不确定性估计）
- 选择得分最高的k个智能体（k≈3~7）

### 阶段II：符号关系图构建（Signed Relational Graph Construction）
- **关系评估函数**（公式6）：
  ```
  sij = sign(f_eval(h_i^0, h_j^0))  # 极性：+1或-1
  wij = |f_eval(h_i^0, h_j^0)|      # 置信强度
  Aij = sij · wij                     # 符号邻接矩阵
  ```
- 划分正负邻居集合：N⁺_i（支持）、N⁻_i（冲突）
- **多跳邻域递归定义**（公式7）：
  - B^(ℓ)_i：ℓ跳平衡邻域（一致推理链）
  - U^(ℓ)_i：ℓ跳不平衡邻域（高阶冲突结构）

### 阶段III：冲突感知的符号消息传递（Conflict-Aware Signed Message Passing）
- 每个智能体维护两个分离的表示向量：
  - `hpos`：支持上下文下的语义状态
  - `hneg`：冲突上下文下的语义状态
- **更新规则**（公式8-9）：
  - 正表示从平衡邻域聚合支持信号，从不平衡邻域聚合校正信号
  - 负表示执行对称的极性反转操作
- 经过T轮迭代、每轮L层消息传递后，融合为最终表示

### 阶段IV：符号共识读出（Signed Consensus Readout）
- **加权聚合**：
  ```
  wi = Σ_j Aij / Σ_p Σ_q |Apq|  # 结构感知权重
  y = Σ_i wi · h_i^(T)
  ```
- 权重反映智能体在符号图中的净支持强度
- 最终由LLM根据聚合输出生成答案

## 7. 优点

**方法设计亮点：**
1. **符号图建模的创新应用**：首次将社会平衡理论引入LLM多智能体推理，显式区分信任/冲突/中立关系
2. **分离表示机制**：维护正/负两个表示向量，避免对称聚合导致语义崩溃
3. **多跳邻域递归**：通过B(ℓ)和U(ℓ)捕获高阶一致性和冲突结构
4. **结构感知权重**：基于净支持强度而非简单平均，提高聚合质量

**实验设计亮点：**
1. **全面的对比基线**：覆盖单智能体、多智能体、不同交互拓扑结构
2. **系统性的消融分析**：逐个模块验证贡献度
3. **多维度鲁棒性评估**：4类恶意智能体 × 5个比例梯度，接近真实对抗场景
4. **可视化案例分析**：定性地展示符号图如何隔离不可信智能体

**理论与实践结合：**
- 提供误差抑制引理和稳定性定理，为方法有效性提供理论支撑

## 8. 不足与局限

**实验覆盖局限：**
1. **任务类型单一**：主要评估具有明确答案的推理任务（选择题、数值计算、代码补全）
2. **未涉及开放性任务**：如创意写作、多轮对话、战略规划等"正确性"模糊的场景
3. **智能体池固定**：使用预定义的智能体角色配置，未探索动态角色生成

**方法局限性：**
1. **恶意智能体检测依赖初始评估**：若初始评估被对抗性攻击误导，后续符号图可能失效
2. **参数敏感性**：虽然λ在{0.4~0.6}范围内稳定，但不同任务可能需要调优
3. **符号图构建依赖相似度函数**：f_eval的实现选择可能影响关系判定的准确性

**潜在偏差风险：**
1. **过度抑制冲突**：在需要"建设性冲突"促进创新的场景中，冲突抑制可能损害多样性
2. **群体极化可能**：结构感知权重可能导致少数正确观点被压制

**可扩展性问题：**
1. **候选智能体数量N**：选择阶段复杂度O(N²d)，N增大时计算成本显著上升
2. **消息传递轮数T**：迭代次数增加时延迟累积，可能不适合实时应用

**应用边界：**
- 当前评估基于英文数据集，中文及其他语言场景未验证
- 主要在问答/代码生成任务上测试，其他类型任务（对话、摘要）的有效性待探索
