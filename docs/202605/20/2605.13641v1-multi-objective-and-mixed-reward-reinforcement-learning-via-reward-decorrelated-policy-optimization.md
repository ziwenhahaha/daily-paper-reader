## 摘要
复杂强化学习环境通常采用多任务和混合奖励设置。在这些场景中，奖励分布的异质性以及奖励维度之间的相关性经常破坏标量优势函数的构建。为解决这些问题，我们提出奖励去相关策略优化（Reward‑Decorrelated Policy Optimization，RDPO），一种明确针对上述两类失效模式的奖励处理方法。RDPO 首先利用幅度感知的分位数归一化（Magnitude‑Aware Quantile normalization）来稳定在二值、分数以及连续奖励下的提示级优势分配。随后，它在每个活跃奖励子空间内应用马哈拉诺比斯白化（Mahalanobis whitening）以在聚合前消除相关性冗余。将 RDPO 应用于 LongCat‑Flash 的后训练阶段后，RDPO 提升了指令遵循、写作质量以及对困难提示的鲁棒性，同时在推理和代码任务上仍保持较强的竞争力。

## Abstract
Complex reinforcement learning environments frequently employ multi-task and mixed-reward formulations. In these settings, heterogeneous reward distributions and correlated reward dimensions often destabilize the construction of scalar advantages. To address these challenges, we propose Reward-Decorrelated Policy Optimization (RDPO), a reward-processing method designed to explicitly target both failure modes. RDPO first utilizes Magnitude-Aware Quantile normalization to stabilize prompt-level advantage allocation across binary, fractional, and continuous rewards. It then applies Mahalanobis whitening within each active reward subspace to mitigate correlation redundancy prior to aggregation. When applied during the post-training of LongCat-Flash, RDPO enhances instruction following, writing quality, and robustness to hard prompts while remaining broadly competitive on reasoning and coding evaluations.

## 1. 核心问题与整体含义（研究动机和背景）

**研究问题：** 在复杂强化学习环境中，大语言模型（LLM）需要同时优化多个目标（如计算效率、人类偏好对齐、提示级约束等），但将多个异质性奖励信号聚合为单一标量优势时存在两个核心失效模式：

- **奖励尺度异质性问题：** 不同任务类型的奖励分布差异巨大（可能是二值、分数或连续型），导致标准化后优势集中在少数rollout上，造成训练不稳定。
- **奖励维度相关性问题：** 同一任务内的多个奖励维度（如正确性分数与长度奖励、规则遵循与评分奖励）往往存在相关性，直接求和会造成信息重复计数或抵消。

**研究背景：** 现有方法如GRPO直接对原始奖励求和，对异质性敏感；GDPO采用Z-score归一化改善了尺度问题，但仍对非高斯分布敏感且无法处理维度间相关性。

## 2. 论文提出的方法论

### 2.1 核心思想

RDPO提出**两阶段奖励处理管道**：
1. **Magnitude-Aware Quantile (MAQ) 归一化**：使提示级优势分配对二值奖励、并列、高度偏斜分布和异常值更鲁棒。
2. **Mahalanobis白化**：在每个活跃奖励子空间内降低维度间相关性冗余后再聚合。

### 2.2 关键技术细节

#### (a) 有效信息效率（Effective Information Efficiency）

论文定义指标 η_eff = η_proj × η_corr 来诊断混合奖励聚合质量：
- η_proj：衡量聚合方向与等权重投影的对齐程度
- η_corr：衡量聚合后保留的独立信息量（基于绝对相关矩阵）

#### (b) MAQ归一化流程

对每个提示 i 和奖励维度 k（给定G个rollout得分 r1 ≤ r2 ≤ ... ≤ rG）：

1. **对数压缩间隙计算**：
   ```
   gap_j = log(1 + |r_{j+1} - r_j| / (β · σ_global))
   ```
   其中 σ_global 是全局四分位距（IQR），β > 0 控制压缩强度。

2. **CDF分配**：归一化间隙后按累积比例分配CDF位置 u(j)。

3. **逆正态映射**：
   ```
   A(j) = Φ^{-1}(u(j))
   ```
   其中 Φ^{-1} 是标准正态分布的逆CDF。

该方法比Z-score归一化更鲁棒，因为对数压缩自然限制极端异常值影响，同时对小幅密集间隙保持近似线性。

#### (c) Mahalanobis白化

1. **白化变换**：
   ```
   W(i,j) = Σ^{-1/2} · A(i,j)
   ```
   其中 Σ^{-1/2} = UΛ^{-1/2}U^T 来自协方差矩阵的特征分解。

2. **运行协方差估计**（EMA）：
   ```
   Σ̂_t = (1 - α)Σ̂_{t-1} + αΣ_batch
   ```
   α 是EMA衰减率，在线跟踪缓慢演化的相关结构。

3. **子空间白化**：仅对观察到的奖励子空间应用白化（异质任务中不同任务激活不同子空间）。

#### (d) 最终优势计算

```
A_sum(i,j) = Σ_k W(i,j)_k = 1^T Σ̂^{-1/2} A(i,j)
```
后接批归一化得到最终优势估计。

## 3. 实验设计

### 3.1 数据集与场景

**训练场景：**
- 在四任务混合上训练：数学推理、代码生成、指令遵循、一般写作
- 每任务使用2-3个奖励维度，形成4个活跃奖励子空间：
  - {math, length}
  - {code, length}
  - {ifeval, rubrics}
  - {length, rm, rubrics}

**奖励类型：**
- Rubrics Reward：基于生成奖励模型的细粒度评分（加权平均，必需项失败则归零）
- IFEval Reward：规则验证器评估格式、内容、行为约束
- Math Reward：精确匹配或等价性检查
- Code Reward：功能正确性评估
- RM Reward：独立奖励模型的软偏好信号（线性缩放到[0,1]）
- Length Reward：鼓励简洁（阈值比较+二次惩罚）

**条件奖励处理：** 辅助信号（RM、Length）仅在核心任务满足基本要求时才生效（条件阈值0.5）。

### 3.2 Benchmark设置

**评估分为四个聚类：**

1. **指令遵循：** IFEval、GuideBench、SOP-Maze

3. **写作与Arena评估：** WritingBench、ArenaHard-v2（AH-Hard、AH-Creative）
