## 摘要

协调图是合作型多智能体强化学习（MARL）的核心抽象，但现有稀疏图学习器缺乏理论支撑的机制来决定哪些边应该存在以及每条边应该携带多少信息。现有方法依赖于启发式标准，对学习到的拓扑没有形式化保证，也没有原则性的方法来为结构上不同的智能体关系分配不同的通信容量。为了解决这一问题，我们提出了异构信息瓶颈协调图（HIBCG），它学习了一个组感知的稀疏图，其中边存在性和消息容量都是有理论依据的。以图信息瓶颈（GIB）为基础工具，HIBCG 首先构建了一个组对齐的块对角先验，为边保留提供了一个封闭形式的准则——决定哪些边应该存在以及每个组块应该以什么密度存在——然后在所得拓扑上控制每个智能体的特征带宽，将消息压缩到仅保留任务相关内容。我们证明组对齐先验严格收紧了拓扑学习的变分界，目标可以按组块分解，实现差异化边控制，容量分配遵循注水原理。

## Abstract
Coordination graphs are a central abstraction in cooperative multi-agent reinforcement learning (MARL), yet existing sparse-graph learners lack a theoretically grounded mechanism to decide which edges should exist and how much information each edge should carry. Current methods rely on heuristic criteria that offer no formal guarantee on the learned topology, and no principled way to allocate different communication capacities to structurally different agent relationships. To address this, we propose Heterogeneous Information-Bottleneck Coordination Graphs (HIBCG), which learns a group-aware sparse graph in which both edge existence and message capacity are theoretically justified. With the graph information bottleneck (GIB) serving as the underlying tool, HIBCG first constructs a group-aligned block-diagonal prior that provides a closed-form criterion for edge retention -- determining which edges should exist and at what density per group block -- and then controls per-agent feature bandwidth on the resulting topology, compressing messages to retain only task-relevant content. We prove that the group-aligned prior strictly tightens the variational bound on topology learning, that the objective decomposes per group block, enabling differential edge control, and that capacity allocation follows a water-filling principle.

### 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：合作型多智能体强化学习（MARL）中，智能体需要交换任务相关信息以协调局部决策。协调图（Coordination Graph）通过指定哪些智能体对之间交换信息来选择性地实现通信。

- **核心问题**：现有稀疏图学习方法存在两大局限：
  1. **边存在性缺乏理论支撑**：使用启发式标准（如固定阈值、Top-K选择、注意力分数等）决定边的取舍，无法形式化地决定组内与组间连接的密度。
  2. **通信容量缺乏结构化控制**：即使学习了拓扑，现有方法也缺乏对不同智能体关系分配不同消息带宽的原则性方法。

- **研究动机**：需要一种原则性框架，能够**联合学习图拓扑**并**控制该拓扑上的特征带宽**，使通信只在结构上不同的地方分配不同容量。

### 2. 论文提出的方法论

#### 2.1 核心思想

提出**异构信息瓶颈协调图（HIBCG）**，将 MARL 协调建模为异构图学习问题：

- 使用**图信息瓶颈（GIB）**作为工具，耦合两个控制：
  - **拓扑学习**（AIB）：使用组对齐的块对角先验决定哪些边应该存在。
  - **消息控制**（XIB）：在已学得的拓扑上调控每个智能体的特征带宽。

#### 2.2 关键技术细节

**三层架构**：

1. **Stage 1 - 组感知初始图**：基于 GACG 产生初始图 $A^{(0)}$ 和组划分 $G_t$，通过高斯分布采样 $z^{(0)}_A \sim N(\mu_t, \Sigma_t^A)$。

2. **Stage 2 - 异构结构剪枝**：每层使用高斯结构编码器：
   - $P(Z^{(l)}_A | A^{(0)}, Z^{(l-1)}_X) = N(\mu^{(l)}_A, \Sigma^{(l)}_A)$
   - **块对角先验**：$Q(Z^{(l)}_A) = N(0, \text{blkdiag}(\Sigma^{(l)}_{0,g})_{g \in G'})$
   - **封闭形式 KL 惩罚**：$\text{KL}(P \parallel Q) = \frac{1}{2}\sum_{d=1}^k \left(\frac{\sigma_d^2 + \mu_d^2}{\sigma_0^2} - 1 + \log\frac{\sigma_0^2}{\sigma_d^2}\right)$
   - **异构剪枝分配**：设置 $\lambda_{\text{cross}} > \lambda_{\text{intra}}$ 使组间边更稀疏、组内边更密集。

3. **Stage 3 - 每智能体消息压缩与决策**：
   - $Z_{\text{out}}^X = \mu_X + \sigma_X \odot \epsilon$，使用对角高斯变分编码器。
   - 每智能体 XIB 惩罚：$\text{XIB}(i) = \mathbb{E}_D[\text{KL}(N(\mu_{X,i}, \text{diag}(\sigma_{X,i}^2)) \parallel N(0, \sigma_{X,0}^2 I))]$
