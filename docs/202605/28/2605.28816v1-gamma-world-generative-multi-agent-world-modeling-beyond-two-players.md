## Abstract
World models for interactive video generation have largely focused on single-agent settings, where future observations are generated from a single control signal. However, many generated environments require multi-agent interaction: multiple players, robots, or embodied agents act simultaneously within a shared space. Scaling world models to such settings requires a principled multi-agent design: agents should remain independently controllable, permutation-symmetric, and support efficient inference while maintaining consistency across time and perspectives. In this paper, we present our generative multi-agent world model for interactive simulation. It introduces Simplex Rotary Agent Encoding, a parameter-free extension of 3D RoPE that represents agents as vertices of a regular simplex in rotary angle space. This gives each agent a distinct phase while making all agents permutation-equivalent, enabling scalable agent identity without learned per-slot identities or a fixed agent ordering. To avoid dense all-to-all attention across agents, we further propose Sparse Hub Attention, where learnable hub tokens mediate token interaction across agents, reducing cross-agent attention cost from quadratic to linear in the number of agents. For real-time rollout, we distill a full-context diffusion teacher into a causal student that generates temporal blocks sequentially with KV caching, enabling action-responsive generation at 24 FPS. Experiments in multiplayer virtual environments show that our model improves video fidelity, action controllability, and inter-agent consistency over slot-based and dense-attention baselines, while generalizing from two to four players without additional training.

## 2. 论文提出的方法论

### 2.1 核心思想

提出 **γ-World**，一个生成式多智能体世界模型，通过三大技术创新解决上述挑战：
- **Simplex Rotary Agent Encoding**：参数无关的智能体身份编码；
- **Sparse Hub Attention**：稀疏的跨智能体通信机制；
- **教师-学生蒸馏**：支持 KV 缓存的实时流式推理。

### 2.2 Simplex Rotary Agent Encoding（单纯形旋转智能体编码）

- **思想**：将每个智能体表示为 $d_p/2$ 维旋转角空间中的正单纯形顶点，而非标量索引或可学习向量。

- **数学形式**：
  - 单纯形顶点构造：$s_v = \sqrt{\frac{V}{V-1}} Q\left(e_v - \frac{1}{V}\mathbf{1}\right) \in \mathbb{R}^{d_p/2},\quad v = 1, \ldots, V$
  - 其中 $V$ 为单纯形池大小（训练时选定），$e_v$ 为 one-hot 向量，$Q$ 为从 $\mathbb{R}^V$ 零均值子空间到 $\mathbb{R}^{d_p/2}$ 的线性等距映射。
  - 顶点性质：$\|s_v\|_2 = 1$，且对任意 $v \neq v'$ 有 $\|s_v - s_{v'}\|_2^2 = \frac{2V}{V-1}$（所有智能体对等距）。

- **智能体旋转角度**：$\theta_p = \alpha \cdot s_{\pi(p)}$，其中 $\pi$ 为注入映射，$\alpha > 0$ 控制分离强度。

- **4D RoPE 算子**：$R_{4D}(t,p,h,w) = \text{diag}(R_t(t), R_{simp}(\pi(p)), R_h(h), R_w(w))$

- **优势**：参数无关、无需固定槽位顺序、支持训练时随机分配顶点，支持从 V 个顶点中任选 P 个智能体，无需重新训练。

### 2.3 Sparse Hub Attention（稀疏枢纽注意力）

- **动机**：稠密跨智能体注意力的复杂度为 $\mathcal{O}(P^2 n^2 L^2)$，随智能体数二次增长。

- **设计**：引入少量可学习的枢纽（hub）令牌作为紧凑的共享通信状态：
  - 智能体令牌仅允许：① 同一智能体流内的自注意力；② 与 hub 令牌的注意力；
  - Hub 令牌允许：与所有智能体及 hub 自身的注意力；
  - 不同智能体之间的直接注意力被遮蔽，信息通过两跳路径传递：智能体 → hub → 智能体。

- **注意力遮蔽矩阵**：$\mathcal{M}_{hub}(i,j) = \mathbb{1}[\rho(i) = \rho(j) \lor \rho(i) = hub \lor \rho(j) = hub]$
- **与块因果遮蔽结合**：$\mathcal{M}(i,j) = \mathbb{1}[b(j) \leq b(i)] \cdot \mathcal{M}_{hub}(i,j)$

- **复杂度**：降低至 $\mathcal{O}(PnL(nL + nK)) + \mathcal{O}(nK(PnL + nK))$，对智能体数 $P$ 线性。

### 2.4 三阶段训练与蒸馏流程

1. **双向教师模型**：全序列双向注意力 + 统一噪声水平，用于高质量条件去噪训练；
2. **因果学生模型**：块因果注意力 + Sparse Hub Attention + Diffusion Forcing（每块独立噪声水平），支持多步自回归rollout；
3. **Self-Forcing 蒸馏**：将因果学生蒸馏为少步生成器（4步去噪），使用 DMD（Distribution Matching Distillation）损失，teacher 提供条件分布匹配信号，学生在自 rollout 上训练（生成块写入 KV 缓存后作为后续块的历史条件）。

### 2.5 动作条件注入

- 共享动作编码器 $f^a$ 将每智能体每帧动作映射为隐藏特征 $u_t^p = f^a(a_t^p)$；
- 通过层特定投影生成动作偏置 $\beta_t^{p,\ell} = g^\ell(u_t^p)$，广播到该智能体该帧所有空间令牌。

### 2.6 KV 缓存流式推理

- 每智能体维护独立的 KV 缓存（最近 24 个潜在帧窗口）；
- 共享 hub 的 KV 缓存记录所有智能体历史和 hub 自身历史；
- 生成新块时，各智能体读取自身历史和 hub 缓存，hub 读取所有智能体缓存和历史 hub。

## 7. 优点

1. **置换对称性理论保证**：通过正单纯形顶点编码，从几何上严格保证了所有智能体对的等距性，从根本上解决了可学习槽位 ID 的不对称问题。

2. **参数无关的可扩展性**：单纯形池大小 $V$ 在训练时固定，但推理时可激活任意 $P \leq V$ 个智能体，无需重新训练或添加新参数。

3. **线性复杂度注意力**：Sparse Hub Attention 将跨智能体交互从二次方降低到线性方，显著提升大规模多智能体场景的推理效率。

4. **两跳信息流设计**：通过 hub 令牌的间接通信路径（智能体→hub→智能体）既保留了跨智能体信息传递，又避免了不必要的全连接开销，是一种优雅的信息路由设计。

5. **完整的实时部署方案**：从双向教师到因果学生再到蒸馏学生，三阶段流程完整覆盖了从高质量训练到实时推理的全链路。

6. **消融实验严谨**：系统地验证了每个组件的独立贡献，从输入组织、智能体编码到注意力机制逐层剥离。

7. **真实世界迁移验证**：不仅在虚拟游戏中验证，还测试了真实机器人协调，展示了方法的多领域适用性。

## 8. 不足与局限

1. **数据集覆盖有限**：实验主要集中在 Minecraft 风格的游戏环境和有限的机器人场景，未在更复杂、异构、长时域的多智能体环境中验证（如自动驾驶多车交互、大规模人群模拟等）。

2. **智能体数量扩展受限**：单纯形池大小 $V$ 受限于旋转头维度 $d_p/2 + 1$，当智能体数量非常大时（如数十个），可能需要更大的旋转带或层次化分组策略，论文未深入探索。

3. **缺乏物理一致性约束**：γ-World 未显式建模 3D 几何或物理约束，长时域 rollout 可能积累视觉不一致（如物体穿透、遮挡关系错误等）。

4. **Hub 容量调优依赖实验**：Hub 令牌数量 $K$ 的选择（实验中发现 $K=8$ 为较好平衡点）依赖手工调参，缺乏理论指导。

5. **两跳通信延迟**：跨智能体信息必须经过 hub 中转，在需要快速直接交互的场景（如碰撞检测）中可能引入信息瓶颈。
