## 摘要
多智能体基础模型部署中，智能体自我报告的置信度常与真实准确率背离，尤其在困难任务上呈负相关，导致协调者难以判断信任哪个响应。传统设计时校准依赖固定修正和保留数据，无法应对实际任务中的分布变化。本文提出MARGlN，一种从任务流在线学习每个智能体、每个置信度带校准因子的方法，无需模型访问或重训练，在19个模型、8个基准、超5万观察中实现3-6倍校准误差降低，将困难任务的成对解析率从45-56%提升至70-89%。 基础模型置信度系统性错误校准，在困难任务上甚至与准确率反相关，导致多智能体协调时无法依赖置信度选择最佳响应。

## Abstract
Foundation model agents increasingly operate in multi-agent deployments where a coordinator must decide which agent's response to trust. The standard approach weights agents by their self-reported confidence, but recent evidence shows that foundation model confidence is systematically mis-calibrated and, on hard tasks, inversely correlated with accuracy. Design-time calibration methods (temperature scaling, Platt scaling, histogram binning) cannot address this problem because they fit a fixed correction to held-out data and degrade under distribution shift.   We present MARGIN (Multi Agent Runtime Grading via Incremental Normalization), an online calibration method that learns per-agent, per-confidence-band calibration factors from the task stream itself, requiring no model access, no held-out data, and no retraining. MARGIN uses symmetric exponentially weighted moving averages with Bayesian shrinkage blending, and has three hyperparameters with robust defaults. Across 19 foundation models, 8 benchmarks, and over 50,000 observations, MARGIN achieves 3-6x lower calibration error than the best design-time baseline under distribution shift. In multi-agent selection, raw verbalized confidence produces pairwise resolution worse than random (45-56%) on hard benchmarks. MARGIN corrects this completely, raising pairwise resolution to 70-89% and surpassing the always-best-model oracle on three of four benchmarks. Six formal propositions characterize convergence, tracking speed, and the optimality of symmetric updates for non-strategic agents, with all predictions illustrated empirically.

## 1. 论文核心问题与整体含义（研究动机与背景）

- **核心问题**：在多智能体基础模型部署中，协调者需要决定信任哪个智能体的响应。传统方法根据智能体自我报告的置信度加权，但基础模型的置信度存在系统性错误校准，尤其在困难任务上与准确率呈**负相关**（即越没把握的模型反而越自信）。
- **关键危害**：在困难基准（如BigCodeBench、LiveCodeBench、CodeContests）上，使用原始置信度的成对解析率仅为44.8%–55.5%，**低于随机选择**（50%），导致多智能体协调系统性能严重退化。
- **现有方法缺陷**：设计时校准方法（温度缩放、Platt缩放、直方图分箱）依赖固定的保留数据集进行一次性修正，当部署分布与校准分布不一致时，修正效果急剧下降（ECE从个位数升至37%–63%）。
- **实际需求**：多智能体系统运行在分布持续变化的环境中（任务类型演变、用户行为变化、模型更新），需要一种能够**在线学习、无需模型访问、无需重训练**的校准方法。

## 2. 论文方法论

### 核心思想

MARGIN（Multi-Agent Runtime Grading via Incremental Normalization）将每个智能体的置信度校准问题转化为**在线统计估计问题**。它从任务流本身持续观察每个智能体在不同置信度水平下的实际准确率，动态计算校准因子，无需任何模型内部信息。

### 关键技术细节

1. **置信度带分层跟踪**：
   - 将置信度范围 [0,1] 划分为 K=3 个等宽区间（[0, 1/3)、[1/3, 2/3)、[2/3, 1]）
   - 为每个（智能体，带）对维护两个EWMA估计：准确率估计 `â_{i,k}` 和平均置信度 `¯c_{i,k}`
   - 校准因子 `γ_{i,k} = â_{i,k} / ¯c_{i,k}` 反映"自称90%置信时实际正确的频率"

2. **对称EWMA更新**：
   - `â_{i,k} ← (1-α)·â_{i,k} + α·o_{i,t}`（正确取1，错误取0）
   - `¯c_{i,k} ← (1-α)·¯c_{i,k} + α·c_{i,t}`
   - 学习率 α=0.04，提供约25个观测的有效记忆窗口
   - **对称性是关键**：基础模型代理是非策略性的，其误差是认知性的（而非战略性操纵），对称EWMA是唯一无偏估计

3. **贝叶斯收缩混合**：
   - 在冷启动阶段，当某（智能体，带）的观测较少时，将该带的校准因子向全局模型级因子收缩
   - `γ_eff = (n_{i,k} / (n_{i,k} + ks))·γ_{i,k} + (ks / (n_{i,k} + ks))·γ_{i,·}`
   - 收缩常数 ks=100，使新进入的智能体能快速达到可靠估计（50个观测后ECE从13.67%降至7.74%）

4. **校准置信度计算**：
   - `˜c_{i,t} = γ_eff · c_{i,t}`
   - 用于多智能体选择时，采用置信度加权投票

### 算法流程（核心步骤）

1. 对每个任务，接收智能体预测和置信度，将其路由到对应置信度带
2. 根据该带的有效校准因子计算校准置信度
3. 选择校准置信度最高的响应
4. 观察真实结果后，更新对应带的准确率和置信度EWMA估计

### 理论贡献

- 6个形式化命题证明：
  - EWMA的指数衰减特性
  - 校准收敛性
  - 追踪速度（偏移后约75个观测恢复）
  - 偏差-方差权衡（最优 α* = O(δ^{2/3})）

## 7. 优点

1. **在线适应性强**：从任务流持续学习，自然适应分布变化，无需重新训练或保留数据。
2. **无模型访问要求**：作为黑盒方法运作，只需观测预测、置信度和结果，适用于任何API模型或私有模型。
3. **理论严谨**：6个形式化命题提供收敛、追踪速度和最优性保证，所有预测均被实验验证。
4. **对称EWMA的洞察**：发现基础模型代理是非策略性的，其误差是认知性的，对称更新是无偏且最优的——这一洞察对声誉系统设计有广泛意义。
5. **鲁棒超参数**：仅3个超参数（α=0.04、K=3、ks=100），默认值在广泛场景下表现良好。
6. **冷启动处理有效**：贝叶斯收缩使新智能体在50个观测后达到7.74% ECE，相比无收缩减少41–44%。
7. **动态池鲁棒性**：智能体退出、冷启动和滚动替换场景下性能稳定，无需额外机制。
8. **双模态支持**：支持口头置信度和一致性置信度两种模态，无需针对模态调参。

## 8. 不足与局限

1. **算力信息缺失**：未报告GPU型号、数量或训练时长，难以评估计算成本和可行性。
2. **二元结果假设**：需要可验证的正确/错误信号，限制了对开放式生成或主观任务的应用。
3. **置信带选择原则性不足**：K=3是经验选择，缺乏理论依据，更细粒度的分区在长时运行中可能更优。
4. **带内i.i.d.假设**：假设同一置信带内的观测是独立同分布的，但实际中连续问题可能相关（如主题聚类），虽然100次洗牌的稳健性分析表明影响有限。
5. **长尾分布敏感性**：EWMA有效记忆窗口约25个观测，对极端罕见的困难任务可能跟踪不足。
6. **扩展性未充分验证**：实验使用19个模型，真实系统可能有数百或数千个代理和用户，带数过多可能导致稀疏性问题。
7. **校准-性能权衡**：在分布稳定时，设计时方法可能略优（偏差更小），但这种情况在实践中罕见。
8. **战略代理的未来风险**：随着模型能观察和响应声誉信号，当前对称方法可能需要调整为非对称形式。
