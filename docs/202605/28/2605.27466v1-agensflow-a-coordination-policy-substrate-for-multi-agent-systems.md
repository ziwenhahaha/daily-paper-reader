## 摘要
基于大型语言模型（LLM）构建的多智能体系统需要许多难以事先确定的协调选择：调用哪种技能协议、哪个智能体角色应执行子任务、将哪个模型绑定到每个角色、角色之间如何交互、何时使用检索或验证，以及何时完全跳过某一步。这些选择与任务模式和操作约束相互影响，因此静态的流程管道和一次性的模型比较只能提供设计空间的有限视角。本文提出了 AgensFlow，一个开源框架，将多智能体协调视为部分可观测条件下的在线策略学习问题。该框架使协调决策可观察、可从重复轨迹中学习，而不是将技能、角色、模型、拓扑结构和评估选择视为固定的流程设计。AgensFlow 在两个语料库上进行了评估：分布式系统事故任务和安全公告任务。评估显示三个主要结果：在协调密集类任务上，学习路由比固定流程基线达到了更高质量的运行点；skip:X 将拓扑压缩隔离为底层的重要组成部分；以及热启动策略图可以在保持稳态质量的同时降低探索成本。总体而言，结果表明，学习的、可审计的路由可以改进相对于静态连线的协调密集型多智能体工作流。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

## 2. 论文方法论

### 核心思想
框架将多智能体编排建模为部分可观测的顺序决策过程，通过可折叠的任务签名（folded signature）实现状态抽象，允许多个相关轨迹共享价值估计。

### 关键技术细节

#### 2.1 状态表示（Folded Task Signature）
- **公式**：$s_t = \phi(x_t, h_t, b_t) = (\ell_t, z_t, \beta(\hat{c}_t), \beta(\hat{u}_t), \beta(\hat{r}_t), \beta(\hat{e}_t))$
- **组成要素**：
  - $\ell_t$：任务模式标签（straightforward, evidence_heavy, ambiguous, contradictory, high_risk, exploratory）
  - $z_t \in \{0,1\}^7$：二进制交接掩码（goal, subproblem, evidence, critique, verification, draft_answer, merged_answer）
  - $\beta(\cdot)$：四个信念估计（正确性 $\hat{c}_t$、不确定性 $\hat{u}_t$、矛盾风险 $\hat{r}_t$、证据充分性 $\hat{e}_t$）

#### 2.2 动作空间
- **集合**：$a_t \in A(s_t) = \{\text{invoke}(k,m)\} \cup \{\text{skip}:X\} \cup \{\text{terminate}\}$
- **创新点**：`skip:X` 将拓扑省略本身变成可学习的决策，支持对可选单元的选择性跳过。

#### 2.3 奖励函数
- **公式**：$r(t) = w_q Q(t) - w_c C(t) - w_\rho \rho(t)$
  - $Q(t)$：RelativeJudge 质量分数（相对轨迹评分）
  - $C(t)$：归一化 token 成本
  - $\rho(t)$：重试/失败惩罚
- **默认权重**：$w_q = 1.0, w_c = 0.3, w_\rho = 0.15$

#### 2.4 策略学习：可靠性感知 UCB1
- **评分公式**：$\text{score}(s,a) = \bar{r}(s,a) + c_s \sqrt{\frac{\log(N_s+1)}{N_{s,a}}} - \lambda f(s,a)$
  - $\bar{r}(s,a)$：均值奖励
  - $N_s$：签名访问次数，$N_{s,a}$：动作访问次数
  - $f(s,a)$：记录失败率
- **探索退火**：$c_s = \max(0.5, 1.4 \cdot 2^{-N_s/50})$，初始 1.4，50 次访问后半衰，地板 0.5

## 6. 论文主要结论与发现

### 结论 1：学习路由优于固定基线
- 在协调密集类任务上显著提升（C3: +0.181, C8: +0.156, C7: +0.131）。
- 程序性任务类统计持平（C6: +0.004），在 C1 和 C5 上有小幅下降。
- 三法官审计质量：main 0.817 vs baseline 0.749（+0.068）。

### 结论 2：skip:X 隔离拓扑压缩
- 启用 skip:X 后，框架学会对可选单元选择性省略。
- 冷启动学习轨迹显示：初期压缩成本 → 质量下降后重新平衡 → 最终找到平衡点。

### 结论 3：热启动降低探索成本
- 使用约 10% 更少 token（稳态）至 21% 更少 token（全程）。
- 稳态质量基本持平（三法官：+0.012），全程质量几乎相同（+0.003）。
- 在 C5（+0.081）和 C7（+0.049）上保留针对性质量提升。

### 结论 4：奖励信号鲁棒性是一等公民
- 单法官信号下热启动质量优势被高估（+0.052 vs 三法官 +0.012）。
- 跨法官审计揭示奖励设计对结论的实质影响。
- 奖励审计应作为协调底层的一部分，而非外部评估步骤。

## 7. 优点

### 方法论亮点
1. **状态抽象创新**：通过折叠签名实现价值共享，避免逐提示记忆化，支持可扩展的在线学习。
2. **拓扑作为动作空间**：`skip:X` 将拓扑选择从配置变为可学习决策，突破传统管道限制。
3. **可靠性感知探索**：UCB1 + 失败惩罚 + 退火探索，平衡利用与探索，防止脆弱动作过度选择。
4. **奖励可审计性**：RelativeJudge + 交叉法官 + 轴分解 + 置信度加权，将奖励信号本身纳入学习闭环。

### 实验设计亮点
1. **跨域验证**：同一框架在完全不同领域（分布式系统 vs 安全公告）验证迁移能力。
2. **热启动价值**：首次系统量化策略图跨域迁移的探索成本节省。
3. **审计必要性论证**：通过实验揭示单法官 vs 多法官对结论的实质影响。

## 8. 不足与局限

### 实验覆盖局限
1. **拓扑类型单一**：仅测试线性 + 跳过拓扑，未覆盖并行、蜂群、分支联盟、层级分解等更复杂拓扑。
2. **特征分类法未变**：跨域迁移假设特征空间相似，未测试不同抽象粒度的影响。
3. **统计表达缺失**：未报告置信区间或显著性检验，结论可靠性难以量化。

### 方法论局限
1. **信念估计启发式**：正确性、不确定性等信念基于经验规则更新，非数据驱动学习。
2. **动作空间依赖预定义**：legal action set 由激活计划、已完成字段、预算状态和终止规则决定，仍依赖人工设计边界。
3. **终止非策略选择**：terminate 动作不由路由器直接选择，仅隐式触发。

### 偏差风险
1. **法官偏见**：即使跨家族审计，LLM 法官仍可能系统性地偏好特定轨迹特征（如更长、更详细）。
2. **任务同质性**：两语料库均为合成任务，可能无法泛化到真实世界长尾分布。

### 应用限制
1. **实时性约束**：在线学习需要足够轨迹密度，冷启动阶段可能性能不足。
2. **资源密集**：6 层架构 + 相对评估，部署成本高于简单管道。
3. **任务类型敏感**：对协调密集类任务效果显著，程序性任务无显著提升甚至可能下降。
