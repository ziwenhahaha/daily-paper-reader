## 摘要
组合泛化仍是深度强化学习（DRL）面临的核心挑战。经典规划通过显式的关系描述提供了一个简单而具挑战性的研究环境，无需从感知中学习。在稀疏奖励领域，标准强化学习通过实时搜索进行的探索效果不佳，而基于学习的规划方法通常依赖专家演示、后见重新标记或从目标状态随机游走。相比之下，规划器依靠最佳优先搜索方法（如 A*）从头解决问题。我们提出了一种自改进的 WA* 学习框架，结合由关系图神经网络表示的值启发式方法：启发式引导搜索，生成的搜索数据通过 Q 学习更新启发式方法。该循环产生能够充当通用策略的启发式方法，并在无需搜索的情况下解决新实例，而 DRL 在此类情况下通常会失败，正如我们在 Sokoban、PushWorld、The Witness 以及 2023 年国际规划竞赛基准测试等谜题上所展示的那样。值得注意的是，我们展示了强大的零样本泛化能力：例如，在少于 30 块的 Blocksworld 实例上训练的启发式方法，无需搜索即可成功解决具有 488 块的实例。

## Abstract
Combinatorial generalization remains a central challenge in Deep Reinforcement Learning (DRL). Classical planning provides a simple yet challenging setting to study this problem through explicit relational descriptions, without requiring learning from perception. In sparse-reward domains, standard RL exploration via real-time search is ineffective, and learning-based planning methods often rely on expert demonstrations, hindsight relabeling, or random walks from the goal state. In contrast, planners rely on best-first search methods such as $\mathrm{A}^\star$ to solve problems from scratch. We propose a self-improving $\mathrm{WA}^\star$ learning framework in combination with a value heuristic represented by a Relational Graph Neural Network: the heuristic guides search, and the resulting search data updates the heuristic via $Q$-learning. This loop yields heuristics that can function as general policies and solve new instances even without search, where DRL otherwise fails, as we show on puzzles such as Sokoban, PushWorld, The Witness, and the 2023 International Planning Competition benchmarks. Notably, we demonstrate strong zero-shot generalization: For example, heuristics trained on Blocksworld instances with fewer than 30 blocks successfully solve instances with 488 blocks without search.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：深度强化学习（DRL）在组合泛化（combinatorial generalization）方面存在重大挑战，即训练好的策略或价值函数难以泛化到训练分布之外的、但具有共同问题结构的新实例。在经典规划（Classical Planning）领域，这种泛化需求尤为突出，因为规划实例会随物体数量、初始状态和目标的变化而产生系统性分布偏移。
- **研究背景**：在稀疏奖励（sparse-reward）的规划域中，标准RL依赖的实时搜索（real-time search）效率极低；而基于学习的规划方法通常需要专家演示、后见经验重放（HER）或从目标状态随机游走来获取信号。相比之下，传统规划器使用最佳优先搜索（如WA⋆）可以有效利用问题结构，但缺乏从经验中学习的机制。如何将自改进的搜索（best-first search）与学习有效结合，是一个核心难题。

## 2. 论文提出的方法论

- **核心思想**：提出 **GSP（Generalized Search for Planning）** 框架，实现“搜索即学习、学习即搜索”的自改进循环。核心在于：用学到的Q函数引导WA⋆搜索，搜索产生的过渡数据又反过来通过Q-learning更新Q函数，形成正向循环。
- **关键技术细节**：
    - **搜索阶段（WA⋆ Search Episode）**：维护搜索树、最佳优先队列F和回放缓冲区D。每次弹出 frontier 中评分最高的 `(s, a)` 对（评分函数 `f(s,a) = g(s) + w·Q_θ(s,a)`），生成后继状态 `s' = T(s,a)`。对死端状态赋予固定惩罚返回 `R_⊥`，对达到目标的路径回溯并赋予真实返回下界 `R(st, at)`，其余非终止状态存入未标注样本。
    - **Q-learning更新**：对死端样本回退到固定惩罚 `y = R_⊥`；对目标路径样本，取 `y = max{R, ˆy(s,a)}` 以结合搜索发现的实际解下界与 bootstrapped 目标，防止回退；其余采用标准一步bootstrapping目标 `ˆy(s,a) = -1 + max_{a'} Q_θ(s', a')`。
    - **Q函数表示**：采用 **关系图神经网络（R-GNN）**。将状态 `s`（一组真原子）和目标 `g`（一组文字）合并为关系集合 `R_{s,g}`，对每个适用动作添加辅助“动作原子”。通过 `L` 层消息传递，聚合来自原子中所有角色的信息，最终通过聚合-拼接-MLP 的方式输出 `Q_θ(s,a)` 值，实现对不同动作模式跨状态泛化的评分能力。
    - **实例采样策略**：维护三个实例池（unsolved/solved/satisficed），按指数加权采样，优先在次优解实例上训练以获取最大信息量。

## 6. 主要结论与发现

- **强零样本泛化**：训练于少于30块Blocksworld实例的启发式方法，无需搜索即可直接解决含488块的新实例（100%覆盖率）。
- **搜索显著优于纯DRL**：在Sokoban上，GSP_π（667/1000）大幅超越Lifted HER（309/1000），且使用GSP_WA⋆可达到完美解决（1000/1000），平均展开207节点，远优于DeepCubeA（1050节点）。
- **贪婪策略与搜索的权衡**：学到的Q函数作为局部动作排名器表现强，但作为全局启发式在分布外数据上校准较差，导致在某些IPC域贪婪策略（GSP_π）反而优于WA⋆搜索（GSP_WA⋆）。
- **AlphaZero对比**：AlphaZero风格的MCTS驱动学习在大多数域泛化极弱，仅Spanner例外，验证了MCTS不适合稀疏奖励的规划任务。
- **组件重要性**：解下界目标对性能贡献最大，死端监督影响最小（仅少数域存在死端），优先级采样策略提供了稳定的训练收益。

## 7. 优点

- **统一的框架**：使用单一架构（R-GNN）和超参数，在多样化域（大规模IPC规划、可解谜题、PushWorld）上均取得竞争性或领先性能。
- **自改进循环的有效性**：搜索提供高质量训练目标（目标路径下界），学习的启发式反过来改进搜索效率，形成正反馈。
- **处理稀疏奖励的优势**：通过WA⋆搜索一次性发现整个解路径并传播监督信号，比逐步实时搜索能更快获得学习信号。
- **显式处理死端**：将死端状态纳入训练数据（这对Sokoban和The Witness至关重要），显著提升了在这些域的表现。

## 8. 不足与局限

- **全局校准问题**：学到的Q函数更适于局部动作排名而非全局价值估计，在分支因子极大的IPC测试实例上，WA⋆搜索可能因全局分数失准而失败（尽管贪婪策略成功）。
- **1-WL表达力限制**：R-GNN基于消息传递，在Rovers和Satellite等域受限于1-WL同构性，无法区分关键结构差异，导致覆盖率远低于LAMA等符号方法。
- **大规模规划域的脆弱性**：Childsnack等域因初始状态动作数达千万级，对所有学习方法仍是巨大挑战，无完美解决方案。
- **算力需求**：12小时训练×10域，以及5并发搜索工作进程，对算力要求较高；且未提供明确GPU型号，难以评估实际硬件门槛。
- **Sliding Tile Puzzle失败**：GSP在5×5滑块谜题上完全失败（0/1000解决），因为初始启发式无信息且固定搜索预算不足以发现解，而固定MLP基线通过增大搜索预算可以成功，强调了训练预算设计的敏感性。
