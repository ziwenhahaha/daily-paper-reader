## Abstract
On-policy self-distillation (SD) improves LLM reasoning by using teacher-side privileged information (PI) to turn sparse verifier outcomes into dense token-level supervision. Existing methods usually assume trusted PI, such as reference answers or successful traces. We ask whether PI can instead come from an experience-derived skill bank, where retrieved skills are compact and reusable but may also be irrelevant or misleading. We propose Skill-Conditioned Gated Self-Distillation (SGSD), which formulates skill-based SD as teacher hypothesis validation rather than unconditional imitation. SGSD retrieves skill-mistake pairs, constructs a multi-teacher pool, and lets all skill-conditioned teachers score the same plain-prompt student rollout. The verifier validates each teacher's polarity: supporting a success or suppressing a failure gives positive supervision, while the opposite stance is reversed. A robust gated objective then distills informative teacher-student disagreements while suppressing uncertain or extreme signals. Experiments on multiple mathematical reasoning benchmarks show that SGSD consistently improves over GRPO and remains competitive with answer-conditioned OPSD under a weaker PI assumption. For example, on Qwen3-1.7B, SGSD outperforms GRPO by 6.2% and OPSD by 1.7% on average on AIME24, AIME25, and HMMT25. Our code is available at https://github.com/walawalagoose/SGSD.

## 1. 论文核心问题与整体含义（研究动机和背景）

- **研究背景**：强化学习与可验证奖励（RLVR）已成为 post-training 大语言模型推理能力的主流范式。RLVR 方法（如 GRPO）仅依赖稀疏的最终答案奖励，存在**奖励信号稀疏、延迟且在整个轨迹上几乎均匀分布**的弱点。
- **核心问题**：on-policy 自蒸馏（SD）通过让模型同时充当教师和学生，将稀疏奖励转化为密集 token 级监督，但现有方法假设教师端拥有**可信的特权信息（PI）**（如参考答案、成功轨迹等）。论文提出一个关键问题：**经验驱动的技能库能否作为 PI 的替代来源**？技能比完整轨迹更紧凑、更可复用，但可能是不相关甚至误导性的。
- **挑战**：
  1. PI 从单一可信源变为不确定相关的检索技能假设集；
  2. 技能的价值依赖结果：同一教师支持可能有利或有害，取决于 rollout 成功与否；
  3. 多教师产生噪声或冲突信号，需提取有用分歧而不被极端 logit 主导。

## 2. 论文方法论

### 核心思想

- **Skill-Conditioned Gated Self-Distillation（SGSD）**：将基于技能的自蒸馏形式化为**教师假设验证**（teacher hypothesis validation），而非无条件模仿。检索技能-错误配对构成多教师池，每个教师评分同一学生 rollout，验证器裁决每个教师的极性（polarity），再通过鲁棒的 gate 目标提取有用分歧。

### 关键技术细节

- **结构化技能库** B = {G, E}：
  - G：通用技能（原则 + 适用条件）；
  - E：常见错误模式（描述 + 成因 + 如何避免）；
  - 通过冷启动提取 + 层级合并构建，支持训练中在线更新。

- **技能条件多教师池**：
  - 对问题 x 检索 Top-K 技能-错误配对 {g_k, e_k}；
  - 教师 k 的上下文：c_k(x) = g_k ⊕ e_k ⊕ x；
  - 教师分布：p_T^(k)(y_t | x, y_{<t}) = π̄_θ(y_t | c_k(x), y_{<t})（stop-gradient）；
  - 教师权重 α_k(x) 由检索相似度归一化得到。

- **结果验证的教师极性**：
  - Token 级信用差距：Δ_t^(k) = log p_T^(k)(...) − log p_S(...)；
  - 教师级支持分数：a_k = (1/T) Σ_t Δ_t^(k)；
  - 极性判决规则（核心）：
    - r > 0 且 a_k > 0 → helpful teacher（蒸馏）；
    - r < 0 且 a_k < 0 → warning teacher（蒸馏）；
    - r > 0 且 a_k < 0 → over-suppressive teacher（反转）；
    - r < 0 且 a_k > 0 → misleading teacher（反转）；
  - 最终极性 ρ_k = sgn_out(r) × sgn(ēa_k)。

- **鲁棒支持估计**：
  - Token 掩码 m_t：过滤特殊 token、格式 token；
  - 差距裁剪：clip(Δ, −c_Δ, c_Δ)；
  - 置信阈值 ε_a：|ēa_k| ≤ ε_a 的教师被忽略。

- **Gate 蒸馏目标**：
  - ℓ_gate(Δ) = log 2 − log(1 + exp(−Δ²/(2τ_g)))；
  - 特性：近零差距几乎无更新，中等差距线性增长，极大差距指数衰减（bounded）；
  - 整体目标：L_SGSD(x) = Σ_k α_k(x)ρ_k ℓ̄^(k)。

- **在线技能更新**：每 25 步从最近成功/失败 rollout 中提取新技能候选，合并去重后更新技能库。

## 6. 论文主要结论与发现

- **SGSD 在较弱 PI 假设下表现竞争力**：SGSD 仅使用验证器和可能误导的技能-错误配对，无需参考答案，仍优于 GRPO 且可与 OPSD 匹敌；
- **技能注入本身不足**：Base+Skill、GRPO+Skill、OPSD+Skill 均表现不佳，证明 SGSD 的价值不在于暴露技能，而在于教师侧的条件化验证；
- **训练稳定性**：SGSD 在训练后期保持稳定提升，而 OPSD 在中期后出现退化；
- **多教师极性关键**：移除极性机制会导致后期崩塌；单教师池持续低于多教师池；
- **具体性能提升**：Qwen3-1.7B 上，AIME24/AIME25/HMMT25 平均分从 37.4% 提升至 43.7%，分别超越 GRPO 6.2% 和 OPSD 1.7%。

## 7. 优点

- **创新的问题形式化**：将技能自蒸馏重新定义为教师假设验证，而非无条件模仿，契合现实场景中技能不可信的实际情况；
- **结果验证的极性机制**：通过验证器结果与教师支持的对齐关系，自动判断教师是 helpful / warning / misleading / over-suppressive，实现对误导性信息的自动反转；
- **鲁棒的 gated 目标**：结合 token mask、gap clipping、confidence threshold 和饱和 loss，保证梯度既不因无关 token 而受噪声干扰，也不因极端 logit 而被主导；
- **理论与工程结合**：完整推导了 gated loss 的梯度性质（符号一致性、bounded informative region、局部 reverse-KL 近似），为方法提供坚实理论支撑；
- **在线技能库演化**：技能库随训练动态更新，使检索技能反映当前策略的成败经验，增强适应性；
- **实验扎实**：多模型规模、多 benchmark、完整消融和变体分析，实验设计系统全面。
