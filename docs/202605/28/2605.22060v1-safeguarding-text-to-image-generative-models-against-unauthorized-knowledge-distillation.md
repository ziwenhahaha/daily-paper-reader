## 摘要
闭权重生成服务正越来越多地通过基于查询的API进行部署，用户可以在模型参数不可访问的情况下获取生成的输出。然而，这种部署并不能防止模型被盗：攻击者可以重复查询服务，收集大量发布的合成图像，并将其作为训练数据用于构建私有的替代模型。这种查询-输出驱动过程实现了未授权的知识蒸馏和能力复制，而无需直接访问原始权重。为了缓解这一威胁，实际的防御应保持发布图像的视觉保真度，提供对扰动幅度的明确控制，并能高效扩展到大量输出发布。我们提出了WaveGuard，一个单次、基于生成器的防护框架，在用户指定的扰动预算下保护发布的合成图像。WaveGuard采用频率感知扰动生成器，注入结构化的、不可察觉的扰动，在保持对良性观看者的感知效用的同时，降低受保护图像作为未授权学生模型训练数据的可用性。在基于WikiArt的合成输出蒸馏设置下进行的大量实验表明，WaveGuard在效能-保真度-效率之间取得了良好的权衡，具备明确的不可感知性控制，并在保护效率上实现了显著提升。

## Abstract
Closed-weight generative services are increasingly deployed through query-based APIs, where users can obtain generated outputs while model parameters remain inaccessible. However, such deployment does not prevent model stealing: an attacker can repeatedly query the service, collect large volumes of released synthetic images, and use them as training data for a private substitute model. This query-output-driven process enables unauthorized knowledge distillation and capability replication without direct access to the original weights. To mitigate this threat, a practical defense should preserve the visual fidelity of released images, provide explicit control over perturbation magnitude, and scale efficiently to large-volume output release. We present WaveGuard, a single-pass, generator-based protection framework that safeguards released synthetic images under a user-specified perturbation budget. WaveGuard employs a frequency-aware perturbation generator to inject structured, imperceptible perturbations that maintain perceptual utility for benign viewers while reducing the usefulness of protected images as training data for unauthorized student models. Extensive experiments under WikiArt-related synthetic-output distillation settings show that WaveGuard achieves a favorable efficacy--fidelity--efficiency trade-off, with explicit imperceptibility control and substantial gains in protection efficiency.

### 1. 核心问题与整体含义（研究动机和背景）

- **威胁场景**：闭源文本到图像生成服务通过查询式 API 部署，用户可获取生成图像但模型参数不可访问。攻击者通过反复查询收集大量合成图像，将其作为训练数据训练替代模型，从而实现无需访问原始权重的未授权知识蒸馏与能力复制。
- **问题本质**：现有防御方法（水印类方法、迭代优化类方法）无法同时满足以下要求：①降低合成图像对替代模型训练的可用性；②保持发布图像的视觉保真度；③提供对扰动幅度的显式控制；④能够高效扩展到大规模输出发布场景。
- **研究目标**：提出一种在用户指定扰动预算下工作的输出级保护方案，既保留图像对良性用户的感知效用，又减少其作为未授权学生模型训练数据的价值。

### 2. 方法论：核心思想、关键技术细节、算法流程

#### 核心思想

WaveGuard 是一个**单次、基于生成器的保护框架**，采用频率感知扰动生成器在一次前向传播中完成图像保护，而非迭代优化。

#### 关键技术细节

- **扰动生成器架构**：采用 U-Net 风格的编码器-解码器结构，输入为clean图像 x 与目标图像 m 的通道级拼接 `[x; m]`，输出加性扰动 δ = Gθ(xin)。通过以下两个关键模块实现频率感知：
  - **频率变换块（FTB）**：使用 Haar 小波变换将中间特征分解为四个子带（LL, LH, HL, HH），分别捕获低频结构（全局布局）与高频细节（边缘轮廓）。
  - **频率感知注入路径（FIP）**：低频残差连接（LL i 注入编码器）保留全局语义；高频三元组缓存（LH, HL, HH）注入解码器，通过逆小波变换重构高频细节，避免深度特征提取中的信息丢失与伪影。

- **扰动约束**：通过 tanh 层输出归一化扰动，并通过显式缩放施加硬 ℓ∞ 约束：`δ = ε · tanh(Dec(Enc(xin)))`，其中 ε 为用户指定的扰动预算。

#### 训练目标

- **潜在对抗损失（Latent Adversarial Loss）**：利用冻结的 SD VAE 编码器 E(·) 在潜在空间中将受保护图像 ˜x 的表示向目标图像 m 对齐：`L_adv = E[∥E(˜x) - E(m)∥²]`，驱使扰动误导生成模型捕获目标特征同时破坏原始图像语义。
- **期望变换（EOT）增强**：对 ˜x 和 m 应用随机可微分变换（可微分 JPEG 压缩、高斯模糊），提升对常见后处理的鲁棒性。
- **加权扰动铰链损失**：基于目标图像构造空间权重图 M = 1 + w·¯m，通过 `L_pert = E[max(0, ∥M ⊙ δ∥₂ - c)]` 正则化扰动幅度，减少视觉失真。
- **整体目标**：`min_θ λ_adv L_adv^EOT + λ_pert L_pert`，仅更新生成器参数。

### 6. 论文的主要结论与发现

1. **WaveGuard 在保真度上最优**：在 ε=8/255 条件下，WaveGuard 取得了最高的 PSNR（36.201）、SSIM（0.930）和最低的 LPIPS（0.089），优于所有对比方法。

3. **效率优势显著**：生成器方法（WaveGuard、AdvWM）单张图像处理时间约 0.025 秒，迭代方法（Mist、CosAttack）需要 7-17 秒，效率提升约 300-700 倍。
4. **频率模块贡献明确**：低频注入保留全局语义，高频注入维持细粒度扰动有效性，两者均对整体性能有正向贡献。
5. **跨检查点迁移有效**：在 SD1.5 上训练的 WaveGuard 对 SD2.1 学生模型仍保持保护效果（CSD 从 0.845 降至 0.663）。
6. **混合数据训练场景下保护有效**：随着受保护数据比例增加，保护效果单调提升，50% 保护比例接近全保护设置。

### 8. 不足与局限

1. **最强保护非最高保真**：迭代方法（如 Mist）在部分保护指标（如 DreamBooth CSD）上表现更强，WaveGuard 代表的是保真度优先的运作点，存在保护强度与保真度之间的权衡取舍。
2. **强净化攻击脆弱**：DiffPure 等对抗净化方法能大幅削弱保护效果，表明当前方法对自适应攻击的鲁棒性不足。
3. **数据集范围有限**：实验仅在 WikiArt 艺术风格数据集上进行，未涵盖人物、场景、自然图像等多种下游应用场景。
4. **替代模型类型单一**：仅评估了 DreamBooth 与 Textual Inversion 两种微调方法，未测试 LoRA 等其他常见的个性化训练方式。

6. **训练数据源敏感性**：在真实图像与合成图像训练源对比中，真实图像表现更优，但获取真实训练数据可能存在实际困难。
7. **未探索 adaptive attacker**：实验未考虑更复杂的攻击者策略，如对抗性数据筛选或专门的防御规避训练。
