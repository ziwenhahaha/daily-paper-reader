## 摘要
搜索增强推理模型的post-training通常依赖复杂外部监督、辅助模块或树搜索等机制，导致训练流程臃肿且资源依赖强。Search-E1提出仅用vanilla GRPO与离线自我蒸馏实现自进化，通过特权上下文对齐推理分布获得密集每步监督。在7个QA基准上使用Qwen2.5-3B达到0.440平均EM，超越所有开源基线，证明了无需复杂机制即可实现高效搜索增强推理训练。 现有搜索增强推理方法通过外部监督、辅助模块或复杂训练策略提升性能，但这些机制增加了训练复杂性和资源依赖，论文质疑其必要性并探索更简洁的训练范式。

## Abstract
Post-training has become the dominant recipe for turning a language model into a competent search-augmented reasoning agent. A line of recent work pushes its performance further by adding elaborate machinery on top of this standard pipeline. These augmentations import external supervision from stronger external systems, attach auxiliary modules such as process reward models or retrospective critics, restructure the rollout itself with tree search or multi-stage curricula, or shape the reward with hand-crafted bonuses and penalties. Each addition delivers a measurable gain, but each also inflates the training pipeline and ties the recipe to resources or designs that may not always be available. We take a step back and ask whether any of this machinery is actually necessary, and propose Search-E1, a self-evolution method that lets a search-augmented agent improve through only vanilla GRPO interleaved with offline self-distillation (OFSD). After each GRPO round, the policy rolls out on its own training questions. A token-level forward KL objective then aligns the policy's inference-time distribution to its own distribution under a privileged context that exposes a more efficient sibling trajectory. Despite this simplicity, the procedure naturally provides dense per-step supervision. On seven QA benchmarks, Search-E1 reaches $0.440$ average EM with Qwen2.5-3B, surpassing all open-source baselines at both scales. Code and complete version will be made public soon.

## 2. 方法论

### 核心思想

Search-E1提出交替式训练管道，将**探索**（GRPO）和**巩固**（OFSD）两个阶段解耦。关键洞察是：GRPO产生的rollouts中自然包含质量差异显著的兄弟轨迹对，其中一个轨迹正确且高效（用更少搜索次数到达正确答案），另一个则表现不佳。这一对比本身已携带每步信号，无需外部注入。

### 关键技术细节

**（1）轨迹格式**：采用 ``（内部推理）、`<search>`（查询检索器）、`<information>`（检索段落）、`<answer>`（最终答案）的结构化格式。

**（2）GRPO阶段**：使用标准GRPO优化，以最终答案正确性（exact match）作为单一轨迹级奖励。每个token共享相同的advantage，导致密集每步监督缺失。

**（3）离线自我蒸馏（OFSD）阶段**：
- **配对挖掘**：从GRPO训练后的策略采样K条轨迹，选取R=1且搜索次数最少的作为reference轨迹，选择与reference差异最大的作为student轨迹
- **非对称条件化**：Teacher以 `[instruction] + q + τ_ref` 为条件，Student以 `[instruction] + q` 为条件
- **Token级别Forward KL**：对齐teacher和student在相同token位置的分布分布，使用逐点裁剪防止稀有高散度token主导梯度

### 算法流程

```
循环迭代：
    Stage 1: GRPO轮次
        → 从策略π采样轨迹组
        → 计算exact-match奖励
        → 计算group-normalized advantage
        → GRPO损失更新策略
    Stage 2: OFSD轮次
        → 从更新后策略采样新rollout池
        → 挖掘(reference, student)配对
        → 冻结GRPO策略作为Teacher，LoRA适配器激活的模型作为Student
        → 最小化Forward KL损失更新LoRA
        → 合并LoRA到基权重
```

## 6. 论文主要结论与发现

1. **简洁机制足以驱动进化**：仅用vanilla GRPO + 离线自我蒸馏即可超越依赖外部教师、辅助模块的复杂方法
2. **兄弟轨迹对比自然提供每步监督**：GRPO rollouts中高效轨迹与低效轨迹的对比包含丰富的step-level信号
3. **多跳任务收益更大**：多跳问题需要更多搜索步骤，兄弟轨迹在查询质量上差异更显著，per-step对齐信号信息量更高
4. **自蒸馏闭合Base-Instruct差距**：通过OFSD信号，Instruct配置超越了Base变体的性能
5. **多轮迭代持续提升**：GRPO+OFSD循环可重复执行，每轮带来正增益且边际递减

## 8. 不足与局限

1. **Bamboogle表现不佳**：该数据集上被GiGPO超越（0.464 vs 0.641），可能因测试集小（125条）导致高方差，且GiGPO的anchor-state分组对该数据集的bridge-entity查询结构特别适配
2. **依赖GRPO稳定性**：OFSD轮次依赖GRPO产生有意义的兄弟轨迹对，若GRPO出现reward collapse，OFSD也会受影响
3. **多轮训练的边际收益**：论文仅测试2个GRPO+OFSD循环，长期迭代的收益递减模式尚需进一步验证
4. **特权上下文单一**：目前使用单条兄弟轨迹作为特权信息，更丰富的特权信号（如多正确轨迹、相关问题成功轨迹）的潜力未被探索
5. **检索器固定**：使用固定的E5-base-v2检索器，未探索更强的检索器或动态检索策略的影响
6. **仅验证QA任务**：方法在其他搜索增强推理场景（如对话、规划）的泛化性未验证
