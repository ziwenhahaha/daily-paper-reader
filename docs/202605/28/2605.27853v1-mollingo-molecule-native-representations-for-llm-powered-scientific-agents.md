## Abstract
We present MolLingo, a multi-agent system that emulates the reasoning process of a chemist to automate molecular design. Existing LLM-based approaches either operate as standalone generative models without access to external tools or lack the multi-agent coordination and shared memory needed for iterative, evidence-driven reasoning across the molecular design pipeline. MolLingo addresses this by coordinating a Literature Agent, a Chemist Agent, and an Orchestrator through a shared memory module, with each agent equipped with domain-specific tools. To enable effective molecular reasoning, we introduce BRICS-based Fragment Enumeration (BFE), a synthesis-aware molecular fragmentation method that decomposes molecules into chemically meaningful building blocks represented as block-based SMILES paired with common chemical names. This representation bridges molecular structure and LLM semantic space, enabling block-level reasoning and editing that is difficult with raw SMILES alone. As a case study in early-stage therapeutic design, MolLingo further grounds the Chemist Agent's reasoning in binding site geometry and residue-level protein context derived from molecular docking to optimize molecules for stronger target binding. Across four benchmarks, MolLingo consistently outperforms frontier LLMs and specialized baselines, including a fourfold docking score improvement over GPT-5.4 despite using the same underlying model, consistent drug property optimization gains across multiple LLM backbones, and state-of-the-art results on TOMG-Bench, surpassing both frontier LLMs and the RL-based optimization method RePO. Our results suggest that LLMs are already capable molecular design assistants when guided through chemically meaningful representations and biologically grounded structural context. ## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 2.1 多智能体系统架构（MolLingo）

MolLingo协调三类专业智能体，通过集中式共享记忆模块持久化所有提取的知识、中间结果和推理轨迹：

| 智能体 | 职责 |
|--------|------|
| **编排器（Orchestrator）** | 管理工作流、将任务分解为子目标、动态适应工作流、支持人机交互 |
| **文献代理（Literature Agent）** | 利用检索增强生成（RAG）从PubMed、UniProt、ChEMBL等来源检索和综合领域知识 |
| **化学家代理（Chemist Agent）** | 负责分子推理和化合物设计，支持hit发现、hit-to-lead和lead优化各阶段 |

**共享记忆模块**作为协调中枢，维护结构化信息（候选分子、对接分数、靶标注释）和非结构化上下文（推理轨迹、文献洞察、设计依据），使各智能体能读取和写入共享存储，实现跨完整发现工作流的迭代细化。

### 2.2 块级SMILES表示法（BFE: BRICS-based Fragment Enumeration）

**问题**：标准SMILES将分子结构编码为扁平字符序列，单个字符替换即可使分子失效，功能有意义的子结构（如芳香环）散落在非连续字符跨度中。

**解决方案**：提出基于块的分子的表示法，将分子图分解为离散、化学有意义的构建块，每个构建块表示为块级SMILES字符串与通用化学名称的配对。

**三步构建流程**：
1. **词汇表构建**：在大规模药物类分子语料（20M分子，来自ChEMBL、ZINC等）上训练分词器。使用BRICS分解分子得到合成可及的片段作为原始单元，通过穷举枚举所有可能的BRICS键对组合（而非迭代合并），在单次语料遍历中生成构建块词汇表。复杂度从图BPE的 $O((V-n_0) \cdot |C| \cdot \bar{n}^4)$ 降至 $O(|C| \cdot \bar{n}^3)$，约提升 $O((V-n_0) \cdot \bar{n})$ 倍。
2. **分子分块**：给定分子，识别所有BRICS键并枚举所有非分支分解方式。选择最优分解的层次标准：
   - 优先碎片最少的分解（更长块携带更丰富的结构和功能信息）；
   - 过滤掉词汇表中频率低于 $f_{min}$ 的块；
   - 在等长候选中选择块频率标准差最低的分解。
3. **带通用名称的块级SMILES**：每个块转换为带通配原子的SMILES字符串（[1*]表示左连接点，[2*]表示右连接点），并映射到通用化学名称（如pyridine、2-aminopyrimidine）。

### 2.3 LLM引导的分子设计管线

**Hit识别**：从ChEMBL检索已知binder（基于实验数据）或使用DrugCLIP进行配体基虚拟筛选（无实验数据时）。

**Hit-to-Lead（上下文感知片段生长）**：
1. 将hit片段对接到靶标蛋白活性位点；
2. 识别**生长热点**：对每个重原子 $a_i$，计算邻域残基 $R_i$（接触阈值 $d_c = 7.0 Å$）和可用体积 $V_i$（构建5.0Å立方网格，排除范德华重叠点）；
3. 按可用体积排序热点，将其生物上下文（原子类型、可用体积、邻域残基）编码为自然语言提示，传给化学家代理提出块级片段扩展。

**Lead优化**：迭代细化候选分子以改善结合亲和力和药物性质，化学家代理在每步接收块表示和ADMET预测谱，推理哪些块可能导致不良ADMET性质并提出保守修饰。

## 3. 实验设计：数据集、场景、Benchmark、对比方法

- **基线**：GPT-5.4、Claude-4.6-Sonnet、Gemini-3-Pro、Qwen2-7B-Instruct、Qwen2-14B-Instruct，均基于原始SMILES；
- **评估指标**：改进率（%）、相似性（%）、Imprv×Sim乘积、药物相似性（QED）、合成可及性（Synth）、有效性（%）。

- **MolLingo额外输入**：对接姿态、热点分析、结合位点的三维生物上下文；
- **基线**：同上5个LLM，均基于原始SMILES+蛋白名称；
- **评估指标**：有效性（%）、对接分数改进率（%）、相似性（%）、药物相似性（QED）、合成可及性（Synth）。

### 3.3 TOMG-Bench分子属性优化

- **评估指标**：成功率（SR）、分子相似性（Sim）、SR×Sim乘积。

### 3.4 Hit识别对比实验

## 6. 论文的主要结论与发现

### 6.1 核心发现
1. **块级表示优于原始SMILES**：MolLingo在所有实验中一致且显著优于基于原始SMILES的基线方法，改进幅度在多个指标上达数倍提升；
2. **四倍对接分数提升**：MolLingo（GPT-5.4骨干）对接分数改进率达10.4%，相比GPT-5.4基线的2.4%提升超过四倍；
3. **模型无关性**：块级表示的优势在GPT-5.4、Claude-4.6-Sonnet、Gemini-3-Pro等不同模型上表现一致，说明受益于表示方法而非特定模型；
4. **SOTA on TOMG-Bench**：在LogP、QED、MR三个优化目标上均取得最高SR×Sim分数，超越强化学习方法RePO；
5. **高有效性保证**：MolLingo在所有变体上均达到100%化学有效性，无需后验过滤；
6. **注意力对齐验证**：块级SMILES使LLM形成明确的结构-功能关联，原始SMILES则呈现随机分布；
7. **BFE分解的关键性**：朴素BRICS分解虽相似性更高（78.7% vs 45.5%），但优化效果大幅下降（1.0% vs 18.5% Imprv×Sim: 81.3 vs 839.8）。

### 6.2 关键结论
> "LLMs are already capable molecular design assistants, they simply need to be probed in the right language, through chemically meaningful representations and biologically grounded structural context."

LLM引导分子设计的瓶颈不在于底层模型的能力，而在于如何向其传递分子、物理和生物上下文信息。当以化学语言与之交流时，LLM已经是有能力的化学助手。
