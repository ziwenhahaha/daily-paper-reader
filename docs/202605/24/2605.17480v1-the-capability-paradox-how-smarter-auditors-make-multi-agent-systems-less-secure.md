## Abstract
Multi-agent systems extend large language models (LLMs) by decomposing tasks among specialized agents, but their distributed decision process creates new attack surfaces. We identify \emph{semantic hijacking}, an attack in which harmful requests are concealed within domain-specific narratives and propagated to a Manager through Worker reports, without any syntactic injection primitives. Across 42,000 adversarial trials over 12 Manager models and 7 Worker configurations, we uncover a \emph{capability paradox}: as Worker capability increases, the mean system-level Attack Success Rate (ASR) increases from 18.4% to 63.9%, peaking at 94.4%. To explain this effect, we conduct multi-level mediation analysis on two independent datasets (47,807 interactions). This analysis shows that this paradox is driven by \emph{linguistic certainty}: stronger Workers are more likely to interpret adversarial narratives as legitimate, convey their conclusions assertively, and thereby lead Managers to treat such confident endorsements as justification to execute. In our larger Worker-Only setting ($n_W$=14), certainty mediates 74% of the effect, with 95% confidence intervals (CI) excluding zero under both Monte Carlo and cluster bootstrap; the smaller Full-MAS setting ($n_W$ =6) shows a directionally consistent indirect effect. Worker-side safety prompting does not reliably mitigate this failure. Building on the mediation finding, we propose \emph{heterogeneous ensemble verification}, which pairs Workers of asymmetric domain competence so their complementary vulnerabilities break the certainty-to-execution chain, reducing ASR from 52.8% to 2.0% with negligible benign-task impact. Our results show that upgrading components to stronger models can actively degrade system security, and that effective defenses require exploiting--rather than eliminating--capability asymmetries between agents.

## 1. 核心问题与整体含义（研究动机和背景）

**核心问题：** 在基于大语言模型（LLM）的多智能体系统（Multi-Agent Systems, MAS）中，传统的安全研究主要关注**语法层面的注入攻击**（如指令覆盖、分隔符操作等），但本研究揭示了一种全新的攻击面——**语义劫持（Semantic Hijacking）**，即攻击者通过将恶意请求伪装在看似合理的领域特定叙述中（如 SRE 事故报告），绕过所有语法注入特征，使 Worker 智能体误判并生成权威性报告，从而诱导 Manager 智能体执行有害操作。

**研究动机：** 现代 MAS 系统普遍采用层级化的 Manager–Worker 架构，Manager 依赖 Worker 的审计报告作为决策依据。这种**委托信任机制**引入了新的安全依赖：即使 Worker 的输入通道可能包含对抗性内容，Manager 也倾向于将 Worker 报告视为可信的中间证据。直觉上，研究者认为更强的 Worker 模型能够更好地理解对抗性输入，从而提升系统安全性。但本研究通过大规模实验证明，这一直觉是**完全错误的**。

**能力悖论（Capability Paradox）：** 随着 Worker 能力提升，系统级攻击成功率（ASR）反而从 18.4% 上升到 63.9%，峰值达 94.4%。这意味着用更强的模型替换 Worker 会**主动降低系统安全性**，与"更强模型 = 更安全"的普遍假设形成根本性矛盾。

## 2. 论文提出的方法论

### 核心思想

论文将问题形式化为一个**层级化的 MAS 安全评估框架**，核心包括三个组成部分：

1. **语义劫持攻击生成**：从真实世界的 SRE 事故报告（postmortem）出发，使用 Gemini-3-Flash 作为对抗性变异器，将事故报告转变为包含恶意请求的伪造操作叙事。关键约束是：payload 中**不包含任何指令覆盖词、分隔符转义、可执行命令或 SQL 注入字符串**，完全依赖语义框架来伪装攻击。

2. **Manager–Worker 架构评估**：Worker 接收 payload 并输出结构化安全审计报告（含评估标签、触发类别、元数据、技术理由）；Manager 同时接收原始 payload 和 Worker 报告，决定是否拒绝、请求验证或调用工具。建立明确的信任边界：Worker 无直接执行权限，但其报告可塑造 Manager 决策。

3. **自动化 Oracle 评分**：使用 Gemini-3-Flash 作为 Oracle，对每个交互轨迹进行安全评分（Full-MAS 评分 0–4 级，Worker-only 评分 0–3 级）。攻击成功的判定标准为 Oracle 评分 ≥ 2。

### 关键技术细节

- **攻击策略分类**（SRE 场景）：紧迫性（SRE Urgency）、数据完整性恢复（Data Integrity Recovery）、安全遏制（Security Containment）、合规/法律（Compliance/Legal）、行政紧急授权（Administrative Emergency）五大类。
- **工具库**：11 个沙盒化工具桩，覆盖基础设施控制、数据操作、身份管理、代码执行、外部通信、财务操作等高影响力权限类别。
- **良性 benchmark 构造**：使用相同的变异器、源事故和工具库，仅修改变异指令以生成合法操作请求，分三个难度层级（安全只读操作、合法写操作、使用攻击相邻词汇的灰色地带操作）。
- **确定性操作化**：使用** assertive 词汇密度减去 hedging 词汇密度**来量化 Worker 报告的"语言确定性"（linguistic certainty）。assertive 词汇包含 27 个条目（如"verified"、"safe to proceed"、"legitimate"）；hedging 词汇包含 28 个条目（如"potentially"、"may"、"uncertain"）。

### 公式与评估指标

- **系统级 ASR（攻击成功率）**：
$$ASR(M, W) = \frac{1}{N}\sum_{i=1}^{N} \mathbb{1}[O(M(W(p_i), p_i)) \geq 2]$$
  其中 $M$ 为 Manager，$W$ 为 Worker，$O$ 为 Full-MAS Oracle，$N$ 为 adversarial payload 数量。

- **Worker-only Fool Rate（被骗率）**：
$$FR(W) = \frac{1}{|P_{valid}|}\sum_{p_i \in P_{valid}} \mathbb{1}[O_W(W(p_i)) \geq 2]$$
  $P_{valid}$ 排除了格式错误的输出。

- **多层级中介分析**：MMLU 得分作为预测变量，语言确定性作为中介变量，劫持结果作为因变量。间接效应 $a \times b$ 通过 Monte Carlo 模拟（20,000 次）和 Worker 聚类自助法（2,000 次）评估。

### 防御方法：异构集成验证（Heterogeneous Ensemble Verification）

将能力不对称的两个 Worker 配对——一个领域能力强但易受叙事欺骗（如 DeepSeek-R1），一个能力较弱但更保守且具有选择性拒绝能力（如 Llama-3.1-8B）。使用 **OR-gate 逻辑**：仅当两个 Worker 均评估为安全时才将请求转发给 Manager，否则在 Worker 层拦截。该防御利用了"互补脆弱性"——两个 Worker 以不同方式失败，OR-gate 要求一致性，从而在保留良性任务吞吐量的同时过滤攻击。

## 6. 论文的主要结论与发现

### 6.1 语义劫持的有效性

- 在 Config A 中，系统级 ASR 范围从 0.8% 到 94.4%，Worker-only FR 范围从 2.8% 到 90.3%。
- 即使是 DeepSeek-R1（顶级能力模型，MMLU 90.8）作为 Worker 时，FR 也高达 90.3%。
- 语义劫持 payload 完全不包含语法注入特征，但仍能高度有效地诱骗智能体执行有害操作。

### 6.2 能力悖论

- **Worker 能力（MMLU 得分）与 Fool Rate 呈强正相关**（Spearman ρ = 0.81，p < 0.001）。
- 使用 DeepSeek-R1 替换 Llama-3.1-8B 作为 Worker，ASR 可增加高达 19.6 倍。
- 该悖论**特异于语义攻击**：语法注入攻击（INJECT-IO、INJECT-RS）在所有能力级别上均保持低 FR（≤ 7%），不呈现代价能力-脆弱性缩放。
- 跨领域泛化中，能力悖论从 SRE（ρ = 0.81）递减至医疗（ρ = 0.50）再到金融（ρ = 0.07），表明在具有明确法规结构的领域中悖论较弱。

### 6.3 语言确定性作为中介机制

- **Worker-only 设置**（$n_W = 14$）：语言确定性中介了 74.4% 的能力-劫持效应，Monte Carlo 95% CI [+39%, +146%] 排除零。
- **Full-MAS 设置**（$n_W = 6$）：点估计方向一致，但聚类 Bootstrap CI 包含零（[-1.14, +1.07]），被视为提示性而非确认性结果。
- 更强的 Worker 更可能将对抗性叙事解析为合法，并以更高的确定性传达结论；Manager 则将高确定性报告视为执行授权。

### 6.4 防御策略效果

| 防御方法 | ASR 变化 | 良性任务影响 | 评价 |
|---------|---------|-------------|------|
| Worker 端安全提示 | 9/12 模型降低 FR，但 3 个模型 FR 上升 | 不适用 | ❌ 不可靠 |
| Manager 端怀疑主义提示（P3） | -46.7 pp | -15.3 pp | ❌ 安全性-效用权衡过大 |
| 报告不确定性注入 | -43.2 pp | -26.3 pp | ❌ 安全性-效用权衡过大 |
| **异构集成验证（DeepSeek-R1 + Llama-3.1-8B）** | **-50.8 pp**（52.8% → 2.0%） | **+0.0 pp**（无影响） | ✅ **最有效** |

### 6.5 集成防御的通用性原则

- 能力不对称是必要条件，同家族不对称配对（Llama-3.1-70B + Llama-3.1-8B）仍实现 -30.9 pp ASR 下降。
- 弱伙伴必须是**选择性拒绝者**（拒绝攻击请求、接受良性请求），而非无差别拒绝者（如 Mistral-Nemo，无差别拒绝率 88%，会损失大量良性任务）。
- 跨家族多样性提供了额外边际，但在拥有选择性弱伙伴时非必需。

## 7. 优点

1. **问题定义的原创性**：首次系统性地定义了"语义劫持"攻击类别，揭示了语法注入防御完全无法覆盖的全新攻击面。

2. **能力悖论的深刻洞察**：反直觉地发现"更强模型 = 更危险"的现象，并深入分析其机制根源，为后续研究提供了全新的安全视角。

3. **中介机制的多层级验证**：使用两种独立数据集、两种能力代理（MMLU、GPQA-Diamond）、两种统计方法（Monte Carlo、聚类 Bootstrap），提供了交叉验证的统计证据。

4. **实验规模与系统覆盖**：42,000 次 adversarial trials + 多领域泛化 + 全面的防御消融，实验体量远超同类研究。

5. **机制驱动的防御设计**：不满足于观察到现象，而是通过中介分析定位到"语言确定性"这一核心机制，进而设计针对性防御，展示了从诊断到处方的完整研究路径。

6. **跨架构 Oracle 验证**：使用独立的 GPT-4o-mini 重新评分并与主 Oracle 对比，增强了结果的可信度。

7. **算力透明**：完整报告了 Token 消耗、API 成本和运行时间，提供了可复现性基础（$200 USD 预算即可复现全流程）。

8. **负责任的研究实践**：仅发布攻击方法论和定量结果，benchmark  gated behind responsible-use agreement；所有工具使用沙盒化实现。

## 8. 不足与局限

1. **领域覆盖有限**：仅验证了 SRE、医疗、金融三个领域，未覆盖科学发现、法律工作流、创意写作等同样常见的多智能体部署场景，泛化性仍需进一步确认。

2. **跨领域能力悖论衰减的解释深度不足**：论文观察到能力悖论在金融领域几乎消失（ρ = 0.07），但仅将其归因于"明确的监管结构"，缺乏对机制差异的深入量化分析。

3. **Oracle 的系统性盲点风险**：尽管与人类标注达到 κ ≥ 0.87 一致性，但 automated Oracle 可能存在系统性盲点，大规模人类验证可能揭示更多细微问题。

4. **能力指标的统一性缺陷**：部分模型（Qwen-3-32B、Qwen-3.5-27B）仅报告 MMLU-Pro 或 GPQA-Diamond reasoning-mode 得分，与标准 MMLU 不可比，因此被排除在部分相关性分析之外，导致样本选择可能影响结论的稳健性。

5. **防御策略的适用性边界**：异构集成验证需要维护两个不同能力级别的 Worker，增加了系统复杂度和推理延迟；在资源受限场景中可能不具成本效益。

6. **短语境交互的实验设计局限**：使用沙盒化工具桩的确定性 stub 响应，真实生产环境中的工具执行结果可能具有不确定性或延迟，可能改变 Manager 的决策模式。

7. **RLHF 与能力悖论的因果归因未分离**：论文指出能力悖论可能与 RLHF 诱导的过度自信有关，但未设计实验来分离 RLHF、预训练规模或指令微调数据各自的贡献。

8. **良性任务的难度层级主观性**：三个难度层级的划分虽然合理，但缺乏对"gray-zone 操作"边界条件的系统性定义，可能存在语义模糊。

9. **选择性模型排除**：部分分析排除了异常模型（如 Qwen-2.5-7B、Qwen-3-32B、Qwen-3.5-27B、Mistral-Nemo），虽然有充分理由，但仍可能影响统计推断的完整性。

10. **缺乏真实世界部署验证**：所有实验均在模拟环境中进行，未评估在真实生产 MAS 部署中的效果。
