## Abstract
Self-distillation enables language models to learn on-policy from their own trajectories by using the same model as both student and teacher, with the teacher being conditioned on privileged information unavailable to the student. Such information can come in different types or views, such as solutions, demonstrations, feedback, or final answers. This setup provides dense token-level feedback without relying on a separate external model, but creates a fundamental asymmetry: the teacher may rely on view-specific information that the student cannot access at inference time. Moreover, the best type of privileged information is often task-dependent, making it difficult to choose a single teacher view. In this work, we address both these challenges jointly by introducing AVSD (Adaptive-View Self-Distillation), a novel method of self-distillation with multiple privileged-information views, which reconstructs token-level supervision by separating stable cross-view consensus from view-specific residual signals. AVSD identifies the consensus signal shared across views, which provides a reliable update direction, and then selectively adds the view-specific residual signal to adjust the update magnitude when it both aligns with the consensus direction and remains proportionate to the consensus signal. Experiments on math competition benchmarks (AIME24, AIME25, and HMMT25) show that AVSD consistently outperforms both single-view self-distillation baselines and GRPO, achieving average Avg@8 gains of 3.1% and 2.2% over the strongest baselines on Qwen3-8B and Qwen3-4B, respectively. Moreover, on code-generation benchmarks (Codeforces, LiveCodeBench v6) using Qwen3-8B, AVSD outperforms the single-view self-distillation baseline by 2.4% on average.

## 1. 核心问题与整体含义（研究动机和背景）

**问题背景：**
- 大语言模型（LLM）在数学、代码等可验证任务中进行后训练时，主要依赖**强化学习与可验证奖励（RLVR）**范式，但该方法提供的监督信号稀疏且仅在结果层面（outcome-level），导致模型失败时学习信号不足
- **蒸馏（Distillation）**提供密集的 token 级指导，但传统离线蒸馏存在训练-测试不匹配问题（off-policy）
- **自蒸馏（Self-Distillation）**通过让学生模型自身同时扮演学生和教师角色来解决上述问题：教师可以在训练时访问学生无法获得的特权信息（如完整解决方案、部分推理链、最终答案等）

**核心挑战：**
1. **信息不对称**：教师依赖的特权信息在推理时不可用，导致学生被迫模仿可能不兼容的分布
2. **特权信息类型选择依赖任务**：不同类型（完整解、部分解、答案）的特权信息在不同数据集上表现差异显著（论文 Fig.1 展示了 Qwen3-4B 上三种视图的巨大性能波动），且最佳选择因任务而异，难以事先确定
3. **单视图假设的局限**：现有自蒸馏方法通常在整个训练过程中固定使用单一特权视图

**核心问题：** 能否同时利用多种特权视图构建比任何单一特权教师更好的 token 级学习信号？

## 2. 论文提出的方法论：核心思想、关键技术细节

**方法名称：** AVSD（Adaptive-View Self-Distillation，自适应视图自蒸馏）

### 2.1 核心思想

将同一训练样本的多种特权视图视为**教师家族（Teacher Family）**，通过分析多视图间的几何结构，将教师信号分解为两部分：
- **共识信号（Consensus Signal）**：跨视图共享的稳定部分，提供可靠更新方向
- **残差信号（Residual Signal）**：视图特定的支持，仅在符合共识方向且比例适当时选择性添加

### 2.2 关键数学定义

**单视图反向 KL 目标函数：**
$$L_{sv}(\theta) = E_{(x,r)\sim D} E_{y\sim P_S^\theta(\cdot|x)} \left[ \frac{1}{|y|} \sum_{t=1}^{|y|} D_{KL}(p_t \parallel q_t) \right]$$

其中 $p_t$ 为学生分布，$q_t$ 为教师分布（相同模型在特权信息下条件化）。

**Token 级蒸馏优势（Token-level Distillation Advantage）：**
$$A_t(v) := \log q_t(v) - \log p_t(v)$$

- $A_t(v) > 0$：促进 token v
- $A_t(v) < 0$：抑制 token v

**多视图设置：** 构建 M 个特权视图 $V(r) = \{r^{(1)}, ..., r^{(M)}\}$，每个视图诱导教师分布 $q_t^{(m)}(v)$。

### 2.3 两个聚合目标

**几何共识目标（Geometric Consensus Target）：**
$$\text{eq}_G^t(v) = \left( \prod_{m=1}^{M} q_t^{(m)}(v) \right)^{1/M} = \exp\left(\frac{1}{M}\sum_{m=1}^{M}\log q_t^{(m)}(v)\right)$$

- 强调跨视图一致支持的 token
- 保守：任何视图低概率的 token 都会被压低
- 对应最小化平均反向 KL 到教师家族

**算术边际目标（Arithmetic Marginal Target）：**
$$q_A^t(v) = \frac{1}{M}\sum_{m=1}^{M}q_t^{(m)}(v)$$

- 保留至少一个视图强烈支持的 token
- 宽松：可捕捉互补推理路径，但对视图特定伪影敏感

### 2.4 跨视图残差定义

$$A_A^t(v) = A_G^t(v) + J_t(v), \quad J_t(v) := \log q_A^t(v) - \log \text{eq}_G^t(v) \geq 0$$

其中 $J_t(v)$ 为残差，非负（由算术-几何均值不等式保证）。

### 2.5 门控重建机制

**核心原则：** 始终保留视图共识，仅在残差可靠时添加。

**重建优势：**
$$\tilde{A}_t(v) := A_G^t(v) + \lambda_t(v) J_t(v), \quad \lambda_t(v) \in [0, 1]$$

**门控函数（核心公式）：**
$$\lambda_t(v) = C_t(v) \cdot R_t(v)$$

其中：
- **对齐组件（Alignment Component）：**
$$C_t(v) := \frac{|A_G^t(v)|}{\frac{1}{M}\sum_{m=1}^{M}|\Delta_t^{(m)}(v)| + \epsilon} \in [0, 1]$$
  - 高值：视图优势同号或不支持视图接近中性
  - 低值：正负更新相互抵消

- **幅度组件（Magnitude Component）：**
$$R_t(v) = \frac{|A_G^t(v)|}{|A_G^t(v)| + J_t(v) + \epsilon}$$
  - 防止残差压倒共识更新方向

**关键保证：** 由 $C_t(v) \leq 1$ 和 $J_t(v) \leq |A_G^t(v)| + J_t(v) + \epsilon$ 可证 $\lambda_t(v)J_t(v) \leq |A_G^t(v)|$，即残差可调整更新幅度，但**不能逆转共识建立的更新方向**。

### 2.6 训练目标

$$L_{\text{AVSD}}(\theta) = E_{(x,r)\sim D} E_{y\sim P_S^\theta(\cdot|x)} \left[ \sum_{t=1}^{|y|} D_{KL}(p_\theta(\cdot|h_t) \parallel \text{sg}[q_*^t(\cdot)]) \right]$$

其中 $q_*^t$ 为重构建目标，梯度仅通过学生分布流动。

### 3.2 特权视图构建

**数学视图（3 种）：**
1. **完整解决方案（Full Solution）**：包含 CoT + 答案的完整推理过程
2. **部分解决方案（Partial Solution）**：保留初始推理步骤的截断版本
3. **最终答案（Final Answer）**：仅包含正确答案

**代码视图（3 种）：**
1. **参考实现（Reference Implementation）**：标准解决方案代码
2. **算法提示（H Algorithmic Hint）**：从参考解衍生的高层次提示
3. **执行反馈（Execution Feedback）**：学生 rollout 在测试用例上的运行结果（pass/fail）

## 6. 论文的主要结论与发现

1. **多视图优于单视图**：AVSD 在所有测试模型和基准上始终优于 OPSD 和 GRPO，证实了利用多种特权信息优于固定单一视图

2. **共识和残差互补**：仅共识目标在某些基准上表现良好（如 AIME24、AIME25），但会过于保守；仅算术目标在部分基准上有增益（如 HMMT25），但可能过度使用视图特定支持；AVSD 通过门控机制结合两者，在所有基准上达到最优

3. **视角数量带来边际增益**：从 1 个增加到 3 个视图，数学基准平均提升 ~2.8%，代码基准平均提升 ~2.0%；从 3 个增加到 4 个视图，增益递减但仍为正

4. **Token 级学习信号更可靠**：
   - GRPO 在 29.6% 的错误 rollout 案例中无法提供学习信号
   - OPSD 在 12.3% 的高影响力 token 上给出错误方向信号
   - AVSD 将此比例降至 2.9%

5. **无单一视图最优**：不同特权视图在不同任务上表现各异，AVSD 通过组合所有视图实现更稳健的性能

6. **门控机制有效**：三个模型的门开启率在 21%~27% 之间，表明 AVSD 主要依赖共识信号，同时选择性添加残差信息

## 7. 优点

1. **理论动机清晰**：通过分析多视图教师家族的几何结构，自然导出共识+残差的分解框架

2. **门控设计巧妙**：
   - 对齐组件防止视图冲突时的不稳定信号
   - 幅度组件保证残差不能翻转共识方向
   - 数学上保证 $\lambda_t(v)J_t(v) \leq |A_G^t(v)|$

3. **计算开销可控**：AVSD 仅增加 ~13% 的训练时间，相比基线 OPSD 具有实际可行性

4. **任务无关的通用性**：方法可应用于任何可构造多视图特权信息的任务（数学、代码均验证）

5. **实验严谨性**：
   - 多模型验证（3 个不同规模模型）
   - 多任务覆盖（数学 + 代码）
   - 充分的消融和缩放实验
   - Token 级信用分析提供深层可解释性

6. **解决真实痛点**：明确针对"特权信息类型选择困难"这一实际挑战，提供系统性解决方案

## 8. 不足与局限
