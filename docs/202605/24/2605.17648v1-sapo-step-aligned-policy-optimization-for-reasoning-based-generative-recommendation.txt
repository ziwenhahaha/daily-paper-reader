Abstract 

Generative recommendation treats next-item prediction as autoregressive item-identifier generation. Specifically, items are encoded as semantic identifiers (SIDs), which are short coarse-to-fine token sequences whose early tokens capture broad semantics and later tokens refine them. Recent work augments this paradigm with reasoning traces and optimizes them via reinforcement learning with veri-fiable rewards, typically outcome-reward algorithm with exact-match feedback on the generated SID. However, in large-catalog recommendation, exact-match feedback on the generated SID only reports whether the final item is correct; when a generated SID mismatches, outcome-reward cannot identify which SID-token prediction caused the mismatch and may penalize matched SID-token positions together with the mismatched position. We identify that the natural unit of credit assignment in this setting is a single reasoning step (one thinking block paired with one SID token). We instantiate this idea in SAPO (Step-Aligned Policy Opti-mization): rather than broadcasting one advantage to the whole response, SAPO computes a separate group-relative advantage for each reasoning step and applies it only to the corresponding thinking block and SID token. Across three real-world recommendation datasets, SAPO stabilizes reinforcement-learning training and consistently improves over existing generative recommendation baselines, with the largest gains where sparse exact-match feedback makes reasoning-step credit assignment important. Our results suggest that reinforcement-learning objectives for structured generation should mirror the decoder’s own decomposition of the output. Our code is available at https://github.com/zhengzaiyi/SAPO .

1 Introduction 

The item recommendation task requires a model to identify items a user is likely to interact with from a large catalog of closely related candidates. Traditional systems typically decouple candidate retrieval from ranking, whereas generative recommendation offers an alternative by casting recommendation as autoregressive item generation. Building on advances in language modeling [ 12 ], this formulation has become increasingly attractive, with prior work demonstrating strong empirical performance [ 1, 14 ]. Because assigning a unique token to every item would require a prohibitively large output vocabulary, items in this paradigm are represented as semantic identifiers (SIDs), which are short token sequences produced by a hierarchical vector quantizer. The language model is then trained to autoregressively generate the SID of the item a user will interact with next. A complementary line of work augments generative recommenders with explicit reasoning traces, typically by prompting the model to produce a reasoning trace before SID decoding [ 3, 9, 10 , 23 ]. These traces are intended to help the model interpret user histories and surface relevant preference

signals prior to generating an item identifier. Recent systems increasingly adopt a post-training recipe that first aligns the backbone to the recommendation vocabulary via supervised fine-tuning (SFT), then activates structured reasoning, and finally applies reinforcement learning (RL) from verifiable rewards (RLVR) [ 6] when reliable outcome checks are available. This recipe bridges SFT-only reasoning recommenders and newer SFT-then-RL variants, including GRPO-style approaches for recommendation [ 29 ]. Once a response contains both a reasoning trace and a multi-token SID, the central training question becomes one of credit assignment under the match feedback of SID: which SID-token prediction caused the final mismatch? 

For hierarchical SID decoding, credit assignment should respect the positional structure of the SID hierarchy. While final-item verification checks only the whole SID tuple, each SID-token position reveals whether the model has captured fine-grained item information or has failed to identify the broad intent. Outcome-reward RL discards this structure by broadcasting a scalar reward across the reasoning tokens and all SID-token predictions. This mismatch produces two failure modes: matched SID-token positions can be penalized together with the mismatched position, and scalar advantage broadcast cannot localize the mismatched SID-token prediction among unrelated reasoning steps. The underlying issue is action granularity: the rollout is too coarse, yet individual tokens are too fine-grained to serve as the credit-assignment unit for structured SID decoding. To address this mismatch, we propose SAPO , a reinforcement learning method for the RL stage of reasoning-based generative recommendation. SAPO exploits the structure already present in SID decoding: each generated SID tuple exposes multiple verifiable token predictions, giving every rollout a natural decomposition into reasoning steps. We therefore define a reasoning step as one thinking block paired with its corresponding SID token and treat this pair as the action unit for reinforcement learning. Once the action unit is fixed, the rest of the update follows naturally: SAPO assigns a verifiable match reward to each reasoning step from SID-token correctness, estimates a group-relative advantage at the reasoning-step level, and optimizes a step-aligned surrogate with step-normalized token aggregation so that each reasoning step receives its own update signal. The resulting credit assignment remains consistent with the original exact-match objective while providing denser supervision when exact SID-tuple matches are sparse. Our main contributions are as follows: • We identify the mismatch between exact-match outcome feedback and reasoning-based SID decoding as a problem of action granularity: rollout-level rewards cannot identify the specific mismatch steps, while rollout-level advantages can penalize matched step positions (Section 3). • We propose SAPO , which treats each reasoning step, pairing one thinking block with one SID token, as the RL action unit and derives from this choice a verifiable per-step match reward, a step-level group-relative advantage, and a step-aligned surrogate with step-normalized token aggregation, and introduces no learned reward model (Section 4). • Experiments on three real-world recommendation datasets show that SAPO produces more stable RL dynamics and yields strong overall Recall and NDCG results, especially in settings where fine-grained credit assignment matters (Section 5).

2.2 Three-Stage Training Setup 

Motivated by recent reasoning-based generative recommenders such as OneRec-Think [ 10 ] and SIDReasoner [ 3 ], we adopt a three-stage setup for reasoning-based SID decoding. Stage 1 aligns the language model to the SID vocabulary and the recommendation prompt format, so the model can produce valid SID tokens for the next-item prediction task. Stage 2 trains the policy to generate 

K thinking blocks followed by the K SID tokens of the predicted item, giving the response one reasoning block for each SID-token position. Stage 3 applies reinforcement learning (RL) from final exact-SID feedback, while the SID hierarchy exposes intermediate SID-token match signals for credit assignment. The next section analyzes why the standard outcome-reward RL objective is misaligned with this per-step decoding structure.

Such shaping tells that a rollout is partially correct, but it still cannot isolate the mismatched SID-token prediction. This is what we refer to as the action-granularity mismatch. The rollout is too coarse because it hides which of the K verifiable SID-token predictions succeeded or failed. Individual response tokens are too fine because arbitrary reasoning tokens do not have verifiable correctness labels, whereas SID-token positions do. The natural credit-assignment unit lies in between: the reasoning step that pairs a thinking block with its SID token. This mismatch causes two concrete problems. First, matched SID-token credit is discarded : matched SID-token predictions receive no positive signal when another SID token is wrong, so plausible near misses are treated as complete failures. Second, reasoning-step advantage localization is lost : an error at one SID-token position assigns the same scalar weight to token gradients from unrelated thinking blocks and SID-token positions. In long reasoning responses, this diffuse broadcast can reinforce or penalize many reasoning tokens together even though the verifier only checks the SID-token predictions, contributing to length drift, format degradation, and weak group-relative contrast. These observations motivate aligning reward, advantage, and token aggregation at the reasoning-step granularity in the next section. 

4 Methodology 

Motivated by the analysis above, SAPO addresses the RL-stage credit-assignment problem in reasoning-based SID decoding. Its central design choice is to use each reasoning step, defined as a SID-token prediction together with its associated thinking block, as the credit-assignment (action) unit. Once this reasoning step is fixed, reward placement, advantage estimation, and token aggregation should all be defined at the same granularity. We now define the reasoning-step decomposition used by SAPO and present the reasoning-step match reward and the step-aligned surrogate. The overall pipeline is illustrated in Figure 2.

5.2 Comparison with Baselines (RQ1) 

To address RQ1 , we evaluate SAPO against the baselines on the held-out test set of each category. Following common practice [ 5], we report Recall@ k (R@ k) and normalized discounted cumulative gain (NDCG@ k, denoted N@ k) at k ∈ { 5, 10 }.Table 1 shows that SAPO improves end-task ranking quality, while the pattern is more informative than uniform dominance. The most direct comparison is SIDReasoner, which shares the same broad reasoning-based setup but uses a rollout-level shaped reward in Stage 3: SAPO improves every NDCG column over SIDReasoner and improves R@10 on two of the three categories, while remaining slightly below SIDReasoner on Office-Products R@5 and Industrial-and-Scientific R@10. This pattern suggests that reasoning activation alone is not sufficient; once reasoning is introduced, the RL credit signal must also respect the hierarchical SID-token structure. Across the full table, SAPO achieves the best NDCG on all columns, indicating that step-aligned credit assignment improves the ordering of the ground-truth item within the top-ranked list. At the same time, SAPO ranks first or second on all reported Recall columns rather than uniformly dominating every Recall setting. SAPO is consistently stronger than non-reasoning generative baselines, supporting the broader conclusion that reasoning benefits most when RL credit assignment follows the reasoning-step structure.

5.5 Case Study 

To complement the quantitative results, we provide qualitative examples showing how SAPO shapes reasoning behavior. Figure 5 compares outcome-reward training and SAPO on the same

SAPO preserves this finer-grained cue by reasoning about electronics and stepper motor control, and ranks the motor-driver SID first. Because both models receive the same prompt and start from the same Stage 2 reasoning-activation checkpoint, the contrast primarily reflects the effect of the Stage 3 credit-assignment objective. The outcome reward only distinguishes complete SID-tuple success from failure, so this broadly relevant but wrong accessory receives the same scalar penalty as a completely unrelated item. SAPO instead assigns feedback to reasoning steps using SID-token correctness, making the update sensitive to which SID-token prediction is correct and which refinement fails. 6 Conclusion 

We introduced SAPO , a step-aligned reinforcement-learning method for reasoning-based generative recommenders that decode items over a K-level SID codebook. The central observation is that outcome-reward RL uses the wrong action granularity for this setting: it obscures which SID-token prediction caused the mismatch and can penalize matched SID-token predictions together with erroneous refinements. SAPO resolves this mismatch by treating each reasoning step (τ (k), s (k))

as the action unit and deriving from that choice a per-step match reward, a per-step group-relative advantage, and a step-aligned surrogate with step-normalized token aggregation. Across the evaluated settings, SAPO materially stabilizes training dynamics and yields competitive-to-improved Recall and NDCG relative to outcome-reward baselines, with the clearest gains where fine-grained credit matters most. B Related Work 

B.1 Generative Recommendation with Semantic Identifiers 

Generative recommendation has emerged as an alternative to discriminative matching: each item is represented as a short sequence of discrete tokens, and the recommender is trained as a language model over these tokens [ 1 , 7 , 14 ]. A central design choice is the tokenization scheme: residual-quantization VAEs produce hierarchical semantic identifiers (SIDs) that capture coarse-to-fine item semantics, which both improves generalization to unseen items and enables compact generation [ 7, 14 ]. Our work inherits this representation and shows how the K-level structure of SIDs provides a natural scaffold for step-level credit assignment, an observation that, to the best of our knowledge, has not been leveraged by prior work on generative recommenders, including recent reasoning-based systems such as SIDReasoner [3] that still rely on a scalar outcome reward. 

B.2 Reasoning and Reinforcement Learning for Large-Language-Model (LLM)-based Recommendation 

Following the success of reinforcement learning from verifiable rewards on math and code [ 2, 6], several recent systems apply group-relative policy-optimization algorithms [ 11 , 15 , 24 , 28 ] to LLM-based recommenders. The systems most closely related to ours include SIDReasoner [ 3 ], OneRec-Think, R 2ec, ReaRec, and RecCoT [ 10 , 17 , 22 , 23 ], all of which augment generative recommenders with chain-of-thought reasoning and train them with a scalar outcome reward fed into GRPO or a close variant. SAPO directly addresses this misalignment by assigning rewards at the per-step granularity; it can be viewed as a form of process-reward RL [ 8, 19 ] that obtains per-step supervision directly from the codebook hierarchy, without any human-annotated or learned reward model. Concurrently, Rank-GRPO [ 29 ] proposes rank-level credit assignment for conversational recommenders; our work is complementary in that it targets the codebook hierarchy of SID decoding rather than the positional hierarchy of a recommendation list. A complementary diagnosis is provided by Zhang et al. [26] ,who identify an inference-time failure mode of reasoning-based generative recommenders: long chain-of-thought traces induce a textual-inertia bias in which a generic-text subspace dominates inference and dilutes the SID evidence needed to decode the correct item. Our analysis targets a 13 different, training-time failure mode: rollout-level credit diffused over reasoning tokens and SID-token predictions. The two diagnoses are nevertheless consistent. An outcome-reward gradient budget spread indiscriminately over reasoning tokens reinforces the same reasoning-token distribution whose dominance Zhang et al. [26] observe at inference time, whereas SAPO concentrates the budget on the verifiable SID tokens. 

C Limitations 

We highlight several directions for extending SAPO beyond the present study. Evaluation scale and diversity. Our experiments focus on three English Amazon Reviews categories with a fixed K = 3 SID hierarchy. Future work can evaluate SAPO on larger catalogs, deeper or less balanced SID hierarchies, cross-lingual item descriptions, and domains with different interaction dynamics. It would also be useful to study how step-aligned credit assignment behaves under different initialization regimes, including weaker Stage 2 checkpoints. Richer feedback settings. This paper uses ground-truth SID tuples to obtain clean per-level verifiable feedback. A natural next step is to extend SAPO to settings with multiple acceptable next items, noisy implicit feedback, or production-oriented click and conversion signals. These settings may require combining SID-level match rewards with calibrated relevance, diversity, or user-satisfaction objectives. System and decoding variants. Our implementation uses a blocked layout in which the policy generates all K thinking blocks before decoding the contiguous SID block. Future work can explore alternative response layouts, decoding constraints, and reward schedules, including interleaved reasoning–SID generation, adaptive step weighting, and deeper integration with trie-constrained decoding. These extensions may further improve efficiency, controllability, and deployment flexibility. 

D Broader Impacts 

SAPO improves the training of generative recommenders that decode items as hierarchical SIDs. On the positive side, per-step credit assignment reduces the tendency of outcome-reward training to inflate reasoning length and collapse reward, which translates into more sample-efficient RL fine-tuning and more interpretable (coarse-to-fine) reasoning traces that practitioners can audit. On the negative side, any improvement to the accuracy of personalized recommenders carries the standard risks of the broader generative-recommendation setting: filter-bubble reinforcement, preference manipulation by adversarial content producers, and information asymmetry between platforms and users. Our contribution targets the RL credit-assignment layer and does not introduce new data sources or deployment pathways beyond those already common in the literature, so we do not identify a novel direct path to harm; we nonetheless encourage downstream practitioners who build on SAPO to pair it with diversity-aware reward shaping and with the auditing tools proposed in the fairness and filter-bubble literature.

F Implementation Layout and Reasoning-Step Pairing 

The policy we actually train and release follows the implementation layout of Eq. (4) , in which all K thinking blocks are generated first and the K SID tokens are decoded as a contiguous block. Under this layout, SAPO defines reasoning step k by pairing the k-th thinking block τ (k) with the 16 k-th SID token s(k). This pairing is used only to assign credit; it is not an alternative autoregressive factorization of the model. The distinction matters because under Eq. (4) every s(k) is conditioned on the full reasoning trace, i.e., all K thinking blocks, and the previously generated SID tokens, not merely on a local step fragment. We adopt the blocked layout because (a) the contiguous SID block enables trie-based constrained decoding and beam search directly over the K-level codebook while reusing the KV cache across SID positions, and (b) later thinking blocks can refine earlier ones before any SID is committed. SAPO therefore never relies on an alternative decode order. Per-step match rewards are placed on SID-token positions (Section 4.2), and per-step advantages are computed from per-step group statistics (Section 4.3); both depend only on the extracted reasoning steps (τ (k), s (k)) and the corresponding level-k match indicators. All definitions and derivations in Sections 3–4.3 and in Appendix E therefore remain valid under the implementation layout, with |y(k)| = |τ (k)| + 1 denoting the size of the reasoning step. 

G Baseline Descriptions 

Traditional sequential recommenders. GRU4Rec [ 4] models user sessions with gated recurrent units (GRUs) and ranks items by session-parallel mini-batch training. SASRec [5 ] applies a left-to-right Transformer to the interaction sequence, predicting the next item with a causal self-attention mechanism. Caser [ 18 ] treats the interaction sequence as a matrix and applies horizontal and vertical convolutional filters to capture point-level and union-level sequential patterns. 

Generative recommenders without reasoning. TIGER [14 ] represents items as sequences of SIDs produced by an RQ-VAE and trains a Transformer to autoregressively generate SID tuples for the next item. HSTU [ 25 ] is a large-scale generative retrieval model that encodes heterogeneous user actions with a hierarchical sequence Transformer. LCRec [ 27 ] aligns a pre-trained language model with collaborative item IDs via a lightweight adapter, enabling generative recommendation without full fine-tuning. LETTER [ 20 ] learns item tokenization end-to-end with the generative recommender, jointly optimizing the codebook and the decoder. 

Reasoning-based generative recommenders. SIDReasoner [ 3] augments a generative recom-mender with chain-of-thought reasoning and trains Stage 3 with outcome-reward RL; it shares our Stages 1 and 2. OneRec-Think [ 10 ] extends the OneRec framework with a reasoning module and outcome-reward GRPO training. R2ec [ 23 ] integrates retrieval-augmented reasoning into generative recommendation and trains with a scalar reward signal. ReaRec [ 17 ] uses a think-then-recommend paradigm with reasoning traces distilled from a teacher model, followed by outcome-reward RL fine-tuning. 

H Dataset Details 

Source and preprocessing. We use the Amazon Reviews corpus [ 13 ] and follow the 5-core sequential recommendation protocol of Kang and McAuley [5] : users and items with fewer than five interactions are filtered out, and the remaining interactions are sorted chronologically per user. We restrict the timespan to October 2016–November 2018 so that all three categories share a comparable activity window. Each user’s interaction sequence is split leave-one-out: the last item is held out for test, the second-to-last for validation, and the remaining prefix for training (with all valid (history, target) sliding-window pairs used as training samples). 

Per-category statistics. Table 4 reports the final split sizes used throughout the paper. The three categories span a range of catalog sizes (3.5k–3.9k items) and activity densities (avg. sequence length 5.96–6.45), providing a consistent but non-trivial test bed for generative recommendation. 

SID tokenization (RQ-VAE). For each category we learn a residual-quantization variational autoen-coder (RQ-VAE) over item text embeddings concatenating title, category, brand, and description. J Implementation Details 

Default backbone. We use Qwen3-1.7B [ 21 ] as the primary backbone and share the Stage 1 and Stage 2 checkpoints across all RL variants so that every comparison isolates the effect of the Stage 3 algorithm. Stages 1 and 2 are trained with AdamW, a cosine schedule, and a 5% warm-up on the enriched SFT and reasoning-activation datasets described in Appendix K. In Stage 3 we build on the VERL [ 16 ] training framework with vLLM-based rollouts. For each prompt we sample G = 16 

rollouts with temperature and top-p both set to 1.0, a maximum response length of 1024 tokens, and a mini-batch of 256 prompts per update. The actor learning rate is 1×10 −6 with a 2% warm-up, the KL coefficient against the Stage 2 reference is set to 10 −3 (low-variance KL), the entropy coefficient is 0, and the clip range ϵ is 0.2 for both SAPO and the outcome-reward baselines. The released SAPO implementation assigns the per-step match reward of Eq. (5) at SID-token positions, attaches the format bonus only to the last SID position, and requires exactly K extracted thinking blocks before any per-step reward is credited.
