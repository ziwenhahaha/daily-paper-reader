## Abstract
Reinforcement learning for multi-turn agents suffers from a credit-assignment mismatch: rewards are sparse and trajectory-level, while success often hinges on a few local decisions. Existing online policy distillation (OPD) provides denser token-level supervision, but typically treats heterogeneous agent trajectories as monolithic strings rather than causal interaction units. We present StepOPSD, a post-rollout preference self-distillation framework that takes the agent step as the unit of credit redistribution. StepOPSD decomposes trajectories into action-centered step segments, rescoring them under hindsight-enriched teacher contexts and converting token-level log-probability gaps into sign-preserving advantage shaping with a normalized per-step credit budget before the GRPO update. Across ALFWorld and Search-QA with Qwen3-1.7B and Qwen2.5-3B-Instruct, StepOPSD attains best or second-best results on subsets most sensitive to local causal errors, including first-place performance on ALFWorld Heat (79.1%), PickTwo (95.0%), Search-QA TriviaQA (61.6%), and tied-best performance on HotpotQA (40.4%). The results further reveal a consistent two-knob law: smaller α_clip acts as a broadly stabilizing local trust region, whereas the optimal global mixing strength λ_mix remains task-dependent. These findings suggest that step-aware distillation is most useful when trajectory-level rewards are weakly aligned with the local action that determines downstream success.

## 2. 方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想
- **StepOPSD（Step‑Aware Online Preference Self‑Distillation）**：以 **“Agent 步（step）”** 为信用再分配的基本单元，而不是整个轨迹。  
- 通过 **事后（post‑rollout）** 对每个步段进行教师‑学生重评分，把教师对步段中 token 的偏好差距转化为对 GRPO 优势的加权调制，而**不改变原始 RL 的更新方向**。

### 2.2 关键技术细节

| 步骤 | 关键操作 | 说明 |
|------|----------|------|
| **1. 步段提取** | 依据任务特定标签（如 `<action>...</action>`）把轨迹切分为原子步段 | - **ALFWorld**：仅提取动作命令（`action_only`）<br>- **Search‑QA**：提取包含内部推理的干净步段，去除外部检索内容（`clean_step_no_observation`） |
| **2. 教师‑学生重评分** | 对每个步段建立教师上下文 `c_T = c_S ⊕ h`（`h` 为后见之明信息） | 采用 **peer‑trajectory hindsight**：在同一 GRPO 组内取第一个成功的同伴轨迹作为教师；教师模型使用 **stale_ref_policy**（10 步前的策略快照），防止移动目标不稳定。 |
| **3. 偏好差距量化** | 对步段内每个 token `z_{k,j}` 计算 log‑概率差<br>`Δ_{k,j} = log π_T(z_{k,j}|c_T) - log π_S(z_{k,j}|c_S)` | Δ 捕捉关键决策点：教师与学生在该 token 上概率分歧越大，Δ 越大。 |
| **4. 优势塑造（Advantage Shaping）** | - 计算原始权重 `w_raw = 2·σ(sign(A_ℓ)·Δ_ℓ)`<br>- 对权重做局部裁剪 `w_ℓ = Π_{[1‑αclip, 1+αclip]}(w_raw)`<br>- 最终优势 `~A_ℓ = (1‑λmix)·A_ℓ + λmix·(w_ℓ·A_ℓ)` | - 权重 `w_ℓ` 在 `[1‑αclip, 1+αclip]` 范围内限制局部更新幅度，**保持优势符号不变**。<br>- `λmix` 控制教师信号的全局混合强度。<br>- **sign‑preserving**：保证更新方向与原始 RL 一致。 |
| **5. 步级归一化（Step‑wise Normalization）** | 对同一步内的 token 权重做均值归一化，使每个步段获得相等的平均修改预算 | 防止冗长推理步因 token 数量多而占据过多信用。 |

### 2.3 整体算法流程（文字描述）

1. **Rollout**：基于当前策略 πθ 在环境中采样完整轨迹 τ（保持原有 GRPO rollout 不变）。  
2. **奖励获取**：计算末端稀疏奖励 R(τ)。  
3. **步段解析**：将 τ 按任务标签拆解为若干步段 {s₁, s₂, …}。  
4. **教师构建**：在同组中选取首个成功的同伴轨迹，构建后见之明教师上下文 c_T。  
5. **偏好差距计算**：对每一步段内的每个 token 计算 Δ（教师‑学生 log‑概率差）。  
6. **优势塑造**：依据 Δ 生成权重 w_ℓ，进行裁剪得到 w_ℓ；混合原始优势得到 ~A。  
7. **步级归一化**：对同一步内的权重进行均值归一化，确保每步获得相同信用预算。  
8. **GRPO 更新**：使用 ~A 进行策略梯度更新，保持优势符号不变，仅重新分配信用。

## 6. 主要结论与发现

1. **步级信用再分配的价值**：当轨迹级奖励与决定最终成功的局部动作错配最严重时（如 Heat、Cool、Pick2、TriviaQA、HotpotQA），StepOPSD 能取得显著提升。  
2. **“两旋钮法则”**  
   - **αclip（局部裁剪）**：收紧（αclip=0.05）在所有场景下均表现为“全局稳健器”，有效抑制后期教师‑学生 gap 的方差爆炸。  
   - **λmix（全局混合强度）**：最优值随任务特性而变：**弱混合（λmix=0.05）** 更适合动作密集型嵌入式任务（如 ALFWorld），**强混合（λmix=0.2）** 更适合检索密集型问答（Search‑QA）。  
3. **训练阶段转折**：约在第 50 步时，RL 基础策略趋于成熟，教师‑学生 gap 方差出现明显转折；弱混合在 ALFWorld 中方差下降，强混合在 Search‑QA 中保持平稳，验证了理论方差上界（Theorem 2）。  
4. **步级归一化防止冗长步主导**：通过 equal_step_mean_abs 约束，每个步段获得相同的信用预算，阻止长推理步因 token 数多而抢夺优化质量。  
5. **最高成绩**（部分子集）：  
   - ALFWorld Heat（79.1%）、Pick2（95.0%）  
   - Search‑QA TriviaQA（61.6%）  
   - HotpotQA（40.4%，并列最佳）

## 7. 优点

- **模块化、非侵入式设计**：仅在 post‑rollout 阶段加入轻量级的 Reward Manager 与 Trainer 子类，保持原有 GRPO rollout 完全不变。  
- **理论保障**：提供 **Sign‑Preserving（命题 1）** 与 **Variance Reduction（定理 2）** 的形式化证明，保证优势塑造不改变最优策略方向并降低梯度方差。  
- **步级归一化**：防止长步垄断信用，保证不同步段在优化过程中拥有公平的影响力。  
- **灵活的后见之明来源**：使用 peer‑trajectory hindsight，无需额外人工标注或外部 oracle，成本低且信息密集。  
- **双层调控（λmix + αclip）**：为不同任务提供细粒度的“全局‑局部”调参空间，兼顾探索与利用。  
- **跨模型规模一致提升**：在 Qwen3‑1.7B 与 Qwen2.5‑3B‑Instruct 两大规模上均表现优异，显示出较好的可扩展性。  
- **实验覆盖面广**：包括 6+7 个子任务、2 种模型规模、多种参数组合及诊断分析，提供了丰富的经验依据。

- **仅事后塑造**：StepOPSD 只在 rollout 结束后介入优势计算，未在生成阶段直接引导 token 级采样决策，因此在极少数极端长序列或高噪声环境下可能仍受限于稀疏奖励的初始学习瓶颈。  
- **依赖 peer‑trajectory hindsight**：后见之明教师的构建基于同组内成功的同伴轨迹，若某任务的成功率极低（如某些 2Wiki、MuSiQue 子集），peer‑hindsight 可能难以获取可靠的成功样本，导致教师信号弱化。  
- **步段切分依赖任务特定的标记规则**：ALFWorld 使用 `<action>...</action>` 标记，Search‑QA 使用清理后的思考链标签。对于全新环境或缺乏结构化标签的 Agent 场景，需要额外的解析模块，迁移成本并非零。  
- **评估指标的单一性**：实验仅使用最终成功率（ALFWorld）和准确率（Search‑QA），未报告如 token 级效率、推理步数或交互轮次等多维度质量指标，难以及评判模型在“效率”与“精度”之间的权衡。  
- **统计显著性缺乏**：在多数子集上的结果未提供跨种子/多次运行的置信区间或显著性检验，可能导致对某几项最高成绩（如 Heat 79.1%）的稳健性评估不足。  
- **计算开销**：虽然只在 post‑rollout 阶段加入教师重评分和步级归一化，但每一次更新仍需对完整的 token 序列做两次策略前向（学生 + 教师），在极大模型（如 7B+）上可能带来不可忽视的显存与时延开销。  
- **对噪声检索的鲁棒性**：Search‑QA 采用固定的 E5 检索器，若检索结果噪声较高，教师‑学生偏好差距可能被错误信息误导，导致优势误塑。文中未对此做针对性消融。
