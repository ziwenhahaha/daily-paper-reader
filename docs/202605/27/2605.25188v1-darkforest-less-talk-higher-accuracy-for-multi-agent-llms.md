## 摘要
多智能体 LLM 系统通过组合多个智能体的输出来提升推理能力，但高度依赖交互的方法可能引入错误传播和高通信开销。当智能体交换原始回答或推理轨迹时，错误的中间推理可能被采纳并放大，导致自信但错误的共识；多轮通信还会增加 token 消耗、延迟和推理成本。本文提出一个名为 DarkForest 的受控通信协调框架。DarkForest 首先保持智能体相互独立，使每个智能体在看不到其他智能体输出的情况下生成答案。随后，它将原始回答解析为结构化候选记录，将语义等价的候选分组成簇，并利用智能体可靠性、置信度、解析质量、支持模式可靠性和独立性校正，估计这些簇上的校准信念分布。协调器只从该信念状态中接收策略允许的证据，从而实现受控通信。在六个推理基准上的实验表明，DarkForest 达到了领先的整体质量，在基准指标上最多比最强基线提升 30.7\%，并且相比重通信基线最多减少 $6.5\times$ 的 token 消耗。

## Abstract
Multi-agent LLM systems improve reasoning by combining outputs from multiple agents, but interaction-heavy methods can introduce error propagation and high communication overhead. When agents exchange raw responses or reasoning traces, incorrect intermediate reasoning may be adopted and amplified, leading to confident but wrong consensus; multi-round communication also increases token consumption, latency, and inference cost. In this paper, we propose a controlled-communication coordination framework named DarkForest. DarkForest first keeps agents independent, so each agent produces an answer without seeing the others' outputs. It then parses the raw responses into structured candidate records, groups semantically equivalent candidates into clusters, and estimates a calibrated belief distribution over these clusters using agent reliability, confidence, parse quality, support-pattern reliability, and independence corrections. A coordinator receives only policy-permitted evidence from this belief state with controlled communication. Experiments on six reasoning benchmarks show that DarkForest achieves leading overall quality, improves the strongest baseline by up to 30.7\% on benchmark metrics, and reduces token consumption by up to $6.5\times$ compared with communication-heavy baselines.

## 1. 核心问题与整体含义（研究动机和背景）

### 1.1 背景

随着大语言模型（LLM）的发展，多智能体 LLM 系统已成为提升测试时推理的重要范式。该系统通过查询多个基于 LLM 的智能体并聚合它们的输出来解决复杂任务。

### 1.2 核心问题

论文识别出当前多智能体 LLM 系统存在的两大关键局限：

- **错误传播（Error Propagation）**：一旦错误或误导性输出被共享，后续智能体可能采纳、细化或放大该错误，导致系统以越来越高的置信度收敛到错误答案。此时智能体间的一致性不再代表独立验证，而可能反映的是模仿、说服性影响或早期响应的污染。

- **通信开销（Communication Overhead）**：跨智能体的重复消息交换大幅增加 token 消耗、延迟和推理成本，使许多多智能体系统在大规模部署时成本高昂。

### 1.3 核心洞察

论文通过实验发现：即使不进行任何跨智能体通信，现有智能体池中已有约 75.2% 的 MATH 样本至少有一个独立查询的智能体生成了正确答案。然而，通信密集型方法的最终准确率远低于这个上限，说明协调过程可能丢弃或覆盖了已有的有用独立证据。

**核心设计问题不是"如何让智能体更多通信"，而是"控制什么信息跨越智能体边界"。**

## 2. 论文提出的方法论

### 2.1 核心思想

DarkForest 从不完整信息博弈理论中汲取灵感，在不确定性、有限信任、高通信成本和高错误成本的环境下，智能体不应暴露超出可靠协调所必需的信息。协调不等于共享更多文本，而是对紧凑的、策略允许的、可验证证据的受控暴露。

### 2.2 三大设计原则

1. **独立候选生成（Independent Candidate Generation）**：智能体不观察其他智能体输出，保持尽可能大的条件独立性。

2. **校准聚合（Calibrated Aggregation）**：一致性由历史可靠性、解析质量、置信度和智能体依赖性加权，而非统一投票。

3. **受控通信（Controlled Communication）**：协调器仅接收策略批准的信息，而非无限制访问所有原始轨迹。

### 2.3 技术流程

#### 2.3.1 解析与规范化（Parsing & Canonicalization）

- 对每个智能体的原始输出应用任务特定解析器 $P$
- 解析后的观察形式为：$o_i = (a_i, z_i, c_i, v_i, q_i)$
  - $a_i$：解析候选
  - $z_i$：规范化候选表示
  - $c_i \in [0,1]$：置信度
  - $v_i \in \{0,1\}$：解析有效性指示
  - $q_i$：解析质量元数据

#### 2.3.2 候选聚类（Candidate Clustering）

- 将语义等价的解析候选分组为簇
- 对每个候选 $z$，定义支持集 $S_z = \{i \in V : z_i = z\}$
- 每个簇表示为 $C_z = (z, S_z, \pi_z)$，其中 $\pi_z$ 是支持模式

#### 2.3.3 校准信念构造（Calibrated Belief Construction）

核心评分公式：

$$s(z) = R_{\pi_z} \sum_{i \in S_z} \alpha_i \cdot \rho_i \cdot \delta_i \cdot \phi(c_i)$$

其中：
- $\alpha_i$：智能体 $i$ 的校准可靠性
- $R_{\pi_z}$：支持模式 $\pi_z$ 的校准可靠性
- $\rho_i$：解析质量惩罚（malformed penalty）
- $\delta_i$：相关智能体的独立性校正
- $\phi(c_i) = 0.5 + c_i$：置信度乘法器（bounded affine form）

归一化为类后验分布：

$$P(z|O) = \frac{\max(0, s(z))}{\sum_{z' \in C} \max(0, s(z'))}$$

#### 2.3.4 离线校准阶段

使用拉普拉斯平滑估计参数：
- $\alpha_i = \frac{n_{corr}^i + 1}{n_{valid}^i + 2}$（智能体可靠性）
- $R_\pi = \frac{n_{corr}^\pi + 1}{n_\pi + 2}$（支持模式可靠性）
- $\lambda_{mal} = \text{clip}(\frac{Acc_{mal}}{Acc_{well}}, \lambda_{min}, 1)$（malformed 惩罚）

#### 2.3.5 受控披露策略（Controlled Disclosure）

- 披露操作符 $E = D(O, B; \Omega)$
- 可暴露：候选标识符、规范候选、解析状态、置信度、紧凑信念摘要
- 禁止暴露：完整原始轨迹
- 记录跨边界 token 消耗：$T_{cross} = \text{tokens}(E)$

#### 2.3.6 协调器与确定性护栏（Coordinator & Guardrail）

**协调器**：接收原始输入和暴露证据，产生最终答案。

**护栏**：仅当信念状态强烈支持与协调器输出冲突的候选时介入。

信任条件：
$$\text{Trusted}(z^*) = (|S_{z^*}| \geq k) \land (P(z^*|O) \geq \tau_p) \land (\Delta \geq \tau_m)$$

最终决策：
$$\hat{y} = \begin{cases} z^*, & \text{if } \text{Trusted}(z^*) \land \hat{z}_{coord} \neq z^* \\ \hat{z}_{coord}, & \text{otherwise} \end{cases}$$

护栏是确定性的，不引入额外模型调用。

### 7.1 理论驱动设计
- 从不完整信息博弈理论中推导设计原则，为多智能体协调提供了坚实的理论基础

### 7.2 全面校准机制
- 不仅考虑智能体可靠性，还校准支持模式可靠性、解析质量、置信度和相关性

### 7.3 受控通信创新
- 首次将通信作为可测量设计变量而非隐式副作用
- 通过策略控制平衡证据保留与错误传播风险

### 7.4 轻量高效护栏
- 确定性护栏不引入额外模型调用，仅使用已有信念状态信息

### 7.5 开源与可复现
- 代码开源，提供完整实验配置

## 8. 不足与局限

### 8.1 实验覆盖局限
- 主要在数学、代码、金融、法律领域测试，未涵盖开放域对话、创意写作等任务
- 智能体池规模有限（默认 3 个），在极端规模下的效果未充分验证

### 8.2 偏差风险
- **校准依赖**：性能依赖于离线校准数据的质量和代表性，若校准集分布与测试集差异大，校准参数可能不准确
- **智能体选择**：使用了特定的预训练模型组合，其结论可能在其他模型架构上有所差异
- **相关性假设**：独立性校正假设智能体间的相关性结构是已知或可估计的

### 8.3 应用限制
- **离线校准要求**：需要准备带标注的校准集，限制了在新领域快速部署
- **任务类型**：在 HumanEval 等代码生成任务上仅与最强基线持平，可能因为代码的细节保留在聚合中更重要
- **护栏阈值敏感性**：虽然论文报告了稳定范围，但实际应用中仍需针对不同任务调优阈值

### 8.4 协调器依赖
- 消融显示协调器选择仍有影响，使用较弱协调器会降低性能
- 虽然护栏提供了保护，但协调器仍是潜在瓶颈

### 8.5 通信模式假设
- 假设独立生成候选是可行的，但对某些需要实时协作的场景（如多智能体环境交互）可能不适用
