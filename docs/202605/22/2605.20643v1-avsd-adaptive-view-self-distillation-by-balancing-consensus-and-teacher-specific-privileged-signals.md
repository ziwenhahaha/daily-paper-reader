## 摘要
引入自适应视图自蒸馏方法，平衡共识与教师特定信号用于语言模型。

## Abstract
Self-distillation enables language models to learn on-policy from their own trajectories by using the same model as both student and teacher, with the teacher being conditioned on privileged information unavailable to the student. Such information can come in different types or views, such as solutions, demonstrations, feedback, or final answers. This setup provides dense token-level feedback without relying on a separate external model, but creates a fundamental asymmetry: the teacher may rely on view-specific information that the student cannot access at inference time. Moreover, the best type of privileged information is often task-dependent, making it difficult to choose a single teacher view. In this work, we address both these challenges jointly by introducing AVSD (Adaptive-View Self-Distillation), a novel method of self-distillation with multiple privileged-information views, which reconstructs token-level supervision by separating stable cross-view consensus from view-specific residual signals. AVSD identifies the consensus signal shared across views, which provides a reliable update direction, and then selectively adds the view-specific residual signal to adjust the update magnitude when it both aligns with the consensus direction and remains proportionate to the consensus signal. Experiments on math competition benchmarks (AIME24, AIME25, and HMMT25) show that AVSD consistently outperforms both single-view self-distillation baselines and GRPO, achieving average Avg@8 gains of 3.1% and 2.2% over the strongest baselines on Qwen3-8B and Qwen3-4B, respectively. Moreover, on code-generation benchmarks (Codeforces, LiveCodeBench v6) using Qwen3-8B, AVSD outperforms the single-view self-distillation baseline by 2.4% on average.

## 1. 论文的核心问题与整体含义

### 研究背景
- **自蒸馏（Self-Distillation）** 是大语言模型后训练的重要范式，使用同一模型同时作为学生和教师，教师借助特权信息（Privileged Information）为学生提供密集的 token 级监督信号
- 特权信息可以是完整解决方案、演示、反馈或最终答案等形式
- 该方法避免了对外部教师模型的需求，但引入了根本性不对称：教师可访问的信息学生无法在推理时获取

### 核心问题
- **问题一：信息不对称下的教师-学生分布偏移**  
  直接匹配特权教师会迫使模型编码视图特定的关联，而这些关联在测试时学生无法观察
- **问题二：最优特权信息类型难以事先确定**  
  不同特权信息类型（如完整方案 vs 部分推理 vs 仅答案）在不同数据集上表现差异显著，且没有单一类型在所有任务上最优
- **核心研究问题**：能否同时利用多个特权视角来构建比任何单一特权教师更好的 token 级学习信号？

### 研究动机
- 现有自蒸馏方法隐含假设：选定特权上下文类型后在整个训练过程中保持固定
- 不同特权信息类型具有不同的信息密度、任务效用和诱导的学生-教师分布偏移
- 选择单一特权视角会显著影响性能和泛化能力

## 2. 论文提出的方法论

### 核心思想
AVSD（Adaptive-View Self-Distillation）是一种多视角自蒸馏框架，核心洞察是：
- **共识信号**：不同特权视角诱导相似的 token 级更新 → 信号稳定且任务相关
- **残差信号**：仅被少数视图强烈支持的 token → 可能捕获互补信息，但也可能反映特权伪影，风险更高

### 关键技术细节

#### 2.2.1 几何共识目标（Geometric Consensus Target）
- **定义**：归一化几何均值  
  $$qGt(v) = \frac{eqGt(v)}{\sum_u eqGt(u)}, \quad eqGt(v) = \left(\prod_{m=1}^M q_t^{(m)}(v)\right)^{1/M}$$
- **特性**：要求 token 被所有教师视图支持才获得高概率，保守且强调跨视图一致性
- **几何优势**：$$AGt(v) = \log eqGt(v) - \log pt(v) = \frac{1}{M}\sum_{m=1}^M \Delta_t^{(m)}(v)$$

#### 2.2.2 算术边际目标（Arithmetic Marginal Target）
- **定义**：算术均值  
  $$qAt(v) = \frac{1}{M}\sum_{m=1}^M q_t^{(m)}(v)$$
- **特性**：保留被至少一个视图强烈支持的 token，允许个别视图的贡献更显著
- **残差定义**：$$J_t(v) = \log qAt(v) - \log eqGt(v) \geq 0$$

#### 2.3 残差门控机制（Gated Residual）
- **目的**：始终保留视图共识，仅在残差可信时添加
- **重建优势**：$$\tilde{A}t(v) = AGt(v) + \lambda_t(v)J_t(v)$$
- **门控公式**：$$\lambda_t(v) = Ct(v) \cdot Rt(v)$$
  - **对齐组件** $Ct(v)$：衡量视图间更新方向一致性  
    $$Ct(v) = \frac{|AGt(v)|}{\frac{1}{M}\sum_m |\Delta_t^{(m)}(v)| + \epsilon}$$
  - **幅度组件** $Rt(v)$：确保残差与共识成比例  
    $$Rt(v) = \frac{|AGt(v)|}{|AGt(v)| + J_t(v) + \epsilon}$$

#### 2.3 重建目标与训练目标
- **重建目标**：$$q^\star_t(v) \propto eqGt(v) \exp(\lambda_t(v)J_t(v))$$
- **训练目标**：$$L_{AVSD}(\theta) = \mathbb{E}_{(x,r)\sim D, y\sim P_{\theta}^S(\cdot|x)}\left[\sum_{t=1}^{|y|} DKL(p_\theta(\cdot|h_t) \parallel sg[q^\star_t(\cdot)])\right]$$

### 算法流程（简化描述）
1. 对每个训练样本，构造 M 个特权视图（完整方案、部分方案、仅答案等）
2. 采样 on-policy rollout $y$
3. 对每个前缀 $h_t$，计算学生分布 $p_t$ 和 M 个视图的教师分布 $q_t^{(m)}$
4. 计算几何共识目标、算术边际目标及其残差
5. 计算门控值 $\lambda_t(v)$，重建教师目标 $q^\star_t$
6. 通过反向 KL 蒸馏损失更新学生模型

## 6. 论文的主要结论与发现

### 核心发现

#### 主结果
| 模型 | 最强基线改进 | 具体提升 |
|------|-------------|----------|
| Qwen3-4B | OPSD | Avg@8 提升 **2.2%** |
| Qwen3-8B | GRPO | Avg@8 提升 **3.1%** |
| DeepSeek-R1-Qwen-7B | OPSD | Avg@8 提升 **2.3%** |
| Qwen3-8B 代码 | OPSD | Avg@8 提升 **2.4%** |

#### 关键实验发现

1. **无单一特权视图最优**：不同数据集上最优特权视图不同
   - AIME24/25：完整方案可能过于约束
   - HMMT25：答案-only 可能信息不足
   - AVSD 在所有数据集上持续最优

2. **消融验证**：
   - 共识目标：保守，在 AIME24/25 强但 HMMT25 弱
   - 算术目标：在 AIME24/HMMT25 提升但 AIME25 有限
   - AVSD：综合两者优势，在所有基准上最优

3. **Token 级学习信号可靠性**：
   - GRPO：29.6% 的错误 rollout 无学习信号
   - OPSD：12.3% 的高影响 token 分配错误符号
   - AVSD：仅 2.9%（显著更可靠的信用分配）

4. **视图数量缩放**：
   - 从 1→3 个视图：AIME24/25 提升约 2.8-2.9%
   - 第 4 个视图：收益递减
   - 代码任务仍能从第 4 个视图获益

5. **门控行为**：
   - Gate 开启率：21-27%（模型间稳定）
   - 表明 AVSD 主要依赖共识，但选择性加入残差

## 8. 不足与局限

### 实验覆盖局限

1. **任务范围有限**  
   主要聚焦推理任务（数学/代码），未验证在对话、摘要、分类等任务上的效果

2. **数据质量依赖**  
   嘈杂的标注解决方案可能导致不可靠的教师信号

3. **未测试工具使用 Agent**  
   作者承认这是有价值但未覆盖的方向

### 方法论局限

1. **依赖反向 KL 目标**  
   门控的正则化和方向保持保证依赖于反向 KL 的策略梯度解释，切换目标可能需要重新分析

2. **视图构建质量**  
   AVSD 性能受限于构造的视图质量，自动化高质量视图构造未探讨

3. **互信息间隙问题**  
   虽然门控减轻了但未完全消除特权信息泄露风险

### 潜在偏差风险

1. **评估基准代表性**  
   主要在竞赛级数学/编程任务，可能不适用于更广泛推理场景

2. **训练数据依赖**  
   使用 OpenThoughts 子集，可能存在数据集特定偏差

3. **模型家族局限**  
   主要在 Qwen 和 DeepSeek 系列验证，迁移到其他模型族（如 Llama、Mistral）未验证
