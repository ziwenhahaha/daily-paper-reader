## 摘要
大型语言模型（LLMs）使得协作式多智能体（MA）系统成为可能，在该系统中交互的智能体通过多样化推理和迭代精炼提升性能。然而，这些系统仍然容易受到错误传播的影响，早期阶段的信息会削弱下游推理。为解决此问题，我们对智能体间通信进行系统分析，以确定哪些信息推动了多智能体系统的性能。我们发现，缺少推理和验证的智能体间通信会显著降低性能。基于这些见解，我们提出了类别感知恢复增强技术（CARA），该技术在通信过程中强制保留关键信息，恢复了高达86.2%的失败案例。结果表明，信息质量在有效的多智能体协作中扮演关键角色。我们的代码可在 https://anonymous.4open.science/r/cara_mas 获取。

## Abstract
Large Language Models (LLMs) have enabled collaborative Multi-Agent (MA) systems, where interacting agents improve performance through diverse reasoning and iterative refinement. However, these systems remain vulnerable to error propagation, where early-stage information degrades downstream reasoning. To address this, we conduct a systematic analysis of inter-agent communication to identify which information drives MA performance. We find that the absence of reasoning and verification in inter-agent communication significantly degrades performance. Based on these insights, we propose Category-Aware Recovery Augmentation (technique), which enforces the presence of critical information during communication. recovers up to 86.2% of failed cases. Our results highlight the key role of information quality in effective MA collaboration. Our code is available at https://anonymous.4open.science/r/cara_mas

## 1. 论文核心问题与整体含义（研究动机与背景）

### 研究动机

大型语言模型（LLM）的快速发展使得多智能体（Multi-Agent, MA）协作系统成为可能。在这类系统中，多个 LLM 智能体通过交换响应、相互批评和迭代改进来共同完成复杂任务。然而，现有研究指出这类系统面临**错误传播（error propagation）**的严重问题——早期阶段的错误信息会在后续交互轮次中被强化和放大，导致最终输出质量下降。现有的改进方案（如调整智能体行为或系统结构）往往是**反应式（reactive）**的，在失败发生后才介入，且主要关注最终任务正确性，**忽视了智能体间通信内容本身的作用**。

### 核心问题

- **RQ1**：多智能体系统中，智能体交换的信息可以归为哪些类别？
- **RQ2**：移除特定信息类别会如何影响多智能体任务的性能？
- **RQ3**：能否通过强制关键信息类别的存在来恢复失败的案例？

### 整体含义

本研究首次**系统化地**对多智能体系统中的智能体间通信进行分类与量化分析，揭示了并非所有信息类别都同等重要，信息质量是决定多智能体协作有效性的关键因素。

## 2. 论文方法论

### 2.1 信息类别定义

通过人工标注分析（400条标注轨迹，80层分层采样），论文识别出智能体响应中的**五种信息类别**：

| 类别 | 定义 |
|------|------|
| **C1 (Answer)** | 智能体的最终答案或决策，通常带有“Answer:”等标记 |
| **C2 (Reasoning)** | 推导答案的逻辑步骤序列，每一步引入新信息以趋向结论 |
| **C3 (Verification)** | 对推理过程和答案的评估，通过回溯验证逻辑是否成立 |
| **C4 (Reference)** | 直接引用其他智能体的推理或答案，以确认、挑战或建立在之上 |
| **C5 (Unchanged)** | 明确声明答案保持不变，如“The answer remains unchanged” |

### 2.2 五种多智能体系统架构

- **Seq-U**（序贯-均匀）：顺序处理，所有先前响应对后续智能体可见
- **Seq-R**（序贯-角色）：角色分工（Planner→Solver→Reviewer），顺序处理
- **Debate**：包含裁判（Judge）的中介，提议者（Proposer）和批评者（Critic）对抗性交互
- **CR-MC**：集体精炼 + 多数共识协议（≥50%智能体同意则终止）
- **CR-SV**：集体精炼 + 简单投票协议（三轮后投票选出最终答案）

### 2.3 遮挡分析（Occlusion Analysis）

采用**留一法遮挡（Leave-One-Out Occlusion）**框架，对每个信息类别进行系统性移除，测量性能变化。实验设置包括：
- 对每个类别 C_t，执行遮挡并测量任务性能变化（ΔC_t）
- 设置长度匹配的**控制遮挡（Control Occlusion）**，确保性能变化由语义内容而非文本长度变化引起
- 总计 **247,980 次**遮挡实验运行

### 2.4 CARA（Category-Aware Recovery Augmentation）

基于遮挡分析发现，论文提出**两层增强策略**：

**第一层：提示增强（Prompt Augmentation）**
- 在系统提示中明确要求必须包含的信息类别（如 Reasoning:、Verification:、Reference:）
- 针对有前置响应的智能体，要求包含 Reference 类别

**第二层：响应验证（Response Verification）**
- 使用正则表达式检查生成响应中是否包含必需关键词
- 若缺失关键信息，最多重试**3次**返回智能体进行修正

## 6. 主要结论与发现

### 6.1 信息类别流行度（RQ1）

| 类别 | 流行度 | 发现 |
|------|--------|------|
| C2 (Reasoning) | 超过90% | 最普遍，推理交换是核心 |
| C1 (Answer) | 超过90% | 答案传播是主要驱动力 |
| C4 (Reference) | Debate系统高达50%，其他<5% | 辩论系统鼓励批评和引用 |
| C3 (Verification) | 整体较低 | 验证环节不足 |
| C5 (Unchanged) | 低于5% | 停止决策讨论不足 |

### 6.2 遮挡分析关键发现（RQ2）

- **移除 C2（Reasoning）** 持续降低性能（高达 -18.63%），是协作推理的关键
- **移除 C1（Answer）** 有时**改善**性能（高达 +17.13%），说明中间答案可能引入冗余
- **移除 C4（Reference）和 C5（Unchanged）** 导致最大性能下降（-38.69%，-32.97%），对维持连贯通信至关重要
- **C3（Verification）** 的作用因任务而异：改善 Code（+23.27%），但降低 Math（-18.08%）

### 6.3 系统间差异

| 系统 | 最关键类别 | 说明 |
|------|------------|------|
| Seq-U | C1, C2 | 答案和推理可能引入冗余 |
| Seq-R | C2 | 角色分工下的推理生成至关重要 |
| Debate | C5 | 确认先前正确响应的能力关键 |
| CR-MC, CR-SV | C4 | 引用先验响应对集体决策至关重要 |

### 6.4 CARA恢复效果（RQ3）

| 指标 | 最高恢复率 |
|------|------------|
| **TF样本** | 86.0%（Seq-R, Math） |
| **FF样本** | 62.2%（Seq-R, Math） |
| **平均TF恢复** | 62.2%（Qwen2.5-Inst） |
| **平均FF恢复** | 30.0%（Qwen2.5-Coder） |

**总体结论**：CARA 能恢复高达 86.2% 的失败案例，验证了信息质量在多智能体协作中的决定性作用。

## 7. 优点

1. **首创性**：首次系统化地对多智能体系统中的智能体间通信进行分类与量化分析
2. **方法论严谨**：遮挡分析结合留一法框架，定量评估每个信息类别的贡献
3. **实验全面**：覆盖5种系统架构、6个数据集、2个模型后端、41,330条通信轨迹
4. **实用干预**：CARA 作为一种轻量级、可迁移的增强策略，无需修改底层模型或架构
5. **统计充分**：所有关键结果均标注统计显著性（p<0.05）
6. **标注可靠性**：LLM标注器与人类标注者的一致性达92%（Cohen's Kappa=0.88）

## 8. 不足与局限

1. **模型覆盖有限**：仅使用两个 Qwen 系列模型，结论可能难以泛化到其他 LLM 架构（如 GPT、Claude 等）
2. **类别不完整**：承认信息类别体系并非穷尽，可能存在其他影响性能的信息类别
3. **系统选择**：仅评估5种MA系统设计，替代架构可能展现不同的通信动态
4. **提示敏感性**：任务结果可能受提示设计影响，尽管已采用已建立的MA提示策略
5. **评估范围**：在6个数据集和2个LLM上评估，泛化到其他任务或模型的结论需谨慎
6. **潜在数据泄露**：虽然影响所有方法且分析聚焦相对比较，但仍存在风险
