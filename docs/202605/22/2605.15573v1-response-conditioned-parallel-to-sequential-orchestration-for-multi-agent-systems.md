## 摘要
多智能体系统能够通过多个大型语言模型智能体之间的协作来解决复杂任务。现有的协作框架通常以并行或顺序模式运行。在并行模式下，智能体独立响应查询，随后对响应进行聚合。相比之下，顺序系统允许智能体通过有向拓扑进行通信，并逐步相互优化。然而，这两种模式都无法同时实现最小化通信和延迟以及最大化最终响应准确性的目标。在本工作中，我们引入了一种名为 Nexa 的混合范式，这是一种可训练的条件化策略，弥合了两种模式之间的差距。Nexa 从并行执行阶段开始，将结果响应嵌入到共享语义空间中，然后预测稀疏有向无环通信图。如果图为空，系统保持纯并行；如果非空，系统执行一次顺序消息传播。该策略是一个轻量级 transformer 模型，且该方法避免了对外 部 LLM 裁判或奖励模型的需求，以及手工设计的测试时拓扑搜索。我们形式化了这个混合执行问题，证明了生成的图在结构上是无环的，且该框架严格包含纯并行执行，并提出了基于策略梯度优化的训练方法。实验结果表明，Nexa 在某一设置下学到的响应条件化策略在智能体数量、任务或底层智能体改变时可以被复用，从而强调了所学通信策略的泛化能力。

## Abstract
Multi-agent systems can solve complex tasks through collaboration between multiple Large Language Model agents. Existing collaboration frameworks typically operate in either a parallel or a sequential mode. In the parallel mode, agents respond independently to queries followed by aggregation of responses. In contrast, sequential systems allow agents to communicate via a directed topology and refine one another step by step. However, both modes are inadequate for achieving the desired objectives of minimizing communication and latency while simultaneously maximizing the accuracy of the final response. In this work, we introduce a hybrid paradigm called Nexa, a trainable response-conditioned policy that bridges the gap between the two modes. Nexa begins with a parallel execution stage, embeds the resulting responses into a shared semantic space, and then predicts a sparse directed acyclic communication graph. If the graph is empty, the system remains purely parallel; if it is non-empty, the system performs one sequential message propagation. The policy is a lightweight transformer model, and the method avoids the need for external LLM judges or reward models, as well as hand-crafted test-time topology search. We formalize this hybrid execution problem, show that the resulting graph is acyclic by construction, and that the framework strictly subsumes pure parallel execution, and present a training procedure based on policy-gradient optimization. Results demonstrate that the response-conditioned policy learned by Nexa under one setting can be reused when the number of agents, the task, or the underlying agent changes, thus emphasizing the generalizability of the learned communication policy.

## 1. 论文的核心问题与整体含义（研究动机和背景）

多智能体系统通过多个大型语言模型（LLM）智能体之间的协作来解决复杂任务。当前主流协作框架分为两类：
- **并行模式（Parallel）**：各智能体独立回答查询，随后对响应进行聚合（如投票、自洽等）。缺点是计算开销大、token消耗高、且无法利用定向通信来修复错误答案。
- **顺序模式（Sequential）**：智能体按有向拓扑逐步传递信息、相互优化。缺点是需要手工设计拓扑结构，且拓扑固定后难以适应不同实例。

两种模式均无法同时实现**最小化通信与延迟**以及**最大化最终响应准确性**的目标。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
提出 **NEXA（NEXA）**，一种响应条件化的混合编排策略：先并行执行产生候选响应池，再基于响应内容预测是否需要顺序通信——如果需要则预测一条稀疏的有向无环通信图（DAG），执行一轮顺序传播。

### 关键技术细节

1. **并行草案阶段**：所有 N 个智能体独立生成响应 $\mathcal{R}^{(0)} = \{R_n^{(0)}\}_{n=1}^N$。
2. **语义嵌入**：使用固定编码器（如 all-MiniLM-L6-v2）将每个响应映射为向量 $r_n = f(R_n^{(0)}) \in \mathbb{R}^d$。
3. **贡献排序（Contribution-Defined Order）**：计算平均嵌入 $r_{\text{avg}} = \frac{1}{N} \sum_n r_n$ 和贡献分数 $\psi_n = \cos(r_n, r_{\text{avg}})$，按 $\psi$ 降序排列得到拓扑序 $\pi$。
4. **响应条件化图策略**：轻量级Transformer编码器 $\text{Enc}_\theta$ 对响应嵌入矩阵 $X = [r_1, ..., r_N]^\top$ 进行全局上下文建模，得到隐状态 $H = \text{Enc}_\theta(X)$。再通过 $\Lambda = HH^\top$ 生成边logit，采样得到稀疏通信图 $G = (V, E, \pi)$。
5. **顺序传播**：若 $E = \emptyset$（空图），直接返回并行聚合结果；否则按 $\pi$ 顺序，每个节点根据父节点最新响应更新自身响应。
6. **无外部裁判的聚合**：计算加权质心 $z_{\text{centroid}} = \frac{\sum_n w_n z_n}{\sum_n w_n}$，选择与质心余弦相似度最高的响应作为最终答案。
7. **DAG保证**：边的可行集被限制为 $E_\pi = \{(m, n) : \pi^{-1}(m) < \pi^{-1}(n)\}$，即只能从高贡献节点指向低贡献节点，从而保证图的无环性。
8. **训练目标**：使用REINFORCE策略梯度优化，奖励函数为 $R_{\text{sp}}(G) = R_{\text{task}}(G) - \lambda_{\text{sp}} \frac{|E|}{M}$（含稀疏性惩罚）。

### 公式概览
- **响应生成**：$R_n^{(0)} = A_n(Q)$
- **贡献分数**：$\psi_n = \cos(r_n, r_{\text{avg}})$

1. **准确率-成本权衡最优**：NEXA在 AQUA-RAT、HumanEval、GSM8K 上取得最高平均准确率（60.90%），同时 token 消耗比 SelfOrg⋆ 减少约35%，比 GPTSwarm 减少超过50%。
2. **跨智能体数量泛化**：在 N=10 训练的策略可直接用于 N∈{5,15,20} 而无需重训练，且始终优于单次调用和 CoT 基线。
3. **跨任务迁移**：AQUA-RAT ↔ GSM8K 的迁移差距仅为0.05–0.18个百分点。
4. **跨模型规模迁移**：1.5B训练的策略迁移到7B模型后表现接近在7B上直接训练的策略（GSM8K：90.48 vs 90.52）。
5. **跨模型代际迁移**：Qwen2.5-1.5B → Qwen3.5-2B 迁移后仅差0.17个百分点。
6. **通信行为特点**：随团队规模增大，救援率提升（19.2%→23.8%），伤害率保持低水平（1.6%–2.5%），保全率超过97.5%；且倾向于选择低边（稀疏）通信计划。
7. **骨干无关性**：Transformer和GNN骨干表现相近，说明核心收益来自NEXA框架设计而非特定网络架构。
8. **优化器鲁棒性**：PG略优于GRPO但差距极小，表明通信策略对优化器选择不敏感。

## 7. 优点

- **混合范式设计**：首次将并行和顺序执行统一为一个可学习的决策问题，而非将二者视为互斥的设计选择。
- **响应条件化**：策略基于实际响应内容而非静态任务标签或角色分配，天然具有跨配置迁移的潜力。
- **DAG结构保证**：通过限制可行边集保证无环性，无需后处理循环检测。
- **稀疏通信学习**：显式稀疏性惩罚使策略倾向于选择最小通信计划，降低开销。
- **无外部裁判**：利用语义嵌入和贡献分数实现端到端、无需LLM法官的聚合与决策。
- **泛化能力强**：在任务、智能体数量、模型规模、模型代际等多个维度均展示了可观的迁移能力。
- **轻量级实现**：策略网络仅一层Transformer，训练和推理开销极低。
- **贡献定义拓扑序**：借用 Shapley 近似概念稳定化通信顺序，使顺序传播具有可解释的基础。
