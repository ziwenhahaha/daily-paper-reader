## 摘要
基于大型语言模型（LLM）的多智能体系统需要进行大量协调选择，这些选择在先验情况下难以确定：调用何种技能协议、哪个智能体角色应执行子任务、将哪个模型绑定到每个角色、角色之间如何交互、何时使用检索或验证、以及何时完全跳过某个步骤。这些选择与任务模式和操作约束相互影响，因此静态流程和一次性模型比较仅能提供设计空间的有限视角。本文提出了AgensFlow，一个开源框架，将多智能体协调视为部分可观测条件下的在线策略学习问题。该框架使协调决策可观察且可从重复轨迹中学习，而不是将技能、角色、模型、拓扑和评估选择视为固定流程设计。AgensFlow在两个语料库上进行了评估：分布式系统事故任务和安全咨询任务。评估显示了三个主要结果：学习路由在协调密集类上达到了比固定流程基线更高质量的运行点；skip:X将拓扑压缩隔离为底层的有意义部分；热启动策略图可以降低探索成本，同时保持平台质量。总体而言，结果表明可学习、可审计的路由可以改进静态连接方式在协调密集型多智能体工作流程中的表现。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
AgensFlow 将多智能体编排形式化为**部分可观测的顺序决策过程**，学习状态抽象空间上的策略。

### 状态表示（Folded Task Signatures）
$$
s_t = \phi(x_t, h_t, b_t) = (l_t, z_t, \beta(\hat{c}_t), \beta(\hat{u}_t), \beta(\hat{r}_t), \beta(\hat{e}_t))
$$

- **$l_t$**：任务领域标签（straightforward, evidence_heavy, ambiguous, contradictory, high_risk, exploratory）
- **$z_t$**：7位二进制交接掩码（goal, subproblem, evidence, critique, verification, draft_answer, merged_answer）
- **$\beta(\cdot)$**：信念估计（正确性、不确定性、矛盾风险、证据充分性）

### 动作空间
$$
a_t \in A(s_t) = \{\text{invoke}(k,m)\} \cup \{\text{skip}:X\} \cup \{\text{terminate}\}
$$

- **invoke(k,m)**：技能协议 k 绑定模型 m（如 solver_concise_haiku）
- **skip:X**：跳过可选单元 X（拓扑压缩的关键机制）
- 动作选择是**顺序的、贪心的**，延迟进行轨迹级奖励备份

### 奖励函数
$$
r(t) = w_q Q(t) - w_c C(t) - w_\rho \rho(t)
$$

- **Q(t)**：RelativeJudge 质量分数（[0,1]）
- **C(t)**：归一化 token 成本
- **$\rho(t)$**：重试/失败惩罚
- 默认权重：$w_q=1.0, w_c=0.3, w_\rho=0.15$

### 策略图与动作选择（可靠性感知 UCB1）
$$
\text{score}(s,a) = \bar{r}(s,a) + c_s \sqrt{\frac{\log(N_s+1)}{N_{s,a}}} - \lambda f(s,a)
$$

- **$\bar{r}(s,a)$**：平均奖励
- **探索系数衰减**：$c_s = \max(0.5, 1.4 \cdot 2^{-N_s/50})$
- **失败惩罚**：$-\lambda f(s,a)$，降低频繁触发验证/可恢复失败的动作权重

### 六层架构
| 层级 | 职责 |
|------|------|
| Layer 5 | 预检 + 治理 + 报告 |
| Layer 4 | 奖励 + RelativeJudge |
| Layer 3 | 策略图 + 路由器 + LangGraph |
| Layer 2 | 智能体 + 传输 + 追踪 |
| Layer 1 | Schema + 领域 + 技能注册 |
| Layer 0 | Web 搜索封装 |

### RelativeJudge 奖励设计
- **同任务相对轨迹评分**：减少奖励黑客攻击压力
- **跨评判模型平均**：多模型家族评判（Anthropic + OpenAI + Qwen）
- **四轴分解**：goal_achievement, grounding, coordination, recovery
- **置信度加权**：根据评判分歧调整备份权重

## 6. 主要结论与发现

### 核心发现

#### 1. 学习路由优于固定流程
- 主方法 plateau（6-8 epochs）在 3 评判审计下 RJ +0.068/token +15%
- **5/8 类别获胜**，集中在协调密集类：
  - C3（跨文档多供应商）：+0.181
  - C8（跨供应商对）：+0.156
  - C7（缓解正确性）：+0.131

#### 2. skip:X 隔离拓扑压缩价值
- 启用 skip:X 比禁用（no-skip）在 plateau 质量更高
- 冷启动学习轨迹显示：初始压缩成本 → 质量下降 → 重新平衡

#### 3. 热启动迁移降低探索成本
- 初始 443 节点分布式系统策略图 → 安全咨询语料库
- **token 减少 ~21%**（全流程）/ ~10%（plateau）
- 审计 plateau 质量：+0.012（全流程 +0.003）
- 跨域先验保留 plateau 质量，降低探索开销

#### 4. 奖励信号鲁棒性
- 单评判 vs 三评判对结论有影响
- 热启动质量优势在单评判下被高估（+0.052 → 审计 +0.012）
- **奖励审计应作为系统设计的一部分**

## 8. 不足与局限

### 实验覆盖局限
- **拓扑类型单一**：仅测试线性-with-skip 拓扑
- **未测试**：并行执行、分支联盟、层级分解、swarm 风格
- **领域覆盖有限**：仅技术领域（分布式系统、安全咨询）

### 偏差风险
- **评判依赖**：在线训练仅用单一评判，可能引入评判家族偏差
- **信念启发式**：初始版本信念估计为启发式，可能不够鲁棒
- **任务相似性**：合成任务可能不完全代表真实生产环境

### 应用限制
- **策略图迁移条件**：当前要求共享特征分类法
- **冷启动开销**：需要足够轨迹积累才能发挥价值
- **复杂系统门槛**：六层架构增加了系统复杂度
