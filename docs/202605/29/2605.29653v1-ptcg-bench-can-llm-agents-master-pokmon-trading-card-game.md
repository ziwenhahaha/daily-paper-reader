## Abstract
Given a strategically complex board game, human players can quickly learn to devise strategies after playing a few rounds. Autonomous agents require similar capabilities in realistic interactive environments, yet existing agent benchmarks often fail to fully capture such strategic and evolving decision-making scenarios. We present PTCG-Bench, a benchmark built on the Pok'{e}mon Trading Card Game (PTCG) that evaluates LLM agents at two complementary levels: (1) their decision-making performance within a single complex environment, and (2) their ability to self-evolving through accumulated experience. We further include a modular harness ablation to better interpret agent performance without conflating it with model capability. Our experiments show that, although LLM agents can achieve non-trivial gameplay performance, sustained and stable self-evolution remains challenging, and performance is sensitive to harness design. We hope that PTCG-Bench will facilitate future research on harness-aware and self-evolving agents in realistic interactive environments.

## 2. 论文方法论：核心思想、关键技术细节

### 核心思想
PTCG-Bench基于宝可梦集换式卡牌游戏（Pokémon Trading Card Game, PTCG），该游戏是一个双人、回合制、零和的策略卡牌游戏，每个玩家使用60张牌组，包含宝可梦、能量和训练师牌。游戏结合了隐藏信息、随机抽卡、动态合法动作和长期资源规划等挑战，是评估LLM智能体战略决策能力的理想环境。

### 关键技术细节

**A. 游戏环境引擎**
- 自定义Python实现的PTCG引擎，确保规则执行的可重复性和合法动作验证
- 实现了完整的游戏机制：宝可梦、能量、训练师、进化约束、撤退成本、特殊状态、Knock Out判定和标准胜利条件
- 提供React前端用于可视化、回放和定性分析

**B. 牌组池设计**
- 使用5张覆盖不同战略原型的竞技牌组：
  - Charizard ex（攻击型/阶段性进化）
  - Gardevoir ex（控制型/递归）
  - Miraidon ex（快攻型/基础宝可莫）
  - Gholdengo ex（组合型/爆发）
  - Lugia Archeops（消耗型/工具箱）

**C. 状态与动作空间**
- 分离私有、公共和全局状态，保留不完全信息
- Agent观察自己的私有区域和公共棋盘信息，对手的手牌、牌组顺序和奖品卡身份保持隐藏
- 动作空间定义为参数化游戏动作的工具schemas

**D. Agent-Environment接口**
- 游戏引擎在每个决策点暴露当前状态和合法动作集
- Agent harness将观察与交互历史结合，渲染为LLM可读的提示
- 骨干LLM返回结构化动作请求（工具调用），由harness解析并由引擎验证
- 无效请求触发重试

**E. 模块化Harness架构**
分解为三个可独立消融的关键模块：

| 模块 | 描述 |
|------|------|
| 观察结构（H1） | 将游戏状态转换为结构化描述 vs. 原始状态或游戏日志 |
| 合法动作masking（H2） | 移除引擎计算的合法动作集，要求Agent从当前状态推断有效动作 |
| 上下文管理（H3） | 移除近期轨迹历史，保留当前状态和合法动作 |

**F. 纵向评估协议**
- 评估自我进化能力：游戏组织为多轮完整PTCG比赛
- 每轮后，进化Agent更新持久状态（反思、战略摘要、记忆条目、修订提示或技能文档）
- 分离局内决策与跨游戏经验更新
- 使用锚定比赛（vs.固定锚基线）防止评分漂移

**G. 评估锦标赛设计**
- **固定Agent**：循环赛评估，配置间直接比较，支持共享评分量表上的整体实力比较
- **自我进化Agent**：锚定比赛，每个进化Agent与固定评分的锚基线对战，锚不变化确保稳定的参考群体

## 3. 实验设计：数据集、场景、benchmark、对比方法

### 骨干模型评估（RQ1）
- **模型**：10个来自5个模型家族的backbones（每个家族包含前沿版和成本效益版）：
  - Google: Gemini 3.1 Pro / Gemini 3 Flash
  - Anthropic: Claude Sonnet 4.6 / Claude Haiku 4.5
  - DeepSeek: DeepSeek V4 Pro / DeepSeek V4 Flash
  - OpenAI: GPT-5.4 / GPT-5.4 Nano
  - Qwen: Qwen3.6 Plus / Qwen3.5 Flash
- **非LLM参考**：Random（均匀采样合法动作）和Heuristic（预定义权重采样）
- **设置**：统一ReAct harness，镜像对战（使用同一牌组）
- **目标**：验证PTCG-Bench能否区分不同backbone的战略能力

### 自我进化评估（RQ2）
- **配置**：5种代表性自我进化方法，均使用Qwen3.5-Flash作为骨干
  1. **Reflexion**：基于反思的语言强化学习（Shinn et al., 2023）
  2. **ExpeL**：经验学习者，聚合经验并提炼为可重用战略经验（Zhao et al., 2024）
  3. **Long-Term Memory**：基于记忆的Agent设计（Park et al., 2023; Packer et al., 2023; Xu et al., 2026）
  4. **Prompt Evolution**：将策略提示本身视为持久状态进行演化（Madaan et al., 2023; Wang et al., 2024）
  5. **Skill Library Evolution**：维护结构化战略技能的持久集合（Anthropic, 2026; Alzubi et al., 2026）
- **设置**：R=8轮，每轮与12个固定锚对战（10个静态LLM Agent + Random + Heuristic），共24场游戏/轮，120场/N
- **目标**：验证PTCG-Bench能否追踪跨序列游戏的自我进化

### Harness消融实验（RQ3）
- **消融组件**：观察结构（H1）、合法动作masking（H2）、上下文管理（H3）
- **配置**：Full Harness、三种单组件消融、最小Harness（移除所有三项）
- **设置**：固定骨干模型，镜像对战锦标赛
- **目标**：量化harness设计对游戏性能的影响

### 跨牌组泛化实验
- 使用5种不同牌组进行镜像对战，验证骨干模型差异是否跨战略设置保持一致

## 6. 论文主要结论与发现

### 主要发现

1. **PTCG-Bench产生广泛且良好解析的评分分布**
   - LLM Agent评分从GPT-5.4 Nano的1237到Gemini 3.1 Pro的1854，差距达617 Glicko-2点
   - 617点差距对应约97.2%的期望胜率
   - 评分差距显著超过大多数个体Agent的评分偏差，表明benchmark能够有效区分智能体

2. **前沿变体持续优于同家族成本效益变体**
   - 表明PTCG-Bench捕捉的是有意义的能力差异，而非锦标赛噪声

3. **游戏强度不由推理成本决定**
   - 成本-评分分布呈非单调分布
   - 更高成本不一定带来更强表现
   - 揭示了实际成本-性能权衡

4. **PTCG-Bench捕捉了通用基准未充分反映的能力**
   - 与LiveBench（ρ=0.31）、SWE-Bench Pro（ρ=-0.14）、GPQA（ρ=0.35）的排序相关性较弱
   - 表明PTCG-Bench评估的是静态语言理解之外的能力，特别是不完全信息推理、长期规划和不确定性下的战略决策

5. **现有自我进化方法在PTCG中难以实现一致改进**
   - 5种方法均未展现出跨越8轮的单调提升
   - 评分波动无稳定上升趋势
   - 无进化配置能可靠地超越非进化同骨干参考
   - 揭示了当前自我进化机制的不足：难以从长期、不完全信息的博弈中提取可重用战略知识

6. **Harness设计对测量性能有实质性影响**
   - 移除结构化观察：评分下降33点（0.6倍相邻骨干差距）
   - 移除合法动作masking：评分下降118点（2.1倍），无效动作率从3.3%升至15.9%
   - 移除历史上下文：评分下降115点（2.1倍），工具调用增加近5倍（68.4→385.7）
   - 最小Harness：评分下降151点（2.7倍），无效动作率27.3%
   - 这些下降超过典型相邻骨干差距（55点），接近家族内模型层级差距

## 7. 优点

1. **统一的基准架构**
   - 同时评估战略博弈能力、自我进化能力和harness设计影响
   - 填补了现有benchmark的空白

2. **高质量的游戏环境**
   - 自定义PTCG引擎确保规则一致性
   - 5种战略原型牌组覆盖多样游戏风格
   - 保留不完全信息特性

3. **模块化、可消融的Harness设计**
   - 独立控制观察结构、动作masking和上下文管理
   - 允许受控分析harness选择如何影响游戏表现
   - 避免将harness效果与骨干能力混淆

4. **纵向评估协议**
   - 通过锚定比赛实现跨轮的稳定评分追踪
   - 有效分离局内决策与跨游戏经验更新
   - 为自我进化评估提供可控设置

5. **稳健的评估指标**
   - Glicko-2评分系统同时提供均值和偏差
   - 头对头胜率补充评分系统，暴露匹配级优势
   - 无效动作率和工具调用数量提供额外洞察

6. **广泛的模型覆盖**
   - 10个backbone来自5个模型家族
   - 允许跨家族和跨层级比较
