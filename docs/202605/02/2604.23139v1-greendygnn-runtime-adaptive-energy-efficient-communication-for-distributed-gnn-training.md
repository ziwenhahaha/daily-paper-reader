## 摘要
分布式GNN训练主要由远程特征获取主导，这可能会带来非常高的开销。多跳邻域采样会跨越分区边界，并触发细粒度的RPC，其固定的启动开销和GPU停滞延迟会造成能源浪费。现有系统试图通过预采样和静态缓存来减少这种开销，但缓存策略无法对运行时的网络变化做出反应。我们表明，在时变网络拥塞情况下，静态缓存可能会增加高达45%的能耗，因为固定的重建调度是不够的。我们提出了GreenDyGNN，它将缓存窗口管理构建为一个序列决策问题。GreenDyGNN执行epoch内缓存重建，并使用一个Double-DQN智能体（该智能体在带有域随机化拥塞的校准模拟器中进行训练），在每个边界处自适应地调整重建窗口大小和每个所有者的缓存分配。一个异步的双缓冲流水线使得这种自适应几乎没有额外开销。在拥塞情况下，GreenDyGNN与默认DGL相比最多可减少43%的总能耗，与最佳静态策略相比可减少4-24%的总能耗，同时在网络状况良好时能取得接近最优的性能。

## Abstract
Distributed GNN training is dominated by remote feature fetching, which can be very costly. Multi-hop neighborhood sampling crosses partition boundaries and triggers fine-grained RPCs whose fixed initiation cost and GPU-stall latency waste energy. Prior systems try to reduce this overhead with presampling and static caching, but cache policies cannot react to runtime network variation. We show that under time-varying congestion, static caching can increase energy by up to 45% because a fixed rebuild schedule is insufficient. We present GreenDyGNN, which formulates cache window management as a sequential decision problem. GreenDyGNN performs intra-epoch cache rebuilds and uses a Double-DQN agent, trained in a calibrated simulator with domain-randomized congestion, to adapt rebuild window size and per-owner cache allocation at each boundary. An asynchronous double-buffered pipeline makes adaptation effectively free. Under congestion, GreenDyGNN cuts total energy by up to 43% over Default DGL and 4-24% over the best static policy, while closely matching the optimum under clean conditions.

#### 1. 核心问题与整体含义（研究动机和背景）

*   **现有痛点**：现有的系统（如 RapidGNN）通常采用静态缓存策略（如每个 Epoch 重建一次缓存）来减少通信。然而，在共享集群中，网络拥塞往往是动态变化的。论文指出，面对时变的网络拥塞，固定的缓存重建策略无法做出反应，甚至可能导致能耗增加高达 45%。
*   **核心问题**：如何设计一种能够感知运行时网络波动，并动态调整缓存策略的机制，以最小化分布式 GNN 训练的总能耗（包含计算与通信能耗）。

#### 2. 提出的方法论：核心思想与技术细节
*   **核心思想**：提出 **GreenDyGNN** 框架，将分布式 GNN 训练中的缓存窗口管理建模为一个**序列决策问题（MDP）**。系统不再使用静态缓存，而是引入 Epoch 内的细粒度窗口缓存重建机制，并利用强化学习（RL）智能体根据实时网络状况动态调整重建窗口大小和缓存分配策略。
*   **关键技术细节**：
    *   **异步双缓冲流水线**：将采样、缓存重建、特征解析和 GPU 训练解耦为并发阶段。通过 Active 和 Pending 两个缓冲区，确保 RL 智能体在任何频率下进行缓存重建都不会阻塞 GPU 训练，实现零额外开销。
    *   **校准模拟器与 Sim-to-Real 迁移**：为了避免在真实集群上低效地训练 RL，系统通过一次性的性能分析（约 15 分钟）收集数据，构建轻量级的能耗与延迟数学模型。随后在模拟器中引入**域随机化**（注入各种随机拥塞模式），训练 RL 智能体，最后无缝部署到真实集群中。
    *   **基于 Double-DQN 的 MDP 构建**：
        *   **状态空间**：包含每个所有者的拥塞乘数、缓存命中率、系统负载比率以及上一步的动作（共 23 维特征）。

#### 7. 优点
*   **问题切入点新颖**：首次将“运行时网络波动”作为分布式 GNN 训练中节能的一等公民问题来对待，打破了传统的静态缓存假设。
*   **工程与算法的完美结合**：通过异步双缓冲流水线巧妙地掩盖了策略决策和缓存重建的开销，使得复杂的 RL 策略能够真正零开销地应用于高频的系统底层控制。
*   **Sim-to-Real 的系统化应用**：利用数学建模+域随机化训练 RL，有效解决了在真实分布式集群上直接训练 RL 成本过高、样本效率低下的问题。

#### 8. 不足与局限
*   **硬件环境相对陈旧**：实验使用的 GPU 为 NVIDIA P100，属于较老的架构。现代数据中心更多使用 A100/H100 等，这些 GPU 的空闲功耗特征和 RDMA 网络特性可能有所不同，结论在最新硬件上的绝对收益幅度可能发生变化。

*   **模型架构单一**：实验主要基于 2 层的 GraphSAGE，对于更深层次、通信模式更复杂的 GNN 架构（如基于 Attention 的机制）未做探讨。
