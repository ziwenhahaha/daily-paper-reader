## 摘要
强化学习（RL）通过奖励信号直接优化模型行为来微调大型语言模型（LLM）。虽然在经典强化学习中，精确的状态价值估计对稳定训练至关重要，但在 LLM 后训练阶段仍是一项未被充分探索的挑战。在本工作中，我们提出了状态价值估计基准（SVEB），用于评估现有 RL 框架中的状态估计，并表明在标准方法（如 PPO）中的评论器会退化为粗糙的群体平均基线。为解决此问题，我们提出了两种技术：Numca，利用数值跨度作为可评分的里程碑进行状态价值估计；以及 Hista，一种利用 LLM 隐藏状态作为表示，对不相交的 rollouts 及其回报进行加权平均的框架。大规模实验表明，两种方法均能产生更精确的状态价值估计，并在不同的 RL 算法和模型规模下提升训练性能，且未产生显著的计算开销。

## Abstract
Reinforcement learning (RL) refines large language models (LLMs) by directly optimizing model behavior through reward signals. While accurate state value estimation is critical for stable training in classical RL, it remains an underexplored challenge in LLM post-training. In this work, we introduce the State Value Estimation Benchmark (SVEB) to assess state estimation within existing RL frameworks and show that critics in standard approaches like PPO collapse to a coarse group-average baseline. To address this, we propose two techniques: Numca, which leverages numerical spans as gradable milestones for state value estimation, and Hista, a framework that uses LLM's hidden states as representation to weighted average disjoint rollouts and their return. Extensive experiments demonstrate that both methods yield more accurate state value estimates and enhance training performance across different RL algorithms and model sizes without incurring significant computational overhead.

- **研究空白**：在 LLM post-training 阶段，如何有效估计细粒度状态价值是一个未被充分探索的挑战。
- **核心问题陈述**：
  > How does accurate state value improve LLM RL training, and how can it be estimated effectively?

## 2. 论文提出的方法论

### 2.1 SVEB（State Value Estimation Benchmark）

- **目的**：量化不同方法估计的状态价值与真实状态价值之间的误差。
- **构建流程**：
  1. 使用固定策略 LLM 采样 rollouts，从每个 rollout 的中间状态中均匀采样得到状态集合 D_s。
  2. 对每个状态 s_t，利用蒙特卡洛采样（MCS@n）对同一策略采样 n 条独立 continuation，以终端奖励均值作为参考值：bV(s_t) = (1/n) Σ r(s_T^{(i)})。
  3. 评估指标为 MAE（Mean Absolute Error）：MAE(f, D_s) = (1/|D_s|) Σ |f(s_t, θ) − bV(s_t)|。

### 2.2 Numca（Numerical Milestone Credit Assignment）

- **核心思想**：在数学推理任务中，将数值（整数、小数、分数）作为自然可解析的里程碑，将文本生成重新建模为稀疏奖励的目标到达问题。
- **关键技术**：
  1. 定义数值模式集合 P（如整数、小数、分数的正则匹配）。
  2. 将状态抽象为里程碑集合：s_M ≜ M(s_t)，即 s_t 中出现的所有里程碑。
  3. 宏观动作 a_M 定义为两个相邻抽象状态之间生成的 token 序列。
  4. 抽象状态的价值通过包含该里程碑的所有 rollouts 的最终奖励均值估计。
- **算法**：基于字典实现，遍历 rollouts 检测里程碑变化，更新 T[s_M] = (count, reward_sum)，最终 V(s_M) = reward_sum / count。计算开销可忽略。

### 2.3 Hista（Hidden State based State Value Estimation）

- **核心思想**：利用 LLM 的隐藏状态作为通用的状态表示，通过概率加权估计法估算状态价值，无需额外 rollouts 或 Critic 训练。
- **关键技术——MinDistance（最小距离）**：对两条长度为 η_1 和 η_2 的隐藏状态序列 X_1 和 X_2，定义 MD(X_1, X_2) 为逐 token 求 L2 距离并取最小匹配的累加和。
- **核心定理**：
  - **Theorem 5.2（隐藏状态与奖励的相关性）**：两个状态获得相同终端奖励的概率与它们隐藏状态之间的 MinDistance 成反比：P(R_1 = R_2) ∝ 1 / MD(X_1^l, X_2^l)。
  - **Theorem 5.5（概率加权估计的优越性）**：在满足特定假设（B.2）的条件下，概率加权估计器的绝对偏置不超过朴素均值估计器。
- **算法流程**（四阶段流水线）：
  1. **隐藏状态提取**：从 LLM 最后一层提取终端状态的隐藏状态序列 X_τ。
  2. **池化与采样**：使用 EMA（平滑因子 α）和压缩间隔 φ 对隐藏状态序列降维，得到 E_τ；再以间隔 δ 均匀采样定义有限状态空间 S_H^τ。
  3. **距离计算**：对给定状态 s_t，用 MinDistance 计算其与 S_H 中所有其他状态的距离。
  4. **价值估计**：取 k 个最近邻，以距离倒数加权平均终端奖励：V(s_t) = (Σ ω_i · r_i) / (Σ ω_i)，其中 ω_i = 1 / MD(s_t, s_i)。
- **优势**：效率与 GRPO 相当（额外延迟 < 1 秒/训练步），空间复杂度 O(⌊T/d⌋²)。

## 6. 论文的主要结论与发现

1. **PPO Critic 退化问题**：标准 PPO 在 LLM 设置下的 Critic 网络价值预测始终退化为群体平均奖励基线，在 SVEB 上与 GRPO 无显著差异，无法提供细粒度信用分配。

2. **Numca 的有效性**：在数学推理领域（SVEB-Number），Numca 显著优于 GRPO 和 PPO，MAE 从 0.164 降至 0.136；但缺乏跨领域泛化能力，在非数学领域退化为 GRPO 水平。

3. **Hista 的通用优越性**：Hista 在所有 5 个 SVEB 领域中均优于 GRPO 和 PPO，MAE 全面降低；在下游 RL 训练中，Hista 与 DAPO/CSIPO 结合后在多数 benchmark 上取得稳定提升（如 MATH-500、GSM8K、OlympiadBench、AMC 等），跨越 0.5B~14B 多种模型规模。

4. **效率保证**：Hista 的计算开销与 GRPO 相当（额外延迟 < 1 秒/步），无需训练额外 Critic 网络或额外 rollouts。

5. **理论支撑**：Theorem 5.2 建立了隐藏状态 MinDistance 与相同终端奖励概率之间的相关性；Theorem 5.5 证明了概率加权估计器的理论优越性。

## 7. 优点

- **问题定位精准**：首次系统地揭示了 LLM RL 中 Critic 退化为群体平均基线的核心问题，并通过 SVEB 基准将其量化。
- **解决方案层次化**：提出两种互补方案——领域特定的 Numca（验证数值里程碑思路）和通用化的 Hista（理论基础扎实），覆盖了从启发式到理论驱动的方法论演进。
- **理论扎实**：通过详细的 Transformer 层操作 Lipschitz 分析（Lemma A.3~A.9），建立了隐藏状态距离与奖励一致性的理论联系，Theorem 5.5 给出了概率加权估计优于朴素均值估计的严格证明。
- **实验全面且公平**：覆盖 5 个推理领域、3 种训练数据集、8+ 类下游 benchmark、多种模型规模（0.5B~14B），且包含完整的超参消融研究。
- **效率友好**：Hista 在不引入额外模型参数和显著计算开销的前提下实现精确状态价值估计，工程可行性高。
- **Benchmark 贡献**：SVEB 作为首个专门针对 LLM 状态价值估计的基准，可为后续研究提供统一的评估框架。

## 8. 不足与局限

- **Numca 的领域依赖性**：Numca 严格依赖数值模式作为里程碑，在科学、通用 QA、编程等领域完全失效，仅适用于数学推理。
- **Theorem 5.2 假设限制**：理论分析依赖 Answer Parsing 假设（最终答案从单一 token 解析），在开放式编程等任务中不适用。虽然实验证明 Hista 在编程任务上仍有效，但理论框架尚未覆盖此类场景。
- **个别配置表现下滑**：在 Qwen3-1.7B 模型上，Hista 在部分 benchmark 上未超越 DAPO 基线，作者将其归因于 RL 的随机性和分布外评测的不稳定性，表明在中等规模模型上的表现可能不够稳健。
- **参数敏感性**：Hista 对 EMA 参数 α、压缩间隔 φ、采样间隔 δ 和最近邻数 k 的组合存在一定敏感性，需要精细调参（论文建议 φ=5, α=0.7, δ=50, k=66）。
- **长序列开销**：MinDistance 计算的空间复杂度为 O(⌊T/d⌋²)，在超长序列（> 100k tokens）场景下可能面临内存压力，论文虽通过 EMA 缓解但未系统评估极端长度。
- **实验覆盖缺口**：GSPO+Hista 仅在 1.5B 模型上测试，未探索更大规模，对 GSPO 与 Hista 的组合泛化性了解有限。
