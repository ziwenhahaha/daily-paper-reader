## Abstract
Moving beyond simple scalar rewards toward richer world feedback is a natural path to more scalable RL post-training. On-policy self-distillation (OPSD) is a promising recent approach that uses arbitrary feedback as learning signal, yet its reliability compared to established methods, such as GRPO, remains unclear. We identify a strikingly consistent linear correlation between the initial student-self-teacher performance gap and the final performance improvement in OPSD. This relationship holds across context types and model families, providing a powerful predictive law for anticipating the outcome of an OPSD configuration without running the full training procedure. Interestingly, we show that this linear predictability holds with model scale, suggesting a potential basis for new empirical scaling laws on larger models with stronger in-context learning capabilities. In essence, our findings show that OPSD performance can be predicted and tuned before training, offering a principled way to incorporate world feedback as a first-class component of the post-training pipeline.

**研究动机与背景：**
- **强化学习后训练的局限**：基于可验证奖励的强化学习（RLVR）已成为大语言模型（LLM）训练的核心组件，但其奖励信号通常局限于二元结果（正确/错误），难以利用更丰富的反馈信息（如代码执行错误、单元测试反馈、环境交互信息等）。
- **OPSD 的潜力与挑战**：同策略自蒸馏（On-Policy Self-Distillation，OPSD）是一种新兴方法，通过特权上下文（privileged context）将任意反馈信号作为学习信号，无需外部奖励模型。然而，该方法的可靠性尚不明确，训练过程中的稳定性问题（如上下文设计敏感、教师分布偏移）限制了其广泛应用。
- **核心问题**：能否在无需完整训练的情况下，提前预测 OPSD 的最终性能表现，从而筛选最优的特权上下文配置并避免浪费计算资源？

**核心发现：**
- 论文发现了一个**显著的线性预测规律**：学生模型与自教师模型在训练前的初始性能差距（student-self-teacher gap）与训练后的最终性能提升（final performance improvement）之间存在近乎完美的线性相关（R² 高达 0.949–0.996）。

## 2. 论文提出的方法论

**核心思想：**
- 利用学生模型自身的特权版本（自教师）作为监督信号，通过同策略蒸馏将特权上下文中的信息内化到学生模型中。
- 关键创新点在于：**无需运行完整训练，即可通过测量初始性能差距来预测最终性能提升**，从而实现 OPSD 配置的快速筛选和可靠性保障。

**关键技术细节：**

- **学生-自教师架构**：
  - 学生策略 πθ 通过提示 x 采样轨迹 y ∼ πθ(·|x)。
  - 自教师策略 π¯θ 与学生共享权重参数，通过指数移动平均（EMA）缓慢跟踪学生参数 θ 的变化，并额外接受特权上下文 c 的条件输入。
  - 特权上下文 c 包含学生模型无法直接获取的信息，例如环境反馈、参考解决方案等，通过上下文学习（in-context learning）引导自教师产生更强行为的预测分布。

- **OPSD 优化目标**（公式 1）：
  $$L_{\text{OPSD}}(\theta; c) = \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(\cdot|x)} \left[ \frac{1}{|y|} \sum_{t} \text{KL}\left( \pi_\theta(\cdot|y_{<t}, x), \text{stopgrad}\left[ \bar{\pi}_\theta(\cdot|y_{<t}, x, c) \right] \right) \right]$$
  - 使用**反向 KL 散度**（reverse KL divergence）作为蒸馏损失，仅在学生采样的轨迹上进行优化，避免了暴露偏差问题。
  - `stopgrad` 操作阻止梯度回传到自教师，确保自教师不因学生更新而改变。

- **六种特权上下文配置**（表 3）：
  1. **None（对照）**：无特权信息。
  2. **Expert Preamble**：包含数据结构与算法清单、复杂度表、模式规则、边界情况、9 步方法论等专家级提示。
  3. **Feedback**：提供环境反馈（如运行时错误）。
  4. **LLM Peer Hints**：直接给出正确解决方案。
  5. **Own Solution + Feedback**：根据当前轨迹是否正确，动态选择提供自己的正确解决方案或环境反馈。
  6. **Peer Solution + Feedback**：同时提供同龄人（peer）的正确解决方案和环境反馈。

## 3. 实验设计

**数据集与基准：**
- **基准测试**：LiveCodeBench v6（2025 年 2 月至 5 月的编程问题），使用 mean@4（每题采样 4 次的通过率）作为评估指标。
- **任务场景**：代码生成与修复任务，作为富含环境反馈的典型场景。

**模型家族：**
- **Qwen3 系列**：0.6B、1.7B、4B、8B 四种规模。
- **Olmo-3-7B-Instruct**：单独进行实验验证。
- 所有模型均禁用“思考模式”（thinking mode），以确保评估的一致性。

**实验内容：**

- **实验 1（模型家族泛化性）**：
  - 对 Qwen3-8B 和 Olmo-3-7B-Instruct 分别使用六种特权上下文配置进行 50 步 OPSD 训练。
  - 测量每种配置下的初始学生-自教师性能差距与最终性能提升。
  - 结果：两种模型均呈现近乎完美的线性关系（Qwen3-8B 的 R² = 0.949，Pearson 相关系数 r = 0.974；Olmo-3-7B-Instruct 的 R² = 0.996，r = 0.998）。

- **实验 2（尺度不变性）**：
  - 对 Qwen3 家族的四种规模模型，使用固定的特权上下文（Peer Solution + Feedback）进行训练。
  - 验证线性预测规律是否跨尺度成立。
  - 结果：四种规模均落在同一条线性拟合线上（R² = 0.977，r = 0.988），表明该规律具有尺度不变性。

- **训练动态分析**：
  - 监控训练过程中学生模型性能的收敛行为，发现学生性能随训练步数稳步提升并逐渐趋近自教师水平，同时自教师能力也在 EMA 更新的驱动下同步提升。

- **留一交叉验证**：
  - 对实验 1 进行留一交叉验证，评估线性预测模型的泛化能力。
  - 结果：Qwen3-8B 的 RMSE = 0.016，Olmo-3-7B-Instruct 的 RMSE = 0.003，表明预测精度高且稳定。

**超参数配置**（表 2）：
- 优化器：AdamW（学习率 1×10⁻⁶，权重衰减 0.01，β₁=0.9，β₂=0.999）。
- 梯度裁剪：最大范数 1.0。

- 验证采样：每提示 4 条轨迹，温度 0.6，top-p = 0.95。
