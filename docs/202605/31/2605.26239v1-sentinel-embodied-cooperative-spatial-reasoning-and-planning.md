## Abstract
In this work, we study Cooperative Spatial Intelligence, the ability of decentralized embodied agents to coordinate effectively under dynamic environmental constraints across city-scale outdoor domains. We introduce Sentinel Challenge, a benchmark where multiple decentralized embodied agents must communicate in natural language to agree on a mutually safe and convenient meeting point within large, city-scale outdoor environments. Each agent must then navigate safely while avoiding dynamic sentinels patrolling the area, using a tool that provides coarse spatial information. To address this, we propose CoSaR (Cooperative Spatial Reasoning and Planning), a framework that bridges the high-level communication and planning abilities of foundation models with the precision of classical spatial navigation algorithms. CoSaR enables agents to exchange situational updates, reason over evolving spatial constraints, and collaboratively replan trajectories. Evaluated across 14 city-level scenes with 3-5 agents, CoSaR consistently leads to faster gathering, shorter path lengths, and improved safety. Our results demonstrate that integrating dynamic communication with spatial reasoning is essential for robust multi-agent cooperation. By formalizing this new setting and providing a scalable benchmark, we aim to build a foundation for advancing cooperative spatial intelligence in embodied multi-agent systems. Code and challenge are available at https://github.com/UMass-Embodied-AGI/Sentinel.

## 2. 论文提出的方法论

### 核心思想
提出 **CoSaR (Cooperative Spatial Reasoning and Planning)** 框架，桥接基础模型的高级通信与规划能力与经典空间导航算法的精度。

### 关键技术细节

#### (1) Map Tool（地图工具）
- 提供粗粒度空间信息，包括：
  - `QueryRoute`: 返回路径航点
  - `QueryNearby`: 返回附近地点列表
  - `QueryPlace`: 返回地点元数据
  - `QueryMap`: 返回航空视角的二值地图
  - `QueryRefinedRoute`: 细化航点连接
- 基于 Dijkstra 算法实现路径查询

#### (2) 感知模块 (Perception Module)
- 使用 Grounding DINO 进行开放集物体检测
- 使用 SAM (Segment Anything) 预测分割掩码
- 计算 CLIP 特征进行语义匹配
- 构建体素网格并标记哨兵危险区域

#### (3) 通信模块 (Communication Module)
- 分析接收消息，提取：
  - 空间信息（其他智能体位置、哨兵位置）
  - ETA（预计到达时间）信息
  - 合作信息（队友意见）
- 生成候选通信消息，协调决策

#### (4) 空间记忆 (Spatial Memory)
核心数据结构，包含：
- **占用地图与危险区域**：记录地面可达性、智能体/哨兵位置
- **ETA 地图**：记录每个智能体到各候选地点的预计时间
- **集合计划**：选定的集合地点和参考路线

#### (5) 空间感知推理模块 (Spatial-aware Reasoning Module)
- 利用 LLM 零样本推理能力
- 决定下一步行动（Query / Communicate / Navigate / Wait）
- 触发条件：新事件观测、意见分歧、定期间隔（每120秒）

#### (6) 导航模块与路线细化 (Route Refinement)
当检测到路线与危险区域相交时：
1. 调用 `QueryMap` 获取粗粒度航空地图
2. 用 VLM 在标注地图上进行零样本空间推理
3. 调用 `QueryRefinedRoute` 验证并细化路线
4. 使用 A* 算法进行局部路径规划

#### (7) 紧急避让机制
- 触发条件：收到哨兵警告信号或视觉检测到哨兵
- 计算避让分数选择远离哨兵的方向
- 执行10步避让后确认安全

## 3. 实验设计

### 数据集与场景
- **平台**：Virtual Community 虚拟社区
- **场景数量**：14个城市规模场景（800m × 800m）
- **地点数量**：每场景50-150个预定义地点
- **智能体数量**：3-5个
- **哨兵数量**：5-20个（固定或巡逻模式）

### Benchmark 指标
| 指标 | 说明 |
|------|------|
| Success Rate | 1500步内所有智能体成功集合的比例 |
| Caught Rate | 被哨兵捕获的智能体比例 |
| Detection Rate | 任一智能体被检测的时间步比例 |
| Time Cost | 所有智能体集合所需步数 |
| Distance Traveled | 所有智能体总行进距离 |

### 对比方法
1. **Oracle Centered**：选择所有智能体几何中心的集合点
2. **Oracle Centered w/ DZ**：增加危险区域避让
3. **MCTS Planner**：使用蒙特卡洛树搜索选择集合点
4. **RoCo**：先规划后执行的协作方法
5. **CoELA**：基于基础模型、支持动态通信的模块化框架
6. **MAT**：多智能体 Transformer（集中式端到端强化学习）

### 消融实验
验证以下组件的贡献：
- Route Refinement（路线细化）
- Message Analyzer（消息分析器）
- Spatial Memory（空间记忆）
- Emergency Avoidance（紧急避让）

## 6. 论文的主要结论与发现

### 核心发现
1. **CoSaR 整体最优**：在所有配置下均取得最高成功率（最高达67.86%），同时具有最低的时间成本和距离成本

2. **空间记忆的重要性**：
   - 禁用空间记忆后，成功率从32.14%下降至21.42%
   - 禁用消息分析器导致最大跌幅（19.26%）
   - 证明协调决策和ETA信息共享至关重要

3. **动态通信的价值**：
   - CoELA（支持动态通信）优于静态规划的 RoCo
   - CoSaR 结合动态通信与空间感知，优于仅支持动态通信的 CoELA

4. **效率-安全权衡**：
   - CoELA 有时达到更低捕获率，但代价是花费更多时间在初始位置躲避
   - CoSaR 在安全性和效率间取得更好平衡

5. **规模化鲁棒性**：
   - 增加智能体或哨兵数量使任务更难
   - CoSaR 的成功率下降幅度小于其他方法，表现更鲁棒

### VLM 路线细化局限
- 在多哨兵场景中成功率仅约20%
- VLM 在全局空间理解和长距离推理上能力有限

## 8. 不足与局限

### 实验层面
1. **仅模拟环境**：所有实验在 Virtual Community 中进行，与真实城市环境可能存在差距

2. **VLM 路线细化成功率低**：约18-27%的成功限制了该模块的实际效用

3. **依赖强大基础模型**：需要 GPT-4o 等闭源大模型，成本较高

### 方法层面
1. **哨兵检测依赖视觉**：若感知模型失效，整个系统崩溃

2. **通信带宽假设**：全局通信通道，无消息丢失或延迟建模

3. **室内安全性假设**：智能体初始位置在室内，规避了初始安全问题

### 应用层面
1. **收敛时间问题**：多智能体协商可能耗时较长，不适合时间敏感场景

2. **对时钟同步的假设**：ETA 计算依赖统一时间基准

- 论文讨论了技术可能用于 surveillance 场景的双重用途风险
