## Abstract
Automating scientific computing workflows requires more than generating executable code: autonomous systems must also select appropriate computational strategies, implement them faithfully, and ensure that the resulting outcomes remain causally attributable to the decisions that produced them. In multi-agent pipelines, this process is particularly fragile, as small inconsistencies between agent intentions and actions can lead to semantic drift, where the eventually executed procedure no longer reflects the originally selected strategy, thereby corrupting downstream evaluation and adaptation. In this work, motivated by the ATHENA framework (Toscano et al., 2025; Toscano et al., 2026) and the concept of empowerment (Yiu et al., 2025), we introduce a multi-agent framework that combines contextual bandits with structured inter-agent communication and, most importantly, semantic checkpoints that preserve action-outcome fidelity throughout the pipeline. The system integrates specialized large language model (LLM) agents, grounded code generation, and self-healing execution loops within an adaptive decision-making architecture. Interpreting the framework through the lens of empowerment, we show that reliable autonomous learning requires not only identifying high-quality actions, but also preserving the integrity of their propagation across agents. Using sensitivity analysis and uncertainty quantification workflows as representative case studies, we demonstrate that unchecked semantic drift degrades policy learning, whereas the proposed framework improves convergence, robustness, and adaptation to novel problem contexts. These results suggest a broader design principle for scientific multi-agent systems: adaptive decision-making must be coupled with explicit mechanisms that guarantee semantic consistency and reliable information flow across the computational pipeline.

## 1. 论文核心问题与整体含义

### 研究动机与背景

- **核心挑战**：自动化科学计算工作流不仅需要生成可执行代码，更需要自主选择合适的计算策略、实施这些策略，并确保结果能够可靠地归因于产生它们的决策。
- **语义漂移问题**：在多智能体管道中，智能体之间的微小不一致会导致语义漂移（semantic drift），即最终执行的程序不再反映最初选择的策略，从而破坏下游的评估与自适应机制。
- **研究缺口**：现有的多智能体系统缺乏对行动-结果因果链完整性的保障机制，导致即使策略选择了正确的方法，奖励信号也可能因实现偏差而被错误归因。

### 论文意义

论文将方法选择问题形式化为上下文多臂老虎机（contextual bandit）问题，并从赋权（empowerment）理论视角揭示：可靠的自主学习需要同时满足两个条件——（1）识别高质量行动；（2）保证行动在智能体间传播的完整性。

## 2. 论文方法论

### 核心思想

提出一种基于赋权最大化的多智能体框架，核心在于**分离统计组件与通信组件**：

- **统计组件**：通过上下文老虎机学习哪些方法配置能产生高质量结果。
- **通信组件**：通过语义检查点和结构化通信确保所选行动忠实传播。

### 系统赋权定义

系统赋权定义为互信息：

$$\text{Empow}_S(s) = I(A; O | s)$$

其中 $A$ 是方法选择行动，$O$ 是计算结果。该量测度方法选择对计算结果的决定程度。

### 关键技术细节

#### （1）结构化 Scheme 通信

- **ProblemScheme**：编码问题结构特征（维度、预算、任务类型等），在会话开始时生成并保持不变。
- **MethodScheme**：附属于每次策略决策，包含完整的方法规范（估计器族、输出模式、最小预算等）。
- **DiagnosticScheme**：评估代理在每次迭代结束时产生，包含受控词汇的根因分析和处方建议。

#### （2）七语义检查点机制（CP0-CP7）

| 检查点 | 比较对象 | 类型 | 功能 |
|--------|----------|------|------|
| CP0 | 当前问题 vs 存档 | 非阻塞 | 跨会话知识转移与异常检测 |
| CP1 | 用户消息 vs 上下文 | 阻塞 | 验证问题解析正确性 |
| CP2 | 行动 vs 上下文 | 阻塞 | 验证方法上下文兼容性 |
| CP3 | 模板 vs 策略 | 非阻塞 | 无匹配模板时警告 |
| CP4 | 单元图 vs 策略 | 阻塞 | 验证实现计划 |

检查点通过余弦相似度量化语义对齐，使用 sentence-transformer all-MiniLM-L6-v2 模型计算嵌入向量。

#### （3）多智能体架构

系统分为四个功能组：

- **概念化团队**：协调器、门卫、模型翻译器——运行一次，生成结构化问题表示。
- **策略团队**：策略家、批评者、顾问——作为策略操作器 $\pi$，基于上下文和历史选择方法。
- **实施团队**：学习代理、重构代理、检查器——将抽象行动翻译为可执行代码。
- **执行与评估团队**：执行运行器、调试器、顾问——运行代码并产生奖励。

#### （4）自适应阈值

阈值采用运行估计更新：

- 初始使用手工调优先验值（表4）
- 当系统一致产生相干转换时收紧
- 问题较难匹配时放松
- 设置自适应下界防止阈值低于空标定水平

## 3. 实验设计

### 测试场景

论文选择**灵敏度分析（SA）和不确定性量化（UQ）**作为代表性任务，使用 UQpy 库实现。

### 基准测试用例

#### （1）Checkpoint 消融实验

- **场景**：对抗性提示，明确要求使用无矩估计器（如 Chatterjee），与语料库主导的 Sobol 模板冲突。
- **目标**：验证语义漂移是否导致行动-代码不匹配。

#### （2）对手鲁棒性实验

- **场景**：Sobol G 函数（d=8），引入方法交换漂移——在策略报告生成后、将代码送达重构代理前，将 SobolSensitivity 替换为 MorrisSensitivity。
- **目标**：评估 CP5 检测语义错配的能力。

#### （3）高维筛选实验

- **场景**：15 维 Sobol G 函数，$a = [0, 0.5, 1, 2, ..., 100]$，预算 N=50,000。
- **目标**：评估系统在 $d_{in} \geq 10$ 条件下的筛选能力。

#### （4）异常检测实验

- **熟悉问题**：悬臂梁灵敏度（$d_{in}=4$，正态输入，N=20,000），存档中有 2 个已完成会话。
- **异常问题**：非线性热扩散模型（$d_{in}=20$，均匀输入，N=100,000），存档中无匹配项。

#### （5）置换鲁棒性实验

- **设计**：固定每会话首轮迭代方法（Sobol、Chatterjee、Morris 等），观察策略演变和最终分布。
- **目标**：量化路径依赖程度。

### 对比方法

- **No-Checkpoint 基线**：所有检查点阈值设为 -1，全部无条件通过。
- **Full Pipeline**：完整语义检查点系统。
- **Ablated 条件**：单独禁用特定检查点（如 CP5）以评估其贡献。

## 6. 主要结论与发现

### 语义漂移的危害

- 无检查点条件下，2/3 的运行产生行动-代码不匹配，奖励被错误归因于错误的方法臂，直接腐蚀策略学习。
- 奖励 $R_n=82$ 被记入 Sobol 臂，但实际测量的是 Chatterjee 方法质量。

### 检查点的有效性

- Checkpoint 有效保护因果链 $A_n \rightarrow S_n \rightarrow O_n \rightarrow R_n$。
- CP2 检测到 Sobol 提议与无矩约束不一致，触发重试。
- CP5 在对手实验中防止漂移达成就绪，在 ablated 条件下系统需要额外 3 次迭代恢复（最终通过 CVM 估计器收敛至 $R=93.61$，而完整系统单次迭代达到 $R=95.17$）。

### 跨会话知识积累

- 三个独立会话在 15 维 Sobol 函数上，最佳奖励单调递增：Session 1 ($R=64.2$) → Session 2 ($R=68.8$) → Session 3 ($R=75.5$)，累计增益 11.3 分。
- 策略从历史观察到 Sobol + 大预算一致优于半预算版本，在后续会话中提前选择全预算变体。

### 异常检测功能

- CP0 在熟悉问题上降低探索，直接选择历史最佳方法，首轮达到 $R_1=91.0$。
- CP0 在异常问题上增加探索并激活筛选优先策略，避免将先前学习的启发式方法误用于未验证的制度。

### 代理多样性

- 三个会话共探索五种不同估计器（Sobol、PCESensitivity、ChatterjeeSensitivity、CVMSensitivity、MorrisSensitivity）。
- 无单一估计器在所有迭代中被选择，符合 empowerment 的变异性要求。

## 7. 优点

### 方法设计亮点

1. **理论统一性**：将 empowerment 概念从单智能体扩展到多智能体管道，提供统一的设计原则而非临时工程技巧。

2. **通信结构化**：用 ProblemScheme、MethodScheme、DiagnosticScheme 替代自由文本，消除智能体边界处的歧义。

3. **检查点层次化**：七检查点覆盖转移、选择、实施、评估四个层次，每层针对不同失败模式，非冗余设计。

4. **自适应机制**：阈值自动调整，空标定确保基线安全，适应不同问题难度。

5. **自愈执行循环**：Debugger → Refactor Agent → Execution Runner 的闭环处理运行时错误，无需人工干预。

6. **双时间尺度学习**：历史 $H_n$ 驱动会话内适应；策略 $\pi$ 驱动跨会话转移。

### 实验设计亮点

1. **对手实验**：引入方法交换漂移模拟真实失败模式，直接测试系统检测能力。

2. **消融研究**：通过移除检查点验证各组件贡献，结果清晰展示无检查点时奖励归因错误。

3. **跨会话设计**：展示系统如何随时间积累和改进，而非仅报告单次运行结果。

## 8. 不足与局限

### 实验覆盖局限

- Benchmark 仅涵盖 SA/UQ 领域，未验证方法在其他科学计算任务（如优化、微分方程求解）上的效果。
- 对手实验仅运行单次，统计显著性有限。

### 潜在偏差风险

- 检查点阈值依赖手工调优和经验默认值，可能对特定问题类型过拟合。
- 语料库（UQpy）覆盖的方法有限，新方法引入可能超出系统处理能力。
- 跨会话学习的长期效果未充分评估（存档随时间增长的行为未探讨）。

### 应用限制

- 系统复杂度高，需要协调多个 LLM 代理和检查点机制，实际部署门槛较高。
- 对 LLM 推理能力的依赖意味着如果底层模型性能下降，整个系统的可靠性会受影响。
- 论文未报告推理延迟或实时性指标，实际工作流集成可能面临挑战。

### 扩展性问题

- 语义检查点的相似度计算随问题规模线性增长，大规模问题可能成为瓶颈。
- 存档匹配（CP0）的计算成本随会话数量增加而增加，未探讨优化策略。
