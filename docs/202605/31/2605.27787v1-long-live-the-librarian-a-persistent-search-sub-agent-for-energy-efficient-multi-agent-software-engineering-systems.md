## Abstract
Multi-agent systems (MAS) have substantially advanced autonomous software engineering (SWE), but their growing inference energy demands raise sustainability concerns. In this paper, we demonstrate that this cost is concentrated in an overlooked source: redundant output tokens generated across agents. Two empirical findings ground this claim. First, our per-token energy attribution for MAS reveals a sharp asymmetry: an output token consumes 30 to 1,000 times more energy than an input or cached token. Second, MAS inflate per-episode output because agents repeatedly re-explore overlapping repository regions. To address this inefficiency, we propose Librarian, a persistent search sub-agent that tracks repository-search history and suppresses redundant exploration actions across agents. By returning short references to file regions instead of full file excerpts, Librarian further reduces output-token volume. On SWE-Bench Verified, Librarian reduces per-episode GPU energy consumption of existing multi-agent SWE systems by up to 25% while preserving task performance.

## 1. 论文的核心问题与整体含义

### 研究动机
随着大型语言模型（LLM）的广泛部署，数据中心的电力需求急剧增长，带来碳排放和淡水消耗等环境问题。现有研究多关注单次调用的能耗，而**多智能体系统（MAS）在软件工程中的应用**（如代码修复、功能实现）却缺乏能源分析。

### 核心问题
论文发现了一个反直觉的现象：**MAS 消耗的 GPU 能源甚至超过单智能体系统，尽管前者的总 Token 数量更低**。通过细致分析，发现问题集中在两个层面：

- **输出 Token 能耗不对称**：输出 Token 的能耗是输入或缓存 Token 的 **30 到 1000 倍**（表2）
- **冗余探索**：MAS 中各子智能体重复浏览重叠的代码区域，导致同一文件内容被多次重新输出

- 被测系统：BOAD（分层软工智能体）和 HyperAgent（导航-编辑-执行三角色智能体）

## 2. 论文提出的方法论

### 核心思想
提出 **Librarian**——一个持久化搜索子智能体，专门处理代码查找任务，通过三种设计削减能源浪费：

### 关键技术细节

| 设计原则 | 具体实现 |
|---------|---------|
| **持久会话** | 整个 episode 期间维护单一会话，所有历史搜索可复用 |
| **新鲜度报告** | 每次调用前通过 `git diff` 识别文件修改范围，指示 Librarian 重新遍历变更区域 |
| **指针式回答** | 返回 `[path, start, end]` 而非内联代码；实际代码由工具执行后转发给编排器 |
| **历史剪枝** | 若调用产出的新内容 < 500 字符，则丢弃该调用记录 |

### 工作流程

```
编排器 → Librarian 调用 → Librarian 追加历史 + 新鲜度报告
→ 检索/浏览代码（利用累积历史） → submit(指针列表)
→ 工具执行指针获取代码 → 转发给编排器
```

### 集成方式
- 替换原有代码导航子智能体
- 重写编排器计划中的查找步骤为自然语言查询
- 附加工具选择策略（已知位置直接查看，未知位置调用 Librarian）

## 6. 论文的主要结论与发现

### 关键发现
1. **输出 Token 主导能源**：输出 Token 能耗是其他类别的 30–1000 倍（表2），aggregate token 计数无法预测能源消耗
2. **重复探索是冗余主因**：BOAD 和 HyperAgent 中 10%–35% 的文件读取输出为重复内容（表3）

### Librarian 效果

| 指标 | 结果 |
|-----|------|
| 能源节省 | 最高 **25%**（+ Caveman + Librarian 配置） |
| 任务性能 | 保持或提升 pass rate |
| 角色级节省 | 导航角色输出 Token 减少 50%–85%（表6） |
| 难度鲁棒性 | 在所有难度档位保持效果，最难任务上提升最显著 |

### 与其他方法对比
- **LastNObservation**：虽减少输入 Token，但导致再探索，输出 Token 反增，能源上升、pass rate 下降
- **Caveman 提示**：单独使用可降能源但损性能；与 Librarian 组合达最优效率

| 局限维度 | 具体说明 |
