## Abstract
Cross-Video Reasoning (CVR) has emerged as a critical frontier in multimodal intelligence, requiring models to retrieve, align, and aggregate evidence distributed across multiple videos. Current Multimodal Large Language Models (MLLMs) often struggle with CVR, as simple single-pass strategies encode multiple videos into a shared compressed context, potentially obscuring rare but critical evidence. In this paper, we propose AgentCVR, a multi-agent framework that treats CVR as an active evidence-acquisition task. AgentCVR employs a Master Agent to iteratively coordinate specialized Visual and Audio Agents for targeted evidence extraction. To ensure efficient training, we introduce Script-Simulated RL, which optimizes the agent's policy with LLM-generated semantic scripts and a lightweight text-based simulator, bypassing costly multimodal inference during online exploration. Experimental results on a comprehensive CVR benchmark show that AgentCVR outperforms single-pass baselines and achieves comparable performance to state-of-the-art closed-source systems, particularly in complex cross-video alignment and localization. To ensure reproducibility, our code is available at https://github.com/wang-jh24/AgentCVR.

### 1. 论文核心问题与整体含义（研究动机与背景）

**核心问题：** Cross-Video Reasoning（CVR，跨视频推理）要求模型从多个独立视频中检索、对齐并聚合稀疏分布的证据，以回答需要跨视频比较和推理的复杂查询。当前主流的多模态大语言模型（MLLMs）在这类任务上表现不佳。

**研究背景与动机：**
- 现有视频理解研究多聚焦于单视频分析，无法评估模型的跨视频推理能力。
- 真实场景越来越复杂，处理孤立视频已不够，驱动向 CVR 的转变。
- 现有 CVR 方法通常采用单次通过（single-pass）策略，将所有候选视频编码到共享的压缩上下文中。这种设计将长视频压缩为固定大小的表示，会掩盖稀疏但关键的线索，导致证据归属弱、跨视频比较不可靠，尤其是在时间和比较型查询上表现糟糕。
- **核心观点：** CVR 不仅是上下文长度问题，更是证据获取问题。框架应迭代决定要检查什么证据、查询何种模态、何时收集到足够支持。

### 2. 论文方法论：核心思想、关键技术细节、算法流程

#### 2.1 核心思想

将 CVR 重新定义为**主动多轮证据获取任务**，而非被动单次上下文压缩。核心设计：

- **多智能体协同：** 由一个轻量级的主控智能体（Master Agent）迭代协调专门的视觉智能体（Visual Agent）和音频智能体（Audio Agent），针对目标证据进行提取。
- **证据获取与推理解耦：** 模态特定智能体负责检索局部化的观察结果，主控智能体负责跨视频比较和推理。
- **脚本模拟强化学习（Script-Simulated RL）：** 为解决在线多模态强化学习成本过高的问题，提出用 LLM 生成的语义脚本和轻量级文本模拟器替代真实的视频交互进行策略训练。

#### 2.2 AgentCVR 框架细节

**动作空间分解：** A = A_vis ∪ A_aud ∪ A_ans
- **视觉查询** (`a_vis ∈ A_vis`)：参数化为 `(vid, τ_start, τ_end, P_focus)`，指定目标视频、时间间隔和可选的焦点提示。
- **音频查询** (`a_aud ∈ A_aud`)：参数化为 `(vid, τ_start, τ_end)`，检索语音或声音描述。
- **答案动作** (`a_ans ∈ A_ans`)：终止过程并生成最终答案。

**POMDP 建模：** 将主动多轮证据获取建模为 MLLM-based POMDP：
- **状态：** s_t = (q, H_t)，包含查询和交互历史。
- **动作：** 从参数化策略 π_θ(at|st) 采样，包括调用工具或终止。
- **观察：** 环境返回文本线索。
- **奖励：** 轨迹级奖励 R_total = R_ans + R_fmt（答案正确性 + 格式有效性）。

#### 2.3 Script-Simulated RL 训练范式

**核心流程：**

1. **合成脚本构建（离线）：** 使用强 LLM 基于结构化模式和任务模板生成语义脚本 W_script。每个脚本定义：
   - 一组视频及其元信息。
   - 时间上接地的事件和属性。
   - 回答查询所需的跨视频关系。

2. **在线文本模拟器：** 在 RL 训练期间，用轻量级 LLM 作为模拟器 M_sim 替代真实视频推理：
   ```
   o_t = M_sim(at, W_script(vid, [τ_start, τ_end]))
   ```
   模拟器执行语义查找和组合，在无相关事件时返回明确的"无证据"响应而非虚构内容，实现约 5 倍延迟降低。

3. **策略更新：** 使用 GRPO（Group Relative Policy Optimization）算法优化主控智能体：
   - 每个查询采样 G = 8 条轨迹。
   - 轨迹级重要性采样比率 r_i(θ) = π_θ(H^(i)) / π_θ_old(H^(i))。
   - 优势函数通过组内奖励归一化计算。

#### 2.4 Sim-to-Real 迁移

训练好的策略直接部署到真实环境：主控智能体调用实际的视觉工具（T_vis，如 Qwen3-VL-8B）和音频工具（T_aud，如 Whisper）处理原始视频，交互历史随新观察更新，直到收集足够证据后终止并输出预测。

### 6. 论文主要结论与发现

1. **AgentCVR-8B 取得最佳开源性能：** 在多个任务上超越所有开源单次通过模型和改造的单视频智能体，整体平均准确率达 **42.03%**（对比最优开源单次通过模型 Qwen3-VL-32B 的 38.37%）。

2. **与闭源系统可比：** AgentCVR-8B 性能高度接近闭源前沿模型（如 Gemini-2.5-Pro 的 49.63%），同时具有开源可复现优势。

3. **多智能体架构有效性：** 替换为多智能体架构后，4B 和 8B 模型分别提升 8.59% 和 7.73%，证明证据获取与推理解耦的有效性。

4. **Script-Simulated RL 有效性：** 引入 RL 训练后，4B 和 8B 模型分别进一步提升 1.88% 和 4.83%，证明显式策略优化对长时域子智能体交互决策的重要性。

5. **视觉工具主导，音频工具辅助：** 视觉工具是提取跨视频证据的关键因素；音频工具提供互补线索，尤其在定位语音描述时有效。

6. **交互效率优势：** AgentCVR 平均交互轮数（14.4/16.7）与单视频智能体相当，但帧覆盖数显著更高（306.8/355.2），表明其能在有限交互内高效访问更广泛的视觉上下文。

7. **Sim-to-Real 高保真度：** 模拟环境与真实环境在 FSA 任务上 IoU 达 87.4%，BU 任务决策重叠率达 83.9%，延迟降低约 5 倍，验证了训练范式的有效性。

### 7. 优点

1. **范式创新：** 首次将 CVR 从被动单次上下文压缩转变为动态多轮主动证据获取，提供了全新的视角。

2. **高效训练范式：** Script-Simulated RL 巧妙解决了多模态 RL 训练成本过高的问题，用文本模拟器替代真实视频交互，实现约 5 倍延迟降低，同时保持高 Sim-to-Real 保真度。

3. **多智能体协同设计：** Master Agent 协调 Visual/Audio Agents 的架构设计清晰，证据获取与最终推理解耦，模块化程度高。

4. **GRPO 算法应用：** 使用 Group Relative Policy Optimization 优化多轮决策，结合稀疏正确性奖励和格式奖励，有效平衡探索与利用。

5. **全面的实验验证：** 覆盖 10 个任务、4 个维度的多层次实验，包含充分的消融分析和定性案例。

### 8. 不足与局限

1. **与闭源模型仍有差距：** AgentCVR-8B 整体平均准确率（42.03%）仍低于 Gemini-2.5-Pro（49.63%），存在显著优化空间。

2. **多轮推理失败模式：**
   - **指令漂移：** 扩展交互中可能出现格式错误。
   - **过早认知闭合：** 在多标签场景中可能过早提交单一答案，忽略其他有效选项。
   - **过度依赖内部先验：** 可能优先考虑通用常识而非具体视觉证据（如 PSS 任务中的烹饪流程假设）。

3. **仅优化 Master Agent：** 当前框架只优化主控智能体，模态特定智能体保持冻结，缺乏任务特定的适配。

4. **模拟器对齐仍有偏差：** 虽然 Sim-to-Real 保真度较高，但模拟环境与真实环境之间仍存在轻微偏差，需要更高保真度的模拟器。

5. **推理延迟增加：** 主动多轮交互范式不可避免地引入比单次通过方法更高的推理延迟，未来需优化决策效率、减少交互轮数。

6. **泛化性验证不足：** 主要在 CrossVid 一个 benchmark 上验证，未在其他 CVR 数据集上测试方法的泛化能力。

7. **多标签任务支持有限：** 实验显示 AgentCVR 在需要选择多个答案的任务（如"选择 A、B、C"）上表现不佳，存在排他性偏差问题。
