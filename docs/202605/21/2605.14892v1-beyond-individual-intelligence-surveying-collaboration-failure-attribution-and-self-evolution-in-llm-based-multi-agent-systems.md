## 摘要
基于 LLM 的自主智能体在推理、规划和工具使用方面已经展现出强能力，但当任务需要跨角色、工具和环境进行持续协调时仍存在局限。多智能体系统通过专业化智能体之间的结构化协作来解决这一问题，但更紧密的协调也会放大一个较少被探索的风险：错误可能跨智能体和交互轮次传播，产生难以诊断且很少转化为结构性自我改进的失败。现有综述分别覆盖单个智能体能力、多智能体协作或智能体自进化，却没有考察它们之间的因果依赖。本文围绕四个因果相连的阶段提供统一综述，我们称之为 LIFE 进程：Lay the capability foundation（奠定能力基础）、Integrate agents through collaboration（通过协作整合智能体）、Find faults through attribution（通过归因发现故障）以及 Evolve through autonomous self-improvement（通过自主自我改进进化）。对于每个阶段，我们提供系统分类，并形式化刻画相邻阶段之间的依赖关系，揭示每个阶段如何既依赖又约束下一个阶段。除综合已有工作外，我们还识别阶段边界处的开放挑战，并提出面向闭环多智能体系统的跨阶段研究议程，使系统能够持续诊断失败、重组结构并细化智能体行为，将当前协调框架扩展到更具自组织性的集体智能形式。通过连接这些先前碎片化的研究线索，本综述旨在为自主、自我改进的多智能体智能提供系统参考和概念路线图。

## Abstract
LLM-based autonomous agents have demonstrated strong capabilities in reasoning, planning, and tool use, yet remain limited when tasks require sustained coordination across roles, tools, and environments. Multi-agent systems address this through structured collaboration among specialized agents, but tighter coordination also amplifies a less explored risk: errors can propagate across agents and interaction rounds, producing failures that are difficult to diagnose and rarely translate into structural self-improvement. Existing surveys cover individual agent capabilities, multi-agent collaboration, or agent self-evolution separately, leaving the causal dependencies among them unexamined. This survey provides a unified review organized around four causally linked stages, which we term the LIFE progression: Lay the capability foundation, Integrate agents through collaboration, Find faults through attribution, and Evolve through autonomous self-improvement. For each stage, we provide systematic taxonomies and formally characterize the dependencies between adjacent stages, revealing how each stage both depends on and constrains the next. Beyond synthesizing existing work, we identify open challenges at stage boundaries and propose a cross-stage research agenda for closed-loop multi-agent systems capable of continuously diagnosing failures, reorganizing structures, and refining agent behaviors, extending current coordination frameworks toward more self-organizing forms of collective intelligence. By bridging these previously fragmented research threads, this survey aims to offer both a systematic reference and a conceptual roadmap toward autonomous, self-improving multi-agent intelligence.

## 2. 论文提出的方法论：核心思想与关键技术

### 2.1 核心思想：LIFE 框架

论文将 LLM 多智能体系统的完整生命周期组织为四个**因果相连的阶段**，合称 **LIFE**：

| 阶段 | 全称 | 对应内容 |
|------|------|----------|
| **L** | Lay capability foundation | 个体智能体能力基础（第2章） |
| **I** | Integrate agents through collaboration | 多智能体协作（第3章） |
| **F** | Find faults through attribution | 失败归因（第4章） |
| **E** | Evolve through autonomous self-improvement | 系统自进化（第5章） |

每个阶段之间存在**双向依赖**：前一个阶段的输出约束后一个阶段的搜索空间，后一个阶段的结果反过来塑造前一个阶段的组织形式。

### 2.2 第2章 — 个体智能（Individual Intelligence）

将单个智能体建模为结构化序贯决策系统：

**智能体形式化定义**（公式1）：

$$a = (X, U, O, M, R, P, T)$$

其中：
- $X$：观察空间
- $U$：动作空间
- $O$：工具返回的观察空间
- $M$：记忆状态空间
- $R$：推理模块（Reasoning）
- $P$：规划模块（Planning）
- $T$：工具执行模块（Tool）

**决策流程**（公式2-7）：观察 → 记忆检索 → 推理 → 规划 → 动作选择 → 工具执行 → 记忆更新，形成闭环系统。

**四大能力维度**：

1. **推理（Reasoning）**：分为输入阶段增强（知识增强 RAG / 多模态感知）、推理过程增强（搜索空间扩展 / 路径验证）、输出阶段调节（可靠性评估 / 响应行为调控）。
2. **记忆（Memory）**：覆盖记忆形成（直接记录 / 抽象蒸馏 / 类型路由）、记忆维护（存储结构 / 动态更新）、记忆检索与利用三个子环节。
3. **规划（Planning）**：分解为基于分解的规划（主动分解 / 渐进分解）和基于搜索的规划（步级搜索 / 计划级搜索）。
4. **工具使用（Tool Use）**：涵盖工具能力获取（轨迹学习 / 协议对齐）、工具调用（单次调用 / 闭环调用 / 工作流编排）、工具泛化（未见工具泛化 / 工具发现与检索）。

### 2.3 第3章 — 多智能体协作（Multi-Agent Collaboration）

**多智能体系统形式化定义**（公式8）：

$$S = (A, E, C, G, \Pi)$$

- $A$：智能体集合 $\{a_1, ..., a_N\}$
- $E$：共享环境
- $C$：通信协议
- $G$：编排拓扑结构（$G = (A, E_c)$）
- $\Pi$：协作策略集合

**四大协作维度**：

1. **角色（Role）**：同质角色 vs 异质角色；静态分配 vs 动态分配。
2. **通信（Communication）**：显式通信（结构化消息 / 自然语言）vs 隐式通信（环境感知 / 行为观察）；内部协议（A2A、ANP）和外部协议（MCP）。
3. **编排（Orchestration）**：中心化编排（中央控制器统筹）、分布式编排（点对点自主协调）、混合编排（层级结构结合）。
4. **交互（Interaction）**：信息流分为顺序交互与并行交互；交互模式分为合作交互与竞争交互。

### 2.4 第4章 — 失败归因（Failure Attribution）

**失败归因的形式化定义**（公式12-13）：

轨迹 $\tau = \{(i_0, u_0), ..., (i_H, u_H)\}$，当轨迹评估函数 $Z(\tau) = 1$ 时，归因模型 $f$ 输入系统配置 $\Omega$、轨迹 $\tau$ 和用户查询 $q$，输出责任智能体 $I$ 和关键时间步 $t$：

$$(I, t) = f(\Omega, \tau, q), \quad \text{if } Z(\tau) = 1$$

**失败分类学**：
- 按系统结构分类（规范/设计失败、智能体间错位、任务验证失败）
- 按执行阶段分类（推理错误、系统执行错误、规划/协调错误）
- 按因果生命周期分类（智能体级、工作流级、平台级根因）

**归因方法三大类**：

| 方法类别 | 核心机制 | 代表工作 |
|----------|----------|----------|
| **数据驱动方法** | 从大量标注失败轨迹中学习归因模式 | AGENTRACER, Spectrum, CORRECT, GraphTracer |
| **约束引导方法** | 通过结构化诊断流程限制搜索空间 | ABDUCT, Role-Trace, DOVER, AgentRx |
| **因果推理方法** | 通过反事实推理和因果图分析定位根因 | MARL, InterDebug, CDC-MAS, AgentFail |

### 2.5 第5章 — 系统自进化（Self-Evolution）

**自进化系统形式化定义**（公式14-16）：

将静态系统扩展为离散时间过程 $S(k)$，自进化映射 $\Gamma$ 基于历史上下文 $H(k)$ 生成下一代系统：

$$S(k+1) = \Gamma(S(k), H(k))$$

目标：$\mathbb{E}_{\xi \sim D_{task}}[J(S(k+1), \xi)] \geq \mathbb{E}_{\xi \sim D_{task}}[J(S(k), \xi)]$

**三层进化分类学**（公式17）：

$$\Gamma = [\Gamma_{agentic}, \Gamma_{systemic}, \Gamma_{meta}]$$

1. **智能体级进化**（$\Gamma_{agentic}$）：修改单个智能体的提示、记忆或神经参数。
2. **系统级进化**（$\Gamma_{systemic}$）：在给定系统实例内重构协作拓扑、通信协议和协作策略。
3. **元级进化**（$\Gamma_{meta}$）：将完整系统配置作为种群中的候选个体，通过专门的元过程跨任务积累可迁移的设计知识。

- LLM 反射（语义导向变化）

- 进化算法（种群多样性维持）
- 文本梯度（因果归因驱动）
- 启发式更新（规则驱动）
- 监督微调（知识传递）

## 6. 论文的主要结论与发现

### 6.1 LIFE 框架的提出

论文的核心贡献是提出了 **LIFE 四阶段因果链框架**，将 LLM 多智能体系统的研究从碎片化状态整合为统一的发展路线图。四个阶段之间的依赖关系如下：

- **L → I**：个体能力（推理、记忆、规划、工具使用）为多智能体协作提供了基础；能力短板往往成为多智能体系统的实际瓶颈。
- **I → F**：协作结构决定了哪些失败可被观测以及如何传播；协作越紧密，错误传播越隐匿，诊断越困难。
- **F → E**：失败归因缩小了进化搜索空间，使自我改进有的放矢，而非盲目随机调整。
- **E → L/I**：进化获得的组织知识反过来重塑个体能力和协作拓扑。

### 6.2 个体智能的关键发现

1. **模块化优化主导，但跨模块耦合未被充分研究**：现有方法几乎仅优化单一模块，但端到部著署中能力错配往往成为实际瓶颈。
2. **推理增强与幻觉抑制正在趋同**：搜索空间扩展与过程奖励模型（PRM）是连接两者的关键桥梁。
3. **记忆设计碎片化**：图结构存储擅长关系检索但维护成本高；基于效用的维护需要大量交互才能收敛。
4. **能力评估为静态快照**，忽略连续适应过程。

### 6.3 多智能体协作的关键发现

1. **从同质智能体到功能专业化**：异质角色配置（规划、编码、审查、评估分工）已成为主流，但角色多样性增加也带来相互依赖性和级联错误风险。
2. **静态 vs 动态角色分配**：静态分配适合可预测的分解结构；动态分配适合开放性任务，但当前系统多为启发式实现，缺乏学习何时专业化或冗余的机制。
3. **显式通信主导但可能无法无限扩展**：随着智能体数量和交互轮次增加，Token 开销、延迟和不一致性问题凸显；隐式通信值得更多关注。
4. **编排拓扑无通用最优**：中心化提供全局控制但易产生瓶颈；分布式提高鲁棒性但共识和冲突解决更困难；混合编排成为趋势，应视为自适应设计变量而非固定架构。

### 6.4 失败归因的关键发现

1. **归因定义尚未统一**：不同研究将归因表述为负责任智能体定位、错误类型识别、交互级失败、平台级工作流根因等多种形式，缺乏跨研究的可比性。
2. **异常传播高度耦合**：局部偏差通过任务分配、工具调用和结果聚合逐渐放大，终点与真正起点往往不一致。
3. **细粒度归因受限**：现有方法在步骤级、模块级、错误类型级归因上可靠性不足，对长程轨迹中的多角色、多层次依赖结构处理能力有限。
4. **数据真实性不足**：自动生成样本与真实部署中的复杂失败模式存在分布差异，人工标注成本高且存在标注者分歧。
5. **评估-修复闭环不完整**：现有评估主要依赖定位准确率，缺乏与下游验证、干预和修复的紧密连接。

### 6.5 自进化的关键发现

1. **三层进化分类学**：智能体级（提示/记忆/参数）、系统级（拓扑/团队构成/共享记忆）、元级（知识空间/生成器优化）。
2. **驱动机制分布不均**：RL 和 LLM 反射覆盖所有三个进化层次；文本梯度和启发式更新目前仅限于系统级。
3. **生命周期差距**：大多数系统级和元级方法采用在线-短暂或离线-持久生命周期，缺乏真正的持续终身学习。
4. **缺乏混合机制**：EA 的种群多样性、RL 的参数精调、反射的语义导向三者互补，但仅有少量工作将它们结合。
5. **安全对齐挑战**：自我进化的多智能体系统可能绕过人类设计的安全护栏，需要将安全约束提升为动态的、贯穿进化过程的选择压力。

### 6.6 跨阶段研究议程

论文提出了闭合循环多智能体系统的研究路线：
1. **自适应协作**：角色、拓扑和通信策略在线选择而非预先固定。
2. **协作高效设计**：以更少的消息、更少的智能体、更紧凑的共享表示获得更好的性能。
3. **可信多智能体推理**：可解释的协调轨迹、故障定位和鲁棒评估协议。
4. **终身进化**：持久积累跨任务协作知识，克服灾难性遗忘。
5. **进化对齐**：安全作为进化的内在、持续选择压力，而非静态约束。

## 8. 论文的局限与不足

### 8.1 作为综述论文的固有局限

1. **文献选择性风险**：综述覆盖的方法虽多，但仍可能遗漏相关工作，且选择标准未完全透明。
2. **时效性限制**：LLM 和多智能体领域发展极为迅速，本综述的某些内容可能在发表时已经部分过时。
3. **二手数据依赖**：对各方法的性能对比依赖原始论文报告的结果，未进行独立验证，存在报告偏差的风险。

### 8.2 具体分析维度上的不足

1. **性能基准对比不充分**：论文提供了大量分类和定性比较，但缺少量化的性能对比（如胜率、平均分数等），读者难以快速判断哪些方法在特定任务上表现最佳。
2. **计算成本分析缺失**：未系统比较各方法在训练和推理阶段的计算成本（如 Token 消耗、API 调用次数、延迟等），这些信息对实践者至关重要。
3. **安全与对齐讨论深度不足**：虽然论文在第5章的进化部分和第6章的研究议程中提到了安全挑战，但安全与对齐作为独立维度未进行系统性深入分析，尤其忽视了多智能体协作和进化过程中的**涌现风险**（emergent risks）。
4. **多模态多智能体覆盖有限**：论文主要聚焦于文本驱动的多智能体系统，对视觉-语言智能体、具身智能体或多模态协作的讨论相对薄弱。
5. **真实世界部署案例不足**：综述主要基于学术基准测试和消融实验，对工业级、真实部署的多智能体系统（如生产环境中的实际应用案例）讨论有限。
6. **经济与伦理维度未充分探讨**：未深入讨论多智能体系统的资源消耗、环境影响（碳足迹）、以及潜在的就业替代等伦理和社会问题。
7. **长程交互中的累积误差**：对多轮交互中微小错误的累积放大效应讨论不够深入，尤其是在长程任务中的错误传播链。
8. **评估标准不统一**：失败归因和自进化领域的评估方法差异巨大，缺乏通用的评估协议和标准化基准，限制了方法间的公平比较。
