## 摘要
Self‑Distilled Agentic Reinforcement Learning（SDAR）将在线自蒸馏作为门控辅助目标，结合 RL 主干优化多轮交互的强化学习。通过 Sigmoid 门控强化正向教师认可 token、软衰减负向拒绝，显著提升跨模型规模在 ALFWorld、WebShop、Search‑QA 上的性能。

## Abstract
Reinforcement learning (RL) has emerged as a central paradigm for post-training LLM agents, yet its trajectory-level reward signal provides only coarse supervision for long-horizon interaction. On-Policy Self-Distillation (OPSD) complements RL by introducing dense token-level guidance from a teacher branch augmented with privileged context. However, transferring OPSD to multi-turn agents proves problematic: compounding multi-turn instability destabilizes supervision, while skill-conditioned privileged guidance requires asymmetric treatment for negative teacher rejections may arise from imperfect skills retrieval or utilization. We introduce SDAR (Self-Distilled Agentic Reinforcement Learning), which treats OPSD as a gated auxiliary objective while keeping RL as the primary optimization backbone. SDAR maps detached token-level signals into a sigmoid gate, strengthening distillation on teacher-endorsed positive-gap tokens and softly attenuating negative teacher rejections. Across the Qwen2.5 and Qwen3 families on ALFWorld, WebShop, and Search-QA, SDAR substantially improves over GRPO (+9.4% on ALFWorld, +7.0% on Search-QA, +10.2% on WebShop-Acc), avoids the instability of naive GRPO+OPSD, and consistently outperforms hybrid RL--OPSD baselines across model scales.

### 1. 核心问题与整体含义（研究动机和背景）

**研究问题：**
- 大语言模型（LLM）智能体在多轮交互场景（如ALFWorld、WebShop、Search-QA）中进行强化学习时，RL仅提供稀疏的轨迹级奖励信号，缺乏细粒度的token级监督。
- On-Policy Self-Distillation (OPSD) 通过特权上下文（如检索到的技能）引入密集的token级指导，但直接迁移到多轮智能体训练时存在两大致命问题：
  1. **多轮OPSD不稳定性**：学生策略一旦偏离教师支持的轨迹，token级监督变得不可信，导致每轮KL散度激增和灾难性性能下降。
  2. **特权引导的非对称信任问题**：教师并非独立更强的模型，而是同一策略附加特权上下文（如技能）。当教师对某token赋值更低概率时，可能源于技能质量不佳、利用率低或多轮漂移，而非真正的负面信号。

**核心洞察：**
论文指出，超过50%的token呈现负gap（教师比学生概率更低），因此需要非对称处理：更信任正向教师认可，更保守对待负向教师拒绝。

### 2. 论文提出的方法论

**核心思想：**
SDAR将OPSD作为**门控辅助目标**，保持RL作为主优化骨干，通过自适应sigmoid门在token级别调节蒸馏强度，实现“让每个token自己决定监督强度”。

**关键技术细节：**

| 组件 | 说明 |
|------|------|
| **整体目标** | $L(\theta) = L_{\text{GRPO}}(\theta) + \lambda_{\text{SDAR}} \cdot L_{\text{SDAR}}(\theta)$ |
| **Token级门控** | $g_t = \sigma(\beta \Delta_t)$，其中 $\Delta_t = \log \pi_T(y_t|s_t^+) - \log \pi_\theta(y_t|s_t)$（teacher-student log-probability gap） |
| **门控策略** | 三种：① Entropy gating $g_t = \sigma(\beta h_t)$；② Gap gating（默认，使用上述公式）；③ Soft-OR gating $g_t = \sigma(\beta[1-(1-h_t)(1-\Delta_t)])$ |
| **蒸馏损失** | $\ell_t^{\text{SDAR}} = g_t \cdot (\log \pi_T(y_t|s_t^+) - \log \pi_\theta(y_t|s_t))$，取反向KL方向 |
| **停止梯度（sg）** | 对$g_t$和teacher log-prob应用sg，确保梯度仅流经学生策略 |

**设计动机：**
- Sigmoid函数将无界的gap转换为有界、平滑的(0,1)权重，防止梯度爆炸。
- 反向KL天然具有“mode-seeking”特性，只追随教师支持的模式，与门控机制互补。
- 停止梯度避免自引用优化路径，确保稳定性。

**算法流程（见Algorithm 1）：**
1. 对每条轨迹采样G个响应，计算环境奖励及GRPO优势。
2. 计算GRPO策略损失（clip surrogate objective）。
3. 构建teacher context（附加特权技能），计算token-level gap和门控值。
4. 联合优化：$L = L_{\text{GRPO}} + \lambda \cdot L_{\text{SDAR}}$。

### 7. 优点

1. **优雅的辅助角色定位**：将OPSD置于RL的辅助地位，不破坏RL优势的语义和无偏性。
2. **自适应token级门控**：细粒度（token-level）的动态课程学习，比刚性调度（如TCOD）或手写规则更灵活。
3. **理论支撑**：提供5个命题证明等价形式、梯度有界性、单调性，理论完备。
4. **对检索质量鲁棒**：即使随机检索也能带来正向收益，降低对下游系统的依赖。
5. **跨尺度一致性**：在1.7B到7B模型上均有效，证明方法的可扩展性。
6. **消融实验充分**：覆盖门控策略、锐度、系数、蒸馏目标等多个维度。

---

### 8. 不足与局限

1. **模型规模覆盖有限**：仅在≤7B模型上验证，更大模型（如14B+、72B）的效果未探索。
2. **技能库依赖**：虽然对检索质量鲁棒，但仍然依赖预定义的SkillBank，可能存在领域覆盖不足的问题。
3. **只验证了Qwen家族**：缺乏对Llama、Mistral等其他模型架构的泛化性验证。
4. **训练时长较短**：仅150步，长期训练是否会导致门控失效或过拟合未充分探索。
5. **门控策略偏向经验设计**：三种门控策略（Entropy/Gap/Soft-OR）均为启发式设计，缺乏自动搜索或理论最优性证明。
6. **多轮设定局限于固定回合**：未讨论如何处理开放式交互（如无限回合对话）。
7. **RL基线单一**：主要对比GRPO，未与PPO、DPO等其他RL算法做深入对比。
