## Abstract
Riichi Mahjong is a multi-player, imperfect-information game characterized by stochasticity and high-dimensional state spaces. These attributes present a unique combination of challenges that mirror complex real-world decision-making problems in reinforcement learning. While prior research has heavily relied on supervised learning from human play logs to pre-train the policy, algorithms capable of learning \textit{tabula rasa} (from scratch) offer greater potential for general applicability, as evidenced by the AlphaZero lineage. To facilitate such research, we introduce \textbf{Mahjax}, a fully vectorized Riichi Mahjong environment implemented in JAX to enable large-scale rollout parallelization on Graphics Processing Units (GPUs). We also provide a high-quality visualization tool to streamline debugging and interaction with trained agents. Experimental results demonstrate that Mahjax achieves throughputs of up to \textbf{2 million} and \textbf{1 million steps per second} on eight NVIDIA A100 GPUs under the no-red and red rules, respectively. Furthermore, we validate the environment's utility for reinforcement learning by showing that agents can be trained effectively to improve their rank against baseline policies.

### 1. 核心问题与整体含义（研究动机和背景）

- **麻将（Riichi Mahjong）本身的特性**：
  - 四人参与、信息不完全、状态空间高维、随机性强，是强化学习（RL）中极具挑战性的决策问题。
  - 与扑克类似，但涉及更长的时序和多方博弈。
- **现有研究的局限**：
  - 目前多数麻将 AI（如 Suphx）依赖**监督学习（SL）**——从人类对局日志中预训练，再做微调，而非真正从零学习（tabula rasa）。
  - 麻将四玩家、长 horizon 的特性导致**数据吞吐量成为瓶颈**，CPU 模拟器无法高效支持大规模自对弈训练。
- **研究缺口**：
  - 缺少一个能够**在 GPU 上大规模并行**、完全向量化的麻将环境，以支撑从零开始的 RL 研究（如 AlphaZero 风格的纯自对弈）。
- **核心动机**：
  - 提供一个高性能、可扩展的麻将模拟器，使研究者能够在 GPU 上进行大规模 rollout 并行，从而实现真正的 tabula rasa 强化学习。

### 2. 论文提出的方法论：核心思想与关键技术细节

#### 2.1 核心思想
- 在 **JAX** 框架下实现一个**全向量化的四人麻将环境**，采用函数式编程范式（不可变 State + 纯函数），以充分利用 JIT 编译和 GPU 并行能力。
- 遵循 **Pgx API** 设计规范，确保与现有 JAX-based RL 生态（如 Pgx、JaxMARL 等）兼容。

#### 2.2 关键技术细节

| 模块 | 说明 |
|---|---|
| **环境 API** | 标准化 `init / step / observe` 接口，支持 `jax.jit(jax.vmap(...))` 并行批量模拟。 |
| **状态管理** | State 数据类包含所有游戏信息（手牌、分值、风圈、副露、掩码等），均为 **JAX 数组（immutable）**。 |
| **规则支持** | 两种主要变体：**Tenhou 红五规则**（含红宝牌，含复杂规则如流局）和**No-Red 规则**（无红宝牌，简化部分规则）。三种模式：single（单局）、east（四局东）、half（八局东南）。 |
| **动作空间** | 离散动作 ID：打牌、杠、吃、碰、立直、荣和、过等；提供 `legal_action_mask` 过滤非法动作。 |
| **观测空间** | 结构化字典观测，面向 Transformer 架构设计，包含：手牌索引、打牌历史、番数/Shanten 数等标量。Ego-centric（以当前玩家为中心）。 |
| **GPU 优化** | ① **向量化逻辑**：将复杂的条件分支（如 if-else）替换为矩阵运算；② **缓存机制**：对番数计算等高开销步骤进行预计算并编码为位掩码。 |
| **可视化** | SVG 矢量图 + Web UI，支持英文本地化，便于调试和人与 AI 对战。 |

### 3. 实验设计：数据集、基准与对比方法

#### 3.1 吞吐量基准（Benchmark）

- **对比对象**：
  - **Libriichi**（Rust 实现，CPU 多线程，用于 Mortal 项目）——仅支持红五规则。
  - **Pgx 将棋（Shogi）**——同类 JAX 向量化环境，作为通用基准参考。
- **测试场景**：
  - 批量大小从 2^3 到 2^14（单卡）或从 8 到 2^14（8 卡）不等。
  - 分别在单卡 NVIDIA A100 和 8 卡 A100 集群上测试。
  - 测量指标：**每秒步数（Steps Per Second, SPS）**。
- **关键实验结果**：
  - 单卡：A100 上 Mahjax 在 batch≈2^10 时趋于饱和（约 600k–800k SPS），Libriichi 在 batch≈2^3 处即饱和。
  - 8 卡：A100×8 上 Mahjax 继续扩展，**峰值达到 2M SPS（no-red）和 1M SPS（red）**，超过 Libriichi **10 倍以上**，并优于 Pgx Shogi。

#### 3.2 强化学习训练验证

- **目标**：验证环境可用于 RL 训练，而非追求 SOTA 性能。
- **训练配置**：
  - 规则：no-red，单局模式（single-round）。
  - 预训练：**Behavioral Cloning（BC）**——基于启发式规则代理生成 500k 样本。
  - 主训练：**PPO + KL 正则化**（向 BC 策略约束），参数：γ=1.0, λ=0.95, lr=3e-4, clip ε=0.2, entropy coeff=0.01, value coeff=0.5, KL penalty=0.2。
  - 并行环境数：1,024；rollout 长度：256；总步数：1 亿。
- **评估方法**：
  - 对手：3 个固定 BC 策略（1 vs 3），共 1,000 局。
  - 指标：**平均排名**（理论期望 2.5，越低越好）。
- **结果**：代理平均排名持续低于 2.5 基线，表明 PPO 训练成功改进策略。

### 6. 主要结论与发现

1. **Mahjax 实现了业界领先的模拟吞吐量**：
   - 8× A100 上 no-red 规则达 **2M SPS**，red 规则达 **1M SPS**，超过 CPU 模拟器 Libriichi 10 倍以上。
2. **GPU 并行化显著优于 CPU**：
   - 单卡即超越 Libriichi 最优性能，8 卡进一步扩展，表明大规模自对弈在算力上已可行。
3. **环境可用于 RL 训练**：
   - 基于 BC + PPO 框架，代理在 1 亿步训练后平均排名优于基线，验证了模拟器的正确性和稳定性。
4. **API 与现有生态兼容**：
   - 遵循 Pgx 规范，便于与 Transformers 等现代 RL 算法集成。

### 7. 优点

- **大规模 GPU 向量化**：首次在 JAX 中实现四人麻将全向量化，突破 CPU 瓶颈，为大规模 tabula rasa RL 铺路。
- **两种规则覆盖**：Tenhou 红五规则（贴近实际竞技环境）和简化 no-red 规则（适合快速迭代），兼顾真实性和效率。
- **多粒度游戏模式**：single/east/half 三种模式提供不同难度梯度，便于渐进式研究。
- **高质量可视化**：SVG/Web UI + 英文支持，降低研究门槛，促进人机交互调试。
- **代码开源**：GitHub 公开，便于复现和社区贡献。

### 8. 不足与局限

- **RL 实验覆盖有限**：
  - 仅在 no-red + single-round 模式下验证，未测试 red 规则或 full-round（half/east）模式。
  - 训练规模（1 亿步）与 Suphx 等实际系统相比仍较小，未展示从零学习的能力。
- **消融实验缺失**：
  - 未对向量化逻辑、缓存机制等关键优化进行独立消融，无法量化各模块的贡献。
- **多 GPU 扩展性边界未探明**：
  - 实验止步于 8 卡，未测试更大规模集群的 scaling behavior。
- **对比基线较少**：
  - 缺少与 Suphx、Mortal 等现有麻将 AI 的实际对战评估，仅 throughput 对比不足以说明策略质量。
- **缺乏完整规则验证**：
  - 当前不支持三麻（3-player）变体或部分复杂规则（如连庄、一发等），离完整 Tenhou 规则集仍有差距。
- **长期决策能力未验证**：
  - single-round 模式侧重即时手牌效率，未充分体现半庄制中的**排名防守、临时同盟**等长线策略。
