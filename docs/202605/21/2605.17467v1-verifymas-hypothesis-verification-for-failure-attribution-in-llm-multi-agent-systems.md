## 摘要
VerifyMAS提出假设验证框架用于多智能体系统失败归因，解决智能体间协调错误和全局失败问题。

## Abstract
Large language model-driven multi-agent systems (LLM-MAS) excel at complex tasks, yet unreliable agents remain a key bottleneck to system-level reliability. Automatic failure attribution is therefore critical, but existing approaches, such as direct prediction of agent-error pairs and agent-first failure attribution, rely on local logs of agents and miss global failures that only manifest over full interaction trajectories, such as cross-step inconsistencies and inter-agent coordination errors. Moreover, directly predicting failures induces a large combinatorial search space, hindering fine-grained attribution. To address these challenges, we propose VerifyMAS, a hypothesis verification framework for agent failure attribution. Instead of directly predicting faulty agents and error types, VerifyMAS formulates and verifies failure hypotheses against full trajectories. This verification-based approach decomposes attribution into trajectory-level error validation and fine-grained agent localization, providing an error-first attribution approach that captures global failure patterns while substantially reducing the search space. We further introduce a hypothesis-based data construction strategy grounded in a structured error taxonomy and fine-tune a specialized LLM verifier model for trajectory-level failure verification and agent attribution. Experiments on Aegis-Bench and Who&When show that VerifyMAS consistently improves diverse backbone models, including open-source Qwen and API-based GPT models, outperforming prior methods without sacrificing inference efficiency for long multi-agent trajectories.

### 1. 论文核心问题与整体含义（研究动机和背景）

**核心问题**：
LLM驱动的多智能体系统（LLM-MAS）在复杂任务中表现优异，但不可靠的智能体仍是系统级可靠性的关键瓶颈。如何自动、准确地将失败归因到具体的智能体和错误类型，是一个亟待解决的问题。

**研究背景与动机**：
- 现有方法（如直接预测智能体-错误对、Agent-first失败归因）依赖智能体的本地日志，难以捕获仅在完整交互轨迹中才显现的**全局失败**（如跨步骤不一致、智能体间协调错误）。
- 直接预测失败会产生**巨大的组合搜索空间**，阻碍细粒度归因。
- 现有的Agent-first方法因从单个智能体出发，容易偏向局部上下文，忽视全局轨迹级证据。

### 2. 论文提出的方法论

#### 核心思想
VerifyMAS提出**假设验证框架**，将失败归因重新定义为"错误优先"的假设验证问题。具体而言：
- 不直接预测错误类型和责任智能体，而是**针对预定义的错误假设进行验证**，判断轨迹是"支持（entail）"、"中立（neutral）"还是"反驳（contradict）"该假设。
- 这种**错误优先（error-first）** 的思路使得模型能够从全局轨迹视角评估失败，而非仅依赖局部智能体行为。

#### 关键技术细节

**零样本推理流程（Zero-shot Inference）**：
1. **假设构建**：基于预定义的14种错误类型（如FM-1.1任务偏离、FM-2.1重复任务等），为每种错误构建自然语言假设。
2. **三路假设验证**：将轨迹与每个假设配对，LLM验证器预测标签 ∈ {A: entail, B: neutral, C: contradict}。采用三路而非二路的原因是很多失败假设证据不足或影响不明确，Neutral类可避免强制分类。
3. **智能体归因**：仅对预测为"entail"的假设进行责任智能体定位，从候选智能体集合中选择最可能的负责任智能体。
4. **两阶段设计**：将失败验证与智能体定位分离，降低组合复杂度。

**有监督微调（SFT）训练策略**：
- **数据构建**：
  - **Entail样本**：直接从标注轨迹构建，每个错误假设对应其真实责任智能体。
  - **Contradict样本**：使用反证关键词表（如表7），当轨迹包含明确反驳证据时，将对应假设标记为矛盾。
  - **Neutral样本**：基于错误类型邻近映射表（表8），选择语义相关但无充分证据支持的假设作为中性样本。
  - **稀有智能体重采样**：对长尾分布的智能体进行过采样，避免偏向高频智能体。
- **训练损失**：标准语言模型交叉熵损失，监督生成结构化JSON输出（包含标签和责任智能体）。

1. **VerifyMAS显著优于现有方法**：在Aegis-Bench上，VerifyMAS相比DPR和CoT-Agent在Pair-F1上提升1.75~3.29个百分点。

6. **效率保持**：VerifyMAS在提供更强性能的同时，运行时间与基线方法相当（~2.1秒/轨迹）。

### 7. 优点

- **创新性框架设计**：将失败归因从直接预测转变为假设验证，分解为轨迹级错误验证和细粒度智能体定位，显著降低组合搜索空间。
- **错误优先范式优势**：从错误类型出发而非智能体，使模型能优先利用全局轨迹上下文，避免局部行为偏差。
- **系统性数据构建策略**：反证关键词表和错误类型邻近映射表提供了高质量的负样本构造方法，提升模型区分能力。
- **稀有智能体处理**：针对长尾分布的重采样策略避免模型偏向高频智能体。
- **三路验证设计**：Neutral类处理证据不足的模糊情况，比二路分类更符合实际场景。
- **实验设计严谨**：覆盖多种模型、多粒度指标、消融研究和跨域泛化测试。

### 8. 不足与局限

- **Agent级性能不稳定**：在某些模型上Agent-level μF1/MF1提升不显著或略有下降，表明责任智能体的准确定位仍是难点。
- **可能产生非保守预测**：论文指出验证驱动设计可能导致在某些轨迹上过度预测错误。
- **假设依赖预定义错误分类**：方法基于14种预定义错误类型，对新出现的错误模式可能需要扩展分类体系。
- **长轨迹复杂度**：虽然验证框架降低了组合空间，但长交互轨迹仍可能包含复杂的跨步骤依赖，验证复杂度随轨迹长度增长。
- **反证关键词覆盖有限**：消融实验中Counter-evidence策略的有效性依赖于预先定义的关键词表，可能无法覆盖所有反驳证据模式。
- **未来改进方向**：探索额外的补充监督信号以实现更精确的失败定位。
