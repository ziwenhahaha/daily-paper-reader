## Abstract
History-dependent policies induced by recurrent neural networks (RNNs) rely on latent hidden state dynamics, making verification in partially observable reinforcement learning (RL) challenging. Existing RNN verification tools typically rely on restrictive modeling assumptions or coarse over-approximations of the hidden state space, which can lead to overly conservative or inconclusive results. We propose $\textbf{RNN}$ $\textbf{Pro}$babilistic $\textbf{Ve}$rification ($\texttt{RNN-ProVe}$), a probabilistic framework that $\textit{estimates the likelihood}$ of undesired behaviors in RNN-based policies. $\texttt{RNN-ProVe}$ uses policy-driven sampling to approximate the set of hidden states that are feasible under a trained policy, and derives statistical error bounds to produce bounded-error, high-confidence estimates of behavioral violations. Experiments on partially observable single-agent and cooperative multi-agent tasks show that $\texttt{RNN-ProVe}$ yields more quantitative, feasibility-aware probabilistic guarantees than existing tools, while scaling to recurrent and multi-agent settings.

- **核心问题**：基于循环神经网络（RNN）的强化学习策略在部分可观测环境中依赖隐状态（hidden state）来编码历史交互信息，但现有 RNN 验证工具难以精确刻画"哪些隐状态是策略实际可达的"，导致验证结果过于保守或不准确。
- **背景**：
  - RNN 广泛用于处理部分可观测性和多智能体协调问题，但其内部记忆机制的验证一直是难题。
  - 现有 RNN 验证主要面向**鲁棒性分析**（bounded input perturbations），而非策略在**实际可达历史**上的行为验证。

  - 传统方法依赖强建模假设或粗粒度几何超近似（如区间传播），会引入虚假隐状态（spurious states），产生误报。
- **研究动机**：在单智能体和合作多智能体 RL 场景中，定量评估 RNN 策略产生不安全行为的概率，同时保证统计误差有界、高置信度。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想

RNN-ProVe 将精确验证松弛为**概率近似验证**：不枚举所有可行隐状态，而是通过策略驱动的采样来估计违反行为占可行隐状态空间的比例，给出带误差上界和置信度的体积估计。

### 关键技术细节

- **可行历史预言机（Feasibility Oracle）**：
  - 在 RL 训练过程中记录"状态-隐状态"对 $(s, h)$，训练一个二分类器 $C: S \times H \to \{0,1\}$，判断给定状态下某隐状态是否可由策略实现。
  - 训练数据来源于最大熵探索策略，覆盖范围广（包含次优但可行的交互）。
- **两阶段验证流程**：
  - **Phase 1-2（离线）**：策略训练 → 收集状态-隐状态对 → 训练可行性分类器 $C$。
  - **Phase 3（在线验证）**：对目标状态 $s$ 采样候选隐状态 $\rightarrow$ 用 $C$ 过滤出可行隐状态 $h^*$ → 对每个 $h^*$ 检查策略行为 $f(h, o)$ 是否违反需求 → 用蒙特卡洛估计计算违反比例 $\tilde{V}$。
- **统计误差分解**：通过 Hoeffding 不等式，将总误差 $\epsilon$ 分解为分类器误差上界 $\epsilon_{\text{clf}}$ 和蒙特卡洛采样误差 $\epsilon_{\text{ver}}$，并确定所需样本量 $N, M \geq \frac{\ln(2/\delta)}{2\epsilon^2}$。
- **关键定理（Theorem 2）**：保证 $\Pr\left[\frac{|\tilde{V} - \text{Vol}(\Gamma(T))|}{\text{Vol}(H)} \leq \epsilon\right] \geq 1 - \delta$，即 bounded-error, high-confidence 证书。
- **多智能体扩展**：将验证问题分解为各智能体独立验证，取最大违反概率作为全局证书，保持可扩展性。

### 算法流程概述

1. 用 DRQN/CTDE-DRQN 训练 RNN 策略；
2. 收集训练期间的状态-隐状态对，构建数据集；
3. 训练可行性分类器 $C$；
4. 对目标状态 $s$ 采样 $N$ 个隐状态，用 $C$ 筛选可行子集；
5. 在可行子集上评估策略行为，计算违反比例 $\hat{p}$；
6. 根据 Hoeffding 不等式给出带误差 $\epsilon$ 和置信度 $1-\delta$ 的体积估计。

## 6. 论文的主要结论与发现

1. **可行历史过滤是关键**：朴素 MC 采样报告高达 21%~24% 的违反率，而 RNN-ProVe 仅报告 1%~13%，差异来源于朴素采样包含了大量不可达隐状态。
2. **规模可扩展**：RNN-ProVe 在所有任务上均可在毫秒级返回结果（0.002s~0.05s），而精确基线在 16×16 导航上超时（>45分钟）。
3. **非平凡违反率被识别**：即便训练性能良好（平均回报高），RNN-ProVe 仍识别出 1%~13% 的可行历史会导致碰撞行为，说明平均回报指标不足以揭示隐状态敏感性问题。
4. **多智能体场景可扩展**：合作 BP 任务中两个智能体的违反概率分别为 1.15%~1.73%，框架自然支持 CTDE 设置下的分散执行验证。
5. **近似 oracle 足够好**：在 4×4 环境中精确 oracle 与近似 oracle 的验证结果几乎相同（1.43% vs 1.44%），说明训练期间收集的数据已充分覆盖可行隐状态空间。

- **策略多样性有限**：仅验证了 GRU-based DRQN 策略，未覆盖 LSTM、Transformer-based RNN 或其他循环架构。

- **未考虑对抗性扰动**：验证的是策略在自然可达历史上的行为，未讨论在可行历史上叠加对抗扰动时的安全性（这超出了本文范围但仍是重要方向）。
- **缺乏真实连续控制任务验证**：实验均为离散动作/网格场景，与机器人控制等实际应用存在差距。
