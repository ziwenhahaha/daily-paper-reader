## 摘要
在线策略自蒸馏在推理模型的自身轨迹上进行训练，同时利用往往同一模型在特权上下文条件下的教师提供密集的 token 级监督。现有目标函数通常在思维链序列中统一加权教师的 token 级信号，尽管教师预测分布的熵存在显著差异。我们提出 EGRSD（熵引导增强自蒸馏），通过三个信号统一 token 级更新：基于奖励的方向、教师‑学生似然比幅度，以及所提出的教师熵置信门控，该门控在降低高熵 token 位置权重的的同时对每个 token 权重保持非零下界。我们进一步提出 CL‑EGRSD，一种因果前瞻变体，能够区分持续高熵区间与短暂高熵位置——后者后续上下文迅速转为低熵。使用 Qwen3‑4B 和 Qwen3‑8B 在思考模式下的实验表明，EGRSD 与 CL‑EGRSD 在所比较的可训练方法中提升了精度‑长度前沿。

## Abstract
On-policy self-distillation trains a reasoning model on its own rollouts while a teacher, often the same model conditioned on privileged context, provides dense token-level supervision. Existing objectives typically weight the teacher's token-level signal uniformly across a chain-of-thought sequence, despite substantial variation in the entropy of the teacher's predictive distribution. We propose EGRSD (Entropy-Guided Reinforced Self-Distillation), which unifies token-level updates through three signals: a reward-grounded direction, a teacher-student likelihood-ratio magnitude, and the proposed teacher-entropy confidence gate that down-weights high-entropy token positions while maintaining a nonzero lower bound on every token weight. We further introduce CL-EGRSD, a causal-lookahead variant that distinguishes sustained high-entropy spans from transient high-entropy positions whose following context rapidly becomes low entropy. Experiments with Qwen3-4B and Qwen3-8B in thinking mode show that EGRSD and CL-EGRSD advance the accuracy-length frontier among the compared trainable methods.

## 1. 核心问题与整体含义（研究动机与背景）

### 问题背景
- 大语言模型（LLM）在链式思维（Chain-of-Thought, CoT）推理时会产生大量冗余的中间推理 token，包含重复验证循环、模拟自纠错标记、重复中间推导等。
- 这种冗长推理导致推理延迟和计算成本显著增加，亟需在保持推理准确率的同时减少不必要的计算。

### 核心问题
- **在线策略自蒸馏（On-Policy Self-Distillation, OPSD）**：模型在自己的采样轨迹上训练，由一个"特权教师"（通常是同一模型在 privileged context 条件下的版本）提供密集的 token 级监督。
- **现有方法的缺陷**：现有 OPSD 目标函数对思维链序列中的所有 token 统一加权教师的 token 级信号，但实际上教师预测分布在不同 token 位置存在显著差异——有些位置低熵（确定性计算），有些位置高熵（分支决策点），有些是短暂高熵后迅速转为低熵的"转折点"。
- **核心矛盾**：均匀加权会过度强调高方差监督信号（如持续高熵位置），但完全抑制高熵 token 又会丢失转折点信号。

### 研究动机
- 识别并利用**教师预测熵**作为第三个未被使用的信号（前两个是奖励方向和师生似然比幅度）。
- 设计一种置信门控机制，在降低高熵 token 权重的同时保留非零下界，避免破坏有效的分支多样性信号。

## 2. 论文提出的方法论

### 核心思想
将 token 级更新分解为**三个信号的乘积**：
1. **方向（Direction）**：基于结果奖励的优势符号 $D_i = \text{sign}(A_i)$
2. **幅度（Magnitude）**：教师-学生似然比 $w_{i,t} = \text{clip}(\exp(D_i \cdot \delta_{i,t}), 1-\varepsilon, 1+\varepsilon)$
3. **置信度（Confidence）**：特权教师的预测熵门控 $\omega_{i,t}$（新增）

### 关键技术细节

#### 2.1 EGRSD（熵引导增强自蒸馏）

**教师熵计算**：
$$H_{i,t} = -\sum_{v \in V} p_T(v | x, s^\star, y_{i,<t}) \log p_T(v | x, s^\star, y_{i,<t})$$

**批量归一化**：
$$\hat{H}_{i,t} = \frac{H_{i,t}}{\max_{(j,k) \in C_{\text{batch}}} H_{j,k}}$$
（分母下界设为 1 nat 以避免数值不稳定）

**置信门控**：
$$\omega_{i,t} = \text{clip}(1 - \gamma \hat{H}_{i,t}, 0.1, 1)$$

- $\gamma$ 控制门控对教师熵的敏感度
- 下界 0.1 确保高熵位置不会被完全丢弃
- 最终 token 级优势：$\tilde{A}_{i,t} = A_i \cdot w_{i,t} \cdot \omega_{i,t}$

#### 2.2 CL-EGRSD（因果前瞻变体）

- **问题**：瞬时熵会将持续高熵区间与短暂高熵转折点混为一谈
- **解决方案**：用短时因果前瞻窗口内的最小熵替代瞬时熵
$$\hat{H}_{i,t}^{\text{CL}} = \frac{\min_{j \in [t, \min(t+W, T_i)]} H_{i,j}}{\max_{(j,k) \in C_{\text{batch}}} H_{j,k}}$$
$$\omega_{i,t}^{\text{CL}} = \text{clip}(1 - \gamma \hat{H}_{i,t}^{\text{CL}}, 0.1, 1)$$

- **理论依据**：最小化是满足保守性、保序性、幂等性和因果性的滤波器族中的极值选择，在转折点位置最大化权重恢复，同时保持对持续高熵（fork）位置的抑制。

#### 2.3 三类 token  Regime 区分

| Regime | 特征 | 处理方式 |
|--------|------|----------|
| **Lock** | 低瞬时熵，教师分布尖锐 | 接近满权重通过 |
| **Fork** | 高瞬时熵且持续偏高 | 衰减权重 |
| **Pivot** | 高瞬时熵但短期内迅速转为低熵 | CL-EGRSD 恢复权重 |

## 6. 论文的主要结论与发现

### 核心结论
1. **精度-长度前沿提升**：EGRSD 和 CL-EGRSD 在 Qwen3-4B 和 Qwen3-8B 上均推进了精度-长度 Pareto 前沿，所有可训练基线均被支配（被 Pareto 优超）
2. **Pareto 支配关系**：
   - Qwen3-4B：EGRSD 在准确率和长度上均 Pareto 支配 OPSD
   - Qwen3-8B：CL-EGRSD 取得最高 Avg.（69.32%），长度（12,232 tokens）与 OPSD（12,097 tokens）相当
3. **适度熵衰减最稳定**：γ=0.1 和 γ=0.3 在 8B 上表现最佳，过强（γ≥0.7）或过弱（γ=0）均导致性能下降
4. **前瞻机制的作用**：W=5 在 8B 上效果最好；前瞻在 8B 上更有效（模型更能利用受保护的转折点），在 4B 上一致性较弱

### 关键发现
- **熵与证据可靠性高度相关**：教师-学生 log 概率差距从最低熵十分位的 0.002 nats 增长到最高熵十分位的 0.329 nats（139 倍差异）
- **Pivot 权重重恢复机制**：CL-EGRSD 对 pivot 位置的平均权重恢复（+0.111）是对持续 fork（+0.053）的 2.1 倍，选择性比 2.1:1
- **冻结教师优于在线更新**：EMA 和 Hard Copy  teacher 更新策略均显著差于冻结教师（Avg. 下降 0.4~1.6 个百分点），因为师生差距会趋于零，熵门控失效

## 7. 优点

### 方法论亮点
1. **信号分解优雅**：将 token 级更新显式分解为方向×幅度×置信度三信号，与 RLSD 的框架无缝衔接

5. **最小侵入性**：不改变采样分布或奖励定义，仅增加乘性置信因子

### 实验设计亮点
1. **公平对比**：所有方法共享相同训练数据、步数、trainer 实现
2. **多规模验证**：覆盖 4B 和 8B 两个模型规模

## 8. 不足与局限

- ⚠️ **模型规模**：仅 4B 和 8B，未在更大模型（如 70B+）或不同架构上验证
- ⚠️ **方差估计**：仅 AIME24 报告了多轮运行的方差（5 次），其他 benchmark 可能存在随机波动风险

### 方法局限
- ⚠️ **额外超参数**：引入 γ 和 W（对 CL-EGRSD），需要为不同模型分别调优
- ⚠️ **熵的语义歧义**：教师熵可能反映认知歧义（推理路径不确定性）或 aleatoric 变化（同义表述、词汇粒度欠规范），两者混淆
- ⚠️ **参考 solution 依赖**：机制诊断使用 held-out benchmark 的代理参考 solution 而非原始训练集参考

### 潜在偏差风险
- ⚠️ **Privileged context 敏感性**：方法依赖特权上下文（参考 solution）的质量，若参考 solution 本身不可靠，熵信号可能误导
- ⚠️ **训练数据偏差**：仅使用 OpenThoughts-114k 子集蒸馏 DeepSeek-R1 轨迹，可能存在数据分布偏差

### 实践局限
- ⚠️ **超参数迁移性**：需要为不同基座模型重新调优 γ 和 W，论文未提供自动调整策略
- ⚠️ **前瞻窗口 W 的物理意义**：W=5 tokens 在实践中是经验选择，缺乏理论指导
