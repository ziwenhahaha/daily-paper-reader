## Abstract
Group Relative Policy Optimization (GRPO) has been a key driver of recent progress in reinforcement learning with verifiable rewards (RLVR) for large language models, but it is typically trained in a low-staleness, near-on-policy regime that incurs substantial system overhead. We ask a simple question: How off-policy can GRPO be? We show that GRPO-style algorithms can tolerate substantially larger rollout staleness than previously assumed, and propose Mu-GRPO, an RL training framework that organizes training into a small number (e.g., four) of large sequential generation-optimization stages. This design induces high rollout staleness while greatly reducing rollout-optimization switching overhead. To stabilize learning under stale data, Mu-GRPO combines relaxed clipping, which preserves useful stale-rollout gradients, with negative-advantage veto, which removes destabilizing post-trigger suffix updates in negative-advantage responses. Across five language models and multiple math reasoning benchmarks, Mu-GRPO matches or exceeds the performance of standard GRPO while achieving around 2x speedup in wall-clock training time, establishing a substantially improved performance-efficiency trade-off for LLM reinforcement learning.

### 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：标准 GRPO（Group Relative Policy Optimization）在近在线策略（near-on-policy） regime 下训练，需要频繁刷新 rollout 数据（μ 较小，如 μ=4），导致 rollout 生成与策略优化之间的大量权重同步开销。如何在不损害学习稳定性的前提下，**大幅容忍 rollout 延迟（staleness）**，从而将离策略学习转化为系统效率收益？
- **研究背景**：
  - RLVR（Reinforcement Learning with Verifiable Rewards）已成为大语言模型推理能力提升的关键技术，DeepSeek R1、OpenAI o1 等系统均基于此范式。
  - 标准 GRPO 实现虽被描述为"在线"，但实践中 rollouts 由若干更新前的策略生成，本质上是低延迟的离策略设置。
  - 频繁的 rollout 生成与优化切换导致推理引擎（如 vLLM）与训练引擎（如 FSDP）之间的同步开销高、GPU 内存竞争严重。
- **核心假设**：GRPO 类算法可以容忍远超常规的 rollout 延迟，只要明确处理高延迟优化的失败模式。

### 2. 论文提出的方法论：核心思想与技术细节

#### 核心思想

将训练组织为**少量大型 rollout 生成-优化阶段**（而非频繁小批量切换），通过容忍极端 rollout 延迟换取系统效率提升，并通过负优势否决（Negative-Advantage Veto）和松弛裁剪（Relaxed Clipping）稳定高延迟学习。

#### 关键技术细节

- **μ 定义**：rollout staleness μ = B_train / B_mini，即一轮 rollout 生成支持多少轮模型更新。标准 GRPO μ≈4，μ-GRPO 将 μ 推至 1024（每个阶段 1024 次优化）。
- **松弛裁剪（Relaxed Clipping）**：
  - 将裁剪范围从 [0.8, 1.2] 放松至 [0, 5]。
  - 动机：在高延迟下标准裁剪会过度抑制有用梯度；松弛裁剪允许使用陈旧 rollout 中的信号，但需要配合负优势否决防止崩溃。
- **负优势否决（Negative-Advantage Veto, NAV）**：
  - 对每个负优势 response，检测是否存在 token 的重要性比率 ρ_t(θ) < τ_c（触发条件），τ_c 默认设为 10⁻⁴。
  - 一旦触发，该 response 的所有后续 token 被遮罩（mask=0），不再参与梯度更新。
  - 采用序列级作用域（S_seq）：一旦触发，整个 response 被遮罩。诊断实验表明，真正有害的是**触发后的非触发后缀 H_κ**，而非触发 token 本身。
- **前缀支持失配（Prefix-Support Mismatch）**：
  - 当触发 token 的极低比率进入后续 prefix occupancy ratio R_{t-1}(θ) 时，后缀 token 在当前策略几乎不会产生的 prefix 上被更新，导致局部比率 ρ_t 非微小但整体失效。
  - 定理 1 与推论 1 形式化证明了即使后缀 token 的局部比率 ≥ τ_c，仍可引发任意大的反 Pearson 前缀分歧 χ²(ν_β ∥ ν^{pre}_θ)。
- **两阶段 pipeline**：
  - **Phase 1（Rollout Generation）**：冻结当前策略 π_k 作为 rollout 策略 β_k，批量生成静态 rollout 数据集 D_k = {(x_i, y, A, b_{1:T})}，存储行为对数概率 b_t。
  - **Phase 2（Policy Optimization）**：在 D_k 上执行 μ 次模型更新，使用存储的 b_{1:T} 计算重要性比率 ρ_t(θ)，应用松弛裁剪和 NAV，得到 μ-GRPO 目标函数 J_{μ-GRPO}。

### 3. 实验设计：数据集、benchmark 与对比方法

#### 数据集与场景

- **主实验**：DeepScaleR 数学数据集（用于训练）
- **评估 benchmark**：5 个数学推理基准
  - AMC23、AMC24（美国数学竞赛）
  - AIME24、AIME25（美国数学邀请赛）
  - Math500
- **额外实验**：代码任务（Qwen2.5-Coder-7B 在 code_contests 数据集上训练，LiveCodeBench 评估）

#### 对比方法

- **GRPO (μ=4)**：标准近在线 GRPO，作为基线。
- **GRPO (μ=1024)**：仅增加延迟，不加任何稳定化机制（隔离 μ-GRPO 稳定机制的效果）。
- **M2PO (μ=1024)**：另一高延迟稳定化方法，使用二阶矩阈值过滤高方差 token。
- **μ-GRPO (μ=1024)**：本文方法，默认四阶段（Stage 0~3，每阶段 1024 次更新）。

#### 训练配置

- 所有方法均训练 4096 次模型更新，batch size = 32（保证相同 rollout 采样预算）。
- 使用 verl 框架，vLLM 生成 rollout，FSDP 优化 actor。
- 学习率 1×10⁻⁶，AdamW，reward 为二元（正确=1.0，错误=0.0），temperature=1.0，每个 prompt 采样 8 个 response。

### 6. 论文的主要结论与发现

1. **GRPO 可容忍远大于常规的 rollout 延迟**：在 μ=1024 条件下，GRPO 在数学推理任务上仍可学习，但需要显式稳定化。
2. **高延迟崩溃的根因是负优势后缀漂移**：触发 token 本身危害有限；真正导致崩溃的是触发后被保留的非触发后缀（H_κ），这些 token 在当前策略不支持的 prefix 上被更新。
3. **松弛裁剪 + 负优势否决有效稳定高延迟训练**：两者缺一不可——松弛裁剪暴露信号，NAV 过滤破坏性更新。
4. **μ-GRPO 在多个模型和任务上匹配或超越标准 GRPO**：5 个模型中 4 个取得平均准确率提升或持平，同时 1.53× 加速。
5. **延迟与刷新频率存在下界**：尽管延迟可很大，但单阶段（μ=4096）性能下降；μ≈1024 是最佳平衡点。
6. **效率提升可迁移至代码任务**：在 Qwen2.5-Coder-7B 上实现了 2.16× wall-clock 加速。

### 7. 优点

- **诊断深入**：通过 importance ratio 轨迹分析精确定位到负优势后缀，而非笼统归因于离策略崩溃，为后续研究提供了清晰的问题定位。
- **稳定机制简洁有效**：NAV 只需逐 token 检查 ρ_t < τ_c，无需批量级统计或迭代遮罩，实现轻量且可解释。
- **理论与实验闭环**：形式化证明（前缀支持失配的 χ² 界限）解释了为何 token 级裁剪无法防止后缀崩溃。
- **系统效率转化直接**：大阶段设计将算法耐受性直接转化为同步次数从 1024 次降至 4 次，GPU 内存争用降低。
- **多模型、多任务验证**：覆盖从 3B 到 7B 跨架构模型，数学与代码双任务，证据较充分。
- **异步兼容性**：即使在完全异步设置下，μ-GRPO 也能保持效率优势（trainer idle ratio 从 24.2% 降至 0.6%）。

---

### 8. 不足与局限

- **任务范围有限**：实验主要集中在数学推理和少量代码生成，未覆盖开放式生成、对话、工具调用等更复杂场景。
- **最佳 μ 值依赖经验调优**：虽然固定优化预算下 μ=1024 最优，但这仍是超参数，不同模型或任务可能需要不同设置。
- **NAV veto 了部分数据**：即使 veto token 比例不高（平均 0.17%~4.03%），在被判断为跨越 off-support boundary 的 response 上可能存在有用的中间 token 被误删，尤其在更复杂推理链中。
- **未探索更大模型**：最大模型为 8B（Qwen3-Base-8B），未在 70B+ 规模上验证。
- **固定 prompt set**：每阶段使用固定训练集 X_k，策略更新期间无数据新鲜度注入，理论上可能在极长训练周期下导致数据分布偏移。
- **评估频率**：每 200 步评估一次，可能错过训练早期的剧烈波动信息（如 Figure 2 中崩溃过程）。
- **异步实验规模有限**：仅使用 2 设备生成 + 2 设备训练的配置，未验证更大并行度下的效率表现。
