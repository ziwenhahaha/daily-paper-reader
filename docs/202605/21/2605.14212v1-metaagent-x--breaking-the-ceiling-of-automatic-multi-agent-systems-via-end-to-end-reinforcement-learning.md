## 摘要
自动多智能体系统旨在无需依赖人工设计或固定编排的情况下实例化智能体工作流。然而，现有的自动多智能体系统方法仍然仅具备部分自适应能力：它们要么在测试时进行无需训练的搜索，要么在保持下游执行智能体冻结的情况下优化元层设计器，从而产生了冻结执行器上限，并导致自设计、自执行智能体模型的端到端训练仍未被探索。为解决此问题，我们提出了 MetaAgent-X，一个端到端强化学习框架，联合优化自动多智能体系统的设计与执行。MetaAgent-X 支持基于脚本的多智能体系统生成、执行轨迹收集以及设计者和执行者轨迹的信用分配。为支持稳定且可扩展的优化，我们提出了执行者-设计者分层 rollout 和阶段性协同演化，以提升训练稳定性并揭示设计者‑执行者协同演化的动态特性。MetaAgent-X 在现有自动多智能体系统基线上持续取得更优性能，实现高达 21.7% 的提升。全面的消融实验表明，设计者和执行者在整个训练过程中均有所提升，并且有效的自动多智能体系统学习遵循阶段性协同演化过程。这些结果表明，可端到端训练的自动多智能体系统是构建自设计、自执行智能体模型的实用范式。

## Abstract
Automatic multi-agent systems aim to instantiate agent workflows without relying on manually designed or fixed orchestration. However, existing automatic MAS approaches remain only partially adaptive: they either perform training-free test-time search or optimize the meta-level designer while keeping downstream execution agents frozen, which creating a frozen-executor ceiling and leaving the end-to-end training of self-designing and self-executing agentic models unexplored. To address this, we introduce MetaAgent-X, an end-to-end reinforcement learning framework that jointly optimizes automatic MAS design and execution. MetaAgent-X enables script-based MAS generation, execution rollout collection, and credit assignment for both designer and executor trajectories. To support stable and scalable optimization, we propose Executor Designer Hierarchical Rollout and Stagewise Co-evolution to improve training stability and expose the dynamics of designer-executor co-evolution. MetaAgent-X consistently outperforms existing automatic MAS baselines, achieving up to 21.7% gains. Comprehensive ablations show that both designer and executor improve throughout training, and that effective automatic MAS learning follows a stagewise co-evolution process. These results establish end-to-end trainable automatic MAS as a practical paradigm for building self-designing and self-executing agentic models.

## 1. 论文的核心问题与整体含义

### 研究背景
- 多智能体系统（MAS）在医疗决策、科学发现、金融交易、软件工程等领域展现出超越单智能体的优势
- 传统的多智能体系统依赖人工设计的固定工作流，缺乏自适应能力
- 元智能体（meta-agent）能够自动为每个任务设计合适的多智能体系统结构

### 核心问题
**"冻结执行器天花板"（Frozen-Executor Ceiling）**：现有自动MAS方法仅具备**部分自适应能力**，存在两大根本性限制：

1. **参数级别脱节**：设计者与执行者仅通过提示级交互耦合，缺少基于下游执行结果的优化信号，导致冻结的执行器对元设计器形成硬性上限

2. **协同演化动态不清晰**：设计者和执行者在联合训练下的演化机制不明确

### 研究动机
突破现有方法中"仅优化设计者而冻结执行者"的局限，探索**端到端可训练的自动MAS**，实现自设计和自执行的智能体模型。

## 2. 论文提出的方法论

### 核心思想
MetaAgent-X 是一个**端到端强化学习框架**，联合优化自动多智能体系统的设计与执行两端，使设计者（Designer）和执行者（Executor）能够协同演化。

### 关键技术组件

#### 2.1 在线元智能体RL流程
- **Designer策略** π<sub>D</sub><sup>ϑ<sub>D</sub></sup>：根据任务查询 q 生成任务特定的多智能体系统设计 d
- **Executor策略** π<sub>E</sub><sup>ϑ<sub>E</sub></sup>：在外部环境中执行生成的多智能体系统
- 完整参数集：ϑ = {ϑ<sub>D</sub>, ϑ<sub>E</sub>}，支持共享策略或分离策略设置

#### 2.2 执行者-设计者分层Rollout（Executor-Designer Hierarchical Rollout）

**树状结构Rollout**：
- **第一层**：设计者为每个问题生成 M 个独立的MAS设计 {d₁, d₂, ..., d_M}
- **第二层**：对每个设计 dᵢ，执行 N 次独立 rollout {e<sub>i,1</sub>, e<sub,i,2</sub>, ..., e<sub>i,N</sub>}
- 结果：每个问题形成 M×N 的评估矩阵

**分解式优势估计**：

- **设计者优势**：对设计 dᵢ 的奖励取 N 次执行的均值
  - `¯R_D_i = (1/N) Σ R(e_{i,j}, d_i)`
  - 优势归一化：`Â_D_i = (¯R_D_i - μ_D_q) / (σ_D_q + ε)`
  
- **执行者优势**：收集同一问题下所有执行轨迹
  - `G_E_q = {τ | τ ∈ T_{i,j}, i ∈ [M], j ∈ [N]}`
  - 优势：`Â_E(τ) = (R(e_{i,j}, d_i) - μ_E_q) / (σ_E_q + ε)`

#### 2.3 阶段性协同演化（Stagewise Co-evolution）

交替优化策略，每 K 步切换活跃角色：
```
(α(t)_D, α(t)_E) = 
  (0, 1) 如果 ⌊t/K⌋ mod 2 = 0 → Executor阶段
  (1, 0) 如果 ⌊t/K⌋ mod 2 = 1 → Designer阶段
```

- 仅活跃角色的轨迹贡献梯度
- 共享参数持续更新
- 通过阶段性隔离减少梯度干扰

#### 2.4 GRPO优化目标

使用 Group Relative Policy Optimization，对每个角色 r：
```
L_r(ϑ_r) = -(1/|G_r|) Σ_{i∈G_r} min(
  ρ_r^i(ϑ_r) Â_r^i,
  clip(ρ_r^i(ϑ_r), 1-ε, 1+ε) Â_r^i
)
```
其中 ρ_r^i 是当前策略与旧策略的比值。

## 6. 论文的主要结论与发现

### 主要性能提升
- MetaAgent-X RL 在 Qwen3-8B 上平均提升 **+11.17%**（38.33% vs 27.16% 基线）
- 在 Qwen3-4B 上平均提升 **+12.80%**（34.18% vs 21.38% 基线）
- AIME24 提升 **+21.7%**（40.0% vs 18.3% SA基线）

### 关键发现

1. **层级Rollout优于扁平Rollout**：M=4, N=4 配置比 M=8, N=1 性能更优，说明重复执行同一设计能提供更可靠的下游效用估计

2. **阶段性协同演化优于耦合训练**：
   - 耦合训练快速提升但后期崩溃
   - 仅优化执行者快速饱和
   - 仅优化设计者提升有限
   - 阶段性训练呈现清晰的阶梯式学习曲线

3. **共享策略优于分离策略**：共享参数允许训练信号从一个角色迁移到另一个角色，改善泛化能力和数据效率

4. **结构选择具有任务依赖性**：
   - 难题（如AIME）：70%+选择Reflection结构
   - 简单任务（如APPS）：更倾向Single结构
   - 竞赛级任务：Ensemble+judge更受青睐

5. **RL同时提升设计者和执行者**：
   - 50%的改进来自执行者在相同结构下更好地解决问题
   - 50%来自设计者选择更优结构
