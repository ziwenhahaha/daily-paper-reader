## 摘要
DiPRL通过架构熵正则化实现训练中逐步离散化，避免事后离散化导致的不可恢复性能崩塌。

## Abstract
Programmatic reinforcement learning (PRL) offers an interpretable alternative to deep reinforcement learning by representing policies as human-readable and -editable programs. While gradient-based methods have been developed to optimize continuous relaxations of programs, they face a significant performance drop when converting the continuous relaxations back into discrete programs. Post-hoc discretization can discard optimized branches and parameters in a program, which results in a collapse of policy expressivity and lowered task performance, leading in turn to a need for additional fine-tuning. To overcome these limitations, we propose Differentiable Discrete Programmatic Reinforcement Learning (DiPRL), a method that learns programmatic policies that become nearly discrete during training, avoiding a separate post-hoc fine-tuning stage. We first analyze the inherent risks of performance drop introduced by post-hoc discretization of gradient-based methods. Then, we introduce programmatic architecture entropy regularization, which enables smooth, differentiable training that encourages convergence toward a discrete program. DiPRL maintains the efficiency of gradient-based optimization while mitigating the risks of post-hoc discretization. Our experiments across multiple discrete and continuous RL tasks demonstrate that DiPRL can achieve strong performance via interpretable programmatic policies.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 2.1 核心思想

提出 **DiPRL（Differentiable Discrete Programmatic Reinforcement Learning）**，在不引入单独事后微调阶段的前提下，让程序策略在训练过程中**逐步趋向离散**，从而从根本上消除事后离散化带来的性能崩塌。

### 2.2 技术细节

#### DSL（领域特定语言）

程序策略使用上下文无关文法定义：
```
Program E := A | if B then A else E
Condition B := ϕ_w(s) > 0
```
其中 `A` 是基础动作，`B` 是由参数化线性谓词 `ϕ_w(s)` 决定的条件分支。程序从根节点开始递归展开，直到达到最大深度 `D_m`。

#### 连续松弛推导树

沿用 π-PRL 的松弛方式，将程序松弛为可微推导树 `T = {V, E}`：
- **架构概率** `P_T(v_j | v_i)`：与状态无关的展开概率，定义程序路径的先验分布。
- **路径概率** `P_T(ζ) = ∏_{k=0}^{n-1} P_T(v_{k+1} | v_k)`，其中 `ζ` 为从根到终端的完整程序路径。
- **谓词松弛**：将硬分支指示器 `1[ϕ_w(s) > 0]` 松弛为 Sigmoid 函数 `σ(ϕ_w(s))`。
- **路径诱导的动作分布**（式1）：
  ```
  π_{ζ,W}(a|s) = (∏_{⟨vi,vj⟩∈ζ} p_{ij}(s)) · π_{w_ζ}(a|s)
  ```
- **松弛推导树策略**（式2）：
  ```
  π_T^W(a|s) = ∑_{ζ∈P} P_T(ζ) · π_{ζ,W}(a|s)
  ```

#### 程序架构熵正则化（核心创新）

- **程序架构熵**：量化程序架构的不确定性。
  - 定义程序深度随机变量 `D`，概率 `p_d = Pr(D = d)`。
  - 深度基架构熵（式3）：
    ```
    H(T) = -∑_{d=1}^{D_m} p_d log p_d
    ```
  - 在本文 DSL 下，深度唯一确定程序路径，因此该熵等于通用的路径熵 `Ĥ(T)`。
- **正则化目标**（式4-5）：将最大熵约束引入策略梯度目标：
  ```
  max_{T,W} J(T,W)  s.t. H(T) ≤ H̄
  L(T, W, α) = E_τ[∑_{t=0}^T log π_T^W(a_t|s_t) Â_t] - α(H(T) - H̄)
  ```
  其中 `α ≥ 0` 是正则化系数，`Ĥ_t` 是优势估计。

#### 自动系数调节

固定 `α` 在不同任务间表现不一致。本文通过 **自动调节** `α`（优化 `β = log α`）实现：梯度为 `∇_α g(α) = H̄ - H(T)`——当架构熵高于目标时增大 `α`，低于目标时减小 `α`。这使得训练早期保持架构探索灵活性，后期逐步收紧到离散程序。

## 6. 论文的主要结论与发现

1. **事后离散化是 π-PRL 的关键失效模式**：在训练结束后突然从连续推导树切换到离散程序会丢弃有用分支，导致不可恢复的性能下降。即使后续微调也无法弥补。
2. **DiPRL 成功实现训练中逐步离散化**：通过架构熵正则化，DiPRL 在整个训练过程中逐渐将程序架构熵推向零（最低可达 0.000），最终提取的离散程序与训练中的松弛策略高度一致。
3. **DiPRL 在离散任务上表现优异**：
   - CartPole-v1 达到满分 500。
   - DoorKey 达到 0.95（远超 π-PRL 的 0.66）。
   - MountainCar-v0 达到 -110.79（优于 π-PRL 的 -171.83 和 π_disc.-PRL 的 -178.58）。
4. **DiPRL 在连续任务上显著优于 π-PRL**：
   - HalfCheetah Hurdle：DiPRL 723.88 vs π-PRL -443.80。
   - Ant RandomGoal：DiPRL 413.12 vs π-PRL 264.75。
   - Ant CrossMaze：DiPRL 220.25 vs π-PRL -223.58。
5. **自动调节正则化系数是关键**：固定 `α` 在不同任务间表现不稳定，自动调节 `α` 在所有任务上保持鲁棒。
6. **DiPRL 发现更深的程序**：由于避免了分支被强制丢弃，DiPRL 能探索更深的程序架构（如 Acrobot-v1：DiPRL 深度 4.33 vs π-PRL 深度 2.33）。

## 7. 优点

1. **问题定位精准**：首次系统性地揭示并理论分析了 π-PRL 事后离散化导致不可恢复性能下降的根本原因（被删路径的概率质量 `m_T` 与路径值差距的乘积）。
2. **方法简洁有效**：仅通过在策略梯度目标中加入一项架构熵正则化，就消除了对单独微调阶段的需求，简化了训练流程。
3. **理论支撑**：提供了完整的形式化分析（附录 B），推导出事后离散化性能上界，并证明架构熵正则化如何从理论上降低性能崩塌风险。
4. **自动调参鲁棒性**：自动调节 `α` 的设计避免了手工调参的痛苦，使方法在不同任务间具有良好的通用性。
5. **实验全面**：离散+连续 9 个任务，与多种基线（PPO、决策树类方法、π-PRL 各阶段）对比，消融研究涵盖两个关键超参数。
6. **可解释性强**：DiPRL 输出的离散程序如 `if (-0.16 - 0.31·f0 - 2.1·f1 - 6.6·f2 - 2.3·f3 > 0): Left else Right`（CartPole-v1），人类可直接理解和编辑。

## 8. 不足与局限

1. **DSL 表达能力有限**：DiPRL 仅支持 if-else 控制流和线性谓词，无法自然地表示循环、函数调用或时序操作符。对于需要重复行为或长时序规划的任务，最优策略可能超出 DSL 的表达能力。
2. **解释性未经验证**：论文未进行人类中心评估（如人类能否理解程序、程序是否可调试），仅声称程序具有可读性。
3. **理论假设的局限性**：理论分析中的值差距上界 `Δ` 是假设的均匀上界，实际情况中 `Δ` 可能因状态分布而变化，分析的实用性有限。
4. **算力细节不透明**：未报告总训练时长，无法评估样本效率（虽然展示了训练曲线但未明确每步耗时）。
5. **自动化调参的额外开销**：自动调节 `α` 需要维护额外的对偶变量更新，虽然论文声称开销不大（学习率 10⁻⁴），但增加了实现复杂度。
6. **部分任务性能未显著超越基线**：在 Pusher2D 等任务上，DiPRL 与 π-PRL 及 VIPER 性能接近，说明在简单任务上优势不明显。
7. **程序分支的探索受限于最大深度**：`D_m = 6` 是预设的，无法自动发现更深层结构的需求。
