## Abstract
Long-horizon multimodal agents in open-world games must stay goal-directed across many low-level interactions under tight token and latency budgets. Existing approaches often trade off costly per-step reasoning against reactive execution that can drift, repeat failures, and recover poorly. Our key idea is to reuse strategic reasoning across locally stable segments and reinvoke it at event boundaries. We present SPIKE, an adaptive dual controller framework for cost-efficient long-horizon game control. Its Strategic Controller performs low-frequency global planning, failure analysis, and recovery, while its Reactive Controller handles fast local execution under a strict token budget. An Event Trigger monitors visual change, task progress, repeated actions, and failure signals to decide when control should stay reactive or escalate to strategic reasoning. Hierarchical Memory separates short-term experience reuse in the State-Action Memory Bank (SA-MB) from structured evidence in the State Action Knowledge Graph (SA-KG), allowing each controller to retrieve the context it needs. This design reuses strategic proposals over multiple reactive steps, supports local override when plans become stale, and reserves expensive reasoning for moments where extra deliberation is useful. On the Lite-100 split of StarDojo, SPIKE improves Lite-100 success rate (SR) by 5.0 percentage points (38.5% relative) over the strongest Lite-100 baseline and Budgeted SR by 9.3 points (75.6% relative) over the strongest budgeted baseline. It also reduces token consumption by 54.9% and latency by 40.8%. Ablations show that event triggering, reactive override, and heterogeneous memory each contribute to success and recovery, supporting selective reasoning rather than reasoning at every step.

# SPIKE: 自适应双控制器框架论文详解

## 1. 论文核心问题与整体含义

**研究背景**：开放世界游戏中的长程多模态智能体面临"规划-延迟-记忆三难困境"：

- 每步进行深度推理会大幅增加计算成本和延迟
- 纯响应式执行虽然快速，但在状态漂移、执行失败、重复行为和恢复方面表现不佳
- 传统的记忆复用引入检索噪声，削弱跨episode的复用效果

**核心问题**：如何在高昂推理成本与不稳定执行质量之间找到平衡，使得智能体在长程任务中既保持目标导向又满足令牌和延迟预算约束？

**研究动机**：现有方法（如CRADLE等规划密集型框架）虽能有效推理，但高频调用大模型导致成本过高；而ReAct等响应式智能体执行成本低但缺乏可靠性。实际场景中大多数步骤处于稳定的本地执行状态，仅在少量边界处需要恢复和重新规划。

## 2. 论文方法论

### 核心思想

SPIKE将战略推理成本分摊到局部稳定的片段上，仅在事件边界处重新调用规划。核心假设：**长程游戏控制的时间分布不均匀——大部分步骤在稳定本地执行，少量边界主导可恢复的失败**。

### 关键技术组件

#### 2.1 自适应双控制器架构

**战略控制器（Strategic Controller）**：

- 激活条件：周期性刷新、卡住状态、恢复信号
- 工作流程：信息收集 → 自反思（可选）→ 任务推理 → 动作提议
- 输出：针对下T步的短期提议（足够指导本地执行但足够紧凑可在场景变化后修订）

**响应控制器（Reactive Controller）**：

- 在严格令牌预算下处理常规步骤
- 使用最新战略提议、当前观察和SA-MB记忆提示发出本地动作
- 当新证据使提议失效时，可进行有限本地修正（回避、重新定位、工具重选）
- 不能改变当前子目标，失败的修正则将控制权交回事件触发器

#### 2.2 事件触发器（Event Trigger）

将预算目标操作化为在线事件边界决策。理想情况下每步选择战略推理的期望效用是否超过其额外成本：

$$g^*_t = \arg\max_{g\in\{0,1\}} \quad \tilde{U}(g | h_t) - \lambda C_t(g, h_t)$$

实际使用可复现的替代事件检测器：

$$g_t = \mathbb{I}[(c_t \geq T) \lor (d_t > \tau_v) \lor (z_t \geq \tau_z) \lor (r_t \geq \tau_r \land z_t \geq \tau_{rz}) \lor (\ell_t \geq \tau_\ell)]$$

检测信号包括：

- $d_t = 1 - \cos(\phi(v_t), \phi(v_{t-1}))$：视觉不连续检测
- $z_t$：进度停滞检测
- $r_t$：行为循环检测
- $\ell_t \in \{0,1,2,3\}$：执行失败级别

#### 2.3 层级记忆（Hierarchical Memory）

**状态-动作记忆库（SA-MB）**：低延迟本地动作先验，用于响应控制器执行

检索评分公式：
$$S_{MB}(i|x_t) = \rho_i(\alpha_C C_i + \alpha_L L_i + \alpha_R R_i + \alpha_P P_i)$$

其中：

- $L_i$：Jaccard重叠
- $C_i$：令牌频率余弦相似度
- $R_i$：奖励指数移动平均
- $P_i$：成功/尝试比率
- $\rho_i$：时间衰减因子

**状态-动作知识图（SA-KG）**：战略证据图，用于检查和重新规划

检索评分公式：
$$S_{KG}(j|x_t) = \beta_C C_j + \beta_P P_j$$

其中 $C_j$ 是向量相似度，$P_j$ 是最佳输出动作成功率。

**融合检索**：$$\tilde{S}_{fuse} = \lambda_{MB} \tilde{S}_{MB} + \lambda_{KG} \tilde{S}_{KG}$$

### 2.4 算法流程

```
Algorithm: Event-triggered amortized deliberation in SPIKE
Input: task x, thresholds Θ, SA-MB, SA-KG, budget B
Initialize: proposal p←∅, history h0←∅
for t=1,...,H do
    Observe ot=(vt,ut,ft) and update history context
    Compute visual change dt, stall streak zt, repetition tail rt, failure level ℓt
    Set gt←EventTrigger(dt,zt,rt,ℓt,ct;Θ)
    if gt=1 then
        retrieve SA-KG/global evidence
        update proposal p with Strategic Controller
    else
        retrieve local hints from SA-MB
    Emit at with Reactive Controller using ot,p,retrieved hints
    Execute at, receive feedback
    write memory only on success, interpretable recovery, or planner-validated progress
end for
```

## 7. 方法论亮点

1. **成本意识的设计理念**：将战略推理视为稀缺资源，仅在必要时分配，而非均匀分配到每一步

2. **事件边界检测的可操作化**：将抽象的预算约束转化为可观测的事件信号（视觉变化、停滞、重复、失败）

3. **控制器的自适应角色分配**：响应控制器不仅是消极执行器，而是具有本地覆盖能力的决策者

4. **记忆的职责分离**：SA-MB和SA-KG分别服务于不同的控制器，避免共享检索上下文带来的噪声

5. **保守的写入策略**：仅在成功、可解释的恢复或规划器验证后才写入记忆，防止瞬态错误成为长期证据

6. **多粒度的失败检测**：F1（本地重试）、F2（硬错误升级）、F3（计划失效）三级失败分级提供分级恢复能力

## 8. 不足与局限

### 8.1 实验覆盖局限

- **RDR2任务数较少**：仅13个任务，多样性有限
- **单一游戏测试**：主要在StarDojo验证，泛化到其他游戏风格的能力需进一步验证
- **战斗和社会任务表现不佳**：最难的类别（战斗8.3%、社会10.7%）成功率仍然较低

### 8.2 方法局限

- **视觉子滤问题的脆弱性**：当进度在视觉上微妙、失败信号嘈杂、或游戏状态变化而屏幕差异不大时，可能错误路由控制
- **记忆陈旧风险**：检索到的记忆可能过时，尤其在动态环境中
- **事件触发器阈值固定**：需要针对不同游戏类型重新校准

### 8.3 应用限制

- **未利用音频线索**：当前系统仅使用视觉观察和动作历史
- **仍依赖显式模型调用**：恢复和重新规划仍需大模型调用，成本和延迟仍是实际部署约束
- **更多移动组件**：相比单控制器基线，引入了更多需要调优的参数（事件触发器阈值、检索权重等）

- **硬件依赖**：结果可能依赖于特定的GPU架构和服务配置
