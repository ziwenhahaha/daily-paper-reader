## Abstract
Latent steering exploits internal representations of Large Language Models (LLMs) to guide generation, yet interventions on dense states can entangle distinct semantic features. In this paper, we investigate attention query activations as a high-fidelity site for precise control, hypothesizing that manipulating the attention mechanism itself offers sharper steerability than general state interventions. We introduce Prototype-Based Sparse Steering, a framework that applies Sparse Autoencoders (SAEs) specifically to query activations, to decompose them into interpretable features, then apply gradient-based optimization during inference to align the sparse representation with class prototypes of target behaviors. To validate this architectural insight, we first analyze the mechanism in Textualized Gridworld, a controlled environment for verifiable planning constraints. We demonstrate that optimizing sparse query features enables effective navigation of rigid planning requirements (i.e., safe vs. short paths), confirming the method's ability to satisfy objective rules. We then demonstrate the framework's versatility by training SAEs on a high-dimensional educational domain, where the framework steers the cognitive complexity of feedback (i.e., Bloom's Taxonomy). Our experiments establish that sparse query representations provide the necessary disentanglement for unified, interpretable control over both logical planning and stylistic nuance.

## 1. 论文的核心问题与整体含义（研究动机和背景）

### 研究动机
- **LLM 控制生成的挑战**：大语言模型（LLM）虽能生成流畅文本，但在实际部署中需要精确满足用户定义的要求（风格、安全性、逻辑一致性），仅靠提示工程（prompt engineering）往往效果有限
- **提示方法的局限性**：长提示上下文中控制指令容易被稀释或忽略；小模型和开源模型仅依赖提示难以实现稳定控制；增加示例会导致显著的内存消耗和计算开销
- **激活干预的潜力**：潜在空间引导（latent steering）通过直接干预模型内部激活实现控制，避免了提示扩展的负担
- **密集激活的问题**：密集激活空间中多个潜在因素相互纠缠（superposition hypothesis），导致干预不够精确，容易产生副作用

### 核心问题
如何在推理时（inference-time）对 LLM 进行精确、可控的生成引导，同时保持：
- 目标属性的精确对齐
- 任务相关内容的完整性
- 生成文本的连贯性

## 2. 论文提出的方法论

### 核心思想
论文提出 **Prototype-Based Sparse Steering** 框架，核心思想是：

1. **选择注意力查询激活（Query Activations）作为干预位点**：相比残差流（residual stream），查询激活更局部化，扰动影响范围更小
2. **使用稀疏自编码器（SAE）分解查询激活**：将高维、纠缠的查询表示分解为可解释的稀疏特征
3. **基于梯度的原型优化**：在推理时通过梯度上升优化稀疏表示，使其与目标行为类原型对齐

### 关键技术细节

#### 2.2.1 属性条件引导目标（Attribute-Conditioned Steering Objective）

- **原型表示**：对于每个属性类 $k$，定义支持集 $D_k = \{((x_i, y_i), a_i) : a_i = k\}$，原型中心为：
$$c_k = \frac{1}{|D_k|} \sum_{((x_i,y_i),a_i) \in D_k} \text{Enc}(\text{LLM}_{\leq \ell}(x_i, y_i))$$

- **度量分布**：使用负距离的 softmax 定义属性类的概率分布：
$$P_\phi(a = k_T | z) = \frac{\exp(-d_{k_T})}{\sum_{j=1}^{K} \exp(-d_j)}$$

- **梯度上升优化**：
$$z^{(t+1)} = z^{(t)} + \eta \nabla_z \log P_\phi(a = k_T | z^{(t)})$$

#### 2.2.2 查询层级干预（Query-Level Intervention）

- **局部性优势**：查询激活影响注意力权重，产生的是值的凸组合，而非像残差流干预那样直接覆盖累积状态
- ** locality 诊断**：通过相对激活偏差 $\delta_\ell$ 和 Jensen-Shannon 散度（JSD）量化干预的局部性

#### 2.2.3 稀疏自编码器训练

SAE 架构为单层 ReLU 自编码器：
- **编码器**：$\text{Enc}(s) = \text{ReLU}(W_e s + b_e)$
- **解码器**：$\text{Dec}(z) = \tilde{W}_d z$（每列归一化）
- **训练损失**：
$$L(s) = \|\hat{s} - s\|_2^2 + \lambda \|z\|_1 + \beta \|b_e\|_2^2$$

### 算法流程（Algorithm 1）

1. 计算每个属性类的原型中心 $c_k$
2. 获取初始模型输出 $y'$
3. 初始化稀疏表示 $z^{(0)} = \text{Enc}(\text{LLM}_{\leq \ell}(x, y'))$
4. 迭代优化直到收敛：
   - 计算到各原型的距离
   - 计算目标原型概率
   - 执行梯度上升更新
5. 解码并继续生成

## 6. 论文的主要结论与发现

### 6.1 查询层级干预的优越性
- **局部性**：查询干预产生有界且缓慢变化的跨层激活偏差，而残差流干预导致 1-2 个数量级的更大持续偏差
- **分布成本**：查询层级干预在相同控制强度下实现更低的任务成功-JSD 权衡

### 6.2 硬约束规划（TGW）
- **SAE-opt 表现**：在大多数设置中达到最高或并列最高的成功率，同时保持最低的违规率
- **具体提升**：Qwen 上 Safe 目标从 3.7%（ICL）提升到 15.2%
- **Long 目标**：SAE-opt 在所有方法中达到最高成功率

### 6.3 软语义控制（Bloom's Taxonomy）
- **SAE-opt 表现**：在两个 judge 下均达到最高平均命中率（GPT-4o: 25.4%，Claude: 27.1%）
- **重大增益**：在 "Remember" 和 "Create" 等困难层级上显著改进
- **相关性保持**：query 层干预在重定向认知风格时不破坏输入代码的关联性

### 6.4 梯度优化 vs 静态向量添加
- **Dense-opt 优势**：在 TruthfulQA 上实现最高 %T×I，同时将平均 JSD 降低一个数量级
- **最坏情况**：静态方法 JSD 峰值超过 0.13-0.28，Dense-opt 保持在 0.03 以下

### 6.5 稀疏性的作用
- **L1 正则化优势**：相比 L2，L1 在所有模型和目标类上产生更低的非目标漂移
- **最大差距**：L2 在较大系数下导致多达 4.9 倍的非目标漂移

### 6.6 层级选择
- **中间层最有效**：无论 TGW 还是认知风格任务，中间层的稀疏查询特征产生最可靠和有效的引导

## 8. 不足与局限

### 8.1 实验覆盖局限
1. **模型规模**：仅测试了 4B-8B 参数模型，未验证在更大模型（如 70B+）上的效果
2. **领域覆盖**：主要集中在编程教育和 Gridworld，对自然对话、创意写作等场景覆盖不足
3. **多属性控制**：未验证同时控制多个属性的可行性

### 8.2 潜在偏差风险
1. **原型质量依赖**：原型的质量高度依赖于支持集的例子数量和质量
2. **模型固有偏置**：模型对 "Evaluate" 层级有强烈固有偏置，可能限制某些控制目标的可达性
3. **Judge 可靠性**：依赖于 LLM-as-Judge 的评估，可能存在系统性偏差

### 8.3 应用限制
1. **推理时开销**：每次推理都需要迭代优化，增加了生成延迟
2. **SAE 训练成本**：需要为每个模型单独训练 SAE，开销较大
3. **层级敏感性**：中间层最有效的结论可能不是通用的，需要针对新模型重新调优
4. **长文本场景**：未测试在极长上下文（>10K tokens）场景下的效率和有效性

### 8.4 理论分析不足
1. **最优性保证**：缺乏对优化收敛到全局/局部最优的理论保证
2. **泛化边界**：未明确说明方法在不同任务类型间的泛化条件和限制
