## 摘要
大语言模型在强化学习训练中面临token级信用分配瓶颈，标准GRPO算法对所有token均匀应用序列级奖励导致训练效率低下。AMR-SD提出不对称元反射自蒸馏机制，通过将诊断信号压缩为自生成的Socratic提示和批评，并结合因果信息增益实现稀疏的token级优势调制，有效解决了答案泄漏和训练崩溃问题。

## Abstract
The alignment of Large Language Models (LLMs) for complex reasoning heavily relies on Reinforcement Learning with Verifiable Rewards (RLVR). However, standard algorithms like GRPO apply sequence-level rewards uniformly to all tokens, creating a severe credit-assignment bottleneck. While on-policy self-distillation attempts to resolve this by conditioning a self-teacher on privileged contexts, direct exposure to raw oracle solutions often induces over-conditioned teacher distributions, implicit answer leakage, and late-stage training collapse. To overcome these limitations, we propose Asymmetric Meta-Reflective Self-Distillation (AMR-SD). Instead of conditioning directly on raw reference traces, AMR-SD inserts a reflection bottleneck: it compresses diagnostic signals -- from verifier outcomes, peer rollouts, or reference feedback -- into concise, self-generated Socratic hints and critiques. Furthermore, we introduce Causal Information Gain (CIG) with an asymmetric, ReLU-gated threshold to translate these reflections into sparse, highly precise token-level advantage modulations. Combined with temporal annealing, this mechanism preserves the base environmental reward while filtering out distributional noise. Experiments across scientific, mathematical, and tool-use benchmarks demonstrate that AMR-SD significantly outperforms existing baselines, achieving robust long-horizon stability and successfully preventing late-stage collapse.

## 2. 论文提出的方法论

### 核心思想
AMR-SD（Asymmetric Meta-Reflective Self-Distillation）提出**反射中介的自训练循环**，不直接让教师看到原始参考答案，而是通过三个递进阶段实现精确的 token 级信用分配：

1. **Meta-Reflection 阶段**：将诊断信号压缩为自生成的 Socratic 提示（hint）和批评（critique）
2. **CIG 量化阶段**：计算教师与学生策略之间的因果信息增益
3. **非对称调制阶段**：通过 ReLU 门控阈值实现稀疏、方向性干预

### 关键技术细节

#### 2.1 Socratic Rescoring via Meta-Reflection
- 定义离散条件映射函数：`ci = freflect(x, yi, sgn(Ai))`
  - 成功轨迹（Ai ≥ 0）：生成鼓励性 hint
  - 失败轨迹（Ai < 0）：生成针对性 critique
- 使用 peer rollouts（同一 prompt 下验证通过的轨迹）作为批评生成的参考
- 通过 stop-gradient 操作将学生模型转变为条件 Socratic 评估器

#### 2.2 Causal Information Gain（CIG）
- **信息增益公式**：`ICIG = log(πθsg(at|st, ci) / πθ(at|st))`
- **有界对称截断**：`bICIG = clip(ICIG, -κ, κ)`，其中 κ = 5 防止数值不稳定
- 正值表示学生知识缺陷，负值表示学生过度自信

#### 2.3 Asymmetric Token-Level Advantage Modulation
- **非对称调制标量**：
  ```
  Δt = I{Ai≥0} · λeff · max(0, bICIG - τ) + I{Ai<0} · γeff · max(0, -bICIG - τ)
  ```
- **ReLU 门控阈值 τ**：仅在置信度超过阈值时触发调整，过滤分布噪声
- **精细化 token 级优势**：`bAi,t = Ai · (1 + Δt)`
- **参数设置**：λ = 0.2，γ = 0.1（因负向 CIG 分布更重，故 λ > γ 补偿）

#### 2.4 Temporal Annealing
- 线性退火公式：`{λeff, γeff} = {λ, γ} · max(0, 1 - tglobal/Tdecay)`
- 作用：训练后期平滑过渡至纯环境奖励，防止学习退化信号

#### 2.5 优化目标
- 修改版 PPO clipped surrogate objective
- 替换标准轨迹优势为密集估计器 `bAi,t`
- 使用 importance sampling ratio ρt(θ) 进行策略更新

## 8. 不足与局限

### 8.1 依赖反思质量
- **核心限制**：CIG 机制有效性依赖模型自生成的 hint/critique 质量
- **能力门槛**：弱模型（如 enable_thinking=False 的 Qwen3-8B）反思质量不足，无法超越 GRPO
- **适用场景**：需要具备强 Chain-of-Thought 或自评估能力的基础模型

### 8.2 推理开销
- **额外生成步骤**：每个轨迹需额外自回归生成反思文本
- **预算限制**：虽设硬性上限（4096/16384 tokens），但仍增加约 22% wall-clock 时间
- **资源受限场景**：对延迟敏感的应用可能存在实际部署障碍

### 8.3 潜在偏差风险
- **反射偏差传播**：若 hint/critique 存在错误或片面性，错误监督信号可能通过 CIG 调制传播
- **CIG 分布依赖**：不对称参数（λ > γ）基于经验分布设计，跨任务泛化性需进一步验证
- **阈值敏感性**：τ 和 κ 的最优值依赖于实验观察，理论推导较弱

### 8.4 评估覆盖局限
- **领域集中**：主要在科学、数学推理验证，缺乏代码生成、对话等更多实际应用场景
- **模型规模**：仅在 7B/8B 模型验证，LLM 缩放规律未探索
- **离线评估**：无人类评估或实际用户反馈，主观推理质量未被直接衡量
