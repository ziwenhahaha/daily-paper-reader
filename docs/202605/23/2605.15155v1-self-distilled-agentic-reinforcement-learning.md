## Abstract
Reinforcement learning (RL) has emerged as a central paradigm for post-training LLM agents, yet its trajectory-level reward signal provides only coarse supervision for long-horizon interaction. On-Policy Self-Distillation (OPSD) complements RL by introducing dense token-level guidance from a teacher branch augmented with privileged context. However, transferring OPSD to multi-turn agents proves problematic: compounding multi-turn instability destabilizes supervision, while skill-conditioned privileged guidance requires asymmetric treatment for negative teacher rejections may arise from imperfect skills retrieval or utilization. We introduce SDAR (Self-Distilled Agentic Reinforcement Learning), which treats OPSD as a gated auxiliary objective while keeping RL as the primary optimization backbone. SDAR maps detached token-level signals into a sigmoid gate, strengthening distillation on teacher-endorsed positive-gap tokens and softly attenuating negative teacher rejections. Across the Qwen2.5 and Qwen3 families on ALFWorld, WebShop, and Search-QA, SDAR substantially improves over GRPO (+9.4% on ALFWorld, +7.0% on Search-QA, +10.2% on WebShop-Acc), avoids the instability of naive GRPO+OPSD, and consistently outperforms hybrid RL--OPSD baselines across model scales.

# 论文总结：Self-Distilled Agentic Reinforcement Learning

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：大语言模型（LLM）智能体（Agent）在多轮交互环境中（如 GUI 自动化、Web 购物、搜索问答）需要完成长时域决策任务。强化学习（RL）是训练这类智能体的核心范式，但其仅提供轨迹级（trajectory-level）的稀疏奖励信号，对长 horizo n 交互只能提供粗糙监督。
- **已有方案局限**：On-Policy Self-Distillation (OPSD) 通过在带特权上下文（privileged context，如检索到的技能/技能库）的教师分支上产生密集的 token 级指导来弥补 RL 的不足。但将 OPSD 直接迁移到多轮智能体训练时面临两大核心问题：
  - **多轮 OPSD 不稳定性**：一旦学生智能体偏离教师支持的轨迹，token 级监督信号会逐渐失效，导致各轮 KL 散度激增，任务性能出现灾难性下降。
  - **特权引导的非对称信任**：教师分支并非一个独立更强的模型，而是同一策略附加了训练时才能获得的特权上下文（如检索到的技能）。当教师对某 token 赋予比学生更低的概率时，这种"负差距"可能源于技能检索质量差、技能利用不充分或多轮漂移，而非该 token 真的应该被抑制。实验发现负差距 token 超过 50%，因此需要对正、负教师引导做非对称处理。
- **核心洞察**：对多轮智能体而言，RL 应作为主要的优化骨干，而 OPSD 应降格为一种精心控制的辅助角色。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想

将 OPSD 损失设计为一个**门控辅助目标**（gated auxiliary objective），叠加在 GRPO 主损失之上。通过一个**自适应 sigmoid 门**，让每个 token 自主决定自身蒸馏强度，从而：
- 对教师认可的正差距 token（teacher-endorsed positive-gap tokens）加强蒸馏；
- 对教师否定的负差距 token（negative teacher rejections）进行柔和衰减；
- 保留 RL 优势的语义完整性和无偏性。

### 2.2 关键符号定义

- $x$：输入任务描述；$y_t$：第 $t$ 个 token；$\pi_\theta$：学生策略；$\pi^+_\theta$：附加了特权上下文 $c^+$ 的教师策略。
- **Teacher-Student log-probability gap**：$\Delta_t = \log \pi^+_\theta(y_t | s^+_t) - \log \pi_\theta(y_t | s_t)$，即教师对学生采样 token 赋予的 log 概率与学生自身 log 概率之差。
- **Token-level gate**：$g_t = \sigma(\beta \cdot \Delta_t)$，其中 $\sigma$ 为 logistic sigmoid，$\beta > 0$ 为锐度参数，将无界的 $\Delta_t$ 映射到 $(0,1)$ 区间。$g_t$ 通过 `stop_gradient`（sg）操作从梯度图中分离，仅作为置信度权重。

### 2.3 三种门控策略

| 门控策略 | 公式 | 说明 |
|---|---|---|
| **Gap Gating**（默认） | $g_t = \sigma(\beta \cdot \Delta_t)$ | 直接利用教师-学生差距，正差距 → 高权重，负差距 → 低权重 |
| **Entropy Gating** | $g_t = \sigma(\beta \cdot h_t)$ | 基于学生熵 $h_t$，在学生最不确定的位置激活 |
| **Soft-OR Gating** | $g_t = \sigma(\beta[1-(1-h_t)(1-\Delta_t)])$ | 综合不确定性与教师差距 |

实验证明 **Gap Gating 表现最优**，因为 $\Delta_t$ 最直接地衡量了教师对当前 token 的认可程度。

### 2.4 蒸馏损失函数

$$\ell^{\text{SDAR}}_t = g_t \cdot \left( \log \pi^+_\theta(y_t | s^+_t) - \log \pi_\theta(y_t | s_t) \right)$$

$$L_{\text{SDAR}} = \text{Agg}(\ell^{\text{SDAR}}_t)$$

总体损失：

$$L(\theta) = L_{\text{GRPO}}(\theta) + \lambda_{\text{SDAR}} \cdot L_{\text{SDAR}}(\theta)$$

其中 $\lambda_{\text{SDAR}}$ 控制蒸馏辅助强度（默认 0.01），GRPO 保持为无偏的主要优化骨干。

### 2.5 理论性质

- **命题 1（等价加权似然形式）**：在 detach 假设下，最小化 $L_{\text{SDAR}}$ 等价于最大化加权的 token 级对数似然。
- **命题 2（梯度形式）**：$L_{\text{SDAR}}$ 的梯度严格受限于有界门控标量 $|\nabla_\theta L_{\text{SDAR}}| \leq \text{Agg}(B_t)$，即门不会放大辅助梯度，从而防止梯度爆炸。
- **命题 3（单调性与平滑性）**：$g_t$ 对 $\Delta_t$ 严格单调递增，$\partial g_t / \partial \Delta_t \in (0, \beta/4]$。
- **命题 5（不 detach 的后果）**：若不 detach 门，会引入不稳定的自引用耦合项，导致优化不稳定。

### 2.6 技能检索

将技能检索建模为多臂老虎机问题，使用 UCB（Upper Confidence Bound）、关键词匹配（KM）、全量检索（Full）和随机检索（Random）四种策略评估鲁棒性。

### 2.7 算法流程（Algo 1 SDAR）

1. **采样**：从任务集采样一批任务，检索对应技能 $c^+$。
2. **On-policy rollout**：通过学生策略 $\pi_\theta$ 采样 $G$ 条响应。
3. **序列级优势**：从环境获取奖励，计算组内相对优势 $A^{(i)}$。
4. **GRPO 策略损失**：基于 $A^{(i)}$ 计算裁剪的代理损失 $L_{\text{GRPO}}$。
5. **Token 级门控蒸馏**：
   - 用特权上下文计算教师 logits；
   - 对每个 token 计算 $\Delta_t$ → 门 $g_t$ → 蒸馏损失 $\ell_t$。
6. **联合更新**：最小化 $L = L_{\text{GRPO}} + \lambda \cdot L_{\text{SDAR}}$。

## 6. 主要结论与发现

1. **性能提升显著**：SDAR 在三个基准上全面超越 GRPO：
   - Qwen2.5-7B：ALFWorld +9.4%、Search-QA +7.0%、WebShop-Acc +10.2%
   - 小模型（1.7B）同样有持续提升（ALFWorld 53.9 vs. 46.1）
2. **避免灾难性不稳定性**：朴素 GRPO+OPSD 在小模型上性能崩溃（ALFWorld 3B: 81.2 → 32.0），而 SDAR 通过门控机制稳定训练过程。
3. **技能内化而非依赖**：SDAR 在推理时不依赖外部技能，但性能优于带技能的 Skill-GRPO*，说明知识被真正内化到策略参数中，而非仅作为推理时的上下文提示。
4. **更强泛化能力**：相比 Skill-SD 和 RLSD，SDAR 在分布内和分布外数据集上均保持优势，尤其在小模型上优势更明显。
5. **训练动态验证**：门激活率在训练早期保持在 0.5 以下（抑制负信号），随训练推进逐步上升（更多正信号），符合自适应课程学习的预期。
6. **检索鲁棒性**：即使随机检索也优于纯 GRPO baseline；高质量检索进一步放大收益，说明门控机制能有效过滤低质量技能带来的噪声。

## 7. 优点

- **问题定位精准**：首次系统识别了多轮智能体场景下 OPSD 的两大核心瓶颈（多轮不稳定性和非对称特权引导），并给出针对性的解决方案。
- **设计简洁优雅**：仅通过一个 sigmoid 门实现 token 级自适应控制，机制直观、可解释性强，且不引入额外的大模型调用或复杂的调度器。
- **理论支撑扎实**：提供了 5 个理论命题严格证明了门控机制的单调性、平滑性、梯度有界性和 detach 的必要性。
- **辅助梯度有界性**：命题 4 证明了门控辅助梯度始终不超过无加权梯度，有效防止了梯度爆炸风险，这是 RLSD 等方法所不具备的。
- **实验全面系统**：覆盖 3 个模型规模、3 个任务领域、5+ 种对比基线、4 种消融维度，实验设计从主性能到训练动态到鲁棒性层层递进。
- **推理时无需特权上下文**：技能仅在训练阶段使用，推理时零额外开销，具备实际部署价值。
- **蒸馏目标选择合理**：选择 Reverse KL 而非 Forward KL 或 JSD，契合"弱教师"场景下的 mode-seeking 需求，避免了不必要的模式覆盖。

## 8. 不足与局限

- **训练规模有限**：仅在 1.7B~7B 参数规模的模型上验证，未在更大规模模型（如 14B+、72B）上验证方法的 scaling 特性。
- **训练步数偏短**：150 步的观察窗口可能不足以捕捉长期收敛或潜在的后期性能波动，缺少数千步级别的长期训练实验。
- **技能库依赖**：虽然对检索质量进行了鲁棒性测试，但技能库本身的质量（覆盖度、结构化程度）对训练效果仍有潜在影响，文中未深入分析技能库构建策略。
- **任务类型有限**：仅覆盖 3 类多轮交互任务（ALFWorld、WebShop、SearchQA），对更复杂的开放世界任务（如 Minecraft、OSWORLD 等）的泛化性未知。
- **无离线验证**：所有实验均为在线训练-在线评测的设定，对离线评估或迁移学习场景未做探索。
- **门控超参数 $\beta$ 的敏感性**：虽然 5.0 被验证为最优，但不同模型规模或任务类型可能需要重新调优，缺乏自适应调节机制。
- **教师-学生差距分布不均衡**：超过 50% 的 token 为负差距，意味着大量 token 受到衰减处理，若任务对某些关键决策的 token 恰好落在负差距区域，可能影响学习效果。
