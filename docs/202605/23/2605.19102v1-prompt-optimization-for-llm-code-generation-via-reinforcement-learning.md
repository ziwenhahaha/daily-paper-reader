## Abstract
Large Language Models (LLMs) can generate code from natural language, but their performance is highly sensitive to prompt formulation. We propose a reinforcement-learning-based framework that models prompt refinement as a sequential decision-making problem. A Proximal Policy Optimization (PPO) agent iteratively improves prompts using a hybrid action space that combines direct generation, genetic lexical mutation and semantic rewriting, guided by shaped rewards derived from unit-test feedback. We evaluate the framework on MBPP+, HumanEval+, and APPS using CodeT5+, CodeLLaMA, and DeepSeek-Coder as frozen code generators. On the 500-task MBPP+ test set, the PPO agent achieves strict Pass@1 scores of 57.58%, 64.80%, and 85.50%, respectively, outperforming EPiC, Reflexion, and Random-Hybrid. Soft-Pass@1 reaches 67.90%, 73.10%, and 88.20%, respectively. Similar improvements are observed on HumanEval+ and APPS across all backbone models. The results demonstrate that reinforcement learning with shaped test-driven rewards improves functional correctness in LLM-based code generation.

# 论文总结：基于强化学习的大型语言模型代码生成的提示优化

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题背景**：大型语言模型（LLMs）能够根据自然语言描述生成代码，但其生成性能对 prompt 的表述方式高度敏感。即使是最先进的 LLMs，在代码生成任务上仍表现出相对较低的准确率，常引入功能性和逻辑性缺陷，需要人工调试。

- **现有方法的局限**：
  - 静态 prompt 方法（如 Chain-of-Thought）缺乏适应性，无法基于中间反馈进行迭代优化
  - 进化算法（如 EPiC）通常依赖预定义算子，缺乏自适应学习能力
  - 强化学习方法主要依赖二元正确性信号，忽视了部分正确性或详细反馈中蕴含的丰富信息

- **核心研究目标**：探索强化学习是否能够提升代码生成任务的 prompt 优化效果，通过自适应地结合词汇变异和语义重写，实现多步 prompt 精炼。

---

## 2. 论文提出的方法论

### 2.1 核心思想

将 **prompt 优化建模为序列决策问题**，使用 PPO（近端策略优化）智能体在迭代过程中学习选择最优的 prompt 转换策略。智能体结合三种动作：直接生成、词汇变异和语义重写，通过单元测试反馈构建的 shaped rewards 引导策略学习。

### 2.2 环境设计

- **状态表示**：使用 MiniLM sentence transformer 将当前 prompt 嵌入为 384 维向量 $s_t = \text{Embed}(p_t)$
- **奖励设计**：基于 pass ratio $\rho_{i,t}$（通过测试用例的比例）构建 shaped reward：
  - $\rho_{i,t} = 1.0$ 时，$R = 1.0$（完全正确）
  - $0 < \rho_{i,t} < 1.0$ 时，$R = \rho_{i,t}$（部分正确给予比例奖励）
  - $\rho_{i,t} = 0$ 时，$R = -1$（完全失败）
  - 执行失败时，$R = -2$（更强惩罚）

### 2.3 动作空间与转换策略

| 动作类型 | 说明 |
|---------|------|
| **直接生成** | 不修改 prompt，利用解码随机性探索 |
| **词汇变异** | 遗传算法驱动：交叉、索引 shuffle、变异（概率 0.2）、锦标赛选择 |
| **语义重写** | 预训练模型执行高级重构，配合正则过滤保留关键词 |

### 2.4 算法流程

```
训练循环（每 episode）：
1. 采样编程任务，嵌入初始 prompt
2. 循环迭代（直到全部通过或达到最大步数）：
   a. 智能体根据当前状态采样动作
   b. 执行对应的 prompt 转换
   c. 使用冻结的代码生成模型生成代码
   d. 在沙箱环境执行代码，评估测试用例
   e. 计算 shaped reward，存储转换
   f. 嵌入更新后的 prompt
3. 使用 PPO 更新策略
```

### 2.5 评估指标

- **Strict Pass@1**：主要指标，只有全部测试通过才算成功
- **SoftPass@1**：辅助指标，衡量迭代过程中的渐进式进展

## 6. 论文的主要结论与发现

1. **语义重写提供最强单步提升**：在单步设置下，语义重写 > 词汇变异 > 直接生成

2. **Shaped reward 显著优于 binary reward**：
   - 提供连续反馈，避免部分正确解被当作完全失败
   - 加速收敛，减少训练不稳定性

3. **PPO 框架一致超越所有基线**：
   - MBPP+ 上：PPO (57.58%/64.80%/85.50%) vs Random-Hybrid (31.12%/49.08%/79.10%)
   - HumanEval+ 上：PPO (60.20%/70.60%/91.30%) vs EPiC (33.80%/41.20%/90.10%)
   - APPS 上同样取得最佳结果

4. **学习到的策略比随机调度更有效**：Random-Hybrid 证明性能提升来自学习到的动作调度，而非仅靠动作空间组合

5. **统计显著性**：McNemar 检验显示 p < 0.001，中等效应量（Cohen's h ≈ 0.27-0.35）
