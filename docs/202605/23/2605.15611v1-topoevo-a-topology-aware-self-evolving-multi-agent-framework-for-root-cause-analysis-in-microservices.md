## 摘要
TopoEvo通过拓扑感知自演化多智能体框架解决微服务根因分析中的症状放大偏差问题，采用MOMA、VQ和HET模块实现，根因定位精度提升3.44%，故障分类F1提升4.39%~16.81%。

## Abstract
Root cause analysis (RCA) in microservices is challenging due to (i) noisy and heterogeneous multimodal observability (metrics, logs, traces), (ii) cascading failure propagation that amplifies downstream symptoms, and (iii) non-stationary topology drift induced by autoscaling and rolling updates. Recent LLM-based RCA agents can generate tool-grounded explanations, yet they often remain topology-agnostic and suffer from \emph{symptom-amplification bias}, misattributing the root cause to salient downstream victims. We propose \textbf{TopoEvo}, a topology-aware self-evolving multi-agent framework that couples graph representation learning with structured, topology-constrained reasoning. TopoEvo first introduces \emph{Metric-orthogonal Multimodal Alignment} (MOMA), which decomposes metric embeddings into complementary subspaces and contrastively aligns logs and traces to reduce modality redundancy and sparsity, yielding stable node representations for graph encoding. It then applies \emph{Vector Quantization} (VQ) to discretize topology-enhanced states into auditable \emph{symptom tokens} with a symptom lexicon, enabling reliable retrieval and token-level evidence grounding. On top of these discrete topology cues, TopoEvo performs a multi-agent \emph{Hypothesis--Evidence--Test} (HET) workflow to explicitly verify propagation-consistent explanations and separate initiating anomalies from amplified downstream symptoms. Finally, a \emph{Self-Evolving Mechanism} refreshes hierarchical incident memory and performs conservative test-time adaptation with high-confidence pseudo-labels to maintain robustness under drift.

## 1. 论文核心问题与整体含义（研究动机和背景）

### 研究背景
- 微服务架构已成为构建大规模云应用的主流范式，但系统复杂性使得运维面临严峻挑战
- 故障很少停留在局部，会沿服务依赖关系快速传播，产生放大且具有误导性的下游症状

### 核心挑战（三座大山）
1. **噪声异构多模态可观测性**：指标、日志、跟踪三种模态的频率、稀疏性、噪声模式和缺失模式各不相同，简单拼接导致不稳定表示和虚假相关
2. **症状放大偏差（Symptom-Amplification Bias）**：现有LLM-based RCA agent缺乏拓扑感知，常将最显著的下游受害者误判为根因
3. **非平稳拓扑漂移**：自动伸缩、滚动更新和演化的调用图导致分布偏移，静态模型在分布外场景下性能严重下降

### 论文目标
提出 **TopoEvo**，一个拓扑感知、自我演化的多智能体框架，实现微服务根因定位和故障类型分类的联合推理。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 整体框架（五组件架构）

#### A. 数据处理与依赖图构建
- **多模态数据预处理**：
  - 指标（Metrics）：归一化多变量时间序列 → TCN + MLP 编码
  - 日志（Logs）：Drain3模板解析 → TF-IDF 向量化 → 轻量投影器
  - 跟踪（Traces）：Span关系解析 → 聚合统计 → 1D膨胀CNN + MLP
- **细粒度同质依赖图**：节点/服务/Pod三个粒度统一建模，边编码交互/传播关系和结构关系

#### B. Metric-Orthogonal Multimodal Alignment (MOMA)
**核心思想**：以指标为锚点，将日志和跟踪对齐到互补的正交子空间

**关键技术**：
- 指标嵌入分解为两个互补分量：
  ```
  u_v = W_u * x_metric_v (log一致分量)
  v_v = W_v * x_metric_v (trace一致分量)
  ```
- **正交正则化**：防止两个分量捕获相同信息
  ```
  L_⊥ = Σ_v (u_v^⊤ * v_v) / (||u_v||₂ * ||v_v||₂ + ε)²
  ```
- **对比对齐（InfoNCE）**：将日志对齐到 u_v，将跟踪对齐到 v_v
- 预训练策略提升训练稳定性

#### C. 拓扑感知增强：VQ + Symptom Vocabulary
**核心思想**：将连续拓扑感知状态离散化为可检索、可审计的症状标记

**关键技术**：
- **向量量化**：使用码本 C = {c_k}^K_{k=1} 将节点表示 h_v 映射到最近码向量
  ```
  q_v = arg min_k ||h_v - c_k||²
  ĥ_v = c_q_v
  ```
- **VQ损失**：码本损失 + 承诺损失（标准 VQ-VAE 形式）
- **症状词典构建**：为每个离散码 k 关联：
  - **Metric Signature**：主导KPI模式（典型z-score、频率）
  - **Log Signature**：模板ID + 突发度指标
  - **Trace Signature**：异常交互模式（入站/出站特征）
  - **Topology Context**：邻居类型分布、度配置文件
  - **Summary**：自然语言简短描述

#### D. Reasoning-Enhanced Multi-Agent Diagnosis (HET)
**核心思想**：将诊断分解为假设→证据→测试的验证循环，而非直接输出结论

**工作流程**：
1. **候选中心子图构建**：选取Top-K候选节点 + H跳诱导子图，限制推理范围
2. **假设生成（Hypothesis Planner）**：
   - 拓扑约束优先（因果传播沿有向边）
   - 必须包含"受害者即根因"替代假设
   - 多级一致性（Node/Service/Pod）
   - 可验证性：每个假设需指定支持/反驳证据
3. **证据获取（Modality Agents）**：
   - Metric Agent：时间戳、幅度带、阈值交叉
   - Log Agent：模板ID、突发分数
   - Trace Agent：异常span链、方向性
   - 返回结构化证据，禁止叙述性输出
4. **验证（Judge Agent）**：
   - **时间优先性**：根因先于下游异常
   - **路径一致性**：证据位于假设传播路径上
   - **模板一致性**：日志/跟踪匹配故障模板
5. **显式决策**：报告Top-1根因 + 说明为何拒绝强替代假设

#### E. Self-Evolving Mechanism（自演化机制）
**核心思想**：在非平稳系统中持续保持鲁棒性

**组件**：
1. **层次Incident Memory**：
   - 存储已解决事件：拓扑指纹、症状标记集、验证假设、缓解措施
   - **层次更新策略**：Pod层比Service/Node层更频繁刷新
   - **相似性感知随机遗忘**：防止冗余崩溃
     ```
     p(i|N_τ) = exp(sim(u_new, u_i)/γ) / Σ exp(sim(u_new, u_j)/γ)
     ```

2. **测试时适应（Test-Time Adaptation）**：
   - 高置信度诊断 → 伪标签样本积累到缓冲区 B_tta
   - 保守优化目标：
     ```
     L_tta = L_pseudo_rca + λ_reg * ||θ - θ₀||²
     ```
   - 正则化项防止灾难性漂移

## 6. 论文主要结论与发现

### 核心发现

1. **性能提升显著**：
   - 根因定位精度绝对提升 **3.44%**
   - 故障分类F1提升 **4.39%~16.81%**

2. **各模块贡献度排序**（基于消融实验）：
   - **HET（假设-证据-测试）影响最大**：移除后Acc@1下降15-16个百分点
   - **MOMA（多模态对齐）次之**：约10个百分点
   - **VQ（向量量化）第三**：约12个百分点
   - **SEM（自演化机制）也有显著贡献**：≥8个百分点

3. **码本大小最优值**：K=128 为最优粒度
   - K过小（32/64）：原型碰撞，不同症状被压缩到同一token
   - K过大（256/512）：过度细分导致噪声编码、泛化能力下降

4. **VQ学习的症状词典优于baseline**：
   - Token Purity和NMI更高
   - 组内方差更低
   - 证明VQ能够学习紧凑且具判别性的离散表示

5. **案例验证了症状放大偏差的修正**：
   - GAT RCL错误地将U2（下游）排在P0（真正根因）之前
   - TopoEvo通过HET验证了P0→U2→G1传播路径，纠正错误定位

## 7. 优点：方法或实验设计亮点

### 方法论亮点

1. **拓扑感知设计独特**：
   - 首次将GAT编码器与症状token化结合
   - 通过码本将拓扑状态离散化，解决症状放大偏差
   - 实现可审计的token级证据锚定

2. **多模态对齐创新**：
   - Metric-Orthogonal策略避免log/trace对齐到相同因子
   - 正交正则化保持模态互补性
   - 指标锚点提供稳定参考，解决日志/跟踪稀疏问题

3. **多智能体HET工作流**：
   - 假设-证据-测试结构化流程，强制可验证性
   - 显式消除强替代假设，提升可解释性
   - 拓扑约束贯穿假设生成到验证全流程

4. **自演化适应机制**：
   - 层次记忆策略（Pod层更频繁刷新）适配不同变化速率
   - 相似性感知遗忘防止记忆崩溃
   - 保守测试时适应避免灾难性遗忘

### 实验设计亮点

1. **多粒度评估**：Service/Pod/Node三个级别独立评估，覆盖实际运维场景
2. **真实场景验证**：Dataset B来自真实生产环境，增强实用性说服力
3. **对比方法全面**：涵盖传统图方法、多模态学习方法和LLM-agent方法
4. **参数敏感性分析**：K值实验揭示离散化粒度的重要性
