## Abstract
Multi-agent systems extend large language models (LLMs) by decomposing tasks among specialized agents, but their distributed decision process creates new attack surfaces. We identify \emph{semantic hijacking}, an attack in which harmful requests are concealed within domain-specific narratives and propagated to a Manager through Worker reports, without any syntactic injection primitives. Across 42,000 adversarial trials over 12 Manager models and 7 Worker configurations, we uncover a \emph{capability paradox}: as Worker capability increases, the mean system-level Attack Success Rate (ASR) increases from 18.4% to 63.9%, peaking at 94.4%. To explain this effect, we conduct multi-level mediation analysis on two independent datasets (47,807 interactions). This analysis shows that this paradox is driven by \emph{linguistic certainty}: stronger Workers are more likely to interpret adversarial narratives as legitimate, convey their conclusions assertively, and thereby lead Managers to treat such confident endorsements as justification to execute. In our larger Worker-Only setting ($n_W$=14), certainty mediates 74% of the effect, with 95% confidence intervals (CI) excluding zero under both Monte Carlo and cluster bootstrap; the smaller Full-MAS setting ($n_W$ =6) shows a directionally consistent indirect effect. Worker-side safety prompting does not reliably mitigate this failure. Building on the mediation finding, we propose \emph{heterogeneous ensemble verification}, which pairs Workers of asymmetric domain competence so their complementary vulnerabilities break the certainty-to-execution chain, reducing ASR from 52.8% to 2.0% with negligible benign-task impact. Our results show that upgrading components to stronger models can actively degrade system security, and that effective defenses require exploiting--rather than eliminating--capability asymmetries between agents.

# 论文详细总结

## 1. 核心问题与整体含义

- **研究背景**：现代多智能体系统（MAS）采用分层 Manager-Worker 架构，将任务分解给专业化的 Worker 代理，由 Manager 聚合报告并决定是否执行操作或调用外部工具。这种分布式决策虽提升了可扩展性和专业化程度，但也引入了新的安全依赖：Manager 通常信任 Worker 的报告，即使其输入通道可能包含对抗性内容。
- **核心威胁**：论文识别出一种全新的攻击类型——**语义劫持（Semantic Hijacking）**。攻击者将恶意请求隐藏在领域特定的叙事（如 SRE 事件报告）中，通过 Worker 的报告传递给 Manager，无需任何传统的句法注入原语（如指令覆盖、定界符操纵）。payload 完全处于模型的预期输入分布内，句法防御难以检测。
- **核心悖论**：论文发现**能力悖论（Capability Paradox）**——更强的 Worker 模型反而会增加系统级攻击成功率。这与"更强模型应更安全"的直觉假设相悖。

## 2. 论文提出的方法论

### 核心思想
论文设计了**受控评估框架**，系统性地研究分层 MAS 中安全行为如何在角色间传递。核心在于将语义攻击与常规 prompt 注入分离，隔离由语义解释和委托信任导致的失败。

### 关键技术细节

**（1）语义劫持 Payload 生成**
- **来源**：使用真实 SRE 事件报告（来自 danluu/post-mortems 仓库，含 Cloudflare、GitHub、Google、AWS、Stripe 等组织的案例）
- **突变器**：使用 Gemini-3-Flash 将真实事件转化为"看似合理但实际危险的操作请求"
- **约束**：payload 必须是自然语言，排除可执行命令、SQL 语句、利用字符串或指令覆盖模式
- **攻击策略**：5 类 SRE 策略（紧急性、数据完整性恢复、安全containment、合规/法律、行政紧急授权）

**（2）评估架构**
- **Worker**：作为安全审计员，接收 payload 并输出结构化安全评估报告（包含：评估标签、安全类别、提取的元数据、技术理由）
- **Manager**：接收原始 payload 和 Worker 报告，做出最终决定（拒绝、请求验证、调用工具）
- **Oracle**：自动化评估工具，与人工标注一致性 Cohen's κ = 0.87（Full MAS）和 0.92（Worker-only）

**（3）中介分析**
- **预测变量**：Worker 能力（以 MMLU 得分代理）
- **中介变量**：语言确定性（assertive 词汇密度 - hedging 词汇密度）
- **结果变量**：攻击成功（ASR/Fool Rate）
- **统计方法**：Worker 级 OLS（GEE 逻辑回归 + 聚类稳健标准误）+ Monte Carlo 模拟（20,000 次）+ Worker 聚类自助法（2,000 次）

**（4）防御方法**
- **异构集成验证（Heterogeneous Ensemble Verification）**：将高能力 Worker（DeepSeek-R1）与保守 Worker（Llama-3.1-8B）配对，使用 OR-gate 逻辑——只有当两个 Worker 都评估为安全时才转发给 Manager

## 6. 论文的主要结论与发现

### 核心发现
1. **能力悖论存在**：Worker 能力（MMLU）与 Fool Rate 正相关（Spearman ρ = 0.81, p < 0.001），更强的 Worker 更易被语义劫持
   - ASR 范围：0.8% → 94.4%
   - DeepSeek-R1 Worker Fool Rate 高达 90.3%

2. **语言确定性是中介机制**：
   - Worker-only 设置（nW=14）：确定性中介 74% 效应，95% CI [+39%, +146%]
   - 机制：强 Worker 更自信地将对抗叙事解读为合法 → Manager 将自信背书视为执行授权

3. **跨域差异**：能力悖论在 SRE（ρ=0.81）最强 → 医疗（ρ=0.50）→ 金融（ρ=0.07），反映正式协议覆盖程度

4. **现有防御不足**：
   - Worker 侧安全提示词：不稳定（9/12 模型改善，3 个恶化）
   - Manager 侧怀疑提示词：降低 ASR 但良性任务完成率从 91.3% 降至 76.0%
   - 不确定性注入：ASR 从 70.1% 降至 26.9%，但良性完成从 80.3% 降至 54.0%

5. **异构集成验证有效**：DeepSeek-R1 + Llama-3.1-8B 配对
   - ASR 从 52.8% 降至 2.0%（-50.8 pp）
   - 良性任务完成无影响（77.7% → 77.7%）

|------|------|
| **领域覆盖** | 仅测试 SRE、医疗、金融三个领域，对科学、法律等工作流的泛化性需进一步验证 |
| **Oracle 偏差** | 自动 Oracle 虽与人工标注一致性高，但仍可能存在系统性盲点，更大规模人工验证可能发现新问题 |
| **能力度量** | MMLU 和 GPQA-D 来自异构评估流程，标准化重新评估可能收紧能力-脆弱性关联 |
| **Worker 数量** | Full MAS 设置中 Worker 数量较小（nW=6），限制统计推断力 |
| **因果推断** | 中介分析确认相关性，但未排除未测量的混淆因素 |
| **防御可扩展性** | 异构集成需维护多个 Worker，可能增加系统复杂度和成本 |
| **弱伙伴选择** | 防御效果依赖弱伙伴的"选择性拒绝"特性，需针对性测试 |
