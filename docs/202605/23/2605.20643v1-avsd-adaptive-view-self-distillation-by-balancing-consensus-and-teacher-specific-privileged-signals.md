## 摘要
自蒸馏使语言模型能够通过将同一模型同时作为学生和教师来从自身轨迹中学习，教师的条件化依赖于学生无法访问的特权信息。这类信息可以呈现为不同的类型或视图，如解决方案、演示、反馈或最终答案。这种设置提供了密集的逐token反馈，无需依赖独立的外部模型，但造成了一种根本性的不对称：教师可能依赖于学生推理时无法访问的视图特定信息。此外，最佳特权信息类型通常与任务相关，使得选择单一教师视图变得困难。在这项工作中，我们通过引入AVSD（自适应视图自蒸馏）来共同应对这两个挑战，这是一种利用多个特权信息视图的新型自蒸馏方法，通过将稳定的跨视图共识与视图特定的残差信号分离开来重建逐token监督。AVSD识别跨视图共享的共识信号，该信号提供了可靠的更新方向，然后有选择地添加视图特定的残差信号来调整更新幅度，前提是该信号与共识方向一致且与共识信号保持适当比例。在数学竞赛基准测试（AIME24、AIME25和HMMT25）上的实验表明，AVSD始终优于单视图自蒸馏基线和GRPO，分别在Qwen3-8B和Qwen3-4B上实现了3.1%和2.2%的平均Avg@8提升。此外，在使用Qwen3-8B的代码生成基准测试（Codeforces、LiveCodeBench v6）上，AVSD平均优于单视图自蒸馏基线2.4%。

## Abstract
Self-distillation enables language models to learn on-policy from their own trajectories by using the same model as both student and teacher, with the teacher being conditioned on privileged information unavailable to the student. Such information can come in different types or views, such as solutions, demonstrations, feedback, or final answers. This setup provides dense token-level feedback without relying on a separate external model, but creates a fundamental asymmetry: the teacher may rely on view-specific information that the student cannot access at inference time. Moreover, the best type of privileged information is often task-dependent, making it difficult to choose a single teacher view. In this work, we address both these challenges jointly by introducing AVSD (Adaptive-View Self-Distillation), a novel method of self-distillation with multiple privileged-information views, which reconstructs token-level supervision by separating stable cross-view consensus from view-specific residual signals. AVSD identifies the consensus signal shared across views, which provides a reliable update direction, and then selectively adds the view-specific residual signal to adjust the update magnitude when it both aligns with the consensus direction and remains proportionate to the consensus signal. Experiments on math competition benchmarks (AIME24, AIME25, and HMMT25) show that AVSD consistently outperforms both single-view self-distillation baselines and GRPO, achieving average Avg@8 gains of 3.1% and 2.2% over the strongest baselines on Qwen3-8B and Qwen3-4B, respectively. Moreover, on code-generation benchmarks (Codeforces, LiveCodeBench v6) using Qwen3-8B, AVSD outperforms the single-view self-distillation baseline by 2.4% on average.

# AVSD 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

### 研究背景
- 大语言模型（LLM）在解决复杂推理任务（如竞赛级数学问题）时，需要将可用的训练监督转化为有效的学习信号
- 强化学习与可验证奖励（RLVR）虽已成为主流范式，但其监督通常是稀疏的、仅在结果层面
- 蒸馏提供密集的 token 级指导，但标准离线蒸馏存在训练-测试分布不匹配问题
- 在线蒸馏仍依赖更强大的外部教师模型

### 核心问题
**问题一：选择何种特权信息的困境**
- 不同的特权信息类型（完整解答、部分推理、仅最终答案等）在不同数据集上表现差异显著
- 没有单一视图是普遍最优的：完整解答可能锁定教师于预定义思维模式，仅答案可能缺乏信息密度
- 分布匹配在信息不对称下存在不可约的互信息间隙，学生被迫编码无法在推理时观察的视图特定相关性

**问题二：教师-学生信息不对称**
- 教师依赖学生无法访问的特权信息
- 直接模仿特权教师会导致推理时无法复现的偏差
- 教师可能产生视图特定的人工痕迹

### 研究目标
能否同时利用多个特权视图来构建比任何单一特权教师更好的 token 级在线学习信号？

## 2. 论文提出的方法论

### 核心思想
**自适应视图自蒸馏（AVSD）**：将多视图教师信号分解为两部分：
1. **共识信号**：跨视图共享的稳定信息，提供可靠的更新方向
2. **残差信号**：视图特定的支持，通过门控机制选择性添加

### 关键技术细节

#### 数学形式化

**单视图设置**：学生 token 分布 $p_t(v)$，教师分布 $q_t(v)$（条件于特权信息），token 级蒸馏优势定义为：
$$A_t(v) = \log q_t(v) - \log p_t(v)$$

- $A_t(v) > 0$：教师赋予更高概率，梯度下降促进该 token
- $A_t(v) < 0$：抑制该 token

**多视图设置**：$M$ 个特权视图 $\mathcal{V}(\mathbf{r}) = \{r^{(1)}, \ldots, r^{(M)}\}$，每个视图诱导教师分布 $q^{(m)}_t(v)$

#### 两个聚合目标

**1. 几何共识目标（Geometric Consensus Target）**
$$\tilde{q}^G_t(v) = \left(\prod_{m=1}^{M} q^{(m)}_t(v)\right)^{1/M} = \exp\left(\frac{1}{M}\sum_{m=1}^{M}\log q^{(m)}_t(v)\right)$$

- 归一化后得到 $q^G_t(v)$
- 强调跨视图一致支持，低概率token被下权重
- 优点：**保守**，不易受单一视图偏差影响
- 对应优势：$A^G_t(v) = \frac{1}{M}\sum_{m=1}^{M}\Delta^{(m)}_t(v)$

**2. 算术边际目标（Arithmetic Marginal Target）**
$$q^A_t(v) = \frac{1}{M}\sum_{m=1}^{M}q^{(m)}_t(v)$$

- 保留至少一个视图强烈支持的 token
- 优点：**宽松**，能捕获补充信息
- 缺点：敏感于视图特定人工痕迹

**残差定义**：
$$J_t(v) = \log q^A_t(v) - \log \tilde{q}^G_t(v) \geq 0$$
- 测量算术边际中几何共识丢弃的支持量

#### 门控重建机制

**门函数**：
$$\lambda_t(v) = C_t(v) \cdot R_t(v) = \frac{|A^G_t(v)|}{\frac{1}{M}\sum_{m=1}^{M}|\Delta^{(m)}_t(v)| + \epsilon} \cdot \frac{|A^G_t(v)|}{|A^G_t(v)| + J_t(v) + \epsilon}$$

两个组件：
- **对齐组件** $C_t(v)$：当视图方向一致时接近1，冲突时接近0
- **幅度组件** $R_t(v)$：防止残差翻转共识方向

**重建优势**：
$$\tilde{A}_t(v) = A^G_t(v) + \lambda_t(v)J_t(v)$$

**关键保证**：$0 \leq \lambda_t(v)J_t(v) \leq |A^G_t(v)|$
- 残差可调整更新幅度，但不能翻转共识方向

**重建目标分布**：
$$q^*_t(v) = \frac{\tilde{q}^G_t(v)\exp(\lambda_t(v)J_t(v))}{\sum_{u}\tilde{q}^G_t(u)\exp(\lambda_t(u)J_t(u))}$$

#### 训练目标

$$\mathcal{L}_{\text{AVSD}}(\theta) = \mathbb{E}_{(x,r)\sim\mathcal{D}} \mathbb{E}_{y\sim P_{\theta}^S(\cdot|x)} \left[\sum_{t=1}^{|y|} D_{\text{KL}}(p_\theta(\cdot|h_t) \parallel \text{sg}[q^*_t(\cdot)])\right]$$

梯度仅通过学生分布流动，教师分布和重建目标作为固定监督。

### 算法流程

```
Algorithm: Multi-View Self-Distillation with Gated Reconstruction
1. 对每个训练迭代：
   2. 采样 minibatch {(xi, r_i)}
   3. 对每个样本 (x, r)：
      4. 构建特权视图 r^{(m)} = T_m(r), m=1..M
      5. 采样在线 rollout y ~ p_theta(·|x)
      6. 对每个前缀 h_t = (x, y_{<t})：
         7. 计算学生分布 p_t(·) = p_theta(·|h_t)
         8. 对每个视图 m：
            9. 计算教师分布 q^{(m)}_t(·) = sg[p_theta(·|h_t, r^{(m)})]
         10. 计算 q^A_t(v) 和 eq^G_t(v)
         11. 计算残差 J_t(v)
         12. 计算门 λ_t(v)
         13. 重建教师 q^*_t(v)
         14. 累积损失 DKL(p_t || sg[q^*_t])
      15. 更新 θ
```

### 核心发现

1. **无单一特权视图是最优的**
   - 不同特权信息类型在不同数据集上表现差异显著

   - 这是任务依赖的，无法事先确定

2. **AVSD 始终优于所有基线**
   - 在数学基准上，AVSD 相对最强基线：
     - Qwen3-4B：提升 2.2%
     - Qwen3-8B：提升 3.1%（相对 GRPO）
     - DeepSeek-7B：提升 2.3%（相对 OPSD）
   - 在代码基准上（Qwen3-8B），AVSD 相对 OPSD 提升 2.4%

3. **共识+门控残差优于单独使用任一目标**
   - 共识目标过于保守，部分数据集表现不佳
   - 算术目标过于宽松，可能引入噪声
   - 门控机制有效平衡两者

4. **更多视图带来更好性能**
   - 从 1 个视图增至 3 个视图：
     - AIME24: +2.9%
     - AIME25: +2.8%
     - Codeforces: +2.0%
   - 边际收益递减，但 4 个视图仍有收益

5. **AVSD 提供更可靠的 token 级学习信号**
   - GRPO 在 29.6% 的错误 rollout 案例中无学习信号
   - OPSD 对 12.3% 的高影响 token 给出错误符号
   - AVSD 仅 2.9% 的高影响 token 有错误符号

6. **训练开销合理**
   - 仅增加 13% 的训练时间
   - 额外成本来自批量教师前向传递，而非 rollout 生成

## 8. 不足与局限

### 方法论局限

1. **视图质量依赖**
   - 性能受限于特权视图的质量
   - 若视图包含过多噪声或人工痕迹，门控可能无法完全过滤

2. **视图构造的工程成本**
   - 需要为不同任务设计不同的视图构造方法
   - 缺乏自动化的视图发现或选择机制

3. **仅验证反向 KL 散度**
   - 未探索其他蒸馏损失（如正向 KL、JS 散度）
   - 假设反向 KL 对此设置最优但未充分论证

4. **门控函数的表达能力**
   - 当前的门函数基于启发式设计
   - 可能存在更优的门控机制未被发现

### 实验局限

5. **任务类型单一**
   - 主要测试推理密集型任务（数学、代码）
   - 未验证在知识问答、摘要、对话等任务上的效果

6. **模型规模有限**
   - 最大测试到 8B 模型
   - 未在更大模型（如 32B+）上验证

7. **缺乏分布外泛化测试**
   - 所有评估数据来自类似训练数据的领域
   - 未测试在全新领域（如科学推理、逻辑推理）上的泛化

8. **未报告方差**
   - 实验结果缺乏统计显著性报告
   - 难以判断性能差异是否显著

9. **消融实验不够充分**
   - 未单独测试对齐组件和幅度组件的贡献
   - 未测试不同门函数设计的效果

### 实践局限

10. **需要特权信息**
    - 依赖于特权信息的可用性
    - 在无法获取特权信息的场景中不适用

11. **推理开销未测试**
    - 仅测试了训练阶段的开销
    - 未分析推理阶段的效率

12. **超参数敏感性**
    - 未充分探索门函数中 ε 的敏感性
    - 未测试其他超参数组合

## 总结评价

AVSD 论文提出了一个简洁而有效的多视图自蒸馏框架，通过几何共识和算术边际目标的门控组合，在数学和代码推理任务上取得了显著的性能提升。该工作的主要贡献在于：

- **理论层面**：清晰的形式化将多视图蒸馏问题分解为共识和残差两个可解释组件
- **实践层面**：方法通用且开销可控，易于集成到现有 RLVR 流程
- **实证层面**：全面的实验验证了方法在多种设置下的有效性

尽管存在视图质量依赖和任务覆盖范围有限等局限，该工作为多视图蒸馏提供了有价值的视角，并对自蒸馏中特权信息选择的问题提供了创新的解决方案。
