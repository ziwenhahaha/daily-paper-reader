## Abstract
Navigation mesh (Navmesh) inconsistencies affect the player experience by directly impacting the navigation systems used by non-playable characters (NPCs) in game environments. While navmeshes are generated from world geometry using well-established algorithms, environments change throughout development as terrain is adjusted and assets are moved or replaced, resulting in mismatches between the navmesh and the actual environment. Existing automated approaches attempt to detect navigation issues using exploration agents and reinforcement learning techniques. However, since these methods rely on the navigation data itself or evaluate navigation behavior indirectly, they do not explicitly verify whether the navigation representation reflects the walkable space defined by underlying geometry.   This paper presents a framework for validating navigation meshes through an independent, geometry-driven analysis of navmesh correctness. The approach reconstructs walkable space directly from environment geometry using a voxel-based representation, followed by constraint-aware traversal and connectivity evaluation. Validation is formulated as a prioritized search problem over the voxel space, where reinforcement learning guides sampling toward regions more likely to exhibit inconsistencies. At each sampled location, reachability derived from the voxel representation is compared against reachability obtained from the navmesh via engine-level queries. Experiments across multiple large-scale open-world game environments show that the approach consistently lowers exploration effort while maintaining similar defect detection coverage. The framework runs offline within the game engine and can be integrated into automated quality assurance pipelines. Since the method relies on geometry, it can be adapted across game engines with minimal changes, making it suitable for production deployment.

## 1. 论文的核心问题与整体含义

- **研究背景**：导航网格（Navmesh）广泛用于游戏中非玩家角色（NPC）的寻路与导航。Navmesh 由底层几何体通过成熟算法生成，但在大规模游戏开发中，环境和几何体频繁变更（地形调整、资产移动/替换），导致 Navmesh 与实际环境之间出现不一致。
- **核心问题**：传统自动化验证方法依赖导航代理的探索行为（如 RL 代理行走、碰撞检测等），间接推断导航问题。这些方法存在以下局限：
  - 依赖运行时行为，无法直接验证几何一致性
  - 计算开销大，难以集成到自动化 QA 流程
  - 只能检测已发生的运行时错误，无法主动发现潜在的结构性缺陷
- **研究动机**：提出一种几何驱动的验证框架，直接将 Navmesh 与从几何体重建的可步行空间进行比较，实现对导航正确性的离线、显式验证。

## 2. 论文提出的方法论

### 2.1 核心思想

将 Navmesh 验证问题**重新定义为一个几何一致性问题**：
- 从环境几何体（地形 + 碰撞几何）独立重建可步行空间
- 在同一采样点比较**体素可达性**与 **Navmesh 可达性**
- 若两者不一致，则标记为潜在的导航缺陷

### 2.2 关键技术细节

**（1）体素化可步行空间重建**

- **地形体素化**：从引擎高度图采样，将地形表面表示为点集 $v_i = (x_i, y_i, h(x_i, y_i))$
- **资产碰撞体素化**：对碰撞网格进行体素-三角形相交测试，标记占据体素 $v_i \in V_{occ}$（当且仅当 $B_i \cap M \neq \emptyset$）
- **体素分辨率 $s$**：控制几何保真度与计算成本的权衡

**（2）约束感知可步行性分类**

每个体素需满足以下遍历约束：
- **坡度限制**：$\theta = \arccos(\mathbf{n} \cdot \mathbf{u}) \leq \theta_{max}$
- **台阶高度**：$|h_i - h_j| \leq h_{step}$
- **碰撞半径**：$d_{obs} \geq r_{agent}$
- **垂直间隙**：$H_{free} \geq H_{agent}$

**（3）可达性图构建**

将满足约束的体素连接为图 $G_v = (V_w, E)$，其中 $(v_i, v_j) \in E$ 当且仅当 $\|\mathbf{p}_i - \mathbf{p}_j\| \leq r$（邻域半径）

**（4）强化学习引导的优先级探索**

- **问题形式化**：将探索建模为有限 MDP，最大化累积奖励 $J(\pi_\theta) = \mathbb{E}\left[\sum_{t=0}^{T} \gamma^t R_t\right]$
- **状态表示**：包含当前体素重要性、探索进度、邻居统计、方向向量等特征的固定长度向量
- **动作空间**：离散移动方向 $A = \{N, NE, E, SE, S, SW, W, NW\}$
- **奖励函数**：$R_t = I(v_t) - \lambda_{step} - P_{revisit}$（重要性奖励 - 步数惩罚 - 重复访问惩罚）
- **语义重要性建模**：$I(v) = \sum_{k \in \mathcal{N}(v)} w_k$（聚合巡逻路径、生成点、交互区等游戏语义信号）
- **策略网络**：采用双深度 Q 网络（DDQN），减少过估计偏差
- **优先经验回放**：基于 TD 误差 $\delta_t$ 优先采样高信息量转换

**（5）一致性验证与后处理**

- 采样点处比较 $R_{vox}(w)$ 与 $R_{nav}(w)$
- 若 $R_{vox}(w) \neq R_{nav}(w)$，标记为不一致
- 应用容差过滤 + 连通分量聚类，去除离散伪影，保留结构性缺陷区域

## 6. 论文的主要结论与发现

1. **几何驱动的验证有效**：通过比较体素可达性与 Navmesh 可达性，可以直接检测传统代理方法难以发现的结构性不一致

2. **RL 引导探索显著提升效率**：
   - 优先级 RL 变体比启发式采样减少约 **24%** 样本

   - 在相似检测率（约 92.6%）和覆盖率（约 87.4%）下实现

3. **探索效率的来源**：语义重要性加权使探索集中在交互区、巡逻路径等高价值区域，而非均匀分布

4. **框架实用性**：
   - 离线运行，无需运行时代理模拟
   - 可并行处理空间区域，支持大规模环境
   - 已集成到 Ubisoft 生产 QA 流程

5. **跨引擎适配性**：方法依赖几何和引擎查询，最小改动即可适配不同游戏引擎

## 7. 优点

1. **创新性问题定义**：将 Navmesh 验证重新定义为几何一致性问题，而非行为推断问题
2. **完全离线验证**：不依赖运行时 NPC 行为，消除对导航系统的假设依赖
3. **几何独立参考**：体素化可步行空间作为独立基准，不依赖 Navmesh 生成过程
4. **RL 仅用于探索**：强化学习不参与验证正确性判断，避免引入策略偏差
5. **语义感知探索**：结合游戏语义元数据（巡逻点、交互区等），优先检测高价值区域
6. **可扩展架构**：区域独立处理 + 并行化 + 批处理，适合大规模生产环境
7. **可解释的检测结果**：报告包含世界坐标、受影响区域和导航上下文，便于 QA 团队定位和分类缺陷

## 8. 不足与局限

1. **体素分辨率依赖**：
   - 更高分辨率提升敏感度但增加计算成本
   - 离散化可能引入局部误差，论文通过连通层级验证缓解

2. **语义优先级依赖元数据**：
   - 当游戏语义元数据（巡逻路径、交互区等）稀缺或不准确时，RL 探索策略效果下降

3. **仅支持地面行走**：
   - 方法假设地面行走，不支持多层垂直导航（如攀爬、梯子）
   - 高于地面的可步行区域（如屋顶）被排除

4. **碰撞几何假设**：
   - 假设碰撞数据与设计意图一致
   - 当碰撞数据简化或错位时，检测到的差异需要人工解读

5. **小规模环境性价比存疑**：
   - 在较小环境中，简单的穷举策略可能表现相当甚至更优

6. **算力信息缺失**：未披露训练硬件配置，难以评估方法的计算门槛和复现难度

7. **未来方向未验证**：层次化表示、多智能体探索、多层连通性等扩展方向尚处于概念阶段
