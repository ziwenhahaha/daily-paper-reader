## 摘要
基于可验证奖励的强化学习（RLVR）在 LLM 推理方面显示出强大潜力，但基于结果的 RLVR 在困难问题上效率较低，因为正确最终答案 rollout 很少，而样本级信用分配无法利用失败尝试中的部分进展。我们引入 SCRL（Subproblem Curriculum Reinforcement Learning），这是一个课程 RL 框架，从参考推理链中导出可验证子问题，并将最后一个子问题固定为原始问题。这将困难问题上的部分进展转化为可验证学习信号。在算法上，SCRL 使用子问题级归一化，在每个子问题位置独立归一化奖励，并将所得优势分配给对应答案片段，从而无需外部 rubric 或奖励模型即可实现更细粒度的信用分配。我们的分析表明，子问题课程能将困难问题带出梯度死区，并且原始问题越难，相对收益越大。跨七个数学推理基准，SCRL 优于强课程学习基线，在 Qwen3-4B-Base 上比 GRPO 平均准确率提升 +4.1 点，在 Qwen3-14B-Base 上提升 +1.9 点。在 AIME24、AIME25 和 IMO-Bench 上，SCRL 进一步使 Qwen3-4B-Base 的 pass@1 提升 +3.7 点、pass@64 提升 +4.6 点，表明其在困难推理问题上具有更好的探索能力。

## Abstract
Reinforcement learning from verifiable rewards (RLVR) has shown strong promise for LLM reasoning, but outcome-based RLVR remains inefficient on hard problems because correct final-answer rollouts are rare and sample-level credit assignment cannot use partial progress in failed attempts. We introduce SCRL (Subproblem Curriculum Reinforcement Learning), a curriculum RL framework that derives verifiable subproblems from reference reasoning chains and fixes the final subproblem as the original problem. This turns partial progress on hard problems into verifiable learning signals. Algorithmically, SCRL uses subproblem-level normalization, which normalizes rewards independently at each subproblem position and assigns the resulting advantages to the corresponding answer spans, enabling finer-grained credit assignment without external rubrics or reward models. Our analysis shows that subproblem curricula lift hard problems out of gradient dead zones, with larger relative gains as the original problem becomes harder. Across seven mathematical reasoning benchmarks, SCRL outperforms strong curriculum-learning baselines, improving average accuracy over GRPO by +4.1 points on Qwen3-4B-Base and +1.9 points on Qwen3-14B-Base. On AIME24, AIME25, and IMO-Bench, SCRL further improves pass@1 by +3.7 points and pass@64 by +4.6 points on Qwen3-4B-Base, indicating better exploration on hard reasoning problems.

## 2. 论文提出的方法论

### 核心思想

受竞赛数学中多部分问题的启发：将难题分解为难度递增的可验证子问题序列，让模型在单次 rollout 中按顺序回答所有子问题，从而：
- 正确解决早期子问题的答案自然成为后续问题的推理基础
- 中间结果提供可验证的过程级监督
- 实现细粒度信用分配

### 关键技术细节

**（1）子问题构建**
- 给定参考解决方案，使用外部 LLM（如 DeepSeek-V3.2）将推理链中的中间节点重写为 K=4 个可验证子问题
- 子问题需满足：
  - 难度递增：s(1) ≺ s(2) ≺ ... ≺ s(K)=原问题
  - 前期为后期提供有用中间结果，但各问题独立可答
  - 每个问题有客观可验证答案

**（2）课程 rollout 结构**
- 使用结构化标签格式回答：`<p1>答案1</p1><p2>答案2</p2>...`
- 标签用于标识答案范围，便于后续信用分配

**（3）进度感知奖励校正（Progress-aware Correction）**
- 计算最大连续成功子问题数 ki（课程进度）
- 奖励向量 r = (r^(1), ..., r^(K)) 校正为：仅保留连续成功前缀
- 示例：(1, 1, 0, 1) → (1, 1, 0, 0)
- 防止奖励黑客：不在后续问题获得信用除非早期问题已解决

**（4）子问题级归一化**
- 对每个子问题位置 j，在小组内独立归一化奖励：
  ```
  A_i^(j) = (R_i^(j) - mean({R_i^(j)})) / std({R_i^(j)})
  ```
- 各位置的归一化优势独立计算，实现细粒度信用分配

**（5）Token 级信用分配**
- 使用结构化响应格式，将子问题级优势映射到对应答案 token 范围
- 若 token 位于 <pj> 和 </pj> 之间，则获得位置 j 的优势
- 答案范围外的 token 获得零优势

**（6）混合组训练**
- 半数 rollout 来自课程提示 tK(x)，使用 token 级优势
- 另一半来自原问题提示 x，使用标准 GRPO
- 减少提示不匹配，确保训练与评估一致

### 算法流程（Algorithm 1）

1. 采样问题 x，构造课程提示 tK(x)
2. 采样 G/2 个课程 rollout
3. 提取 K 个答案范围，验证每个子问题答案
4. 计算课程进度 ki，应用进度感知校正
5. 对每个子问题位置 j 独立计算优势 A_i^(j)
6. 分配 token 级优势给对应答案 token
7. 采样 G/2 个原问题 rollout，计算标准 GRPO 优势
8. 使用 SCRL 目标函数更新策略

## 6. 论文的主要结论与发现

### 核心发现

1. **SCRL 一致超越强基线**：在 Qwen3-4B 上平均提升 +4.1 分，Qwen3-14B 上 +1.9 分
2. **难题增益更大**：
   - 在 AIME24/25、IMO-Bench 上 pass@1 提升 +3.7 分
   - pass@64 提升 +4.6 分
3. **子问题级归一化有效**：比仅用子问题作为提示的 Both-GRPO 和去除校正的版本表现更好
4. **不依赖高质量生成器**：即使使用较弱的 Qwen3-4B-Instruct 生成子问题，仍有 +2.7 分提升
5. **课程进展可迁移**：训练中解决的子问题比例提升，且该能力迁移回原问题解决

### 理论洞察

- 子问题分解可将难题从梯度死区中提升
- 恢复比随问题难度增加而增长（Ω(1/δ)）
- 难题越难，子问题课程相对增益越大
