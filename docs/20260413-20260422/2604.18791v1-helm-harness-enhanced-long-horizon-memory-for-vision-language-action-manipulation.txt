# Abstract 

Vision-Language-Action (VLA) models fail systematically on long-horizon manip-ulation tasks despite strong short-horizon performance. We show this failure is not resolved by extending context length alone in the current reactive execution setting: it arises from three execution-loop deficiencies—the memory gap (loss of cross-phase task context), the verification gap (no pre-execution action validation), and the recovery gap (no failure detection or rollback). We present HELM , a model-agnostic framework addressing each gap with a dedicated learned component: an Episodic Memory Module (EMM) with CLIP-indexed keyframe retrieval, a learned State Verifier (SV) that predicts action failure before execution from (observation, action, subgoal) triples, and a Harness Controller (HC) implementing memory-conditioned rollback and replanning. The SV is the core learning contribution: we show it consistently outperforms rule-based feasibility checks and ensemble uncer-tainty baselines, and that its effectiveness depends critically on memory-augmented context. On LIBERO-LONG, HELM improves task success rate by 23.1 pp over OpenVLA (58.4% → 81.5%); extending the VLA context window to H=32 yields only +5.4 pp, suggesting the gap is not closed by longer context windows in the current reactive execution setting. Ablations and mechanism analyses isolate each component’s contribution. We release LIBERO-Recovery, a perturbation-injection evaluation protocol for failure recovery. 

# 1 Introduction 

VLA models [Brohan et al., 2023a, Kim et al., 2024, Black et al., 2024, Octo Model Team et al., 2024] achieve impressive results on short-horizon manipulation: OpenVLA reaches 91.2% on LIBERO-SPATIAL (avg. 2.3 subgoals). On LIBERO-LONG (avg. 5.8 subgoals) the same model drops to 58.4%—a 32.8 pp gap that does not close with longer context windows or larger backbones (§3). 

The gap is not closed by extending context length in the current reactive execution setting. 

We ran OpenVLA with history H=32 (4 × the default): TSR rises to only 63.8%, leaving 17.7 pp unexplained. We identify three execution-loop deficiencies that persist regardless of backbone scale: 1. Memory gap ( FM ). The fixed context window discards completed-subgoal evidence. At step t=47 , the model cannot recall that the mug was placed in the cabinet at t=12 ; it re-attempts the placement, corrupting task state. 2. Verification gap ( FV ). Actions are proposed reactively with no pre-execution feasibility check. Infeasible grasps, wrong-object contacts, and out-of-workspace motions execute silently and propagate error. 3. Recovery gap ( FR). After a failed action the model continues on a corrupted state, causing cascading failures across subsequent subgoals.

HELM addresses each gap with a dedicated component (Figure 1). The Episodic Memory Module 

(EMM) maintains a CLIP-indexed keyframe store and injects retrieved context into the VLA’s language input. The State Verifier (SV)—our core learning contribution—is a lightweight MLP trained on rollout data to predict action failure before execution; crucially, it takes memory-augmented context as input, making its effectiveness dependent on the EMM. The Harness Controller (HC) orchestrates the loop: retrieve → propose → verify → execute-or-recover. 

Why the SV is the core contribution. Pre-execution failure prediction is qualitatively different from post-hoc reflection [Shinn et al., 2023] or uncertainty-based filtering: it must reason jointly over the current visual state, the proposed action, and the task history. We show (§5) that (a) a rule-based verifier achieves only +6.8 pp vs. the SV’s +8.4 pp; (b) ensemble uncertainty achieves +9.5 pp but requires 5 × the inference cost; (c) the SV degrades by 6.1 pp when memory context is removed, confirming the EMM–SV coupling.

LIBERO-Recovery evaluation protocol for failure recovery (§5). 22 Related Work 

VLA models. RT-2 [Brohan et al., 2023a], OpenVLA [Kim et al., 2024], π0 [Black et al., 2024], and Octo [Octo Model Team et al., 2024] all share a reactive execution paradigm: map current observation to action without persistent memory or external verification. RT-1 [Brohan et al., 2023b] and Diffusion Policy [Chi et al., 2023] are strong task-specific baselines but lack generalization. HELM is orthogonal to backbone choice. 

Memory for embodied agents. FILM [Min et al., 2022] uses semantic maps for navigation; Voy-ager [Wang et al., 2023] maintains a skill library for open-world exploration; Generative Agents [Park et al., 2023] use importance-weighted memory streams. None addresses multi-stage manipulation with physical state constraints. Our EMM differs in three ways: keyframe-boundary write triggers, CLIP-based visual retrieval (not text-only), and tight coupling with a learned failure predictor. 

LLM agent frameworks. ReAct [Yao et al., 2023], Reflexion [Shinn et al., 2023], and CRITIC [Gou et al., 2024] operate on text and perform post-hoc correction. HELM’s SV performs pre-execution 

prediction on visual-physical state—a fundamentally different and harder problem. Inner Mono-logue [Huang et al., 2022] uses hand-crafted success detectors; our SV is learned and requires no manual engineering. 

Failure recovery. SayCan [Ahn et al., 2022] and ProgPrompt [Singh et al., 2023] do not model execution-time failure. Code as Policies [Liang et al., 2023] generates plans but has no recovery loop. PaLM-E [Driess et al., 2023] integrates perception and language but operates reactively. The closest prior work is Inner Monologue [Huang et al., 2022], which replans on failure but uses hand-crafted detectors and has no pre-execution verification. ACT [Zhao et al., 2023] and RLBench [James et al., 2020] provide strong manipulation baselines but do not address long-horizon failure recovery. 

Pre-execution verification. Learned feasibility predictors have been studied in motion plan-ning [James et al., 2020] but not in the VLA setting where the action space is high-dimensional and the failure signal is semantic rather than purely geometric. HELM’s SV is the first learned pre-execution failure predictor that conditions on episodic memory, enabling it to detect failures that are only apparent in the context of prior task history.

4.3 State Verifier (SV): Core Learning Contribution Motivation. Pre-execution failure prediction requires reasoning over the joint distribution of visual state, proposed action, subgoal context, and task history. This cannot be reduced to rule-based feasibility checks (which miss semantic errors and require environment-specific engineering) or uncertainty heuristics (which are action-agnostic and cannot distinguish a confident-but-wrong action from a genuinely feasible one). Formally, we seek P (fail t | ot, a t, g t, Mt); the memory context Mt

is essential because whether an action is valid often depends on what has already been completed (e.g., placing an object that was already placed is a failure regardless of current visual feasibility). The novel problem formulation—memory-conditioned pre-execution failure prediction—is the core contribution; the MLP architecture is a deliberate choice for low-latency inference (12 ms/step) and is validated by the ablation showing that the memory-augmented input ˆot is the critical factor, not model capacity (§5.4). We show empirically that a learned SV consistently outperforms both rule-based and ensemble alternatives (§5). 

Model. The SV is fψ : (ˆ ot, a t, g t) → [0 , 1] , a 3-layer MLP [1024 →512 →256 →1] with ReLU and dropout 0.1. Input: 

ˆot = [ ϕ(ot); ktop-1 ] ∈ R1024 , (2) concatenated with projected action Waat ∈ R256 and subgoal embedding ϕtext (gt) ∈ R256 . The memory-augmented observation ˆot is critical: removing ktop-1 degrades AUROC from 0.847 to 0.791 (§5.4). 

Training. Collect 50K (ot, a t, g t, y t) triples from VLA rollouts on training tasks; yt=1 if failure occurs within 5 steps. Binary cross-entropy with positive weight 4.0. Adam, lr 10 −4, batch 256, ∼2h on one A100.

4.4 Harness Controller (HC) 

The HC maintains a subgoal stack S (initialized by prompting πθ to decompose τ ) and a completion detector (same architecture as SV, trained on completion labels). On failure signal ( pfail >θ v or completion detector negative): retrieve the most recent checkpoint /success entry from EMM, issue a goal-conditioned recovery sequence (prompt: “return to the state shown”), re-push the failed subgoal, and append the failure entry to context. Maximum Rmax =3 recovery attempts before marking task failed. 

Forward recovery variant (HELM-Fwd). For environments where rollback is infeasible, HC instead generates a forward recovery plan from the current corrupted state. We evaluate both variants (§5).

5.2 Main Results 

Table 2 reports all baselines. Key findings: 

Longer context is insufficient. H=32 yields only +5.4 pp (63.8%), confirming the memory gap is structural. Oracle Memory yields +14.0 pp (72.4%), showing EMM’s contribution is real but not the only factor. 

Same-budget backbone adaptation is insufficient. OpenVLA + LoRA (50K) reaches 69.3% TSR— better than H=32 but 12.2 pp below HELM. This confirms that the same rollout budget is more effectively spent on training the SV than on fine-tuning the backbone. 

Learned SV outperforms alternatives. Rule Verifier: +6.8 pp. Ensemble (×5): +9.5 pp at 5 ×

inference cost. HELM’s SV: +8.4 pp (from ablation, Table 3) at 1 × cost. The SV is both more accurate and more efficient than ensemble uncertainty. 

Rollback vs. forward recovery. HELM-Fwd achieves 76.3% TSR and 38.7% RSR, confirming forward recovery is useful but rollback provides an additional +5.2 pp TSR and +15.5 pp RSR. For real-robot deployment, HELM-Fwd is the practical variant. 

Model-agnosticism. HELM (Octo) improves over Octo by 21.6 pp, a similar relative gain to HELM (OpenVLA), confirming the framework generalizes across backbones. 

5.3 Ablation Study 

Table 3 ablates each HELM component. EMM is the largest single contributor ( −11.2 pp without it). SV contributes −8.4 pp; notably, SV without memory context (“SV w/o mt”) contributes only −2.3 pp, confirming the EMM–SV coupling. Rollback contributes −6.3 pp to TSR and −31.1 pp to RSR.

# 6 Limitations 

SV data dependency. Training requires ∼50K rollout steps ( ∼2 h) per environment. One-time cost, but limits zero-shot deployment. 

Rollback in real robots. Rollback assumes reversible actions. HELM-Fwd addresses this but yields lower RSR (38.7% vs. 54.2%). Real-robot validation remains future work. 

Sim-to-real transfer. All experiments are conducted in simulation (LIBERO, CALVIN). Real-robot deployment introduces additional challenges: (a) the SV must generalize to visual domain shifts not present in rollout training data; (b) CLIP embeddings may be less discriminative under real sensor noise; (c) the perturbation magnitudes in LIBERO-Recovery ( ±5 cm) may not reflect the full range of real-world disturbances. We view HELM as a framework whose components (EMM, SV, HC) can be adapted to real settings, but direct sim-to-real transfer of the trained SV is not guaranteed and requires further study. 

Subgoal decomposition. VLA-generated decompositions fail in 4.3% of episodes; oracle decompo-sition adds +2.7 pp, suggesting a small but real bottleneck. 

Overhead. ∼12 ms/step (15% over backbone alone) on one A100. 

# 7 Conclusion 

HELM addresses three recurring failure modes of VLA models in long-horizon manipulation. The core learning contribution—the memory-conditioned State Verifier—consistently outperforms rule-based and uncertainty-based alternatives and is most effective when paired with the EMM. Com-prehensive experiments with 9 baselines, mechanism analyses, and oracle upper bounds show that the gains are not explained by simply adding more context or more modules. LIBERO-Recovery provides a reusable evaluation protocol for failure recovery. We hope HELM motivates further work on execution-time augmentation as a complement to backbone scaling. 7FM FV FR

A.1 Failure Taxonomy Annotation Protocol Annotators. Two annotators with robotics ML backgrounds independently labeled all 83 failed episodes from 200 OpenVLA LIBERO-LONG rollouts. 

Label definitions. 

• FM : the agent takes an action that is inconsistent with a subgoal completed more than H=8 

steps ago (e.g., re-attempting a completed placement, opening an already-open drawer). Identified by comparing the proposed action against the ground-truth subgoal completion log. • FV : the agent proposes an action that is physically infeasible or semantically incorrect given the current simulator state (e.g., grasping an object not in the gripper’s reachable workspace, targeting the wrong object). Identified by checking the simulator’s collision and reachability oracle at the proposed action. • FR: after a failed action (detected by a drop in task progress score), the agent continues executing subsequent subgoals on a corrupted state, leading to at least one additional failure within 10 steps. Identified by tracking task progress score over time. 

Multi-label policy. Episodes can receive multiple labels (18% of failed episodes had two or more modes). Annotators assigned all applicable labels. 

Agreement. Cohen’s κ=0 .81 (substantial agreement) computed over the binary label for each mode independently, then averaged. Disagreements were resolved by discussion; no episode required a third annotator. 

Distribution. Of 83 failed episodes: 41% FM , 33% FV , 26% FR (multi-label episodes counted in each applicable category). 

A.2 Implementation Details VLA backbone. OpenVLA-7B [Kim et al., 2024] in inference mode (no fine-tuning). For Octo experiments: Octo-Base checkpoint [Octo Model Team et al., 2024]. 

CLIP encoder. ViT-B/32 from OpenAI release. 512-dim embeddings, L2-normalized. 

State Verifier. 

• Architecture: MLP [1024, 512, 256, 128, 1], ReLU, dropout 0.1. • Input: [ϕ(ot); ktop-1 ] ∈ R1024 (Eq. 2), projected action Waat ∈ R256 , subgoal text embed-ding ∈ R256 .• Loss: BCE, positive weight 4.0. Adam lr 10 −4, wd 10 −5, batch 256. • Data: 50K steps from OpenVLA rollouts on LIBERO training tasks (not LIBERO-LONG). Failure label: yt=1 if task progress drops within 5 steps. • Training time: ∼2 h on one NVIDIA A100 80GB. 

Completion detector. Same architecture as SV; input [ϕ(ot); ϕtext (gt)] ∈ R1024 ; trained on comple-tion labels from same rollout data. 

Subgoal decomposition prompt. 

10 List the sequential subgoals to complete: [TASK]. Output as a numbered list. 

Fallback to single-subgoal if parsing fails. 

Memory parameters. k=3 , ∆c=20 , Nmax =50 , Rmax =3 .

Evaluation. 3 seeds. LIBERO-LONG: 500 episodes/seed. CALVIN: 1000 chains/seed. LIBERO-Recovery: 300 perturbed episodes/seed.

A.4 LIBERO-Recovery Protocol 

LIBERO-Recovery is an evaluation procedure applied to the existing LIBERO-LONG simulator, not a new dataset. At a randomly selected subgoal boundary k∗∈{ 2, . . . , K −1}, one of two perturbations is injected with equal probability: (a) object displacement ∼ U (−5cm , 5cm )3; (b) gripper state flip. The perturbation is silent (no visual indicator). RSR = fraction of perturbed episodes where the agent completes the full task.
