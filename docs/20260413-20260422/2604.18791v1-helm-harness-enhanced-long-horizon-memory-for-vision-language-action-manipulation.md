## 摘要
视觉-语言-动作（VLA）模型尽管在短程任务中表现强劲，但在长程操纵任务中却存在系统性失效。我们表明，在当前的反应式执行设置下，仅靠延长上下文长度无法解决这一失效问题；相反，它源于执行循环中反复出现的三个缺陷：记忆鸿沟（memory gap）、验证鸿沟（verification gap）和恢复鸿沟（recovery gap）。我们提出了 HELM，这是一个与模型无关的框架，通过三个组件解决这些缺陷：一个情景记忆模块（EMM），通过 CLIP 索引的关键帧检索关键任务历史；一个学习型状态验证器（SV），根据观测、动作、子目标和基于记忆的上下文在执行前预测动作失败；以及一个护航控制器（HC），负责执行回滚和重新规划。SV 是核心的学习贡献：它始终优于基于规则的可行性检查和集成不确定性基准，且其有效性关键取决于对情景记忆的访问。在 LIBERO-LONG 数据集上，HELM 将任务成功率比 OpenVLA 提高了 23.1 个百分点（从 58.4% 提升至 81.5%），而将上下文窗口延长至 H=32 仅带来 5.4 个百分点的提升，且在相同预算下的 LoRA 适配仍比 HELM 低 12.2 个百分点。HELM 还提高了在 CALVIN 上的长程表现，并显著增强了受控扰动下的恢复成功率。消融实验和机制分析分离了各组件的贡献，我们发布了 LIBERO-Recovery，这是一个用于评估长程操纵中故障恢复的扰动注入协议。

## Abstract
Vision-Language-Action (VLA) models fail systematically on long-horizon manipulation tasks despite strong short-horizon performance. We show that this failure is not resolved by extending context length alone in the current reactive execution setting; instead, it stems from three recurring execution-loop deficiencies: the memory gap, the verification gap, and the recovery gap. We present HELM, a model-agnostic framework that addresses these deficiencies with three components: an Episodic Memory Module (EMM) that retrieves key task history via CLIP-indexed keyframes, a learned State Verifier (SV) that predicts action failure before execution from observation, action, subgoal, and memory-conditioned context, and a Harness Controller (HC) that performs rollback and replanning. The SV is the core learning contribution: it consistently outperforms rule-based feasibility checks and ensemble uncertainty baselines, and its effectiveness depends critically on access to episodic memory. On LIBERO-LONG, HELM improves task success rate by 23.1 percentage points over OpenVLA (58.4% to 81.5%), while extending the context window to H=32 yields only a 5.4-point gain and same-budget LoRA adaptation remains 12.2 points below HELM. HELM also improves long-horizon performance on CALVIN and substantially boosts recovery success under controlled perturbations. Ablations and mechanism analyses isolate the contribution of each component, and we release LIBERO-Recovery as a perturbation-injection protocol for evaluating failure recovery in long-horizon manipulation.

### 1. 论文的核心问题与整体含义
现有的视觉-语言-动作（VLA）模型（如 OpenVLA, Octo）在短程任务中表现优异，但在长程操纵任务中面临系统性失效。研究发现，这种失效并非单纯因为上下文长度不足，而是源于执行循环中的三个结构性缺陷：
*   **记忆鸿沟 (Memory Gap)**：固定窗口的上下文会丢弃已完成子目标的证据，导致机器人重复已完成的动作。
*   **验证鸿沟 (Verification Gap)**：模型反应式地生成动作，缺乏执行前的可行性检查，导致错误动作静默执行并传播误差。
*   **恢复鸿沟 (Recovery Gap)**：一旦发生错误，模型会在损坏的状态下继续执行，引发级联故障。

### 2. 论文提出的方法论：HELM 框架
HELM 是一个模型无关的增强框架，通过三个核心组件闭合上述“鸿沟”：
*   **情节记忆模块 (EMM)**：维护一个由 CLIP 索引的关键帧存储库。它在子目标完成、检测到失败或固定步长时写入关键帧。执行时，通过余弦相似度检索最相关的历史上下文，并将其序列化为文本注入 VLA 的语言输入。
*   **状态验证器 (SV) —— 核心贡献**：这是一个轻量级的 3 层 MLP 模块。它在动作执行前，综合当前观测 ($o_t$)、拟执行动作 ($a_t$)、当前子目标 ($g_t$) 和检索到的记忆 ($M_t$)，预测动作失败的概率 $P(fail)$。
*   **护航控制器 (HC)**：管理子目标栈。当 SV 预测失败概率超过阈值时，HC 触发恢复机制。它支持**回滚 (Rollback)**（返回到记忆中的最近检查点）或**前向恢复 (Forward Recovery)**（从当前损坏状态重新规划）。

### 3. 实验设计
*   **数据集与场景**：
    *   **LIBERO-LONG**：包含 10 个任务，平均 5.8 个子目标。
    *   **CALVIN (ABC→D)**：评估在未见环境中的长程链式操作。
    *   **LIBERO-Recovery (本文提出)**：在子目标边界注入随机扰动（如物体位移、夹具状态翻转），专门测试恢复能力。
*   **对比方法 (9 个基准)**：
    *   基础模型：OpenVLA (H=8, 16, 32, 64) 和 Octo。

### 4. 资源与算力
*   **硬件**：所有实验均在单张 **NVIDIA A100 80GB GPU** 上运行。
*   **训练时长**：状态验证器 (SV) 的训练需要约 5 万步的 rollout 数据，在 A100 上训练耗时约 **2 小时**。

*   **实验规模**：进行了大规模评测，LIBERO-LONG 包含 500 个评估回合，CALVIN 包含 1000 个链式任务，均使用 3 个随机种子取平均值。
*   **充分性**：实验设计非常全面。不仅对比了不同长度的上下文窗口，还通过消融实验验证了 EMM、SV 和回滚机制的各自贡献。
*   **客观性**：作者特别设置了“同等预算 LoRA”基准，证明了将数据用于训练验证器比直接微调骨干模型更有效，确保了对比的公平性。

### 6. 主要结论与发现
*   **结构性缺陷**：将上下文窗口从 8 延长到 64 仅能提升 6.7% 的成功率，证明长程任务的瓶颈在于执行逻辑而非模型容量。
*   **HELM 的优越性**：在 LIBERO-LONG 上，HELM 将 OpenVLA 的成功率从 **58.4% 提升至 81.5%** (+23.1 pp)。
*   **验证器的关键性**：学习到的 SV 优于基于规则的检查和集成方法。且 SV 的性能高度依赖于记忆上下文，移除记忆后其 AUROC 显著下降。
*   **恢复能力**：在扰动测试中，HELM 的恢复成功率 (RSR) 达到 54.2%，远高于原始模型的 12.3%。

### 7. 优点
*   **模型无关性**：可以无缝集成到任何现有的 VLA 模型（如 OpenVLA, Octo）之上。
*   **预执行干预**：不同于事后反思（Reflexion），SV 在错误发生前进行拦截，避免了不可逆的物理损坏。
*   **轻量高效**：通过 MLP 和 CLIP 检索实现，训练和推理成本极低，适合实时机器人控制。

### 8. 不足与局限
*   **数据依赖**：SV 的训练需要特定环境的 rollout 数据（约 2 小时采集量），限制了其完全零样本（Zero-shot）迁移的能力。
*   **回滚假设**：回滚机制假设环境动作是可逆的或可以重置的，在某些现实场景（如液体倾倒、物体破碎）中难以实现。
*   **Sim-to-Real 挑战**：实验主要在仿真环境（LIBERO, CALVIN）中完成，真实世界的传感器噪声和视觉偏移可能会影响 SV 的预测准确性。
