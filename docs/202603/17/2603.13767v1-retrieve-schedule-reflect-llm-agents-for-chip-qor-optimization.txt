## Abstract 

Modern chip design requires multi-objective opti-mization of timing, power, and area under strin-gent time-to-market constraints. Although pow-erful optimization algorithms are integrated into EDA tools, achieving high QoR hinges on effec-tive long-horizon scheduling, which relies heav-ily on manual expert intervention. To address this issue and automate chip design, we pro-pose an agentic LLM framework that schedules chip optimizations through direct interaction with EDA tools. The agent is grounded in natural lan-guage expertise expressed as a search tree through retrieval-augmented generation (RAG). We fur-ther improve scheduling quality with Pareto-driven QoR feedback through language reflection. Experimental results show that, compared with black-box search methods such as reinforcement learning, our framework achieves 10% greater timing improvement while consuming less power and area, with more than 4× speedup. The post-optimization QoR is also comparable to that achieved by human experts. Finally, the agent sup-ports customized tasks expressed in natural lan-guage, enabling preferential QoR trade-offs. The code and chip design data will be publicly avail-able at https://github.com/YiKangOY/ Open-LLM-ECO .

## 2. Preliminaries 

2.1. Chip Optimization 

Chips are optimized by EDA tools through iterative opti-mization algorithm usage. Among many optimization meth-ods, gate sizing (Ning, 1994; Fishburn & Dunlop, 1985) and buffer insertion/removal (Shi & Li, 2005; Van Ginneken, 1990) are the most prevalent algorithms. They directly trade off timing against power/area, and can be applied at multiple stages across logic synthesis, placement, and routing. Prac-tical implementations of these algorithms are integrated in both academic and commercial EDA tools (Ajayi & Blaauw, 2019; Synopsys, 2022). However, achieving satisfactory QoR still requires extensive human intervention to deter-mine objective priorities and algorithm configurations for optimization (Coudert, 2002). 

2.2. LLM for Chip Optimization 

In chip design, LLMs are commonly used as tool-using agents that read reports/logs and propose edits to flow scripts. For example, ChipNeMo studies domain adaptation and evaluates chip-design assistants for EDA script generation and bug analysis (Liu et al., 2023). ChatEDA similarly pro-poses an LLM-powered autonomous agent that decomposes tasks and generates scripts for RTL-to-GDSII flows (He et al., 2023). ORFS-agent is a representative example that optimizes an RTL-to-GDS flow by tuning flow parameters with an LLM (Ghose et al., 2025). However, practical QoR optimization is not a single-shot search over parameters. Engineers run many optimization iterations and decide the next optimization target, algorithms (e.g., sizing vs. buffer-ing), and many other detailed algorithm options based on both QoR metrics and rich diagnostic information in tool logs and reports. Our work targets this missing layer by automating the scheduling of the iterative chip optimization process, rather than optimizing one stage or tuning a fixed set of EDA tool scripts. 

2.3. LLM for Task Scheduling 

Many recent LLM agent systems work as schedulers that iteratively choose of actions/tools, observe intermediate re-sults, and revise the schedule over long horizons. ReAct illustrates this view by interleaving reasoning with action selection to update the next-step plan after each environment interaction (Yao et al., 2023). Two mechanisms are especially useful for improving scheduling quality. 2.4. Problem Formulation 

In this paper, we aim to automate the scheduling of chip QoR optimization with an multi-agent framework. Given a designer-defined optimization task, the framework iter-atively selects an objective from power, timing, and area, and then schedules an EDA tool algorithm together with its options/parameters. Due to QoR saturation, the agent launches new rounds of scheduling and execution iterations with reflection on feedback from previous rounds. The opti-mized chip produced in the final scheduled round is used to evaluate the resulting QoR. 

## 3. Search Tree–based Retrieval-augmented Scheduling 

We adopt a Retrieval-augmented Generation (RAG) mecha-nism to guide agents during scheduling with domain knowl-edge and EDA tool manuals. Previous work proposes mod-eling sequential chip optimization as a decision tree (Pei et al., 2023) to explore different optimization paths. Inspired by this formulation, we model scheduling as a search tree (ST) whose nodes contain natural-language contents that encode engineer scheduling strategies. At runtime, the agent retrieves relevant ST paths and uses them as guidance for scheduling decisions.

## 4. Agentic Schedule–Execute–Reflect Framework 

Building upon the ST-based RAG in Section 3, we present an agentic framework for long-horizon chip QoR optimiza-tion scheduling. An overview is shown in Figure 3. The framework consists of three agents: a scheduler, an executor, and a reflector. Starting from an unoptimized design, the framework per-forms iterative schedule–execute steps within each round. During these iterations, RAG provides state-matched expert guidance and tool-manual grounding so that scheduling de-cisions are informed and tool execution remains legal. Once QoR improvement saturates within a round, the framework restarts from the unoptimized design and launches a new round, as shown in Figure 3 (a). Across rounds, the reflector agent summarizes feedback from historical traces and trans-fer it to the scheduler and executor to improve subsequent schedules.

4.2. Across-round Pareto-driven Reflection 

As more optimization algorithms are applied, QoR improve-ment often saturates and the flow can get stuck in a local optimum (Coudert, 2002; Neto et al., 2022). Engineers therefore draw feedback from previous optimization traces and restart from the unoptimized design to explore alter-native schedules that can reach better QoR. Since QoR is multi-objective, Pareto dominance is widely adopted to mea-sure trade-offs (Geng et al., 2022; Bai et al., 2024). 5.2. Optimization Result 

Our primary objective and the prompt is: Achieve timing closure by ensuring non-negative slack, while minimizing power and area . Following common industrial practice, timing slack is treated as a hard constraint and is prioritized first, since negative slack indicates functional failure of the design. Once timing violations are resolved, power and area are optimized as secondary objectives under the satisfied timing constraint. We report results in Table 1, comparing our method against reinforcement learning (RL) and manual scheduling by an experienced engineer. We first evaluate our framework by post-optimization tim-ing slacks. Compared with both human scheduling and our agentic framework, the RL baseline exhibits substan-tially larger negative slack, although it leaves only 10.92% negative slacks unsolved. Our framework improves total slack substantially, leaving only 0.23% unsolved and ex-ceeds human baseline. This indicates that timing violations are mitigated broadly across the design rather than being confined to a small subset of paths. 6Retrieve, Schedule, Reflect: LLM Agents for Chip QoR Optimization 

In terms of worst slack, our method delivers improvements of over 90% relative to the original designs, bringing the most critical timing paths close to or into the non-negative regime. This level of improvement is comparable to that achieved by experienced human engineers and consistently surpasses the RL baseline, which demonstrating that our proposed framework maintains competitive critical-path re-pair capability while more effectively managing multiple interacting critical and near-critical paths in designs with se-vere timing violations. For designs that still exhibit negative slack after optimization (“NVDLA partition m/p” and “hid-den5”), our method achieves larger improvements in both worst slack and total slack relative to the original designs, indicating the strong capability in mitigating widespread and interacting timing violations. We next compare the post-optimization power and area across all methods. Relative to the original designs, our method reduces area by 4.6% on average, while achieving slight power improvement without sacrificing timing clo-sure, and it consistently outperforms both the human and RL baselines. Compared with human scheduling, our method achieves comparable or lower power and area on most de-signs, while significantly improving over RL, which often incurs unnecessary area and power overheads. On several designs such as “NVDLA partition (m/p)”, our method re-sults in slightly higher area and power than the baselines. This behavior is expected, as these designs suffer from se-vere timing violations and require aggressive timing repair, where buffers insertions and upsizing cells gates introduce inevitable area and power trade-offs. In contrast, the RL baseline tends to over-optimize tool command sequences without explicit long-horizon task coordination, leading to suboptimal trade-offs between timing repair and power/area recovery. Overall, the results indicate that our framework ef-fectively balances timing closure with power/area efficiency when scheduling iterative optimizations through commer-cial EDA tools. The scheduler prioritizes timing repair in early iterations; once timing violations are resolved, it shifts attention to power and area optimization. In contrast, the RL baseline tends to remain focused on timing-oriented actions without explicit long-horizon coordination, which can lead to unnecessary power/area increases even after timing is largely repaired.

## 6. Conclusion 

Long-horizon chip QoR optimization remains a bottleneck in modern EDA workflows, not because optimization al-gorithms are lacking, but because effective optimization requires the scheduling of EDA tool usage with exper-tise chip design knowledge. However, black-box search is inefficient since it only relies on numerical QoR feed-back, without domain knowledge. Motivated by how expert engineers schedule the optimization, we introduced Re-trieve–Schedule–Reflect , the first agentic LLM framework that automates chip optimization by explicitly modeling scheduling as an iterative, knowledge-guided process with self-improvement through reflection. By integrating search tree-based retrieval with Pareto-driven reflection, our framework combines domain expertise, long-horizon scheduling, and quantitative feedback to progres-sively refine schedule strategies. Experiments on 8 designs using commercial EDA tools show that our approach con-sistently outperforms reinforcement learning baselines and matches or exceeds expert human scheduling. Moreover, the ability to directly interpret optimization preference in natu-ral language enables flexible, user-defined multi-objective trade-offs that are difficult to achieve with conventional methods. Ablation studies further confirm that both retrieval augmentation and multi-round reflection are essential to robust performance. 8Retrieve, Schedule, Reflect: LLM Agents for Chip QoR Optimization

## A. Scheduling Search Tree 

Here we discuss the contents in the nodes of search tree and further paths. Each node v in the search tree contains a corresponding objective o among power, timing, area. There is also a high-level summary s over design state that, like “The design is suffering from severe timing violation of x% of clock period”, or “The power compared to initial design increased by x%”, etc. It also contains a language-instructed strategy G over the use of EDA tools like “explore various gate sizing options” or “aggressively recover power even with negative slack”. There is also a segment of tool manual associated with it that contains the correct use of EDA tools, which we would not show for license restrictions. The tree is built by summarizing academic papers and technical reports and are filtered by engineers to ensure the tree is valid in terms of chip optimization. For example, there will not be the case that vi have summary si of “timing violation is small” while the next node vi+1 has si+1 denotes that “timing violation is significant” when timing optimization is performed. By extracting nodes in the search tree into a path, we could develop scheduling template that contains possible transition of designs states s and corresponding strategies. And example path is shown in the following paragraph. It use natural language to summarize the design state changes and objectives selected during the path. It also have a sequence of description regarding high-level usage of EDA tools like exploring different options or increasing the efforts. The schedulers perform RAG at each schedule step so they can always retrieve the most relevant paths based on design state and QoR changes in one round of execute-schedule iterations.

> Timing (Aggressive) →Timing (Incremental) →Power →Timing (Incremental) ** Design summary **: The design exhibits severe timing violation, after aggressive timing optimization, there are several remaining violations. So we perform incremental timing optimization with different algorithm. After they are fixed, we do power recovery. The power recover reveals new changes to incrementally optimize timing degradation brought by power optimization. ** Target selection **: Use an aggressive timing pass to gain slack fast, then an incremental timing pass to stabilize, then optimize power, and end with incremental timing cleanup. Switch away from timing if it becomes well-exploited with no gains. ** Option selection **: Aggressive timing is ** exploration-heavy ** (broad option coverage) to expose limits. Incremental timing is ** exploitation-heavy **, using only options shown to help and avoiding prior no-effect attempts. Power stage starts with exploration to find which power components can be reduced safely, then exploits the best component-focused options. ** Guidelines **: Don’t chase truly non-fixable timing violations. If aggressive timing increases power/area, explicitly recover power afterward (and note whether area needs alater pass). After power changes, always run atiming cleanup to repair any regressions while minimizing new side-effects.
