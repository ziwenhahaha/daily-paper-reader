# Abstract 

Modern deep neural network architectures are trained via backpropagation, which requires errors to be sequentially propagated through all layers before parameters can be updated. This introduces two limitations: locking, where layer-wise updates are strictly interdependent and cannot proceed in parallel, and the weight transport problem, which requires symmetric forward and backward pathways for exact gradient computation. These constraints restrict parallelism, increase memory and communication overhead, and pose challenges for scalable learning. In this work, we propose Hierarchical Block-Local Learning (HBLL), a framework that decom-poses deep neural networks into hierarchically linked blocks trained using local learning objectives derived from variational principles, eliminating the need for full end-to-end backpropagation while maintaining effective information propagation across the network. HBLL is the first algorithm that is able to train deep neural networks in O(log N ) parallel time complexity, where N is the number of network layers. We show that HBLL implicitly defines a family of subnetworks corre-sponding to different hierarchical paths, enabling flexible inference with different effective numbers of layers. We evaluate HBLL on a set of challenging vision and language modeling tasks, achieving competitive performance. We also extend HBLL to recurrent sequence architectures, applying to settings that otherwise rely on backpropagation through time. 

# 1 Introduction 

Error Backpropagation [Rumelhart et al., 1986], has emerged as the de facto standard method for training deep neural networks. This method computes gradients by propagating activations sequen-tially through all layers of a deep neural network, computing the loss at the output and propagating the gradients sequentially backward through the network. Backpropagation has demonstrated un-precedented scaling properties enabling deep neural networks with trillions of parameters and close to human-level performance on complex tasks including natural language modeling. However, these gains come at increasing computational and energy costs, driven not just by the model size but also by the training paradigm itself.

One major disadvantage of Backpropagation stems from the sequential error propagation scheme. Updates are locked until a full forward-backward pass across all layers of the network was performed. This locking problem limits the parallelization properties in learning and leads to reduced hardware utilization [Jaderberg et al., 2017]. The limitations of backpropagation have motivated the search for alternative learning methods that relax or remove the need for global error propagation. In particular, local learning approaches aim to train different parts of networks using locally defined rules, reducing inter-layer dependencies and improving scalability. However, while such approaches have shown promise in vision and smaller-scale settings, their ability to scale to large network sizes remains limited. In this work, we propose Hierarchical Block-local learning (HBLL), a framework that enables scalable gradient estimation in deep neural networks without full end-to-end backpropagation. HBLL decomposes networks into hierarchically organized blocks trained using local objectives derived from a variational formulation, providing a general and theoretically grounded framework for local learning. By enabling communication between levels of the hierarchy through invertible transformations, HBLL allows information to propagate across the network while avoiding global gradient dependencies. We provide a principled probabilistic interpretation of learning without global backpropagation, that enables distributed training. Our results demonstrate that HBLL scales to vision classification and autoregressive language modeling, achieving competitive performance on CIFAR-10, CIFAR-100 [Krizhevsky, 2009] and WikiText-103 [Merity et al., 2017]. We also apply HBLL to recurrent neural networks (RNNs), formulating a hierarchical local-learning approach for training sequential models. The main advantage of our hierarchical network learning approach is that it can be efficiently parallelized. The parallel compute complexity allows us to reason about the compute efficiency of algorithms that run on multiple parallel resources. The parallel compute complexity T∞ denotes the optimal level of parallelization an algorithm can achieve on an optimal number of parallel compute nodes (c.f. Cormen et al. [2022], p.779f). The compute complexity of conventional error backpropagation is O(N ), since it propagates information forward and backward through the network. We present here the first T∞ = O(log N ) parallel algorithm. The contributions of this paper are summarized as follows: • We propose a Hierarchical Block-Local Learning (HBLL), a framework for training deep neural networks without full end-to-end backpropagation by decomposing the network into hierarchically organized blocks optimized using local objectives. • We introduce a probabilistic learning framework, derived from a variational formulation, that enables learning without global gradient propagation while supporting distributed and parallel training with O(log N ) parallel compute complexity. • We demonstrate that HBLL achieves competitive performance on tasks including vision classification and autoregressive language modeling, on benchmarks such as CIFAR-10, CIFAR-100 and WikiText-103. 2• We show that HBLL implicitly defines a family of subnetworks corresponding to different hierarchical paths, enabling flexible inference under varying computational constraints. • We formulate HBLL-RNN, a recurrent variant of HBLL that trains a sequential model using hierarchical local learning. 

# 2 Related Work 

A large number of training algorithms based on local learning in deep neural networks (DNNs) have been proposed in the literature. Early approaches relied on fixed random feedback weights [Lillicrap et al., 2014, Nøkland, 2016] to propagate the model error to individual layers. Subsequent work introduced learnable feedback weights [Kolen and Pollack, 1994, Akrout et al., 2019], which improved performance over fixed random projections. Another line of research, known as target propagation [Lee et al., 2015, Meulemans et al., 2020], replaces error signals with targets propagated through the network [Frenkel et al., 2021, Ernoult et al., 2022, Shibuya et al., 2023]. While our method also employs learnable feedback weights, it differs in that it derives a principled hierarchical feedback structure to update weights and scales naturally to language modeling. Another family of methods adopts contrastive learning objectives [Chen et al., 2020, Oord et al., 2019], where models are trained using positive and negative examples, from which local learning rules at the layer or block level can be derived [Xiong et al., 2020, Illing et al., 2021]. The forward-forward algorithm [Hinton, 2022, Zhao et al., 2023] is a prominent example, in which inputs paired with correct or incorrect labels are propagated through the network. Extensions include adding a prediction head at the output layer and using inter-class samples as alternatives [Aghagolzadeh and Ezoji, 2025], show that Forward-Forward-style training can be scaled successfully to Vision Transformers [Dosovitskiy et al., 2021]. Forward-pass-only methods such as layer-wise zero-order gradient approximations estimate parameter updates through parameter perturbations [Malladi et al., 2023] rather than backpropagated gradients. In contrast, our work does not rely on contrastive objectives or zeroth-order gradient estimates but instead focuses on probabilistic block-wise local learning. A number of approaches target block-wise or layer-wise optimization directly [Ye et al., 2026]. Decoupled neural interfaces [Jaderberg et al., 2017] use auxiliary “utility networks” to approximate backward gradients, thereby decoupling layer training. More recently, Ma et al. [2024] propose using parts of subsequent layers as auxiliary networks, introducing complex dependencies across blocks during backpropagation. In our case, auxiliary networks are independent, except for a shared predic-tion head across blocks. Kappel et al. [2023] derive local losses from a probabilistic interpretation of DNN training, but do not scale to ImageNet. We also adopt a probabilistic formulation for local losses [Salvatori et al., 2025]; however, our learning is hierarchical, and auxiliary networks are not all directly connected to the output layer. Finally, Nøkland and Eidnes [2019] combine a matching loss with a local cross-entropy loss but do not scale to ResNet architectures. To the best of our knowledge, these approaches have not been evaluated across both attention-based vision architectures, such as Vision Transformers, and natural language processing tasks. 

# 3 Hierarchical Block-Local Learning 

3.1 Neural nets, chains and trees 

Deep neural networks (DNNs) are universal function approximators that map an input x to an output 

y = f (x), via a transfer function f : x → y, (x, y) ∈ D x × D y, where Dx, Dy are some (possibly high-dimensional) domains. Typically f is constructed by chaining together a sequence of N nested operations, (f1 ◦ f2 ◦ · · · ◦ fN )( x) = f (x) = y, where an = fn(an−1) are the transfer functions with activations an. N can denote the number of layers in the network. However, in many modern neural network architectures it is natural to think of this chain of operations in terms of more complex functional blocks, such as transformer blocks that consist themselves of multiple layers, and the theoretical framework we present here is agnostic to the level of granularity that is chosen. It can be insightful to adopt a probabilistic interpretation of the chain of operations performed by a DNN. To make this explicit we use the reparameterization trick [Kingma and Welling, 2013] to introduce latent variables zn = an + εn, where εn is block-wise independent noise with small amplitude. Using this, we can interpret the sequence of transformations to define a Markov chain 3over latent representations. # 4 Experiments 

We evaluate HBLL across supervised classification, autoregressive language modeling, and recurrent sequence modeling tasks. We first study MNIST [Lecun et al., 1998] using multilayer perceptrons to test whether HBLL can train deeper feedforward networks with local objectives. We then evaluate Vision Transformer-style architectures [Dosovitskiy et al., 2021] on CIFAR-10 and CIFAR-100 [Krizhevsky, 2009] to assess whether HBLL remains effective on attention-based vision models under increasing task complexity. Next, we evaluate HBLL on WikiText-103 [Merity et al., 2017] to study its applicability to autoregressive language modeling. Finally, we extend HBLL to recurrent neural networks(RNNs) and evaluate the resulting HBLL-RNN models on sequential MNIST and permuted-sequential MNIST. Across these experiments, we compare HBLL primarily against standard end-to-end backpropagation (BP) using matched architectures and training settings. Where applicable, we also report contrastive Forward-Forward local-learning results from prior work as an additional reference point [Aghagolzadeh and Ezoji, 2025]. Implementation details and hyperparameters for each experiment are provided in Appendix 6.5. Additional subnetwork analyses are reported in Appendix 6.6.

4.5 Recurrent Sequence Modeling 

Although the main experiments in this work focus on feedforward and Transformer-style architec-tures, the HBLL formulation can also be adapted to recurrent sequence modeling. This is natural because recurrent architectures define Markovian transitions over hidden states, which align with the conditional factorization in Theorem 1. In this setting, the hierarchy is interpreted over time indices, and each node summarizes a block of past inputs. However, the recurrent cell deployed at inference must be causal: the lowest-depth cell consumes only the previous hidden state and the current input, yielding a standard autoregressive recurrence. To ensure this, we obtain a causal variant through a single change to how the inter-node message is formed. 4.5.1 Experimental Setup 

We evaluate the recurrent variant on sequential MNIST and permuted-sequential MNIST. In both, each image is flattened to its 784 pixels; permuted-sequential MNIST additionally reorders those 784 pixels by a single fixed random permutation (the same permutation for every image, shared across train and test). This destroys spatial locality, so the model can no longer rely on neighbouring-pixel structure and must integrate information across the whole sequence. At tree depth D the input is split into N = 2 D steps, each consuming ⌈784 /2D ⌉ consecutive (permuted) pixels; the pixels are start-padded with leading zeros to N ⌈784 /2D ⌉ so the split is even and the deployed chain ends on real pixels. In every case the deployed model is the single depth-0 base cell unrolled over this full sequence, which we compare against a backpropagation-through-time (BPTT) recurrent baseline of identical cell type, hidden width, and parameter count. We test two cells, a GRU [Cho et al., 2014] and a gateless Elman-style RNN cell (a single linear layer with sigmoid activation) [Elman, 1990], over depths 1 through 8 for the GRU and 1 through 6 for the gateless cell. Optimizer, input preprocessing, and training schedule are shared between the two methods within each task.

# 5 Discussion 

Our results demonstrate that hierarchical block-local learning can scale to vision classification tasks and autoregressive language modeling. We first show that HBLL remains effective on MNIST classification with simple feedforward MLPs across increasing depths. On CIFAR-10, HBLL achieves competitive accuracy with Vision Transformer-style architectures and produces useful shorter hierarchical paths. On CIFAR-100, HBLL further remains competitive under a larger label space, matching or slightly improving over the corresponding backpropagation baseline in top-5 accuracy. In addition, the CIFAR-10 corruption experiments show that HBLL improves robustness under missing-data and label-noise settings, suggesting that the hierarchical block-local objective can provide useful regularization when inputs or supervision are degraded. On WikiText-103, while there is a slight performance gap compared to end-to-end backpropagation, HBLL provides a viable approach to language modeling without relying on full end-to-end backpropagation. We further show that the same hierarchical local-learning principle can be extended to recurrent neural networks, where HBLL-RNN remains competitive with BPTT on sequential and permuted-sequential MNIST while using a causal recurrent cell at inference. By removing the requirement for full backpropagation 12 across the entire network, HBLL addresses the locking problem and reduces inter-layer dependencies during training. This enables parallel and distributed training across blocks. The local learning objectives in HBLL are derived from a variational formulation, providing a principled probabilistic foundation for training without global backpropagation. HBLL also implicitly defines a family of subnetworks corresponding to different hierarchical paths, enabling flexible inference with varying computational budgets. Another interesting direction is to investigate whether the subnetworks induced by different hierarchical paths can support efficient inference schemes such as speculative decoding, where shallower paths could serve as draft models and deeper paths as verification models. Despite the promising results, a gap remains between HBLL and fully backpropagated models, particularly at larger depths. Future work includes improving the efficiency of local objectives, exploring richer invertible transformations (e.g., flow-based couplings) etc.

6.4 Linear Gaussians with known variance 

In the vision and language modeling experiments we consider case of the linear Guassian random varialbes with known scalar variance σ2. Thus, in our HBLL framework, the latent random variables are distributed according to a Normal distribution zn ∼ p (zn | zn−1) = N (zn | μzn|zn−1 , σ 2I),where μn = an is the mean directly governed by the block activations. For this convenient choice we can compute the posterior distribution p (zn | zn−1, zn+1 ), as 

zn ∼ p (zn | zn−1, zn+1 ) = N (zn | 12 μzn|zn−1 + 12 μzn|zn+1 , 12 σ2I) . (29) To perform the Bayesian inversion μzn|zn+1 with an arbitrary neural network block, we used the first order Taylor series approximation 

μzn|zn+1 ≈ an + 12 ∇∥ an+1 − zn+1 ∥2 , (30) where ∥·∥ 2 denotes the Euclidean norm, and zn+1 is here the sampled latent variable from the next level in the hierarchy. Thus, variational posterior samples were generated according to 

zn = an + 14 ∇∥ an+1 − zn+1 ∥2 + ε , with ε ∼ N (ε | 0, 12 σ2I) . (31) 

6.5 Experimental Details 6.5.1 MNIST Classification 

For MNIST, we use a simple multilayer perceptron with sigmoid activations and no residual con-nections or normalization layers. We evaluate hierarchy depths D ∈ { 2, 3, 4}, corresponding to networks with 4, 8, and 16 layers, respectively. For each depth, we vary the hidden dimensionality over {50 , 100 , 500 }. The same architecture depth and hidden dimensionality are used for the BP and CFF+M baselines whenever applicable. The exact hyperparameters for the MNIST classification are as follows: we use a batch size of 64, the AdamW optimizer [Loshchilov and Hutter, 2019], a learning rate of 0.001, and train for 100 epochs. For the HBLL model , the latent posterior variance is fixed to one and activations are treated deterministically without adding any noise. 18 6.5.2 CIFAR-10 Classification 

For CIFAR-10, we use a Vision Transformer architecture consisting of a patch embedding layer, Transformer blocks, and a classifier head. The HBLL model uses a depth-3 hierarchy. The full path contains seven Transformer blocks, while the short path contains three Transformer blocks. We compare against end-to-end backpropagation with corresponding effective depths. We report CFF+M baselines from the prior work Aghagolzadeh and Ezoji [2025] when available. For all CIFAR-10 experiments, we use the same data augmentation pipeline across HBLL and BP consisting of random cropping, horizontal flipping, and RandAugment [Cubuk et al., 2020]. The reported CFF+M result from prior work uses the same augmentation settings. For missing-data and label-noise experiments, M denotes the percentage of missing input data and 

N denotes the percentage of noisy labels. The setting (M, N ) = (0 , 0) corresponds to the clean uncorrupted training setting. The exact hyperparameters for the CIFAR-10 classification are as follows: we use a batch size of 256, the AdamW optimizer [Loshchilov and Hutter, 2019], a learning rate of 0.0005, and train for 500 epochs. For the HBLL model, posteriors are sampled using the reparameterization trick with a constant noise scale of 0.005 .

6.5.3 CIFAR-100 Classification 

For CIFAR-100, we use the same Vision Transformer architecture as in the CIFAR-10 experiments, consisting of a patch embedding layer, Transformer blocks, and a classifier head. The HBLL model uses a depth-3 hierarchy. The full path contains seven Transformer blocks, while the short path contains three Transformer blocks. We use the same data augmentation pipeline across HBLL and BP, consisting of random cropping, horizontal flipping, and RandAugment [Cubuk et al., 2020]. The reported CFF+M result from prior work uses the same augmentation settings. We compare against end-to-end backpropagation and CFF+M baselines with corresponding effective depths. Since CIFAR-100 contains 100 classes, we report top-5 classification accuracy, following Aghagolzadeh and Ezoji [2025]. The exact hyperparameters for CIFAR-100 classification are as follows: we use a batch size of 256, the AdamW optimizer [Loshchilov and Hutter, 2019], a learning rate of 0.0005, and train for 500 epochs. For the HBLL model, posteriors are sampled using the reparameterization trick with a constant noise scale of 0.002 .

6.5.4 Language Modeling on WikiText-103 

For language modeling, we use a GPT-2-style Transformer architecture based on Karpathy [2022]. Each HBLL block corresponds to one Transformer block. We evaluate a depth-3 hierarchy, where the full path contains seven Transformer blocks and the short path contains three Transformer blocks. Performance is measured using perplexity on WikiText-103. All language-modeling experiments use the same tokenization, optimizer settings, and training schedule for HBLL and the corresponding backpropagation baselines. The exact hyperparameters for the language-modeling experiments are as follows: we use a batch size of 2, a context window of 1024 , the AdamW optimizer [Loshchilov and Hutter, 2019], a learning rate of 10 −4, weight decay of 0.01 , and train for 10 epochs. We use mixed-precision training with bfloat16. For the HBLL language model, posteriors are sampled using the reparameterization trick with a constant noise scale of 0.005 .

6.5.5 (Permuted-)Sequential MNIST Classification (HBLL-RNN) 

For sequential and permuted-sequential MNIST we use the recurrent variant of HBLL (HBLL-RNN), whose N time steps are arranged in a binary tree and trained block-locally; the deployed model is the single depth-0 cell rolled over the full sequence as an ordinary RNN. We evaluate two cells: a gated GRU and a gateless Elman cell (a single linear layer with sigmoid activation), both at a fixed hidden width of 2048 . We sweep the tree depth D, with sequence length N =2 D ; we report D ∈ { 1, . . . , 8}

for the GRU and D ∈ { 1, . . . , 6} for the gateless Elman cell, beyond which the deployed Elman chain becomes unstable. Each image is flattened to its 784 pixels and, at depth D, split into N steps of 

⌈784 /2D ⌉ consecutive pixels with minimal start-padding; permuted-sequential MNIST additionally 19 applies a single fixed random permutation to the 784 pixels, shared across train and test. The same cell type, hidden width, and parameter count are used for the BPTT baseline, which is trained by full 

O(N ) backpropagation through time. The exact hyperparameters for the (permuted-)sequential MNIST experiments are as follows: we use a batch size of 128 , the AdamW optimizer [Loshchilov and Hutter, 2019], a learning rate of 

0.001 , weight decay of 10 −4, gradient-norm clipping at 5.0, and train for 200 epochs. We use mixed-precision training with bfloat16. HBLL-RNN is trained block-locally at O(log N ) span, with all nodes at a given depth evaluated in one batched call and inter-node states detached. The deployed chain is additionally trained with the parallel fixed-point (consistency) correction at consistency weight 1.0, differentiating only through the final sweep; we select the number of iterations K per depth (the best K reported in each table, with K=0 denoting tree-only, no correction, which is used throughout for the gateless Elman cell). Activations are treated deterministically without added noise, and a single recurrent cell is shared across the tree so that the deployed depth-0 chain is one cell rolled. For the timing comparisons we implement the GRU cell explicitly, as plain matrix multiplications, for both HBLL and the BPTT baseline, rather than using PyTorch’s nn.GRU . The latter dispatches to a fused cuDNN kernel, which would give the baseline a kernel-level speed advantage unrelated to the training algorithm; since HBLL does not yet have an equivalent fused CUDA kernel, using the unfused cell for both sides isolates the algorithmic (training-span) speedup and keeps the comparison fair.
