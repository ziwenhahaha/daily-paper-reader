## Abstract
Despite the remarkable progress of Large Language Model (LLM) based Multi-Agent Systems, most research focuses on optimizing coordination topology while largely underexploring the equally critical problem: how to transmit and optimize messages among agents effectively? Current communication schemes typically rely on the direct concatenation of first-order neighbor responses, which induces a restricted evidence receptive field and leads to the dilution of crucial insights over multi-hop paths. To address these limitations, we propose the Multi-Order Communication (MOC) scheme, which reconstructs the inter-agent communication to capture multi-hop dependencies and incorporates a structural message consolidation strategy to ensure efficiency. Specifically, we formalize the communication mechanism to construct a structured multi-order evidence stream, and subsequently design a Semantic-Topological Merging algorithm to optimize semantic fidelity within token constraints. Extensive experiments across six diverse datasets and LLM backbones of varying parameter scales demonstrate that MOC consistently improves task performance and reduces communication costs. The code is available at https://github.com/yao-guan/MOC.

## 1. 核心问题与整体含义（研究动机和背景）

### 研究问题
当前LLM多智能体系统（MAS）面临一个关键但被忽视的问题：**如何有效地在智能体之间传输和优化消息？**

### 现存局限
- **受限的证据接收范围**：现有通信方案仅拼接直接（一阶）邻居的响应，导致智能体只能获取近邻信息
- **关键信息稀释**：多跳传播过程中，中间智能体对上游消息的转述、摘要和重新解释会引入语义衰减，造成信息累积性遗漏和偏差
- **上下文效率瓶颈**：直接扩展证据范围会导致提示长度快速增长，上下文利用率下降

### 研究动机
现有研究主要聚焦于拓扑优化（如链式、树状、完全连接图、动态拓扑等），但对通信机制本身的设计关注不足。本文旨在填补这一空白，提出一种**拓扑感知的通信方案**，使目标智能体能够访问多跳距离的原始上游响应。

## 2. 论文提出的方法论

### 核心思想
受图神经网络中MixHop的启发，MOC将多跳聚合思想引入LLM多智能体通信，通过**重建智能体间通信结构**来捕捉多跳依赖，同时结合**结构化消息合并策略**确保效率。

### 关键技术细节

#### 2.1 形式化建模
- 将LLM多智能体系统建模为有向无环图（DAG）：$G = (V, E)$
- 智能体 $v_i$ 关联生成函数 $f_i(\cdot)$，由内部状态 $\theta_i = (\text{Role}_i, \text{Plugins}_i)$ 参数化
- 边 $e_{i \to j}$ 表示从 $v_i$ 到 $v_j$ 的信息流

#### 2.2 K阶消息流构造
定义智能体 $v_j$ 的k阶响应集合：
$$S_j^{(k)} := \{R_i \mid k_{\min}(i, j) = k\}$$

其中 $k_{\min}(i, j)$ 是从 $v_i$ 到 $v_j$ 的最短可达跳数。通过远-近全局排序，构造输入上下文：
$$M_j := \bigoplus_{k=1}^{K} S_j^{(k)}, \quad \Pi_j := \pi_j^{(K)} \| \pi_j^{(K-1)} \| \cdots \| \pi_j^{(1)}$$

#### 2.3 语义-拓扑合并算法（Semantic-Topological Merging）
**目的**：在token约束下压缩多阶证据，同时保持语义保真度和拓扑优先级

**步骤**：
1. 使用轻量级文本编码器（如sentence-transformers）将消息映射到语义空间
2. 计算排序后的相似度矩阵 $S_{uv} = \cos(h_u, h_v)$
3. 识别高相似度对 $(u, v)$（$u < v$，$S_{uv} \geq S_{\max} - \epsilon$）
4. 使用小型预训练模型（如TinyLlama）蒸馏合并候选集
5. 选择与原始源语义最接近的合并结果
6. 迭代执行直到满足消息数量约束 $B_{\text{msg}}$

**合并选择准则**：
$$R_v^* = \arg\max_{\hat{r} \in C_v} \sum_{x \in \{u,v\}} \cos(T(\hat{r}), h_x)$$

**消息数量约束**：
$$B_{\text{msg}} = \left\lfloor \frac{|M_j|}{K} \right\rfloor + \gamma K$$

#### 2.4 统一执行框架
$$R_j = f_j(\text{Psys},j \oplus Q \oplus \Psi \circ \Phi(M_j \mid \Pi_j))$$

其中 $\Phi$ 为消息合并操作符，$\Psi$ 为上下文组合操作符。

## 6. 论文的主要结论与发现

### 6.1 性能提升
- **Gemma-2-27B**：平均提升1.18%-2.30%，AQuA数据集增益最大（6.77%，$\rho=0.3$）
- **Qwen2.5-32B**：MMLU提升3.84%（$\rho=0.7$），HumanEval提升3.68%（$\rho=0.3$）
- **DeepSeek-V3.2-685B**：MMLU-Pro提升0.82%

### 6.2 最优参数
- **K=2** 被确认为最优接收范围，平衡了信息丰富度和冗余控制
- 过高K值（K=3）在密集图上可能引入冗余，性能反而下降

### 6.3 效率改进
- **Token节省**：20智能体场景下，输入Token从13.69×10⁵降至12.49×10⁵，低于Vanilla MAS的13.38×10⁵
- **输出Token减少**：DeepSeek-V3.2上MMLU-Pro输出从795k降至759k，表明更丰富的上下文带来更果断的生成

### 6.4 稀疏拓扑优势
MOC在稀疏连接（$\rho=0.3$）下优势更明显，因为多阶消息传输有效扩展了受限的证据接收范围

### 6.5 互补性
MOC可与拓扑优化方法（如G-Designer）互补，在任务自适应拓扑上仍有提升（1.0%-4.8%）
