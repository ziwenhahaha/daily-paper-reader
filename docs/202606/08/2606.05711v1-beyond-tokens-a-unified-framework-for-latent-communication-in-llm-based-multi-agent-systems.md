## Abstract
Multi-agent systems built on large language models (LLMs) have become a prevailing paradigm for tackling complex reasoning, planning, and tool-use tasks. The dominant communication protocol in such systems is natural language: agents exchange messages token-by-token, verbalising their internal reasoning so that peers can read, verify, and respond. While convenient and interpretable, this protocol suffers from three structural drawbacks -- high inference cost, irreversible information loss during discretization, and ambiguity/redundancy of natural language. A growing body of work therefore explores an alternative protocol -- latent communication -- in which agents exchange continuous representations (embeddings, hidden states, or KV-caches) directly, bypassing the bottleneck of text generation. This paper presents a unified framework for organising the rapidly expanding literature on latent communication. We analyse existing methods along three orthogonal axes: (1) WHAT information is communicated (Embeddings, Hidden States, KV-Caches, or other continuous state); (2) WHICH sender-receiver alignment is used (latent-space alignment and layer alignment); and (3) HOW the communicated information is fused into the receiver (concatenation, prepending, mathematical operations, cross-attention, or cache restoration). Under this 3-axis framework, we systematically categorise eighteen representative methods proposed between 2024 and 2026, identify five major design patterns, and surface a set of open challenges -- including cross-architecture alignment, security of latent channels, compression for edge deployment, and the relationship between latent communication and latent chain-of-thought. We hope that this framework both lowers the barrier to entry for new researchers and provides a vocabulary for comparing future work.

## 1. 论文核心问题与整体含义（研究动机和背景）

### 研究动机
当前基于LLM的多智能体系统主要采用**自然语言通信协议**，即智能体间通过逐token生成自然语言消息进行交互。这种协议存在三个结构性缺陷：

- **高推理成本**：每次通信需要发送方解码（T tokens）× 接收方重新编码，额外开销约为O(L·T·d) FLOPs
- **离散化过程中的信息丢失**：隐藏状态携带~40,000 bits信息，但每个token仅能传递~15-17 bits，压缩比达10³-10⁴倍
- **自然语言的冗余与歧义**：生成的文本优化的是语言流畅性而非任务相关信息密度

### 核心问题定义
**如何让LLM智能体在保持系统协作能力的同时，通过连续表示（embeddings、隐藏状态或KV-caches）直接交换信息，绕过自然语言瓶颈？**

### 背景
- 2024-2026年间，该领域已出现超过15种不同的潜在通信方法
- 方法设计碎片化，缺乏统一术语和分类体系
- 亟需一个框架来组织快速扩张的文献

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
提出一个**三维正交框架**，任何潜在通信方法都可唯一描述为：
```
Method = (WHAT | 信息类型, WHICH | 对齐方式, HOW | 融合策略)
```

### 三大轴线详解

#### 轴线1：WHAT — 通信信息的类型

| 类型 | 描述 | 代表方法 | 特点 |
|------|------|----------|------|
| **Embeddings** | 输入层连续向量 e_i ∈ R^d | CIPHER | 最低信息量，最低架构依赖性 |
| **Hidden States** | 中间层隐藏状态 h_i^(ℓ) | AC, Interlat, SDE, ThoughtComm, MoT | 中等信息量，需架构对齐 |
| **KV-Caches** | 注意力机制的key-value缓存 | KVComm, Cache-to-Cache, LatentMAS, Q-KVComm等9种方法 | 最高信息量，最高架构依赖性 |
| **其他** | 状态增量、持久化内存等 | SDE (delta), Agent Memory (Q4) | 特殊场景优化 |

#### 轴线2：WHICH — 发送方-接收方对齐方式

**潜在信息对齐**：
- **无需对齐**：同模型实例（相同backbone）
- **学习投影**：同架构异版本（如Llama-3的不同微调）
- **通用视觉编码器**：异构架构（如Llama ↔ Qwen）

**层级对齐**（Layer Alignment）：
- **Last → First**：发送方最后一层 → 接收方第一层
- **All → Corresponding**：所有层级一一对应
- **Selected → Selected**：选择关键层级（如KVComm的top-k选择）

#### 轴线3：HOW — 信息融合策略

| 策略 | 操作 | 代表方法 |
|------|------|----------|
| **Concatenation** | 沿token轴拼接 | CIPHER, Interlat |
| **Prepending** | 预置到接收方KV-cache | KVComm, LatentMAS, Q-KVComm |
| **Mathematical Operation** | 逐元素加法/投影 | AC, SDE, ThoughtComm |
| **Cross-Attention** | 交叉注意力机制 | MoT |
| **Cache Restoration** | 直接替换/恢复缓存 | RelayCaching, Agent Memory |
| **Hybrid** | 混合方式 | Edge LLM Handover |

### 五条通用设计规律（Takeaways）

1. **信息-成本-依赖性权衡**：KV-Cache > Hidden State > Embedding（信息量↑、传输成本↑、架构依赖性↑）
2. **Prefill vs. Decode选择**：KV-Cache方法集中于prefill阶段；Embedding/Hidden State方法集中于decode阶段
3. **层级映射策略**：同构智能体可选"Last→First"或"All→Corresponding"；异构智能体需学习对齐
4. **融合策略谱系**：Concatenation/Prepending最简单；Cross-Attention最表达力强但需训练
5. **设计空间未饱和**：3×3×5=45个概念位置，18种方法仅覆盖约17个

## 6. 论文的主要结论与发现

### 核心发现

1. **潜在通信的优势条件**：
   - 智能体紧密耦合（规划→执行链路）
   - 通信为中间过程（用户无需查看消息）
   - 发送方与接收方可对齐到共同潜在空间
   - 延迟是硬约束（实时流水线、边缘部署、大规模智能体数）

2. **方法选择指南**：
   - 相同backbone：Last→First + Concatenation 简单有效
   - 同架构异版本：学习投影头
   - 异构架构：通用视觉编码器或交互层

3. **性能数量级**：
   - 延迟降低：2-24×
   - TTFT加速：4.7×-136×
   - Token节省：3-4×
   - 信息压缩比：10³-10⁴（相对自然语言）

4. **领域趋势**：
   - KV-cache方法正在成为主流（9/18种方法）
   - 训练-free方法占主导（约13/18种）
   - 跨架构通信仍是开放挑战

### 六大开放问题

1. **跨架构对齐**：需O(N)通用对齐而非O(N²)成对翻译器
2. **安全与鲁棒性**：潜在通道的对抗攻击和隐私泄露未充分研究
3. **压缩与量化**：KV-cache传输的压缩空间巨大
4. **理论理解**：缺乏何时/为何潜在通信优于自然语言的理论保证
5. **与Latent CoT的统一**：单智能体内推理与多智能体通信的框架整合
6. **真实场景部署**：边缘设备的电池、内存、带宽约束

## 8. 不足与局限

### 实验覆盖局限

1. **直接对比缺失**：各方法在各自backbone和benchmarks上评估，缺乏统一比较
2. **规模有限**：18种方法可能未覆盖全部重要工作（领域仍在快速演进）
3. **模拟依赖**：部分方法（如Edge LLM Handover）仅模拟评估

### 偏差风险

1. **发表偏差**：可能更多收录训练-free/成功方法，失败案例未充分报告
2. **backbone偏见**：大量方法假设同构智能体，异构场景覆盖不足
3. **短期评估**：缺乏长期部署稳定性、累积误差等问题的分析

### 应用限制

1. **解释性牺牲**：潜在通道对人类不透明，难以审计和调试
2. **安全空白**：对抗攻击、隐私泄露风险未充分研究
3. **跨组织场景**：当发送方和接收方由不同组织控制时，潜在通道共享的实际可行性存疑
4. **实时性假设**：部分方法假设通信同步进行，未充分考虑异步场景

### 理论局限

1. **无理论保证**：缺乏何时潜在通信必然优于自然语言的收敛性/最优性证明
2. **信息度量模糊**：未精确定义"信息量"的度量标准和上界
