## Abstract
The application of multi-agent systems in robotics is a very challenging field. Several competitions involving such systems are proposed to foster research and development of strategies and mechanisms using games as the underlying domain. Among them are the ones from the \textit{IEEE Very Small Soccer (VSSS)} category, which is the case study described in this paper. In VSSS, two teams of three robots each compete in a very dynamic environment of a soccer game. Thus, coordination of robots' behavior during the game is crucial to win it. In this paper, we present a Behavior-Tree-based approach to support multi-robot coordination within the VSSS team of the ThundeRatz robotics team from the Universidade de S$\tilde{a}$o Paulo. Moreover, a comparison between the proposed approach and the previous one, which was based on a Finite State Machine (FSM), was conducted using the FIRASim simulator. Besides that, the performance of this new strategy was further evaluated in an academic robotics competition.

- **研究动机**：探索一种更具模块性、灵活性和可维护性的控制架构，以实现动态任务切换和角色分配，同时不牺牲原有的实时响应能力。  

## 2. 方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想
- **组织模型**：使用 MOISE⁺ 对团队结构进行建模，定义了七种角色（守门员、射手、助理、全场后卫、边路后卫、点球踢手、点球防守者）以及角色之间的兼容性（组内/组间兼容），并引入 “Coach” 作为全局领导者负责角色分配。  
- **行为树（BT）**：用行为树替代 FSM 来实现 Coach 的决策逻辑。BT 具备层次化、模块化、可视化优先级等优势，能够自然表达“若满足条件 A 则执行动作 B，否则执行 C”这类逻辑。  
- **黑板（Blackboard）**：使用共享黑板存放全局状态（如 `team_state` = attacking/defending、`use_two_strikers_mode`、`use_penalty_mode`），实现子节点之间的状态读写，便于统一管理团队进攻/防守状态。

### 2.2 关键技术细节
| 关键组件 | 功能描述 |
|----------|----------|
| **初始化子树（Initializer Subtree）** | 根据比赛暂停或自动裁判（VSSReferee）的事件消息设置初始角色；取代了原 FSM 中的多个初始状态。 |
| **角色交换子树（Roles Swapper Subtree）** | 动态判断当前是进攻还是防守，并通过对应的交换子树执行角色切换。根节点采用 **Fallback** 结构，优先检查进攻分支，若不满足则执行防守分支。 |
| **进攻交换子树（Attack Swapper Subtree）** | 依次检查：① 双射手模式 → 切换至单射手；② 点球模式 → 替换射手；③ 射手 ↔ 助理角色交换。 |
| **防守交换子树（Defense Swapper Subtree）** | 依次检查：① 点球防守模式 → 替换守门员；② 全场后卫 ↔ 边路后卫 交换；③ 守门员 ↔ 边路后卫 交换。 |
| **优先级隐式表达** | 通过 BT 的顺序（Sequence）和回退（Fallback）节点显式排列角色切换的先后顺序，避免 FSM 中隐藏的转移优先级。 |
| **实现库** | 采用开源的 **BehaviorTree.CPP** 库，支持 tick‑driven 执行、动作/条件节点分离、插件式扩展。 |

### 2.3 算法流程（伪代码）

1. **比赛启动/暂停结束** → 进入 **Initializer Subtree**；读取裁判消息 → 设定初始角色集合并写入黑板。  
2. 进入 **Roles Swapper Subtree**，循环执行：  
   - 读取黑板 `team_state`；  
   - 若 `team_state == attacking` → 调用 **Attack Swapper Subtree**；  
   - 否则 → 调用 **Defense Swapper Subtree**。  
3. **Attack/Defense Swapper Subtree** 内部按顺序检查对应条件节点：  
   - 若满足 `use_two_strikers_mode == true` → 执行 “切换至单射手” 动作；  
   - 若满足 `use_penalty_mode == true` → 执行 “点球角色切换” 动作；  
   - 若满足 “射手 ↔ 助理” 交换条件 → 执行角色交换。  
4. 所有交换完成后返回成功（Success），进入下一轮 tick。  
5. 若需要切换进攻/防守状态（如防守转为进攻），在 **Roles Swapper** 中通过 “Set Blackboard `team_state` = attacking/defending” 触发切换。

## 7. 优点

- **模块化、可复用**：子树（初始化、角色交换）可独立开发、调试，便于团队分工。  
- **优先级显式**：通过 BT 结构自然呈现角色切换顺序，避免 FSM 中隐式的转移优先级导致的行为不透明。  
- **状态共享简洁**：使用黑板统一管理团队状态，条件检查与动作执行解耦，提升可读性。  
- **实验验证充分**：250 场仿真 + 5 场正式比赛，提供统计显著性验证。  
- **实战适配**：方案基于官方仿真器和自动裁判，符合真实竞赛规则，可直接部署。
