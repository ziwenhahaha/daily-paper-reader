## Abstract
Reinforcement learning with verifiable rewards (RLVR) improves language-model reasoning, but GRPO-style optimization remains prone to collapse. We analyse this instability through token-level gradient dynamics, deriving a taxonomy that predicts how updates affect next-token probabilities and entropy. The taxonomy shows that stability depends jointly on the advantage sign and token distribution under the current policy. Motivated by this finding, we propose Winner Advantage Policy Optimization (WAPO), a simple online clipped policy-gradient objective that updates only on positive-advantage completions. Across mathematical reasoning and multi-hop QA benchmarks, WAPO improves training stability and matches or outperforms baselines across multiple model families. Full code can be found at https://github.com/layer6ai-labs/wapo.

### 2. 方法论

#### 2.1 理论框架：梯度激励的 Token 分类体系

论文从一阶近似出发，分析策略更新对 next-token 概率的影响。对于采样的 token $s$，其概率变化满足：

$$\Delta p_i = \eta A p_i (C(p) - p_s - p_i) + O(\eta^2)$$

其中 $C(p) = \sum_j p_j^2$ 是概率分布的平方和，$A$ 是序列级优势，$\eta$ 是学习率。

基于此，论文定义了 **Peak** 和 **Valley** 两类 token：

- **Peak token**：$p_s \geq C(p)$（高概率采样 token）
- **Valley token**：$p_s < C(p)$（低概率采样 token）

结合优势符号（正/负），形成四象限分类：

| 分类 | 优势符号 | 熵变化 | 稳定性 |
|------|----------|--------|--------|
| Pos-Peak | 正 | 降低 | 稳定但保守 |
| Pos-Valley | 正 | **升高** | 易导致随机崩溃 |
| Neg-Peak | 负 | **升高** | 易导致随机崩溃 |
| Neg-Valley | 负 | 降低 | 初期有帮助，但可能引发低熵崩溃 |

#### 2.2 WAPO 算法

**核心思想**：仅在正优势完成上执行策略梯度更新，移除非正优势项。

**目标函数**：

$$J_{\text{WAPO}}(\theta) = \mathbb{E}_{x \sim \mathcal{D}} \left[ \frac{1}{GT} \sum_{i=1}^{G} \sum_{j=1}^{T_i} A_i^+ \cdot \min\left(\rho_{ij}(\theta), 1+\epsilon\right) \middle| \exists i \text{ s.t. } A_i^+ > 0 \right]$$

其中 $A_i^+ = \max(A_i, 0)$ 为截断后的正优势，$\rho_{ij}(\theta)$ 为 token 级重要性比率。

**关键特性**：

- 保留 GRPO 风格的优势归一化和裁剪机制
- 使用 $(1 - q_x)$ 作为自适应因子，$q_x$ 为当前策略的正确回答概率
- 自适应因子会在模型对某提示已较熟练时自动减弱更新幅度

### 6. 主要结论与发现

1. **梯度分类体系的预测能力**：四象限分类可准确预测不同更新对熵的影响，验证了理论分析的有效性

2. **崩溃机制**：
   - Pos-Valley 和 Neg-Peak 更新会升高熵，导致高熵崩溃（随机输出）
   - Neg-Valley 更新虽初期可提升奖励，但会降低熵，最终导致低熵崩溃（重复输出）

3. **WAPO 的优势**：
   - 在所有测试设置中保持训练稳定，无崩溃现象
   - 多跳 QA 任务提升显著：Qwen3-4B 在 OTT-QA 上达 37.44%（次优基线 27.57%）
   - 数学任务上与基线方法性能相当或略有优势

4. **保留探索能力**：WAPO 不仅在 Pass@1 上表现最佳，Pass@K 曲线也显示其维持了采样多样性

5. **域外泛化**：Hotpot-QA → 2-wiki 迁移中 WAPO 在所有模型上均取得最佳性能
