## Abstract
As LLM agents become capable of increasingly long-horizon tasks, evaluating their performance in economic systems is becoming increasingly important. Unlike existing benchmarks that primarily evaluate a single agent interacting with a passive environment, economic systems are inherently multi-agent, requiring autonomous agents to communicate, negotiate, and transact while pursuing their own objectives over extended periods. We introduce CoffeeBench, a benchmark for evaluating LLM agents in a long-horizon multi-agent economy composed of heterogeneous firms. In CoffeeBench, two farmers, two roasters, and two retailers autonomously operate their businesses over a 90-day simulation, each seeking to maximize cumulative net income through communication and transactions while managing cash, inventory, and pricing. The evaluated model controls one coffee roaster, while the remaining firms are controlled by fixed reference agents. Across several recent open-weight and proprietary LLMs, all models outperform a passive baseline that takes no actions, with most achieving positive net income. Analysis of agent behavior reveals substantial differences in long-horizon economic interaction: higher-performing models communicate more actively with other firms, whereas Claude~Haiku~4.5 exhibits an idle-drift failure mode, repeatedly choosing inaction despite producing coherent assessments and plans. We release our code and agent trajectories to support future research.

## 1. 论文的核心问题与整体含义

### 研究动机

- **现有评估范式的局限**：当前 LLM Agent 基准测试主要评估单一智能体与被动环境的交互，无法覆盖真实经济系统中多主体之间的**通信、协商、交易**等复杂动态行为
- **长时序自主决策的挑战**：随着 LLM 能力向长时序任务扩展（如代码智能体、Web 智能体），如何在**动态多智能体经济系统**中评估其表现成为关键问题
- **异构企业的缺失**：已有基准（如 Vending-Bench、Vending-Bench Arena）仅包含单一经济角色或同质化企业，缺乏异构企业间的交互评估

### 研究目标

- 构建一个能够模拟**异构多企业经济系统**的基准测试平台
- 评估 LLM 智能体在**供应链多层级、长期博弈**场景下的决策与协调能力
- 研究不同模型在经济交互中的行为模式差异

## 2. 论文提出的方法论

### 核心思想

**CoffeeBench** 是一个 90 天咖啡供应链经济模拟平台，包含六家企业：

被测模型控制 **roaster_A**，其余五家企业由固定参考模型（Claude Sonnet 4.6）控制。

### 关键技术细节

#### 2.1 时间管理机制

- **每日业务窗口**：09:00 - 19:00
- **主动行动时间成本**：每次工具调用推进本地时钟 30 分钟
- **空闲与重激活**：调用 `wait_for_next_day()` 进入空闲状态，可被外部事件（消息、交易报价等）重新唤醒
- **夜间系统更新**：消费者销售、运营成本、库存腐损、财务应计等

#### 2.2 工具集设计

**共享工具**（所有角色可用）：
- `post_listing()` - 发布销售挂牌
- `make_offer()` / `accept_offer()` - 交易谈判
- `pay_invoice()` - 结算应付账款
- `send_message()` - 自由形式消息通信

**角色专用工具**：

| 角色 | 专用工具 | 功能 |
|------|----------|------|
| 农民 | `produce_item()` | 生产绿咖啡豆，含成本与延迟 |
| 烘焙师 | `roast()` | 烘焙绿咖啡为成品，含成本与产出率 |
| 零售商 | `set_retail_price()` | 设置消费者价格 |
| 零售商 | `view_consumer_sales()` | 查看销售历史 |

#### 2.3 消费者需求模型

采用零售商级价格竞争模型，包含：
- **价格弹性**：基于消费者保留价格（商品 $30/kg，特色 $80/kg）
- **品牌忠诚度因子**：`loyalty_i ~ U(0.85, 1.15)`（每零售商随机）
- **季节性波动**：春季促销（Day 40-53）需求乘数 3.0
- **随机噪声**：`ε_t ~ N(0, 0.5²)`

需求池计算公式：
```
M_t = min( f(t) * M_max * D_0 / D_base, max(0, f(t)*D_0 + ε_t) * max(0, 1 - p̄/p_res) )
```

#### 2.4 经济约束

- **固定每日运营成本**：农民 $25/天，烘焙师 $30/天，零售商 $50/天
- **库存腐损**：每日 0.5%
- **存储容量限制**：农民/烘焙师 120kg，零售商 80kg
- **赊账条款**：Net-30，逾期罚款 0.1%/天
- **破产机制**：现金余额为负则退出模拟

#### 2.5 KPI 定义

$$\text{NetIncome} = \text{Revenue} - \text{COGS} - \text{OpEx} - \text{InterestExp} + \text{InterestRev}$$

- **COGS**：基于加权平均成本法，防止自交易套利
- **OpEx**：固定成本 + 腐损 + 坏账 + 库存减值

### 6.2 关键发现

#### 活跃通信与盈利能力正相关

- **GPT-5.5** 发送约 140 条消息，主要面向上下游企业
- **Gemini 3.1 Pro** 仅发 16 条消息但读取 90 次，呈现"反应式"风格
- 同一层级竞争对手间**几乎没有直接通信**（≤1条/运行）

#### Idle-Drift 失败模式（Claude Haiku 4.5）

- 平均约 **40 天**处于空闲状态
- 推理痕迹**保持连贯**（如 Day 26 早晨评估"业务运行顺利"）
- 持续调用 `wait_for_next_day()` 而非采取行动
- 可能原因：长上下文积累导致行为转变，或过度保守的行动选择

#### 工具使用策略差异

- 高性能模型集中使用 `make_offer()` 和 `accept_offer()`
- **工具调用量 ≠ 盈利能力**：Kimi K2.6 调用量高但利润低
- **定价策略 > 交易量**：Gemini 3.1 Pro 交易量低但净收入高于 Kimi

#### 理论天花板分析

- 乐观双重垄断估计：**$23,800**
- GPT-5.5 实际表现：**$3,109**（约达 13%）
- 表明当前模型**仍有巨大提升空间**

## 8. 不足与局限

### 8.1 模拟真实性局限

- **供应链简化**：实际供应链层级更深、更复杂
- **生产过程抽象**：真实生产涉及更长时间跨度和不确定性
- **宏观经济因素缺失**：无融资机制、监管、复杂契约安排
- **单一角色测试**：仅评估 roaster_A，其他角色能力未知

### 8.2 统计可靠性不足

- **每模型仅 3 次运行**，方差较高时小差异可能不显著
- 建议未来增加运行次数或降低环境随机性以换取可靠性

### 8.3 行为多样性受限

- 参考代理固定为 Claude Sonnet 4.6，可能**限制市场动态多样性**
- 未探索不同背景代理组合对被测模型的影响

### 8.4 策略深度评估缺失

- 理论天花板基于极度乐观假设，实际可达性能可能更低
- 未充分测试**复杂协调策略**（如同层竞争者间的价格协调）

### 8.5 伦理与安全考量

- 论文承认可研究潜在不良行为（串通、操纵交易）
- 当前模型未展现此类行为，但风险随能力提升可能增加
