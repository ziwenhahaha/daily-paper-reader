## Abstract
Large language model (LLM) agents are increasingly deployed in multi-agent systems where they must coordinate and agree on shared decisions. We ask whether classical resilient consensus theory, developed for deterministic agents, transfers to LLM agents that may behave adversarially. Framing LLM agreement as a Byzantine consensus game, we run controlled experiments on complete and general communication graphs. We find that prompted LLM agents fail to reach agreement that is achievable in principle: consensus can fail even in settings where classical theory guarantees that a convergent algorithm exists, and this failure persists across temperatures and horizons. At the same time, wrapping the agents with classical resilient consensus filters improves agreement. The benefit of filtering depends on how much robustness the underlying topology already provides. Our results suggest that classical resilient consensus theory is a useful lens for the safety of agentic AI.

## 2. 方法论：核心思想、关键技术细节、算法流程

### 2.1 形式化：拜占庭共识游戏
- 将 LLM 智能体的协商过程建模为**同步、离散轮次的拜占庭共识游戏**。  
  - **诚实智能体**拥有初始整数（在 0‑50 范围内均匀采样），在每轮收到所有其他智能体的提案后，利用 LLM 生成新提案。  
  - **拜占庭智能体**可随意产生任意值（实验中被设定为极端低/高方向，以“拉散”诚实智能体的取值）。  
  - 目标：在 **T_max ≥ B+1** 轮后，所有诚实智能体的值相同（**有效共识**），否则视为**无效共识**。

### 2.2 经典过滤器的引入
- **Global MSR‑type Filter（适用于完全图）**  
  - 对于每个节点收到的多集合 \(V\)（所有智能体的提案），去除 **B** 个最小和 **B** 个最大元素：  
    \[
    \Pi_{\text{MSR}}(V, B) = \text{sort}(V)[B:|V|-B]
    \]  
  - 过滤后使用剩余值进行后续 LLM 提示（prompt）生成。  
- **Local MSR‑type Filter（适用于一般有向图）**  
  - 对每个节点 \(i\) 收到的邻居值集合 \(V_i\)，同样去除 **B** 个最小和 **B** 个最大：  
    \[
    \Pi_{\text{MSR}}(V_i, B) = \text{sort}(V_i)[B:|V_i|-B]
    \]  
  - 需要满足 **|V_i| ≥ 2B+1**（即每个节点的入度足够大），并在满足 **Theorem 2**（图划分条件）时保证近似共识。  

### 2.3 协议流程（每轮）
1. **提议阶段（Proposal）**：智能体根据上一轮的提案（以及过滤后残余的提案）构造提示，向 LLM 查询得到新整数提案。  
2. **广播阶段（Broadcast）**：所有智能体把自己的提案发送给所有邻居（或在完全图中发送给全体）。  
3. **过滤阶段（Filter）**（若使用过滤器）：每个诚实节点执行 Global 或 Local MSR‑type 过滤。  
4. **终止**：达到 \(T_{\max}\) 轮后判定共识是否达成。  

### 2.4 关键理论依据
- **完全图**：Theorem 1（Pease‑Shostak‑Lamport）指出 **N ≥ 3B+1** 为可达精确拜占庭协议的充要条件。  
- **一般有向图**：Theorem 2（Vaidya‑Tseng‑Liang）给出在 **N ≥ 3B+1** 且图满足 **C∪R ⇒ L 或 L∪C ⇒ R**（所有满足条件的划分）时，可通过 Local MSR‑filter + 确定性更新实现近似共识。

## 6. 主要结论与发现

1. **Prompt‑only LLM 智能体在对抗环境下容易失效**  
   - 完全图中，即使在 **N ≥ 3B+1**（理论保证可达共识）区域，baseline 的共识概率随 B 增大显著下降，且在 **B=0** 时也出现不低（约 70‑90%）的失败率。  

2. **经典 MSR‑type 过滤器显著提升共识成功率**  
   - **Global MSR‑filter**：在完全图上实现约 **100%** 共识概率，几乎消除了对抗影响（即使在理论上限之外仍保持高效）。  
   - **Local MSR‑filter**：在满足 Theorem 2 的图上同样提升共识率；在已经具备强连通的稠密图中增益有限（说明**拓扑本身的鲁棒性**决定过滤器的边际收益）。  

3. **失效不是理论错误，而是实现差距**  
   - 经典理论给出**存在**可达共识的算法，而不是所有提示（prompt）策略都能实现。LLM 的**“自然”提示**并不能自动满足弹性共识的数学要求，导致实际共识率低于理论上限。  

4. **过滤器的鲁棒性**  
   - **温度变化**、**轮数（Horizon）变化**、以及 **B 参数错配**（过滤器参数 f 大于真实 B）均未显著削弱过滤效果，说明该方法对实际部署中的不确定性具备一定容错能力。  

5. **图拓扑决定过滤器收益**  
   - 在高度连通（完全图）时，过滤器作用显著；在本来就满足强鲁棒条件的图中，增益有限。提示**在设计系统时需结合网络拓扑评估过滤器的必要性**。

## 8. 不足与局限

- **对抗行为简化**：实验中的 Byzantine 智能体仅采用“极端值”策略，未模拟更高级的策略（如“渗透式”“协作式”攻击），可能导致对真实最坏情况的估计不足。  
- **模型规模单一**：仅使用 **3B 参数** 的 LLM，未在其他尺寸或不同厂商的模型上验证，结论的可迁移性有待进一步验证。  
- **缺乏对结构化决策的检验**：任务仅是**整数共识**，未涉及实际多智能体任务（如计划生成、任务分配等），因此对更复杂、真实场景的适用性仍需后续工作。  
- **实验规模限制**：完整图实验 30 次、消融 10 次，虽提供置信区间，但仍有统计噪声，尤其是高 B 情形下的共识概率估计可能波动较大。  
- **未披露算力细节**：缺少 GPU 型号、运行时间等资源信息，读者难以直接复现实验成本。  
- **过滤器的理论保证依赖已知 B**：在实际系统中，真实 Byzantine 数量往往是未知或仅上界已知。虽然文中提供了错配实验（B 被高估时仍有效），但对**低估 B** 的鲁棒性未进行系统评估。
