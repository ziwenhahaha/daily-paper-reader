## 摘要
基于大语言模型（LLM）的自动多智能体系统（MAS）生成已成为解决复杂任务的关键前沿。然而，现有方法在模型能力与经验保留之间面临两难局面。推理时 MAS 利用冻结的前沿 LLM，但在不学习过去经验的情况下重复相同的搜索。相反，训练时 MAS 通过梯度更新内化经验，但受限于较小模型的低能力上限，且难以扩展到大规模前沿 LLM。为弥合这一差距，我们提出 Skill‑MAS，这是一种将经验保留与参数更新解耦的第三条路径，将高层编排能力概念化为可演进的 Meta‑Skill。Skill‑MAS 通过闭合优化循环对该结构化知识进行细化：(1) Multi‑Trajectory Rollout 在当前 Meta‑Skill 下为每个任务采样行为分布； (2) Selective Reflection 自适应地选择优先任务，并运用层次化对比分析将系统性经验提炼为可泛化的策略级原则。在四个复杂基准和四个不同 LLM 上进行的大量实验表明，Skill‑MAS 不仅取得了显著的性能提升，而且保持了良好的成本‑性能权衡。进一步分析表明，进化后的 Meta‑Skill 具有高度鲁棒性，并在未见任务和不同 LLM 上表现出强迁移能力。

## Abstract
Large Language Model (LLM)-based automatic Multi-Agent Systems (MAS) generation has become a crucial frontier for tackling complex tasks. However, existing methods face a dilemma between model capability and experience retention. Inference-time MAS leverages frozen frontier LLMs but repeats identical searches without learning from past experience. Conversely, Training-time MAS internalizes experience via gradient updates but is constrained by the low capability ceiling of smaller models, and is hard to scale to large frontier LLMs. To bridge this gap, we propose Skill-MAS, a novel third path that decouples experience retention from parametric updates by conceptualizing the high-level orchestration capability as an evolvable Meta-Skill. Skill-MAS refines this architectural knowledge through a closed optimization loop: (1) Multi-Trajectory Rollout samples a behavioral distribution for each task under the current Meta-Skill; and (2) Selective Reflection adaptively selects priority tasks and applies hierarchical contrastive analysis to distill systemic experience into generalizable, strategy-level principles. Extensive experiments across four complex benchmarks and four distinct LLMs demonstrate that Skill-MAS not only achieves remarkable performance gains but also maintains a favorable cost-performance trade-off. Further analysis reveals that the evolved Meta-Skills are highly robust and exhibit strong transferability across unseen tasks and different LLMs.

## 2. 论文提出的方法论

### 核心思想
- **关键创新**：将 Meta-agent 的高层编排能力概念化为**可演进的 Meta-Skill**（结构化自然语言文档），而非参数化权重
- **设计哲学**：Meta-Skill 编码策略级原则，涵盖任务分解、智能体工程和工作流编排，但不包含任务特定的解决方案

### Meta-Skill 的三模块结构

| 模块 | 功能 | 说明 |
|------|------|------|
| **1. Task Decomposition（任务分解）** | "What" | 分析用户查询、识别宏观目标、将请求分解为离散的子任务、定义可评估的成功标准 |
| **2. Agent Engineering（智能体工程）** | "Who" | 实例化专业子智能体、分配角色画像和工具、提供操作所需的上下文输入 |
| **3. Workflow Orchestration（工作流编排）** | "How" | 选择拓扑结构（顺序/层级/循环）、定义智能体间的输入输出映射、生成可执行的 MAS |

### 核心算法流程

**闭合优化循环（Closed Optimization Loop）**：

#### 阶段一：Multi-Trajectory Rollout（多轨迹 rollout）
- 对每个任务 $t_i$ 在当前 Meta-Skill $S^{(r)}$ 下独立采样 $K$ 条轨迹
- 每条轨迹记录：$\tau_{i,k} = \{id_i, k, s_{i,k}, S^{(r)}, \Phi_{i,k}\}$
- 计算**任务不确定性** $u_i = \sqrt{\frac{1}{K}\sum(s_{i,k} - \bar{s_i})^2}$（衡量跨运行的一致性）
- 计算**任务难度** $d_i = -\bar{s_i}$（难度越高值越大）

#### 阶段二：Selective Reflection（选择性反思）
1. **优先级驱动任务选择**：
   - 将 $u_i$ 和 $d_i$ 归一化后融合为统一优先级 $p_i = \frac{1}{2}(\tilde{u}_i + \tilde{d}_i)$
   - 使用离散二阶差分检测"肘点"（elbow），自动确定需优先分析的任务子集

2. **层次化轨迹反思**：
   - **阶段1 - 任务内对比分析**：将 K 条轨迹分为高分集 $H_i$ 和低分集 $L_i$，对比诊断分歧点、成功因素、失败模式，归因根因，生成候选补丁
   - **阶段2 - 跨任务综合**：汇总各任务报告，识别跨任务的系统性弱点/优势，形成结构化证据包 $E$

3. **技能优化**：
   - 基于证据包 $E$ 移除/改写无效规则
   - 保留三模块脚手架，针对性优化特定模块
   - 所有修改必须抽象为可泛化的编排原则

## 6. 论文的主要结论与发现

### 核心结论
1. **性能提升显著**：Skill-MAS-optimized 在几乎所有实验设置中超越所有基线方法，取得最高平均性能

2. **成本-性能权衡优越**：
   - 训练时 MAS：成本最低但性能最差（生成简单的 MAS，缺乏泛化能力）
   - 推理时 MAS：性能提升但推理成本最高（每样本重复迭代优化）
   - **Skill-MAS**：一次性生成 MAS，达到最佳性能，同时保持中等成本

3. **Meta-Skill 迁移性强大**：
   - **同任务跨LLM**：表现最佳（如 VITA 任务从 DeepSeek→GPT 提升 63.10）
   - **同LLM跨任务**：具有竞争力（验证了任务无关策略的有效性）
   - **跨LLM+跨任务**：最弱但仍有提升（最极端挑战场景）

4. **技能演化轨迹一致**：
   - 从任务分解设计 → 智能体级认知控制 → 系统级韧性，逐步演进
   - 各数据集的优化技能收敛于可靠性导向控制（约束感知推理、结构化输出契约、验证门、回溯机制）

## 7. 优点

1. **范式创新——第三条路径**：
   - 首次将 Meta-agent 的高层编排行为建模为可演进的 Meta-Skill
   - 成功解耦经验保留与参数更新，使冻结前沿 LLM 具备迭代学习能力

2. **方法设计优雅**：
   - **多轨迹统计**：将单次结果转化为分布表征（不确定性+难度），区分系统性缺陷与瞬时噪声
   - **肘点自适应**：无需人工设定阈值，自动识别优先分析任务
   - **层次化对比分析**：从任务内到跨任务，逐层抽象系统性经验

3. **实验设计严谨**：
   - 多样化的基准覆盖（深度研究、数学、问答、工具调用）
   - 跨闭源/开源模型的泛化验证
   - 包含迁移性、消融等多元化分析

4. **实践价值**：
   - 一次性生成可复用，无需每样本迭代优化
   - 演化后的 Meta-Skill 可跨任务、跨模型迁移，降低部署成本

5. **可解释性强**：
   - 三模块结构明确归因失败模式
   - 优化过程可追踪、可解释

## 8. 不足与局限

1. **标签依赖性**：
   - 依赖 ground-truth 标签计算轨迹分数，在弱监督/无监督场景下效果下降
   - 消融实验显示（表3），无自适应优先级选择时性能显著下降

2. **实验规模相对有限**：
   - 部分测试集仅84-168条样本，统计可靠性待加强
   - 未提供置信区间或统计显著性检验

3. **消融实验不够深入**：
   - 缺少对 Multi-Trajectory Rollout 和 Selective Reflection 各子组件的逐个 ablation
   - 难以精确评估各模块的独立贡献

4. **演化效率**：
   - 10轮演化周期较长，K=5的rollout数在大规模场景下成本较高
   - 未讨论收敛性保证或早停策略

5. **多任务学习效果有限**：
   - 在聚合数据集上演化后，部分数据集（BCP）性能下降
   - 缺乏跨域适应的显式机制

6. **算力透明度不足**：
   - 缺乏训练/演化阶段的 GPU 使用、训练时长等详细信息
   - 难以与其他方法进行公平的资源消耗对比

7. **基线方法的公平性**：
   - 某些基线（如 MAS²）出现"Failed"（无法生成可执行代码），可能影响对比的公平性

8. **泛化边界未充分探索**：
   - 主要在专业基准上测试，对简单/模糊任务的适用性未知
   - 未探索 Meta-Skill 退化的风险（如过度拟合特定数据集模式）
