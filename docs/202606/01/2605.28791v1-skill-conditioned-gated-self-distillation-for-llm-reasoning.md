## 摘要
在策略自蒸馏 (SD) 通过使用教师侧特权信息 (PI) 将稀疏的验证器输出转化为密集的 token 级别监督，从而提升大语言模型的推理能力。现有方法通常假设可信的 PI，例如参考答案或成功的轨迹。我们探索 PI 是否可以来源于经验衍生的技能库，其中检索到的技能紧凑且可复用，但也可能不相关或具有误导性。我们提出技能条件门控自蒸馏 (SGSD)，将基于技能的 SD 表述为教师假设验证而非无条件模仿。SGSD 检索技能‑错误对，构建多教师池，并让所有技能条件教师对相同的纯提示学生 rollout 进行评分。验证器验证每位教师的极性：支持成功或抑制失败产生正向监督，而相反立场则被反转。随后，鲁棒的门控目标蒸馏出信息丰富的师生分歧，同时抑制不确定或极端的信号。在多个数学推理基准上的实验表明，SGSD 一致地优于 GRPO，并在较弱的 PI 假设下与答案条件的 OPSD 保持竞争力。例如，在 Qwen3‑1.7B 上，SGSD 在 AIME24、AIME25 和 HMMT25 上平均分别超过 GRPO 6.2% 和超过 OPSD 1.7%。我们的代码可在 https://github.com/walawalagoose/SGSD 获取。

## Abstract
On-policy self-distillation (SD) improves LLM reasoning by using teacher-side privileged information (PI) to turn sparse verifier outcomes into dense token-level supervision. Existing methods usually assume trusted PI, such as reference answers or successful traces. We ask whether PI can instead come from an experience-derived skill bank, where retrieved skills are compact and reusable but may also be irrelevant or misleading. We propose Skill-Conditioned Gated Self-Distillation (SGSD), which formulates skill-based SD as teacher hypothesis validation rather than unconditional imitation. SGSD retrieves skill-mistake pairs, constructs a multi-teacher pool, and lets all skill-conditioned teachers score the same plain-prompt student rollout. The verifier validates each teacher's polarity: supporting a success or suppressing a failure gives positive supervision, while the opposite stance is reversed. A robust gated objective then distills informative teacher-student disagreements while suppressing uncertain or extreme signals. Experiments on multiple mathematical reasoning benchmarks show that SGSD consistently improves over GRPO and remains competitive with answer-conditioned OPSD under a weaker PI assumption. For example, on Qwen3-1.7B, SGSD outperforms GRPO by 6.2% and OPSD by 1.7% on average on AIME24, AIME25, and HMMT25. Our code is available at https://github.com/walawalagoose/SGSD.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想

将技能条件自蒸馏重新定义为教师假设验证问题：检索到的技能-错误对作为多教师假设，验证器根据 rollout 结果判断各教师的极性（helpful 或 misleading），通过门控目标过滤不确定或极端信号。

### 2.2 关键技术细节

#### （1）结构化技能库构建（Structured Skill Bank）

- 技能库包含两类结构化知识：
  - **通用技能（General Skills）**：编码可复用的推理原则，包含 `skill_id`、`title`、`principle`、`when_to_apply`
  - **常见错误模式（Common Mistakes）**：描述错误发生原因及避免方法，包含 `mistake_id`、`description`、`why_it_happens`、`how_to_avoid`
- 技能库通过冷启动（cold-start）流程初始化：采样训练问题生成轨迹 → 成功轨迹提取通用技能 → 失败轨迹提取错误模式 → 分层合并去冗余

#### （2）技能条件多教师池（Skill-Conditioned Multi-Teacher Pool）

- 对每个问题 x，通过语义相似度检索从技能库获取：
  - $G(x) = \{g_1, ..., g_{K_x}\}$（通用技能）
  - $E(x) = \{e_1, ..., e_{K_x}\}$（错误模式）
- 每个技能-错误对 $(g_k, e_k)$ 形成一个独立的教师上下文 $c_k(x) = g_k \oplus e_k \oplus x$
- 教师权重 $\alpha_k(x)$ 来自检索相似度得分的 softmax 归一化
- 所有教师共享同一策略模型（stop-gradient），但接收不同的技能条件 PI

#### （3）结果验证的教师极性（Outcome-Validated Teacher Polarity）

- 计算 token 级教师-学生 log 概率差距：$\Delta_t^{(k)} = \log p_T^{(k)}(y_t | x, y_{<t}) - \log p_S(y_t | x, y_{<t})$
- 汇总为教师级支持分数：$a_k = \frac{1}{T} \sum_{t=1}^T \Delta_t^{(k)}$
- 验证器结果 $r \in \{+1, -1\}$ 验证立场：
  - **正向监督**：支持成功 rollout 或抑制失败 rollout
  - **反转监督**：支持失败 rollout 或抑制成功 rollout
- 决策规则表：极性 $\rho_k = sgn_{out}(r) \cdot sgn(\tilde{a}_k)$

#### （4）鲁棒支持估计（Robust Support Estimation）

- **Token 遮罩**：移除特殊 token、格式 token 等，仅保留有效推理位置
- **差距截断**：$\tilde{\Delta}_t^{(k)} = clip(\Delta_t^{(k)}, -c_\Delta, c_\Delta)$
- **置信阈值**：当 $|\tilde{a}_k| \leq \epsilon_a$ 时忽略该教师

#### （5）门控蒸馏目标（Gated Distillation Objective）

- 使用有界门控损失：
  $$\ell_{gate}(\Delta) = \log 2 - \log\left(1 + \exp\left(-\frac{\Delta^2}{2\tau_g}\right)\right)$$
- 特性：接近零时梯度消失，极大差距时饱和，适中差距处提供信息性监督
- 最终目标：$L_{SGSD}(x) = \sum_{k=1}^{K_x} \alpha_k(x) \rho_k \bar{\ell}^{(k)}$
- 极性 $\rho_k$ 控制更新方向：helpful 教师蒸馏立场，misleading 教师反转立场，不确定教师无贡献

#### （6）在线技能更新与维护

- 固定间隔从最近 rollout 中提取新技能候选
- 成功轨迹提取通用技能，失败轨迹提取错误模式
- 新候选与现有条目合并，按容量上限剪枝

### 2.3 算法流程（综合 Procedure）

1. **采样**：从训练集采样 mini-batch
2. **检索**：对每个问题从技能库检索 top-K 技能-错误对
3. **生成**：学生模型基于纯问题提示采样 rollout，验证器评估结果
4. **评分**：每个技能条件教师对相同 rollout 进行评分
5. **极性判断**：计算鲁棒支持分数，结合验证器结果判断极性
6. **蒸馏**：通过门控损失计算师生分歧的梯度，更新学生策略
7. **技能更新**：定期从最近成功/失败轨迹中提取新技能并维护技能库

5. **技能库可动态更新**：在线更新机制使技能库能够反映当前策略的成功和失败模式

6. **严格的师生信息边界**：技能仅在教师侧可见，学生仅接收纯问题提示，确保蒸馏过程中的师生对比
