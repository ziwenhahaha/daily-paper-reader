## Abstract
Integrated sensing, communication, and computation (ISCC) is emerging as a unified design paradigm for future vehicular networks that require joint environment perception, safety-critical information exchange, and latency-sensitive task processing. In New Radio Vehicle-to-Everything (NR-V2X) Mode 2, autonomous resource selection is performed through sensing-based semi-persistent scheduling (SB-SPS), which is effective for distributed communication resource reservation but does not explicitly consider sensing-resource demand, task-induced computation workload, and the additional latency introduced by mobile edge computing (MEC) offloading. This paper develops multi-agent proximal policy optimization-based SB-SPS (MAPPO-SPS), an ISCC-aware cross-layer scheduler that jointly adapts SB-SPS reservation, radio-resource partitioning, and overflow-driven computation-offloading decisions at control epochs. The scheduling problem is formulated as a cooperative partially observable Markov game and solved using MAPPO with centralized training and decentralized execution (CTDE). Simulation results show that MAPPO-SPS achieves a balanced tradeoff among CRLB-based sensing accuracy, packet reception ratio (PRR), effective throughput, energy consumption, and end-to-end delay.

## 2. 论文提出的方法论

### 核心思想

- 提出**MAPPO-SPS（Multi-Agent Proximal Policy Optimization-based SB-SPS）**框架，将感知、通信与计算联合纳入SB-SPS访问过程，在去中心化操作下实现ISCC调度。
- 将调度问题建模为**合作部分可观察马尔可夫博弈（Cooperative POMDP）**，使用MAPPO在集中训练-分散执行（CTDE）范式下求解。

### 关键技术细节

- **系统模型**：
  - 考虑N辆车在Road Segment上行驶，MEC-enabled RSU提供路边计算支持
  - 采用双时间尺度模型：时隙级别（slot-level）演化与控制周期级别（epoch-level）调度
  - PC5 sidelink资源被划分为感知和通信两部分：$N_{si}(k) + N_{ci}(k) \leq N_{sl}^{i}(k)$
  - V2I上行资源独立建模用于MEC卸载：$0 \leq N_{o}^{i}(k) \leq N_{o,\max}^{i}(k)$

- **感知模型**：
  - 基于OFDM的ISAC收发器，感知带宽$B_{si}(k) = 12N_{si}(k)\Delta f$
  - 2D DFT处理产生Range-Doppler地图，产生感知数据量$D_{si}(t) = 2N_{s,i}(k)M_{s,i}(k)b$
  - 感知质量指标采用**CRLB（Cramér-Rao Lower Bound）**：范围CRLB $\epsilon_{ri,\ell}(t)$和速度CRLB $\epsilon_{vi,\ell}(t)$
  - 车辆级感知惩罚$\epsilon_{sens}^{i}(t)$由目标特定CRLB加权聚合得到

- **通信模型**：
  - PC5 sidelink通信，SB-SPS预留资源，感知窗口计算CBR（Channel Busy Ratio）
  - SINR模型：$\Gamma_{ci,j}(t) = \frac{P_{ci}(t)h_{i,j}(t)}{\sigma^2 + \sum_{\ell \in U\setminus\{i\}} \mathbf{1}\{r_\ell(t)=r_i(t)\}P_{c\ell}(t)h_{\ell,j}(t)}$
  - 有效 sidelink 速率：$R_{eff}^{i}(t) = R_{ci}(t) \cdot PRR_i(t)$
  - 通信效用：$U_{ci}(t) = \alpha_{prr}PRR_i(t) + \alpha_{rate}\frac{R_{eff}^{i}(t)}{R_{eff,\min}^{i}}$

- **计算模型**：
  - 三层计算架构：车载CPU、MEC-enabled RSU、云端
  - 通信任务优先本地处理，溢出部分按决策$\eta_{ci}(k)$卸载至MEC
  - 感知溢出可按$\eta_{si}(k) \in \{0,1,2\}$选择：无卸载、MEC卸载或云卸载
  - 计算惩罚：$\psi_{comp}^{i}(t) = \alpha_{dc}\frac{T_{comp,c}^{i}(t)}{\delta_{ci}} + \alpha_{ds}\frac{T_{comp,s}^{i}(t)}{\delta_{si}} + \alpha_e\frac{E_{tot}^{i}(t)}{E_{\max}^{i}}$

- **跨层成本函数**：
  - $\ell_i(t) = \omega_s \tilde{\epsilon}_{sens}^{i}(t) + \omega_c \tilde{\Phi}_{ci}(t) + \omega_{comp} \tilde{\psi}_{comp}^{i}(t)$，其中$\omega_s + \omega_c + \omega_{comp} = 1$

### 算法流程（MAPPO训练）

1. 初始化去中心化Actor和集中式Critic参数
2. 每 episode 初始化移动性、信道、队列、MEC积压和SB-SPS预留
3. 每个控制周期 $k$：
   - 每个车辆观察 $o_i(k)$，获取候选资源集 $R_{cand}^{i}(k)$
   - 若 $\bar{RC}_i(k)=0$ 或触发重新评估，从掩码动作域采样动作
   - 否则保留先前SB-SPS预留，仅更新允许的跨层组件
   - 执行联合动作，模拟时隙级ISCC动态，计算合作奖励
4. 使用PPO裁剪目标更新Actor，使用时序差分更新Critic
5. 重复直到收敛

## 6. 论文的主要结论与发现

- **MAPPO-SPS优势**：在感知精度、通信可靠性和计算效率之间实现了最佳平衡
- **关键结果**：
