## Abstract
Communication protocol design is a central challenge in large language model-based multi-agent systems. Existing single-channel approaches face an inherent communication trilemma: text-based methods are interpretable but verbose, while latent-space methods are efficient but opaque and limited to unidirectional workflows. Inspired by multi-channel communication theory, we propose HyLaT, a hybrid latent-text communication protocol that transmits elaborate cognitive signals through a latent channel for efficiency, while expressing concise critical signals in natural language to preserve interpretability and precision. We introduce a two-stage training framework combining single-agent hybrid generation learning and multi-agent interactive co-training, enabling agents to generate and interpret hybrid messages across multiple rounds of interaction. Experiments demonstrate that HyLaT reduces communication overhead significantly while maintaining competitive task performance, with strong generalization and robustness across diverse settings.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在大语言模型（LLM）驱动的多智能体系统中，智能体间的通信协议设计面临关键挑战。
- **研究背景**：现有通信范式均为**单通道**设计，陷入"通信三难困境"（communication trilemma）：
  - **文本通信**（Text-based）：完全可解释，但消息冗长、token 开销大、效率低。
  - **潜在空间通信**（Latent-space）：信息传递密集高效，但对外部观察者不透明（缺乏可解释性），且主要支持单向或单轮交互，难以处理多轮、多智能体交互场景。
- **整体含义**：论文认为智能体消息天然包含两类异构信息——**详细认知信号**（如推理过程、解释）与**简洁关键信号**（如最终答案、承诺），应通过不同通道传输。借鉴多通道通信理论，提出用混合通道设计解决效率与可解释性之间的根本矛盾。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

#### 核心思想

提出 **HyLaT（Hybrid Latent-Text）** 协议，采用双通道架构：
- **潜在通道（Latent Channel）**：传输密集的中间认知信号（如推理、解释），提升机器间传输效率。
- **文本通道（Text Channel）**：传递简洁的关键信号（如最终答案、结论），保证可解释性与人机可读性。

#### 关键技术细节

**双通道响应格式**：智能体输出由两部分组成：
$$o_i^{(t)} = [\langle bot \rangle\ z_1, ..., z_k\ \langle eot \rangle]\ ||\ [y_1, ..., y_m]$$
- `Z = (z_1, ..., z_k)`：k 个连续潜在向量，由模型自回归生成，用 `<bot>` 和 `<eot>` 包裹。
- `Y = (y_1, ..., y_m)`：标准自然语言序列，生成在 `<eot>` 之后。

**通道分工（Channel Specialization）**：
- 文本通道保留摘要性内容（答案、承诺）。
- 潜在通道压缩冗长信息（解释、推理过程）。

#### 两阶段训练框架

**阶段 1：单智能体混合生成学习（Single-Agent Hybrid Generation Learning）**
- 训练模型从纯文本输入生成混合输出（潜在+文本）。
- 包含两个前向分支：
  - **混合前向**：生成潜在向量 Z 后生成文本 Y，损失函数 $L_{hybrid}$ 基于 Y 计算语言模型损失，引导模型基于自身产生的潜在信息推导答案。
  - **纯文本前向**：生成 `[e, y]`（解释+答案），损失函数 $L_{text}$ 采用教师强制，保持模型显式推理能力。
  - **跨通道对齐损失** $L_{align}$：在预测最终答案的 token 位置（如 "My answer is" 后的 token），对齐两分支的隐藏激活，将解释信息压缩到潜在空间：
$$L_{align} = \frac{1}{L} \sum_{l=1}^{L} ||sg(h_l^T) - h_l^S||^2$$
- 阶段 1 总损失：$L_{stage1} = \alpha L_{hybrid} + \beta L_{text} + \gamma L_{align}$。

**阶段 2：多智能体交互协同训练（Multi-Agent Interactive Co-Training）**
- 多智能体多轮对话数据，每个智能体接收来自其他智能体的混合响应。
- 构造输入 $x_i^{(t)} = [x_{i}^{(t-1)},\ \oplus_{j \neq i} o_{j}^{(t-1)}]$。
- 聚合所有智能体的损失：$L_{stage2} = \frac{1}{N} \sum_{i=1}^{N} (\alpha L_{hybrid}^{(i)} + \beta L_{text}^{(i)} + \gamma L_{align}^{(i)})$。
- 使智能体能解释来自同伴的潜在向量并生成适当的混合响应。

### 6. 论文的主要结论与发现

1. **HyLaT 有效平衡效率与性能**：
   - Token 效率比最高效的文本方法（EcoLang）提升约 **10.6 倍**，比 LatentMAS-V 提升约 **3.6 倍**。
   - 运行时间接近单智能体基线，同时保持竞争性任务性能。
   - 在 SocialIQA 上显著超越所有基线，表明潜在通道可补充文本无法充分传达的信息。

2. **两阶段训练缺一不可**：
   - 无 Stage 1：早期潜在向量噪声大，错误跨轮传播，性能大幅下降（38.43% vs 57.83%）。
   - 无 Stage 2：模型未见过同伴的混合输出，测试时格式错误率高达 22.27%，性能严重退化。

3. **双通道设计是最优选择**：
   - 纯文本变体 token 开销高（639.18 tokens）。
   - 纯潜在变体准确率低（50.28%）且格式错误率高（6.12%），表明文本信号对多轮连贯理解至关重要。

4. **通信协议兼容性**：HyLaT 智能体可与 NL 智能体有效通信，增加 HyLaT 智能体比例可同时降低 token 消耗并提升准确率。

5. **鲁棒性**：对潜在向量的高斯噪声注入（σ≤0.5）下性能保持稳定，关键答案通过文本通道传达，提供了噪声隔离层。

6. **可扩展性**：随着智能体数量和交互轮次增加，HyLaT 的 token 使用量保持相对稳定，效率优势愈发明显。通过针对性扩展训练数据可缓解规模扩大带来的性能下降。

7. **社会模拟适用性**：在重复信任游戏中，HyLaT 以极低通信开销（182 tokens）达到最高的平均返还额（10.84），展示了在模拟人类信任行为方面的潜力。

### 7. 优点：方法或实验设计上的亮点

1. **问题洞察深刻**：识别出多智能体通信的"三难困境"，并从多通道通信理论出发提出结构化解法，而非在单一通道内优化。
2. **混合协议设计巧妙**：通过通道分工（潜在传认知信号，文本传关键结论），同时解决了效率与可解释性的矛盾。
3. **两阶段训练框架合理**：先在单智能体场景学习混合生成，再在多智能体场景学习跨智能体理解，循序渐进，降低训练难度。
4. **跨通道对齐损失创新**：通过在预测答案的关键 token 位置对齐两分支隐藏激活，将显式推理压缩到潜在空间。
5. **格式兼容性强**：保留文本通道使 HyLaT 可与纯文本智能体互操作，克服了纯潜在方法的"黑盒"局限。
6. **实验覆盖全面**：主实验、消融、鲁棒性、异构模型、可扩展性、社会模拟等多维度验证。
7. **效率提升显著**：在多个尺度模型（1B、1.5B、3B）上均验证了效率优势，具有良好的跨模型家族泛化性。

### 8. 不足与局限

1. **模型规模局限**：所有实验均在相对小规模模型（1B~3B 参数）上进行，未验证在更大规模模型（如 7B、70B）上 HyLaT 收益是否可维持。

2. **推理框架兼容性问题**：当前修改后的生成流程与 vLLM 等推理框架不兼容，限制了在超大规模或长时程多智能体场景中的部署。

3. **文本通道多样性受限**：
   - 训练数据以 QA 辩论风格为主，缺乏丰富的通信行为类型。
   - 遵循 Shen et al. (2025) 采用固定答案提示对齐生成，限制了文本通道的表达范围。
   - 高质量多智能体通信数据和人类通信数据稀缺制约了文本通道的发展。

4. **潜在通道可解释性仍不足**：虽然比纯潜在方法有所改善，但潜在通道本身仍是不透明的，如何平衡潜在可解码性与通信性能是未解决的开放问题。

5. **统计显著性分析缺失**：未提供置信区间或显著性检验，难以判断方法间性能差异是否具有统计意义。

6. **社会模拟场景单一**：仅在重复信任游戏中验证，未涉及更复杂的社会交互场景（如谈判、协作规划）的泛化性验证。

7. **训练-评估配置不一致**：训练使用 N=2、T=2，而主实验评估使用 N=3、T=2，导致性能随规模扩展出现退化，需额外补充大规模训练数据。

8. **潜在偏差风险**：训练数据合成依赖 GPT-5 等强模型，可能引入特定模型的偏差；弱-强配对策略可能遗漏其他类型的对话动态。
