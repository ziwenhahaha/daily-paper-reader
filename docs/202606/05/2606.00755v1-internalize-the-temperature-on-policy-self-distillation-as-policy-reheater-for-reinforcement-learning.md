## Abstract
Reinforcement learning from verifiable rewards improves the reasoning ability of large language models, but often suffers from entropy collapse, in which increasingly concentrated policies reduce rollout diversity and useful learning signals. Existing remedies either constrain the RL objective (e.g., entropy regularization) or adjust sampling temperature during rollout collection, but these interventions remain external to the model parameters. We propose Temperature-Scaled On-Policy Self-Distillation (TS-OPSD), a lightweight policy reheating method that internalizes the exploratory effect of temperature into model parameters. Starting from an entropy-collapsed RL checkpoint, TS-OPSD constructs a self-teacher by applying high-temperature scaling to the model's own logits, then distills the resulting smoother distribution back into the student. This policy reheating requires no external teacher, privileged data, or additional inference cost. Experiments on Qwen3-4B-Base and Qwen3-8B-Base show that policy reheating yields a stronger initialization for continued RL than both standard continued RL and rollout-level temperature reheating. Further analyses show that TS-OPSD mainly reduces output sharpness while preserving intermediate representations, top candidate sets, and reasoning capability. These results suggest that entropy restoration can serve as a simple post-collapse intervention for extending reasoning-oriented RL.

- **核心问题**：在基于可验证奖励的强化学习（RLVR）中，随着训练进行，策略的熵会快速下降（熵崩溃），导致输出分布极度尖锐，rollout 多样性降低，有效的梯度信号减少，模型难以继续提升。
- **现有方案局限**：  
  - *目标层面*：熵正则化、熵保持更新或自适应裁剪等方法在优化目标上施加额外约束，会牺牲奖励最大化。  
  - *Rollout 层面*：在推理时提升采样温度（Rollout Reheating）只改变行为策略，未改变模型自身分布，导致行为策略与优化策略之间的错配。
- **研究动机**：能否把温度的“探索效应”直接嵌入模型参数，使模型自身具备更高的熵，从而在后续 RL 中作为更好的初始化？

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 2.1 核心思想
- **Temperature‑Scaled On‑Policy Self‑Distillation (TS‑OPSD)**：将模型自己的 logits 经高温缩放后得到一个更平滑的自老师，然后通过 KL 散度把该平滑分布蒸馏回模型本身，从而把温度的探索效应“内部化”到模型权重中。
- 与传统自蒸馏不同，**不需要外部模型、特权信息或额外推理**，仅利用当前模型的 logits。

### 2.2 关键技术细节
- **自老师构造**：给定学生 logit \(z\)，温度 \(T>1\) 下的老师分布  
  \[
  p_T(v) = \frac{\exp(z_v / T)}{\sum_u \exp(z_u / T)} = \frac{q(v)^{1/T}}{Z_\alpha},\quad \alpha = \frac{1}{T}
  \]  
  其中 \(q(v)=\text{softmax}(z)_v\) 为学生的原始分布，\(p_T\) 与 \(q\) 共享相同的 token 排名（单调幂变换），仅在分布尖锐度上不同。
- **KL 目标**  
  - **Forward KL (FKL)**：\( \mathcal{L}_{\text{FKL}} = D_{\text{KL}}(p_T \parallel q) \)  
    梯度：\(\nabla_z \mathcal{L}_{\text{FKL}} \propto q - p_T\)（压缩高概率 token 的 logit 边缘，保持语义顺序）。  
  - **Reverse KL (RKL)**：\( \mathcal{L}_{\text{RKL}} = D_{\text{KL}}(q \parallel p_T) \)  
    在自温度耦合下，梯度简化为  
    \[
    \nabla_z \mathcal{L}_{\text{RKL}} = -\bigl(1-\frac{1}{T}\bigr)\nabla_z H(q)
    \]  
    即等价于最大化学生熵，温度仅作为梯度幅度的标量因子。
- **训练流程（Policy Reheating）**  
  1. 从熵崩溃的 RL 检查点 \(\theta_{\text{collapse}}\) 开始。  
  2. 使用 **OPSD**（GKD 框架）进行少量步（≤15 步），教师温度 \(T=1.2\)，学习率 \(1\times10^{-6}\)，有效批大小 1024，on‑policy rollouts。  
  3. 当平均 token 熵恢复到 0.12 nats 以上时停止，得到恢复后的检查点 \(\theta_{\text{reheat}}\)。  
  4. 以 \(\theta_{\text{reheat}}\) 为起点，使用原始 DAPO 超参数继续 RL 训练，直至进一步崩溃或收敛。

## 6. 论文的主要结论与发现

1. **政策重热显著优于基线**：在所有基准上，DAPO w/ PR‑FKL 均超过标准 DAPO 与 Rollout Reheating，尤其在较难的 AIME、OlymMATH 上提升更为明显。  
2. **FKL 优于 RKL**：FKL 通过结构化平滑（按模型自身偏好重新加权）保留了推理结构，而 RKL 更接近纯熵最大化，导致在后续 RL 中的表现略逊。  
3. **TS‑OPSD 是自限的**：在熵已恢复的模型上继续高温 OPSD 会导致失控熵增长，说明该方法仅适用于 **低熵崩溃阶段**。  
4. **内部表征与推理能力保持不变**：CKA、Top‑10 重叠、准确率均显示 OPSD 只降低输出尖锐度，未破坏中间层表示或推理能力。  
5. **训练动态特征**：存在“熵跳跃”——先缓慢提升，随后出现不连续跃升，伴随分支位置提前、log‑prob 降低，标志着模型开始探索低概率推理分支。  
6. **计算成本可接受**：一次性约 15%~17% 的额外 RL 时间即可换来持续 RL 的性能提升，性价比高。  
7. **可逆性证据**：低‑温 OPSD 与高‑温 OPSD 的参数更新方向高度相反，提供了对方法机理的直观理解。

## 7. 优点

- **轻量且无外部依赖**：不需要额外 teacher、 privileged data 或额外的推理步骤，实现成本极低。  
- **与 RL 解耦**：将熵恢复作为独立的后处理阶段，不影响原始 RL 目标，避免目标层面的权衡。  
- **理论分析完整**：对 FKL 与 RKL 在自温度耦合下的梯度形式进行严格推导，并给出直观解释。  
- **实验覆盖面广**：跨模型规模、多个基准、多种对比与消融，验证了方法的稳健性。  
- **可视化与可解释性**：通过 CKA、熵跳跃、log‑prob 轨迹等分析，揭示了“内部化温度”的具体表现。  
- **适用性强**：在数据稀缺场景下同样有效，具备成为生产管线的潜力。

## 8. 不足与局限

- **模型规模局限**：仅在 4B 与 8B 参数模型上验证，未在大模型（如 70B+）或不同架构上测试，泛化性尚需进一步考察。  
- **经验性阈值**：熵崩溃阈值（0.08 nats）与恢复目标（0.12 nats）为经验设定，缺乏自适应或理论支撑，可能随模型族或训练配置而变化。  
- **单轮重热**：实验仅使用一次 OPSD + 继续 RL，未探索多轮循环（OPSD → RL → OPSD）以及相应的触发/停止策略。  
- **超参数敏感性未系统搜索**：学习率、批大小、温度选择等关键超参数未做网格搜索，依赖经验设定。  
- **计算资源仍较高**：虽然相对 RL 只增加约 15%~17% 的时间，但完整 RL 训练仍需 30+ 小时，对资源有限的团队仍有门槛。  
- **通用性待验证**：目前仅在数学推理任务上验证，方法在其他 RL 场景（如代码生成、自然语言生成）中的有效性未知。  
- **潜在风险**：过度熵提升可能导致推理路径退化，尽管实验显示当前阈值安全，但在更极端的温度或更长的训练下可能出现不稳定。
