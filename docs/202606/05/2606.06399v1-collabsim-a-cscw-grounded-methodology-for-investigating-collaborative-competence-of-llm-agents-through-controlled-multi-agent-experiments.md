## 摘要
基于大型语言模型构建的多智能体系统（MAS）已展现出日益增长的潜力，其效能取决于智能体通过文本渠道进行协调的能力，正如人类团队一般。然而，近期研究表明，MAS 经常失败并非因为智能体缺乏个体任务解决能力，而是因为它们缺乏协作能力：即建立共同基础、维持共享任务理解、平衡个人与集体激励、以及在交互展开过程中修复错位的能力。数十年来，计算机支持的协同工作（CSCW）研究已经刻画了在受限通信条件下人类团队的这些需求，然而现有的 MAS 评估主要关注任务结果或单个智能体在推理、规划和工具使用方面的熟练程度。为了对 MAS 中智能体的协作能力进行系统分析，我们提出了 CollabSim，一个可配置仿真框架，它结合了基于理论的合作能力定义、交互条件的受控操作以及智能体内部状态的动作级探测。在四种 LLM 上的实验表明，CollabSim 能够捕捉条件效应、分离模型性能模式，并揭示智能体设计的任务依赖效应。

## Abstract
Multi-agent systems (MAS) built on large language models have shown growing promise, with their effectiveness resting on agents' ability to coordinate through text-based channels much as human teams do. Yet recent study suggests that MAS often falter not because agents lack individual task-solving ability, but because they lack collaborative competence: the capacity to establish common ground, maintain shared task understanding, balance individual and collective incentives, and repair misalignment as interaction unfolds. Decades of research in Computer-Supported Cooperative Work have characterized these requirements for human teams coordinating under constrained communication, yet existing MAS evaluations focus mainly on task outcomes or single-agent proficiency in reasoning, planning, and tool use. To enable a systematic analysis of agents' collaborative competence in MAS, we introduce CollabSim, a configurable simulation framework that combines a theory-grounded definition of collaborative capabilities, controlled manipulation of interaction conditions, and action-level probing of agents' internal states. Experiments across four LLMs show that CollabSim can capture condition effects, separate model performance patterns, and reveal task-dependent effects of agent design.

## 2. 论文提出的方法论

### 核心思想

提出 **CollabSim**，一个可配置的仿真框架，用于系统评估 LLM 智能体的协作能力。框架结合了：
- 基于 CSCW 理论的协作能力定义
- 交互条件的受控操作
- 智能体内部状态的动作级探测

### 关键技术细节

**系统架构（两层结构）：**

| 层次 | 功能 |
|------|------|
| 交互层 | governs how agents perceive the task environment |
| 控制层 | orchestrates experiment configuration, execution, and evaluation |

**形式化定义：**
```
实验 E = (A, S, {X_i}_{i=1}^{n}, {O_i}_{i=1}^{n}, T)
```
- A = {a₁, ..., aₙ}：智能体集合
- S：状态空间
- X_i：智能体 a_i 的动作空间
- O_i：智能体 a_i 的观察空间
- T：终止条件

**Agent Context Protocol：**
- 在每个回合 t，系统通过组合两部分构建智能体的观察 o_t^i：
  1. **公共更新**（public update）：当前共享任务状态 s_t，经信息可见性规则过滤
  2. **私有状态**（private state）：智能体特定信息（如剩余产能、分配的材料）

**Experiment Controller：**
- 通过 .yml 配置文件驱动，基于回合的执行循环
- 管理动作验证、状态更新、触发探测模块

### 可配置的交互条件

| 条件类别 | 说明 |
|----------|------|
| **通信带宽** | 控制消息频率和长度，类比媒体丰富度操纵 |
| **信息可见性** | 控制哪些共享状态信息可被访问 |
| **团队规模** | 变化每会话的智能体数量 |

### 探测模块（Probing Module）

在每个动作后查询每个智能体三个维度：
1. 感知的任务状态（perceived task state）
2. 感知到的队友意图（perceived teammate intents）
3. 自我推理（self-reasoning）

**量化指标：**
- 自我报告置信度分数（self-reported confidence score）
- 配对响应相似度（pairwise response similarity）：通过 SBERT 编码的余弦相似度计算

- 实时机制，900秒总时长
- 每10秒触发所有智能体，生产一个形状需30秒

**Map Task：**
- 基于步进，总步数120
- Guide 只能发送消息，Follower 有完整画布动作空间

## 6. 论文的主要结论与发现

### 6.1 交互条件操纵的效果

**通信带宽限制的影响：**
- 降低通信带宽一致性地减少了原始消息量
- 过程指标显示智能体合作和协调普遍下降：
  - DayTrader：所有四个模型的合作率下降
  - Shape Factory：三个模型的消息-交易比率下降
  - Hidden Profile：关键候选人提及率在三个模型中下降
  - Map Task：三个模型的沟通效率下降

**信息可见性的影响：**
- 在 Shape Factory 和 Map Task 中启用信息可见性增加了智能体参与度
- 但更好的参与不必然转化为更好的任务结果
- Map Task 中三个模型显示路线绘制准确率提升
- Shape Factory 中财富未增加

**团队规模的影响：**
- Shape Factory：更大的团队提高财富但降低订单完成率（机会与协调负担的权衡）
- DayTrader：九智能体条件下理论驱动智能体的合作率下降
- 显示智能体自我报告对齐与实际合作行为之间存在差距

### 6.2 不同模型后端的效果

| 模型 | 特点 |
|------|------|
| **GPT-5.5** | 最可靠的强表现者（四个任务中三个排名前二），但 DayTrader 平均财富最低 |
| **Claude 4.6** | DayTrader 表现突出（唯一收益随团队规模单调递增的模型），但 Hidden Profile 两个条件下准确率为 0% |
| **Qwen3.6** | 对条件最敏感的模型，Shape Factory 所有条件下订单完成率为 0% |
| **Llama-4** | 整体表现最弱，所有 Shape Factory 条件下订单完成率为 0%，Map Task 路线准确率最低 |

**总体趋势：** 专有模型通常领先或与最强结果持平，开源模型表现欠佳。

### 6.3 智能体设计的效果

- 两种智能体设计产生可测量的差异
- 在某些设置下，理论驱动智能体一致性地改进性能（如 Shape Factory 意识仪表板条件下的接受率）
- 但效果因任务而异，并非普遍有益：
  - Hidden Profile 带宽限制下，Qwen3.6 和 GPT-5.5 的任务准确率从 100% 降至 0%
  - DayTrader 六智能体条件下，Llama-4 财富大幅下降

### 6.4 探测模块发现

**对齐是任务依赖的：**
- 平均对齐（跨所有模型/设计/条件）：DayTrader 最高（0.894），其次 Hidden Profile（0.886）、Shape Factory（0.868）、Map Task 最低（0.797）
- 与每个任务的协调结构一致

**队友意图对齐最困难：**
- 三个任务中，Q1（任务状态）对齐最高，Q2（队友意图）最低
- Q1-Q2 差距在高度受限的 DayTrader 很小（+0.004），在更多信息不对称任务中更大（Hidden Profile +0.056，Map Task +0.053）

**团队规模在不同任务中产生相反效果：**
- Shape Factory：对齐随团队规模增加而下降
- DayTrader：对齐随团队规模增加而增加（但反映的是对背叛策略的共同认识，而非更好的合作）

- **协作能力的操作化**：将抽象的"协作能力"概念分解为可测量、可操纵的具体维度

- **自我报告与行为对比**：能够诊断"报告对齐但行为不一致"的失败模式

### 7.4 系统性实验设计

- **多维度变化**：条件 × 模型 × 智能体设计的全面组合
- **任务覆盖**：涵盖四种经典 CSCW 实验范式，覆盖多种协作挑战类型
- **对照实验**： Persona-Based 与 Theory-Informed 智能体的对比提供了有价值的消融分析

### 7.5 定性分析补充

- 通过交互日志分析揭示了具体的失败模式：
  - 围绕共享目标的协调失败
  - 个人与集体激励平衡失败
  - 任务相关信息对齐失败

- **实验范式的人工性**：经典 CSCW 任务是高度控制的实验室设置，可能不完全反映现实世界的协作场景
- **任务类型的局限性**：四个任务覆盖的协作维度有限，可能无法代表所有类型的协作挑战

### 8.4 方法论局限

### 8.5 未明确的问题
