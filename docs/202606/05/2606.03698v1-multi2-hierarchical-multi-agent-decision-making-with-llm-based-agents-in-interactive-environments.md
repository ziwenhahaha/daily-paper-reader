## Abstract
A central goal of large language model (LLM) research is to build agentic systems that can plan, act, and adapt through sustained interaction with dynamic environments. While recent LLM-based agents exhibit impressive contextual reasoning, their long-horizon decision-making remains fragile, often suffering from objective drift, where goals and plans drift over extended interactions. We introduce Multi$^2$, a hierarchical multi-agent decision-making framework that explicitly decomposes agent behavior into complementary roles. A high-level agent (System 1) focuses on context-aware sub-goal generation using supervised fine-tuning (SFT), while a low-level agent (System 2) executes atomic actions through offline-to-online reinforcement learning (RL) in interactive environments. This separation enables stable long-horizon control, mitigates objective drift, and allows efficient adaptation. Across diverse interactive environments, Multi$^2$ consistently outperforms strong agentic baselines, demonstrating improved robustness and coordination in multi-turn interaction. Beyond performance, we introduce and release three hierarchical benchmark datasets, filling a long-standing gap in training and evaluating hierarchical decision-making for LLM-based agents.

### 1.1 研究问题
当前基于大语言模型（LLM）的智能体（Agent）在与动态环境进行长时间、多轮交互时，存在以下核心挑战：

- **目标漂移（Objective Drift）**：在长时交互过程中，智能体的目标与计划逐渐偏离原始意图。
- **执行脆弱性**：现有层级式方法虽然能进行任务分解，但执行器（Executor）并未被显式训练来纠正交互过程中累积的错误，导致执行端仍然会在长时交互中出现崩溃。
- **令牌效率低下**：智能体依赖不断增长的交互历史来维持意图，导致推理时令牌消耗过高。

### 1.2 核心动机
论文旨在回答一个关键问题：**如何构建能够可靠且高效地解决长时域任务的交互式 LLM 智能体？**

作者认为可靠的多轮交互需要满足三个条件：
1. 显式的任务级分解以保持意图；
2. 能通过与环境交互而改进的鲁棒动作执行；
3. 令牌高效的交互方式。

## 2. 论文提出的方法论

### 2.1 核心思想：Multi² 框架

Multi² 是一个**层级化多智能体决策框架**，将智能体行为显式分解为两个互补的角色：

| 角色 | 名称 | 职责 | 训练方式 |
|------|------|------|----------|
| 高层智能体 | **System 1** | 基于上下文的子目标（Sub-Goal）生成，保持全局意图 | 监督微调（SFT） |
| 低层智能体 | **System 2** | 在动态环境中执行原子动作，完成子目标 | 离线到在线强化学习（Offline-to-Online RL） |

### 2.2 关键技术细节

#### 2.2.1 层级数据集构建

- **D_sys1**（System 1 数据集）：包含任务描述 K_n 和观察-子目标对序列 `{(o_h, g^h)}`，用于训练高层规划。
- **D_sys2**（System 2 数据集）：包含子目标 g^(i) 和对应的低层转移序列 `{(o_t, a_t, r_t, o_{t+1})}`，用于训练动作执行。

#### 2.2.2 System 1 训练（SFT）

使用标准交叉熵目标进行行为克隆：

$$L_\phi = -\mathbb{E}_{(K_n, o_t, g^h) \sim D_{sys1}} [\log \pi_\phi(g^h | o_t; K_n)]$$

#### 2.2.3 System 2 离线训练（Offline RL）

- **Q 函数训练**：通过最小化时序差分（TD）误差训练 Q 网络：
$$L_\omega = \mathbb{E}_{(g^h, o_t, a_t, r_t, o_{t+1}) \sim D_{sys2}} [(r_t + \gamma V_\psi(o_{t+1}; g^h) - Q_\omega(o_t, a_t; g^h))^2]$$

- **值函数训练**：使用期望分位数回归（Expectile Regression）：
$$L_\psi = \mathbb{E}_{(g^h, o_t, a_t, r_t, o_{t+1}) \sim D_{sys2}} [L_\tau(A(o_t, a_t))]$$

- **策略训练**（离线目标）：引入**策略锚定优势项（Policy-Anchored Advantage Term）**，避免过度模仿：
$$L^{offline}_\theta = -\mathbb{E}_{(g^h, o_t, a_t) \sim D_{sys2}} [\lambda A(\ldots) + \exp(\beta A(\ldots)) \times \log \pi^{offline}_\theta(\ldots)]$$

#### 2.2.4 System 2 在线训练（Online RL）

通过 KL 正则化实现稳定自改进：
$$L^{online}_\theta = -\mathbb{E}_{(g^h, o_t, a_t) \sim B_{sys2}} [w_t \log \pi^{online}_\theta(\ldots)] + \eta \mathbb{E}_{(g^h, o_t) \sim B_{sys2}} [D_{KL}(\pi^{online}_\theta \parallel \pi^{offline}_\theta)]$$

其中：
- `B_sys2` 是混合离线数据集和新收集交互的回放缓冲区
- `w_t = exp(1/α * A(...))` 根据优势值加权
- KL 正则化项防止策略在在线微调过程中发生剧烈偏移

#### 2.2.5 LoRA 参数高效微调

System 1 和 System 2 共享预训练 backbone，但使用**独立的 LoRA 适配器**实现参数解耦，训练和推理时只激活对应系统的适配器。

### 2.3 算法流程（Algorithm 1）

```
1. 系统初始化：System 1 (π_φ), System 2 (π_offline_θ, Q_ω, V_ψ)
2. System 1 训练（SFT，epochs EP1）
3. System 2 离线训练（Offline RL，epochs EP2）
4. System 2 在线训练（Online RL，epochs EP3）：
   a. 接收任务和观察，采样子目标
   b. 循环执行动作并存储转移
   c. 子目标达成时调用 System 1 生成下一子目标
   d. 更新 Q_ω, V_ψ 和 π_online_θ
```

## 6. 论文的主要结论与发现

### 6.1 性能提升

Multi² 在三个基准环境和三个 backbone 上**一致性地优于所有基线方法**：

| 方法 | ScienceWorld ID | ScienceWorld OOD | ALFWorld ID | ALFWorld OOD | TextCraft |
|------|-----------------|------------------|-------------|--------------|-----------|
| ReAct (Llama) | 23.23% | 10.30% | 6.72% | 7.46% | 9.00% |
| Glider (Llama) | 60.48% | 34.36% | 43.57% | 37.86% | 9.50% |
| **Multi² (Llama)** | **67.61%** | **30.68%** | **57.86%** | **56.43%** | **35.60%** |

### 6.2 关键发现

1. **层级结构有效**：显式层级分解（System 1 + System 2）比扁平单智能体设计效果更好。
2. **角色专用训练至关重要**：SFT 适用于高层规划（SFT 对上下文感知规划更稳定），RL 适用于低层执行（能纠正累积错误），互换训练方式会导致性能显著下降。
3. **独立 LoRA 适配器有效**：参数解耦比共享适配器效果更好，尤其在 TextCraft 上提升显著。
4. **离线到在线 RL 训练有价值**：在线适应阶段带来约 20% 的额外训练开销，但能可靠提升性能（约 14% 相对提升）。
5. **令牌效率优势**：Multi² 实现了更高的性能/令牌比，在 OOD 划分上优势尤为明显。
6. **目标漂移鲁棒性**：在高难度任务（长时交互）上，Multi² 相比基线方法保持更稳定的性能。
7. **子目标错误非主要失败源**：Planner 相关失败仅占失败案例的 3.70% (ID) 和 5.88% (OOD)，说明提升执行层策略更为关键。
8. **模型规模效应**：在 OOD 划分上，更大的 backbone 带来更显著的泛化提升（1.5B → 7B 性能增幅明显）。

- **长时记忆机制缺失**：当前设计依赖 System 1 显式规划，未探索通过记忆机制维持长时意图的替代方案。
