## Abstract
Agentic reinforcement learning can induce tool abuse, where models overuse external tools even for queries solvable by internal reasoning. Existing approaches mitigate this issue with uniform tool-use penalties or hard limits, which reduce tool frequency but may also suppress useful tool-assisted exploration. We propose EAPO, an Efficient Agentic Policy Optimization framework that learns selective tool use. EAPO introduces tool-free trajectories into each rollout group, applies difficulty-aware reward shaping to penalize redundant tool calls mainly on easier queries, and uses confidence-aware token reweighting to improve policy learning. Across nine mathematical and knowledge-intensive reasoning benchmarks, EAPO consistently improves the accuracy efficiency trade-off on Qwen2.5-3B, Qwen2.5-7B, and Llama3.1-8B. Compared with GRPO, EAPO improves average performance by 10.45%, 7.27%, and 9.69%, while reducing average tool calls by 18.33%, 18.33%, and 24.59%, respectively. These results show that agents can learn when not to use tools without compromising tool-integrated reasoning.

**研究问题：** 智能体强化学习中的**工具滥用（Tool Abuse）** 问题。

**背景：**
- 近年来，大语言模型（LLMs）与外部工具（如搜索引擎、代码解释器）的结合显著提升了复杂任务的解决能力。
- 现有的智能体RL目标函数主要奖励工具可用环境下的任务成功，导致模型逐渐将工具调用视为默认推理策略。
- 结果是模型即使面对简单的算术题或常识问题（如“1+1=？”），也会过度依赖Python或搜索工具，造成不必要的计算、延迟和部署成本。

**现有方法的局限性：**
- 统一惩罚工具使用（如OTC）或硬性限制工具调用次数，可能抑制对真正困难问题的工具辅助探索。
- 关键问题不是“如何减少工具使用”，而是**如何识别和惩罚不必要的工具使用**——即模型本可以独立解决却过度依赖工具的情况。

## 2. 论文提出的方法论：核心思想与技术细节

### 核心思想

EAPO（Efficient Agentic Policy Optimization）旨在让智能体学习**何时使用工具、何时不使用工具**，而非简单地减少工具调用。

### 三大核心组件

#### (1) 效率感知rollout（Efficiency-Aware Rollout）
- 在每个rollout的M条轨迹中，**确保至少n条轨迹不使用任何工具**。
- 通过修改提示明确禁止工具使用，并禁用工具执行模块。
- 对比工具无关轨迹与工具增强轨迹，使模型能够判断工具使用是否真正带来性能提升。
- 定义**效率系数 ε(Ti)**：
  $$\varepsilon(T_i) = \exp(-\beta(c_i - c_{\min}))$$
  其中 $c_i$ 是轨迹中的工具调用次数，$c_{\min}$ 是正确轨迹中的最小工具调用数。

#### (2) 难度感知奖励塑造（Difficulty-Aware Reward Shaping）
- 定义查询难度 $d(q)$：
  $$d(q) = 1 - \frac{1}{M} \sum_{i=1}^{M} I(T_i)$$
  其中 $I(T_i)=1$ 表示轨迹正确，$0$ 表示错误。
- 调整后的奖励：
  $$\tilde{R}_i = [d(q) + (1-d(q))\varepsilon(T_i)] R_i$$
- **核心机制**：当模型已掌握某查询（难度低）时，更多惩罚工具使用；当查询仍困难时，保留工具辅助探索。

#### (3) 置信度感知优势重加权（Confidence-Aware Advantage Reweighting）
- 根据token级别的置信度（log概率）分配不同学习权重。
- 对正确答案中低置信度token上权重（强调信息密集token），对错误答案中高置信度token上权重。
- 公式：
  $$\omega_{i,u,t} = \begin{cases} W_{pos}, & \text{正确且置信度低} \\ W_{neg}, & \text{错误且置信度高} \\ 1, & \text{其他} \end{cases}$$
- 最终优势：$A_{i,u,t} = \omega_{i,u,t} A_i$

### 完整算法流程

详见Algorithm 1，主要步骤包括：
1. 对每个查询生成n条工具无关轨迹 + M-n条工具可用轨迹
2. 计算每条轨迹的正确性、工具调用数、难度系数
3. 应用难度感知奖励塑造
4. 对token进行置信度感知重加权
5. 使用重加权策略梯度更新策略，包含KL正则化

## 6. 论文的主要结论与发现

### 核心结论

1. **性能提升**：与GRPO相比，EAPO在三个模型上分别提升平均性能10.45%（3B）、7.27%（7B）、9.69%（8B）。

2. **效率提升**：同时减少平均工具调用18.33%（3B/7B）、24.59%（8B）。

3. **关键发现**：
   - **简单提示方法不足**：TIR性能不佳，与自主推理一致
   - **规模效应**：3B模型效率提升较小（本身工具滥用不严重），7B+模型提升更明显
   - **工具使用并非越多越好**：β=0时，更多工具调用并不提升性能，表明 indiscriminate tool invocation可能有害
