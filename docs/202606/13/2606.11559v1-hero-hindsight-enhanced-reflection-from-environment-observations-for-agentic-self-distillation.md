## Abstract
Reinforcement learning typically improves multi-turn agent capabilities through the terminal outcome of the trajectories, which makes it difficult to determine credit assignments for each intermediate turns. Recent on-policy self-distillation methods offer a promising alternative by converting privileged feedback into dense token-level supervision through a self-teacher. Our study is motivated by the unexpected performance degradation observed when naively extending this paradigm to multi-turn settings, which we attribute to a lack of alignment between privileged feedback, such as successful trajectories or terminal outcomes, and the student's current decision context. We introduce HERO, a hindsight-enhanced self-distillation framework that uses next environment observations as locally aligned feedback. After each rollout, HERO reflects on the completed interaction to convert each observation into a compact turn-level diagnosis, that captures actionable feedback about the original action such as its necessity, validity or failure cause. On TauBench and WebShop, HERO improves task success and reduces unnecessary turns over environment-feedback-only self-distillation and GRPO. It is especially effective under limited training turn budgets, where successful rollouts are rare and GRPO provides weak reward-contrast signals.

## 1. 核心问题与整体含义

**研究动机与背景：**

- **问题背景**：大型语言模型（LLM）智能体在多轮工具调用任务中表现出色，但优化其交互策略仍然具有挑战性
- **现有方法的局限**：
  - 基于结果的强化学习方法（如 PPO、GRPO）依赖轨迹的终点奖励，对中间工具调用的信用分配存在严重问题
  - 在长horizon交互中，成功的轨迹往往很少，因为智能体必须在有限的交互预算下推理不断演变的观测并获得延迟的结果
  - 即使智能体最终成功，结果奖励仍可能强化包含冗余工具调用的轨迹
- **自蒸馏方法的尝试**：近期提出了将特权反馈转换为密集 token 级监督的在线自蒸馏方法（如 SDPO、OPSD），在单轮设置（如竞争性编程）中表现良好
- **核心问题发现**：将这些方法朴素地扩展到多轮设置会导致性能下降，原因是特权反馈（如成功轨迹或终态奖励）与学生当前决策上下文之间存在**对齐缺失**

## 2. 论文提出的方法论

**核心思想：**

- HERO（**H**indsight-**E**nhanced **R**eflection from **O**bservations）提出使用**下一环境观测**作为局部对齐的反馈信号
- 关键洞察：在多轮交互中，最容易获得的局部反馈信号是每个动作后返回的下一环境观测，这与终端奖励或成功演示不同，它立即跟随学生的动作，反映环境如何响应智能体的实际行为
- 方法框架包含三个阶段：
  1. 学生策略在标准因果交互历史下生成多轮轨迹
  2. 反射器（Reflector）检查完整轨迹，为每轮生成结构化的 turn-level 提示
  3. 自我教师在事后增强的上下文中重新评估学生的原始动作 token，产生密集的分布级监督

**关键技术细节：**

- **反射格式**：每轮反射表示为 $h_t = (d_t, \hat{a}_t)$，其中 $d_t$ 是自然语言诊断，$\hat{a}_t$ 是可选的修正局部动作或目标行为
- **教师上下文构建**：对于第 t 轮，构建事后增强的教师上下文 $\tilde{H}_t = P(x, y_{<t}, o_t, R(\tau), h_t)$，其中 P 是固定的提示模板
- **自我蒸馏损失**：使用广义 Jensen-Shannon 散度作为自蒸馏目标，对选中的助手轮次应用损失

**公式流程：**

- 轨迹生成：给定初始任务 $x$，智能体交互最多 T 轮，$\tau = (x, y_1, o_1, ..., y_T, o_T)$
- 反射生成：$h_{1:T} = R(x, \tau, R(\tau))$，反射器 R 在训练期间是学生策略本身（自反射）
- 学生和教师分布：
  - 学生分布：$p^{stu}_{t,k} = \pi_\theta(\cdot | H_t, y_{t,<k})$
  - 教师分布：$p^{tea}_{t,k} = sg[\pi_{\bar{\theta}}(\cdot | \tilde{H}_t, y_{t,<k})]$
- HERO 目标函数：
  $$L_{HERO}(\theta) = E_{\tau \sim \pi_\theta} \left[ \frac{1}{|I(\tau)|} \sum_{t \in I(\tau)} \ell_t \right]$$
  $$\ell_t = \frac{1}{|y_t|} \sum_{k=1}^{|y_t|} D_{JSD}(p^{stu}_{t,k}, p^{tea}_{t,k})$$
  其中 $I(\tau)$ 是成功解析且非空的反射提示对应的助手轮次集合

**核心发现：**

1. **性能提升**：HERO 在所有设置中一致地改进成功率，超越基线模型和环境反馈基线
   - Qwen3-4B：TauBench-Retail 34.7% vs GRPO 33.3%，WebShop 68.9% vs GRPO 65.2%
   - Qwen3-30B-A3B：TauBench-Retail 50.3% vs GRPO 47.8%，WebShop 79.6% vs GRPO 76.1%

2. **效率改进**：HERO 在保持或提高成功率的同时，减少了平均交互轮数
   - TauBench-Retail：平均轮数从 13.4（GRPO）降至 9.6（HERO）

3. **跨域泛化**：在 TauBench-Airline（OOD 领域）上表现稳健，Qwen3-4B 从 18.0% 提升至 19.5%

4. **从失败中学习**：HERO 可以在没有成功演示的情况下学习，通过诊断局部执行错误（如工具调用格式、冗余 API 调用）

5. **低预算场景优势**：在严格的训练轮数预算下（成功 rollout 很少），GRPO 收到弱奖励对比信号，而 HERO 仍能改进

6. **单 rollout 可训练性**：HERO 在 G=1 时仍可训练，GRPO 因组相对优势为零而无法定义

7. **时钟效率**：HERO 每步训练时间约为 GRPO 的一半，因为只需单次 rollout 而 GRPO 需要 G=8

8. **能力保留**：在 MMLU（59.60→59.54）、MMLU-Pro（34.38→34.35）、IFEval（82.62→82.80）上无明显退化

## 7. 优点

**方法亮点：**

1. **问题定位精准**：准确识别出多轮设置中特权反馈与学生决策上下文的对齐缺失问题
2. **利用自然反馈**：巧妙地利用最容易获得的环境下一观测作为局部对齐信号，而不是依赖难以获得的成功演示
3. **事后压缩**：将完整轨迹的事后信息压缩为紧凑的 turn-level 提示，避免直接条件化完整的事后成功轨迹
4. **学习信号丰富**：即使在失败轨迹中也能产生非零监督，解决了 GRPO 在无奖励变化时的梯度消失问题
5. **端到端自包含**：反射器是学生策略本身，无需外部强模型或额外标注

**实验设计亮点：**

1. **多维度验证**：在多个模型规模、多个基准数据集、分布内和分布外场景下验证
2. **公平比较**：所有基线使用相同的接口、环境和评估协议
3. **消融充分**：系统性地消融了教师提示的各个组件和反射器质量
4. **效率分析**：提供了时钟效率的详细对比，证明方法不仅效果更好而且计算更高效
5. **质量评估**：手动评估反射质量，提供定量和定性分析

## 8. 不足与局限

**实验覆盖局限：**

1. **基准数量有限**：主要在 TauBench 和 WebShop 两个基准上验证，未在更多样化的多轮任务（如代码生成、数学推理）中测试
2. **模型规模覆盖**：仅测试了 Qwen 系列模型，未验证在其他模型族（如 LLaMA、GPT）上的效果
3. **任务复杂度**：任务主要是结构化的工具调用和客户服务场景，对于更开放的对话或多步骤推理任务的效果未知

**方法局限：**

1. **自反射能力依赖**：HERO 依赖模型自身的反思能力，对于复杂数学推理或深度多步问题解决可能不足，因为模型可能无法自行诊断潜在错误
2. **指令遵循依赖**：自我教师需要正确解释反射提示并将其转化为 token 级偏好变化，更适合指令微调模型而非基模型
3. **强模型优势更明显**：更大更强的指令微调模型受益更多，可能因为它们产生更准确的诊断提示并更可靠地遵循事后条件化提示

**失败模式和偏差风险：**

1. **事后泄露**（Hindsight Leakage）：反射使用仅在后续轮次中可用的信息
2. **近因偏差**（Recency Bias）：反射将失败归因于最新轮次而非更早的分叉点
3. **过度自信的修正**：提示提出可信但不支持的下一步动作
4. **无操作提示**：反射识别错误但不提供修复建议

**应用限制：**

1. **错误类型**：主要对可事后识别的智能体错误（如格式错误的工具调用、过度指定的参数、重复 API 调用）有效
2. **复杂推理任务**：对于复杂数学推理或深度问题解决可能不适用
3. **无成功轨迹场景**：虽然 HERO 可以从失败中学习，但当所有轮次的反射都是空时仍无法提供监督
