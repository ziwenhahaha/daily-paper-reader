## 摘要
本文研究在离线强化学习（RL）中使用线性时序逻辑在有限轨迹（LTLf）表达的时序扩展任务约束。最近，基于Transformer的方法（如轨迹Transformer和决策Transformer）已被用于将RL作为序列建模问题。然而，这些方法仅优化奖励，未考虑高层时序需求。在此，我们提出一种神经符号框架，将LTLf背景知识注入到此类基于Transformer的强化学习策略中。我们的方法将LTLf公式编译为确定有限自动机（DFA），并通过可微分表示和基于逻辑的损失函数将其集成到学习过程中。具体而言，我们从DFA演进中导出可微分满足信号，并将其作为训练时的正则化项。结果表明，该方法在不同的模型结构上具有架构无关性。我们在导航环境中评估了所提框架，测试规范涵盖安全性与可达性的时序属性组合。实验结果表明，引入背景知识不仅提升了约束满足率，而且在与普通基线相比仍保持具有竞争力的回报。

## Abstract
In this work we study offline reinforcement learning (RL) under temporally extended task constraints expressed in Linear Temporal Logic over finite traces (LTLf). Recently, transformer-based approaches such as Trajectory Transformers and Decision Transformers have been adopted to address RL as a sequence modeling problem. However, these methods optimize purely for reward and do not account for high-level temporal requirements. Here, we introduce a neurosymbolic framework that injects LTLf background knowledge into such transformer-based RL policies. Our approach compiles LTLf formulas into deterministic finite automata (DFAs) and integrates them into the learning process through a differentiable representation and a logic-based loss function. In particular, we derive differentiable satisfaction signals from DFA progression and use them as a regularization term during training. The resulting method is architecture-agnostic across different models. We evaluate the proposed framework on navigation environments with specification suites covering combinations of safety and reachability temporal properties. Experimental results show that incorporating background knowledge not only improves constraint satisfaction, but also maintains competitive return compared to vanilla baselines.

- **离线强化学习（Offline RL）**：智能体只能使用事先收集好的固定数据集进行学习，无法在线交互检测并纠正不安全行为。  
- **时序扩展约束**：在安全关键场景（如机器人导航）中，许多约束需要跨整个轨迹进行验证（如“最终到达目标且全程不踏入炸弹”），而不能仅靠单步奖励或即时状态判断。  
- **Transformer‑based RL 的局限**：Trajectory Transformer（TT）和 Decision Transformer（DT）等模型把 RL 当作序列建模问题进行训练，**仅最大化累计回报**，缺少对高层时序逻辑需求的表达与保证。  
- **核心问题**：如何在不修改底层奖励结构的前提下，将 **LTLf（Linear Temporal Logic over finite traces）** 形式的时序约束注入到基于 Transformer 的离线 RL 策略中，使生成轨迹同时满足逻辑规范和任务回报。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 2.1 核心思想
- **神经符号（Neuro‑Symbolic）框架**：把符号化的时序约束（LTLf）编译为确定有限自动机（DFA），并在训练阶段通过 **可微分的逻辑损失** 对 Transformer 策略进行正则化，使模型学习既高回报又符合约束的轨迹分布。  
- **架构无关性**：只要模型是自回归的（能估计下一个 token 的概率分布），便可直接使用该框架，无需改变模型结构。

### 2.2 关键技术细节

| 步骤 | 关键技术 | 说明 |
|------|----------|------|
| **① LTLf → DFA** | `ltlf2DFA` 工具 | 将任意 LTLf 公式 φ 转换为等价的确定性有限自动机 A_φ（字母表为 2^Σ） |
| **② 可微分 DFA** | **DeepDFA**（神经概率松弛） | 用矩阵形式实现 DFA，使状态转移对输入符号的软（概率）分配可微 |
| **③ 采样保持可微** | **Gumbel‑Softmax** | 在训练时从模型输出的离散分布中产生近 one‑hot 的软 token（温度 temp 控制锐度） |
| **④ 逻辑损失** | `L_φ = -log( P̂_{θ} ⊨ φ )` | 通过 Monte‑Carlo 采样 N 条软轨迹 ˜τ^(i)，用 DeepDFA 估算每条轨迹满足 φ 的概率，取平均后取负对数得到可微损失 |
| **⑤ 组合训练目标** | `L = α·L_φ + (1‑α)·L_D` | α 为平衡系数（0≤α≤1），L_D 为标准的序列预测交叉熵损失（如 TT/DT 的最大似然） |
| **⑥ 端到端语义统一** | **Logic Adapter** | 统一数据集的 token‑schema、token→symbol 映射、DFA rollout 与逻辑评估，防止因缺少 EOT（end‑of‑trace）标记导致不一致 |

### 2.3 算法流程（文字概述）

1. **输入**：离线轨迹数据集 ℒ、LTLf 规范 φ、模型 `f_θ`（TT 或 DT）。  
2. **编译**：把 φ 编译为 DFA A_φ，扩展其接受 EOT 符号。  
3. **训练循环**（每个 batch）  
   - a. 使用 `f_θ` 对当前子序列预测下一个 token 的分布 ˜y_t。  
   - b. 用 Gumbel‑Softmax 从 ˜y_t 采样得到软 token ˜x_t（保持梯度）。  
   - c. 将软 token 输入 DeepDFA，累加转移矩阵得到每条采样轨迹的满足概率 `P_DDFA(˜τ ⊨ φ)`。  
   - d. 计算逻辑损失 `L_φ = -log( (1/N) Σ_i P_DDFA(˜τ^(i) ⊨ φ) )`。  
   - e. 结合监督损失 `L = α·L_φ + (1‑α)·L_D`，反向更新 `θ`。  
4. **推理**：使用普通贪婪解码（或 Beam Search），不额外施加约束，但因训练时已正则化，生成轨迹更可能满足 φ。

## 3. 实验设计：数据集、场景、基准（Benchmark）与对比方法

### 3.1 环境

- **安全（Invariant Safety）**：`G(¬bomb)` → 整个轨迹必须永不踏入炸弹。  
- **可达安全（Reach‑while‑Safe）**：`G(¬bomb) ∧ F(goal)` → 必须最终到达目标且全程不踏入炸弹。  
- 通过 `ltlf2DFA` 自动生成对应的 DFA（见文中图 2）。

## 6. 论文的主要结论与发现

1. **逻辑正则化显著提升约束满足率**  
   - 在 Decision Transformer 中，α = 0.8 时 **100% 满足率**，bomb‑hit 率降至 0，Goal Rate 达到 100%。  
   - 在 Trajectory Transformer 中，α = 0.8 对安全规范可将 satisfaction 从 35% 提升至 50%；对可达安全规范最佳表现为 α = 0.1（satisfaction 4.7% vs 1.8%）。  

2. **回报保持竞争力**  
   - 加入逻辑正则化后，Return 并未出现显著下降；在 DT 中甚至出现 **回报提升**（从 -1.460 提升至 -0.780），因为避免了负奖励的炸弹触发。  

3. **安全‑性能之间的权衡**  
   - 高 α 强行使模型放弃某些短期高奖励（如进入炸弹），但在可达安全任务中仍可能出现 **goal‑reach 率低**（路径更长或更难发现），体现了内在的权衡。  

4. **TT 与 DT 差异**  
   - DT 对逻辑正则化更敏感，提升幅度大；TT 的提升相对温和，可能因为 TT 本身就具备更好的序列分布建模能力，导致正则化的边际收益较小。  

5. **框架通用性**  
   - 同一个逻辑适配器（Logic Adapter）可无缝对接 TT 与 DT，说明方法在架构层面是 **模型无关** 的。

## 7. 优点：方法或实验设计上的亮点

- **神经符号融合**：首次将 **LTLf → DFA** 与 **DeepDFA**（可微自动机）结合，形成端到端可训练的逻辑损失。  
- **架构无关**：不依赖特定 Transformer 变体，适用于任何自回归序列模型。  
- **训练时注入**：相较于仅在推理阶段约束解码（如 constrained beam search），在训练阶段加入逻辑正则化使模型“内化”约束，推理时不需额外搜索。  
- **统一评估栈**：通过 Logic Adapter 统一 token‑schema、EOT 处理与 DFA 评估，避免因语义不匹配导致的错误约束评估。  
- **可扩展性**：理论上可以支持任意 LTLf 公式（不受安全片段限制），并可进一步结合 Declare pattern 等业务约束。  
- **公开代码**：提供 GitHub 仓库，便于复现与进一步实验。
