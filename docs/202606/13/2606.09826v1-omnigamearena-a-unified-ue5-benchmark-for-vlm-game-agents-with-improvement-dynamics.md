## Abstract
Vision-language model (VLM) agents are increasingly deployed in interactive game environments. Yet game benchmarks for VLM agents typically report a single first-attempt score per (agent, game) pair, focus on single-agent Solo play, and lack unified protocols for evaluating heterogeneous agent classes (commercial VLMs, open-weight VLMs, and specialized game policies) on the same footing. We address these gaps with OmniGameArena, a real-time benchmark of twelve newly built Unreal Engine 5 games spanning Solo (7), PvP (3), and Coop (2) with unified action interfaces, and the Improvement Dynamics Curve (IDC), an agentic-reflection harness in which a tool-using reflector LLM autonomously refines a bounded skill prompt across multiple rounds. Beyond cold-start leaderboard scores, IDC exposes two additional observables for each (agent, game) pair: how the score evolves across reflection rounds, and how the learned skill behaves on held-out task variants. We report these observables for twelve VLM agents on the cold-start leaderboard and four top agents under IDC.

### 1. 论文的核心问题与整体含义（研究动机和背景）

**背景**：视觉-语言模型（VLM）智能体正越来越多地被部署在交互式游戏环境中。然而，当前的游戏基准测试存在两个关键缺陷：

- **单一评分陷阱**：大多数基准仅报告每个（智能体，游戏）对的单次尝试分数，无法揭示智能体在重复交互中的改进轨迹。
- **模式单一化**：现有基准高度集中在单人 Solo 模式，对抗性（PvP）和合作性（Coop）模式严重不足，无法评估对手建模、角色分配和队友失误恢复等能力。

**核心问题**：如何构建一个统一的实时基准，能够在**冷启动排行榜**之外，进一步揭示智能体在**多轮反思改进**中的动态提升曲线，以及所学技能在**未见变体上的迁移能力**。

### 2. 论文提出的方法论

#### 核心思想

论文提出了两个核心贡献：

1. **OmniGameArena**：一个基于 Unreal Engine 5（UE5）的 12 款全新实时游戏基准，涵盖 Solo（7款）、PvP（3款）和 Coop（2款）三种交互模式。所有游戏共享统一的动作接口（键盘-鼠标、游戏手柄），支持异构智能体的公平评估。

2. **改进动态曲线（IDC, Improvement Dynamics Curve）**：一个智能体反思框架（agentic-reflection harness）。在该框架中，一个配备工具使用的反射器 LLM 在多个轮次中自主精炼一个**有界技能提示（bounded skill prompt）**，从而产生完整的分数演化轨迹，而不仅仅是冷启动的单次分数。

#### 关键技术细节

- **游戏环境**：12 款全新 UE5 游戏，覆盖 7 个能力维度：视觉感知（VP）、空间导航（SN）、反应（RT）、记忆（MEM）、规划（PLN）、对抗性（ADV）、合作（COOP）。游戏进度统一归一化到 [0, 1] 连续尺度。
- **IDC 框架**：包含三个模块：
  - **经验获取模块**：每个轮次运行 K 集 episodes（在当前技能提示下），产生轨迹和轮次分数 $S_r$。
  - **反思模块**：反射器 LLM 通过四个自主阶段处理轨迹：Explore（探索）、Diagnose（诊断）、Validate（验证）、Distill（提炼）。反射器使用工具（`list_dir`, `read_text`, `read_image`, `grep`, `submit_diagnosis`, `validate_skill`）自主决定检查内容，而非执行固定脚本。
  - **持久化模块**：存储经验笔记本（experience notebook）、已验证技能和分数曲线 $[S_0, S_1, ..., S_R]$。引入**最佳技能回滚机制**（best-skill rollback）：当轮次分数急剧下降时（$S_{r+1} < \alpha \cdot S_r^*$，$\alpha = 0.5$），将提示重置为历史最佳技能。
- **评估协议**：使用两种时钟模式——
  - **PDQ（Paused Decision Quality）**：推理期间冻结环境，决策时间不计。
  - **LCRT（Latency-Controlled Real-Time）**：在推理时间之后注入模拟延迟，将延迟计入游戏时钟。
- **技能格式**：每轮生成一个**有界技能提示**（skill prompt，最多 1200 tokens），作为自然语言策略指导，包含从经验中提炼的游戏特定启发式规则。

#### 算法流程（IDC）

```
for round r = 0 to R:
    # 经验获取
    run K episodes under skill prompt m_r
    compute round score S_r = (1/K) * Σ s(τ_{r,k})
    
    # 反思
    if r > 0:
        reflector reads trajectories + persistent state
        runs Explore → Diagnose → Validate → Distill
        outputs refined skill m_{r+1}
        
    # 持久化与回滚
    update notebook, skill cache
    if S_{r+1} < 0.5 * S_best:
        m_{r+2} = m_best  # rollback
```

### 6. 论文的主要结论与发现

1. **无单一模型主导**：领导地位在不同游戏间轮转——GPT-5.5 领先 4/7 款 Solo 游戏，Claude Opus 4.6 赢得 CueChase，Gemini 3.1 Pro 领先 MonsterShoot。
2. **新旧模型表现不单调**：Claude Opus 4.6 在 5/7 款 Solo 游戏中优于 Opus 4.7；GPT-5.4 在 SceneEscape 中超过 GPT-5.5。
3. **开源 VLMs 和专用策略普遍失效**：两款 Qwen3.5 MoE 检查点在所有游戏中均低于 0.15，NitroGen 和 Open-P2P 几乎全线崩溃，表明 OmniGameArena 的任务多样性远超其训练分布。
4. **IDC 中所有模型均能改进**：4 个顶级智能体在 LastStand 上提升 130%-437%，在 SharedFloor 上也有显著进步。但**峰值通常出现在曲线中段**而非最终轮次，支持最佳技能回滚机制的必要性。
5. **技能迁移与原始任务提升存在分歧（核心发现）**：
   - SharedFloor 的协调启发式规则在所有变体上**普遍正向迁移**（16/16）。
   - LastStand 的技能迁移**高度依赖技能风格**：Opus 模型收敛到"静止保守"策略，在聚类掉落变体上崩溃（-72%~-76%）；GPT-5.5 的"短暂移动后重新评估"策略在所有变体上均正向迁移。
   - **原始任务提升不能预测变体迁移**——这是 IDC 揭示的隐藏可观测现象。
6. **延迟敏感性因任务而异**：MonsterShoot 对延迟极度敏感；LastStand 的近静止最优策略反而从较少、较晚的动作中受益；吞吐量任务（SoloCraft、SharedFloor）的分数下降主要由动作预算收缩驱动，而非单次动作效率损失。

### 7. 优点

1. **全新 UE5 游戏资产**：12 款专为基准构建的游戏，0% 识别率，显著降低预训练污染风险（对比其他基准的 66.7%-100% 识别率）。
2. **三模式统一评估**：首次在同一实时环境中系统评估 Solo、PvP、Coop 三种交互模式，揭示了不同模式下的能力差异。
3. **异构智能体公平比较**：通过统一适配器连接商业 VLMs、开源 VLMs 和专用策略，在相同条件下评估。
4. **IDC 揭示隐藏动态**：通过多轮反思轨迹和变体迁移测试，揭示了单轮排行榜分数所隐藏的技能过拟合和迁移性差异。
5. **框架设计轻量且功能完整**：IDC 通过小规模固定工具集实现反射器完全自主性，平衡了复杂度和功能性。
6. **双时钟评估协议**：PDQ 和 LCRT 分别隔离决策质量和延迟影响，为真实部署提供互补洞察。
7. **最佳技能回滚机制**：有效防止灾难性技能漂移，保证 IDC 曲线的稳健性。

### 8. 不足与局限

1. **IDC 规模受限**：仅在 2 款游戏（LastStand、SharedFloor）上运行 IDC，3 个变体和 4 个智能体，扩展到更多游戏和模型受算力限制。
2. **单技能格式**：反射器仅维护单个有界技能提示，未采用 Voyager 式的技能库积累方式，可能限制长期改进空间。
3. **玩家与反射器同模型**：每个智能体使用相同的底层模型同时作为玩家和反射器，未测试非对称配置（如小模型玩家 + 大模型反射器）的效果。
4. **反射器工具使用缺乏控制**：反射器自主决定检查内容和工具调用次数，导致过程不完全可复现，结果可能受模型随机性影响。
5. **PvP 场景未纳入 IDC**：IDC 实验排除了 PvP 以避免对手行为混淆，但对抗性场景下的自我改进能力未被评估。
6. **评分归一化掩盖绝对难度**：所有游戏归一化到 [0, 1]，可能掩盖了不同游戏间的绝对难度差异，且满分上限可能限制了对顶级智能体的区分度。
7. **商业 API 延迟不透明**：LCRT 实验中部分模型因后端延迟信号不准确被排除，导致对比仅限于 4 个模型，可能引入选择偏差。
8. **IDC 超参数未系统调优**：K（每轮集数）、R（轮次数）、$\alpha$（回滚阈值）等超参数未做系统性敏感性分析。
