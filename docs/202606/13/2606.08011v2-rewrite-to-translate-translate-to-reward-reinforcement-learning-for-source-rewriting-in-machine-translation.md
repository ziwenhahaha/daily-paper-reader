## Abstract
Rewriting source text with large language models (LLMs) before translation has been shown to improve machine translation (MT) quality. However, we find that prompt-based rewriting can degrade translation quality rather than improve it, particularly when smaller LLMs, such as 4B-parameter models, are used. We argue that this limitation stems from the difficulty of controlling rewriting behavior through natural-language prompts alone: a rewrite is useful only if it improves downstream translation, yet existing prompt-based methods do not explicitly optimize for this signal. To address this issue, we propose RLSR (Reinforcement Learning for Source Rewriting), a reinforcement learning framework that trains the rewriting model with a reward based on the downstream translation-quality improvement produced by each rewrite. Experiments across six MT systems and 16 language pairs show that our 4B RLSR-trained rewriting models significantly outperform both the no-rewriting baseline and prompt-based rewriting baselines at the same model scale, while remaining competitive with baselines that use a 235B LLM.

## 2. 论文提出的方法论

### 核心思想
提出 **RLSR（Reinforcement Learning for Source Rewriting）** 框架，直接用下游翻译质量提升作为奖励信号训练重写模型。

### 关键技术细节

**奖励定义**：
- 定义奖励 R(s,˜s,r) 为重写后翻译质量与原始翻译质量的差异
- 公式：$R(s, \tilde{s}, r) = Q(s, M(\tilde{s}), r) - Q(s, M(s), r)$
- 其中 Q 为自动评估函数（如 xCOMET-XL），M 为下游 MT 模型

**KL 正则化**：
- 为防止奖励黑客（生成无意义文本），引入 KL 散度惩罚
- 调整后奖励：$R_{total}(s, \tilde{s}, r) = R(s, \tilde{s}, r) - \beta \log \frac{R_\theta(\tilde{s}|s)}{R_{ref}(\tilde{s}|s)}$

**训练算法**：
- 使用 DAPO（Decoupled Clip and Dynamic Sampling Policy Optimization）算法
- 属于 on-policy 强化学习方法，基于 GRPO 改进

**联合训练扩展**：
- 可聚合 K 个 MT 模型的翻译质量提升作为联合奖励
- 实现跨 MT 模型的泛化能力

## 6. 论文的主要结论与发现

### 核心发现

**1. 提示式重写在 4B 规模失败**
- 所有 Qwen3 4B 提示式重写基线均差于无重写基线
- 提示无法可靠引导小型模型生成有益重写

**2. RLSR 显著优于同规模基线**
- 4B RLSR 模型在 xCOMET、GEMBA 等指标上显著优于无重写和 4B 提示式基线
- 在大多数设置下具有统计显著性（p < 0.05）

**3. RLSR 与 235B 模型竞争**
- 4B RLSR 模型性能与 235B 提示式重写模型相当
- 表明直接优化比单纯增大模型更高效

**4. RL 优于 SFT**
- SFT 训练导致"复制退化"（89.74% 源文本被原样输出）
- RLSR 仅 6.33% 原样复制
- SFT 无法区分近义但不同奖励的序列

**5. 跨 MT 泛化能力**
- 为一个 MT 模型训练的 RLSR 模型可直接应用于其他 MT 模型
- 联合训练的单一模型可在多个 MT 模型上同时提升性能
- 重写策略主要针对通用翻译障碍，而非特定模型偏好

**6. 重写行为差异**
- 提示式方法几乎完全重写（相似度仅 0.09-0.19）
- RLSR 仅做局部编辑（相似度 0.69-0.85），保持长度
- RLSR 学会针对性干预真实翻译障碍（不流畅、歧义等）

## 7. 优点

**1. 直接优化目标**
- 用下游翻译质量提升作为奖励，直接优化任务目标

**2. 避免指标过拟合**
- 使用 KL 惩罚约束策略偏离参考模型
- 结合 xCOMET（学习指标）和 GEMBA（提示式指标）双重验证

- 推理时仅需单次前向传播（重写模型 + MT 模型）
- 对比迭代式重写方法（需多次运行）更高效

**4. 跨模型泛化**
- 单个训练好的重写模型可服务于多个下游 MT 模型
- 降低实际部署成本

## 8. 不足与局限

### 指标偏差风险
- RLSR 奖励直接基于 xCOMET-XL，可能对该指标过拟合
- 学习指标（xCOMET、MetricX）可能对 RLSR 重写源文本有利
- 缓解措施：使用 GEMBA 验证，但未进行人类评估

### 实验覆盖局限
- 仅覆盖 6 个 MT 模型、16 个语言对
- 领域限于新闻/通用领域，未测试其他领域
- Gemma4 家族实验不完整（仅 2 个 MT 模型）

### 训练成本
- RLSR 训练（约 60 GPU-hours）显著高于 SFT
- 每个 MT 模型需单独训练，增加总体成本
- 对计算资源有限的场景可能不实用

### 适用性限制
- 假设可访问下游 MT 模型以计算奖励
- 黑盒 MT 系统可能无法应用此方法
- 依赖自动评估指标的质量和可靠性

### 未来改进方向
- 改进 SFT 目标（如对关键 token 加权），以降低训练成本
- 扩大联合训练规模，涵盖更多样化的 MT 模型
- 跨领域、跨语言对的更广泛验证
