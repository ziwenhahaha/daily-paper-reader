## Abstract
Modern language agents which perform multi-step reasoning have shown strong performance in knowledge-intensive question answering. However, existing approaches typically couple evidence acquisition and answer generation within a single policy. This forces a single model to play multiple potentially conflicting roles, inducing a combinatorial explosion in the policy space and hindering efficient exploration. It also introduces a credit assignment problem during training: a search action that retrieves sufficient evidence may still be penalized when generation fails, and vice versa. We propose DAC (Divide and Cooperate), a role-decomposed multi-agent training framework that divides agentic search into two cooperative subtasks, each handled by a dedicated agent trained with role-specific learning signals. The generator serves a dual role as both an answer producer and an evidence sufficiency verifier, abstaining when retrieved evidence is insufficient. This abstention signal is incorporated into the search agent's reward, providing structured cross-agent learning signals that improve credit assignment. Conversely, the searcher exposes the generator to diverse and challenging evidence environments by hard-positive evidence augmentation, improving its robustness. Experiments on general and multi-hop QA benchmarks show that DAC, implemented via parameter-efficient LoRA modules over a shared backbone, achieves strong performance against prior baselines that rely on full fine-tuning of monolithic models.

# 论文总结：Divide and Cooperate（DAC）

## 1. 核心问题与整体含义

**研究背景：**
现代语言智能体在知识密集型问答任务中通过多步推理展现出强大的性能。然而，现有方法通常将证据获取（evidence acquisition）和答案生成（answer generation）耦合在单一策略中，遵循 ReAct 框架的设计范式。

**核心问题：**
- **角色冲突**：单一模型被迫同时扮演多个潜在冲突的角色，导致策略空间组合爆炸，阻碍有效探索。
- **信用分配困难**：检索到充分证据的搜索动作可能在生成失败时被惩罚，而弱检索在生成器依赖参数知识补偿时可能逃脱惩罚。
- **训练效率低下**：端到端优化难以实现角色特化，导致训练信号模糊。

## 2. 方法论

### 核心思想
DAC 将智能体搜索分解为两个协作子任务：
- **搜索器（Searcher, S）**：负责迭代证据获取
- **生成器（Generator, G）**：负责基于证据生成答案

### 关键技术细节

**1. 序列分解流程**
1. 问题 Q 输入系统
2. 搜索器迭代执行证据采集，生成工具查询或终止
3. 搜索终止后，证据聚合为 E = Σ Et
4. 生成器接收 (Q, E) 并产生答案 A

**2. 跨验证奖励设计**
生成器被赋予弃权（abstention）能力，当判断证据不足时输出 "unknown"。弃权信号被纳入搜索器奖励：

- **搜索器奖励**：$r_S(Q, E, A) = \mathbb{I}[s(Q,E)=1 \land v(A)=1]$
  - 证据必须外部充分且生成器接受（不弃权）
- **生成器奖励**：$r_G(Q, E, A) = \mathbb{I}[a(Q,A)=1 \lor (s(Q,E)=0 \land v(A)=0)]$
  - 正确回答或正确弃权

**3. 硬正例证据增强**
为防止生成器过度保守（over-abstention），对充分证据集添加无关噪声文档：
- 构造 $E' = E \cup E_N$（E 充分，EN 为无关噪声）
- 保持答案可解性同时增加输入难度，迫使生成器扩展可答区域

**4. 轮次级差分奖励**
为搜索器提供更密集的信用信号：
- 每轮搜索后，用部分证据运行生成器计算奖励
- 轮次奖励为相邻状态奖励的差值
- 保持望远镜性质：累积轮次奖励等于最终奖励变化

## 6. 主要结论与发现

1. **性能优势**：DAC 在几乎所有基准上显著超越 Search-R1，包括通用和多跳 QA，且使用参数效率更高的 LoRA（2.12% 参数量）对比全参数微调的 Search-R1。

2. **弃权作为学习信号**：DAC 的生成器在证据充分性判断上表现更优（更高的 abstention accuracy 和 selective accuracy），证明跨验证奖励有效训练了生成器的证据评估能力。

3. **角色分解与跨代理奖励互补**：单独的角色分解（Decomp-AR）已优于 Search-R1，但 DAC 的跨代理奖励设计进一步提升性能，两者协同有效。

4. **硬正例增强防止过度保守**：无增强时生成器非弃权召回率显著下降，噪声证据迫使模型学习处理不完美输入。

5. **轮次奖励改善搜索学习**：提供更密集的信用信号，加速检索质量提升。

6. **Stackelberg 博弈视角**：无硬正例时均衡阈值坍缩为 0（完全保守），存在硬充分样本时阈值取决于 α > γcG(1) 条件。

## 7. 优点

1. **角色特化设计**：明确分离证据获取与答案生成职责，使每个代理专注单一目标。

2. **创新的跨代理信用分配**：利用生成器的弃权决策作为搜索器的验证信号，将监督边界动态对齐。

3. **参数效率卓越**：仅用两个 LoRA 适配器超越全参数微调基线，降低部署门槛。

4. **游戏理论分析严谨**：Stackelberg 博弈形式化解释系统行为，为设计选择提供理论支撑。

5. **轮次级差分奖励**：通过中间状态评估提供密集信号，改善长序列信用分配。

6. **消融实验全面**：验证每个组件贡献，实验设计透明可复现。

7. **弃权机制的实际价值**：生成器的充分性判断可用于触发额外检索，实现自纠正流水线。

## 8. 不足与局限

1. **任务范围有限**：实验仅覆盖短格式问答，未扩展至长格式生成、开放域规划或复杂工具调用场景。

2. **验证依赖 EM**：证据充分性通过精确匹配判断，可能过于严格；复杂场景可能需要 LLM 评判。

3. **硬正例增强的局限性**：依赖噪声文档采样，在检索库覆盖不足或格式特殊时可能效果有限。

4. **检索系统依赖**：论文未分析检索质量对整体性能的影响边界，检索偏差可能传导至答案。

5. **训练稳定性风险**：虽然 DAC 本身未观察到发散，但搜索-R1 基线存在训练突然崩溃现象，需更稳定的检查点选择策略。

6. **对抗性场景未覆盖**：未在包含误导性信息或恶意干扰的环境中验证鲁棒性。

7. **实际部署风险**：更强大的智能体搜索系统可能使生成输出显得更权威，即使证据有偏或不完整；弃权机制不保证事实性或高风险领域的安全性。
