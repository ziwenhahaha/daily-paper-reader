## Abstract
Constrained Multi-agent reinforcement learning (CMARL) faces two intertwined challenges: the joint action space grows exponentially with the number of agents, and additional requirements couple agents in ways that reward structure alone does not capture. We introduce Coordination Graphs for Constrained Multi-Agent Reinforcement Learning (CG-CMARL), a framework that addresses both challenges by combining coordination graphs with Lagrangian duality. The system decomposes the joint problem into pairwise regions, each served by a set of shared Q-functions, one for the primary objective and one for each of the constraints, so that the number of learned models is independent of the number of agents. At execution time, Max-Sum message passing coordinates actions across the factor graph, while a Lagrangian multiplier controls the objective--constraint tradeoff, allowing a single trained model to trace a Pareto front without retraining. We provide convergence guarantees under mild conditions, together with a compositional error bound that decomposes into separate interpretable sources, each traceable to a specific design choice and independently controllable. Experiments on cooperative navigation tasks (where teams of up to 10 agents must coordinate to reach target positions while satisfying pairwise constraints) show that our method produces Pareto fronts dominating established baselines trained at fixed reward-shaping ratios, while scaling to team sizes where centralized approaches become intractable.

### 1. 核心问题与整体含义

**研究背景：**  
约束多智能体强化学习（CMARL）面临双重挑战：

- **联合动作空间指数爆炸**：随着智能体数量 N 增加，联合动作空间 |A|^N 呈指数增长，使集中式方法在小规模团队（≥6智能体）后即变得不可行
- **约束耦合问题**：安全约束（如碰撞避免、能量预算）将智能体以奖励结构无法刻画的方式耦合在一起

**研究动机：**  
现有方法存在根本性矛盾——集中式原始-对偶方法需要全局状态和联合动作空间访问（正是因子化方法力图克服的瓶颈），而独立学习器无法协调约束满足。论文旨在设计一个同时具备**可扩展性**（O(1)网络数量）和**约束处理能力**（拉格朗日对偶）的去中心化框架。

### 2. 论文提出的方法论

**核心思想：**  
CG-CMARL 将协调图与拉格朗日对偶结合，将联合问题分解为成对区域，每个区域维护两个共享 Q 函数（主目标 + 约束），通过 Max-Sum 消息传递协调动作选择，拉格朗日乘子 λ 控制目标-约束权衡，实现单模型多 Pareto 点追踪。

**关键技术细节：**

- **双头 Q 网络架构**（核心创新）：
  - 主 Q 函数 `Qprim,R`：标准 Q-learning 更新，最大化主奖励
  - 成本 Q 函数 `Qcost,R`：在主 Q 函数的贪心策略下进行 TD 策略评估（**关键设计**）
  - 增强 Q 函数：`Qaug,R = Qprim,R + λ · Qcost,R`
  - 该设计使 λ 可在评估时调整而无需重训练

- **Max-Sum 消息传递**：
  - 因子图结构：变量节点（智能体动作）+ 因子节点（区域 Q 函数）
  - 因子→变量消息：每个因子向智能体发送基于其他智能体消息的最优收益
  - 变量→因子消息：智能体汇总来自其他区域的价值信息
  - 消息阻尼（d=0.3）用于稳定环状图的收敛

- **拉格朗日约束机制**：
  - 每个智能体维护局部乘子 λi，按 episode 更新：`λi ← Π[0,λmax](λi - η(ĉi - cthresh))`
  - 使用团队平均乘子 `λ̄ = (1/N)Σλi` 参与动作选择
  - 评估时外部设置 λ 并 sweep，追踪 Pareto 前沿

- **参数共享**：所有成对区域共享单一网络，O(1) 参数数量与 N 无关

**算法流程（简化版）：**
1. 每时刻 t：构建区域观察 {oR}，计算 `Qaug,R ← Qprim,R + λ̄·Qcost,R`
2. 运行 K 轮阻尼 Max-Sum，输出协调动作
3. 执行动作，观察 reward 和 constraint cost
4. 存储成对区域的 transition 到共享 replay buffer
5. 采样 mini-batch，分别用公式 (3) 和 (4) 更新两个 Q 头
6. 每 episode 末：用公式 (12) 更新各智能体的局部乘子

### 3. 实验设计

**数据集/场景：**  
- **Simple Spread 环境**（PettingZoo/MPE）：N 个智能体和 N 个地标在 [-1,1]² 连续空间中协作导航，需覆盖所有地标同时避免碰撞

- 额外测试三种固定地标布局：spread_uniform（圆形均匀分布）、line（直线分布）、clustered_pair（双簇分布）

**Benchmark：**  
- **评估指标**：Coverage (%)——episode 结束时被覆盖的地标比例；Collisions——每步平均碰撞智能体对数
- **Pareto 前沿**：通过 sweep λ ∈ {0, 0.05, 0.1, 0.2, 0.5, 1, 2, 5, 10, 40} 生成

**对比方法：**  
| 方法 | 特点 |
|------|------|
| IQL | 独立 Q 学习，无协调 |

基线方法通过 `local_ratio ∈ {0.0, 0.3, 0.6}`（碰撞惩罚混入比例）产生单点，每个模型对应单一运行点。

### 6. 论文主要结论与发现

1. **Pareto 前沿支配**：单 CG-CMARL 模型生成的 Pareto 前沿在所有 team size 下**支配**所有基线方法的单点配置

2. **可扩展性**：O(1) 参数量与 N 无关；计算复杂度 O(N²·K·|A|) 多项式增长，在 N=10 时集中式方法已不可行（|A|^N ≈ 10¹³），而 CG-CMARL 仍高效运行

3. **双 Q 函数优势**：解耦主目标学习和约束评估，避免 reward shaping 干扰；保守近似（Qcost 评估主策略）反而产生更安全的策略

4. **λ sweep 有效性**：通过调整评估时 λ 可追踪连续 Pareto 前沿，无需 retraining；极端 λ（≈0 或 ≥5）会导致退化行为，被 moderate λ 配置支配

5. **理论保证**：在独立转移和奖励因子化假设下，Q 函数收敛、主-对偶收敛、误差四源可加分解（结构误差 β、Max-Sum 误差 ε_MS、采样误差 O(1/√n)、神经网络误差 ε_NN）

6. **团队安全稳定性**：尽管原始碰撞对数随 N 增长，但 per-pair collision rate 保持稳定（约 0.001-0.002），表明算法在团队规模增大时仍能维持个体安全水平

### 7. 优点

- **架构创新性**：首次将协调图价值分解与拉格朗日约束处理整合到去中心化框架
- **参数共享设计**：所有成对区域共享单一网络，O(1) 复杂度使方法天然可扩展到大团队

- **理论完备性**：提供 Q 收敛保证、主-对偶收敛、误差分解等多层次理论分析
- **可解释误差边界**：四源误差可加分解，每项可追溯到具体设计选择并独立控制

- **保守安全性**：Qcost 评估主策略而非约束最优策略，使约束预测偏低估计，策略实际更安全

### 8. 不足与局限

- **场景单一性**：仅测试 Simple Spread，未验证过渡依赖场景（β > 0 的主要来源）或高阶交互任务
- **λ-增强变体未验证**：理论上自洽的 λ-condition 变体因增加采样复杂度未实现，理论-实践存在 gap
- **实验重复性有限**：仅 2 个种子，缺乏更大规模的统计显著性验证
- **保守近似缺陷**：两头设计使 Qcost 低估约束奖励，导致 Pareto front 可能非最优（论文承认）
- **λ sweep 需人工干预**：需手动设置 λ 范围，缺乏自适应调整机制
- **静态协调图**：使用固定成对图，对 N≥3 团队存在结构误差 β > 0；未采用动态/稀疏图拓扑
- **参数敏感**：Max-Sum 迭代次数 K、阻尼系数 d、dual 学习率 η 等超参数对收敛有影响
- **双时间尺度假设**：理论上需 η/α → 0，实际实现中隐式依赖该条件
- **合作假设**：所有方法基于完全合作设置，未讨论竞争或混合场景
- **缺乏真实机器人验证**：仅仿真环境测试，Sim-to-Real 迁移未验证
