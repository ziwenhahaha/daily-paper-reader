## Abstract
On-policy self-distillation (SD) improves LLM reasoning by using teacher-side privileged information (PI) to turn sparse verifier outcomes into dense token-level supervision. Existing methods usually assume trusted PI, such as reference answers or successful traces. We ask whether PI can instead come from an experience-derived skill bank, where retrieved skills are compact and reusable but may also be irrelevant or misleading. We propose Skill-Conditioned Gated Self-Distillation (SGSD), which formulates skill-based SD as teacher hypothesis validation rather than unconditional imitation. SGSD retrieves skill-mistake pairs, constructs a multi-teacher pool, and lets all skill-conditioned teachers score the same plain-prompt student rollout. The verifier validates each teacher's polarity: supporting a success or suppressing a failure gives positive supervision, while the opposite stance is reversed. A robust gated objective then distills informative teacher-student disagreements while suppressing uncertain or extreme signals. Experiments on multiple mathematical reasoning benchmarks show that SGSD consistently improves over GRPO and remains competitive with answer-conditioned OPSD under a weaker PI assumption. For example, on Qwen3-1.7B, SGSD outperforms GRPO by 6.2% and OPSD by 1.7% on average on AIME24, AIME25, and HMMT25. Our code is available at https://github.com/walawalagoose/SGSD.

## 1. 核心问题与整体含义（研究动机和背景）

**研究问题**：On-policy self-distillation（SD）通过教师侧的 privileged information（PI）将稀疏的验证器奖励转化为密集的 token 级监督，从而提升 LLM 推理能力。然而，现有方法依赖可信的 PI（如参考答案或成功轨迹），获取成本高且灵活性差。

**核心探索**：能否用经验衍生的 skill bank 替代传统 PI？Skill bank 中的检索技能更紧凑、可复用，但可能存在无关或误导性内容。因此需要解决三大挑战：
- PI 从单一可信源变为不确定的检索假设集
- 技能价值依赖结果：同一教师支持对成功/失败 rollout 效果相反
- 多教师产生噪声或冲突信号，需提取有用分歧

**研究定位**：将基于技能的 SD 重新定义为**教师假设验证**问题，而非无条件模仿。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 核心思想

SGSD 维持一个结构化技能库（General Skills + Common Mistakes），为每个问题检索相关技能-错误配对，构建多教师池。所有教师对同一学生 rollout 评分，由验证器裁决每个教师的极性（polarity），最后通过鲁棒的 gated 目标进行蒸馏。

### 2.2 关键技术细节

#### (1) 结构化技能库构建

- **General Skills (G)**：编码可复用的推理原则，包含 title、principle、when_to_apply
- **Common Mistakes (E)**：描述错误类型、成因及避免方式，包含 description、why_it_happens、how_to_avoid
- **冷启动**：从训练集采样问题，用初始策略生成轨迹，通过 verifier 分为成功/失败两类，分别提取为通用技能和常见错误，并进行层次合并去重
- **技能配对**：检索时将 general skill 与 common mistake 配对，形成平衡的教师上下文

#### (2) Skill-Conditioned Multi-Teacher Pool

对于问题 x，检索得到 top-K 技能对：
- 第 k 个教师上下文：$c_k(x) = g_k \oplus e_k \oplus x$
- 教师分布：$p^{(k)}_T(y_t | x, y_{<t}) = \bar{\pi}_\theta(y_t | c_k(x), y_{<t})$（stop-gradient）
- 学生分布（无技能）：$p_S(y_t | x, y_{<t}) = \pi_\theta(y_t | x, y_{<t})$
- 教师权重：$\alpha_k(x) = \exp(s_k(x)) / \sum_{j} \exp(s_j(x))$，其中 $s_k(x)$ 为检索相似度均值

#### (3) Outcome-Validated Teacher Polarity

对每个 token 计算教师-学生 log 概率差距：
- $\Delta^{(k)}_t = \log p^{(k)}_T(y_t | x, y_{<t}) - \log p_S(y_t | x, y_{<t})$
- 教师级支持分数：$a_k = \frac{1}{T} \sum_t \Delta^{(k)}_t$
- 极性判定规则：

| 情况 | 解释 |
|------|------|
| r > 0 且 $a_k > 0$ | Helpful teacher：蒸馏 |
| r < 0 且 $a_k < 0$ | Warning teacher：蒸馏 |
| r > 0 且 $a_k < 0$ | Over-suppressive teacher：反转蒸馏 |
| r < 0 且 $a_k > 0$ | Misleading teacher：反转蒸馏 |

#### (4) 鲁棒支持估计

- **Token 掩码**：移除特殊符、格式符等，保留有效推理位置
- **差距裁剪**：$\tilde{\Delta}^{(k)}_t = \text{clip}(\Delta^{(k)}_t, -c_\Delta, c_\Delta)$
- **鲁棒分数**：$\hat{a}_k = \frac{\sum_t m_t \tilde{\Delta}^{(k)}_t}{\sum_t m_t + \epsilon}$
- **置信阈值**：$|\hat{a}_k| > \epsilon_a$ 才参与蒸馏，uncertain 教师忽略

#### (5) Gated Distillation Objective

$$
\ell_{\text{gate}}(\Delta) = \log 2 - \log\left(1 + \exp\left(-\frac{\Delta^2}{2\tau_g}\right)\right)
$$

- 特性：零附近几乎为零、中等差距增长、极端差距饱和
- 梯度因子：$g_\tau(\Delta) = \frac{\Delta}{\tau_g(1 + \exp(\Delta^2/(2\tau_g)))}$
- 属性：sign consistency、有界信息区域（$|g_\tau(\Delta)| \leq 1/\sqrt{e\tau_g}$）、局部近似 reverse KL

#### (6) 在线技能更新与维护

- 每 F = 25 步收集最近 rollout 窗口 W
- 若成功率 < γ = 0.8，分割为成功/失败子集，分别提取新技能候选
- 与现有动态条目合并，保持最多 C = 30 个动态条目
- 冷启动条目静态保留

### 2.3 完整算法流程（Alg. 1）

1. 采样 minibatch X ⊂ D
2. 对每个问题 x：
   - 检索 top-K 技能-错误配对，计算教师权重

## 6. 主要结论与发现

1. **SGSD 在较弱的 PI 假设下仍具竞争力**：仅依赖验证器反馈和可能误导的技能对，SGSD 在 Qwen3-1.7B 上平均超越 GRPO 6.2%，超越 OPSD 1.7%。

2. **技能注入本身不足**：Base+Skill、GRPO+Skill、OPSD+Skill 等简单技能增强方法效果有限甚至负向，说明技能需要以结构化方式（如教师假设验证）使用。

3. **训练稳定性优势**：OPSD 早期快速提升但中后期退化，SGSD 保持更稳定的改进趋势，验证器和 gated 目标共同防止错误累积。

4. **多教师池优于单教师**：单一检索对可能不相关或部分有用，多教师池提供多种假设，提高蒸馏相关性。

5. **Answer PI 对 SGSD 有害**：将参考答案附加到技能教师上下文导致性能骤降并最终崩溃，说明 SGSD 受益于可复用的技能 PI，而非更丰富的 privileged 信息。

## 7. 优点

1. **问题定义创新**：将 skill-based SD 重新框架为教师假设验证，而非无条件模仿，直面技能不可信的现实挑战。

2. **多教师池设计**：K 个技能-错误配对形成多假设验证，比单一技能更可靠，且避免将技能直接暴露给学生。

3. **极性验证机制**：通过验证器 outcome 决定蒸馏方向，支持成功/抑制失败为正，反之为反，解决了同一教师在不同结果下的效果差异问题。

4. **Gated objective 理论保障**：提供完整的梯度分析、sign consistency、有界信息区域和局部 reverse KL 近似，兼顾理论严谨性和实践效果。

5. **在线技能更新**：冷启动 + 动态更新，使技能库随训练演进，保持教师信号的时效性。

6. **全面实验覆盖**：主实验 + 多维度消融，覆盖 3 个模型规模、3 个 benchmark、7 种方法。

## 8. 不足与局限

1. **任务范围有限**：主要评估数学推理（自动验证器），开放域生成需校准评判器或可靠 outcome 信号。

2. **技能库构建基础**：简单冷启动 + rank-wise 配对，适应性检索、更复杂配对或加权策略未被探索。

3. **评估协议**：报告 best checkpoint 而非 held-out 选择，与持续训练的自蒸馏设置一致，但未来研究可补充 cross-run 统计。

4. **消融规模限制**：大多数消融在 Qwen3-1.7B 上进行，大模型完整超参搜索未完成，计算成本限制了全面性。

5. **技能库偏见风险**：存储经验衍生技能，需确保与目标训练数据对齐，避免混入评估问题。

6. **推理能力泛化**：数学推理性能提升不应解读为高风险领域的广泛可靠性，需独立验证。
