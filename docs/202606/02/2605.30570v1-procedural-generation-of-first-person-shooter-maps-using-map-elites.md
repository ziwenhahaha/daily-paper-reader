## 摘要
我们研究了MAP-Elites（一种著名的质量多样性算法）在设计第一人称射击（FPS）游戏关卡中的应用。我们考虑两种著名的地图表示（All‑Black和Grid‑Graph），并引入两种新的表示（点线表示和空间布局表示），以更好地刻画FPS地图。我们定义了一系列指标来描述地图的拓扑属性（仅依赖于地图布局）和突现属性（需要通过实际游戏过程来评估）。我们进行了深入分析，以识别最适合引导MAP‑Elites光照过程的特征。我们采用滑动边界MAP‑Elites（MESB）来演化FPS地图种群。我们的结果表明，新的表示能够生成比先前用于演化FPS地图的表示具有更高多样性和质量的地图。

## Abstract
We investigate the application of MAP-Elites (a well-known quality diversity algorithm) to design levels for First-Person Shooter (FPS) games. We consider two well-known map representations (All-Black and Grid-Graph) and introduce two novel representations (Point-Line and Spatial-Layout) that improve the characterization of FPS maps. We define a series of metrics to describe maps' topological properties (which solely depend on maps' layout), and emergent properties (which must be evaluated through actual gameplay). We perform an in-depth analysis to identify the most suitable features to guide MAP-Elites illumination process. We apply MAP-Elites with Sliding Boundaries (MESB) to evolve populations of FPS maps. Our results show that the new representations can generate maps with higher diversity and quality than the representations previously used for evolving FPS maps.

## 2. 论文提出的方法论

### 核心思想
采用**MAP-Elites with Sliding Boundaries (MESB)**算法，结合两种新颖的地图表示方法（Point-Line和Spatial-Layout），通过拓扑特征和涌现特征对搜索空间进行光照，生成多样化且支持平衡游戏体验的FPS地图。

### 关键技术细节

#### 2.2.1 四种地图表示方法

| 表示方法 | 描述 | 特点 |
|---------|------|------|
| **All-Black** | 从充满墙壁的地图中雕刻可步行空间；地图为房间和走廊的列表 | 存在局部性问题（微小变异导致重大变化）和冗余性（不同基因型产生相同地图）；常产生死胡同 |
| **Grid-Graph** | 将地图划分为R×C网格，每个格子最多一个房间，房间间用走廊连接 | 较紧凑、冗余性低；但拓扑数量受限于预设网格大小，灵活性有限 |
| **Point-Line**（新） | 基因组为P个元组⟨p1, p2, s1, s2, c⟩，每对元组定义两个由L形走廊连接的房间 | 改善局部性，显式连接房间避免死胡同；长走廊和短走廊同等概率生成 |
| **Spatial-Layout**（新） | 基因组包含：①矩形房间列表（定义宽高）；②线段列表；③最小间距参数s | 使用Z3 SMT求解器确定房间位置，基于Delaunay三角剖分生成最小生成树走廊；可能产生多种拓扑，增加多样性 |

#### 2.2.2 地图特征体系

**拓扑特征**（46个，仅依赖地图布局，无需模拟）：
- 面积（area）：可步行瓦片占总瓦片的比例
- 对称性（maxSymmetry）：地图沿某轴对称的瓦片比例的最大值
- 平均偏心率（averageEccentricity）：各房间到其他所有房间最大距离的平均值
- 环路数量、平均房间距离、替代路径数量等

**涌现特征**（23个，需通过游戏模拟评估）：
- **节奏（pace）**：衡量战斗接触频率，归一化到[0,1]区间
  - 公式：`pace = 2 / (1 + exp(-5 * NF / TE)) - 1`
  - 其中NF为战斗次数，TE为交战时间
- 平均战斗时间、目标丢失率、击杀差异等

#### 2.2.3 适应度函数

基于游戏平衡性计算，使用两种差异化的机器人大脑（技能15%和85%）和两种武器（狙击步枪、霰弹枪）：
- 通过模拟5场1v1死亡竞赛（Duel模式）评估地图
- 计算各场次击杀数分布的**熵**：
  - 公式：`entropy = -Σ(ki/ktot) * log2(ki/ktot)`
- 适应度为5场匹配熵的平均值，熵越高表示游戏越平衡

#### 2.2.4 MESB算法流程

1. 初始化：创建10×10的档案库（每特征10个箱子），随机生成20个个体
2. 每轮迭代：
   - 使用10个发射器生成10个新个体
   - 对每个个体进行5场游戏模拟，计算适应度（平均熵）
   - 计算光照特征值并更新档案
   - 定期重新计算箱子边界（滑动边界机制）
3. 重复400次迭代，最终档案包含100个多样化的精英解

## 6. 论文的主要结论与发现

### 主要结论

1. **新表示方法的优势**：
   - Point-Line和Spatial-Layout能够生成比All-Black和Grid-Graph**更高多样性和质量**的地图
   - Point-Line在光照特征覆盖范围上表现最佳，特别是面积可达50%可步行区域
   - Spatial-Layout能快速填充档案库，生成多样化的优秀地图

2. **Grid-Graph的局限性**：
   - 虽然能生成最对称的地图，但地图面积过小，对人类玩家来说缺乏趣味性
   - 档案填充不足，多样性低

3. **特征对的影响**：
   - 纯拓扑特征对（area + maxSymmetry）和结合特征对（pace + averageEccentricity）均能有效引导光照过程
   - pace和averageEccentricity特征对能产生更丰富的地图拓扑变化

4. **平衡性与设计质量的权衡**：
   - 仅使用熵作为适应度可能生成"平衡但不有趣"的地图
   - Point-Line和Spatial-Layout在平衡性和设计质量之间取得了最佳折中
   - Spatial-Layout能快速生成多样高质量地图，但后期改进困难

### 关键发现
- Point-Line消除了死胡同问题，生成长走廊和短走廊概率相同
- All-Black的地图布局"嘈杂"，包含许多死胡同和锐角转弯，有利于霰弹枪bot但不利于平衡
- Spatial-Layout生成的地图在环路、覆盖物和替代路径方面表现居中，但档案填充速度最快
