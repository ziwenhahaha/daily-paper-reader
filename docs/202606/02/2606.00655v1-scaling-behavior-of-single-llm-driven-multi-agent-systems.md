## 摘要
基于大语言模型的多智能体系统（MAS）这一新兴领域有望通过协作智能解决复杂任务，然而关于其规模行为和内在集体动力学的基本问题仍未得到充分探索。本文系统地研究了同质多智能体系统的性能如何随智能体数量的增加而演变，将协作变量从模型或知识异质性中隔离出来。我们提出了顺序迭代多智能体系统（SIMAS）框架，这是一种以顺序智能体间通信为中心的极简架构，用于清晰地观察规模效应。通过在多种任务和模型规模下的广泛实验，我们确立了多智能体系统性能并非随智能体数量单调扩展，而是遵循收益递减的模式，受协作协同与协调开销之间的权衡所支配。我们的发现表明，有效的多智能体系统需要足够能力的基础大语言模型，任务类型关键性地调节最优智能体数量，集体智能是依赖于策略性交互设计的涌现特性，而非智能体多样性的必然结果。性能下降源于协调开销而非仅仅是长上下文失败，且规模趋势在不同交互架构（如结构化辩论拓扑）中具有普遍性。本工作提供了多智能体系统规模定律的基础性理解，为设计高效协作系统提供了实践指导，并挑战了更多智能体必然带来更好性能这一普遍假设。

## Abstract
The burgeoning field of LLM-based Multi-Agent Systems (MAS) promises to tackle complex tasks through collaborative intelligence, yet fundamental questions regarding their scaling behavior and intrinsic collective dynamics remain underexplored. This paper systematically investigates how the performance of a homogeneous MAS evolves as the number of agents increases, isolating the variable of collaboration from model or knowledge heterogeneity. We propose the Sequential Iterative Multi-Agent System (SIMAS) framework, a minimalist architecture centered on sequential inter-agent communication, to clearly observe scaling effects. Through extensive experiments across diverse tasks and model scales, we establish that MAS performance does not scale monotonically with agent count but follows a pattern of diminishing returns, governed by a trade-off between collaborative synergy and coordination overhead. Our findings reveal that effective MAS requires a sufficiently capable base LLM, that task type critically modulates the optimal agent count, and that collective intelligence is an emergent property contingent on strategic interaction design rather than a guaranteed outcome of agent plurality. The performance degradation stems coordination overhead rather than merely long-context failure, and the scaling tendency generalizes across interaction architectures like structured debate topologies. This work provides a foundational understanding of MAS scaling laws, offering practical guidance for designing efficient collaborative systems and challenging the prevailing assumption that more agents invariably lead to better performance.

## 2. 论文提出的方法论

**核心框架：SIMAS（Sequential Iterative Multi-Agent System）**

### 核心思想
- **极简主义设计**：仅保留顺序智能体间通信作为核心机制，剥离场景特定的复杂组件
- **同质假设**：所有智能体共享同一个底层LLM，仅通过配置文件（性格、信仰、专业知识）引入多样性
- **协作权衡范式**：性能受"协作协同（Collaborative Synergy）"与"协调开销（Coordination Overhead）"之间的博弈支配

### 关键技术细节

**数学形式化定义**：
- MAS定义为六元组：M = ⟨A, Q, H, T, Π, O⟩
  - A = {a₁, a₂, ..., aₙ}：智能体集合
  - Q：输入问题空间
  - H：对话历史空间
  - T：讨论轮数
  - Π：响应生成函数
  - O：最终答案综合函数

**属性配置**：每个智能体配置为四元组 ⟨LLM, dᵢ, pᵢ, bᵢ, eᵢ⟩
- dᵢ：描述
- pᵢ：性格特质（如Skeptical、Aggressive、Friendly）
- bᵢ：核心信念（如"Precision outweighs speed"）
- eᵢ：专业知识领域（如Logical Verification）

### 算法流程（文字描述）

```
1. 初始化对话历史 h₀ ← {q}，包含原始问题
2. 对每轮 t = 1 to T：
   a. 对每个智能体 aᵢ ∈ A：
      - 基于当前对话历史 hᵢ₋₁,ₜ 生成响应 rᵢ,ₜ
      - 更新对话历史 hᵢ,ₜ ← hᵢ₋₁,ₜ ∪ {rᵢ,ₜ}
   b. 确保对话历史单调增长
3. 第一智能体 a₁ 综合完整历史 hₙ,ₜ 生成最终答案
```

**关键属性**：
- **顺序通信**：确保每个智能体在看到所有先前响应后生成响应
- **历史累积**：对话历史单调增长，实现信息积累

## 6. 论文的主要结论与发现

### 发现1：倒U型规模规律（Inverted-U Scaling）
- **适用条件**：仅当MAS被"充分驱动"（sufficiently driven）时成立
- **定义**：存在n₁ ≠ n₂ ≥ 2，使得A(Mₙ₁) ≥ A(M₁)且A(Mₙ₂) ≥ A(M₁)
- **规律**：性能随智能体数量增加呈先升后降的倒U型曲线
- **解释**：初期收益于协作协同，后期受制于协调开销（信息冗余、推理路径冲突）

### 发现2：模型能力门槛
- **小模型（7B/8B）**：完全无法从多智能体协作中获益，呈现单调性能下降
- **大模型（70B+/72B+）**：能够展现倒U型规律
- **结论**：存在一个性能阈值，模型需达到此阈值才能有效驱动MAS

### 发现3：任务类型调制效应
| 任务类型 | 规模行为 | 典型示例 |
|----------|----------|----------|
| 逻辑推理型 | 性能急剧下降（n=1到n=8下降27.45%） | Abstract Algebra、Formal Logic |
| 事实检索型 | 相对稳定或呈U型恢复 | Philosophy、Global Facts |
| 混合推理型 | 非单调波动 | College Physics |

### 发现4：性能下降根因
- **不是长上下文失败**：固定上下文长度后，倒U型模式依然存在
- **是协调开销**：包括信息冗余、推理碎片化、累积误差

### 发现5：跨架构普遍性
- AutoGen辩论架构可提高峰值性能并延缓下降
- 但**无法消除**倒U型权衡规律
- 结构性辩论降低方差，但宏观模式一致

### 发现6：vs 单智能体推理（CoT）
- **推理密集型任务**：CoT一致优于SIMAS，尤其在AIME上SIMAS灾难性失败
- **编码/开放式任务**：结果混合且模型依赖
- **核心结论**：集体智能是交互架构的涌现特性，非智能体数量的必然产物

### 发现7：U型恢复现象（Philosophy任务）
- **机制**：在n=4-5出现性能低谷后，n=8时部分恢复
- **根因**：大规模多数投票统计抑制噪声，而非真正协作智能
- **本质**：伪稳定性（Pseudo-stability）

## 8. 不足与局限

### 实验覆盖局限

1. **智能体规模限制**：仅测试到8个智能体，未探索更大规模（如100+）的潜在涌现行为或DAG拓扑下的新动态

2. **模型规模受限**：未评估超过70B参数的更大模型，可能遗漏更大规模模型的行为差异

3. **封闭书问答局限**：主要使用QA基准，未充分覆盖工具增强型、纵向协作型或创造性任务中的协作收益

### 理论偏差风险

4. **同质模型假设的理想化**：实际系统多采用异质模型和工具，互补效应可能改变规模规律

5. **SIMAS极简性限制**：缺少投票、动态工作流等协调机制，可能低估了精心设计的协调机制对开销的缓解作用

6. **伪稳定性的误判风险**：U型恢复现象可能被误解为协作有效，需更精细的语义分析工具区分

### 应用限制

7. **任务适用性边界**：对于复杂多步推理任务，简单顺序MAS可能不如精心设计的单智能体CoT有效

8. **缺乏自适应机制**：未提供动态调整智能体数量的策略，纯启发式设计

9. **微观机制分析不足**：虽通过案例展示了协作失败的机制，但缺乏系统性的对话动态、语义多样性、信念分歧的量化分析
