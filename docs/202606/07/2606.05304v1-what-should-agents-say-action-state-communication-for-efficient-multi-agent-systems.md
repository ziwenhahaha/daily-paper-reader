## Abstract
Multi-agent systems (MAS) built on large language models are typically organized around roles, pipelines, and turn schedules, while the content that agents pass to one another is often left as unconstrained natural language. However, this free-form communication can rapidly inflate token usage, consume the shared context window, and ultimately affect both system performance and inference cost. We analyze five common inter-agent communication strategies across two MAS topologies, finding that no fixed strategy is universally optimal. Instead, effective inter-agent messages consistently preserve action-centered information needed by downstream agents. Building on this, we propose the PACT (Protocolized Action-state Communication and Transmission), which treats inter-agent communication as a public state-update problem and projects each raw agent output into a compact action-state record before it enters shared history. Across different MAS topologies, PACT consistently improves the performance-cost trade-off, achieving comparable or stronger task performance with substantially fewer tokens. The gains extend to production coding harnesses: PACT lifts OpenHands' resolve rate at -10% tokens-per-resolved, and is resolve-neutral on SWE-agent while halving input tokens. Our code is publicly available at https://github.com/iNLP-Lab/PACT.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：基于大语言模型（LLM）的多智能体系统（MAS）正被广泛部署，智能体之间通过自然语言进行通信。然而，这种自由形式的通信内容往往未被约束，导致 token 使用量快速膨胀、共享上下文窗口被消耗，最终影响系统性能和推理成本。
- **核心问题**：在多智能体系统中，一个智能体应该向另一个智能体传递什么内容？即 inter-agent message 的内容应该如何设计才能同时优化性能和成本。
- **现有研究的不足**：当前 MAS 研究主要关注角色分工、轮次调度和工具分配，而将智能体间的消息内容几乎完全留给了无约束的自由文本，这是尚未被深入探索的设计维度。
- **观察发现**：有效的 inter-agent 消息始终以行动为中心（action-centered），包含下游智能体所需的信息。

## 2. 论文提出的方法论

### 核心思想

- 将 inter-agent 通信视为**公共状态更新问题**，而非简单的文本摘要或压缩。
- 智能体之间传递的应当是**最小化的、行动相关的信息**，而非发送者的完整推理过程。
- 在每条消息进入共享历史之前，将其投影为紧凑的 action-state 记录。

### 关键技术：PACT（Protocolized Action-state Communication and Transmission）

- **原始更新规则**（标准 MAS）：直接将智能体原始输出 $y_t$ 追加到共享历史 $H_t^{std}$。
  $$H_t^{std+1} = H_t^{std} \oplus y_t$$
- **PACT 更新规则**：在追加之前，通过投影函数 $\Pi_{PACT}$ 将原始输出映射为紧凑的公共消息。
  $$H_t^{PACT+1} = H_t^{PACT} \oplus \Pi_{PACT}(y_t, o_t, r_t)$$

### Action-State 消息空间

PACT 将每条公共消息表示为三个面向接收者的字段：

1. **ACTION（行动）**：说明发送者已经做了什么，或下一个智能体应该做什么，使交接明确而非让接收者自行推断。
2. **STATE（状态）**：记录支撑消息的证据、观察、环境反馈或工具结果，使接收者能够将消息作为有据可查的公共状态来消费。
3. **RESULT（结果）**：包含行动输出本身，如检索到的事实、改进的解决方案或工具调用结果。

### 协议属性

- **不约束智能体私有计算**：智能体仍可进行内部推理和规划，PACT 仅控制哪些部分成为公共通信。
- **与标准 MAS 设计正交**：不引入新智能体、不改变轮次调度、不训练压缩器、不假设特定角色结构。
- **可序列化适配**：不同拓扑下可灵活序列化，例如 split-evidence 交互中显式写出所有字段，而顺序管道中可能通过角色调度隐含部分 action 信息。

## 6. 论文的主要结论与发现

1. **无固定策略是最优的**：五种常见通信策略在两个 MAS 拓扑中表现不一，没有一种策略在所有场景下均最优。

2. **全量内容冗余**：传递完整输出始终成本高昂，在顺序管道中 token 成本最高，且性能并非最优。

3. **简单缩短不够**：Concise Generation 在证据分割交互中表现良好，但在角色专业化管道中较弱；Brief Summary 行为不稳定，显示通用简洁性不是可靠的通信规则。

4. **结论仅在拓扑适配合适时有优势**：Conclusion Only 在固定角色顺序的管道中相对有效，但在需要交换证据的交互设置中很弱。

5. **Artifact Only 指明了正确方向**：Artifact Only 在交互设置中常达到最佳，且在管道中具有竞争力，说明接收者最受益于发送者的行动相关产物而非完整推理 trace。

6. **PACT 改善性能-成本前沿**：
   - 在 split-evidence 交互中，PACT 平均减少 38.7% token 使用量，同时保持或提升 F1
   - 在顺序管道中，PACT 使用 Multi-Agent Debate 所需 tokens 的 19%–23%，达到相当或更好的性能
   - 在 OpenHands 上，resolved 率从 19.40% 提升至 23.00%，tokens-per-resolved 减少 10.3%
   - 在 SWE-agent 上，输入 tokens 减少 50.4%，tokens-per-resolved 减少约 47%

7. **更强模型从紧凑通信中获益更多**：随着模型规模增大，PACT 的 token 使用量减少而性能提升，表明更强智能体能更好地利用紧凑证据和改进后的产物，无需发送者完整推理过程。

8. **字段消融验证**：完整的三字段（A+S+R）结构是最优的，每个字段的缺失均导致性能下降，其中移除 action 字段 F1 从 69.9 降至 64.9，移除 result 字段导致最大退化（F1 降至 64.3，token 增加 12.9%）。

## 7. 优点

- **问题定位精准**：将 inter-agent message 内容设计作为核心研究对象，而非配角或默认自由文本，这是一个新颖且重要的视角。
- **实验设计系统**：在两个结构不同的 MAS 设置上系统评估五种策略，覆盖多种模型规模和基准，实验设计完整且可复现。
- **方法简洁有效**：PACT 不需要额外训练、不引入新智能体、不改变底层 agent 能力，仅通过投影函数控制公共消息内容，易于实现和部署。
- **理论与实践结合**：不仅在受控研究中验证，还成功迁移到 OpenHands 和 SWE-agent 等真实生产系统中，证明了方法的实用性。
- **消融验证清晰**：通过完整的三字段消融实验，明确验证了 Action、State、Result 各字段的贡献。
- **性能提升显著**：在 token 使用量大幅减少（38.7% 平均减少、50% 在 SWE-agent 上）的同时，还能提升或保持任务性能。

## 8. 不足与局限

- **适用场景有限**：PACT 主要针对共享对话历史是主要 token 成本来源的 MAS 场景，对于短交互或不重复暴露先前智能体输出的架构，其收益未被充分探索。
- **实验覆盖不完整**：论文未覆盖开放式辩论、重度工具规划或动态路由智能体网络等协作形式。
- **硬件细节缺失**：未明确 GPU 型号、数量和具体训练/推理时间，算力成本评估不够透明。
- **真实场景迁移有限**：仅在两个编码智能体上验证，其他领域的生产系统（如客服、搜索、规划等）尚未测试。
- **消息格式灵活性**：PACT 的三个字段结构可能需要针对不同任务进行调优或适配，目前尚无自动化的格式推断机制。
- **潜在偏差风险**：在 artifact-only 策略中发现，小模型可能因 artifact 省略了有用的 state 信息而表现较弱，PACT 在极端弱模型上的表现可能存在类似风险。
- **结论依赖特定模型族**：所有实验基于 Qwen3 模型族，其他 LLM 系列（如 Claude、GPT 系列）的表现未经验证。
