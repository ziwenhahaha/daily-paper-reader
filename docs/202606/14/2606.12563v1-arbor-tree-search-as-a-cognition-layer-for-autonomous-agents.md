## Abstract
Arbor is a multi-agent framework that introduces structured tree search as a cognition layer for autonomous agents operating in large, stateful action spaces. Prior autonomous optimization systems operate on isolated targets with stateless evaluation. Arbor instead maintains an explicit search tree of scored hypotheses that serves as the shared working memory across agents, evolving with every measurement, treating failures as diagnostic signal that reshapes subsequent exploration, and expanding as prior successes shift the bottleneck distribution.   We validate Arbor on full-stack LLM inference optimization, a domain where achieving peak performance has historically required coordinated effort from engineering teams across the application, framework, compiler, kernel, and hardware stack. Arbor pairs an Orchestrator agent, which drives optimization by delegating to Domain Specialists across the inference stack, with a Critic agent that safeguards stability through root-cause analysis, introspection, and measurement validation -- a checks-and-balances architecture where neither agent can unilaterally drive the system. Agent capabilities are decomposed into hard skills (domain expertise) and soft skills (coordination protocols that determine how contributions compose), enabling fully autonomous multi-day campaigns. Arbor achieves up to 193% inference throughput-latency Pareto improvement over vendor-optimized baselines, while a single agent without the harness plateaus at +33% throughput improvement and crashes irrecoverably within hours. Arbor generalizes to multiple generations of hardware platform, and run-to-run variance is within 2 percentage points demonstrating that the method is hardware-agnostic and reproducible.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题本质**：大型 LLM 推理系统横跨应用、框架、编译器、内核与硬件等多层技术栈，协同优化历来需要工程团队协调配合且耗时巨大。现有自主优化系统（如 AlphaEvolve、AVO、KernelSkill 等）仅针对孤立目标进行无状态评估，无法处理跨层干预的级联效应——某一层的优化可能导致另一层的回归，或暴露出先前不可见的新瓶颈。
- **核心挑战转变**：随着智能体被应用于越来越复杂的系统，核心挑战已从候选方案生成转向候选方案选择——如何在庞大、有状态的动作空间中进行有效导航。当优化导致系统故障时，故障必须被诊断且诊断结果需作为后续搜索的约束条件传回；当成功干预使瓶颈转移到不同层时，搜索必须通过重新性能分析检测该转移并扩展到先前不存在的动作空间区域。
- **研究空白**：现有系统既不容忍跨层探索产生的级联失败，也不从这些失败中提取可复用的信号。

## 2. 论文提出的方法论

### 核心思想

Arbor 将全栈性能优化形式化为**启发式树搜索**，在一组动态扩展的动作空间上进行搜索。系统维护一个**显式搜索树**作为跨智能体共享的工作记忆（working memory），该树以评分假设的形式存在，随每次测量结果演化。失败被视为诊断信号，重塑后续探索方向；先前的成功则通过重新性能分析触发瓶颈分布变化，进而扩展搜索前沿。

### 关键技术细节

#### 多智能体架构

| 智能体 | 角色定位 | 核心职责 |
|--------|----------|----------|
| **Orchestrator（编排器）** | 类似技术负责人（tech lead） | 驱动优化流程：分析工作负载、构建评分动作栈、委派给领域专家、迭代实现-验证-重新评分循环 |
| **Domain Specialists（领域专家）** | 动态构建的领域智能体 | 在内核、框架、通信、编译器、操作符调度等各层执行迭代式优化与局部验证，返回可合并的补丁 |
| **Critic（批评者）** | 类似质量保障功能 | 通过根因分析、内省、护栏和稳定性监控保障系统稳定性，对 Orchestrator 实施制衡 |

#### 硬技能与软技能分解

- **硬技能**：各智能体的领域专业知识（如内核优化、编译器配置等）
- **软技能**：协调协议（何时委派工作、如何整合他人发现、何时升级超出自身能力范围的问题）

#### 评分启发式函数

对每个候选动作 $a$ 的评分为：

$$h(a) = \frac{g(a)}{c(a)} \cdot (1 - r_{acc}) \cdot (1 - r_{crash}) \cdot m_{gap} + \frac{C_s \ln N}{1 + n_{cat}}$$

其中：
- $g(a)$：预期收益（基于目标内核的 GPU 时间占比与类别加速先验的乘积）
- $c(a)$：估算的墙上时钟成本（实现变更、重启推理服务器、运行端到端基准测试所需时间）
- $r_{acc}$、$r_{crash}$：动作类别的经验失败率
- $m_{gap}$：紧迫度乘数（距吞吐量目标的剩余差距，范围 1.0~2.0）
- UCB 风格的第二项鼓励探索采样不足的动作类别

#### 搜索流程（深度优先循环）

1. 对当前系统状态进行性能分析，识别瓶颈及其 GPU 时间占比
2. 使用启发式 $h(a)$ 对所有候选动作评分，选择得分最高者
3. 将动作委派给适当的领域专家进行实现
4. 通过准确性验证和端到端基准测试对返回结果进行门控
5. 根据结果更新树（保留的动作成为后续性能分析的新基线；回退的动作记录诊断注释；崩溃的动作触发批评者的根因分析）
6. 重新评分剩余候选者（因瓶颈分布可能已转移）
7. 循环直至动作分数低于阈值或墙上时钟预算耗尽

#### 知识库（Knowledge Base）

持久化知识库使 Arbor 的搜索在多次运行间具有累积性：
- 每次运行的动作结果被记录，Orchestrator 通过相似性检索相关条目以更新评分先验
- 运行结束时，原始追踪被提炼为结构化条目（已验证技术、失败模式、参数级发现）
- 继承的失败模式在首次测量前剪枝分支；参数级先验根据特定设置条件化评分

## 6. 论文的主要结论与发现

### 性能提升

- 在 AMD MI355X 上，Arbor 在 6 个生产模型上实现 **+40% 至 +193%** 的吞吐量改进
- 在 AMD MI300X 上实现 **+62% 至 +99%** 的吞吐量改进，验证了跨代泛化能力

### 关键发现

1. **多智能体制衡架构的必要性**：单智能体配置（无树搜索）仅达到 +33% 改进且在 4 小时内崩溃；无领域专家的配置在 6 小时后耗尽；无批评者的配置产生无效优化（+12.9%/+16.5%）或违反约束（如准确率降至 0%）

2. **跨层交互的显著影响**：在 GLM-5-FP8 上，将张量并行度从 TP=8 降至 TP=4 并协同优化 NSA 注意力内核和 MoE 调度，实现 2.9 倍吞吐量提升——这是任何单层优化都无法达到的

3. **稳定性-吞吐量权衡的长期价值**：DeepSeek-R1-0528 优化过程中，为换取无崩溃服务而牺牲 GEMM 内核约 5-10% 的吞吐量，最终实现 +90.0% 的整体提升

4. **端到端验证的重要性**：39% 的内核级改进在部署到服务管道时回归端到端吞吐量（因布局变化、融合通过禁用等）

5. **可重复性**：两次独立运行的优化轨迹收敛到 2 个百分点以内，表明搜索形式化可靠地收敛到相当的结果

## 7. 优点

1. **创新的树搜索认知层**：首次将结构化树搜索作为自主智能体的认知层，维护显式搜索树作为跨智能体共享工作记忆，将失败转化为诊断信号，使先验成功触发搜索前沿动态扩展

2. **制衡式多智能体架构**：Orchestrator 追求激进的性能提升，Critic 强制保障稳定性的约束，两者相互制衡，任何一方都不能单方面主导系统——解锁了通过持续探索才能获得的收益

3. **硬技能与软技能的优雅分解**：将智能体能力分解为领域专业知识（硬技能）和协调协议（软技能），使贡献如何组合变得显式且可管理

4. **动态领域专家构建**：领域专家不是预定义的智能体，而是在运行时由 Orchestrator 动态构建，根据任务、硬件上下文和相关知识库条目组装提示，实现上下文自适应的专业化

5. **知识积累与迁移**：持久化知识库使每次新运行从严格强于上一次运行的先验开始，直接将跨运行累积的领域知识编织到专家智能体提示中

6. **实用的搜索机制**：基于经典 MCTS 的深度优先搜索配合 UCB 风格的探索项，同时利用知识库先验，成本/风险惩罚自然优先探索上层干预

7. **全面的实验验证**：在多种模型架构、量化方案和两代 AMD GPU 上验证，辅以系统性消融和可重复性分析
