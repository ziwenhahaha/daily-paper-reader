## 摘要
评分规则已成为开放域中 RLVR 的替代方案，因为在这些领域单一的真实最终答案并不存在。现有的基于评分规则的训练方法依赖 LLM 验证器根据评分规则对每次 rollout 进行评分。这带来了大量训练时间开销，使优化暴露于验证器特定的偏差，并将评分规则反馈降为稀疏的轨迹末端信号。我们提出基于评分规则的自蒸馏（RGSD），这是一种无需验证器的训练方法，其中基于评分规则的条件化基础策略作为无条件的学生的教师。RGSD 逐 token 地将评分规则条件化的教师分布蒸馏到学生中，用密集的每 token 学习信号替代稀疏的轨迹级奖励，并完全从训练循环中移除 LLM 评判器。在医学和科学领域的 Qwen-2.5（3B, 7B）和 Qwen3-Thinking（4B, 8B）模型上，RGSD 实现了与基于评判器的 GRPO 相当的评分规则满足度，同时每个 prompt 仅使用一次 on-policy rollout，且无需训练时验证器调用。消融实验表明，原始评分规则比自生成的参考响应提供了更强的教师丰富信号，而更强的 GRPO 评判器在某些场景下可以优于 RGSD，这使得 RGSD 成为当验证器成本或可靠性成为瓶颈时的互补性无验证器替代方案。

## Abstract
Rubrics have emerged as an alternative to RLVR in open-ended domains where a single ground-truth final answer is not available. Existing rubric-based training methods rely on an LLM verifier that scores each rollout against rubrics. This introduces substantial training-time overhead, exposes optimization to verifier-specific biases, and reduces rubric feedback to a sparse end-of-trajectory signal. We propose Rubric-Guided Self-Distillation (RGSD), a verifier-free training method in which the base policy, conditioned on the rubric, serves as the teacher for the unconditioned student. RGSD distills the rubric-conditioned teacher distribution into the student token-by-token, replacing sparse trajectory-level rewards with dense per-token learning signals and removing the LLM judge from the training loop entirely. Across Qwen-2.5 (3B, 7B) and Qwen3-Thinking (4B, 8B) models on medical and science domains, RGSD achieves rubric satisfaction comparable to judge-based GRPO while using one on-policy rollout per prompt and no training-time verifier calls. Ablations show that raw rubrics provide a stronger teacher enrichment signal than self-generated reference responses, while a stronger GRPO judge can outperform RGSD in some settings, positioning RGSD as a complementary verifier-free alternative when verifier cost or reliability is the bottleneck.

## 1. 核心问题与整体含义

**研究背景与动机：**
- 在开放域任务（如临床建议、科学论述、指导遵循、创意写作）中，传统的强化学习与可验证奖励（RLVR）方法不适用，因为这些任务缺乏单一的真实最终答案
- 社区转向使用评分规则（Rubrics）——针对每个提示的加权标准清单，通过 LLM 验证器对响应进行评分

**现有方法的核心问题：**
- **计算成本高昂**：每次 rollout 都需要调用 LLM 验证器进行评分，产生显著的训练时间开销
- **验证器盲点**：LLM 评判器存在固有偏差，导致奖励黑客（reward hacking）问题持续存在
- **信用分配困难**：评分规则将多标准评估压缩为单一标量奖励，仅提供稀疏的轨迹级信号，难以精确定位哪些决策改善或损害了评分规则满足度

**核心研究问题**：如何在不使用 LLM 验证器的情况下，利用评分规则实现有效的后训练？

## 2. 论文提出的方法论

**核心思想：**
RGSD（Rubric-Guided Self-Distillation）利用一个关键观察：当基础模型在上下文中看到每个提示的评分规则时，其响应评分比无条件的基线模型高出 30-45 个百分点（rubric-conditioning gap）。RGSD 将这种"条件化提升"转化为训练信号，通过自蒸馏实现。

**关键技术设计：**

| 组件 | 说明 |
|------|------|
| **Student（学生）** | 基于基础模型，可训练，仅以提示 q 为条件 |
| **Teacher（教师）** | 同一基础模型的冻结副本，以提示 q + 评分规则 R 为条件 |
| **蒸馏目标** | 教师在每个学生访问的前缀位置提供逐 token 的目标分布 |

**算法流程：**
1. **初始化**：学生权重 θ_S ← θ_base，教师权重 θ_T ← θ_base（冻结）
2. **采样**：从学生策略采样 on-policy rollout：y ~ π_S(·|q)
3. **前向传播**：将 (q, R, y) 通过冻结教师，获取每个位置的教师分布 π_T(·|q, R, y<t)
4. **更新**：使用裁剪的 Jensen-Shannon 散度更新学生，仅学生接收梯度

**损失函数：**
$$L_{RGSD}(θ_S) = E_{(q,R) \sim D, y \sim π_S(·|q)} \left[ \frac{1}{T} \sum_{t=1}^{T} D_{clip}^{\beta}(π_S(·|q, y<t) || π_T(·|q, R, y<t)) \right]$$

其中 D_clip^β 是裁剪的 Jensen-Shannon 散度，插值于正向 KL（β=0）和反向 KL（β=1）之间，本文使用 β=0.5。

**特殊处理（针对推理模型）：**
- 对于 Qwen3-Thinking 等推理模型，教师可能在思考痕迹中引用具体评分标准
- 掩码  和 </think> 之间的 token，使其不参与损失计算，避免泄漏

## 6. 论文的主要结论与发现

**核心发现：**

| 指标 | RGSD | GRPO |
|------|------|------|
| 医学领域平均提升 | +6.1 pp | +5.9 pp |
| 科学领域平均提升 | +4.9 pp | +4.5 pp |
| 训练时评判器调用 | 0 | ~1M（医学）/ ~1.6M（科学） |
| Rollouts/prompt | 1 | 16 |
| 响应长度（Qwen-2.5） | ~700 tokens | ~1200 tokens |

**详细结论：**

1. **质量可比性**：RGSD 在评分规则满足度上与 GRPO 相当，平均增益相近，个别模型差异混合

2. **计算效率优势**：RGSD 完全移除训练时 LLM 评判器，每个提示仅需一次 on-policy rollout

3. **长度控制**：在 Qwen-2.5 模型上，RGSD 避免 GRPO 出现的严重冗长性增长，最终响应长度约为 GRPO 的 1/2

4. **虚假声明问题**：GRPO 在医学领域训练后虚假声明率从 30.5% 升至 45.1%，而 RGSD 仅升至 35.1%

5. **评分规则条件化差距的诊断价值**：
   - 医学：条件化提升 ~43-46 pp，RGSD 恢复约 20%
   - 科学：条件化提升 ~31-35 pp，RGSD 恢复约 11%

6. **强评判器的权衡**：更强的 GRPO 评判器可超越 RGSD（如 gpt-oss-120b 在科学领域），但仍需高昂的评判器成本

7. **互补性定位**：RGSD 适合评判器成本高、不可靠或操作困难的场景；强评判器 GRPO 适合追求最高评分规则得分的场景

## 7. 优点

**方法设计亮点：**

1. **优雅的验证器移除**：通过自蒸馏将外部评分反馈转化为内部教师分布，无需任何 LLM 评判器介入训练循环

2. **密集 vs 稀疏监督**：将稀疏的轨迹级标量奖励替换为密集的逐 token 蒸馏信号，提供 O(N) vs O(1) 的监督效率

3. **评分规则双重用途**：创新地将评分规则从"外部评分工具"重新定位为"特权教师上下文"，展示了评分规则的新范式

4. **条件化差距作为诊断**：提出用条件化差距诊断 RGSD 潜在效果，为实践者提供轻量级前期评估手段

5. **推理模型特殊处理**：针对思考模型的思考痕迹泄露问题提出掩码策略，体现工程细节的完善

**实验设计亮点：**

1. **多维度覆盖**：4 个基础模型、2 个领域、4 个评估基准（含 OOD）

2. **成本透明**：明确量化 GRPO 的评判器调用成本（~1M/1.6M），便于实践者权衡

3. **事实性审计**：超越表面评分，深入分析响应内容的事实准确性

4. **多种子验证**：对关键配置进行 3 种子实验，量化训练噪声

## 8. 不足与局限

**实验覆盖局限：**

1. **单次运行普遍性**：主表结果基于单次运行，虽有 3 种子实验验证关键配置，但其他配置缺乏方差估计

2. **基线对比有限**：未与 PPO、DPO、或其他蒸馏方法对比，难以全面评估 RGSD 在方法谱中的位置

3. **跨家族实验不完整**：OLMo、Mistral 的 GRPO 对比未运行，削弱了跨家族泛化结论的可信度

**偏差与可靠性风险：**

1. **评估依赖 LLM 评判器**：虽然移除训练时评判器，但评估仍使用 gpt-5.4，评分仍可能继承评判器特定偏差

2. **虚假声明审计的局限性**：基于 LLM 的提取-验证流程本身可能存在偏差，论文将其定位为"支持性证据"而非主要基准

3. **条件化差距作为代理**：评分规则条件化差距是 RGSD 效果的上限代理，但二者的关系并非严格线性

**应用范围限制：**

1. **依赖基础模型的条件化能力**：RGSD 效果受限于基础模型在看到评分规则时的响应提升幅度；若条件化差距小，RGSD 增益有限

2. **不适合最强评判器场景**：当可用且负担得起更强评判器时，GRPO 可能优于 RGSD

3. **推理模型需要特殊处理**：思考 token 掩码是实用工程技巧，但增加了实现复杂度，且掩码策略的效果因领域而异

4. **评分规则可用性假设**：假设每个训练实例都配有高质量的评分规则，但评分规则的质量本身未被评估

**可扩展性考量：**

1. **更大规模未验证**：实验限制在 3B-8B 模型家族，更大规模模型的表现未知

2. **多轮对话场景缺失**：仅验证单轮提示场景，多轮对话中的评分规则应用未探索

3. **领域覆盖有限**：仅测试医学和科学领域，其他开放域（如创意写作、法律）的适用性待验证
