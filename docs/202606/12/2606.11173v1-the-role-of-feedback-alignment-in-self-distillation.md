## Abstract
Conditioning a language model on additional context, such as feedback on a previous attempt, typically improves its response. Self-distillation trains the model to retain this improvement when the context is not present. The method works by matching the model's output distribution under two settings: a student that sees only the question, and a self-teacher that also sees the context. What the model learns therefore depends on what context the self-teacher receives, yet the design of this context remains largely unexplored.   We study context design for self-distillation by training a solver on feedback from a frozen critic. We compare three conditions: (i) a binary reward (GRPO), (ii) the reference solution, and (iii) a step-by-step critique aligned to the solver's reasoning trace.   Step-aligned critique yields the largest gains, outperforming GRPO by 16.11 points and reference-solution-conditioned self-distillation by 5.27 points (Avg@12). Per-token advantage analysis reveals why: step-aligned feedback targets only the tokens where reasoning fails, leaving correct behavior intact. Conditioning on the reference solution, by contrast, pressures the model to change its behavior at every token (even correct steps) because an alternative derivation inevitably differs in phrasing and approach. This suggests that structural alignment between feedback and the solver's reasoning is a key driver of self-distillation effectiveness.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究问题**：自蒸馏（Self-Distillation）通过让同一个语言模型扮演"学生"（只看问题）和"自老师"（额外看到上下文反馈）两个角色来训练模型。模型学什么取决于自老师收到的上下文（context），但**如何设计这个上下文**此前没有被系统研究过。
- **核心动机**：在数学推理领域，向语言模型提供反馈（如来自另一个模型的评价）是提升推理能力的常见手段。但不同形式的反馈（binary reward、reference solution、step-by-step critique）究竟哪种最有效，目前缺乏系统比较。
- **研究空白**：现有自蒸馏工作将上下文视为固定选择，未探索上下文设计对学习效果的影响。

## 2. 论文提出的方法论

### 核心思想
- **自蒸馏框架**：同一模型 π_θ 在两种条件下产生输出分布——学生只看问题 x，自老师额外看到上下文 c（反馈）。训练目标是最小化两个分布之间的逐 token 散度（forward KL）。
- **损失函数**（式 3）：L_SD = E_{y~π_θ(·|x)}[ D( π_θ(y|x) || sg[π_θ(y|x,c)] ) ]
- **逐 token 优势**（式 4）：A_SD^t = log π_θ(ŷ_t|x,c,y<t) − log π_θ(ŷ_t|x,y<t)。该优势量化了上下文如何改变模型在每个 token 位置上的预测，提供天然的密集信用分配。

### 三种上下文条件对比
| 条件 | 描述 | 教师来源 |
|------|------|----------|
| **GRPO** | 标准 RLVR 基线，仅有二元奖励（对/错），无自蒸馏 | N/A（纯 RL） |
| **RefSol** | 自老师看到参考解答（reference solution），直接作为上下文 | 数据集提供的标准解法 |
| **StepAlignFB** | 自老师看到**逐 step 对齐的批判性反馈**，批评家针对学生的推理轨迹逐步骤修正 | 冻结的批评家模型（QwQ-32B）生成 |

### StepAlignFB 的关键设计
- **忠实的抄写员约定（Faithful-Scribe Convention）**：批评家将学生**正确的步骤逐字复制**，仅在错误步骤处替换为正确内容，续写后续推导。
- **避免完整原文引用**：完整引用学生错误步骤会触发 induction head 复制机制，使模型继续复现错误，导致优势信号弥散为正。
- **机制解释**：部分重复（正确前缀逐字复制 + 错误步骤不锚定）恰好利用了 induction head 的复制偏向，将其从错误源转化为信用分配工具。

## 3. 实验设计

### 数据集与 Benchmark
- **训练集**：OpenMathReasoning 的难度和格式过滤子集。经多轮筛选（pass_rate_72b_tir > 0.85、Avg@16 < 5/16、格式准确率 > 0.9），最终保留 **312 道题目**（282 训练 / 30 评估）。
- **筛选理由**：自蒸馏的优势在难题上更显著；同时要求题目对 1.7B 模型可解且批评家能解，以确保反馈有效。
- **评估指标**：Avg@12（平均正确率）、Maj@12（多数投票正确率）、Pass@12（至少一个正确的题目比例）、平均答案长度。

### 对比方法
1. **GRPO**（二元奖励基线）：group size G=8，8× 推理量以匹配计算。
2. **RefSol**（参考解答条件化）：自老师看到标准参考答案。
3. **StepAlignFB**（逐步对齐反馈）：批评家生成结构化逐步骤批判。

### 模型配置
- **求解器（Solver）**：Qwen3-1.7B，LoRA 微调（rank=64, α=128），Thinking-Mode-Off。
- **批评家（Critic）**：Qwen/QwQ-32B（冻结），贪婪解码，仅提取结构化评判输出。

## 6. 论文的主要结论与发现

1. **StepAlignFB 效果最佳**：在 Avg@12 上比 GRPO 高 **16.11 分**，比 RefSol 高 **5.27 分**；Maj@12 提升尤为显著（+13.33 分），说明策略更集中地将概率集中在正确答案上。

2. **逐 token 优势分析揭示机制**：
   - StepAlignFB：在错误步骤处产生尖锐负优势，在正确步骤处产生正优势——类似 PRM 的过程监督效果，无需训练奖励模型。
   - RefSol：在所有步骤上产生弥散负优势，因为替代解法即使在正确步骤上也与学生轨迹的表述形式不同，混淆了风格分歧与真实错误。

3. **Induction Head 复制机制是核心**：
   - 完全逐字引用错误步骤 → 错误被锚定，优势弥散为正。
   - 完全省略正确步骤 → 正确轨迹的优势也变负，模型漂移。
   - **部分逐字复制**（复制正确前缀 + 替换错误步骤）恰好将 induction head 的复制偏向转化为信用分配工具。

4. **自蒸馏的隐式过程监督**：StepAlignFB 无需人工逐步骤标注或训练 PRM，仅通过文本反馈 + 自蒸馏机制即获得 token 级密集信用分配。

5. **实践权衡**：StepAlignFB 需要能力强的批评家模型，增加训练成本和系统复杂度；RefSol 的参考解法可跨多次训练复用，但效果不及 step-aligned 反馈。

## 7. 优点

- **问题定位精准**：首次系统研究自蒸馏中"上下文设计"的影响，填补了该方向的研究空白。
- **机制解释深入**：不仅报告性能提升，还通过逐 token 优势分析揭示了"为何有效"的机制，并将现象归因于 induction head 复制机制。
- **实验设计严谨**：唯一变量原则（反馈形式）、GRPO 计算量匹配、checkpoint 独立选择，确保公平比较。
- **实践指导价值高**：忠实抄写员约定（Hard Rules）提供了可直接操作的 prompt 设计指南。
- **理论联系紧密**：将 step-aligned 反馈的效果与 PRM（过程奖励模型）文献关联，指出自蒸馏可从文本反馈中"隐式蒸馏"出 token 级过程监督。

## 8. 不足与局限

- **数据集单一**：仅在 OpenMathReasoning 数学推理数据集上验证，结论是否泛化到其他领域（代码、科学问题、通用推理）尚不清楚。
- **评估规模较小**：30 道测试题的评估集可能导致统计波动，Maj@12 的大幅提升（+13.33 分）需更大规模验证。
- **批评家模型依赖**：StepAlignFB 依赖 QwQ-32B 这类强推理模型，成本高；未探索更小模型作为批评家的效果。
- **任务范围受限**：仅聚焦数学推理，未覆盖需要外部知识或事实核查的开放域任务。
- **Checkpoint 敏感性**：各指标最优 checkpoint 差异大，实际应用中需要额外的验证集来选择最优模型，增加了调优复杂度。
- **公平性潜在风险**：RefSol 使用的参考解法来自强模型（与 StepAlignFB 的批评家同源），若使用更弱的参考解法可能影响对比结果。
