## 摘要
多智能体系统（MAS）可以通过将复杂问题分解为并行子任务，在测试时扩展大型语言模型的推理能力。然而，现有的多智能体系统大多依赖中心化的编排——即由主智能体分配任务、收集输出并合并结果。随着子任务数量的增加，这种控制器会成为通信与整合的瓶颈。我们提出去中心化语言模型（DeLM），这是一种通过并行智能体、共享已验证上下文和任务队列来实现去中心化协同的 MAS 框架。智能体以异步方式认领子任务、读取累计进度、执行本地推理并回写紧凑的已验证更新。共享上下文充当公共通信基垫，使智能体能够基于彼此已验证的进度进行构建，而无需将每次更新都通过中心控制器路由。实验表明，DeLM 在软件工程测试时扩展和长上下文推理方面均有所提升。在 SWE‑bench Verified 上，DeLM 在 Avg.@1、Pass@2 和 Pass@4 上取得了最佳性能，相较于最强基线提升高达 10.5 个百分点，同时将每个任务的成本降低约 50%。在 LongBench‑v2 Multi‑Doc QA 上，DeLM 在四个前沿模型家族中实现了最高的平均准确率，较最强基线提升最高 5.7 个百分点。代码已发布在我们的项目网站 https://yuzhenmao.github.io/DeLM/。

## Abstract
Multi-agent systems (MAS) can scale large language model reasoning at test time by decomposing complex problems into parallel subtasks. However, most existing MAS rely on centralized orchestration, where a main agent assigns work, collects outputs, and merges results. As the number of subtasks grows, this controller becomes a communication and integration bottleneck. We propose Decentralized Language Models (DeLM), a MAS framework that decentralizes coordination through parallel agents, a shared verified context, and a task queue. Agents asynchronously claim subtasks, read accumulated progress, perform local reasoning, and write back compact verified updates. The shared context acts as a common communication substrate, enabling agents to build on one another's verified progress without routing every update through a central controller. Empirically, DeLM improves both software-engineering test-time scaling and long-context reasoning. On SWE-bench Verified, DeLM achieves the best performance across Avg.@1, Pass@2, and Pass@4, with gains of up to 10.5 percentage points over the strongest baseline, while reducing cost per task by roughly 50%. On LongBench-v2 Multi-Doc QA, DeLM achieves the highest average accuracy across four frontier model families, improving over the strongest baseline by up to 5.7 percentage points. The code is available on our project website at https://yuzhenmao.github.io/DeLM/.

### 研究背景
- **多智能体系统（MAS）的价值**：在大模型测试时，通过将复杂问题分解为并行子任务，可以有效扩展 LLM 的推理能力

  - 长上下文推理（Long-context Reasoning）：如多文档问答，智能体并发处理不同证据簇

### 核心问题
- **现有 MAS 的中心化瓶颈**：大多数系统依赖中心化编排（Centralized Orchestration），即主智能体分配任务、收集输出并合并结果

  - 主智能体在路由过程中可能稀释、遗漏或扭曲有用细节
  - 随着子任务增长，协作变得更慢、更迭代，受限于单个过载的主智能体

### 研究动机
将协调从中心控制器转移到共享问题状态，使智能体能够通过共享上下文直接交换进展，而非通过中心控制器路由每次交互。

## 2. 论文提出的方法论

### 核心思想
**DeLM（Decentralized Language Models）** 提出去中心化协同框架，通过三个核心组件实现：
1. **并行智能体（Parallel Agents）**
2. **共享验证上下文（Shared Verified Context）**
3. **任务队列（Task Queue）**

### 关键技术细节

#### 2.2.1 共享上下文与任务队列
- **共享上下文 C**：存储紧凑、可复用的进展状态，包括源证据、完成结果、失败假设、中间约束
- **任务队列 T**：存储待处理子任务，智能体异步认领
- **关键设计**：C 包含 **gist 条目 Gi**（压缩摘要），而非原始轨迹；每个 gist 指向可按需展开的详细证据

#### 2.2.2 压缩与验证准入机制
- **分层表示**（长源单元）：$r_i \rightarrow S_i \rightarrow G_i$
  1. **阶段 1（参考锚定摘要）**：将源单元压缩为带引用标签的子弹列表，每个子弹指向原文中连续的支撑片段
  2. **阶段 2（紧凑 gist）**：进一步压缩为高度紧凑的摘要，仅保留与问题最相关的核心信息
- **选择性展开（Selective Unfolding）**：
  - 默认读取 gist
  - 按需展开到摘要层 $G \rightarrow S$（UNFOLD 指令）
  - 或进一步展开到原始内容 $S \rightarrow raw$（DEEP_UNFOLD 指令）
- **准入时验证（Admission-Time Verification）**：
  - 推理轨迹：验证 gist 是否忠实保留原始结果中的发现、失败、反馈或约束
  - 长源单元：双重验证——摘要层需锚定原始源，gist 层需保留摘要中的关键声明和限定词
  - 不支持的 gist 被拒绝或重生成

#### 2.2.3 五阶段执行流程（Algorithm 1）
1. **初始化**：从输入生成初始子任务，填入任务队列
2. **并行执行**：智能体异步认领任务，读取共享上下文，执行本地推理
3. **压缩 + 验证 + 准入**：将结果压缩为 gist，验证后加入共享上下文
4. **生成更多子任务**：队列为空时，检查是否需要额外子任务
5. **生成最终答案**：当队列为空且无更多子任务时，从验证后的共享上下文生成答案

## 8. 不足与局限

### 8.1 实验覆盖局限
- **Benchmark 选择有限**：主要集中在软件工程和长上下文问答，未覆盖更多实际应用场景（如自动化研究）
- **资源信息缺失**：未明确报告算力需求，难以评估部署成本

### 8.2 方法局限
- **验证开销**：准入时验证带来额外计算开销，虽然文中表明开销适度，但可能限制极端规模部署
- **分解质量依赖**：DeLM 继承智能体的分解质量；过于粗粒度导致子任务不明确，过于细粒度导致不必要的智能体和复杂推理
- **提示适配**：不同模型家族可能需要定制提示以获得预期行为

### 8.3 潜在偏差风险
- **验证器可靠性**：依赖 LLM 作为验证器，可能存在误判风险；更轻量的验证器（如学习模型或规则检查）尚未充分探索
- **适应性分解**：缺乏自适应决定何时拆分、合并或终止子任务的机制

### 8.4 应用限制
- **结构化数据处理**：在需要精确代码执行的场景（如 OOLONG 的聚合任务），纯自然语言共享上下文不如代码中介执行可靠
- **超大规模**：随着智能体数量增长，共享上下文的一致性和访问效率可能面临挑战

DeLM 提出了一种创新的去中心化多智能体协调框架，通过共享验证上下文和异步任务队列有效解决中心化编排的瓶颈问题。实验表明，该方法在软件工程测试时扩展和长上下文推理任务上均显著优于现有方法，同时大幅降低成本。DeLM 的验证准入机制和层级压缩设计为可靠、可扩展的多智能体协作提供了坚实基础。
