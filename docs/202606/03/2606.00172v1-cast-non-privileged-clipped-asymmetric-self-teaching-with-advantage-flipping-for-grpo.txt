## ABSTRACT 

Reinforcement learning with verifiable rewards (RLVR), especially Group Relative Policy Opti-mization (GRPO), has been widely used to improve reasoning in large language models. However, outcome-level rewards provide only sparse supervision, and group-relative advantages vanish when all sampled trajectories for a prompt are either correct or incorrect. On-Policy Self-Distillation (OPSD) offers dense token-level guidance, but its token preferences are not necessarily aligned with trajectory correctness; empirical diagnostics show that OPSD signals behave differently on correct and incorrect rollouts, with teacher-positive and teacher-negative gap signals exhibiting different noise profiles. These diagnostics are conducted under an OPSD-style privileged teacher context for analysis only, whereas CAST training uses answer-free self-teacher scoring. Motivated by these observations, this work proposes CAST (Non-Privileged Clipped Asymmetric 

Self-Teaching with Advantage Flipping for GRPO), an answer-free self-distillation method for GRPO-style RLVR. CAST keeps the verifier-grounded GRPO objective, but uses a stop-gradient self-teacher to shape token-level advantages according to trajectory correctness. Unlike prior self-distilled RLVR methods, CAST does not require reference-solution-conditioned teacher scoring, keeps the self-teacher log-probability gap active throughout training, and applies bidirectional local advantage sign reversal: teacher-negative tokens in correct trajectories can receive negative token-level advantages, while teacher-positive tokens in incorrect trajectories can receive bounded positive local advantages. For zero-variance all-correct and all-wrong groups, CAST assigns bounded sign-constrained base advantages, so these otherwise zero-gradient groups can contribute verifier-signed token feedback. Experiments on mathematical reasoning show that CAST improves RLVR training while retaining a lightweight, verifier-grounded trajectory-level objective. 

## 1 Introduction 

Reinforcement learning with verifiable rewards (RLVR) has become a widely used paradigm for improving reasoning in large language models [4, 28, 22, 7, 21]. Group Relative Policy Optimization (GRPO) [20] samples multiple trajectories per prompt and estimates group-relative advantages from verifier outcomes, offering a scalable alternative to critic-based policy optimization [24, 3, 8]. Despite its success, GRPO provides only sparse, outcome-level supervision [15, 30, 29, 10]: it says little about which tokens help or hurt, and the group-relative advantage vanishes when all rollouts for a prompt are correct or all are incorrect [30, 13]. These zero-variance groups still contain useful consolidation or failure signals, motivating dense token-level guidance that remains verifier-grounded. On-policy self-distillation [17, 1, 32, 31, 6] supplies such guidance, and recent RLVR variants [29, 12] already exploit teacher–student gaps beyond sparse rewards. However, three issues remain: gaps are not interpreted differently on

## 3 Method 

3.1 Motivation: Correctness-Aware Self-Teacher Shaping 

Self-distilled RLVR methods use On-Policy Self-Distillation (OPSD) to provide dense token-level guidance. Given a student rollout, OPSD compares the probability assigned by a stop-gradient self-teacher and the current student to each generated token. Standard OPSD treats this teacher–student gap as a generic token preference: tokens with higher teacher log-probability are encouraged, while tokens with lower teacher log-probability are suppressed. In RLVR, however, each trajectory is also labeled by verifier correctness. This trajectory-level label changes how the teacher signal should be interpreted. A teacher-positive token inside an incorrect trajectory should not receive the same credit as one inside a correct trajectory, and a correct trajectory may still contain locally undesirable tokens such as redundant answer templates or termination markers. Uniformly reinforcing every token in a successful rollout can therefore consolidate spurious reasoning; conversely, local suppression may benefit some teacher-negative tokens when the gap indicates low support, while final advantage clipping limits over-suppression risk. To make this concrete, an OPSD token-shift diagnostic on 2,048 rollouts from 256 prompts (926 correct and 1,122 incorrect trajectories) compares four strata formed by trajectory correctness and the sign of the per-token gap ∆t =log pteacher (yt) − log pstudent (yt). The diagnostic studies gap structure under an OPSD-style privileged teacher context; all CAST training runs below use answer-free self-teacher scoring (Appendix 8.2). 3.2 Non-Privileged Self-Teacher Scoring 

Under answer-free self-teacher scoring, CAST computes the self-teacher gap for each token yi,t in trajectory yi as 

gi,t = log πϕ(yi,t | x, y i,<t ) − log πθold (yi,t | x, y i,<t ), (4) 3where x is the prompt, πθold is the detached rollout/reference student policy, and πϕ is a stop-gradient self-teacher. The teacher receives the same problem context and generated prefix as the student. The ground-truth answer and reference solution are used only by the verifier to compute the binary outcome reward, not in the self-teacher scoring context. Reference-solution-conditioned teacher scoring is used only as an ablation in Table 7; implementation details are given in Appendix 8.2.

3.4 Correctness-Aware Token Weighting 

CAST converts the teacher–student gap into a signed token weight: 

zi,t = exp (sign( Bi)gi,t ) . (7) When Bi > 0, a teacher-positive gap ( gi,t > 0) means the teacher assigns higher log-probability to the sampled token than the student, so zi,t > 1 strengthens the positive update. When Bi < 0, the sign is reversed in this sign-preserving weighting stage: a teacher-positive token in a negative-base trajectory receives weaker negative pressure, while a teacher-negative token receives stronger negative pressure. In our binary-reward mixed groups, Bi = 0 does not occur unless the group is uniform, and uniform groups are handled separately by fixed-sign base advantages; for completeness, a zero-base token can be masked or assigned zero advantage. ## 4 Experiments 

4.1 Training Details 

We train Qwen3-1.7B, Qwen3-4B, and Qwen3-8B [28] on DAPO-Math-17K [30] for 300 optimizer steps under a matched rollout and optimization budget. We verify no exact problem-text overlap after normalization between DAPO-Math-17K and the in-domain evaluation benchmarks (AIME24, AIME25, AIME26, MATH-500, and HMMT25). The main comparison spans all three model sizes, while Qwen3-4B is used for mechanistic ablations and diagnostics. All runs use LoRA [9]. The RLVR-style methods use binary verifier rewards, 8 completions per prompt, and 128 on-policy trajectories per update; the OPSD-only baseline instead optimizes the reference-solution-conditioned self-distillation objective. All main runs fit on two NVIDIA A800 80GB GPUs and use only a few hundred optimizer steps. The comparison includes GRPO, OPSD, GRPO+OPSD, RLSD, RLRT [12], and CAST. Here, OPSD denotes the standard on-policy self-distillation objective without a verifier-grounded GRPO term. The main method comparison uses a matched 300-step budget; 600-step CAST checkpoints are reported later only as scaling evidence under longer training, not as the main matched-budget comparison. GRPO+OPSD denotes an additive baseline, 

LGRPO+OPSD = LGRPO + αk LOPSD ,

where αk is the OPSD auxiliary coefficient. This baseline does not use CAST’s correctness-aware group splitting, uniform-reward base-advantage branch, answer-free self-teacher scoring, or bidirectional advantage sign reversal. Full optimizer, decoding, hardware, and method-specific hyperparameters are provided in Appendix 8.4; additional training diagnostics are provided in Appendix 8.5. ## 6 Limitations 

Several limitations remain. • Scale. The experiments evaluate Qwen3-1.7B, Qwen3-4B, and Qwen3-8B with LoRA and up to 600 optimizer steps. Larger models, longer training, full-parameter updates, and broader model families remain untested. • Diagnostic scope. Mechanistic analyses are limited primarily to the 300-step Qwen3-4B setting and an OPSD diagnostic set. Broader answer-free self-teacher-gap diagnostics would further test the mechanism. • Non-thinking evaluation. CAST is trained and evaluated with extended reasoning disabled. Explicit chain-of-thought generation is left to future work. • Efficiency and sensitivity. CAST uses validation-tuned clipping and token-gap hyperparameters, and it often produces longer outputs. This may improve exploration but also increases inference cost; length- or compute-normalized evaluation would be useful. 

## 7 Conclusion 

This work introduced CAST (Non-Privileged Clipped Asymmetric Self-Teaching with Advantage Flipping for GRPO). CAST combines verifier-grounded trajectory correctness with answer-free self-teacher scoring, allowing token-level advantages to depend on both rollout correctness and teacher–student gaps. This design can suppress locally teacher-negative tokens in correct rollouts and provide bounded positive credit to locally teacher-positive tokens in incorrect rollouts, requires no reference reasoning traces, and does not expose the ground-truth answer or reference solution to the self-teacher. Under a matched rollout and optimization budget, CAST achieves the strongest overall Avg@16 in the comparisons and improves Pass@16 over GRPO, OPSD, GRPO+OPSD, RLSD, and RLRT on mathematical reasoning benchmarks. The results suggest that correctness-aware token shaping can make self-distilled RLVR more reliable, especially when zero-variance groups are explicitly used rather than discarded. 10 References

DAPO [30], for example, introduces asymmetric clipping, dynamic sampling, token-level policy-gradient loss, and overlong reward shaping. The experiments adopt its asymmetric clipping rule, 

clip( ρ, 1 − ϵ, 1 + ϵhigh ), ϵhigh > ϵ, (20) 13 which gives positive-advantage tokens a larger upper clipping range. DAPO handles zero-variance groups mainly through dynamic sampling, whereas CAST gives such groups bounded sign-constrained base advantages. 

Zero-variance prompt handling in GRPO. All-correct and all-wrong groups are commonly handled before the policy update rather than inside the advantage estimator. A standard preprocessing strategy is to discard or resample groups whose sampled rewards have zero variance, because their group-normalized GRPO advantages are zero. DAPO’s dynamic sampling follows this philosophy: it seeks to build updates from prompts whose sampled completions contain both correct and incorrect responses, thereby concentrating computation on prompts that provide relative ranking signal [30]. This is effective for gradient utilization, but it also changes the on-policy prompt distribution toward boundary-difficulty examples and leaves no direct learning signal from prompts that are currently too easy or too hard. Recent work argues that zero-variance prompts should not necessarily be discarded. RL-ZVP/No Prompt Left Behind keeps all-correct and all-wrong prompts and constructs entropy-guided advantages so that easy prompts can still consolidate confident behavior and hard prompts can still receive exploratory pressure [13]. ERPO [16] similarly focuses on residual prompts whose sampled outcomes lack informative reward contrast, using an entropy-aware policy objective and adaptive sampling to reactivate these cases. Prompt Replay addresses a related sampling problem by maintaining an online prompt buffer and replaying prompts that are likely to produce informative, non-saturated rollouts [2]. These methods operate mainly at the prompt-selection or auxiliary-shaping level. CAST is complementary: it keeps zero-variance groups in the same GRPO-style update, assigns only bounded verifier-signed base advantages, and uses the answer-free self-teacher gap to decide which tokens within those all-correct or all-wrong trajectories should be amplified or suppressed. 

OPD and OPSD. On-Policy Distillation (OPD) uses an external teacher to provide token-level supervision on student-sampled trajectories, reducing off-policy mismatch between teacher-generated data and student-visited states. On-Policy Self-Distillation (OPSD) removes the external teacher by using a stop-gradient self-teacher, often with privileged contexts such as the ground-truth answer or verified solution traces [32]. Although OPSD supplies dense token information, its direction is teacher-driven rather than verifier-grounded. Prior work suggests that OPSD may be more reliable for compacting successful reasoning than repairing failed trajectories [11], motivating correctness-aware use of self-teacher signals. 

Reasoning self-improvement and self-training. CAST is also related to methods that improve reasoning from model-generated traces. Chain-of-thought prompting and self-consistency show that intermediate reasoning paths and multiple sampled solutions can substantially improve mathematical and symbolic reasoning [26, 24]. STaR bootstraps reasoning by fine-tuning on self-generated rationales that lead to correct answers [31], while ReST generates policy samples and reuses filtered data for reinforced self-training [6]. CAST differs from these offline or prompt-time self-improvement approaches by shaping on-policy RLVR updates directly, using verifier correctness to decide the trajectory-level direction and an answer-free self-teacher gap to refine token-level credit assignment. 

Self-distilled RLVR. Self-distilled RLVR methods such as RLSD, SDPO-style training, and related hidden-distillation variants seek to combine two complementary signals: the verifier reward decides whether a rollout should be encouraged or discouraged, while the teacher-student gap provides dense token-level structure inside the rollout [29, 10, 5]. This design is attractive because it keeps the update anchored to outcome correctness while avoiding the extreme sparsity of sequence-level rewards. It also explains why RLSD-type baselines are stronger than OPSD-only training in these experiments: they do not merely imitate the teacher, but use the verifier to set the rollout-level direction. Most RLSD-style objectives, however, use the teacher–student gap primarily as a magnitude modulator rather than as a source of signed token-level feedback. A typical formulation maps the gap to a bounded weight, 

wi,t = clip (exp(sign( Ai)gi,t ), 1 − δ, 1 + δ) , (21) and sets ˜Ai,t = Ai ((1 − λ) + λw i,t ) . (22) This preserves the GRPO update sign through Ai while using distillation for dense token shaping. Consequently, every token in a positive-advantage trajectory remains positive and every token in a negative-advantage trajectory remains negative; the teacher gap changes the strength, but not the local direction, of the update. This is a conservative and stable choice, but it is limited when a correct trajectory contains locally undesirable tokens or when an incorrect trajectory contains locally useful partial reasoning. Another limitation is data utilization. Since the base sign is inherited from the group-relative advantage, all-correct and all-wrong groups become weak or receive zero group-relative advantage when Ai = 0 . These groups are common in RLVR training: easy prompts may become all-correct, while hard prompts may remain all-wrong. Discarding or 14 ignoring them wastes verifier-labeled trajectories that could still teach consolidation or failure avoidance. Finally, some self-distillation setups score the teacher with the ground-truth answer or a reference solution in the context, creating a privileged token-scoring signal that is stronger than what the policy would have at inference time. CAST keeps the useful verifier anchoring of RLSD-style training but changes all three design choices. First, it interprets teacher gaps through trajectory correctness and allows two-sided local sign changes: teacher-negative tokens inside correct trajectories can receive negative token-level advantages, and teacher-positive tokens inside incorrect trajectories can receive bounded positive token-level advantages. Second, it gives all-correct and all-wrong groups bounded sign-constrained base advantages, so zero-variance data remains active. Third, it uses answer-free self-teacher scoring and reserves ground-truth answers and reference solutions for verification only. 

RLRT and advantage sign reversal. RLRT provides a different interpretation of the teacher–student gap [12]. Instead of treating teacher-negative tokens as uniformly undesirable, it reverses the teacher signal on verified-correct trajectories so that tokens where the student departs from the teacher can be strengthened. This is motivated by the observation that a smaller or lagged teacher may suppress alternative valid reasoning paths that the student has discovered. RLRT is therefore closely related to the advantage sign-reversal motivation in CAST: both methods reject pure teacher imitation on successful rollouts. CAST differs in three ways. It uses answer-free self-teacher scoring, applies the gap asymmetrically according to both trajectory correctness and gap sign with bidirectional local sign reversal, and explicitly uses uniform-reward all-correct and all-wrong groups through bounded base advantages. 

8.2 Implementation Details for CAST Advantages 

CAST constructs token-level advantages in two separated stages. First, after rollouts and verification, teacher and student log-probabilities are computed on the sampled tokens to form a detached gap 

¯gi,t = sg [log πϕ(yi,t | x, y i,<t ) − log πθold (yi,t | x, y i,<t )] , (23) where θold denotes the student parameters used for rollout/log-probability collection before the policy-gradient update. This detached ¯gi,t , together with the detached verifier reward, is used to build ˜Ai,t . Second, the online policy is updated only through the importance ratio ρi,t in the DAPO/GRPO objective. Thus, gradients are not propagated through the teacher logits, the old-policy student logits used in the gap, the exponentiation, or the clipping operations that define the shaped advantages. For mixed groups, Bi is the usual group-normalized GRPO advantage. For A LL CORRECT and A LL WRONG groups, Bi

is set to +bcorrect and −bwrong , respectively. These fixed base-advantage values determine only the sign and bounded scale of the token-level update; they do not create a separate reward model and do not expose the answer to the self-teacher. Truncated completions are masked out before advantage construction, so uniform-reward base-advantage updates are applied only to valid generated tokens.

8.4 Training Details 

Unless otherwise specified, experiments use DAPO-Math-17K for 300 optimizer steps on Qwen3-1.7B, Qwen3-4B, and Qwen3-8B. DAPO-Math-17K is used only for training; we verify no exact problem-text overlap after normalization with the evaluation benchmarks (AIME24, AIME25, AIME26, MATH-500, and HMMT25), so reported results are not affected by exact-match train–test contamination. The main method comparison applies the same protocol and matched baseline set across all three model sizes, while mechanistic ablations and diagnostics use Qwen3-4B. The training-step ablation additionally evaluates CAST checkpoints at 100 and 600 steps for each model size. LoRA with rank 64 and alpha 128 is used while keeping the backbone frozen. LoRA adapters are applied to q_proj , k_proj ,

v_proj , o_proj , gate_proj , up_proj , and down_proj . Each run uses two NVIDIA A800 80GB GPUs: one for vLLM rollouts and one for policy optimization. For each prompt, 8 on-policy completions are sampled. With prompt batch size 2 and gradient accumulation 8, each optimizer update contains 16 prompt groups and 128 trajectories. This resource-constrained configuration uses a rollout micro-batch of 2 prompts, and the default run stops after 300 optimizer steps rather than relying on long-horizon or large-cluster post-training. Training uses bfloat16 precision, scaled dot-product attention (SDPA), gradient checkpointing, maximum prompt length 1024, and maximum completion length 3072. Unless otherwise specified, generation uses temperature 0.7, top-p 0.95, top-k 20, min-p 0.0, repetition penalty 1.0, and presence penalty 0.2. Completions that reach the maximum completion length without termination are masked from the policy loss, so uniform-reward base-advantage updates apply only to valid, non-truncated completion tokens. The optimizer is AdamW with zero weight decay. The learning rate is 1 × 10 −6

with 5% warmup and polynomial decay to 1 × 10 −7, and the maximum gradient norm is 1.0. Rewards are binary verifier correctness rewards; no additional format or repetition rewards are used. Following DAPO, all GRPO-style methods use asymmetric clipping with ϵ = 0 .2 and ϵhigh = 0 .28 , and set the Kullback-Leibler (KL) coefficient to 0. For self-distillation methods, the stop-gradient teacher is refreshed every 10 optimizer steps unless a baseline uses a fixed teacher. OPSD uses the reference-solution-conditioned OPSD objective with coefficient 1.0 and no verifier-grounded GRPO term. GRPO+OPSD adds the standard OPSD token auxiliary to the GRPO objective, using an auxiliary coefficient initialized to 0.2 and linearly decayed over 50 steps; it does not use CAST’s correctness-aware group handling or advantage sign reversal. RLSD uses verifier rewards to set rollout-level directions and teacher–student gaps to modulate token-level magnitudes, with token-level weights clipped to [0 .8, 1.2] . RLRT follows the reversed-teacher rule on verified-correct trajectories [12]: it forms di,t = log πθold (yi,t | x, y i,<t ) − log πϕ(yi,t | ci, y i,<t ), clips 

wi,t = exp(sign( Ai)di,t ) to [1 − ϵw, 1 + ϵw] with ϵw = 1 .0, and sets ˜Ai,t = Ai[(1 − λRLRT ) + λRLRT wi,t ] with 

λRLRT = 0 .5 and no decay. This reweighting is applied only to verified-correct rollouts; incorrect rollouts retain the standard GRPO sequence advantage. The RLRT teacher context uses the problem plus a successful rollout sampled 16 for the same prompt group when available. CAST uses a token-gap coefficient of 1.0 with no decay and answer-free self-teacher scoring: 

TAF (x) = {x} ∥ "\n\n[Student response]\n" ,

with solution set to y∗ from reward_model.ground_truth on DAPO-Math-17K. In both settings, the verifier alone uses the ground truth to compute binary correctness rewards. GRPO uses only verifier rewards and group-relative advantages. OPSD uses token-level self-distillation on on-policy rollouts without verifier rewards. GRPO+OPSD combines the GRPO policy-gradient loss with standard OPSD token distillation, but its OPSD signal remains correctness-unaware and does not reactivate zero-variance groups. RLSD combines GRPO with OPSD-style token weighting by using verifier rewards to determine rollout-level update directions and teacher–student gaps to modulate token-level magnitudes. RLRT further emphasizes exploration by treating teacher-negative tokens in correct rollouts as potentially useful departures from the teacher. CAST uses the same GRPO training loop, but interprets teacher–student gaps according to trajectory correctness. In the main CAST configuration, token shaping is applied to mixed, all-correct, and all-wrong groups; zero-variance groups receive bounded sign-constrained base advantages instead of being dropped. CAST uses asymmetric token-weight clipping with (ℓ+, u +) = (0 .8, 1.05) 

for positive-base trajectories and (ℓ−, u −) = (0 .95 , 1.2) for negative-base trajectories, applies negative sign reversal to teacher-negative positive-base tokens and bounded positive sign reversal to teacher-positive negative-base tokens under the smaller positive-base cap, and clips final token-level advantages to [−1.2, 1.2] .In terms of training cost, CAST requires about 1.25 × the wall-clock time of GRPO for a full 300-step run. A full CAST run takes about 5 hours, compared with about 4 hours for GRPO, 4 hours and 40 minutes for RLSD, and 4 hours and 30 minutes for OPSD. The extra cost is modest and mainly comes from computing self-teacher logits for token-gap shaping and applying correctness-aware token-level advantages. See Table 8.

8.8 Additional Explorations 

Table 17 summarizes the exploratory path from RLSD to CAST. All results are evaluated with 16 samples under the non-thinking 32K setting. The row labeled RLSD (zero-variance updates) denotes an exploratory variant that also updates zero-variance all-correct and all-wrong groups. Starting from RLSD, the variants progressively add zero-variance updates, strict group handling, sign flipping, removal of the ground-truth teacher reference, non-decayed token-gap shaping, and finally the incorrect-path positive flip used by CAST. 

8.8.1 RLSD 

The RLSD baseline applies token-level shaping on top of the sequence-level GRPO advantage. It improves over sparse sequence-level feedback by using the teacher–student token log-probability gap to modulate each token’s advantage. However, this formulation still depends on the original group-relative advantage, which becomes weak when all sampled responses for a prompt are correct or all are incorrect. 

8.8.2 RLSD with Zero-Variance Updates 

The RLSD (zero-variance updates) variant additionally applies RLSD-style updates to all-correct and all-wrong groups. Because this changes the handling of zero-variance groups while retaining the RLSD shaping rule, it is treated as an exploratory variant rather than as the main RLSD baseline. 

8.8.3 Strict RLSD 

Strict RLSD addresses this issue by splitting groups into mixed, all-correct, and all-wrong cases. Mixed groups keep the original GRPO advantage, while all-correct and all-wrong groups receive explicit fixed base advantages. This gives nonzero training signal even when group-relative normalization would otherwise collapse. 

8.8.4 Flip RLSD 

This variant introduces local advantage sign reversal. A token from a positive-advantage rollout can still receive negative credit if the teacher–student gap is negative. This avoids uniformly reinforcing every token in a globally successful response and makes the token-level feedback more selective. 

8.8.5 Flip RLSD without Ground-Truth Teacher Reference 

This variant switches to answer-free self-teacher scoring by removing the reference solution from the teacher prompt, so the token-level signal depends only on the problem and the student response. This variant tests whether RLSD can provide useful token-level supervision without access to the ground-truth solution or reference solution. 

8.8.6 Flip RLSD without Ground-Truth Reference and without Decay 

This variant keeps the token-gap shaping active throughout training instead of annealing it away. It treats the teacher– student token gap as a persistent training signal rather than only as an early-stage regularizer. 

8.8.7 CAST with Incorrect-Path Positive Flipping 

CAST adds the final bidirectional sign-reversal rule: teacher-negative positive-base tokens receive negative reversal, and teacher-positive negative-base tokens receive bounded positive reversal under the positive-base clipping cap. This is the configuration used in the main results.
