## Abstract
Agentic reinforcement learning can induce tool abuse, where models overuse external tools even for queries solvable by internal reasoning. Existing approaches mitigate this issue with uniform tool-use penalties or hard limits, which reduce tool frequency but may also suppress useful tool-assisted exploration. We propose EAPO, an Efficient Agentic Policy Optimization framework that learns selective tool use. EAPO introduces tool-free trajectories into each rollout group, applies difficulty-aware reward shaping to penalize redundant tool calls mainly on easier queries, and uses confidence-aware token reweighting to improve policy learning. Across nine mathematical and knowledge-intensive reasoning benchmarks, EAPO consistently improves the accuracy efficiency trade-off on Qwen2.5-3B, Qwen2.5-7B, and Llama3.1-8B. Compared with GRPO, EAPO improves average performance by 10.45%, 7.27%, and 9.69%, while reducing average tool calls by 18.33%, 18.33%, and 24.59%, respectively. These results show that agents can learn when not to use tools without compromising tool-integrated reasoning.

## 1. 核心问题与整体含义（研究动机和背景）

### 研究问题
- **工具滥用（Tool Abuse）**：在智能体强化学习（Agentic RL）中，模型即使对于可以通过内部推理解决的简单查询，也会过度依赖外部工具（如搜索引擎、Python解释器）
- **问题表现**：模型会对"1+1=?"等trivial问题调用Python工具，对"太阳从东方升起"这类常识问题进行网络搜索

### 现有方法的局限
- **统一工具惩罚**：不加区分地惩罚工具调用，可能抑制有用的工具辅助探索
- **硬限制**：直接限制工具调用次数，无法区分真正需要工具的复杂查询与简单查询
- **核心矛盾**：目标是让模型学会"何时"使用工具，而非简单地"减少"工具使用

### 研究动机
- 不必要的工具调用增加计算成本、延迟和部署开销
- 需要一种机制来区分必要工具使用与工具滥用

## 2. 方法论：EAPO（Efficient Agentic Policy Optimization）

### 核心思想
EAPO是一个多工具智能体强化学习框架，通过三大核心组件优化效率而不损害工具辅助推理能力。

### 关键技术细节

#### 2.1 效率感知 rollout（Efficiency-Aware Rollout）
- **机制**：在每个rollout的M条轨迹中，确保至少有n条轨迹完全不调用工具
- **实现**：通过修改提示词禁止工具使用，或在模型尝试调用工具时禁用执行
- **效率系数定义**：
$$\varepsilon(T_i) = \exp(-\beta(c_i - c_{min}))$$
  其中$c_i$是轨迹中的工具调用数，$c_{min}$是正确轨迹中的最少工具调用数

#### 2.2 难度感知奖励塑形（Difficulty-Aware Reward Shaping）
- **查询难度定义**：
$$d(q) = 1 - \frac{1}{M}\sum_{i=1}^{M} I(T_i)$$
  其中$I(T_i)=1$表示轨迹正确
- **调整后的奖励**：
$$\tilde{R}_i = [d(q) + (1-d(q))\varepsilon(T_i)] R_i$$
- **核心思想**：当模型已掌握某查询时惩罚工具使用；对于困难查询保留工具辅助探索

#### 2.3 置信度感知优势重加权（Confidence-Aware Advantage Reweighting）
- **置信度定义**：$C_t = \log \pi_\theta(y_t | x, o_{<t})$
- **token级权重**：
$$\omega_{i,u,t} = \begin{cases} W_{pos}, & \text{若正确且低置信度} \\ W_{neg}, & \text{若错误且高置信度} \\ 1, & \text{其他} \end{cases}$$
- **重加权优势**：$A_{i,u,t} = \omega_{i,u,t} A_i$

### 算法流程
1. 对每个查询生成n条无工具轨迹 + (M-n)条有工具轨迹
2. 计算每条轨迹的工具调用数、难度系数、调整后奖励
3. 计算轨迹级优势
4. 对token进行置信度感知重加权
5. 使用重加权策略梯度目标更新策略，同时保持与参考策略的KL正则化

### 关键发现
