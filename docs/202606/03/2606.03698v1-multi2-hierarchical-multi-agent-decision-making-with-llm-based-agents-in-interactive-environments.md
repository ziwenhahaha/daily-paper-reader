## 摘要
大语言模型（LLM）研究的核心目标是构建能够通过与动态环境的持续交互进行规划、行动和自适应的智能体系统。近期基于LLM的智能体在上下文推理方面表现出色，但其长期决策仍然脆弱，常常出现目标漂移——在长时间交互中目标和计划逐渐偏离。我们提出了Multi$^2$，一种层次化多智能体决策框架，将智能体行为显式分解为互补的角色。高层智能体（系统1）专注于使用监督微调（SFT）进行上下文感知的子目标生成，而低层智能体（系统2）通过离线到在线强化学习（RL）在交互环境中执行原子动作。这种分离实现了稳定的长时控制，减轻目标漂移，并支持高效适应。在多样的交互环境中，Multi$^2$持续优于强基线智能体系统，展示了在多轮交互中改进的鲁棒性和协同能力。除了性能之外，我们还发布并开源了三个层次化基准数据集，填补了LLM基智能体层次化决策训练与评估中长期存在的空白。

## Abstract
A central goal of large language model (LLM) research is to build agentic systems that can plan, act, and adapt through sustained interaction with dynamic environments. While recent LLM-based agents exhibit impressive contextual reasoning, their long-horizon decision-making remains fragile, often suffering from objective drift, where goals and plans drift over extended interactions. We introduce Multi$^2$, a hierarchical multi-agent decision-making framework that explicitly decomposes agent behavior into complementary roles. A high-level agent (System 1) focuses on context-aware sub-goal generation using supervised fine-tuning (SFT), while a low-level agent (System 2) executes atomic actions through offline-to-online reinforcement learning (RL) in interactive environments. This separation enables stable long-horizon control, mitigates objective drift, and allows efficient adaptation. Across diverse interactive environments, Multi$^2$ consistently outperforms strong agentic baselines, demonstrating improved robustness and coordination in multi-turn interaction. Beyond performance, we introduce and release three hierarchical benchmark datasets, filling a long-standing gap in training and evaluating hierarchical decision-making for LLM-based agents.

### 研究动机

大语言模型（LLM）智能体在动态环境中进行多轮、长时 horizon 交互时面临严峻挑战：

- **目标漂移（Objective Drift）**：在长时间交互过程中，智能体的目标和计划逐渐偏离原始意图，这是由于小错误累积导致的状态与初始意图逐步错位。
- **Token 效率低下**：现有方法依赖不断增长的交互历史来维持意图，导致推理时 token 消耗过高。
- **执行脆弱性**：分解任务（hierarchical planning）虽然降低了规划失败，但执行端在长时 rollouts 中仍会因错误累积和约束密集的动作而崩溃。

### 核心问题

> 如何构建能在长时 horizon 任务中可靠且高效运行的交互式 LLM 智能体？

现有方法虽然引入了层次化结构，但执行器（executor）并未被显式训练来纠正交互过程中的错误累积，导致长期交互仍然脆弱。

## 2. 论文提出的方法论

### 核心思想

**Multi²** 是一种层次化多智能体决策框架，将智能体行为显式分解为两个互补的角色：

| 角色 | 名称 | 功能 | 训练方式 |
|------|------|------|----------|
| 高层智能体 | System 1 | 上下文感知的子目标生成，保持全局意图 | SFT（监督微调） |
| 低层智能体 | System 2 | 在子目标条件下执行原子动作 | 离线到在线 RL |

### 关键技术细节

#### 2.1 层次化数据集构造

- **D_sys1**（System 1 数据集）：`{K_n, {(o_h, g_h)}}`，包含任务描述和观察-子目标序列对。
- **D_sys2**（System 2 数据集）：`{g(i), {ξ(i)_t}}`，包含子目标和对应的低层转移序列（观察、动作、奖励、下一步观察）。

#### 2.2 System 1 训练（基于 SFT）

使用标准交叉熵目标进行行为克隆：

$$L_\phi = -\mathbb{E}_{(K_n, o_t, g^h) \sim D_{sys1}} [\log \pi_\phi(g^h | o_t; K_n)]$$

该目标促进任务级别的一致性，确保长期规划中的语义连贯性。

#### 2.3 System 2 离线 RL 训练

**Q 函数训练（TD 误差）：**

$$L_\omega = \mathbb{E}_{(g^h, o_t, a_t, r_t, o_{t+1}) \sim D_{sys2}} [(r_t + \gamma V_\psi(o_{t+1}; g^h) - Q_\omega(o_t, a_t; g^h))^2]$$

**价值函数训练（Expectile 回归）：**

$$L_\psi = \mathbb{E}_{(g^h, o_t, a_t, r_t, o_{t+1}) \sim D_{sys2}} [L_\tau(A(o_t, a_t))]$$

其中 $L_\tau(u) = \tau - \mathbf{1}\{u < 0\} u^2$。

**Actor 离线更新（策略锚定优势）：**

$$L_\theta^{offline} = -\mathbb{E}_{(g^h, o_t, a_t) \sim D_{sys2}} [\lambda A(o_t, \pi^{off}_\theta(a_t | o_t; g^h); g^h) + \exp(\beta A(o_t, a_t; g^h)) \times \log \pi^{off}_\theta(a_t | o_t; g^h)]$$

关键创新：**策略锚定优势项** 鼓励向更高回报的动作学习，减少过度模仿，促进泛化。

#### 2.4 System 2 在线 RL 训练（自改进）

$$L_\theta^{online} = -\mathbb{E}_{(g^h, o_t, a_t) \sim B_{sys2}} [w_t \log \pi^{on}_\theta(a_t | o_t; g^h)] + \eta \mathbb{E}_{(g^h, o_t) \sim B_{sys2}} [D_{KL}(\pi^{on}_\theta \parallel \pi^{off}_\theta)(o_t; g^h)]$$

- **优势加权**：`w_t = exp(\frac{1}{\alpha}A(o_t, a_t; g^h))` 强化高回报动作。
- **KL 正则化**：约束在线策略接近离线策略，防止过度偏离，提升训练稳定性。

#### 2.5 参数化设计

- **LoRA（低秩适配）**：角色特定的适配器，共享预训练 backbone 但参数解耦，实现高效的角色特化。
- **Actor-Critic 结构**：Q 网络估计动作价值，V 网络估计状态价值，支持优势计算。

## 6. 论文的主要结论与发现

### 6.1 性能提升

- **ScienceWorld**：Multi² 在 ID 和 OOD 分割上均显著优于所有基线，例如 Llama-3.1 8B 上 ID 达 67.61%（Glider 60.48%），OOD 达 30.68%（Glider 34.36%）。
- **ALFWorld**：持续领先，Llama-3.1 8B 上 ID 57.86%，OOD 56.43%。
- **TextCraft**：显著领先，Llama-3.1 8B 上成功率达 35.60%（Glider 9.50%）。

### 6.2 关键发现

1. **层次化 + 角色特化是关键**：消融实验显示 Shared Adapter 导致性能大幅下降（ScienceWorld ID 45.57% vs Proposed 60.68%）。

2. **SFT for System 1 + RL for System 2 的最优分配**：仅 RL 配置性能较低且方差大；仅 SFT 配置中位性能竞争但鲁棒性不足。

3. **在线适应带来一致改进**：尤其在 OOD 场景和简单任务上增益更大（Easy 任务 OOD 提升 14.15%）。

4. **Token 效率显著提升**：Multi² 达到最佳性能/Token 比，ScienceWorld OOD 上归一化效率 13.77（ReAct 为 1.0）。

5. **子目标错误非主要失败源**：Planner 相关失败仅占 ID 的 3.70%、OOD 的 5.88%，主要失败来自执行端。

6. **模型规模在 OOD 上更关键**：从 1.5B 到 7B，ALFWorld OOD 从 48.57% 提升到 64.29%（ID 仅从 53.57% 到 62.10%）。

## 7. 优点

### 7.1 方法创新

- **显式角色分解**：将规划（planning）和执行（execution）作为不同的优化问题，而非仅通过 prompt 进行角色分离。
- **离线到在线 RL 管道**：结合离线 RL 的稳定初始化和在线 RL 的持续自改进，设计了 KL 正则化防止策略崩溃。
- **策略锚定优势机制**：缓解过度模仿，鼓励探索超越离线轨迹的动作。

### 7.2 工程贡献

- **角色特定 LoRA 适配器**：实现参数解耦而不增加 backbone 负担，支持高效角色特化。
- **代码化数据集构造**：使用确定性规则进行子目标提取和 prompt 模板标准化，提高可复现性。
- **公开资源**：发布处理好的层次化数据集、prompt 模板、提取规则和训练评估代码。

### 7.3 实验设计亮点

- **严格 Pass@1 评估**：强调可靠性和单次效率。
- **任务难度分层**：揭示不同难度下方法表现的差异。
- **多维度消融**：从训练配置、结构、参数化、损失函数多角度验证组件贡献。

## 8. 不足与局限

### 8.1 实验覆盖局限

- **仅文本交互环境**：未涉及视觉输入或物理机器人操控场景，限制了方法的通用性。
- **固定子目标完成检测**：需要环境提供明确的完成信号或手工规则，可能不适用于更开放的任务。

### 8.2 潜在偏差风险

- **数据依赖**：离线 RL 阶段依赖预收集数据集的质量；若数据集覆盖不足，策略可能受限。
- **ID/OOD 分割定义**：分割基于任务模板，可能存在边界模糊（尤其 ScienceWorld 官方分割与论文自定义分割的差异）。
- **Reflexion 对比不公平**：Pass@6 vs Pass@1 的对比可能高估 Reflexion 的单次成功率。

### 8.3 应用限制

- **计算资源要求**：离线+在线 RL 训练 pipeline 需要显著算力，可能限制资源有限场景的应用。
- **LoRA 参数敏感性**：性能对 LoRA rank、α 等超参数可能敏感，需针对不同 backbone 调优。
- **子目标依赖**：System 1 生成的子目标质量影响整体性能，若子目标规划失败（虽占比小），可能无法恢复。

### 8.4 未探索方向

- **多智能体协作**：框架仅关注单个智能体的层次化，未涉及多智能体协调。
- **终身学习**：在线适应限于单次训练阶段，未探索持续学习的长期效果。
- **安全关键场景**：缺乏对约束违反、安全边界的显式建模和分析。
