## Abstract
Continual reinforcement learning aims to produce agents that learn not only to improve at their current tasks but also to adapt as task distributions change. Training an agent on many diverse tasks can induce zero-shot generalization, but previous work generally evaluates this generalization after training -- with frozen weights. Whether task diversity also improves an agent's ability to continue learning across distribution shifts remains unclear. We introduce Banyan, a GPU-accelerated continual RL domain in which task diversity factors into three independently controllable axes: the map layouts an agent must navigate, the objects it must interact with, and the hierarchical structures of sub-goal dependencies. Across individual distribution shifts, increasing diversity along each axis causes agents to begin training on the new tasks near the performance attained on the previous one, even when the shift changes the structure of the optimal policy. However, as the number of shifts increases, this local transfer does not by itself yield sustained continual learning: longer-horizon tasks plateau, and earlier task distributions are forgotten after later training. Banyan is a benchmark for studying when controlled task diversity produces transferable learning, when that transfer persists, and where it falls short of proper continual learning.

## 1. 核心问题与整体含义

持续强化学习（Continual RL）旨在使智能体在任务分布变化时能够持续学习与适应。先前研究表明，在大规模多样化任务上训练可以诱导零样本泛化能力，但这些工作通常在训练结束后（权重冻结状态下）评估泛化能力。**核心问题在于**：任务多样性是否也能提升智能体在权重持续更新过程中跨分布迁移的学习能力？

此前缺乏可控制任务多样性的环境来研究这一问题。现有持续 RL 基准测试通常最多包含数百个任务，而大规模程序化生成环境虽有百万/十亿量级任务，但未设计用于独立控制不同类型的多样性。

## 2. 方法论：核心思想与关键技术

### 2.1 核心思想
引入 **Banyan**：一个 GPU 加速的持续 RL 领域，将任务多样性解耦为三个独立可控轴：
- **布局（Layout, L）**：地图布局决定导航问题
- **任务树实例（Object Assignment, O）**：具体对象身份，决定合成配方
- **任务树拓扑（Topology, T）**：抽象结构（深度、分支、转换/组合节点），决定子目标依赖关系

### 2.2 任务定义
任务 τ = ⟨L, T, O⟩：
- 叶子节点为环境中放置的对象
- 内部节点为通过转换或组合子节点产生的新对象
- 根节点为目标对象

### 2.3 关键公式

**前向迁移差距（Transfer Gap）：**
$$Δ_i = S_{end}(d_{i-1}) - S_{start}(d_i)$$
衡量智能体从分布 d_{i-1} 进入 d_i 时的性能落差，越接近 0 表示迁移越好。

**后向迁移（Backward Transfer）：**
$$B(t, i) = S_{end}(d_i; t) - S_{end}(d_i; i)$$
正值表示正向后向迁移，负值表示遗忘。

### 2.4 实验设置
- 使用 PPO（策略梯度）和 PQN（并行化 Q 网络）作为基线算法
- 采用 Continual Backprop（CBP）对抗 loss-of-plasticity 问题

## 6. 主要结论与发现

### 6.1 单次迁移：多样性促进系统性迁移
- **前向迁移**：沿每个轴增加多样性使 Δ₂ → 0，即智能体进入新分布时保持在上一分布的绩效水平
- **后向迁移**：仅拓扑多样性 T 显著改善后向迁移；布局和对象多样性对后向迁移无明显影响
- **机制解释**：高多样性产生更优特征，导致更小的 TD-error，从而提升预测质量

### 6.2 多次迁移：多样性抑制持续学习
- **转移差距依然关闭**：在每个边界，Δᵢ → 0，高多样性下每个新分布开始于前一分布的绩效水平
- **但长期学习停滞**：最高多样性（n=256）下 Send(dᵢ) 在 ~0.80 附近停滞；中等多样性（n=16）下从 d₁ 的 ~0.74 提升至 d₁₀ 的 ~0.95
- **矛盾现象**：多样性关闭了每个边界，却导致后续分布学习停滞
- **后向迁移反向**：高多样性下 B(10,1) 增强——智能体在 d₁ 上持续改进，但无法在新分布上专精

### 6.3 关键悖论
> 增加任务多样性既驱动了每次边界处的系统性迁移，又在长期上抑制了智能体在新分布上专精化学习的能力

## 7. 优点

1. **问题定义清晰**：明确提出先前工作未解决的科学问题——多样性是否也提升持续学习期间的迁移能力
2. **环境设计优秀**：Banyan 将任务多样性解耦为三个独立轴，使控制实验成为可能
3. **可扩展性强**：GPU 加速程序化生成支持数十亿任务，适合 many-task 场景
4. **实验严谨**：多算法验证、多随机种子、跨配置对比，结果可靠
5. **对照验证**：连续控制实验排除网格世界特异性，增强结论泛化性
6. **理论贡献**：发现多样性对前向迁移和长期专精化的双面效应，为后续研究奠定基础

## 8. 不足与局限

1. **生态有效性**：Banyan 为网格世界环境，其效果在真实视觉/物理场景中的迁移性未知
2. **架构限制**：全程使用循环网络架构，未探索 Transformer 等自注意力架构是否同样受多样性抑制
3. **算法选择局限**：仅使用最小化基线（PPO/PQN），未测试 replay、模块化策略、基于模型的规划等已有辅助机制是否能恢复多样性的复合收益
4. **视觉多样性缺失**：连续控制实验未变化视觉多样性，仅变化任务结构
5. **潜在偏差**：10K 布局/对象规模实验覆盖可能不够，拓扑实验规模相对较小（最大 256）
6. **内在权衡假设**：论文推测多样性抑制可能是权重共享函数近似的固有特性，但未直接验证该假设
