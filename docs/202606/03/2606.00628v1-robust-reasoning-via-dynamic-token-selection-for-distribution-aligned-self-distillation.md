## Abstract
Self-distillation improves learning efficiency by rewriting reference answers as training data that better matches the model's own distribution. However, reference answers also introduce strong stylistic biases, causing the generative model to imitate surface forms rather than learn useful reasoning patterns. We observe that the rewriting data contains a large number of high-perplexity (PPL) tokens, coming from two distinct sources: beneficial knowledge-enhancing logical corrections, and harmful stylistic drift induced by reference imitation. Treating all such tokens equally can disrupt the base model's original distribution and degrade performance, especially on difficult reasoning tasks. To address this, we propose Distribution-Aligned Self-Distillation (DASD), which uses an answer-aware reference model to generate candidate tokens and dynamically filters them according to the base model's confidence. DASD preserves tokens that encode useful logical knowledge while suppressing distributionally misaligned style noise. Experiments on math, code, and commonsense reasoning benchmarks show that DASD consistently outperforms competitive baselines, reduces high-PPL tokens, and improves robustness across tasks of varying difficulty.

- **核心问题**：大语言模型（LLM）在下游任务微调时，自蒸馏（Self-Distillation）方法会产生分布偏移（Distribution Shift），导致模型性能下降，尤其在困难推理任务上表现明显。

- **研究背景**：
  - **灾难性遗忘**：下游数据与预训练分布差异大时，直接微调会扭曲模型原有参数空间。
  - **自蒸馏机制**：将下游样本重写为更匹配模型自身生成分布的训练数据，减少分布偏移。
  - **潜在缺陷**：参考答案（Reference Answer）引入了强烈的风格偏差，导致生成内容模仿参考的表面形式而非学习有用的推理模式。

- **关键观察**：
  - 自蒸馏数据中存在大量高困惑度（High-PPL, PPL > 2.5）Token，来源于两类：
    - **有益的逻辑修正**：对基模型来说不太可能但对解决问题必要的推理步骤。
    - **有害的风格漂移**：从参考答案继承的冗余表面形式、推理模板等。

## 2. 论文提出的方法论

### 核心思想

提出 **Distribution-Aligned Self-Distillation (DASD)**，一种基于置信度的动态Token选择方法，在保证答案正确性的同时，将生成内容限制在基模型的原生输出分布内。

### 关键技术细节

#### 符号定义
- $M_\theta$：基模型（待优化）
- $M_{ref}$：参考模型（基于基模型并以标准答案为条件）
- $D = \{(x_i, y^*_i)\}$：训练数据集，其中 $x_i$ 为输入，$y^*_i$ 为标准答案
- $\tau$：置信度阈值
- $K$：候选Token数量

#### 候选Token生成（Reference-Guided Candidate Generation）
在每个生成步 $t$，参考模型基于问题、标准答案和已生成前缀计算条件概率分布：
$$P_{ref}(\cdot | x, y_{<t}, y^*)$$
选取Top-K个最高概率Token组成候选集：
$$C_t = \{c_1, c_2, ..., c_K\}$$

#### 动态选择策略（Dynamic Selection Strategy）
对候选集中每个Token $c_i$，计算基模型的置信度：
$$p = P_\theta(c_i | x, y_{<t})$$

采用两规则选择机制：
1. **分布对齐优先**：遍历候选集，选择第一个满足 $p > \tau$ 的Token，确保输出符合基模型原始分布。
2. **知识备份机制**：若所有K个候选均未通过置信度检查，则直接选择 $c_1$（参考模型贪婪解码结果），以保持推理逻辑正确性。

#### 训练目标
使用标准因果语言建模损失在分布对齐数据集 $D_{aligned}$ 上优化基模型：
$$L(\theta) = -\sum_{(x,y) \in D_{aligned}} \sum_{t=1}^{|y|} \log P_\theta(y_t | y_{<t}, x; \theta)$$

### 算法流程
1. **阶段一：分布对齐数据生成**
   - 对每个样本，迭代生成Token直到EOS
   - 每步从参考模型获取Top-K候选
   - 用基模型置信度动态筛选
2. **阶段二：模型训练**
   - 过滤生成样本，仅保留答案正确的样本
   - 在 $D_{aligned}$ 上进行标准微调

## 7. 优点

1. **问题定位精准**：首次系统揭示自蒸馏中高PPL Token的双重来源（有益逻辑修正 vs 有害风格漂移）

3. **双模型协同机制**：参考模型保证逻辑正确性，基模型置信度保证分布一致性，两者互补
4. **硬Token回退机制**：确保在所有候选Token均不可信时，仍能获得正确推理
5. **通用性强**：在三种不同推理任务和多个模型架构上均表现优异
6. **Token级可解释性**：提供了细粒度的Token分类分析，理解方法行为
