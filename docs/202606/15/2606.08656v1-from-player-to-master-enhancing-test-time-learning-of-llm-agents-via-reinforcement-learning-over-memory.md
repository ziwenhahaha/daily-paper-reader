## 摘要
大型语言模型（LLM）智能体正越来越多地被部署在长期运行的环境中，此时通过测试时的经验进行改进变得尤为重要。常见的做法是在每次交互后更新显式记忆，以指导未来的决策。然而，现有大多数方法依赖于手工设计的提示规则，导致难以在多步 horizon 上始终将记忆更新与下游目标对齐。我们提出 MemoPilot，一种插件式记忆副驾驶，它显式地训练记忆更新过程，以提升冻结的 LLM 在序列交互中的表现。我们将记忆更新建模为多轮决策问题，并使用多轮 GRPO 端到端优化它。我们的训练配方引入了（i）回合奖励信号和（ii）在 rollouts 之间上下文无关的回合级优势估计，从而实现更细粒度的信用分配和多轮设置下更稳定的训练。我们在两个测试平台：多轮剪刀石头布（RPS）和限注德州扑克（LHE）上评估 MemoPilot。在这两个环境中，MemoPilot 在强基线上显著提升了冻结玩家的测试时学习，在两个游戏的 Elo 评分中均排名第一（LHE 为 1762，RPS 为 1590），并超过了所有基线记忆方法和包括 DeepSeek-V3.2 在内的专有模型。

## Abstract
Large language model (LLM) agents are increasingly deployed in long-running settings where improving through experience at test time becomes important. A common approach is to update an explicit memory after each interaction to guide future decisions. However, most existing methods rely on hand-designed prompting rules, making it difficult to align memory updates with downstream objectives over multi-step horizons consistently. We propose MemoPilot, a plug-in memory copilot that explicitly trains the memory update process to improve a frozen LLM's performance across sequential interactions. We formulate memory updating as a multi-turn decision problem and optimize it end-to-end with multi-turn GRPO. Our training recipe introduces (i) a turn-wise reward signal and (ii) a context-independent, turn-level advantage estimation across rollouts, enabling finer-grained credit assignment and more stable training in multi-turn settings. We evaluate MemoPilot on two testbeds: multi-round Rock-Paper-Scissors (RPS) and Limit Texas Hold'em (LHE). Across both environments, MemoPilot substantially improves test-time learning of a frozen player over strong baselines, ranking first in Elo ratings on both games (1762 on LHE and 1590 on RPS) and outperforming all baseline memory methods and proprietary models, including DeepSeek-V3.2.

## 2. 论文提出的方法论

### 核心思想
将**记忆更新过程建模为多轮决策问题（MDP）**，使用多轮 GRPO（Group Relative Policy Optimization）端到端优化记忆模型，使其学会从交互经验中提取战略洞察并生成有效指导。

### 关键技术细节

**（1）多轮记忆 MDP 建模**
- **状态空间** $S$：观察元组 $s_t = (e_t, m_{t-1})$，其中 $e_t$ 是最新游戏轨迹，$m_{t-1}$ 是前一轮记忆
- **动作空间** $A$：文本记忆空间，生成器采样 $m_t \sim G_\theta(\cdot | s_t)$
- **奖励函数** $R$：返回游戏结果 $r_t$
- **回合回报**：$R(\tau) = \sum_{t=1}^{T-1} r_{t+1}$（跳过第一局作为初始探索）

**（2）多轮 GRPO 训练配方**
- 使用组内归一化的单步优势估计：$\hat{A}_{i,t,k} = R_{i,t} - \text{mean}(\{R_{i,t}\}_{i=1}^G)$，其中 $R_{i,t} = r_{i,t+1}$
- 采用 clipped surrogate objective 避免策略崩塌
- 优化目标扩展为（组，回合，token）三维结构

**（3）3层结构化记忆空间**
| 层级 | 功能 |
|------|------|
| **REASONING（推理层）** | 分析证据与结果，推断对手策略 |
| **KNOWLEDGE MAINTENANCE（知识维护层）** | 维护假设状态，记录观察计数和证据标签 |
| **FINAL STRATEGY PROMPT（行动层）** | 提炼可执行的动作规则，供玩家使用 |

### 算法流程
```
for each training iteration:
    采样对手策略 σ
    for each rollout i (共G个):
        初始化 m_{i,0} = ∅
        for t = 1 to T:
            (e_{i,t}, r_{i,t}) ← Play(π, σ, m_{i,t-1})
            if t < T:
                m_{i,t} ∼ G_θ(· | e_{i,t}, m_{i,t-1})
        for u = 1 to T-1:
            R_{i,u} ← r_{i,u+1}
    计算优势估计 ˆA_{i,t,k}
    通过 GRPO 更新 θ
```

**关键发现：**

4. **结构化记忆的优势**：3层格式优于自由形式，RL训练对性能至关重要（无RL的3层记忆仅略微改善）

5. **长horizon训练的重要性**：使用5轮rollout训练比2轮产生更稳定、更持久的收益

6. **单步奖励优于累积回报**：避免未来环境随机性导致的方差爆炸

7. **全历史不一定有用**：简单追加历史会引入噪声，降低表现

## 7. 优点

1. **端到端优化记忆更新**：首次将记忆更新策略作为可训练组件，使用 RL 直接优化下游任务表现

2. **插件式设计**：冻结玩家无需参数更新，即可适配更强的或闭源模型

3. **创新的多轮 GRPO 配方**：
   - 回合级优势估计实现细粒度信用分配
   - 单步代理奖励避免环境噪声
   - 组内归一化提升训练稳定性

4. **结构化记忆架构**：3层设计（推理→知识维护→行动指导）提供清晰的知识表示和更新机制

5. **全面的实验验证**：
   - 多游戏场景 + 真实世界基准
   - 丰富的消融实验
   - 跨玩家泛化验证
   - 系统化的对手池设计

6. **可复现性**：公开代码，便于后续研究跟进

## 8. 不足与局限

1. **依赖信息丰富的经验和可观测奖励**：
   - 当轨迹信息量低或奖励极度稀疏时，记忆更新可能缺乏有效证据
   - 论文未探索使用辅助信号（如token效率、轨迹质量评分）缓解稀疏奖励问题

2. **有限的记忆容量**：
   - 512-token 的记忆预算是超参数，复杂单任务长轨迹仍需分块或摘要预处理
   - 未探索自适应记忆压缩机制

3. **非平稳/自适应对手下的性能退化**：
   - 对手频繁切换或配备记忆时，性能明显下降（Table 7）
   - 维护-修正权衡：当对手故意反转行为时，积累的假设可能过时

   - 仅验证于两人博弈游戏（RPS、LHE）
   - 对手通过指令预设，缺乏真实人类行为的不可预测性

5. **评估场景相对受限**：
   - 游戏场景虽可控制但可能无法完全代表真实世界复杂交互
   - StreamBench 仅包含 32 个 episode，样本量有限
