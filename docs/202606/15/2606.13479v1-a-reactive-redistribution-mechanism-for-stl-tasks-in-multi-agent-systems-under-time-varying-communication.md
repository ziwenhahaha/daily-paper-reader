## Abstract
We present a communication-aware task decomposition framework for multi-agent systems with collaborative relative configuration objectives specified in Signal Temporal Logic (STL), allowing for dynamic task reallocation under time-varying communication networks. Building on our prior work, the framework supports the direct use of existing feedback controllers for reactive task satisfaction. We address two key challenges: disjunctive STL specifications and time-varying communication networks. Disjunctive specifications are handled through a graph transition system that captures the alternative task sequences induced by logical OR operators. To address time-varying connectivity, we introduce a redistribution mechanism that transfers tasks from disconnected agents to connected ones as the network evolves while preserving decentralized execution. Simulations and experiments on a swarm of Crazyflie drones demonstrate scalability in the number of agents, communication connectivity, and specification complexity.

- **核心问题**：现有基于反馈的多智能体 STL 控制方法通常假设任务依赖关系与底层通信拓扑兼容，即耦合的 agents 必须能够交换状态信息以强制执行约束。但在实际场景中，通信受限于距离约束（通信半径 r），任务耦合的 agents 可能并不直接相连，导致某些规范在纯局部反馈律下不可实现。

  1. **析取型 STL 规范**：实际任务中常包含逻辑 OR 运算符，需要在多种任务序列之间做出选择或切换。
  2. **时变通信网络**：agents 之间的通信连接随时间动态变化，进一步加剧了任务执行的不确定性。

## 2. 论文提出的方法论

### 核心思想

论文提出了一个**通信感知的任务分解框架**，通过两个主要机制解决上述挑战：

### 关键技术细节

#### 2.1 相对构型与 STL 规范

- **相对构型定义**：对于 agents i 和 j，其相对状态 $e_{ij} = x_j - x_i$，相对构型 $H_{ij}(\eta_{ij})$ 是一个平移的超矩形（hyper-rectangle），由目标相对向量 $c_{ij}$ 和容差向量 $\alpha_{ij}$ 定义。
- **STL 公式片段**：考虑如下形式的 STL 规范 $\psi$，包含 Always（$G$）和 Eventually（$F$）时序操作符以及逻辑合取（$\wedge$）和析取（$\vee$）：
  $$\phi := TI\theta,\quad \psi := \phi | \psi_1 \wedge \psi_2 | \psi_1 \vee \psi_2$$
- **任务图映射**：将任务 $\psi$ 映射为静态图 $G(\psi) = G(V, E_\psi)$，其中 $E_\psi = \{(i,j) | \exists \mu_{ij} \in \psi}$。

#### 2.2 通信一致性（Communication Consistency）

- **定义**：任务图 $G(\psi_\wedge)$ 在时刻 t 是通信一致的，当且仅当每个邻居都在通信图中，即 $N_i(\psi_\wedge) \subseteq N_{i,c}(t), \forall i \in V$。
- **不一致情况**：agents 之间不相连 $(i,j) \notin G_c(t)$，或相对构型在通信半径内不可达（$H_{ij}(\eta_{ij}) \cap C_{ij} = \emptyset$）。

#### 2.3 图转换系统（GTS）

- **目的**：将析取型全局任务 $\psi_{glob}$ 转换为图转换系统 $\Omega$，其状态为任务图 $G(\psi_{\wedge p_i})$，转换关系 $A$ 捕获了由逻辑 OR 诱导的替代任务序列。
- **构造流程**（Alg. 1）：
  1. 将 $\psi_{glob}$ 转换为析取范式（DNF）：$\psi_{glob} = \bigvee_{k=1}^K \psi_{\wedge k}$。
  2. 对每个时期 $p_i$ 的任务进行分组，具有相同第一时期任务的指标归为一组。
  3. 利用合取/析取分配律：$\bigvee_{k \in K_l} (\bar{\psi}_{\wedge p_1}^{K_l} \wedge \psi_{\wedge(p_i > p_1)}^k) = \bar{\psi}_{\wedge p_1}^{K_l} \wedge \bigvee_{k \in K_l} \psi_{\wedge(p_i > p_1)}^k$。
  4. 递归构造节点和转换关系，直到所有时期穷尽。
- **关键引理（Lemma 1）**：对于 GTS $\Omega$ 的每个运行 $\sigma \in L_\Omega$，满足 $\sigma$ 即满足 $\psi_{glob}$。

#### 2.4 任务分解（反应式重分配）

- **核心思想**：将通信不一致的相对谓词 $\mu_{ij}$ 替换为沿着通信图中路径 $\pi_{ij}$ 上的一系列局部相对谓词 $\tilde{\mu}_{ij}^{rs}$。
- **Minkowski 包含约束**：确保分解后的局部构型集合的 Minkowski 和包含在原构型集合内：
  $$\bigoplus_{(r,s) \in \epsilon(\pi_{ij})} \tilde{H}_{ij}^{rs}(\tilde{\eta}_{ij}^{rs}) \subseteq H_{ij}(\eta_{ij})$$
- **处理过载（Overloading）**：当两个任务的分解路径相交于同一 edge $(r,s)$ 时，确保新引入的谓词与已存在的谓词保持包含关系 $\tilde{H}_{ij}^{rs}(\tilde{\eta}_{ij}^{rs}) \subseteq H_{rs}(\eta_{rs})$。
- **循环依赖（Cycle Dependencies）**：对于包含至少 $L-1$ 个 Always 操作的循环 $\pi_\omega$，必须满足 $0 \in \bigoplus_{(r,s) \in \epsilon(\pi_\omega)} H_{rs}(\eta_{rs})$，通过引入辅助变量 $\zeta_{aux}^{rs}$ 施加线性约束。
- **优化问题（式 16）**：求解凸规划 $D\langle t, \psi_{\wedge p_i} \rangle$：
  $$\min_\xi f\langle t, \psi_{\wedge p_i} \rangle(\xi) \quad \text{s.t.} \quad g\langle t, \psi_{\wedge p_i} \rangle(\xi) \leq 0$$
  其中代价函数 $f$ 鼓励新分解的相对构型接近当前 agents 的实际相对位置。

#### 2.5 整体算法（Alg. 2）

1. 初始化：构建 GTS $\Omega$，设置候选任务图集合 $\hat{\Gamma}_{on} = \hat{\Gamma}_{p_1}$。
2. 主循环：
   - 在 $\hat{\Gamma}_{on}$ 中搜索可分解的任务图 $G(\psi_{\wedge p_i})$。
   - 通过 $D\langle t, \psi_{\wedge p_i} \rangle$ 获取分解后的任务图 $G(\tilde{\psi}_{\wedge p_i})$。
   - 执行反馈控制器 $u_i \langle t, G(\tilde{\psi}_{\wedge p_i}) \rangle$ 直到时期结束。
   - 更新 $\hat{\Gamma}_{on} = A(G(\psi_{\wedge p_i})) \subseteq \hat{\Gamma}_{p_{i+1}}$。
3. 到达终端节点时算法终止。
4. **定理 1**：闭环轨迹 $x$ 满足 $x \models \psi_{glob}$。

## 8. 不足与局限

1. **STL 片段限制**：仅支持 $G$ 和 $F$ 时序操作符的析取范式，无法处理更复杂的时序嵌套结构（如 $G(F(\cdot))$）。
2. **路径长度约束**：分解路径长度必须满足 $L \geq l \lfloor \|c_{ij}\| / r \rfloor$，在极端稀疏的通信网络中可能无法找到可行路径。

4. **缺乏基准对比**：未与其他通信感知的多智能体 STL 方法（如分布式状态估计）进行性能对比。
5. **假设理想动态**：假设 agents 遵循指定的非线性输入仿射动态，未考虑实际系统中的扰动、延迟或建模误差。
