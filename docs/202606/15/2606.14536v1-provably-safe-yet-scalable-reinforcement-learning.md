## Abstract
Safe reinforcement learning (RL) aims to learn policies that optimize rewards while satisfying constraints. Predominant approaches rely on soft-constrained policy optimization, which has achieved empirical success but does not provide formal safety guarantees for the learned policy. In contrast, methods with strict guarantees typically rely on explicit certificate functions, whose construction requires the direct synthesis and verification of control-invariant sets, a process that scales poorly with state dimension and often yields overly conservative behavior. In this paper, we present the Provably Safe, yet Scalable RL (PS2-RL) framework, a novel two-phase architecture for learning provably safe policies in a scalable manner, designed to overcome the key bottlenecks of prior methods. Rather than explicitly computing invariant sets, PS2-RL leverages a learned backup policy to forward-integrate the system dynamics, generating an implicit control-invariant set online. In the first phase, the backup policy is trained with our proposed safe-arrival value function, which characterizes the optimal backup policy for invariant-set construction. In the second phase, an RL policy is trained end-to-end through a differentiable projection layer that strictly enforces the safety guarantees induced by the learned backup policy. By maximizing the volume of the implicit control-invariant set in the first phase, the resulting PS2 policy from the second phase is performant and scalable, while maintaining provable safety. Crucially, PS2-RL imposes no restrictions on the underlying RL algorithm and can be plugged into any existing training pipeline. We establish theoretical guarantees for the proposed framework and evaluate it on robotic control tasks with state dimensions up to 10, a regime in which prior provably safe RL methods struggle or become impractical.

**研究动机：**
- 强化学习（RL）在控制复杂机器人系统方面取得了显著成功，但缺乏安全性保证阻碍了其在真实系统上的部署。
- 现有的安全RL方法主要分为两类：
  - **软约束策略优化**（如Lagrangian方法）：虽有一定经验效果，但无法提供**形式化安全保证**，可能在部署时发生灾难性失败。
  - **可证明安全的方法**（基于CBF、安全指数等显式证书）：需要直接综合和验证**控制不变集**，该过程随状态维度呈指数级增长，导致**可扩展性差**，且往往产生过度保守的行为。

**核心问题：**
- 如何设计一种可扩展的安全RL框架，使其能够在高维、输入受限系统中学习**可证明安全**且**高性能**的策略？

## 2. 方法论：核心思想与技术细节

### 2.1 核心思想

PS2-RL（Provably Safe, yet Scalable RL）提出了一种**两阶段架构**，核心思路是：

1. **不显式计算全局不变集**，而是利用学习到的备份策略通过**前向积分**动态生成**隐式控制不变集**。
2. 在第一阶段（Phase I），训练一个**安全到达策略**（safe-arrival policy）来最大化隐式不变集的体积。
3. 在第二阶段（Phase II），通过一个**可微分投影层**（控制不变层，CIL）将RL策略投影到BCBF容许集合中，实现端到端训练，同时严格保证安全性。

### 2.2 关键技术细节

#### 2.2.1 备份控制屏障函数（BCBF）框架

- **系统模型**：控制仿射连续时间系统 $\dot{x} = f(x) + g(x)u$
- **备份诱导控制不变集** $C_T(\pi_b)$：从状态 $x$ 出发，在备份策略 $\pi_b$ 下，在备份 horizon $T$ 内到达已知基集 $B$，且全程保持在安全集 $S$ 内。
- BCBF的约束是**仿射的**（在固定备份策略下），因此构成的可行集是凸的，可高效求解。

#### 2.2.2 基集与基控制器的构造

- **假设4.1**（局部可镇定性）：存在平衡点 $(x_*, u_*)$，其线性化 $(A,B)$ 是局部可镇定的。
- **定理4.1**：在假设4.1下，可构造一个局部椭球基集 $B_c$ 和线性反馈控制器 $\pi_B(x) = u_* - K(x - x_*)$，使其是控制不变的，且 $B_c \subseteq S$。这避免了全局不变集综合的高计算复杂度。

#### 2.2.3 安全到达价值函数（Safe-Arrival Value Function）

- **定义**：对于备份策略 $\pi_{SA}$，定义折扣安全到达Q函数：
$$Q^{\pi_{SA}}_{SA,\beta}(x,u) = \mathbb{E}\left[\sum_{k=0}^{\infty}\beta^k \left(\prod_{\tau=0}^{k-1}c(x_\tau)\right) b(x_k)\right]$$
- **含义**：$b(x) = 1_B(x)$（基集指示函数），$c(x) = 1 - b(x) - f(x)$（持续集指示函数），$f(x) = 1_F(x)$（失败集指示函数）。该价值函数只在**首次安全到达基集**时给予奖励，折扣因子鼓励**时间最优**到达。
- **自洽条件**：$Q^{\pi_{SA}}_{SA,\beta}(x,u) = b(x) + \beta c(x) Q^{\pi_{SA}}_{SA,\beta}(F(x,u), \pi_{SA}(F(x,u)))$
- 该价值函数满足Bellman方程，可使用标准RL pipelines（如TD3）训练备份策略。

#### 2.2.4 控制不变层（Control-Invariant Layer, CIL）

- **定义**：给定备份策略 $\pi^*_b$，CIL将策略输出投影到BCBF容许集合：
$$P_{CIL}(\pi_\phi)(x) = \arg\min_u \|u - \pi_\phi(x)\|_2^2 \quad \text{s.t.} \quad u \in K_{BCBF}(x; \pi^*_b)$$
- 这是一个**凸二次规划**（BCBF约束 + 输入箱约束），通过可微分QP求解器（如qpax）实现端到端可微。
- **PS2策略**：$\pi^{PS2}_\phi(x; \pi^*_b) := P_{CIL}(\pi_\phi)(x)$

#### 2.2.5 理论保证

- **定理4.2**（分解与精确性）：两阶段分解提供了Problem 1的一个**可证明内近似**，且在一定条件下与全局最优解一致。
- **定理4.3**（安全性保证）：任何通过CIL投影的策略都是BCBF可行的，从而满足Problem 1的约束。
- **定理4.4**（通用逼近）：CIL保持了策略网络的通用逼近能力，即PS2策略类能够以任意精度逼近BCBF可行策略类。

### 2.3 算法流程

**Phase I：训练安全到达策略**
1. 初始化安全到达策略参数 $\theta$ 和价值函数参数。
2. 从设计区域 $\Omega \setminus B$ 采样初始状态。
3. 在环境中收集rollout，记录每步的基集/失败集/持续集指示函数。
4. 使用折扣安全到达Bellman递归更新价值函数。
5. 改进 $\theta$（如TD3风格的策略更新）。
6. 返回冻结的组合备份策略 $\pi^*_b$。

**Phase II：通过CIL训练PS2策略**
1. 冻结 $\pi^*_b$，构建BCBF容许集合 $K_{BCBF}(x; \pi^*_b)$。
2. 使用SAC等RL backbone训练任务策略 $\pi_\phi$。
3. 每个控制步，将 $\pi_\phi(x)$ 通过CIL投影到 $K_{BCBF}(x; \pi^*_b)$。
4. 任务奖励梯度通过CIL端到端反向传播。
5. 返回PS2策略 $\pi^{PS2}_\phi$。

## 6. 主要结论与发现

1. **100%安全性**：PS2 SA在Unicycle lane-keeping和Quadrotor powerloop两个任务中均实现了**100%的训练和部署安全率**，在最坏情况下也能保持安全。

2. **性能最优**：在所有100%安全的方法中，PS2 SA的跟踪误差最低：
   - Unicycle：y-RMSE从PS2 ABP的0.97m降至0.52m（降低46%），ψ-RMSE从0.15降至0.11 rad。
   - Quadrotor：位置RMSE从1.39m降至0.60m（降低57%），所有RMSE指标降低40-57%。

3. **学习备份策略的价值**：相比使用解析备份策略（PS2 ABP），学习安全到达策略显著扩大了隐式不变集（在Unicycle中安全到达比例从0.23增至0.33；在Quadrotor近天花板区域从35.5%增至69.3%），这直接转化为更少的保守性和更好的跟踪性能。

4. **可扩展性验证**：在10维四旋翼系统（传统可证明安全方法难以处理的维度）上成功验证，推理时延仅为0.80ms，远低于20ms控制周期。

5. **控制不变层不可或缺**：消融实验表明：
   - **移除CIL后**：安全率从100%降至0%，策略网络未"内化"安全约束。
   - **仅在评估时添加CIL**：虽保证安全，但跟踪性能比端到端训练差43-53%。

## 7. 优点

1. **理论完备性**：
   - 提供了安全保证的正式证明（定理4.2/4.3/4.4），且证明了通用逼近性质，保证了表达力不因安全约束而损失。

2. **两阶段解耦设计**：
   - 将安全（最大化不变集）和性能（最大化任务奖励）解耦为两个可独立优化的子问题，每个子问题可使用成熟的RL方法。

3. **安全到达价值函数的创新**：
   - 提出了一种基于指示函数的折扣价值函数，直接编码"首次安全到达基集"的语义，简化了信用分配，无需手工设计几何塑形函数。

4. **端到端可微训练**：
   - 通过可微分QP层，策略可以直接学习在BCBF容许集合内优化性能，而非仅依赖后验过滤。

5. **方法通用性**：
   - PS2-RL不限制底层RL算法（SAC/TD3/Q-learning均可），也不依赖特定网络架构，只需保证策略网络具有通用逼近能力。

6. **实际可行的高维安全控制**：
   - 突破了传统CBF/SOS/HJ方法在高维系统上的指数级复杂度瓶颈，通过局部基集+学习备份+在线投影的组合实现了高维可证明安全控制。

## 8. 不足与局限

1. **对模型的依赖**：
   - 形式化保证要求有**控制仿射解析模型或可微分仿真器**，在模型未知或难以获取的情况下无法直接应用。

2. **基集构造仍需局部镇定性假设**：
   - 虽然比全局不变集综合容易得多，但需要假设4.1（局部可镇定性），对于强非线性或无平衡点的系统可能不满足。

3. **备份策略/BCBF约束随状态变化重新求解QP**：
   - 虽然推理时延低（<1ms），但每个时间步需解决一个凸QP，在极端低功耗嵌入式控制器上可能成为瓶颈。

4. **实验覆盖有限**：
   - 仅在两个仿真任务上验证，未涉及真实硬件部署、多模态感知输入或随机动力学。

5. **对MPS的比较可能存在不公平**：
   - MPS被给予更长的恢复horizon，但论文未提供消融实验验证horizon差异对MPS性能的影响程度。

6. **安全到达策略训练可能不稳定**：
   - Phase I使用课程学习（curriculum）来扩展初始状态分布，且依赖超参数（如课程成功率阈值、增量步长），在不同任务上可能需要大量调参。

7. **对输入约束的处理**：
   - BCBF约束在基集边界处可能因输入约束而退化为空，论文通过slack正则化QP缓解，但理论上可能在某些边缘情况下导致约束违反。
