## Abstract
Agentic reinforcement learning (RL) equips large language models (LLMs) with tool-use capabilities that substantially improve reasoning on complex tasks. However, integrating external tools often destabilizes training: over-reliance on tools can induce input distribution shift, while overly conservative tool use limits effective exploration. To address this issue, we propose a unified framework TAO-RL that couples tool-aware trajectory filtering with entropy-guided exploration for efficient policy optimization. Specifically, at the data level, TAO-RL filters rollout trajectories along two criteria: discarding those where all tool invocations fail to execute, and removing those where all rollouts are either correct or incorrect, as both cases yield degenerate advantage estimates that contribute no discriminative learning signal. This joint filtering retains data that are both tool-capable and informative, establishing a high-quality training distribution. At the algorithmic level, we introduce a tool-aware entropy-guided bonus that reshapes the advantage function at post-tool-call tokens, encouraging the policy to explore more diverse reasoning paths at critical decision points. These two components are mutually reinforcing: trajectory filtering establishes a clean and informative training foundation, while entropy-guided exploration drives stronger reasoning behaviors at critical tool-interaction junctures. Extensive experiments on 7 challenging reasoning benchmarks across 3 model scales demonstrate the superiority of TAO-RL over existing methods.

### 1. 论文核心问题与整体含义

**研究背景**：
- Agentic 强化学习（RL）赋予大语言模型（LLM）外部工具调用能力，显著提升复杂多步推理任务的性能
- 典型工具包括代码解释器、搜索引擎等

**核心问题**：
- **训练不稳定**：工具调用后外部执行结果注入上下文，导致 token 模式急剧变化（输入分布偏移），使策略更新脆弱
- **探索受限**：过于保守的工具使用策略限制了有效探索，阻止策略发现高价值的行为模式
- **矛盾本质**：需要在训练稳定性与有效探索之间取得平衡

**研究动机**：现有方法或仅关注格式过滤（忽略工具执行成功与否），或仅基于模型内部信号定位关键 token（忽视工具输出对后续推理的信息价值），均未能系统解决工具交互引入的双重挑战。

### 2. 论文方法论

**核心框架**：TAO-RL（Tool-Aware Optimization with Entropy Guidance）

#### 2.1 数据层面：工具感知轨迹过滤（两标准）

**标准 1：工具执行有效性过滤**
- 丢弃同时满足「答案错误」且「所有工具调用均失败」的轨迹
- 理由：此类轨迹既不提供正确推理信号，也不提供有效环境反馈，属于双重无信息量样本

**标准 2：答案可区分性过滤**
- 计算经验通过率 ρ(q) = (1/G)∑ri
- 丢弃 ρ(q)=1（全对）或 ρ(q)=0（全错）的查询
- 理由：全对产生零优势估计（全轨迹奖励相同），全错无法提供正向对比信号，两者都产生退化的优势估计

**过滤效果**：保留同时满足「工具可用」且「答案对比信号明确」的轨迹，构建高质量训练分布。

#### 2.2 算法层面：熵引导探索

**后工具调用 Token 识别**：
- 定位工具执行后的 token 位置集合 Mpost
- 计算每个 token 的预测熵 Ht = -∑v π(v|x,o<t)·log π(v|x,o<t)

**熵门控机制**：
- 保留预测熵位于 top-qH（默认20%）的后工具调用 token
- 定义二元门控 gt = 1[t ∈ Mpost ∧ Ht ≥ τH]

**策略更新**：
- logit 变化分解为「基础更新」+「探索奖励」两部分
- 探索奖励仅作用于高熵后工具调用 token，集中优化压力于最不确定的关键决策点

#### 2.3 理论分析

**梯度信噪比（SNR）提升**：通过移除零均值、高方差的退化轨迹，TAO-RL 的 SNR > 标准 GRPO 的 SNR：
$$\frac{SNR_{TAO-RL}}{SNR_{GRPO}} = 1 + \frac{(G-n)\sigma^2_{deg}}{n\sigma^2_{valid}} > 1$$

**两步互补机制**：
- 无轨迹过滤：熵奖励会在噪声轨迹上放大梯度，导致不稳定
- 无熵奖励：过滤后的梯度对所有 token 一视同仁，无法集中于关键位置

### 6. 主要结论与发现

#### 6.1 性能提升

**Qwen2.5-7B 上最佳结果**：
- AIME24：Avg@16 = 35.42（+2.09 vs SimpleTIR），Pass@16 = 73.33（+6.66）
- AIME25：Avg@16 = 23.96（+2.92），Pass@16 = 46.67（+20.00）
- MATH500：Avg@16 = 78.80（+1.29），Pass@16 = 95.60（+2.20）
- OlympiadBench：Avg@16 = 46.99（+4.04），Pass@16 = 70.00（+3.40）

#### 6.2 关键发现

1. **训练稳定性**：TAO-RL 在 AIME24/25 上呈现稳定上升曲线，而 TIR 波动剧烈、SimpleTIR 有起伏

2. **推理深度提升**：响应长度显著增加（7B 模型平均 Len@16 = 5208 vs SimpleTIR 的 4413），尤其在 AIME24/25 等难题上

3. **工具使用质量**：
   - Valid Code 达 99.28%（接近 SimpleTIR 的 99.03%），远超 TIR 的 54.17%
   - Code Line 平均 59.48 行（+9.45 vs SimpleTIR）

4. **熵引导探索优势**：相比 AEPO，TAO-RL 以更少的工具调用（2.26 vs 3.59 on AIME24）获得更高准确率，说明「质量 > 数量」

5. **分布外泛化**：LiveCodeBench 上 Pass@16 = 45.34%（+1.48 vs SimpleTIR），Len@16 = 1971.88（+237.90），验证跨领域迁移能力

6. **超参数敏感性**：
   - α=0.01 达到最佳平衡（α=0.1 导致不稳定，α=0.001 激励不足）
   - qH=20% 最优（过严限制探索空间，过松引入噪声）

6. **Case Study 直观**：通过具体问题展示 TAO-RL 的「策略切换」「错误恢复」能力，对比基线的「幻觉螺旋」「超时失败」

### 8. 不足与局限

1. **超参数依赖**：α 和 qH 需要针对不同任务调优，增加了实际应用成本

2. **工具类型单一**：论文仅考虑代码解释器，未涉及搜索、数据库、多步组合工具调用等更复杂场景

3. **开放任务未验证**：实验集中于数学/代码推理任务，对开放式 agentic 场景（如对话、任务规划）的适用性未知

4. **对比方法可扩展**：可增加与 RAG-LLM、CoT prompting 等非 RL 方法的对比，强化 TAO-RL 在不同范式中的定位

5. **计算开销**：额外的熵计算和过滤步骤带来少量 overhead，虽文中说明影响有限，但未给出精确时间对比

6. **重采样策略**：采用 2.0 倍过采样补偿被过滤轨迹，可能增加总体训练成本
