## Abstract
We present AgentJet, a distributed swarm training framework for large language model (LLM) agent reinforcement learning. Unlike centralized frameworks that tightly couple agent rollouts with model optimization, AgentJet adopts a decoupled multi-node architecture in which swarm server nodes host trainable models and run optimization on GPU clusters, whereas swarm client nodes execute arbitrary agents on arbitrary devices. This design provides capabilities that are difficult to support in centralized frameworks: (1) heterogeneous multi-model reinforcement learning, enabling the training of heterogeneous multi-agent teams with multiple LLM as brains; (2) multi-task cocktail training with isolated agent runtimes; (3) fault-tolerant execution that prevents external environment failures from interrupting the training process; and (4) live code iteration, which allows agents to be edited during training by replacing swarm client nodes. To support efficient RL in multi-model, multi-turn, and multi-agent settings, AgentJet introduces a context tracking module with timeline merging, which consolidates redundant context and achieves a 1.5-10x training speedup. Finally, AgentJet introduces an automated research system that takes a research topic as input and autonomously conducts long-horizon, multi-day RL studies on large-scale clusters. By leveraging the swarm architecture, this system reproduces key exploratory workflows of RL researchers without human intervention during execution.

  - **调试摩擦**：修改智能体代码或奖励函数需重启整个训练流程（重新加载模型和 vLLM），一次迭代周期长达 5–10 分钟。
  - **多模型约束**：现有系统多只能优化单一策略模型，难以支持异构多智能体系统（如规划用 32B 模型、执行用 7B 模型）。
  - **冗余上下文**：多轮交互中对话上下文累积大量冗余（重复系统提示、工具定义），导致策略梯度更新时计算浪费。
  - **环境锁定**：多任务混合训练需强隔离，单体架构难以提供。
- **核心问题**：如何设计一个能够在异构设备（有无 GPU 均可）上灵活训练一个或多个 LLM 模型的分布式智能体 RL 框架，同时支持容错、热插拔调试、异构多模型训练和多任务隔离？

## 论文提出的方法论

### 核心思想：Swarm（蜂群）架构

- **解耦训练与执行平面**：将模型优化平面（Swarm Server Nodes）与智能体执行平面（Swarm Client Nodes）完全分离，通过客户端-服务器范式组建动态训练网络。
  - **Swarm Server Node**（优化节点）：运行在 GPU 服务器或集群上，负责 LLM 策略梯度更新，托管 vLLM/SGLang 推理 API，内置自动上下文追踪模块，管理 episode 生命周期。每个服务器可独立托管和训练不同模型（如 Qwen3-32B、Qwen3-14B），支持非参数共享的多智能体训练。
  - **Swarm Client Node**（采样节点）：轻量级、CPU 专用进程，可在任意设备（工作站、笔记本）上运行，负责执行任意智能体工作流、读取数据集、运行智能体循环和计算任务特定奖励。客户端通过推理 API 与服务器交互，服务器透明地捕获所有上下文用于训练。

### 关键技术细节

1. **Swarm RL 范式流程**：
   - 设置 Swarm 服务器（从空白状态启动，仅注册地址等待客户端唤醒）。
   - 连接 Swarm 客户端（智能体运行器和 RL 轨迹生成器），指定一个客户端为控制器激活/停用服务器并同步训练参数。
   - 收集轨迹：服务器暴露 OpenAI 兼容推理 API，客户端执行 episode 时服务器为每个 episode 创建唯一的 BaseUrl/ApiKey 对并绑定内部上下文追踪器。Episode 可被客户端或服务器中止，完成后客户端提交奖励，轨迹进入服务器样本池。
   - 优化模型：样本池满时，触发时间线合并（Timeline Merging）减少冗余，计算策略梯度并更新权重。
   - 评估可在任意时间、任意节点调度，与训练并行进行。

2. **Episode Batching（片段批处理）**：支持五种收集策略（C1–C5）：
   - **C1（Episode-count）**：累计 episode 数量达阈值时触发训练。
   - **C2（Task-count）**：累计完成的任务数达阈值时触发。
   - **C3（Non-dummy task-count）**：仅当任务有非零奖励方差（提供可靠策略梯度信号）时视为有效，触发训练。
   - **C4（All-clients-agree）**：所有活动客户端节点一致同意后触发。
   - **C5（Any-client-agrees）**：任一客户端同意即触发。

3. **Context Tracking 与 Timeline Merging（上下文追踪与时间线合并）**：
   - **上下文追踪器**：每个被拦截的 LLM 调用生成独立时间线（Timeline），记录消息块（作者标签、文本、token ID、逐 token log 概率、二值损失掩码）。
   - **时间线合并**：多轮交互中后续 turn 会复用早期 turn 作为提示，时间线间存在长公共前缀。合并算法反向遍历，使长时间线吸收可匹配的前缀较短时间线，保留 LLM 生成 token 的标记（1.5–10× 训练加速）。支持两种匹配策略：
     - **Token-level**：精确 token ID 匹配，保证训练-推理一致性。
     - **Text-level**（默认）：精确文本匹配，更激进地合并，允许 tokenizer 漂移。

4. **框架无关的智能体支持**：服务器暴露 OpenAI 兼容端点，用户仅需将智能体的 LLM base URL 和 API key 替换为 AgentJet 提供的即可。支持 LangChain、AgentScope、MetaGPT、AutoGen、CrewAI、OpenAI SDK 及自定义 HTTP 实现。

5. **Automated Research（自动化研究）**：
   - **Vibe Training**：自然语言描述训练任务 → AI 编码助手生成 Swarm 客户端代码、奖励函数、数据集 → 自动初始化 Swarm 服务器并挂载客户端 → 训练中可热插拔调试（秒级重启客户端）。
   - **A3R（Alpha Auto Research）**：领导者代理（chief scientist）制定多阶段实验计划，生成标准化的实验蓝图（含目的、代码路径、超参数、命令、时间预算），通过云集群 API 或 SSH 将蓝图分发到多个 GPU 服务器并行执行工作者代理，实现多日长周期 RL 研究（超参数搜索、模型规模对比、LoRA 配置分析等），无需人类干预执行。

## 论文的主要结论与发现

1. **异构多模型训练可行且有效**：非共享参数训练使狼人智能体发展出不同人格（behavioral diversity），相比共享参数基线（64.7%）提升至 66.5%，因为共享参数下狼人行为模式相似易被村民识别。
2. **Timeline Merging 实现 1.5–10× 训练加速**：在 AppWorld 上实现 6.25× 墙钟时间加速，且不损害训练质量（奖励曲线和 LLM 调用数几乎不变）。
3. **鸡尾酒混合训练是构建多技能通用模型的低成本方案**：相比分离训练，在 AIME 上几乎无损（差距 < 0.01 reward point），在 AppWorld 上有约 10 个 reward point 的损耗，代价是工具密集型任务的梯度稀释；但该方法将 N+1 次训练（专家+蒸馏）压缩为单次运行。
4. **框架无关性得到验证**：四种不同智能体框架在相同 GRPO 配方下最终评估奖励 spread 仅 0.025，最大跨框架差距 < 0.04，证明 AgentJet 的训练捕获对上层框架透明。
5. **批大小与响应长度关系**：AIME GRPO 中，mr=10000 时最小稳定高效 batch size 为 16；batch 超过 16 不再提升性能；相比增大 batch size，将算力投入增加响应预算（mr=12000）更为有效（bs=16, mr=12000 达 60.00% pass@1，73.33% pass@2）。
6. **A3R 能自主完成多日 RL 研究**：领导者代理将研究问题分解为自适应多阶段计划，跨集群并行执行实验蓝图，实现超参数搜索和模型规模对比，无需人工干预。

## 优点

1. **架构创新性强**：首次提出将智能体执行平面与模型优化平面完全解耦的 Swarm 架构，支持 many-to-many 拓扑和动态节点管理。
2. **能力覆盖全面**：同时解决了容错、热插拔调试、异构多模型 RL、多任务隔离、上下文冗余、框架无关性等多个实际问题，而非单点改进。
3. **Timeline Merging 设计精巧**：利用多轮交互中时间线的公共前缀结构，自动合并冗余，算法简洁（逆向吸收），实现显著加速且不损失训练质量。
4. **Episode Batching 策略灵活**：提供 5 种触发策略（C1–C5），覆盖纯服务端控制和客户端协同控制，适应不同实验需求。
5. **端到端自动化程度高**：Vibe Training 和 A3R 从自然语言需求到完整实验报告全链路覆盖，降低了 RL 研究门槛。
6. **实验设计丰富**：涵盖多智能体对抗/合作场景、共享/非共享参数对比、鸡尾酒训练、框架无关性验证、长期稳定性测试等，维度多样。
7. **开源且框架无关**：支持 LangChain、AgentScope 等主流框架，零代码修改接入黑盒智能体。
