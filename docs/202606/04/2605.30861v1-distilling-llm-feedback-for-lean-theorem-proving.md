## Abstract
Post-training for reasoning models typically combines supervised fine-tuning with reinforcement learning from verifiable rewards, most commonly with GRPO. However, this algorithm suffers from sparse rewards, limited exploration, and mode collapse. Building upon recent works on self-distillation, we propose Feedback Distillation, a training method where the model is trained to match, at the token level, its own distribution conditioned on privileged feedback produced by a language model. Feedback Distillation offers token-level supervision and can inject external knowledge. Evaluating our method for Lean4 theorem-proving, we find that Feedback Distillation maintains greater diversity in generated trajectories than GRPO, yielding higher policy entropy and better pass@k scaling. The two methods are complementary: initializing GRPO from a Feedback Distillation checkpoint outperforms either method alone. All in all, our results suggest a promising avenue to improve post-training for complex reasoning.

## 1. 论文的核心问题与整体含义（研究动机和背景）

当前大语言模型（LLM）在复杂推理任务（特别是数学定理证明）中的后训练（post-training）通常采用**SFT+GRPO**范式：先用监督微调（SFT）在专家解决方案上训练，再用强化学习（如GRPO）最大化验证器提供的奖励。然而，该范式存在三个核心问题：

- **稀疏奖励**：GRPO仅提供轨迹级奖励，当所有采样均失败时无法提供学习信号
- **有限探索**：GRPO不会提升模型输出的多样性，反而趋向于模式崩溃（mode collapse），收敛到训练前就存在的高奖励模式
- **SFT的局限性**：依赖高质量专家数据的生成，成本高昂且可能导致灾难性遗忘

本论文的动机正是为了解决这些问题，提出一种既能提供**细粒度 token 级监督**、又能**保持轨迹多样性**、还能**注入外部知识**的训练方法。

## 2. 论文提出的方法论：Feedback Distillation

### 2.1 核心思想

Feedback Distillation 是一种**自蒸馏（self-distillation）变体**，其核心思想是：

> 让模型（学生）学习在获得**特权反馈（privileged feedback）**时自身的分布行为，从而将反馈信息内化到模型权重中。

具体而言，给定一个问题 x 和学生的尝试回答 y，引入一个**教师模型**，它在生成时额外接收一段反馈 F(y)（由另一个LLM或编译器输出生成），学生则不接收反馈。训练目标是让学生分布尽可能接近教师分布。

### 2.2 关键技术细节

| 组成部分 | 说明 |
|----------|------|
| **教师模型** | π'_μ,y(·|x, F(y))，接收原始问题 x 和反馈 F(y) |
| **学生模型** | π_θ(·|x)，只接收原始问题 x |
| **教师更新** | 每5步通过 EMA（指数移动平均）更新：μ ← αμ + (1−α)θ，默认 α=0.9 |
| **损失函数** | per-token KL 散度（等价于交叉熵）：L̂_θ = −∑_t ∑_a π_μ(a|x,F(y),y<t) log π_θ(a|x,y<t) |
| **Top-K 截断** | 为降低计算开销，仅对教师概率最高的 K=25 个 token 计算损失 |
| **反馈来源** | 可使用外部强模型（Claude Opus 4.6）、ground-truth 解决方案、或 Lean 编译器输出 |

### 2.3 算法流程（文字描述）

1. **采样阶段**：从数据集 D 中采样一个问题 x，让学生模型 π_θ 生成尝试回答 y
2. **反馈生成**：将 y 和 x 发送给反馈模型，生成特权反馈 F(y)
3. **教师分布计算**：在学生生成的前缀条件下，计算教师模型 π'_μ,y 的 token 级概率分布
4. **损失计算**：计算学生分布与教师分布的 per-token KL 散度
5. **梯度更新**：仅对学生模型参数 θ 更新，教师参数 μ 通过 EMA 缓慢跟随
6. **循环迭代**：重复以上步骤

## 6. 论文的主要结论与发现

### 6.1 核心发现

1. **Feedback Distillation 保持更高轨迹多样性**：在整个训练过程中，Feedback Distillation 的策略熵显著高于 GRPO，表明生成的轨迹更加多样化

2. **Pass@k 缩放性能更优**：在相近 Pass@1 准确率下，Feedback Distillation 的 Pass@k 随 k 增长更快，证明高熵反映的是真实的答案多样性而非均匀噪声

3. **更快的引理发现速度**：Feedback Distillation 能在训练中更快、更持续地发现 Mathlib 库中的新引理，表明探索能力更强

4. **GRPO over FD 优于单独方法**：先用 Feedback Distillation 训练，再切换到 GRPO，能获得比单独使用任一方法更高的峰值性能（Qwen3.5-9B：GRPO 59% → FD+GRPO 75%）

5. **Claude 反馈与 Ground-truth 反馈效果相近**：两者均优于仅使用 Lean 编译器输出的反馈

6. **Self-Feedback 仍有效**：即使反馈来自同一模型的冻结副本，结合 GRPO 仍优于单独 GRPO，但性能低于使用外部反馈

7. **EMA 参数权衡**：更激进的 EMA（更低的 α）加速早期学习但增加不稳定性；α=0.9 在当前设置下表现最佳

8. **反馈的内化验证**：训练后，模型能将固定反馈字符串（如"Be concise"）的行为内化到权重中，KL 散度收敛至零

9. **对认知语言表达的保留**：与 Ground-truth 反馈不同，Feedback Distillation（使用 critique 形式的反馈）不会显著抑制模型的不确定性词汇（"wait"、"hmm"等）的使用

## 7. 优点

1. **细粒度 token 级监督**：每个 token 都能获得独立的学习信号，无需等待整个轨迹完成
2. **保持探索多样性**：相比 GRPO，不易陷入模式崩溃，对大规模搜索空间（如 Lean Mathlib）尤为重要
3. **可注入外部知识**：不依赖验证器信号，可通过更强的外部 LLM 注入知识
4. **方法简洁、无需额外模型蒸馏**：无需托管反馈模型或共享 tokenizer，仅通过自然语言反馈传递知识
5. **样本效率优势**：在训练早期阶段，Feedback Distillation 的样本效率显著高于 GRPO
6. **即插即用的增强组件**：可作为 GRPO 的预训练阶段，提升 GRPO 的最终性能

## 8. 不足与局限

1. **长期训练不稳定**：Feedback Distillation 存在类似 GRPO 的训练不稳定性问题，且即使使用大 α 值也无法完全消除
2. **性能未达 SOTA**：论文明确承认当前结果尚未匹配大规模 SFT+GRPO pipeline 的性能水平
3. **反馈质量依赖反馈模型**：性能上限受限于反馈 LLM 的能力，且 Claude 等商业模型的调用成本需考虑
4. **实验领域单一**：所有实验均在 Lean 定理证明环境进行，未扩展到其他形式化或非形式化推理任务（如代码生成、通用问答）
5. **工具使用场景有限**：模型仅配备三个 Lean 专用工具，未探索更复杂的 agentic pipeline（如迭代式搜索、树搜索等）
6. **EMA 超参敏感性**：最佳 α 值可能随模型规模、数据分布变化，需要额外调优
7. **缺乏大规模验证**：实验基于 8B/9B 量级模型，未验证方法在更大规模模型（如 70B+）上的效果
