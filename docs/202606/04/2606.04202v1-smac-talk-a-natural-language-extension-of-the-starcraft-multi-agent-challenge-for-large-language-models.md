## Abstract
As LLMs become more widely deployed, they are increasingly expected to work alongside other AI agents rather than operating in isolation. Effective coordination in these settings requires agents to communicate, share information and make decisions under uncertainty. We introduce SMAC-Talk, a natural language extension of the StarCraft Multi-Agent Challenge for evaluating LLM-based agents in cooperative multi-agent environments. The environment has several key features such as decentralized control, partial observability and long-horizon decision making. SMAC-Talk includes a natural language communication channel which is used to probe agent coordination and trust. We use this communication channel to construct different evaluation scenarios, including settings with an embedded deceptive communicator that tries to disrupt and deceive allies through communication alone. We provide three agents for benchmarking using 4 models from the Qwen3.5 family and study how reasoning structure, memory and model scale affect coordination between agents. We release SMAC-Talk as an open benchmark to support the research community in developing and evaluating LLM agents in cooperative multi-agent settings.

## 2. 论文提出的方法论

### 核心思想
将SMACv2的数值观测和离散动作替换为自然语言动态，同时保留SMACv2的核心特性（部分可观测性、去中心化执行），并新增**自然语言通信通道**，支持智能体间的语言交互与协调评估。

### 关键技术细节
SMAC-Talk引入三个核心组件：

1. **观察转文本适配器（Observation-to-Text Adapter）**
   - 将数值观测转换为自然语言描述
   - 每条观测包含三部分：自我信息、友军信息、敌军信息
   - 自我信息：单位类型、当前血量、位置、视野范围、攻击范围、可执行动作
   - 友军信息：可见友军的状态，通信启用时附加其广播的消息
   - 敌军信息：可见敌军的类型、血量、位置、距离、是否在攻击范围内

2. **文本转动作适配器（Text-to-Action Adapter）**
   - 将LLM输出的自然语言动作映射为离散游戏动作
   - 可执行动作包括：四向移动（东南西北）、攻击指定敌人、治疗指定友军、停止、无操作
   - 验证每个动作，无效动作替换为stop并记录日志

3. **自然语言通信层**
   - 智能体每时间步提交一条自然语言消息
   - 消息广播给所有可见友军（保留部分可观测性）
   - 消息在提交后下一时间步送达，避免瞬时信号传递
   - 消息不被过滤或修改，允许自由表达意图和战术协调

### 评估智能体设计
- **Zero-Shot Agent**：最小指令，无游戏知识、无推理、无通信指导，使用非思考模式
- **ReAct Agent**：结构化链式思维推理，包含战斗策略启发（集中火力、定位、通信规则），维护2个时间步的历史记忆
- **Reasoning Agent**：使用LLM内部思考模式（1024 token预算），自由链式思维推理，接收2个时间步的历史观测和动作
- **Deceptive Communicator**（对抗性智能体）：不参与战斗，仅通过移动和通信干扰队友，策略包括伪造敌人信息、建议次优行动、虚假意图声明

## 3. 实验设计

### 场景设置
共定义8种评估场景，变体维度包括：

| 团队规模 | 通信类型 | 描述 |
|---------|---------|------|
| 5v5 | no_comm | 无通信基线 |
| 5v5 | comm | 自由通信 |
| 5v5 | KDC | 已知欺骗通信者（智能体被告知存在欺骗者） |
| 5v5 | UDC | 未知欺骗通信者（智能体不知情） |
| 10v10 | no_comm | 更大规模无通信 |
| 10v10 | comm | 更大规模自由通信 |
| 10v10 | KDC | 更大规模已知欺骗 |
| 10v10 | UDC | 更大规模未知欺骗 |

- 敌方控制：StarCraft II内置AI，难度设为Very Easy
- 团队组成：Marine（概率0.5）和Marauder（概率0.5）随机生成
- 位置生成：使用SMACv2的surround和reflect放置类型

### 模型对比
- 使用Qwen3.5家族4种规模：4B、9B、27B、122B-A10B
- 对比方法：三种智能体类型（Zero-Shot、ReAct、Reasoning）
- 评估指标：胜率（win rate）、平均奖励（SMACv2 reward信号）、动作错误率（invalid outputs）

### Benchmark特性
- 开源发布，支持研究社区
- 推理无关设计：支持vLLM、Llama.cpp、Cerebras、OpenAI兼容API

## 6. 论文主要结论与发现

### 关键发现

1. **模型规模效应**
   - 4B模型不足以可靠执行任务，动作错误率较高，协调能力弱
   - 9B是Qwen3.5家族中的最低有效规模
   - 27B与122B之间性能趋于平缓，暗示存在**收益递减**

2. **推理结构的影响**
   - **Reasoning Agent（内部链式思维）始终优于Zero-Shot和ReAct Agent**
   - ReAct的结构化推理在通信启用时**性能崩溃**（所有模型规模均如此）
   - 原因可能是结构化格式与通信自由格式不兼容，或启发式规则与推理模式冲突

3. **通信的双面性**
   - 对Zero-Shot和Reasoning Agent有**有限且不稳定的提升**
   - 对ReAct Agent产生**负面影响**，原因尚不明确

4. **欺骗通信场景**
   - 较大模型在**KDC（已知欺骗）场景中明显优于UDC（未知欺骗）**
   - 示例：122B-A10B Reasoning在5v5 KDC达到41%胜率，但在UDC仅10%
   - 表明被告知存在欺骗者能显著提升模型的**信任校准能力**

5. **提示工程的重要性**
   - 在多智能体设置中，提示架构选择的影响被放大
   - 单智能体设置中"可接受"的刚性输出结构，在多智能体、多时间步交互中会导致**复合误差**

## 7. 优点

- **填补研究空白**：首次将MARL的SMAC benchmark扩展为自然语言版本，支持LLM智能体评估
- **丰富的场景设计**：8种场景覆盖通信、无通信、欺骗通信的多种组合
- **多维度分析**：同时考察模型规模、推理结构、记忆长度、通信效果
- **对抗性评估**：引入欺骗通信者，评估智能体在**信任与鲁棒性**上的表现
- **工程完善**：支持多种推理后端（vLLM、LLaMA.cpp、Cerebras），便于复现
- **开源共享**：提供代码和prompt模板，促进社区研究

## 8. 不足与局限

- **模型泛化性**：仅测试Qwen3.5家族，其他LLM（如Claude、GPT、LLaMA）的表现未知
- **难度与种族覆盖**：仅使用Terran单位和Very Easy难度，Protoss和Zerg、高难度敌方未探索
- **推理预算未调优**：Reasoning Agent的1024 token思考预算的影响未被研究
- **ReAct崩溃根因未明**：通信导致ReAct性能下降的具体原因尚不清楚
- **计算成本高**：小模型（4B）性能不足，需要至少9B以上模型才能获得有意义结果，限制了资源受限研究者的使用
- **欺骗通信者规则约束**：虽然设计为"域内语言"，但可能的对抗性仍然受限
