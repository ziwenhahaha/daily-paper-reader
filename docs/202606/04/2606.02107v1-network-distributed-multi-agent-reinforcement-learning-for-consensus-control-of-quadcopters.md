## Abstract
This paper proposes a Network Distributed Multi-Agent Reinforcement Learning (ND-MARL) framework for quadcopter consensus control. Compared to conventional multi-agent MARL formulations that rely on centralized planning or fully decentralized execution, ND-MARL incorporates the swarm communication graph into the decision process. Under a 2-Neighbor communication topology, each agent observes information of only two neighbors and outputs an action through a distributed policy. A high-level distributed consensus planner is trained using Multi-Agent Soft Actor-Critic (MASAC) and embedded in a hierarchical stack to generate reference target positions tracked by a low-level quadcopter controller. Results demonstrate smooth consensus trajectories and planner-tracker integration when compared to a centralized MARL controller. Most notably, the learned controller exhibits zero-shot scalability, as policies trained on a three-agent system are deployed to swarms of up to 250 agents under the same 2-Neighbor communication topology without retraining or fine-tuning, achieving consistent convergence with increasing steady-state spread at large team sizes due to sparse information propagation. These findings highlight ND-MARL as a stable framework for distributed, communication-aware quadcopter consensus control.

## 1. 核心问题与整体含义（研究动机与背景）

- **研究问题**：无人机（UAV）蜂群的共识控制（consensus control），即如何让多个四旋翼飞行器在没有中央控制器的情况下协调一致地移动到同一位置。
- **现有方法的局限**：
  - **集中式MARL**：将整个蜂群视为单一系统，观测全局状态并输出联合动作。虽然能保证一致性和避免非平稳性，但状态-动作空间随智能体数量指数增长，通信开销大，难以扩展到大规模蜂群。
  - **去中心化MARL**：每个智能体仅依赖本地观测进行决策，虽提升可扩展性，但多智能体同时更新策略导致环境非平稳，学习不稳定或效率低下。
  - **CTDE（集中训练-去中心执行）**：虽在训练时可利用全局信息，但集中式评论器限制了其随蜂群规模增大的可扩展性。
- **本文动机**：提出一种**网络分布式多智能体强化学习（ND-MARL）**框架，将通信图结构融入决策过程，通过固定2邻域通信拓扑实现局部交互，从而在保持稳定性的同时实现零样本可扩展性。

## 2. 论文提出的方法论

### 核心思想

- 将全局共识问题分解为局部邻域内的成对共识子问题，而非要求每个智能体获取全局信息。
- 训练一个高层次的一致性路径规划器（consensus path planner），再将其嵌入分层控制栈中，由低层控制器执行实际飞行控制。

### 关键技术细节

#### （1）通信拓扑（2-Neighbor Topology）

- 每个智能体$i$仅与两个邻居通信，即$|N(i)| = 2$，且图是无向对称的。
- 该设计使得每个智能体的通信和计算成本与蜂群规模$N$无关，保持$O(1)$复杂度。

#### （2）状态空间与动作空间

- **单粒子规划层**：每个智能体控制一个无质量粒子，动作空间为二维加速度$[a_x, a_y]$，积分两次更新位置。
- **状态空间**（式3）：
  $$S_i = [v_x, v_y, p_{ij}, p_{ik}, v_{ij}, v_{ik}]$$
  - 包含自身速度、相对邻居的位置和速度，均为相对量，确保平移不变性。

#### （3）奖励函数设计

- **共识奖励**（式5）：
  $$r_i^{cons} = 1 - \tanh\left(\frac{\tilde{d}_i}{\kappa_c}\right), \quad \kappa_c = 0.15$$
  - $\tilde{d}_i$为智能体与邻居的平均距离归一化值。
- **速度奖励**（式7，仅在共识达成后激活）：
  $$r_i^{vel} = 1 - \tanh\left(\frac{\|v_i\|/v_{max}}{\kappa_v}\right), \quad \kappa_v = 0.2$$
- **共识损失惩罚**（式8）：
  $$r^{pen} = \begin{cases} -1, & \text{若共识已达成且} \bar{d}_{nh} > \delta_c \\ 0, & \text{其他} \end{cases}$$
- **总奖励**（式9）：
  $$R_i = w_c r_i^{cons} + w_v r_i^{vel} + r^{pen}, \quad w_c = 1.0, w_v = 0.5$$

#### （4）全局奖励因式分解

- 根据局部拓扑，全局奖励可因式分解为：
  $$R_G(s, a) = \sum_{i \in I} R_i(s_i, s_{N(i)}, a_i, a_{N(i)})$$
  - 这确保了分布式执行的可行性。

#### （5）MASAC算法训练

- 使用**多智能体软Actor-Critic（MASAC）**训练高层规划器，因其熵最大化特性鼓励随机探索，防止过早收敛，增强鲁棒性。
- 网络架构：两层MLP（256节点，ReLU激活），策略网络和评论器网络结构相同。
- 关键超参数：学习率$\lambda = 0.001$，折扣因子$\gamma = 0.99$，目标更新系数$\tau = 0.01$，目标熵$H_{target} = -2$，初始熵系数$\alpha_0 = 0.2$。

#### （6）分层控制栈

- **高层**：MASAC一致性规划器，输出目标位置。
- **低层**：基于先前工作的推力矢量控制器（thrust-vector controller），追踪目标位置，输出推力$U$和滚转/俯仰命令$(\phi, \theta)$。
- 低层状态（式13）包含姿态、线速度、角速度和位置误差。

## 6. 论文的主要结论与发现

1. **性能可比性**：分布式ND-MARL达到的平均共识时间为3.7秒，集中式为3.6秒；终端共识误差分别为0.001m和0.007m，差异仅6mm，两种方法性能相当。

2. **零样本可扩展性**：仅在3智能体上训练的三策略，可直接部署到250智能体蜂群而不需重训练或微调。

3. **通信与计算效率**：每个智能体的通信和计算成本恒定（$O(1)$），不随蜂群规模增长；而集中式方法的观测空间按$N^2 + N$增长，动作空间按$2N$增长。

4. **性能随规模退化**：随着$N$增大，共识时间增加（TCE从0.0013m增至0.644m），这是由于固定2邻域拓扑下信息传播速度变慢，局部邻域可能形成紧密局部簇而非全局共识。

5. **分层栈有效性**：高层MASAC规划器与低层推力矢量控制器的集成运行稳定，产生了平滑的轨迹和协调的飞行行为。
