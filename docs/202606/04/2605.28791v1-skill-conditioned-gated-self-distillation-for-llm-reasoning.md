## 摘要
策略自蒸馏（SD）利用教师端特权信息（PI）将稀疏的验证器输出转化为密集的token级监督，从而提升大语言模型的推理能力。现有方法通常假设PI是可信的，例如参考答案或成功轨迹。我们探索PI是否可以从经验构建的技能库中获取，其中检索到的技能紧凑且可复用，但可能也存在无关或误导的情况。我们提出技能条件化门控自蒸馏（SGSD），将基于技能的SD表述为教师假设验证而非无条件的模仿。SGSD检索技能-错误对，构建多教师池，并让所有技能条件化的教师对相同的普通提示学生rollout进行评分。验证器验证每个教师的极性：支持成功或压制失败给出正监督，而相反立场则反转。然后通过鲁棒的门控目标提炼信息丰富的师生分歧，同时抑制不确定或极端的信号。在多个数学推理基准上的实验表明，SGSD相比GRPO一致性地提升了性能，并在较弱的PI假设下与答案条件化的OPSD保持竞争力。例如，在Qwen3-1.7B上，SGSD在AIME24、AIME25和HMMT25上平均分别优于GRPO 6.2%和优于OPSD 1.7%。我们的代码发布于https://github.com/walawalagoose/SGSD。

## Abstract
On-policy self-distillation (SD) improves LLM reasoning by using teacher-side privileged information (PI) to turn sparse verifier outcomes into dense token-level supervision. Existing methods usually assume trusted PI, such as reference answers or successful traces. We ask whether PI can instead come from an experience-derived skill bank, where retrieved skills are compact and reusable but may also be irrelevant or misleading. We propose Skill-Conditioned Gated Self-Distillation (SGSD), which formulates skill-based SD as teacher hypothesis validation rather than unconditional imitation. SGSD retrieves skill-mistake pairs, constructs a multi-teacher pool, and lets all skill-conditioned teachers score the same plain-prompt student rollout. The verifier validates each teacher's polarity: supporting a success or suppressing a failure gives positive supervision, while the opposite stance is reversed. A robust gated objective then distills informative teacher-student disagreements while suppressing uncertain or extreme signals. Experiments on multiple mathematical reasoning benchmarks show that SGSD consistently improves over GRPO and remains competitive with answer-conditioned OPSD under a weaker PI assumption. For example, on Qwen3-1.7B, SGSD outperforms GRPO by 6.2% and OPSD by 1.7% on average on AIME24, AIME25, and HMMT25. Our code is available at https://github.com/walawalagoose/SGSD.

## 1. 核心问题与整体含义（研究动机与背景）

### 研究问题
- **核心矛盾**：现有自蒸馏（Self-Distillation, SD）方法依赖可信的特权信息（Privileged Information, PI），如参考答案或成功轨迹，将稀疏的验证器奖励转化为密集的 token 级监督。但这类可信 PI 在实际场景中往往难以获取。
- **关键探索**：能否从经验构建的技能库（Skill Bank）中获取 PI？技能库虽提供紧凑、可复用的知识，但检索到的技能可能不相关、部分相关或具有误导性，无法直接用于自蒸馏。
- **核心挑战**：
  1. PI 不再是单一可信源，而是检索到的技能假设集
  2. 技能的价值依赖于结果：同一教师支持可能有益或有害
  3. 多教师产生噪声或冲突信号，需提取有用分歧

### 核心观点
> 将技能-错误对视为**教师假设**，通过验证器结果判断其极性，而非无条件模仿检索到的技能。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 核心思想
提出 **Skill-Conditioned Gated Self-Distillation（SGSD）**，将基于技能的 SD 表述为**教师假设验证**问题。核心创新：
- **技能条件化多教师池**：检索技能-错误对，为每个对构建独立教师
- **结果验证的教师极性**：通过验证器结果决定教师的指导方向
- **鲁棒门控蒸馏目标**：提炼信息丰富的师生分歧，抑制不确定/极端信号

### 关键技术细节

#### （1）结构化技能库构建
- **一般技能（General Skills）G**：编码可复用的推理原则，包含 `title`、`principle`、`when_to_apply`
- **常见错误模式（Common Mistakes）E**：描述错误、原因及避免方法，包含 `description`、`why_it_happens`、`how_to_avoid`
- **冷启动**：从训练集采样问题生成轨迹，成功轨迹提取技能，失败轨迹提取错误模式，通过分层合并去冗余
- **在线更新**：定期从最近成功/失败轨迹提取新技能，合并入技能库

#### （2）技能条件化多教师池
- 对问题 x 检索 top-K 技能和 top-K 错误，配对形成 K 个教师
- 教师 k 的上下文：`ck(x) = gk ⊕ ek ⊕ x`（技能 ⊕ 错误 ⊕ 问题）
- 教师权重 αk(x) 由检索相似度 softmax 归一化得到

#### （3）结果验证的教师极性
- **token 级差距**：`Δt^(k) = log p_T^(k)(yt|x, y<t) - log p_S(yt|x, y<t)`
- **教师级支持分数**：`ak = (1/T) * Σ Δt^(k)`
- **极性决策规则**（表 2）：

| 条件 | 解释 |
|------|------|
| r > 0, ak > 0 | **Helpful teacher**（支持成功）→ 蒸馏 |
| r < 0, ak < 0 | **Warning teacher**（压制失败）→ 蒸馏 |
| r > 0, ak < 0 | **Over-suppressive teacher**（压制成功）→ 反转 |
| r < 0, ak > 0 | **Misleading teacher**（支持失败）→ 反转 |

#### （4）鲁棒支持估计
为防止极端 gap 干扰极性判断：
- **Token 掩码 mt**：移除格式 token、特殊 token、思考分隔符
- **差距裁剪**：`eΔt^(k) = clip(Δt^(k), -cΔ, cΔ)`
- **置信阈值**：`ρk = sgn_out(r) * sgn(eak)` if |eak| > εa，否则 ρk = 0

#### （5）门控蒸馏目标
- **门控损失**（式 14）：
  ```
  ℓgate(Δ) = log 2 - log(1 + exp(-Δ²/(2τg)))
  ```
  - 特性：near-zero gap 权重小；medium gap 提供有用监督；extreme gap 被指数级抑制
  - 梯度 `gτ(Δ) = Δ/(τg * (1 + exp(Δ²/(2τg))))`，满足 sign 一致性和有界性（|gτ(Δ)| ≤ 1/√eτg）
- **整体目标**（式 16）：
  ```
  L_SGSD(x) = Σk αk(x) * ρk * ℓ̄(k)
  ```
  其中 ρk 控制更新方向（Helpful → 蒸馏，Misleading → 反转，Uncertain → 无贡献）

#### （6）整体流程（Algorithm 1）
1. 采样 mini-batch
2. 对每个问题检索 K 个技能-错误对
3. 学生模型采样 rollouts 并获取验证器结果 r
4. 每个教师对同一 rollout 打分，计算 gap 和极性 ρk
5. 计算门控损失，更新策略参数
6. 在线更新技能库

## 8. 不足与局限

### 实验覆盖
- **消融主要在小模型**：Qwen3-1.7B 上的结论需在大模型上进一步验证
- **超参数缺乏系统搜索**：τg、cΔ、εa 等关键参数使用默认值，缺乏敏感性分析
- **数据集单一**：仅数学推理任务，未涵盖开放式生成、代码生成等其他领域

### 方法局限
- **技能库构建依赖 LLM 生成**：冷启动和在线更新中的技能提取依赖模型质量，可能引入偏差
- **检索策略简单**：仅使用语义相似度，配对策略为 rank 对齐，未探索自适应权重
- **教师数量固定**：top-8 技能 + top-8 错误，可能存在冗余或不足
- **极性决策二值化**：ρk ∈ {-1, 0, +1}，缺乏更细粒度的软加权

### 潜在偏差风险
- **Checkpoint 选择偏差**：报告最佳 checkpoint 可能导致过拟合评估基准
- **技能库污染风险**：需确保技能库不包含评估问题，否则存在信息泄漏
- **验证器依赖**：极性判断完全依赖验证器结果，验证器错误会级联传播

### 应用限制
- **任务范围**：目前仅在自动验证答案的数学推理任务上验证，开放式生成需可靠 judge
- **安全风险**：改进数学能力不等于在关键领域可靠，不能直接推断到高风险应用
- **冷启动成本**：需要初始采样和技能提取，增加训练前开销
