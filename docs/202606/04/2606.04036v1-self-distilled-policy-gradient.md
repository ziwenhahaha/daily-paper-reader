## 摘要
在策略自蒸馏，即语言模型在特权上下文的条件下监督自身生成，是一种有前景的稀疏奖励强化学习的稠密监督来源。实际上，它可以实例化为一个辅助的全词汇表学生到教师的反向Kullback-Leibler散度损失。因此我们提出SDPG，一个自蒸馏策略梯度框架，结合了组相对验证器优势与归一化标准差、精确的全词汇表在策略自蒸馏，以及参考策略KL正则化。在实验上，SDPG相对于RLVR和自蒸馏基线提高了稳定性和性能。代码可在 https://github.com/lauyikfung/SDPG 获取。

## Abstract
On-policy self-distillation, where a language model conditions on privileged context to supervise its own generations, is a promising source of dense supervision for sparse-reward reinforcement learning. Actually, it can be instantiated as an auxiliary full-vocabulary student-to-teacher reverse Kullback-Leibler divergence loss. We therefore propose SDPG, a self-distilled policy-gradient framework that combines group-relative verifier advantages with normalized standard deviation, exact full-vocabulary on-policy self-distillation, as well as reference-policy KL regularization. Empirically, SDPG improves stability and performance over RLVR and self-distillation baselines. The code is available at https://github.com/lauyikfung/SDPG.

# Self-Distilled Policy Gradient 论文总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：强化学习与可验证奖励（RLVR）在提升大语言模型（LLM）推理能力时面临两个关键挑战：
  1. **奖励稀疏性**：序列级奖励在长序列生成中对所有token一视同仁，导致信用分配困难
  2. **训练不稳定性**：训练初期大量负优势会导致模型基础能力退化

- **研究背景**：
  - GRPO等RLVR方法已成为后训练推理模型的标准范式
  - 近期提出的Dr.GRPO、DAPO、GSPO等方法解决了部分问题，但稀疏性仍未根治
  - On-policy distillation（OPD）和自蒸馏方法被提出以提供稠密的token级监督信号，但传统方法需要独立的大规模教师模型，内存开销大
  - 纯自蒸馏方法虽可复用同一模型作为教师，但可能强化错误轨迹中的局部合理token

- **研究目标**：将RLVR的探索与选择能力与自蒸馏的稠密监督优势相结合，在保持训练稳定性的同时提升推理性能

## 2. 论文提出的方法论：核心思想、关键技术细节、公式与算法流程

### 核心思想

SDPG将自蒸馏重新解释为一种**策略梯度形式**：通过特权上下文（privileged context）条件下的同一模型产生教师分布，与部署时的学生分布进行全词汇表反向KL散度匹配，从而提供稠密的token级监督信号。

### 关键技术细节

#### (1) 分布定义
- **学生分布**（无特权）：$p_t(a) = \pi_\theta(a | x, y_{<t})$
- **教师分布**（有特权）：$q_t(a) = \pi_\theta(a | c, x, y_{<t})$
- 特权上下文 $c$ 包含标准答案和解答路径

#### (2) 完整词汇表OPD损失
$$\ell_{OPD}^{i,t}(\theta) = D_{KL}(p_{i,t} \| SG[q_{i,t}]) = \sum_{a \in V} p_{i,t}(a) \log \frac{p_{i,t}(a)}{SG[q_{i,t}(a)]}$$

#### (3) 策略梯度等价性（Proposition 3.1）
反向KL全词汇表OPD的梯度等价于一种detach-sampling策略梯度更新，其token优势为中心化对数比率：
$$A_{dist}^t(a) = SG\left[\bar{D}_t - \log \frac{\bar{p}_t(a)}{\bar{q}_t(a)}\right]$$

#### (4) SDPG总目标函数
$$L_{SDPG}(\theta) = L_{out}(\theta) + \beta(k) L^+_{OPD}(\theta) + \alpha L_K(\pi_\theta, \pi_{ref})$$

其中：
- **$L_{out}$**：基于验证器的REINFORCE风格策略梯度损失，使用组相对优势
- **$L^+_{OPD}$**：正向优势门控的蒸馏损失
- **$L_K$**：参考策略的非归一化KL正则化（UFKL或URKL）

#### (5) 正向优势门控
$$m_i = \mathbb{1}[A_{out}^{(i)} > 0]$$
仅在验证器认可的轨迹上应用蒸馏信号，防止错误轨迹的局部合理token被强化。

#### (6) β调度器（warmup-decay）
$$\beta(k) = \beta_{base} \times \min\left(1, \frac{k}{T_{warm}}\right) \times \min\left(1, \frac{T - k}{T_{decay}}\right)$$

- **warmup阶段**（前50步）：防止早期蒸馏目标噪声干扰探索
- **decay阶段**（后50步）：释放学生模型，避免不可约的条件互信息偏差

### 算法流程

1. 对每个prompt从当前策略采样G个响应
2. 计算二值验证器奖励和组相对优势
3. 应用正向优势门控确定蒸馏信号范围
4. 在采样前缀上计算全词汇表OPD损失
5. 联合最小化输出损失、门控蒸馏损失和参考KL正则化
