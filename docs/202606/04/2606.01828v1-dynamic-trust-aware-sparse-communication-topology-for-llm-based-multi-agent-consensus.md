## 摘要
大型语言模型驱动的多智能体系统通过多轮讨论、角色专长和交叉验证来增强复杂推理任务的可靠性。然而，现有的多智能体辩论与协作框架通常采用全连接通信方式，导致消息数量、token费用和端到端延迟随智能体数量呈近似二次增长；尽管固定稀疏拓扑能降低开销，但它们无法根据不同的任务实例或中间推理状态调整通信关系，因而容易保留低价值交互或丢失关键的纠错信息。为解决这一问题，本文提出DySCo（动态稀疏共识），一种动态信任感知的稀疏共识机制。在每一轮推理中，DySCo基于智能体可靠性、答案分歧度和任务相关性评估通信边的价值，并在预算约束下选取少量高价值边进行消息交换；随后通过动态信任权重聚合不同智能体的答案，并在共识一旦稳定时提前终止讨论。该机制以按需通信替代全网广播，从而在降低通信开销的同时保留必要的交叉验证信息。我们进一步给出了通信复杂度和共识稳定性的分析，并在数学推理、逻辑推理和事实问答任务上评估了DySCo的性能。

## Abstract
Large language model-driven multi-agent systems enhance the reliability of complex reasoning tasks through multi-round deliberation, role specialization, and cross-validation. However, existing multi-agent debate and collaboration frameworks typically adopt fully connected communication, causing the number of messages, token costs, and end-to-end latency to grow approximately quadratically with the number of agents; although fixed sparse topologies reduce overhead, they cannot adapt communication relationships to different task instances or intermediate reasoning states, making them prone either to preserving low-value interactions or to losing critical error-correction information. To address this problem, this paper proposes DySCo (Dynamic Sparse Consensus), a dynamic trust-aware sparse consensus mechanism. In each round of reasoning, DySCo estimates the value of communication edges based on agent reliability, answer divergence, and task relevance, and selects a small number of high-value edges for message exchange under budget constraints; it then aggregates the answers of different agents through dynamic trust weights and terminates the discussion early once consensus stabilizes. This mechanism replaces universal broadcasting with on-demand communication, thereby reducing communication overhead while preserving essential cross-validation information. We further present analyses of communication complexity and consensus stability, and evaluate the performance of DySCo on mathematical reasoning, logical reasoning, and factual question-answering tasks.

## 2. 论文提出的方法论

**核心思想**：提出 **DySCo（Dynamic Sparse Consensus，动态稀疏共识）** 机制，将通信拓扑从固定结构转变为由任务状态驱动的自适应决策变量。

**四大核心模块**：

### (1) 个体初始化
- 每智能体独立推理：$(y_0^i, r_0^i, c_0^i) = \text{LLM}_i(P_{\text{solve}}(x))$
- 初始信任权重设为 $T_0^i \leftarrow 1/n$

### (2) 动态边选择（核心创新）
- **边评分公式**：基于多个信号评估每条候选通信边的价值：
$$s_{ij}^t = \alpha T_j^t + \beta c_j^t + \gamma D_{ij}^t + \delta H_{ij}(x) - \eta L_j^t$$
  - $T_j^t$：智能体$j$的历史信任权重
  - $c_j^t$：智能体$j$的当前置信度
  - $D_{ij}^t$：答案/推理分歧度（选择题为指示函数，开放式任务用句子向量距离）
  - $H_{ij}(x)$：任务依赖度（如代码审查中测试-编码智能体间的高依赖）
  - $L_j^t$：消息token成本（负向惩罚）
- **Top-K选择**：在预算约束下，每位接收者选择得分最高的$k$个邻居：$N_i^t = \text{TopK}_{j \neq i}(s_{ij}^t, k)$

### (3) 消息压缩与信任感知更新
- **压缩消息**：不广播完整推理轨迹，而是生成结构化压缩消息，包含当前答案、关键支持理由和针对接收者的潜在反例/修订建议。
- **信任感知状态更新**：接收者根据本地状态和加权邻居消息更新答案，明确区分"保持原答案"、"部分修订"或"完全改变答案"三种情况。
- **信任权重更新**：
$$T_i^{t+1} = \lambda T_i^t + (1-\lambda)(\rho V_i^t + (1-\rho)A_i^t)$$
  - $V_i^t$：外部验证器评分（可执行验证器或独立判断LLM）
  - $A_i^t$：与最终/临时多数共识的一致性
  - $\lambda$：历史平滑系数

### (4) 共识聚合与早停
- **信任加权投票**：最终共识答案通过加权多数投票确定。
- **早停机制**：当加权答案分布熵低于阈值$\epsilon$，或共识答案连续两轮不变且平均置信度增量小于阈值时，提前终止通信。

**通信复杂度分析**：
- 全连接：$O(Rn^2\bar{\ell})$
- DySCo：$O(Rnk\bar{\ell}_c)$，其中$k \ll n$且$\bar{\ell}_c \leq \bar{\ell}$

## 6. 论文的主要结论与发现

**核心发现**：

1. **效率与质量兼得**：DySCo相比Dense MAD实现约**70% token消耗降低**、近**50%延迟降低**，同时在GSM8K上准确率从82.0%提升至84.3%。

2. **线性可扩展性**：Dense MAD的token消耗随智能体数量呈**二次增长**（$n=12$时达58.4k），而DySCo保持**线性增长**（$O(Rnk)$），$n=12$时仅10.5k。

3. **快速共识收敛**：DySCo在第一轮通信后共识熵即降至0.62（vs Dense MAD的0.88），更快达成稳定共识并触发早停。

4. **各组件必要性**：消融实验表明：
   - 去除信任权重：准确率下降2.8%
   - 去除分歧评分：准确率下降2.2%
   - 去除消息压缩：token增加82.2%但准确率仅+0.2%
   - 去除早停机制：token增加25.4%无准确率收益

5. **超参稳健性**：DySCo对关键超参数（如$k, R, \epsilon$，边评分权重$\alpha, \beta, \gamma, \delta, \eta$）的敏感性较低，性能保持稳定。

## 7. 优点

1. **动态拓扑创新**：首次将通信拓扑从固定结构转变为由推理状态、信任、分歧度等信号驱动的自适应决策变量。

2. **理论支撑**：提供了通信复杂度$O(Rnk)$的形式化分析，以及基于时变图的弱共识收敛命题。

## 8. 不足与局限

1. **置信度校准问题**：LLM自报告置信度可能校准不佳，直接使用$c_j^t$可能放大过度自信错误，需要外部验证器或历史校准。

2. **马太效应风险**：信任权重可能导致"富者愈富"效应，使部分智能体随时间被边缘化，需要引入探索机制（如$\epsilon$-greedy或UCB）。

3. **局部共识风险**：过于稀疏的通信可能形成局部意见集群，无法达成全局共识，需要周期性引入随机跨簇边。

4. **适用范围有限**：对于开放式生成、价值判断或多解任务，边价值信号（高置信度、大分歧）不一定反映答案质量，不适合作为事实正确性的通用判定机制。

5. **少数派声音消失**：信任加权聚合可能放大早期偶然正确的智能体，导致后续通信集中在少数节点，降低少数推理路径的可见性。

6. **单一模型测试**：实验仅使用gpt-3.5-turbo，未验证在异构LLM（如GPT-4、Claude、Llama）或不同尺寸模型上的效果。

7. **可验证性依赖**：信任更新依赖可执行验证器（如数学答案检查、代码测试），对于难以验证的任务适用性受限。
